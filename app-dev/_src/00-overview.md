大模型时代，最稀缺的不是「会调 API」的人，而是**能把大模型能力封装成可落地产品**的工程师。本指南采用「自顶向下」路径：

- **先做应用**，再回溯底层 —— 避免被数学公式劝退
- **每一章都能跑通代码**，告别「学完即忘」
- **对齐行业最新栈**：Prompt → Function Call → RAG → MCP → Agent

:::callout 💡
**适合谁：** 有基础编程能力、想转入大模型应用层的开发者；已经在做 LLM 但希望系统化的工程师。
:::
