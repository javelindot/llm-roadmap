6 大模块按顺序推进，全程预计 **4–6 周**（每周 8–10 小时）。每个模块完成后都能产出可独立运行的 demo。

## 全栈推进路线

| 阶段 | 模块 | 核心能力 | 预计耗时 | 产出 |
|------|------|----------|----------|------|
| ①   | Prompt 工程   | 结构化提示词、CoT、Few-shot、评测 | 5–7 天 | 稳定输出的 prompt 模板库 |
| ②   | Function Call | JSON Schema、并行调用、错误处理 | 3–4 天 | 能让模型调外部 API 的 demo |
| ③   | RAG           | 切片、Embedding、混合检索、Reranker | 7–10 天 | 私域知识问答系统 |
| ④   | MCP           | 协议规范、Server、Client、生态 | 3–5 天 | 标准化的工具集成方案 |
| ⑤   | Skills        | Claude Skills 能力包 | 2–3 天 | 按需加载的领域能力 |
| ⑥   | AI Agent      | 规划、记忆、反思、多 Agent | 7–10 天 | 自主完成任务的智能体 |

## 学习节奏建议

```
Week 1   ① Prompt 工程         ←  入门必读，建立直觉
Week 2   ② Function Call + ③ RAG（上）  ←  解锁工具能力
Week 3   ③ RAG（下）            ←  最复杂的一站，慢一点
Week 4   ④ MCP + ⑤ Skills      ←  生态对接
Week 5–6 ⑥ AI Agent            ←  集大成
```

## 三种典型路径

不是所有人都需要从头到尾。按目标挑路径：

:::callout 💼
**做企业问答 / 客服系统**：① → ③ → ②（按需）。RAG 是核心，FC 是辅助。
:::

:::callout 🤖
**做自动化 Agent**：① → ② → ⑥ → ④。Agent 离不开扎实的 Prompt 和 FC 基础。
:::

:::callout 🔌
**做 SaaS / 插件生态**：① → ② → ④ → ⑤。重点在让你的产品被各家模型平台接入。
:::

## 学完之后

| 能力 | 验收标准 |
|------|----------|
| 独立设计 LLM 应用 | 看到一个需求能 5 分钟拆解出技术栈选型 |
| 写生产级代码 | 能加错误处理、监控、评测、降级 |
| 评估技术方案 | 知道每种方案的 trade-off 和上下限 |
| 跟进社区进展 | 看 Anthropic / OpenAI 新发布能立刻判断价值 |
