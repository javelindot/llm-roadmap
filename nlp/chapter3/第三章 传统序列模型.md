# 传统序列模型
## 3.1 RNN（循环神经网络）
### 3.1.1 概述
在自然语言中，词语的顺序对于理解句子的含义至关重要。虽然词向量能够表示词语的语义，但它本身并不包含词语之间的顺序信息。

为了解决这一问题，研究者提出**RNN（Recurrent Neural Network，循环神经网络）**。

RNN 会逐个读取句子中的词语，并在每一步结合当前词和前面的上下文信息，不断更新对句子的理解。通过这种机制，RNN 能够持续建模上下文，从而更准确地把握句子的整体语义。因此RNN曾是序列建模领域的主流模型，被广泛应用于各类NLP任务。

**说明：**

随着技术的发展，RNN已经逐渐被结构更灵活、计算效率更高的Transformer 模型所取代，后者已经成为当前自然语言处理的主流方法。

尽管如此，RNN 仍然具有重要的学习价值。它所体现的"循环建模上下文"的思想，不仅为 LSTM 和 GRU 等改进模型奠定了基础，也有助于我们更好地理解 Transformer 等更复杂的架构。

### 3.1.2 基础结构
RNN（循环神经网络）的核心结构是一个具有循环连接的隐藏层，它以时间步（time step）为单位，依次处理输入序列中的每个 token。

在每个时间步，RNN 接收当前 token 的向量和上一个时间步的隐藏状态（即隐藏层的输出），计算并生成新的隐藏状态，并将其传递到下一时间步。

### 3.1.3 多层结构
为了让模型捕捉更复杂的语言特征，可以将多个 RNN 层按层次堆叠起来，使不同层学习不同层次的语义信息。

这种设计的核心假设是：底层网络更容易捕捉局部模式（如词组、短语），而高层网络则能学习更抽象的语义信息（如句子主题或语境）。

多层RNN结构中，每一层的输出序列会作为下一层的输入序列，最底层RNN接收原始输入序列，顶层 RNN的输出作为最终结果用于后续任务。

### 3.1.4 双向结构
对于某些任务（如文本分类），当前词的预测不仅需要"上文"信息，往往还需要"下文"信息。例如，在句子"今天天气很好，我们去____"中，预测下一个词时需要综合考虑前文"今天天气很好，我们去"以及后文信息。

双向RNN通过使用两套独立的RNN网络来解决这个问题：

+ 正向RNN按时间顺序处理输入序列
+ 反向RNN按逆时间顺序处理输入序列

每个时间步同时得到两个隐藏状态，通常将它们进行拼接，形成最终的输出。

### 3.1.5 多层+双向结构
多层结构和双向结构可以组合使用，形成更复杂的网络结构。在这种结构中，每一层的每个时间步都会同时获得正反向的信息。

### 3.1.6 API使用
`torch.nn.RNN` 是 PyTorch 中实现循环神经网络的模块。它用于对序列数据建模，在自然语言处理（NLP）、时间序列预测等任务中广泛使用。

#### 参数说明
构造 RNN 层所需的参数如下：

```python
torch.nn.RNN(
    input_size, 
    hidden_size, 
    num_layers=1, 
    nonlinearity='tanh', 
    bias=True, 
    batch_first=False, 
    dropout=0.0, 
    bidirectional=False, 
    device=None, 
    dtype=None
)
```

| 参数名 | 类型 | 说明 |
| --- | --- | --- |
| input_size | int | 每个时间步输入特征的维度（词向量维度） |
| hidden_size | int | 隐藏状态的维度 |
| num_layers | int | RNN 层数，默认为 1 |
| nonlinearity | str | 激活函数，'tanh'（默认）或 'relu' |
| bias | bool | 是否使用偏置项，默认 True |
| batch_first | bool | 输入张量是否是 (batch, seq, feature)，默认 False 表示 (seq, batch, feature) |
| dropout | float | 除最后一层外，其余层之间的 dropout 概率 |
| bidirectional | bool | 是否为双向 RNN，默认 False |
| device | torch.device or str | 模块的初始化设备，如 'cuda', 'cpu' |
| dtype | torch.dtype | 模块初始化时的默认数据类型，如 torch.float32, torch.float64 |

```python
rnn = torch.nn.RNN(input_size, hidden_size, num_layers)
```

### 3.1.7 案例实操（智能输入法）
#### 需求说明
本案例的目标是基于 RNN 构建一个智能输入法模型：根据用户输入的历史字，预测下一个可能输入的字。

#### 需求分析
##### 数据集处理
本案例使用公开的文本语料库（如新闻、小说等），从中提取句子作为训练数据。训练时，将每个句子拆分为输入-输出对：输入是句子前 N 个字，输出是第 N+1 个字。

例如，句子"今天天气很好"可构造如下训练样本：

+ 输入："今天天气"，输出："很"
+ 输入："今天天气很"，输出："好"

##### 模型结构设计
模型整体由以下三个主要部分组成：

+ **嵌入层（Embedding）**：将输入的字索引映射为稠密向量表示
+ **RNN层**：用于建模输入序列的上下文信息，输出最后一个时间步的隐藏状态作为上下文表示
+ **输出层（Linear）**：将 RNN 的隐藏状态输出映射为词表大小的向量，经 softmax 后表示各字出现的概率

```python
import pandas as pd
from pathlib import Path

def process():
    """
    数据预处理主函数。
    """
    print("开始处理数据")
    
    # 1. 读取原始数据文件
    df = pd.read_csv(config.RAW_DATA_DIR / 'online_shopping_10_cats.csv', 
                     usecols=['review'], encoding='utf-8')
    
    # 2. 数据清洗：去除空值和空字符串
    df = df.dropna()
    df = df[df['review'].str.strip().ne('')]
    
    # 3. 构建字符级词表
    all_chars = set()
    for text in df['review']:
        for char in str(text):
            all_chars.add(char)
    
    # 添加特殊字符
    char_list = ['<pad>', '<unk>'] + sorted(list(all_chars))
    
    # 保存词表
    with open(config.PROCESSED_DATA_DIR / 'vocab.txt', 'w', encoding='utf-8') as f:
        for char in char_list:
            f.write(char + '\n')
    
    # 4. 生成训练样本
    train_data = []
    for text in df['review']:
        text = str(text)
        for i in range(len(text) - config.SEQ_LEN):
            input_seq = text[i:i+config.SEQ_LEN]
            output_char = text[i+config.SEQ_LEN]
            train_data.append((input_seq, output_char))
    
    # 保存训练数据
    with open(config.PROCESSED_DATA_DIR / 'train.txt', 'w', encoding='utf-8') as f:
        for input_seq, output_char in train_data:
            f.write(f"{input_seq}\t{output_char}\n")
    
    print(f"数据处理完成，共生成 {len(train_data)} 条训练样本")

if __name__ == '__main__':
```

**自定义数据集 dataset.py**

```python
import torch
from torch.utils.data import Dataset, DataLoader

class CharDataset(Dataset):
    """
    字符级数据集。
    """
    def __init__(self, file_path, vocab_file, seq_len):
        """
        初始化数据集。
        """
        # 加载词表
        with open(vocab_file, 'r', encoding='utf-8') as f:
            self.vocab_list = [line.strip() for line in f.readlines()]
        
        self.char2index = {char: idx for idx, char in enumerate(self.vocab_list)}
        self.index2char = {idx: char for idx, char in enumerate(self.vocab_list)}
        self.unk_index = self.char2index['<unk>']
        self.pad_index = self.char2index['<pad>']
        
        # 加载数据
        self.data = []
        with open(file_path, 'r', encoding='utf-8') as f:
            for line in f:
                parts = line.strip().split('\t')
                if len(parts) == 2:
                    self.data.append(parts)
        
        self.seq_len = seq_len
    
    def __len__(self):
        return len(self.data)
    
    def __getitem__(self, idx):
        input_seq, output_char = self.data[idx]
        
        # 将输入序列转换为索引
        input_indices = [
            self.char2index.get(char, self.unk_index) 
            for char in input_seq
        ]
        
        # 转换为张量
        input_tensor = torch.tensor(input_indices, dtype=torch.long)
        output_tensor = torch.tensor(
            self.char2index.get(output_char, self.unk_index), 
            dtype=torch.long
        )
        
        return input_tensor, output_tensor
```

**模型定义 model.py**

```python
import torch
import torch.nn as nn

class InputMethodModel(nn.Module):
    """
    基于RNN的输入法模型。
    """
    def __init__(self, vocab_size, embedding_dim, hidden_size, num_layers=1, dropout=0.1):
        super(InputMethodModel, self).__init__()
        
        # 嵌入层
        self.embedding = nn.Embedding(vocab_size, embedding_dim, padding_idx=0)
        
        # RNN层
        self.rnn = nn.RNN(
            embedding_dim, 
            hidden_size, 
            num_layers, 
            batch_first=True,
            dropout=dropout if num_layers > 1 else 0
        )
        
        # 输出层
        self.fc = nn.Linear(hidden_size, vocab_size)
    
    def forward(self, x):
        """
        前向传播。
        """
        # 嵌入层
        embedded = self.embedding(x)  # (batch, seq_len, embedding_dim)
        
        # RNN层
        output, hidden = self.rnn(embedded)  # output: (batch, seq_len, hidden_size)
        
        # 取最后一个时间步的输出
        output = output[:, -1, :]  # (batch, hidden_size)
        
        # 输出层
        logits = self.fc(output)  # (batch, vocab_size)
        
        return logits
```

**训练脚本 train.py**

```python
import torch
import torch.nn as nn
from torch.utils.data import DataLoader
import config
from dataset import CharDataset
from model import InputMethodModel

def train():
    """
    训练模型。
    """
    device = torch.device('cuda' if torch.cuda.is_available() else 'cpu')
    
    # 加载数据集
    train_dataset = CharDataset(
        config.PROCESSED_DATA_DIR / 'train.txt',
        config.PROCESSED_DATA_DIR / 'vocab.txt',
        config.SEQ_LEN
    )
    train_loader = DataLoader(
        train_dataset, 
        batch_size=config.BATCH_SIZE, 
        shuffle=True
    )
    
    # 初始化模型
    model = InputMethodModel(
        vocab_size=train_dataset.vocab_size,
        embedding_dim=config.EMBEDDING_DIM,
        hidden_size=config.HIDDEN_SIZE,
        num_layers=2
    ).to(device)
    
    # 损失函数和优化器
    criterion = nn.CrossEntropyLoss()
    optimizer = torch.optim.Adam(model.parameters(), lr=config.LEARNING_RATE)
    
    # 训练循环
    for epoch in range(config.EPOCHS):
        total_loss = 0
        for batch_idx, (inputs, targets) in enumerate(train_loader):
            inputs, targets = inputs.to(device), targets.to(device)
            
            # 前向传播
            outputs = model(inputs)
            loss = criterion(outputs, targets)
            
            # 反向传播
            optimizer.zero_grad()
            loss.backward()
            optimizer.step()
            
            total_loss += loss.item()
        
        print(f"Epoch {epoch+1}/{config.EPOCHS}, Loss: {total_loss/len(train_loader):.4f}")
    
    # 保存模型
    torch.save(model.state_dict(), config.MODELS_DIR / 'model.pt')
    print("模型训练完成！")

if __name__ == '__main__':
    train()
```

**评估脚本 evaluate.py**

```python
import torch
from torch.utils.data import DataLoader
import config
from dataset import CharDataset
from model import InputMethodModel

def evaluate(model, dataloader, device, topk=5):
    """
    评估模型。
    """
    model.eval()
    correct = 0
    topk_correct = 0
    total = 0
    
    with torch.no_grad():
        for inputs, targets in dataloader:
            inputs, targets = inputs.to(device), targets.to(device)
            
            outputs = model(inputs)
            
            # Top-1 准确率
            _, predicted = torch.max(outputs, 1)
            correct += (predicted == targets).sum().item()
            
            # Top-K 准确率
            _, topk_pred = torch.topk(outputs, topk, dim=1)
            topk_correct += (topk_pred == targets.view(-1, 1)).any(dim=1).sum().item()
            
            total += targets.size(0)
    
    return correct / total, topk_correct / total

    # 执行评估

    print(f"Top-5 准确率: {topk_acc:.4f}")
    print("========================")

if __name__ == '__main__':
    run_evaluate()
```

### 3.1.8 存在问题
#### 概述
尽管循环神经网络（RNN）在处理序列数据方面具有天然优势，但它在实际应用中面临一个非常严重的问题：**长期依赖建模困难**。这指的是：在训练过程中，当输入序列很长时，模型难以有效学习早期输入对最终输出的影响。

<!-- 这是一张图片，ocr 内容为： -->
![](images/image_1772902934572_736007876.png)

#### 问题分析
上述问题的根本原因在于训练过程中存在的**梯度消失**或**梯度爆炸问题**。

在训练RNN时，采用的是时间反向传播（Backpropagation Through Time, BPTT）方法，在反向传播过程中，梯度需要在每个时间步上不断链式传递，下图为RNN在训练过程中的计算图：

<!-- 这是一张图片，ocr 内容为： -->
![](images/image_1772902934573_494634382.png)

高清大图：

<!-- 这是一张图片，ocr 内容为： -->
![](images/image_1772902934574_88842729.x-emf)

根据上述计算图，可以得出：

$ \frac{\partial L_t}{\partial h_0} = \frac{\partial L_t}{\partial h_t} \prod_{k=1}^{t} \frac{\partial h_k}{\partial h_{k-1}} $

其中每一项 $ \frac{\partial h_k}{\partial h_{k-1}} $ 表示每条路径对 $ h_0 $ 的贡献。

展开早期时间步的某一条路径（例如 $ h_0 \to h_1 \to ... \to h_t $）可以得到：

$ \frac{\partial h_k}{\partial h_{k-1}} = W^T \cdot \text{diag}(1 - h_k^2) $

展开其中一环（为简单起见，按照标量推导）：

现有 $ h_k = \tanh(W \cdot h_{k-1} + U \cdot x_k) $，令 $ z_k = W \cdot h_{k-1} + U \cdot x_k $，则有：

$ \frac{\partial h_k}{\partial h_{k-1}} = \frac{\partial \tanh(z_k)}{\partial z_k} \cdot W = (1 - \tanh^2(z_k)) \cdot W $

令 $ \beta_W = |W| $，则有：

$ \frac{\partial h_k}{\partial h_{k-1}} \approx \beta_W $

可得：

$ \frac{\partial L_t}{\partial h_0} \approx \frac{\partial L_t}{\partial h_t} \cdot \beta_W^t $

所以，早期路径的展开可以写为：

$ \frac{\partial L_t}{\partial h_0} \approx \frac{\partial L_t}{\partial h_t} \prod_{k=1}^{t} \beta_W $

可以看到上述公式中有很多次的连乘，其中的 $ \beta_W $ 的范围是$ (0,1] $，如下图所示：

<!-- 这是一张图片，ocr 内容为： -->
![](images/image_1772902934574_105082923.png)

所以若 $ \beta_W $ 也小于1，那么经过的多次连乘，早期路径（例如 $ h_0 $ 的梯度）的值就会指数级衰减，并迅速接近于0，这个现象称为**梯度消失**。

由于早期时间步的梯度值几乎为0，所以总梯度几乎只会受到最近时间步的输入影响，换句话说，在权重参数更新（$ W \leftarrow W - \eta \cdot \frac{\partial L}{\partial W} $）时，早期输入的信息几乎不会对 $ W $ 的更新产生贡献。

这就导致模型只能学到短期依赖，而无法学到长期依赖。

另外，若 $ \beta_W $ 大于1（大到大于1），那么经过的多次连乘，早期路径（例如 $ h_0 $ 的梯度）的值就会指数级增长，这个现象称为**梯度爆炸**，梯度爆炸又会使得参数更新极不稳定。

这两个问题是制约RNN 学习长期依赖的主要瓶颈。

---

## 3.2 LSTM（长短期记忆网络）
### 3.2.1 概述
为了缓解RNN梯度消失或者梯度爆炸的问题，Hochreiter 和 Schmidhuber 于 1997 年提出了**长短期记忆网络（Long Short-Term Memory, LSTM）**。

### 3.2.2 基础结构
LSTM 通过引入特殊的记忆单元（Memory Cell，图中的 $ C_t $），有效提升了模型对长序列依赖关系的建模能力。其沿时间步展开后的结构如下图所示：

![](images/image_1772902934575_424110100.png)

其内部结构如下图所示，核心结构是三个"门"，分别是遗忘门、输入门和输出门。

<!-- 这是一张图片，ocr 内容为： -->
![](images/image_1772902934575_376356731.png)

各部分具体说明如下：

#### 记忆单元（Memory Cell）
记忆单元负责在序列中长期保存关键信息。它相当于一条"信息通道"，在多个时间步之间直接传递信息（记忆单元是缓解梯度消失和梯度爆炸问题的核心），如下图中的 ➡。

<!-- 这是一张图片，ocr 内容为： -->
![](images/image_1772902934575_702554553.png)

#### 遗忘门（Forget Gate）
遗忘门决定当前时间步要忘记多少过去的记忆。

例如在上一节的输入法智能提示案例中，假如历史输入为："小帅是一名程序员，他每天都加班；"，然后当前时间步的输入为"小美"，意味着当前的主语变为了"小美"，后续应该生成和"小美"相关的内容，所以此时记忆单元就应该忘记之前的主语信息"小帅"。

遗忘门会根据上一个时间步的隐藏状态和当前时间步的输入，生成一个0到1之间的控制系数，然后与上一个时间步的记忆单元状态相乘，从而动态调整哪些信息应该被遗忘。

<!-- 这是一张图片，ocr 内容为： -->
![](images/image_1772902934576_783019317.png)

遗忘门的计算公式为：

$ f_t = \sigma(W_f \cdot [h_{t-1}, x_t] + b_f) $

图中的 ⊙ 符号为阿达玛乘积（Hadamard product），表示逐元素相乘。

#### 输入门（Input Gate）
输入门控制要从当前时间步的输入向记忆单元存入多少新的信息。例如上述案例中，当前时间步的输入为"小美"，所以此时记忆单元就应该存入新的主语信息"小美"。

当前时间步的信息由当前输入和上一个隐藏状态计算而成，同时输入门也由当前输入和上一个隐藏状态计算而成，然后新的信息和输入门相乘得到需要存入记忆单元的信息，如下图所示：

<!-- 这是一张图片，ocr 内容为： -->
![](images/image_1772902934577_768733650.png)

当前时间步的信息计算公式为：

$ \tilde{C}_t = \tanh(W_c \cdot [h_{t-1}, x_t] + b_c) $

输入门的计算公式为：

$ i_t = \sigma(W_i \cdot [h_{t-1}, x_t] + b_i) $

综上所述可以得到记忆单元更新的完整公式为：

$ C_t = f_t \cdot C_{t-1} + i_t \cdot \tilde{C}_t $

#### 输出门（Output Gate）
输出门控制从记忆单元中读取多少信息作为当前时间步的隐藏状态进行输出。例如上述输入法智能提示案例中，记忆单元中存入新主语信息"小美"之后，当前时间步就应该从记忆单元中提取该主语信息，生成与"小美"相关的内容。

输出门同样由当前时间步的输入和上一个时间步的隐藏状态计算而成，如下图所示：

<!-- 这是一张图片，ocr 内容为： -->
![](images/image_1772902934579_279603766.png)

输出门的计算公式为：

$ o_t = \sigma(W_o \cdot [h_{t-1}, x_t] + b_o) $

当前时间步输出的隐藏状态计算公式为：

$ h_t = o_t \cdot \tanh(C_t) $

LSTM通过引入记忆单元（Memory Cell），在时间步之间提供了一条稳定的梯度传播路径。

记忆单元的更新公式为：

$ C_t = f_t \cdot C_{t-1} + i_t \cdot \tilde{C}_t $

所以（简单起见，按照标量推导）：

$ \frac{\partial C_t}{\partial C_{t-1}} = f_t $

在反向传播时，沿记忆单元路径，梯度传播实际上是多个 $ f_t $ 连乘的结果。虽然每个 $ f_t $ 的取值小于1，但通常较接近于1。这是因为 $ f_t $ 由遗忘门生成，在一般任务中，遗忘门倾向于"记得多、忘得少"，因此 $ f_t $ 的值通常较大。

由于乘积中的每一项 $ f_t $ 较接近1，整体衰减速度远小于传统RNN中隐藏状态链式传播时的指数衰减。这使得早期时间步的输入，能够通过记忆单元路径稳定地影响到最终的总梯度，从而有效参与参数的更新，保证了模型对长序列依赖的学习能力。

### 3.2.3 多层结构
与RNN类似，LSTM 也可以通过堆叠多个层来构建更深的网络，以增强模型对序列特征的建模能力。

在多层 LSTM 中，每一层 LSTM 的输出隐藏状态，会作为下一层 LSTM 的输入，同时每一层都维护独立的记忆单元。通过层层传递和提取信息，多层结构能够捕捉更复杂、更抽象的时序特征。

对于 LSTM，同样可以通过双向机制，利用序列中的过去信息和未来信息，进一步提升模型的建模能力。

在双向 LSTM 中，使用两套独立的 LSTM 网络：

+ 正向 LSTM 按时间顺序处理输入序列；
+ 反向 LSTM 按逆时间顺序处理输入序列。

### 3.2.6 API使用
`[torch.nn.LSTM](https://pytorch.org/docs/stable/generated/torch.nn.LSTM.html)`是 PyTorch 中实现长短期记忆网络（Long Short-Term Memory, LSTM）的模块。它用于对序列数据建模，在自然语言处理（NLP）、时间序列预测等任务中广泛使用。该模块支持单层或多层 LSTM，可选择是否使用双向结构（bidirectional）。

`torch.nn.LSTM`与`torch.nn.RNN`的API十分相似，主要区别在于相较于RNN，多了一个记忆单元需要处理。

**自定义分词器 tokenizer.py**

```python
import jieba
from tqdm import tqdm

jieba.setLogLevel(jieba.logging.WARNING)

class JiebaTokenizer:
    """
    基于 jieba 的分词器，用于分词、编码和词表管理。
    """
    unk_token = '<unk>'
    pad_token = '<pad>'
    
    @staticmethod
    def tokenize(sentence):
        """
        对句子进行分词。
        """
        return jieba.lcut(sentence)
    
    @classmethod
    def build_vocab(cls, sentences, vocab_file):
        """
        构建词表并保存到文件。
        """
        unique_words = set()
        for sentence in tqdm(sentences, desc='分词'):
            for word in cls.tokenize(sentence):
                unique_words.add(word)
        
        vocab_list = [cls.pad_token, cls.unk_token] + list(unique_words)
        
        with open(vocab_file, 'w', encoding='utf-8') as f:
            for word in vocab_list:
                f.write(word + '\n')
    
    @classmethod
    def from_vocab(cls, vocab_file):
        """
        从文件加载词表。
        """
        with open(vocab_file, 'r', encoding='utf-8') as f:
            vocab_list = [line.strip() for line in f.readlines()]
        return cls(vocab_list)
    
    def __init__(self, vocab_list):
        self.vocab_list = vocab_list
        self.vocab_size = len(vocab_list)
        self.word2index = {word: index for index, word in enumerate(vocab_list)}
        self.index2word = {index: word for index, word in enumerate(vocab_list)}
        self.unk_token_index = self.word2index[self.unk_token]
        self.pad_token_index = self.word2index[self.pad_token]
    
    def encode(self, sentence, seq_len):
        """
        将句子编码为索引列表。
        """
        tokens = self.tokenize(sentence)
        indexes = [self.word2index.get(token, self.unk_token_index) for token in tokens]
        
        if len(indexes) >= seq_len:
            return indexes[:seq_len]
        else:
            return indexes + [self.pad_token_index] * (seq_len - len(indexes))
```

**模型定义 model.py**

```python
import torch
import torch.nn as nn

class SentimentClassificationModel(nn.Module):
    """
    基于LSTM的情感分类模型。
    """
    def __init__(self, vocab_size, embedding_dim, hidden_size, num_layers=1, dropout=0.3):
        super(SentimentClassificationModel, self).__init__()
        
        self.embedding = nn.Embedding(vocab_size, embedding_dim, padding_idx=0)
        self.lstm = nn.LSTM(
            embedding_dim,
            hidden_size,
            num_layers,
            batch_first=True,
            dropout=dropout if num_layers > 1 else 0
        )
        self.fc = nn.Linear(hidden_size, 1)
    
    def forward(self, x):
        embedded = self.embedding(x)
        output, (hidden, cell) = self.lstm(embedded)
        hidden = hidden[-1, :, :]
        logits = self.fc(hidden)
        return logits.squeeze(1)
```

**训练脚本 train.py**

```python
import torch
import torch.nn as nn
from torch.utils.data import DataLoader
import config
from dataset import ReviewAnalyzeDataset
from model import SentimentClassificationModel
from tqdm import tqdm

def train():
    device = torch.device('cuda' if torch.cuda.is_available() else 'cpu')
    
    # 加载数据
    train_dataset = ReviewAnalyzeDataset(config.PROCESSED_DATA_DIR / 'indexed_train.jsonl')
    train_loader = DataLoader(train_dataset, batch_size=config.BATCH_SIZE, shuffle=True)
    
    # 加载词表
    with open(config.PROCESSED_DATA_DIR / 'vocab.txt', 'r', encoding='utf-8') as f:
        vocab_size = len(f.readlines())
    
    # 初始化模型
    model = SentimentClassificationModel(
        vocab_size=vocab_size,
        embedding_dim=config.EMBEDDING_DIM,
        hidden_size=config.HIDDEN_SIZE,
        num_layers=2
    ).to(device)
    
    # 损失函数和优化器
    criterion = nn.BCEWithLogitsLoss()
    optimizer = torch.optim.Adam(model.parameters(), lr=config.LEARNING_RATE)
    
    # 训练循环
    for epoch in range(config.EPOCHS):
        model.train()
        total_loss = 0
        for inputs, labels in tqdm(train_loader, desc=f'Epoch {epoch+1}'):
            inputs, labels = inputs.to(device), labels.to(device)
            
            outputs = model(inputs)
            loss = criterion(outputs, labels)
            
            optimizer.zero_grad()
            loss.backward()
            optimizer.step()
            
            total_loss += loss.item()
        
        print(f'Epoch {epoch+1}/{config.EPOCHS}, Loss: {total_loss/len(train_loader):.4f}')
    
    torch.save(model.state_dict(), config.MODELS_DIR / 'model.pt')
    print('模型训练完成！')

if __name__ == '__main__':
    train()
```

## 3.3 GRU（门控循环单元）
### 3.3.1 概述
**GRU（Gated Recurrent Unit，门控循环单元）** 是由 Cho 等人于 2014 年提出的另一种改进的循环神经网络结构。GRU 可以看作是 LSTM 的一种简化变体，它在保持 LSTM 效果的同时，结构更加简洁，计算效率更高。

### 3.3.2 基础结构
GRU 的核心是通过两个"门"来控制信息的流动：

+ **重置门（Reset Gate）**：决定有多少过去的隐藏状态需要被忽略
+ **更新门（Update Gate）**：决定有多少过去的隐藏状态需要传递到当前时刻

<!-- 这是一张图片，ocr 内容为： -->
![](images/image_1772902934628_951672281.png)

各部分具体说明如下：

#### 重置门（Reset Gate）
重置门决定了当前时间步的候选隐藏状态 $ h_t' $ 有多少依赖于之前的隐藏状态。当重置门值接近0时，模型会"忘记"之前的大部分信息，这有助于捕获更短期的依赖关系。

重置门的计算公式为：

$ r_t = \sigma(W_r \cdot [h_{t-1}, x_t] + b_r) $

#### 更新门（Update Gate）
更新门是 GRU 中最重要的组件，它决定了有多少过去的隐藏状态需要保留到当前时刻。更新门值越大，表示保留的历史信息越多。

更新门的计算公式为：

$ z_t = \sigma(W_z \cdot [h_{t-1}, x_t] + b_z) $

#### 候选隐藏状态
候选隐藏状态的计算结合了当前输入和经过重置门调整的上一时刻隐藏状态：

$ h_t' = \tanh(W_h \cdot [r_t \odot h_{t-1}, x_t] + b_h) $

#### 隐藏状态更新
最终隐藏状态通过结合上一时刻的隐藏状态和当前时刻的候选隐藏状态得到：

$ h_t = (1 - z_t) \odot h_{t-1} + z_t \odot h_t' $

### 3.3.3 多层结构
与 LSTM 类似，GRU 也可以堆叠多层以增强模型的表达能力。在多层 GRU 中，每一层的输出作为下一层的输入。

### 3.3.6 API使用
`[torch.nn.GRU](https://pytorch.org/docs/stable/generated/torch.nn.GRU.html)`是 PyTorch 中实现 GRU 的模块，用法与 LSTM 基本相同。

#### 参数说明
```python
torch.nn.GRU(
    input_size, 
    hidden_size, 
    num_layers=1, 
    bias=True, 
    batch_first=False, 
    dropout=0.0, 
    bidirectional=False, 
    proj_size=0, 
    device=None, 
    dtype=None
)
```

| 参数名 | 类型 | 说明 |
| --- | --- | --- |
| input_size | int | 每个时间步输入特征的维度 |
| hidden_size | int | 隐藏状态的维度 |
| num_layers | int | GRU 层数，默认为 1 |
| bias | bool | 是否使用偏置项，默认 True |
| batch_first | bool | 输入张量是否是 (batch, seq, feature) |
| dropout | float | 除最后一层外，其余层之间的 dropout 概率 |
| bidirectional | bool | 是否为双向 GRU，默认 False |
| proj_size | int | 隐藏状态的投影输出维度 |
| device | torch.device or str | 模块的初始化设备 |
| dtype | torch.dtype | 模块初始化时的默认数据类型 |


#### 输入输出
```python
gru = torch.nn.GRU(input_size, hidden_size, num_layers)
output, h_n = gru(input, h_0)
```

**输入：**

| 输入 | 说明 |
| --- | --- |
| input | 输入序列，形状为 (seq_len, batch_size, input_size) |
| h_0 | 可选，初始隐藏状态 |


**输出：**

| 输出 | 说明 |
| --- | --- |
| output | GRU层的输出 |
| h_n | 最后一个时间步的隐藏状态 |

#### 完整代码
**模型定义 model.py（GRU版本）**

```python
import torch
import torch.nn as nn

class SentimentClassificationModel(nn.Module):
    """
    基于GRU的情感分类模型。
    """
    def __init__(self, vocab_size, embedding_dim, hidden_size, num_layers=1, dropout=0.3):
        super(SentimentClassificationModel, self).__init__()
        
        self.embedding = nn.Embedding(vocab_size, embedding_dim, padding_idx=0)
        self.gru = nn.GRU(
            embedding_dim,
            hidden_size,
            num_layers,
            batch_first=True,
            dropout=dropout if num_layers > 1 else 0
        )
        self.fc = nn.Linear(hidden_size, 1)
    
    def forward(self, x):
        embedded = self.embedding(x)
        output, hidden = self.gru(embedded)
        hidden = hidden[-1, :, :]
        logits = self.fc(hidden)
        return logits.squeeze(1)
```

训练脚本和评估脚本与 LSTM 版本基本相同，只需修改模型导入即可。
