定位 & 节奏 / PART 1 / LLM 岗位全景

LLM 岗位全景

2025–2026，「大模型」三个字下面其实是 3 个截然不同的方向。先搞清楚自己要打哪一个，再决定怎么打。

三大主流方向

方向核心技能面试重点代表公司 / 岗位
① 算法 / 训练 预训练、SFT、RLHF、对齐、Scaling Law 八股深度 + 论文 + 调参实战 OpenAI/Anthropic 中国分部、月之暗面、智谱、阶跃
② 应用 / Agent Prompt、RAG、Agent、工具集成、Eval 项目深挖 + 系统设计 大厂业务线、独角兽、垂直 SaaS
③ 基建 / 推理 vLLM、TensorRT、量化、分布式、CUDA 性能优化 + 工程实战 云厂商、芯片公司、大厂基础架构

每个方向更细的细分

① 算法 / 训练方向细分

  • 预训练 (Pretrain): 数据清洗、模型架构、训练 infra。岗位极少, 顶级公司。
  • 对齐 (Alignment): SFT / DPO / RLHF / 奖励模型。岗位中等, 各家都在做。
  • 评测 (Eval): 评测集设计、自动评测、Red Team。岗位较多, 容易切入。
  • 领域微调: 行业垂直 LoRA / 数据合成。岗位多, 但单价相对低。

② 应用 / Agent 方向细分

  • RAG 工程师: 企业知识库、智能问答。岗位最多 + 应届友好。
  • Agent / 工作流: 多智能体、自动化编排。岗位中等, 新方向, 简历快速升值。
  • LLM 应用开发: 业务系统 + LLM 集成。最容易转过来, 但工资天花板较低。
  • Prompt 工程: 复杂 prompt 设计、效果调优。岗位少, 多作为兼职职能。

③ 基建 / 推理方向细分

  • 推理引擎开发: vLLM / SGLang / 自研引擎。门槛极高, 应届基本进不去。
  • 推理优化: 量化、KV cache、PagedAttention 等。门槛较高, 需要 CUDA。
  • 分布式训练: 多卡 / 多机训练、并行策略。门槛极高 + 岗位少。
  • 模型部署: K8s + Docker + 监控。最容易切入, 偏运维但好上手。

2025-2026 行情

  • 最热门: 应用 / Agent。各行各业都在做 LLM 应用, 岗位最多。
  • 最高薪: 算法 / 训练。顶级公司应届 SP 价能到 70-100w。
  • 最稀缺: 推理优化。会写 CUDA + 懂 vLLM 的人极少, 工资虚高。
  • 最稳定: 大厂业务 LLM 集成。岗位多, 不卷, 但成长慢。
🎯

应届生建议: 没有强算法背景 (没顶会论文 / 数学不强) 就别冲算法方向, 大概率被刷。从 RAG / Agent / 部署切入最稳, 1-2 年后再决定向算法还是基建深挖。

定位 & 节奏 / PART 1 / 选准方向 3 步

选准方向 · 3 步法

「我到底该投算法岗、应用岗还是基建岗?」—— 不用纠结, 3 步走完, 答案就出来了。

第 1 步 · 评估自己当前能力

能力算法应用基建
数学 / 论文阅读★★★★★ 必须★★ 一般★★ 一般
大型分布式系统★★★ 重要★★ 一般★★★★★ 必须
CUDA / 性能优化★★★ 重要★ 不需★★★★★ 必须
业务理解 / 产品思维★ 不需★★★★★ 必须★ 不需
Python / API 开发★★★ 重要★★★★★ 必须★★★ 重要
Java / Go 后端★ 不需★★★★ 重要★★ 一般

对照打分, 在哪一列「必须」项最齐, 就有「打这个方向」的资格。

第 2 步 · 评估自己的兴趣 / 偏好

  • 喜欢看论文 + 调参实验 → 算法方向。
  • 喜欢做产品 + 解决真实问题 → 应用方向。
  • 喜欢性能优化 + 系统底层 → 基建方向。
  • 三个都喜欢 → 选应用 (转其他方向最容易)。
  • 三个都不喜欢 → 重新想想是不是真的要做 LLM。

第 3 步 · 评估岗位匹配度

找10 个你感兴趣的 JD, 对照每条要求打分:

# JD 评估模板
公司: ___ | 岗位: ___ | 级别: ___

要求 1: ___ → 我的匹配度: 0-5 分
要求 2: ___ → 0-5 分
要求 3: ___ → 0-5 分
...
─────────────────
总分: __/__

# 决策
- 60% 以上 → 直接投, 大概率有面试
- 40-60% → 投 + 准备 1-2 个项目补短板
- 40% 以下 → 暂时跳过, 选更匹配的

常见纠结及破解

「算法岗工资高, 但我没论文」

3 条路:

  1. 不去顶级: 二三线算法岗也能接受 1-2 篇水会 + 项目证明。
  2. 转应用: 应用岗虽然工资比顶级算法低 30%, 但比头部应用岗高 0%。
  3. 先去做应用, 1-2 年后转算法: 工作中发论文 + 攒经验, 再跳。

「应用岗听起来很 low, 想做算法」

2 条提示:

  1. 应用岗不 low: ChatGPT、Cursor、Claude 这些产品的应用工程师工资和算法持平甚至更高。
  2. 「真应用工程师」: 能从产品端反推模型需求 + 设计 RAG/Agent 架构, 比纯调参的算法岗还稀缺。

「我想做基建但没 CUDA 经验」

3 条路:

  1. 先做模型部署: K8s / Docker / 监控相对好上手, 1-2 年后可以转性能优化。
  2. 自学 vLLM 源码: 读 PagedAttention 实现 + 改一个小 bug 提 PR, 简历神器。
  3. 大学算法竞赛 / CUDA 课程: 应届生这个是硬门票, 已工作的不太来得及。
✅

最终建议: 不要追风口, 选你能在 3 年内做到「头部 10%」的方向。在自己最擅长的领域做到顶级, 比在风口方向做到平均收入更高、更稳定。

定位 & 节奏 / PART 1 / 应届 vs 社招

应届 vs 社招路径差异

应届校招和社招是两套截然不同的游戏规则。选错赛道再努力都白搭。

核心差异对照

维度应届校招社招
时间窗口9-11 月秋招主战 / 3-4 月春招补全年皆可, 但 3-5 月最活跃
HC 池大批量, HC 多但竞争激烈按需开放, HC 少但匹配度高
简历重点教育 + 实习 + 项目工作经历 + 项目 + 量化产出
笔试比重★★★★★ 算法笔试硬门票★★ 有些公司也有, 但易跳过
面试轮次HR + 群面 + 2-3 技术面1-2 技术面 + 老板面 + HR
谈薪空间较小 (HC 价基本固定)较大 (按个人议价)
关注点潜力、学习能力、可塑性已有产出、能否立刻干活
入职时间毕业当年 7 月统一入职2-4 周内入职

应届校招独有套路

① 必须刷算法题

大厂校招 80%+ 都有算法笔试 (LeetCode 难度), 不刷题直接出局。

  • 必刷题量: Hot 100 + 大厂高频 50 题 = 150 题左右。
  • 时间投入: 暑假 2-3 个月专项刷, 每天 3-5 题。
  • 语言选择: Python 最快, C++ 在硬核公司加分。

② 必拿核心实习

没有实习的应届生在大厂校招中竞争力大幅下降:

  • 大厂实习留用: 转正率 50-70%, 是性价比最高的校招路径。
  • 实习时机: 大三暑假 / 研一暑假 / 研二寒暑假, 至少 1 段 3+ 月实习。
  • 没拿大厂实习: 用顶会论文 / 大型开源 / Kaggle 名次替代。

③ 注意秋招节奏

7-8 月 暑期实习 + 暑假留用通知
8-9 月 提前批 (大厂 SSP 通常在这阶段)
9-10 月 正式秋招 + 主流 HC
11-12 月 秋招补录 + 春招准备
2-4 月 春招 + 校招最后机会

社招独有套路

① 简历要可量化

「3 年经验」不重要, 「线上 CTR +0.8%」才重要。社招简历每条必须有数字。

② 重视内推

社招 60%+ 通过内推, 内推回复率是官网 5-10 倍。没内推就先建立人脉:

  • 知乎 / 即刻 / V2EX 上的从业者私信。
  • 过去同事 / 学长学姐。
  • 技术社区 / 大会上认识的人。
  • LinkedIn / 脉脉 - 业内招聘官。

③ 关注「黄金跳槽点」

  • 3 月 - 4 月: 年终发完, 大批人开始动。
  • 9 月 - 10 月: 校招 + 社招同时开启, 公司预算多。
  • 避开 12-1 月: 年终未发, 很多人犹豫不跳。

两条路径的转换

实习留用 → 直接成为社招中的「校招生」, 不走校招渠道。第 1 年薪资介于校招和社招之间, 通常更接近社招。

⚠️

应届生最大坑: 不要把应届身份浪费。应届 offer 失败可以再读 1 年 (gap year), 但应届身份用过了就没了, 后面所有招聘都按社招走 (要求工作年限和工资证明)。

定位 & 节奏 / PART 2 / 30 天加速版

30 天备战计划 · 加速版

手上有 30 天集中准备 (在职 / 待业)？这个节奏适合3-5 年经验 + 想跳大厂的社招同学。应届生看下面的 3 个月版本。

4 周节奏概览

Week 1: 盘点简历 + 项目复盘
Week 2: LLM 八股 + 系统设计
Week 3: 算法刷题 提高篇
Week 4: 模拟面试 + 投递 + 复盘

Week 1 · 简历 + 项目复盘

每日任务

  • Day 1-2: 简历重构 (STAR 化, 见 简历优化)。
  • Day 3-4: 项目 1 深度复盘, 写出 STAR + 3 层追问预案。
  • Day 5-6: 项目 2 + 项目 3 同样深度复盘。
  • Day 7: 自我介绍 1 分钟版 + 3 分钟版录音, 自己听 3 遍。

产出

  • ✅ 1 份新简历 (STAR + 量化)。
  • ✅ 3 个项目的「2 分钟讲法 + 10 分钟深挖 + 反问 list」。
  • ✅ 2 个版本的自我介绍音频备份。

Week 2 · LLM 八股 + 系统设计

每日任务

  • Day 8-9: Transformer 细节 (Self-Attention, 位置编码, MHA), 见 tech 文档。
  • Day 10-11: 预训练 + 微调 (Pretrain, SFT, LoRA, QLoRA)。
  • Day 12: 对齐 (RLHF, DPO, GRPO)。
  • Day 13: 系统设计 · RAG (按 5 步法走一遍完整答案)。
  • Day 14: 系统设计 · Agent 平台。

产出

  • ✅ 八股 7 个核心主题各能讲 5 分钟。
  • ✅ 2 个系统设计题能在 25 分钟内画完架构 + 讲清关键决策。

Week 3 · 算法刷题

每日任务

  • Day 15-18: 必刷 7 大题型 (每天 1 题型 4-5 题): 双指针、滑窗、BFS/DFS、动规、堆、回溯、图。
  • Day 19-20: LeetCode Hot 100 高频 30 题。
  • Day 21: 复盘做错的题, 整理「错题集」反复看。

产出

  • ✅ Hot 100 中至少 70 题能在 25 分钟内 ac。
  • ✅ 错题集 ~20 题, 自己能讲出最优解和复杂度。

Week 4 · 模拟面试 + 投递

每日任务

  • Day 22-23: 找朋友模拟 2 场技术面 (1 算法 + 1 项目 + 系统设计)。
  • Day 24-25: 用 Claude/GPT 模拟 2 场问答, 把所有八股问一遍。
  • Day 26-27: 投递 8-10 家公司 (大厂 4-5 + 独角兽 2-3 + 备胎 2)。
  • Day 28: 跟进面试时间, 准备每家公司的针对性问题。
  • Day 29-30: 完成第一轮面试 1-2 家, 复盘改进。

产出

  • ✅ 至少 2 场完整模拟面试经验。
  • ✅ 投递 ≥ 8 家, 进入面试流程 ≥ 3 家。
  • ✅ 1 份「面试问题汇总 + 自己的答案」笔记。

每周日小计 (固定动作)

  1. 列下周「Top 3 重点」, 不超过 3 件事。
  2. 整理本周错题 / 答得不好的题, 第 2 天最先重过。
  3. 记录本周遇到的真实面试题 → 加到下周复习清单。
  4. 给自己写 1 段反馈: 哪里做得好 / 哪里有进步空间。
⚠️

大忌: 不要把 30 天全用来「学新东西」。简历 + 项目复盘是 ROI 最高的, 必须 Week 1 优先做。技术深挖永远学不完, 但简历做好可以让你直接拿面试。

定位 & 节奏 / PART 2 / 3 个月稳健版

3 个月备战计划 · 稳健版

手上有 3 个月时间 (应届校招提前 / 在职从容跳)？这个节奏适合需要补强短板的同学, 每天投入 2-4 小时即可跑完。

3 个月节奏概览

Month 1: 基础 + 简历 (打牢底子)
Month 2: 八股 + 系统设计 + 项目 (能力建设)
Month 3: 算法 + 投递 + 面试 (实战收割)

Month 1 · 基础 + 简历

Week 1-2 · 简历 + 自我介绍

  • 简历重构 (按 简历优化所有 12 篇看一遍, 做笔记)。
  • 列出 5-8 段实习/项目经历, 全部 STAR 化。
  • 1 分钟 + 3 分钟自我介绍, 录音改 3 遍。

Week 3-4 · 选方向 + 调研行情

  • 对照「选准方向 3 步法」, 确定主投方向 (算法/应用/基建)。
  • 找 20+ 个目标 JD, 提取常见技术栈和高频考点。
  • 用「岗位关键字」反向更新简历。

Month 2 · 八股 + 系统设计 + 项目

Week 5-6 · LLM 八股

  • Transformer 系列 (Self-Attention, 位置编码, MHA): 每个能讲 5 分钟。
  • 预训练 + 微调 (Pretrain, SFT, LoRA/QLoRA, DPO): 各能讲 5 分钟。
  • 读 1 篇高频论文 (例如 LoRA / DPO 原文)。

Week 7 · 系统设计

  • 5 步答题法练习: 澄清需求 → 架构 → 关键决策 → 工程 → 扩展。
  • RAG 系统 + Agent 平台 2 个经典题各画 3 遍架构图。
  • 录音演讲, 每个题控制在 25 分钟内。

Week 8 · 项目深挖

  • 3 个最强项目, 写出「2 分钟版 + 10 分钟版 + 3 层反问」预案。
  • 跟同学/前辈对练, 互相挑刺。
  • 把答得不好的点回到代码 / 文档里查清楚。

Month 3 · 算法 + 投递 + 面试

Week 9-10 · 算法刷题

  • Hot 100 第一遍刷完 (每天 5 题, 50 题 / 周)。
  • 大厂高频 30 题第二遍 (重点公司针对性刷)。
  • 整理「错题集」+ 模板代码 (BFS / 二分 / 滑窗等)。

Week 11 · 投递 + 第一轮面

  • 批量投递 (大厂 6-8 + 独角兽 3-5 + 备胎 2-3)。
  • 第 1 周做 2-3 家技术面, 复盘每场后立刻改进。
  • 每场面后 30 分钟内写「面试复盘」: 哪里好/哪里差/改进点。

Week 12 · 收割 + 谈薪

  • 聚焦 2-3 家最匹配的公司, push 加速。
  • 开始有口头 offer, 按「谈薪四原则」拉锯。
  • 对照 TC 计算模板, 横向比较多 offer。
  • 签字, 走离职流程, 入职准备。

每周固定动作

日任务
周一列本周 Top 3 重点 (≤3 件), 拒绝其他诱惑
周二-周五按周计划推进, 每日 2-4 小时
周六1 场模拟面 (找朋友 / 用 Claude / 录音自演)
周日复盘: 完成情况 + 调下周计划 + 错题重过
💡

3 个月版的好处: 每周只用 10-15 小时, 在职完全可控。劣势: 时间长容易拖延。关键是「周日复盘」不能省, 强制让自己跟上节奏。

定位 & 节奏 / PART 2 / 在职 vs 全职准备

在职 vs 全职准备策略

「我应该裸辞备战, 还是边工作边面?」—— 取决于你的财务安全垫和时间利用效率。

两种模式对比

维度在职准备裸辞全职
时间投入每天 2-4 小时每天 6-10 小时
备战周期3-6 个月1-2 个月
压力低 (有收入兜底)高 (时间 + 金钱双压力)
谈薪筹码★★★★★ 上家薪资作锚★★ 已无收入, 被动
面试灵活度★★ 请假难、面试时间紧★★★★★ 随时面
身体 / 心态累, 但收入稳不上班反而容易焦虑
简历空窗期无有 (建议 ≤2 个月)

在职准备建议

① 时间分配

工作日早晨: 30 分钟刷题 (起床 / 早餐时)
工作日晚上: 1.5-2 小时学习 (按周计划推进)
周六上午: 1 场模拟面试 (4-6 小时, 含复盘)
周六下午: 休息 / 玩
周日上午: 2-3 小时实战面试 (公司面试或线上)
周日下午: 复盘 + 列下周计划

每周 ~15 小时, 3 个月 ~180 小时, 足够准备一份 LLM 工程师面试。

② 上家工作策略

  • 不松懈: 工作上别明显怠惰, 等 offer 签字后再松。
  • 不抱怨: 不在工作中表现不满, HR 圈子小, 容易传到下家。
  • 留好关系: 走时给老板和 1-2 个同事留长期联系, 未来背调 / 行业八卦都用得上。

③ 面试时间安排

  • 优先早晨 / 晚上: 8:00 / 19:00 起的电话面, 不影响工作。
  • 需请假时: 一次性请 1-2 天集中面 3-4 家, 不要分散。
  • HR 排面试时: 主动给 2-3 个时段供 HR 选, 而不是问「您什么时候方便」 (主动权在自己)。

裸辞全职准备建议

① 财务安全垫

至少存 6 个月生活费 (含房租 / 吃 / 备用), 否则 2-3 个月没拿到 offer 就会焦虑乱投。

② 时间投入

9:00-12:00 上午 (高效时段)
   学习新内容 (八股 / 系统设计)
14:00-17:00 下午
   刷题 / 项目复盘 / 简历改
19:00-21:00 晚上
   面试 / 模拟面 / 复盘
21:00 后 放松, 必须休息

③ 心态管理

  • 每天保持「外出」: 在家容易抑郁, 哪怕去咖啡馆学习也好。
  • 每周面试 ≥ 2 场: 否则会陷入「准备-准备-准备」死循环。
  • 设置 deadline: 「2 个月内必须有 offer」, 避免无限延期。
  • 有规律运动: 每周 3 次, 不然身体和精神都会下滑。

关键决策点：什么时候裸辞？

下面 3 个条件同时满足再裸辞:

  1. 财务: 存款 ≥ 6 个月生活费。
  2. 市场: 行业不在大裁员周期, 你目标方向 HC 充足。
  3. 动机: 上家工作严重影响备战 (例如 996 完全没时间), 而不是仅仅「不想上班」。

不建议裸辞的情况

  • 存款 ≤ 3 个月。
  • 行业寒冬 (2022-2023 互联网寒冬时, 裸辞失业 6+ 个月的人很多)。
  • 上家工作还可以, 只是「想换换」 → 边工作边面更稳。
  • 家庭有较大经济压力 (房贷 / 养孩子)。
⚠️

裸辞最大坑: 简历空窗期超过 3 个月, 后续公司会反复追问「这段时间在做什么」。准备好可信的答案 (项目 / 学习 / 健康原因 / 家庭原因), 别让空窗成为减分项。
