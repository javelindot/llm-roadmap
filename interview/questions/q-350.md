# （嵌套问题变体）用户一次输入多个不相干任务，是否要创建新的对话栈？

## 答案

答：可以创建新的，但本项目没有。一次图流程，创建一个State，读取持久化的Tracker信
息，将解析出来的多个命令逐个压栈，逐个处理。添加栈深变量，监控栈深不要过长。

### 补充说明

用户一次输入多个不相干任务时，有两种做法：

- 创建多个独立对话栈，适合任务复杂、互不相关、执行时间长的场景。
- 在同一个 State 里解析多个命令，按顺序压栈处理，适合轻量任务。

选择取决于任务是否共享上下文、是否会互相影响、是否需要并行执行。无论哪种方式，都要控制栈深和最大动作数，防止模型陷入无限流程。

### 结合跨境支付项目怎么说（面试口径）

> 在跨境支付客服里，如果用户一次说“查一下这笔付款为什么失败，顺便问下退款多久到账”，这其实是两个任务。付款失败需要查订单和通道，退款时效可能只需要查规则。我的做法是先解析成两个结构化命令，再按优先级处理。
>
> 如果两个任务都很轻，可以在同一个会话 State 里依次压栈；如果其中一个涉及复杂风控或人工审核，就应该拆成独立工单或独立流程，避免上下文互相污染。
>
> 我会给对话栈设置最大深度和超时，避免用户不断插入新问题导致流程失控。

**钩子**：可以引到“多命令解析、同栈顺序处理 vs 独立流程、栈深控制”。
