# 多任务学习中如何平衡不同任务的损失函数？

## 答案

多任务学习通常优化加权总损失：

```text
L_total = sum_t lambda_t * L_t
```

难点是不同任务的数据量、loss 数值、梯度大小、收敛速度和业务重要性不同。直接全部设为 1，可能让某个大任务主导共享参数。

### 1. 固定权重

根据业务重要性和实验手工设置：

```text
L = 1.0 * L_click
  + 2.0 * L_conversion
  + 3.0 * L_risk
```

优点是简单、可解释；缺点是调参成本高，训练不同阶段的最佳权重可能变化。

在加权前应尽量统一 loss 统计口径，例如按有效样本数取均值，避免样本数不同直接改变尺度。

### 2. 基于不确定性的权重

将任务噪声参数作为可学习变量：

```text
L =
sum_t [L_t / (2*sigma_t^2) + log(sigma_t)]
```

噪声大的任务权重会相对降低。它减少人工调参，但学习到的权重不一定等同于业务价值。

### 3. GradNorm

根据各任务在共享层的梯度范数和训练速度动态调整权重：

- 梯度过大的任务降权；
- 学得过慢的任务适当升权；
- 让各任务训练速度更平衡。

### 4. 动态权重平均

Dynamic Weight Average 根据近期 loss 下降速度调整权重。下降慢的任务得到更高关注，方法简单，但对噪声和 loss 抖动敏感。

### 5. 处理梯度冲突

只调 loss 权重不一定够。若两个任务梯度方向冲突，可以使用：

- PCGrad；
- CAGrad；
- MGDA/Pareto optimization；
- task-specific layer/expert；
- MMoE、PLE；
- 将冲突严重的任务拆分。

### 6. 采样层平衡

- 调整不同任务 batch 比例；
- 对小任务过采样；
- 对大任务降采样；
- 保证每个 batch 有足够任务覆盖；
- 对延迟标签使用成熟样本窗口。

样本比例和 loss 权重共同决定真实梯度贡献。

### 7. 跨境支付场景

假设同时训练：

```text
支付成功率
拒付风险
到账时效
通道成本
```

不能只按 loss 数值自动平衡。风险和合规目标可能是硬护栏，应设置最低召回或门禁；成本和成功率则可以在可行域内做多目标权衡。

### 8. 如何验收

- 每个任务与单任务基线对比；
- 观察 loss、梯度范数和梯度 cosine；
- 检查关键任务是否负迁移；
- 做权重消融；
- 评估 Pareto front；
- 线上设置主指标和护栏指标。

### 9. 面试总结

多任务 loss 平衡可以从固定权重、可学习不确定性、GradNorm、动态权重和梯度冲突处理几层展开。最终权重不仅由数学尺度决定，还要反映业务优先级；安全合规任务不能被总损失自动降到无效。
