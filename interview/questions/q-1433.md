# 继续预训练怎么做的？

## 答案

继续预训练通常指在已有基座模型上，使用新的通用语料、领域语料或新语言继续做自监督 next-token prediction，也称 Continual Pretraining、Domain-Adaptive Pretraining。

### 1. 先确定是否需要

适合继续预训练：

- 模型缺少大量领域语言和概念；
- 有规模较大的无标注领域语料；
- 需要补充一种语言或代码分布；
- SFT 无法解决底层表示不足。

不适合把实时费率、规则和订单状态写入参数。这些变化快的信息更适合 RAG、配置和工具。

### 2. 数据准备

- 收集文档、手册、代码、论文、对话等语料；
- 去重和近重复；
- 质量过滤；
- PII、版权和许可证检查；
- 语言识别；
- 删除过期或相互矛盾的规则；
- train/validation 时间切分；
- tokenizer 覆盖率分析；
- 按文档边界 packing。

跨境支付可使用脱敏的产品文档、通道规范、合规说明和技术手册，但不能直接训练客户账号、证件和交易明细。

### 3. 数据配比

只用领域语料容易灾难性遗忘，通常混入通用 replay 数据：

```text
domain corpus + general corpus + safety corpus
```

比例由领域强度和遗忘评测决定，并做多组消融。高质量和覆盖通常比单纯扩大 token 数更重要。

### 4. 训练目标与方式

Decoder-only 模型通常继续优化 causal LM loss：

```text
L = -sum_t log p(x_t | x_<t)
```

实践中：

- 从较小学习率开始；
- warmup + cosine/linear decay；
- bf16；
- sequence packing；
- FlashAttention；
- gradient checkpointing；
- DeepSpeed/FSDP；
- 监控 loss、梯度和吞吐。

如果要改变底层语言能力，全参继续预训练通常更充分；资源有限也可做 LoRA/adapter，但容量和知识吸收上限需要实测。

### 5. Tokenizer

先检查领域术语是否被过度切碎。扩词表会改变 embedding 和 LM Head，需要初始化新 token 并继续训练，还要处理 checkpoint 与兼容性。不是看到专有名词就必须改 tokenizer。

### 6. 评测

- 领域 validation perplexity；
- 领域知识和任务评测；
- 通用能力回归；
- 多语言；
- 长文本；
- 安全和拒答；
- SFT 后的下游增益；
- 灾难性遗忘；
- 训练 token 与收益曲线。

### 7. 与 SFT 的关系

```text
基座模型
-> 继续预训练，补语言和领域表示
-> SFT，学习任务格式与行为
-> 偏好/RL 对齐
```

### 8. 面试总结

继续预训练用大规模无标注语料继续做语言模型目标，重点是数据质量、通用语料混合、低学习率和遗忘评测。稳定领域知识可进入参数，实时业务状态仍应通过 RAG 和工具提供。
