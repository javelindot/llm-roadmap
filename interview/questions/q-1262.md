# 如何构造排序模型训练数据？如何解决正负样本不均？

## 答案

排序模型训练数据应按 Query 或 Session 分组，包含候选、相关性标签、曝光位置、时间和上下文。关键不是简单凑成 1:1 正负样本，而是让训练候选接近线上排序面对的真实难度。

### 1. 正样本来源

- 人工相关性标注；
- 点击、长停留、购买、任务完成等行为；
- RAG 中能支撑答案的黄金文档/chunk；
- 专家确认的 FAQ 和规则；
- 多级标签，如强相关、部分相关、无关。

行为标签要去偏：未点击不一定不相关，可能只是没有曝光或位置靠后；点击也可能受标题吸引而非内容正确。

### 2. 负样本分层

**随机负样本**

差异大，训练稳定但很快变得过于简单。

**同类负样本**

来自相同类目、国家、产品或文档类型，提升细粒度区分。

**Hard negative**

由 BM25/Dense 高分召回但人工判为不相关，例如相似错误码、过期规则、不同币种。

**模型挖掘负样本**

用当前模型检索误排结果，人工或规则复核后加入下一轮。

**对抗最小变化**

只修改数字、否定、主体、时间或状态，防止模型只学主题。

负样本组合应包含 easy、medium、hard，不能全用极难负样本导致标签噪声和训练不稳定。

### 3. 训练目标

- Pointwise：每个候选做分类/回归；
- Pairwise：优化正样本得分高于负样本，如 RankNet；
- Listwise：对一个 Query 的候选列表整体优化；
- LambdaRank/LambdaMART：根据 NDCG 变化构造梯度权重；
- 对比学习：用于双塔召回，结合 in-batch negatives。

目标应与线上指标一致。只用 pointwise CE 不一定能得到最好的 Top-K 顺序。

### 4. 正负不均处理

排序场景负样本天然远多于正样本，不应把全量负样本直接输入，也不应盲目过采样正样本：

- 每个 Query 固定或动态采样负样本；
- 提高 hard negative 比例；
- 使用 class/sample weight；
- focal loss 处理大量 easy negative；
- pairwise/listwise 直接比较正负；
- batch 内保证不同难度和业务分桶；
- 使用曝光倾向权重 IPS 修正位置偏差；
- 对多级相关性保留 graded label。

采样后要保存 sampling probability，必要时在 loss 或评测中校正，避免训练分布与线上分布严重失真。

### 5. 防止假负样本和泄漏

- 同一问题可能有多个正确文档，不能把未标注相关文档当负样本；
- 同一文档不同 chunk 可能互补；
- 时间切分防止未来行为泄漏；
- 同用户、商品或文档跨集合泄漏要检查；
- 训练 hard negative 不能来自测试标注；
- 点击日志需过滤机器人、异常流量和重复曝光。

### 6. 评测与迭代

离线按 Query 分组计算 NDCG、MRR、Recall@K、Precision@K，并分头部/长尾、新内容和跨类目。上线做影子与 A/B，同时监控 CTR/CVR、覆盖、多样性、延迟和风险 guardrail。

跨境支付 RAG 中，可将正确规则作为正样本，将不同国家、币种、版本但措辞相似的规则作为 hard negative。ACL 违规文档应在检索前过滤，不能只靠排序模型学会压低。
