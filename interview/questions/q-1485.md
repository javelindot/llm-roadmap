# 你的推荐项目里面提到了用了腾讯广告算法的 trick 去提升模型能力，具体是什么？

## 答案

这是一道项目真实性追问。不能只说“用了腾讯广告的 trick”，而应明确到论文、模块、改动位置、消融实验和收益；如果简历没有写清具体算法，面试时应按真实情况补充，不能把通用做法包装成腾讯内部方案。

### 1. 推荐回答结构

```text
基线模型是什么
-> 原方案的问题是什么
-> 借鉴了哪篇公开论文或哪个公开模块
-> 在采样、特征、损失或网络哪一层修改
-> 为什么理论上有效
-> 离线和线上如何验证
```

例如，若真实使用的是多任务学习，可以回答：

> 我们借鉴的是公开广告推荐方案中的多任务建模思路，不是腾讯未公开的内部算法。原模型只优化点击率，点击样本多但转化样本稀疏，容易让排序目标偏离最终转化。我们把共享表示接入多个任务塔，并通过 MMoE/PLE 的任务门控减少任务冲突，同时对延迟转化、曝光偏差和样本不均衡做了处理。最终通过单变量消融确认，收益来自任务结构，而不是额外参数量。

### 2. 可能被称为 “trick” 的公开做法

- 多任务学习：ESMM、MMoE、PLE，联合优化 CTR、CVR 和价值目标；
- 负样本处理：in-batch negative、hard negative、分层采样；
- 偏差校正：位置偏差、曝光偏差、IPS 加权；
- 延迟标签：等待窗口、延迟转化建模；
- 特征交叉：Wide、DeepFM、DCN 或序列兴趣特征；
- 损失处理：类别权重、focal loss、辅助任务、蒸馏；
- 分数校准：Platt scaling、isotonic regression 或分桶校准。

只能选择项目中真正做过的方案，并讲清实验。

### 3. 跨境支付场景

若任务是支付通道路由排序，可将广告中的多目标方法迁移为：

```text
成功率任务 + 时延任务 + 成本任务 + 拒付风险任务
```

共享底层学习国家、币种、支付方式、商户和时段特征，各任务使用独立门控与任务塔。最终排序分数由业务约束下的多目标策略计算。牌照、制裁、限额和商户合同必须先做硬过滤，不能作为模型的软偏好。

### 4. 如何证明有效

- 与原基线同数据、同特征、同训练预算对比；
- 做模块、损失和采样策略的逐项消融；
- 离线看 AUC、GAUC、NDCG、校准误差及分群结果；
- 线上看成功率、转化率、成本、时延和风险护栏；
- 报告置信区间、实验周期及实际收益，不只报最优单次结果。

### 5. 面试总结

回答重点不是“腾讯”二字，而是把公开来源、问题、具体改动、原理、消融和业务收益串起来。若说不出算法名称和实验设计，“用了某厂 trick”反而会成为明显风险点。
