# Attention计算中有哪些显存优化策略？

## 答案

Attention 显存问题要区分训练和推理。训练主要是激活和 `N x N` 注意力矩阵；推理主要是随层数、上下文和并发增长的 KV Cache。不同优化解决的对象不同。

### 1. 朴素训练显存

设：

```text
Q,K,V: [B,H,N,D]
Scores: [B,H,N,N]
```

注意力分数和概率是 `O(BHN^2)`，长序列下成为主要激活显存。

### 2. FlashAttention

- 将 Q/K/V 分块放入 SRAM；
- 在线计算 stable softmax；
- 不将完整 `N x N` 矩阵写回 HBM；
- 保持精确 Attention；
- 理论 FLOPs 仍二次，但显存与 IO 显著下降。

这是训练和 Prefill 的首选基础优化。

### 3. Activation Checkpointing

前向只保存部分中间结果，反向时重算：

- 以计算换显存；
- 可对每层或 Attention/FFN 分块；
- 与 FlashAttention 可组合；
- 重算比例过高会明显降低吞吐。

### 4. 序列与上下文并行

- Sequence Parallel：沿 token 维切分部分激活；
- Context Parallel/Ring Attention：长序列分布到多卡，通信 K/V block；
- Ulysses 类 all-to-all 拆 head/sequence；
- 适合超长上下文训练，但通信和负载均衡重要。

### 5. 结构性减少

- Sliding Window/Local Attention；
- Block Sparse/Global token；
- GQA/MQA 减少 KV head；
- MLA 压缩 KV latent；
- Linear Attention/混合架构；
- 降低 head 数、head dim 或序列长度。

这些可能改变模型结构和质量，不是透明优化。

### 6. 推理 KV Cache

每层缓存历史 K/V，近似随：

```text
layers * tokens * kv_heads * head_dim * 2
```

线性增长。优化：

- PagedAttention 减少碎片；
- GQA/MQA/MLA；
- FP8/INT8 KV 量化；
- KV eviction/compression；
- prefix cache 共享公共前缀；
- sliding window；
- 限制 max context/output；
- Prefill/Decode 分离。

### 7. 其他工程手段

- BF16/FP16/FP8；
- fused QKV projection；
- packing 减少 padding；
- variable-length attention；
- 梯度累积降低 micro-batch；
- 及时释放无效 cache；
- profiler 区分 Attention、FFN、参数和通信。

### 8. 跨境支付

长规则和历史工单可通过 RAG 缩短输入，比无条件扩大上下文更直接。KV/Prefix Cache 必须包含租户和权限维度，避免跨用户复用敏感内容。

**面试总结**：训练优先 FlashAttention + checkpoint + 并行；推理优先分页 KV + GQA/MLA + 量化和调度。
