# 🔥 介绍一下 RAG

## 答案

**RAG（Retrieval-Augmented Generation）**是在模型生成前，从外部知识库检索与问题相关的证据，并把证据放进上下文，让模型基于可更新、可追溯的资料回答。

### 1. 离线入库链路

```text
数据采集
→ 解析与清洗
→ 按语义/结构分块
→ 添加标题、时间、权限等元数据
→ 生成 Embedding
→ 建立向量索引和关键词索引
```

难点包括 PDF/表格解析、Chunk 粒度、版本更新、重复内容和权限继承。

### 2. 在线问答链路

```text
用户问题
→ 意图识别 / Query Rewrite
→ 元数据过滤
→ 向量 + BM25 混合召回
→ Reranker 精排
→ 上下文去重、排序和压缩
→ LLM 基于证据生成
→ 引用、校验和反馈
```

### 3. RAG 的价值

- 知识可以独立更新，不必频繁微调模型。
- 回答可附引用，便于审计和核验。
- 能接入私有知识和实时数据。
- 相比把所有文档塞进长上下文，成本和噪声更可控。

### 4. 主要失败点

- 文档中有答案，但没有召回。
- 召回内容相关但不完整，或切片破坏了语义。
- 正确证据排位太低，被无关上下文稀释。
- 模型没有忠实使用证据，仍按参数记忆作答。
- 知识过期、权限泄漏或引用错误。

### 5. 如何评测

应分层评估：

- 检索：Recall@K、MRR、NDCG。
- 生成：正确性、忠实度、完整性、引用准确率。
- 系统：延迟、成本、任务解决率和人工转接率。

**面试总结**：RAG 不是“向量库 + Prompt”这么简单，而是一条从数据治理、检索排序到生成校验的完整工程链路。
