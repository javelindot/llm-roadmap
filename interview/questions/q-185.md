# 有测过回答的准确率吗？

## 答案

> 测过，我们有**离线 + 线上**两套评估体系。
>
> **离线评估（每周跑一次）**
> - **黄金集**：500 条人工标 (query, 期望答案, 相关 chunk_id)
> - **指标**：
>   - 召回 Recall@5 = 92%
>   - 端到端正确率 = 88%（LLM judge + 人工抽检）
>   - Faithfulness（答案忠于召回）= 95%
>   - Answer Relevance = 89%
> - 用 **RAGAS 框架**，judge model 用 Qwen-72B（不出公网）
> - 任何代码改动必须先过离线，不掉点才能合并
>
> **线上监控（实时）**
> - **业务指标**：
>   - 点赞率 / 点踩率
>   - 转人工率（核心 KPI，目标 < 25%）
>   - 客户后续投诉率
> - **技术指标**：
>   - p50 / p95 / p99 延迟
>   - LLM token 消耗
>   - 召回 score 分布
> - 每日 sample 100 条人工标（spot check）
>
> **分场景细分准确率**：
> - 订单状态查询：96%（最容易）
> - 政策查询（汇率、手续费）：91%
> - 退款相关：85%（多轮对话挑战）
> - 复杂客诉：72%（最难，长尾问题）
>
> **失败 case 怎么处理**：
> - 召回失败 → 知识库补内容 / 重训 embedding
> - 召回到但生成错 → DPO 增强 / prompt 调优
> - 引用错误 → 强化 cite enforcement prompt
>
> 准确率 88% 不算特别高，但**模型必须说"我不知道"的拒答率也是 KPI 之一**——宁可拒答 10% 也不能编。这是金融场景的硬要求。
