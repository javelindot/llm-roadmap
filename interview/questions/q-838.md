# rerank 用的什么模型

## 答案

应先给出项目真实模型，再解释为什么选它。没有项目上下文时，可按延迟和精度分层选型。

### 常见 reranker

- **Cross-Encoder**：将 `[query; document]` 一起输入 BERT/RoBERTa/DeBERTa，输出相关性分数，精度高但每个候选都要前向；
- **BGE Reranker**：如 `bge-reranker-v2-m3`，适合多语言与通用检索；
- **Qwen3-Reranker**：基于生成式基础模型的多语言 reranking 系列，适合需要更强语义判断的场景，可参考[官方介绍](https://qwenlm.github.io/blog/qwen3-embedding/)；
- **ColBERT/Late Interaction**：保留 token 级向量交互，成本介于双塔与完整 cross-encoder；
- **LLM pointwise/pairwise/listwise rerank**：能力强但慢，适合少量高价值候选或离线 teacher。

### 项目回答模板

> 第一阶段 BM25+向量召回 top-[K]，使用[模型名]重排到 top-[N]。选择它是因为[语言、领域、延迟]，在相同候选上比[基线]的 nDCG/MRR 提升[真实值]，P95 为[真实值]。最终根据效果、延迟与成本决定。

### 优化与评估

- hard negative 用于训练或微调；
- 动态 top-k：简单 query 少排，困难 query 多排；
- batching、量化、蒸馏和缓存控制延迟；
- 同时看 MRR/nDCG、Recall 保留率、业务切片和 P99；
- 防止 reranker 偏爱长文档，检查时效、权限和版本。

跨境支付中，重排应加入国家、币种、通道和生效时间，但权限过滤必须在召回前完成，不能指望 reranker 消除越权文档。

### 面试总结

高精度常用 Cross-Encoder/BGE/Qwen reranker，超低延迟可用轻量 cross-encoder 或 late interaction。模型名不是答案，必须说明候选规模、指标、延迟和业务切片。
