# 🔥🔥 Multi-Head Attention 的作用是什么?

## 答案

**一句话**：Multi-Head Attention 把同一份输入投影到多个不同的表示子空间，让模型可以同时关注不同位置、不同关系和不同特征，再把各头结果融合。

### 1. 计算形式

```text
head_i = Attention(XW_i^Q, XW_i^K, XW_i^V)
MHA(X) = Concat(head_1, ..., head_h) W^O
```

如果模型维度是 `d_model`，通常拆成 `h` 个头，每个头维度约为 `d_model / h`。

### 2. 主要作用

- **关注不同关系**：有的头偏局部邻近，有的头偏长距离依赖，有的头可能关注指代、语法或边界。
- **提高表示能力**：单头只有一套相似度空间，多头可以学习多套 Q/K/V 投影。
- **并行聚合信息**：每个 Token 能从多个角度读取上下文。
- **稳定优化**：相比只使用一个同等总维度的大头，多个子空间通常更容易学习互补模式。

### 3. 常见误区

- 多头并不会把注意力复杂度从 `O(n²)` 降下来。
- 在总隐藏维度不变时，多头不意味着参数和计算量简单乘以头数，因为每个头的维度被缩小。
- 不同头不保证天然互补，实际模型中可能存在冗余头。
- 头数越多不一定越好；每头维度过小会限制表达能力。

### 4. 与推理优化的关系

标准 MHA 每个 Query 头都有独立 K/V 头，KV Cache 较大。MQA 让所有 Query 头共享一组 K/V，GQA 则让若干 Query 头共享一组 K/V，在质量和推理效率之间折中。

**面试总结**：Multi-Head 的价值不是“重复算多次”，而是让模型在多个可学习的关系空间中并行读取上下文。
