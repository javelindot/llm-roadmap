# 3.新结构增加的参数量

## 答案

这道题缺少“新结构”上下文，面试时第一步应确认是在问 LoRA、Adapter、MoE、额外 Attention，还是某个自定义模块。不要直接猜数字；先写输入输出维度，再按矩阵形状计算。

### 1. 通用计算原则

一个线性层：

```text
Linear(d_in, d_out)
参数量 = d_in * d_out + d_out
```

若不使用 bias：

```text
参数量 = d_in * d_out
```

LayerNorm 通常有缩放和偏置：

```text
2 * d_model
```

RMSNorm 通常只有缩放：

```text
d_model
```

### 2. LoRA 增加参数

对原矩阵 `W ∈ R^(d_out x d_in)` 加 rank 为 `r` 的 LoRA：

```text
Delta W = B * A
A: [r, d_in]
B: [d_out, r]

新增参数 = r * (d_in + d_out)
```

若在每层 Q、K、V、O 四个 `d x d` 投影都加 LoRA：

```text
每层新增 = 4 * 2 * d * r = 8dr
总新增 = n_layers * 8dr
```

还要根据实际 target modules 判断，不可机械套公式。

### 3. Adapter 增加参数

瓶颈维度为 `b`：

```text
down: d -> b
up:   b -> d

新增约 = 2db
```

若每层放两个 Adapter，再乘 2，并加 bias/Norm 参数。

### 4. MoE 增加参数

若原来一个 FFN 的参数约为 `P_ffn`，改为 `E` 个专家：

```text
总专家参数约 = E * P_ffn
每 token 激活参数约 = top_k * P_ffn
```

另外有 Router：

```text
d_model * E
```

所以 MoE 的总参数量和每 token 激活参数量必须分开说。

### 5. 新 Attention 模块

标准 Q/K/V/O 投影若都是 `d x d`：

```text
约 4d^2
```

GQA/MLA 会改变 K/V 的投影维度，应按真实 head 数和 latent 维度逐项展开，不能仍按 `4d^2`。

### 6. 面试表达

> 我先确认新结构插在哪几层、是否共享、是否带 bias，以及训练时增加参数还是推理时也保留。然后列每个矩阵的 shape 求和。参数量之外，我还会区分可训练参数、总参数、每 token 激活参数、FLOPs 和 KV Cache，因为这几个指标不是一回事。

这类题真正考察的是维度意识和建模严谨性，不是背一个固定数字。
