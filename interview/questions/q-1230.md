# 具体介绍一下RAG项目中的应用，如何处理文档，如何计算相似度

## 答案

RAG 在跨境支付中的适用范围，是产品规则、操作手册和合规知识等“需要更新、私有、可引用”的内容；订单状态、余额、实时汇率和通道回执由业务工具查询。完整链路为：

> 文档治理与解析 → 结构化切块 → 多路索引 → 查询理解与鉴权 → 混合召回 → Rerank → 上下文构造 → 受约束生成与引用。

### 1. 文档来源和治理

知识源可以包括：

- 全球收款、付款、换汇和收单产品文档；
- KYC/KYB 材料要求和操作 SOP；
- 国家、币种、通道支持范围；
- API 文档、错误码、Webhook 与接入指南；
- 退款、退汇、拒付和结算流程；
- PDF、Word、HTML、Markdown、FAQ、表格和扫描件。

入库前记录 `source、owner、version、effective_time、language、tenant、ACL`。重复、过期或无权限文档不能直接进入索引；扫描件保留 OCR 置信度，低质量页面转人工复核。

### 2. 文档解析

不同格式使用对应解析器，目标不是只提取纯文本，而是保留：

- 标题层级、段落和列表；
- 表格的表头、单位和合并单元格；
- 页码、图片说明和脚注；
- 代码块、API 字段和示例；
- 文档版本与生效范围。

支付规则中“国家、币种、客户类型、前置条件、例外、结论”必须保持关联。表格不能把数值与表头切开，API 示例不能与字段定义分离。

### 3. 切块与元数据

先按结构和业务语义切，再处理长度：

- FAQ：一个完整问答一个 chunk；
- 错误码：错误码、原因和处置建议一个 chunk；
- 规则：条件、例外和结论保持完整；
- 表格：按行组或业务实体切，重复必要表头；
- 长章节：递归切分并使用父子块；
- 流程：前置条件、步骤和异常分支尽量同块。

每个 chunk 携带：

```text
chunk_id, document_id, title_path, page,
country, currency, product, channel,
version, effective_time, language, ACL
```

子块用于精确召回，命中后回取父块或相邻块补齐上下文。`chunk_size` 不设固定答案，可以从 300 到 800 tokens 做实验，并同时评估召回、上下文噪声、延迟和成本。

### 4. Embedding 与相似度

文档 chunk 和 Query 必须使用兼容的 embedding 模型与相同预处理。常见 dense 相似度：

**余弦相似度**

```text
cos(q, d) = (q · d) / (||q|| * ||d||)
```

只比较方向。若向量已做 L2 normalize，则 cosine ranking 与 inner product ranking 等价。

**点积**

```text
score(q, d) = q · d
```

保留向量范数影响，适用于训练目标与索引配置匹配的模型。

**欧氏距离**

```text
distance(q, d) = ||q - d||_2
```

距离越小越相似。单位向量下，它与余弦相似度具有单调关系。

不能只看向量库返回的分数：不同模型、语言和索引的分数分布不可直接共用阈值，必须在业务验证集上校准。

### 5. 为什么需要 BM25 和 Rerank

Dense retrieval 擅长口语、同义词和跨语言语义，但对 `E10027、SWIFT、PIX、USD` 等精确标识不一定稳定。BM25 基于词频、逆文档频率和长度归一化，对错误码、产品名、国家码和 API 字段更可靠。

两路召回后可以使用 RRF：

```text
RRF(d) = Σ_i 1 / (k + rank_i(d))
```

RRF 不要求 BM25 与 cosine 分数处于同一尺度。合并后再用 cross-encoder reranker 计算 Query 与候选的联合相关性，并执行版本、权限、国家、币种和产品过滤。

### 6. 在线回答

用户问“这笔 EUR 付款为什么还没到”时：

1. 解析订单号、商户与币种并鉴权；
2. 调用订单、通道和清算工具获取当前事实；
3. 检索对应产品和通道的到账、失败与退汇规则；
4. Rerank 后保留少量完整证据；
5. 模型区分“当前状态”和“规则解释”，逐条引用；
6. 证据不足、冲突或涉及不可公开风控原因时转人工。

### 7. 评测

- 召回：Recall@K、MRR、NDCG、关键证据覆盖率；
- 上下文：Context Precision/Recall、重复率、过期或越权 chunk 比例；
- 生成：faithfulness、答案正确性、完整性和引用准确率；
- 业务：首次解决率、人工改写率、错误承诺率；
- 系统：P95/P99、token、检索与模型成本、索引新鲜度。
