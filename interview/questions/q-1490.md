# MMOE的结构及其解决的问题

## 答案

MMoE（Multi-gate Mixture-of-Experts）是一种多任务学习结构，用多个共享专家学习不同模式，再让每个任务通过独立门控选择专家组合，主要缓解任务相关性不一致和负迁移。

### 1. 结构

输入 `x` 进入 `n` 个专家：

```text
e_i(x) = Expert_i(x)
```

任务 `k` 有独立门控：

```text
g_k(x) = softmax(W_k x)
h_k(x) = sum_i g_k_i(x) * e_i(x)
y_k = Tower_k(h_k(x))
```

完整结构包括：

- 共享输入与 Embedding；
- 多个 Expert，通常为 MLP；
- 每个任务独立的 Gate；
- 每个任务独立的 Tower 和输出层。

### 2. 解决什么问题

Shared-Bottom 强制所有任务共享同一表示。当 CTR、CVR、时延或风险目标相关程度不同甚至梯度冲突时，容易负迁移。MMoE 允许不同任务按样本选择不同专家，同时保留可共享的信息。

与普通 MoE 的区别是：普通 MoE 常由一个路由器为模型稀疏选专家；MMoE 主要服务多任务学习，每个任务都有自己的 gate，经典实现也常是稠密加权。

### 3. 局限与改进

- 专家可能塌缩，多个专家学到相似功能；
- 所有专家仍完全共享，任务差异很大时隔离不足；
- Gate 不一定天然可解释；
- 任务损失尺度和样本量不平衡仍需处理；
- 专家数增加会提高计算和参数量。

可通过熵或负载均衡正则、任务损失动态加权、专家使用率监控改善。PLE 进一步区分共享专家与任务专属专家，常用于任务冲突更强的场景。

### 4. 跨境支付场景

通道路由可联合预测成功率、成本、时延和拒付风险。不同任务共享国家、币种、商户和支付方式表示，但通过独立 Gate 选择专家。最终决策层再按业务约束组合各任务输出。

制裁、牌照、额度和强风控规则不能作为普通任务分数与收益互相抵消，必须单独硬约束。

### 5. 面试总结

MMoE 的核心是“多个共享专家 + 每个任务独立 Gate + 独立任务塔”。它比 Shared-Bottom 更灵活地控制共享程度，主要缓解多任务负迁移，但仍需关注专家塌缩、任务不平衡和硬约束边界。
