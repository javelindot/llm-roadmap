# vllm的原理是什么？

## 答案

核心原理：PagedAttention（分页注意力）
核心思路：
将KV Cache拆分成固定大小的page（比如16/32个tokens）；
每个请求的KV Cache不再占用连续显存，而是由多个离散的page组成；
用一个page table记录每个请求的KV Cache页存在显存的哪个位置。

### 补充说明

vLLM 的核心价值是提升大模型推理吞吐。传统推理里，每个请求的 KV Cache 往往需要连续显存，长短请求混在一起容易产生显存碎片。PagedAttention 借鉴操作系统分页思想，把 KV Cache 拆成固定大小的 block/page，用 page table 管理映射，从而更灵活地复用显存。

它还支持 continuous batching，也就是请求动态进入和退出 batch，不用等一批请求全部生成完再处理下一批，因此并发吞吐更好。

### 结合跨境支付项目怎么说（面试口径）

> 在跨境支付客服场景里，用户问题长短差异很大。有人只问“多久到账”，有人会贴一大段订单信息和通道回执。如果用普通推理服务，长短请求混合容易造成显存浪费和排队。vLLM 的 PagedAttention 能更好管理 KV Cache，提高 GPU 利用率。
>
> 但我也会控制 `max_model_len` 和并发参数，因为上下文越长，KV Cache 越大。比如客服实时问答要优先保证低延迟，离线文档解析不能和它抢资源。
>
> 所以我理解 vLLM 的价值不是让模型更聪明，而是让模型服务在高并发场景下更稳定、更高吞吐。

**钩子**：可以引到“PagedAttention、KV Cache 分页、continuous batching、长短请求混部”。
