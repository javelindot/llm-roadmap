# lora原理，bert结构，截断长度token个数如何提升，rag大规模数据怎么处理，自然语言处理方面的了解程度，nlp大数据挖掘方面怎么理解，现在的大模型如何应用到搜索检索层面？

## 答案

这是一组综合追问，回答时应分块，不要把所有概念混在一起。

### 1. LoRA 原理

冻结原权重 \(W\)，只训练低秩增量：

\[
W'=W+\frac{\alpha}{r}BA
\]

其中 \(A\in\mathbb R^{r\times d_{in}}\)、\(B\in\mathbb R^{d_{out}\times r}\)，且 \(r\) 很小。它减少可训练参数和优化器状态，但不会自动降低基座推理成本。

### 2. BERT 结构

BERT 是双向 Transformer Encoder，由 token、position、segment embedding，多层 MHA、FFN、残差和 LayerNorm 构成；使用 MLM 预训练，适合分类、NER、抽取和编码任务。

### 3. 如何提升截断 token 长度

- 修改位置编码并继续训练，如 RoPE scaling、YaRN/LongRoPE 类方法；
- 使用滑窗、分块、层次编码或长上下文模型；
- FlashAttention 降低 IO 和中间显存；
- 稀疏/局部注意力改变二次成本；
- 业务上优先用检索、摘要和记忆压缩减少无效 token。

只修改 tokenizer 的 `max_length` 不能让模型真正理解更长文本，还要考虑位置外推、训练分布、显存和长上下文评测。

### 4. 大规模 RAG 数据

- 使用分布式解析、去重、版本化与增量 CDC；
- 按文档结构切分，建立父子索引和 metadata；
- 向量索引分片，冷热分层，支持删除和重建；
- BM25、向量、多路召回加 reranker；
- 缓存热点查询，控制 top-k 和上下文预算；
- 分层评估 Recall、MRR、引用正确率、延迟和成本；
- 权限过滤必须在检索阶段执行，而不是生成后补救。

### 5. NLP 大数据挖掘

从海量文本中做实体、主题、关系、情感、事件和异常模式抽取，再通过弱监督、聚类、图谱或表示学习形成可检索、可分析资产。关键是数据质量、时间漂移、采样偏差和隐私治理，而不是只跑一个模型。

### 6. 大模型如何用于搜索

- Query 理解、改写、纠错和意图分类；
- Embedding 召回与跨语言检索；
- LLM/reranker 精排；
- 基于证据的答案生成与引用；
- Agent 多跳检索和工具查询；
- 合成训练数据、hard negative 和评测样本。

大模型不应替代倒排索引、向量库和权限系统，而应与经典搜索组成级联。

### 跨境支付场景

对支付规则、错误码和订单问题，使用国家、币种、通道、生效时间和租户权限过滤；精确错误码走 BM25，语义问题走向量召回，费率和状态查实时工具。模型只基于证据解释，缺失证据则转人工。

### 面试总结

这组题的主线是：LoRA 解决参数高效适配，BERT 解决上下文编码，长文本依靠位置与 Attention 工程共同扩展，大规模 RAG 依靠数据和索引治理，大模型则增强搜索的理解、排序和生成环节。
