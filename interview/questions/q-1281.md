# CLIP损失函数

## 答案

CLIP 的核心损失是图像到文本、文本到图像两个方向的对称 InfoNCE/交叉熵。它不是对每对样本独立做二分类，而是在一个 batch 的 `N × N` 相似度矩阵上完成匹配。

### 1. 构造相似度矩阵

设图像编码器和文本编码器输出经过投影及 L2 归一化：

```text
I_i = normalize(f_image(image_i))
T_j = normalize(f_text(text_j))
```

相似度 logits：

```text
S_ij = I_i dot T_j / tau
```

矩阵形式：

```text
S = I @ T^T / tau
```

如果第 `i` 张图与第 `i` 段文本配对，那么正样本是对角线 `S_ii`，标签为：

```text
y = [0, 1, 2, ..., N - 1]
```

### 2. 两个方向的损失

图像到文本：

```text
L_i2t = -1/N * sum_i log(
    exp(S_ii) / sum_j exp(S_ij)
)
```

这里每一行表示“给定图像 `i`，在 N 个文本里找到对应文本”。

文本到图像：

```text
L_t2i = -1/N * sum_j log(
    exp(S_jj) / sum_i exp(S_ij)
)
```

这里每一列表示“给定文本 `j`，在 N 张图里找到对应图像”。

总损失：

```text
L_CLIP = (L_i2t + L_t2i) / 2
```

伪代码：

```python
image_emb = normalize(image_encoder(images))
text_emb = normalize(text_encoder(texts))

logits = image_emb @ text_emb.T / temperature
labels = arange(global_batch_size)

loss_i = cross_entropy(logits, labels)
loss_t = cross_entropy(logits.T, labels)
loss = (loss_i + loss_t) / 2
```

### 3. 梯度如何工作

对每个 anchor：

- 提高对应正样本 `S_ii`；
- 降低其他负样本相似度；
- softmax 概率越高的错误负样本，梯度越大；
- temperature 决定分布尖锐程度和 hard negative 的权重。

L2 normalization 防止模型只靠增大向量范数降低损失，使相似度主要表达方向；整体尺度由可学习的 `logit_scale = 1/tau` 控制。

### 4. 分布式实现要点

多卡训练要构造全局负样本：

- AllGather 各 rank 的 image/text embedding；
- 本地样本的正标签要加全局 offset；
- 确认 gather 是否保留本地梯度；
- 避免只在单卡 batch 内计算，导致有效负样本过少；
- 大矩阵会增加通信和显存，可使用分块 logits 或局部损失实现。

### 5. 数据问题比公式更容易出错

标准 CLIP 假设一一配对，但实际数据可能有：

- 一张图对应多段正确文本；
- 多张近重复图片；
- 同义文案在 batch 中被当成负样本；
- OCR 错误、语言错配；
- 正负样本高度不均。

这会产生 false negatives。可通过去重、multi-positive loss、监督对比学习、hard-negative 过滤或更合理的 batch 构造处理。

跨境支付材料检索中，同一模板、同一银行回单格式可能有多个合理匹配，不能机械地把非对角线全部当绝对负样本；还要避免模型学习账户号、姓名等敏感捷径。
