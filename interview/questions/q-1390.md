# 精确率和召回率

## 答案

精确率和召回率都基于二分类混淆矩阵：

```text
TP：预测为正，实际为正
FP：预测为正，实际为负
FN：预测为负，实际为正
TN：预测为负，实际为负
```

### 1. 精确率 Precision

```text
Precision = TP / (TP + FP)
```

含义是“模型预测为正的样本中，有多少是真的正样本”。Precision 高说明误报少。

例如系统识别出 100 笔高风险交易，其中 80 笔经复核确实高风险：

```text
Precision = 80 / 100 = 80%
```

### 2. 召回率 Recall

```text
Recall = TP / (TP + FN)
```

含义是“所有真实正样本中，有多少被模型找出来”。Recall 高说明漏报少。

如果真实共有 200 笔高风险交易，模型识别出其中 80 笔：

```text
Recall = 80 / 200 = 40%
```

### 3. 二者的权衡

降低分类阈值，通常会：

- 预测更多正样本；
- Recall 上升；
- Precision 可能下降。

提高阈值通常相反。

跨境支付场景要按错误成本选择：

- 制裁、欺诈初筛更怕漏报，优先 Recall；
- 人工审核成本很高时，要提高 Precision；
- 最终应通过分层策略，把高风险召回和人工容量一起优化。

### 4. F1

```text
F1 = 2 * Precision * Recall / (Precision + Recall)
```

F1 是调和平均，适合需要同时兼顾 Precision 和 Recall 的场景。但它默认二者同等重要，可以使用 `F_beta` 调整权重：

```text
F_beta = (1 + beta^2)PR / (beta^2 P + R)
```

`beta > 1` 更重视 Recall，`beta < 1` 更重视 Precision。

### 5. 多分类和检索

多分类要说明：

- macro：各类别等权，关注少数类；
- micro：汇总所有样本，受大类影响；
- weighted：按类别样本数加权。

检索中的 `Precision@K` 和 `Recall@K` 则衡量前 K 个结果的相关比例和相关文档覆盖率。

### 6. 面试总结

Precision 关注“查出来的有多少是真的”，Recall 关注“真的正样本有多少被查出来”。二者通常通过阈值权衡，业务上应根据误报和漏报成本选择，而不是脱离场景追求单一最高值。
