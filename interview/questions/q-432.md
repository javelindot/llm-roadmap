# 🔥 强化学习

## 答案

**一句话**：强化学习是让智能体通过和环境交互，根据奖励信号学习策略，目标是最大化长期累计收益。

### 1. 基本概念

- Agent：做决策的智能体。
- Environment：环境。
- State：当前状态。
- Action：动作。
- Reward：奖励。
- Policy：策略，决定在某个状态下采取什么动作。
- Value：某个状态或动作的长期价值。

强化学习和监督学习不同，监督学习有明确标签；强化学习更多依赖试错和奖励反馈。

### 2. 常见算法

- Q-learning / DQN：学习动作价值。
- Policy Gradient：直接优化策略。
- Actor-Critic：策略和价值函数结合。
- PPO：通过裁剪目标函数稳定策略更新。

### 3. 结合大模型

在大模型里，强化学习常用于 RLHF，用 Reward Model 或规则奖励指导模型输出更符合偏好。但高风险业务里不能只靠 RL 奖励，要有规则和人工兜底。

### 4. 结合跨境支付项目怎么说（面试口径）

> 如果结合跨境支付业务，我不会让强化学习直接决定放款或拒付，因为资金和合规风险太高。更适合用在客服话术优化、Agent 策略选择和排序策略上。
>
> 比如 Agent 可以学习在什么情况下先查订单、先查知识库还是转人工；奖励可以来自问题解决率、人工改写率、用户满意度、是否合规。但涉及风控放行、资金操作，最终仍要由规则系统和人工审批决定。
>
> 所以我理解强化学习在企业场景里更多是优化辅助策略，而不是替代核心业务决策。

**钩子**：可以引到“Agent 策略优化、RLHF、奖励设计、支付资金动作不能完全交给 RL”。
