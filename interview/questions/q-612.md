# 介绍下LLAMA的架构

## 答案

LLaMA 是一类 decoder-only、自回归 Transformer。不同代际和尺寸在词表、上下文长度、注意力头配置等方面不同，不能把某个版本的参数当成整个系列的固定配置，但其公共骨架可以概括如下。

### 基本结构

输入文本先经过 tokenizer 转成 token，查表得到 embedding。模型堆叠多个 Transformer decoder block，每层通常包含：

1. RMSNorm。
2. 带因果 mask 的多头自注意力。
3. 残差连接。
4. RMSNorm。
5. SwiGLU 前馈网络。
6. 残差连接。

最后经过归一化和语言模型输出头，得到下一 token 的词表 logits。

### 关键设计

- Decoder-only：只能关注当前位置及之前的 token，适合 next-token prediction 和自回归生成。
- Pre-Norm：归一化放在注意力或 FFN 之前，深层训练通常更稳定。
- RMSNorm：只根据均方根缩放，不减去均值，计算比 LayerNorm 更简洁。
- RoPE：将位置信息以旋转方式作用于 Q/K，使注意力分数编码相对位置信息。
- SwiGLU：可写成 $SiLU(xW_1)\odot(xW_3)$ 后再乘 $W_2$，相比普通 ReLU FFN 通常有更好的容量利用。
- 无或少 bias：许多 LLaMA 线性层不使用偏置，以简化计算。
- GQA：较新的部分 LLaMA 模型使用 Grouped-Query Attention，让多个 query 头共享较少的 K/V 头，降低 KV Cache 和推理带宽。

### 前向与生成

训练时输入右移后的 token 序列，通过交叉熵预测每个位置的下一个 token。推理分为 prefill 和 decode：prefill 一次处理 prompt 并建立每层 KV Cache；随后每步只输入新 token，复用历史 K/V，采样一个 token 后继续，直到生成 EOS 或达到长度限制。

面试中最好先说明具体版本和参数规模，再讨论其 head 数、词表、上下文窗口等配置，避免把 LLaMA 1、2、3 的细节混在一起。
