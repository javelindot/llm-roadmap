# 单卡显存装不下一个完整的模型怎么办

## 答案

**一句话**：单卡装不下完整模型，可以从降低模型占用、切分模型、切分优化器状态和换推理/训练策略几个方向解决。

### 核心回答
- **推理场景**：
  - 量化：INT8、INT4、GPTQ、AWQ。
  - CPU/GPU offload。
  - 张量并行，把权重切到多张卡。
  - 使用 vLLM、PagedAttention 优化 KV Cache。
  - 换更小模型或蒸馏模型。
- **训练场景**：
  - LoRA/QLoRA，只训练少量参数。
  - 梯度检查点，减少激活显存。
  - ZeRO/FSDP，切参数、梯度、优化器状态。
  - 张量并行、流水线并行。
  - 减小 batch size 和序列长度。

判断时先区分是参数装不下，还是 KV Cache/激活/优化器状态装不下。

### 1. 先区分训练还是推理

推理场景单卡装不下，通常优先考虑：

- 换小模型或蒸馏模型。
- 权重量化到 INT8/INT4。
- 多卡张量并行。
- CPU/GPU offload。
- 优化 KV Cache，避免上下文和并发撑爆。

训练场景更复杂，因为还要保存梯度、优化器状态和激活值，所以更常用：

- LoRA/QLoRA，只训练少量参数。
- 梯度检查点，用计算换显存。
- ZeRO/FSDP，切分参数、梯度、优化器状态。
- 减小 batch size、序列长度，配合梯度累积。

### 2. 工程判断顺序

我的判断顺序是：

1. 业务是否真的需要这么大的模型。
2. 是权重装不下，还是训练状态/KV Cache 装不下。
3. 能否通过量化和上下文控制解决。
4. 是否可以用 LoRA/QLoRA 代替全量训练。
5. 最后才考虑多卡并行和复杂部署。

### 3. 面试总结

单卡显存不够不是只有"加卡"一个答案。面试里要按训练/推理、权重/缓存/优化器状态分别拆，体现你有部署经验。

### 4. 结合跨境支付项目怎么说（面试口径）

> 在跨境支付项目里，如果只是做风控助手、合规问答或客服辅助，我一般不会硬上超大模型。很多场景用 7B/14B 加 RAG、工具调用和好的 prompt 就能满足，没必要为了参数量把部署复杂度拉满。
>
> 如果推理阶段单卡装不下，我会先尝试量化、vLLM、限制上下文和并发；如果训练阶段装不下，我会优先 LoRA/QLoRA，而不是全量微调。这样成本低，也方便为风控、客服、对账分别维护 adapter。
>
> 只有模型能力确实不够，或者公司要训练金融领域基座模型，我才会考虑多卡张量并行、FSDP 这类复杂方案。业务落地里，稳定、便宜、可回滚比一味追大模型更重要。

**钩子**：先判断瓶颈是权重、KV Cache 还是训练状态，再选量化、LoRA 或并行。
