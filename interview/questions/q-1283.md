# 2.分类模型微调前如果比较确定，但sft后可能会把概率输出变得不那么确定了，你觉得是什么导致的

## 答案

SFT 后分类概率变得不那么极端，既可能是模型退化，也可能是校准变好。不能只看最大概率下降，要同时看准确率、NLL、ECE、Brier Score 和可靠性图，再判断是合理降低过度自信，还是出现 under-confidence。

### 1. 先确认概率的计算口径

需要确认：

- 是独立分类头的 softmax，还是生成式模型对标签 token 的概率；
- 微调前后 temperature 是否相同；
- Prompt、模板、截断和 tokenizer 是否一致；
- 是否只比较了候选标签 token，还是全词表概率；
- verbalizer 是否由多个 token 组成；
- 是否启用了 dropout、label smoothing 或其他后处理。

生成式分类中，“正向”“负向”等标签的首 token 概率不一定等于完整类别概率，tokenization 变化会制造假象。

### 2. 数据使决策边界变得更平滑

SFT 数据可能包含：

- 标签噪声；
- 同一输入对应不同标注；
- 类间边界样本增加；
- 类别比例变化；
- 新领域与预训练分布不一致。

模型发现任务本身存在歧义后，输出不再接近 0 或 1 是合理现象。若准确率和 NLL 改善，概率降低可能代表校准更好。

### 3. Loss 和正则化影响 logit 尺度

- label smoothing 明确阻止目标概率逼近 1；
- KL regularization 限制模型偏离参考分布；
- entropy bonus 会鼓励更高熵输出；
- weight decay、dropout 可能缩小 logit margin；
- 多任务 loss 让分类目标不再独占梯度；
- knowledge distillation 的 soft target 会产生更平滑分布。

Softmax 对 logit 尺度敏感。即使 argmax 不变，若最后一层权重范数或 hidden-state norm 下降，概率也会明显变平。

### 4. 优化过程造成能力干扰

学习率过大、训练轮次过多或全参数更新可能导致：

- catastrophic forgetting；
- 原分类表示被指令跟随目标覆盖；
- 分类头与 backbone 不再匹配；
- 类别 margin 缩小；
- 少数类被多数类梯度主导。

如果 SFT 主要训练开放式回答，而分类样本占比较低，模型可能更会解释，却不再保留原来的分类边界。

### 5. 排查实验

按以下顺序做受控对比：

```text
固定评测集、Prompt、tokenizer 和 temperature
比较微调前后 logits、margin、hidden norm、head weight norm
绘制 confidence histogram 和 reliability diagram
报告 accuracy、NLL、ECE、Brier
按类别、难度、领域和长度分桶
做 clean data / noisy data 消融
做不同学习率、epoch、LoRA rank 和冻结层消融
```

还要比较正确样本与错误样本的置信度：正确样本置信度下降、错误样本不变通常是退化；错误样本的过度自信显著下降而准确率持平，可能是改善。

### 6. 修复方法

- 清洗冲突标签，统一标注规范；
- 类别均衡采样或调整 loss 权重；
- 减小学习率和训练轮次；
- 使用 LoRA，冻结部分 backbone 或保留分类头；
- 在 SFT 中混入原分类任务 replay 数据；
- 增加独立分类 CE 辅助损失；
- 移除不必要的 label smoothing/entropy regularization；
- 训练后用验证集做 temperature scaling。

Temperature scaling 只能校准概率，不能修复准确率下降。跨境支付风险分类还要按国家、币种、商户类型和时间分桶校准，并将高风险阈值、人工复核和规则 guardrail 独立于生成模型概率。
