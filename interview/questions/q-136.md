# 什么是评估框架RAGAS？

## 答案

**一句话**：RAGAS 是一个评估 RAG 系统的框架，用来评估检索上下文和生成答案的质量。

### 核心回答
RAGAS 常见指标：
- **Faithfulness**：答案是否忠实于检索上下文，有没有幻觉。
- **Answer Relevancy**：答案是否回答了用户问题。
- **Context Precision**：召回内容中有多少是有用的。
- **Context Recall**：应该召回的证据是否被召回。
- **Answer Correctness**：答案与标准答案的一致性。

它可以结合 LLM-as-judge 自动评估，适合快速对比不同切块、embedding、rerank、prompt 方案。

局限：LLM 评估也可能有偏差，高风险场景仍要人工抽检。

### 1. RAGAS 评估的是什么

RAGAS 不是只评估最终答案，它把 RAG 拆成上下文和答案两个部分：检索上下文是否相关、上下文是否覆盖了回答需要的信息、答案是否忠实于上下文、答案是否真正回答了问题。

这比只看 BLEU/ROUGE 更适合 RAG。

### 2. 使用时要注意什么

RAGAS 经常依赖 LLM-as-judge，所以评估本身也可能有偏差。高风险业务里，不能完全用自动评估替代人工。

更合理的做法是自动评估做日常回归和方案对比，人工专家做抽样和关键场景验收，bad case 进入持续迭代。

### 3. 面试总结

RAGAS 适合快速比较 RAG 方案，尤其是切片、召回、rerank 和 prompt 的迭代，但高风险场景要结合人工评审。

### 4. 结合跨境支付项目怎么说（面试口径）

> 在跨境支付 RAG 里，我会用 RAGAS 做离线评估，比如比较不同切片策略、混合检索和 reranker 的效果。重点看 faithfulness，确保回答没有编规则；也看 context recall，确保该召回的国家/通道规则被召回。
>
> 但合规和资金相关场景我不会完全依赖自动评估。比如制裁名单、国家限制、拒付规则这类问题，最后一定要业务专家抽样验。
>
> 我的定位是：RAGAS 帮我快速发现哪个方案更好，人工评审保证高风险答案真的能用。

**钩子**：RAGAS 适合方案对比，高风险支付规则仍要人工评审。
