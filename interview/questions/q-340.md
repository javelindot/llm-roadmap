# 多Agent场景，如何处理共享记忆与私有记忆隔离？子Agent的记忆隔离，用什么实现？

## 答案

答：独立State；独立CheckPoint，记忆前缀；独立对话历史，独立上下文

### 补充说明

多 Agent 记忆隔离通常分为共享记忆和私有记忆：

- **共享记忆**：主任务目标、全局约束、最终结论、公共知识。
- **私有记忆**：子 Agent 自己的中间推理、工具结果、局部上下文。

实现上可以给每个 Agent 独立 state、独立 namespace、独立 checkpoint、独立向量库 collection 或 metadata 前缀。主 Agent 只接收子 Agent 的摘要结果，而不是完整中间上下文。

### 结合跨境支付项目怎么说（面试口径）

> 在跨境支付多 Agent 场景里，记忆隔离非常重要。比如风控 Agent 可能看到设备指纹、风险标签和历史交易频次；通道 Agent 只需要看到通道回执和错误码；合规 Agent 关注国家、主体和材料状态。它们不应该共享所有原始数据。
>
> 我会给每个子 Agent 独立 State 和 checkpoint，长期记忆也用 namespace 隔离。主 Agent 分发任务时只给必要上下文，子 Agent 返回摘要、证据和置信度。
>
> 共享记忆只放公共任务目标和最终可见结论，比如“这笔付款失败需要人工审核”。完整风控规则和敏感字段不会进入共享记忆，避免越权和信息扩散。

**钩子**：可以引到“共享记忆/私有记忆、namespace 隔离、子 Agent 只回传摘要”。
