# Transformer为什么使用多头Attention?

## 答案

多头 Attention 的核心价值是允许模型在同一层使用多套独立的注意力分布和表示子空间。单头每个 Query 只有一张 Attention 权重图，所有 Value 维度共享这张图；多头可以同时聚合不同位置和关系。

### 1. 公式

第 `h` 个 head：

```text
head_h =
softmax(
  QW_Q^h (KW_K^h)^T / sqrt(d_head)
) VW_V^h
```

最后：

```text
MultiHead(Q,K,V)
= Concat(head_1,...,head_H) W_O
```

通常：

```text
d_head = d_model / H
```

### 2. 为什么优于一个大头

- 每个 head 有独立 Q/K/V 投影；
- 可形成不同的相似度度量；
- 同一个 token 可同时关注近邻、远程实体和结构位置；
- 拼接后保留多组关系，再由 `W_O` 融合；
- 在固定 `d_model` 下，总 Attention 计算量与一个同宽单头大致同阶，并适合并行矩阵计算。

不能把具体 head 机械解释成“语法头/实体头”，头的功能是训练中形成的，可能冗余或混合。

### 3. 表达角度

单头只有：

```text
one attention distribution per query
```

多头有：

```text
H different attention distributions
```

即使总 hidden size 相同，多头也能把不同关系放在不同低维子空间，避免一张 softmax 权重图折中所有需求。

### 4. 局限与演进

- head 太多会让 `d_head` 太小；
- 部分 head 冗余，可被剪枝；
- MHA 的每个 head 都保存 K/V，KV Cache 大；
- GQA 让多组 Query heads 共享较少 KV heads；
- MQA 只保留一个 KV head；
- MLA 使用 latent 压缩；
- FlashAttention 是高效实现，不改变多头语义。

### 5. 跨境支付场景

处理支付工单时，不同 head 可能分别利用订单实体、错误原因、时间和跨句关系，但这只是表示能力；金额、状态和权限仍需结构化工具确认。

### 6. 面试总结

多头不是简单“重复 H 次”，而是用 H 套投影和注意力分布并行学习不同关系。它提高表示灵活性，代价是参数和 KV Cache，因此现代模型常用 GQA 等折中。
