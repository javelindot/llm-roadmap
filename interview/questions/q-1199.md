# 自适应学习率：使用具有自适应学习率的优化器（如Adam、RMSprop），这些优化器能够自动调整每个参数的学习速率，有助于缓解因小批量引起的训练不稳定。

## 答案

Adam、RMSprop 等自适应优化器可以缓解不同参数梯度尺度差异，但不能从根本上解决小 batch 的高方差，也不能增加显存中的 batch size。

### 1. Adam 的自适应机制

Adam 维护梯度的一阶和二阶矩：

```text
m_t = β1 m_(t-1) + (1-β1)g_t
v_t = β2 v_(t-1) + (1-β2)g_t²

θ_t = θ_(t-1)
    - η × m̂_t / (sqrt(v̂_t) + ε)
```

二阶矩使梯度波动较大的参数使用较小有效步长，一阶动量平滑短期噪声，因此比普通 SGD 更容易训练 Transformer。

### 2. 为什么不能替代大 Batch

小 batch 的问题包括：

- 梯度估计方差大；
- batch 内正负样本不足；
- loss 和梯度出现尖峰；
- 多任务中低频标签可能连续缺失；
- 对比学习的负样本数量不足。

Adam 只能按历史梯度调整步长，不能创造缺失的样本，也不能让单个 step 的梯度等价于大 batch。

优先使用：

- Gradient Accumulation；
- 多卡 Data Parallel；
- 长度分桶和动态 token batch；
- 对比学习中的跨卡 negatives 或队列；
- 分层采样保证低频任务进入 batch。

### 3. 小 Batch 下的调参

- 学习率通常需要重新 sweep，不能机械套线性放缩；
- 增加 warmup，避免早期二阶矩估计不稳定；
- 使用 Gradient Clipping；
- 检查 `beta2`：过大响应慢，过小二阶矩噪声大；
- `eps` 影响低精度训练中的数值稳定；
- 使用 AdamW 将权重衰减与梯度更新解耦；
- 按 optimizer step 更新 scheduler；
- 报告多随机种子均值和方差。

较小 batch 有时带来更好的泛化，不应只追求越大越好；目标是达到稳定且资源可承受的有效 batch。

### 4. 长文本场景

长文本 OOM 主要来自激活和 Attention：

> 自适应优化器不减少激活显存。

应配合 BF16、FlashAttention、Gradient Checkpointing、LoRA、ZeRO/FSDP、Sequence/Context Parallel。8-bit optimizer 可以减少优化器状态，但仍不解决单条超长序列激活。

### 5. 跨境支付场景

微调支付客服模型时，如果每个 batch 的国家、语言和意图分布波动大，需要分层采样和梯度累积；仅换 Adam 不会让长尾币种或高风险场景自动进入训练。应同时监控分桶 loss、梯度范数、格式通过率和业务回归。
