# PLE和MMoE有什么区别？

## 答案

**一句话**：MMoE 用多个专家共享给多个任务，通过不同 gate 组合专家；PLE 在此基础上区分共享专家和任务专属专家，减少任务之间的负迁移。

### 1. MMoE

MMoE 有多个 expert，每个任务有自己的 gate。不同任务通过 gate 对专家输出加权组合。它比硬共享更灵活，适合多任务学习。

### 2. PLE

PLE 引入共享专家和任务专属专家，并且可以分层堆叠。共享专家学习共性，专属专家学习任务特性，从而缓解不同任务之间互相干扰。

### 3. 区别

- MMoE：专家共享，任务 gate 不同。
- PLE：有共享专家和任务专属专家。
- PLE 对任务冲突处理更细。
- PLE 结构更复杂，训练和调参成本更高。

### 4. 结合跨境支付项目怎么说（面试口径）

> 在跨境支付风控或推荐场景里，多任务很常见。比如同时预测交易欺诈风险、拒付风险、到账失败概率和通道成功率。MMoE 可以让这些任务共享底层特征，又通过 gate 学习不同任务关注点。
>
> 如果任务之间冲突比较明显，比如“提高支付成功率”和“降低风险损失”目标不完全一致，PLE 会更合适，因为它能把共享能力和任务专属能力拆开。
>
> 我会根据任务相关性、数据量和线上指标选择，不会一上来就用最复杂结构。

**钩子**：可以引到“多任务学习、负迁移、支付成功率和风险任务冲突”。
