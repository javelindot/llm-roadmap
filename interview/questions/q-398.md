# （如果答的好会追问）为什么要严格划分边界答：

## 答案

企业服务必须可控；
如果涉及敏感信息，Agent权限必须最小化；
防止单Agent出错影响全局；
便于迭代、排查问题；
满足法务合规要求。

### 补充说明

严格划分边界的本质是降低复杂系统的风险。Agent 越多、工具越多、业务越敏感，越不能让一个模型同时拥有所有上下文和所有工具。

边界包括：

- 任务边界：每个 Agent 只做一类事。
- 上下文边界：只给必要信息。
- 工具边界：只绑定必要工具。
- 权限边界：按用户、角色、业务线校验。
- 输出边界：敏感信息和高风险承诺过滤。
- 责任边界：出问题能定位到哪个节点。

### 结合跨境支付项目怎么说（面试口径）

> 在跨境支付里必须严格划分边界，因为系统里既有资金动作，也有风控规则、合规数据和用户隐私。如果一个 Agent 同时能看全部数据、调全部工具，一旦理解错或被提示注入，就可能造成越权查询、错误解释甚至误操作。
>
> 我会把客服、风控、合规、通道、账务这些能力拆开。客服 Agent 负责解释和引导；风控 Agent 只输出风险大类和处理建议；合规 Agent 只判断材料和国家限制；资金动作必须走人工审批或业务系统状态机。
>
> 这样做虽然架构上多了一些边界，但好处是可控、可审计、可排查。企业级 Agent 最重要的不是“看起来很智能”，而是出问题时能收得住。

**钩子**：可以引到“任务边界、上下文边界、工具边界、支付资金安全和合规审计”。
