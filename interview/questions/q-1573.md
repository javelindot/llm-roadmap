# 大模型相关岗位的核心定义是什么？

## 答案

大模型相关岗位的核心定义是：以基础模型或生成式模型能力为主要生产要素，对模型、数据、系统或应用效果承担可量化责任，而不是仅在普通业务里调用一次模型 API。

### 1. 四类核心岗位

- **基础模型/后训练**：预训练、SFT、偏好优化、RL、多模态；
- **训练/推理系统**：分布式训练、kernel、推理服务和模型平台；
- **应用算法**：RAG、Agent、搜索、代码、文档理解；
- **数据与评测**：数据治理、合成、benchmark、安全和反馈闭环。

产品、解决方案和工程岗位也可以是大模型岗位，前提是其主要工作与成功指标真正围绕模型系统。

### 2. 核心工作闭环

```text
定义任务和错误成本
-> 构建数据与评测
-> 选择/训练/编排模型
-> 工程部署
-> 灰度和监控
-> bad case 回流
```

只会 Prompt 或只会调 API 不构成完整能力；只研究模型而无法设计可靠评测也难形成生产价值。

### 3. 所需能力

- Transformer、训练和推理原理；
- 数据、统计和实验设计；
- Python/工程与分布式系统；
- RAG/Agent 或目标领域技能；
- 性能、成本、可靠性和安全；
- 能把业务问题转换为可验证模型问题。

不同岗位深度不同，不要求一个人同时精通 CUDA、RL 和产品，但要在一个主方向上形成闭环。

### 4. 跨境支付场景

大模型岗位可以负责多语言客服、合规 RAG、KYC 文档、多步调查 Agent 或模型平台，但必须理解权限、数据驻留、审计和资金风险。若工作只是把模型包装成聊天页面，通常不属于高质量的大模型核心岗位。

### 5. 面试总结

大模型岗位的本质不是岗位名，而是是否围绕模型能力建立“数据-训练/编排-评测-部署-反馈”闭环，并对质量、成本和安全结果负责。
