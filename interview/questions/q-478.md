# Agent怎么评估效果？

## 答案

**一句话**：Agent 评估要看最终任务是否完成，也要看中间过程是否正确，包括工具调用、规划、状态管理、安全和成本。

### 1. 评估维度

- 任务成功率：最终是否解决用户问题。
- 工具调用正确率：是否选对工具、参数是否正确。
- 步骤效率：平均调用几步，是否绕路。
- 事实准确率：答案是否基于工具和知识库。
- 安全合规：是否越权、泄露敏感信息。
- 失败恢复：工具失败后能否重试、降级、转人工。
- 成本延迟：token、工具调用次数、总耗时。

### 2. 评估方法

- 构建真实任务集。
- 回放 Agent trace。
- 自动评分 + 人工复核。
- badcase 分类。
- A/B 测试不同流程和 prompt。

### 3. 结合跨境支付项目怎么说（面试口径）

> 在跨境支付 Agent 里，评估不能只看最后回答好不好。比如用户问“付款为什么失败”，Agent 是否先查订单、是否查了正确通道回执、是否误用了风控工具、最终解释是否有依据，这些都要看。
>
> 我会把每次工具调用、参数、返回结果和最终答案都记录 trace。然后用真实工单评估任务完成率、工具调用正确率、转人工率和人工改写率。
>
> 对高风险场景，比如风控拦截和资金冻结，还要单独看是否遵守安全边界，不能为了完成任务而越权。
