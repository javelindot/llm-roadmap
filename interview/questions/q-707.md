# 长期记忆如何存储？

## 答案

Agent 长期记忆不是简单保存全部聊天记录，而是把值得复用的信息结构化、持久化，并在正确作用域和时间检索回来。

### 存储内容

- episodic：用户或 Agent 发生过的事件与任务轨迹；
- semantic：稳定事实、用户偏好和领域知识；
- procedural：成功策略、工具使用方法和失败教训；
- profile：经用户授权保存的长期属性。

每条记忆应包含正文、类型、来源、创建/更新时间、作用域、权限、置信度、有效期和引用。敏感信息按最小化原则保存并支持删除。

### 存储架构

原始事件写入关系库或事件日志作为事实源；长文本存对象存储；摘要和 embedding 写入向量库；实体关系可进入图数据库。稳定 ID 与版本用于更新、去重和审计。不能只依赖向量库，因为它不擅长事务、精确删除和结构化约束。

### 写入流程

对会话或轨迹进行事实抽取、摘要、去重和冲突检测。只有达到置信度、复用价值和权限要求的内容才写入长期记忆；失败反思需经验证，避免错误经验固化。新事实与旧事实冲突时保留时间和来源，不盲目覆盖。

### 检索与遗忘

根据当前任务生成 query，先做租户、用户、时间和权限过滤，再综合语义相关性、重要度、时效性和访问频率排序。返回少量带引用片段。通过 TTL、衰减、压缩和用户删除实现遗忘。

评估包括记忆命中率、过期/错误记忆率、隐私泄漏、上下文成本和任务成功率。长期记忆的目标是提升决策，而不是无限增长。
