# 冷启动问题的解决方案

## 答案

“冷启动”要先说明场景。推荐系统中通常分为 **新用户、新物品和新系统/新领域** 三类，解决方案不同。

### 1. 新用户冷启动

系统缺少行为历史，可使用：

- 注册时选择兴趣、目标或行业；
- 利用地域、设备、入口等合规上下文特征；
- 展示热门、多样化内容作为安全基线；
- 用少量探索流量快速收集反馈；
- 根据相似人群做迁移，但避免过度刻板化。

### 2. 新物品冷启动

新内容没有点击和转化数据，可使用：

- 标题、文本、图片、类目等内容特征；
- 预训练 Embedding 计算与用户兴趣的匹配；
- 供应方质量、作者历史和规则先验；
- 给新物品保留受控曝光，收集真实反馈。

### 3. 新系统或新领域冷启动

- 从相邻业务做迁移学习或参数初始化。
- 使用专家规则建立可用 Baseline。
- 利用公开数据、合成数据和人工标注启动。
- 先在小流量、Shadow 或灰度环境验证。
- 用主动学习优先标注模型最不确定的样本。

### 4. 常见系统方案

生产系统常用混合策略：

```text
规则 / 热门兜底
 + 内容模型
 + 协同过滤
 + 探索策略
```

数据逐渐积累后，再动态提高个性化模型的权重。

### 5. 风险与评测

- 探索不能破坏用户体验或安全边界。
- 不能只优化点击，应同时看留存、多样性和长期价值。
- 新旧对象要分桶评测，平均指标会掩盖冷启动问题。
- 对低置信结果要有回退策略。

**面试总结**：冷启动的本质是“没有交互数据时如何利用先验，并用最低风险快速获得反馈”。答案要同时覆盖先验、探索和兜底。
