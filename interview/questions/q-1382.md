# 手撕排序链表

## 答案

链表不支持随机访问，不适合直接套用数组快排。最常用的是归并排序：用快慢指针找中点，递归排序左右两段，再合并两个有序链表。

### Python 实现

```python
class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next


def sort_list(head):
    if not head or not head.next:
        return head

    # 找到左半段尾节点
    slow = head
    fast = head.next

    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next

    right = slow.next
    slow.next = None

    left = sort_list(head)
    right = sort_list(right)

    return merge(left, right)


def merge(a, b):
    dummy = ListNode()
    tail = dummy

    while a and b:
        if a.val <= b.val:
            tail.next = a
            a = a.next
        else:
            tail.next = b
            b = b.next
        tail = tail.next

    tail.next = a or b
    return dummy.next
```

### 复杂度

```text
时间复杂度：O(nlogn)
额外空间：O(logn)
```

额外空间来自递归栈。链表节点本身是原地重连，没有额外创建 n 个节点。

### 如何做到 O(1) 额外空间

可以使用自底向上的迭代归并：

1. 先统计链表长度；
2. 子链长度依次取 `1, 2, 4, 8...`；
3. 每轮把相邻两段有序链表合并；
4. 直到子链长度大于等于总长度。

迭代写法时间仍是 O(nlogn)，额外空间 O(1)，但代码更长。面试时间有限时先写递归版本，再说明可以改成 bottom-up。

### 易错点

- 找中点后忘记断链，会导致无限递归；
- 合并时丢失后继节点；
- 快慢指针起点不当，两个节点时无法正确切分；
- 忘记处理空链表和单节点。

### 面试总结

排序链表首选归并排序，因为链表切分和合并都能通过指针完成。递归版本 O(nlogn) 时间、O(logn) 栈空间；若严格要求 O(1) 额外空间，则使用自底向上的迭代归并。
