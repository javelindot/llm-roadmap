# Decoder 文本生成有哪几种方法

## 答案

Decoder 文本生成方法可分为确定性搜索、随机采样、受约束搜索和加速解码。

### 确定性搜索

- Greedy Search：每步选择概率最大 token，速度快，但容易陷入局部最优和重复。
- Beam Search：保留 $B$ 条累计分数最高的序列，适合翻译、语音识别等答案较确定的任务；需长度惩罚，开放式生成中可能缺少多样性。

### 随机采样

- Temperature：用 $z/T$ 调节分布，$T<1$ 更确定，$T>1$ 更多样。
- Top-k：只在概率最高的 $k$ 个 token 中采样。
- Top-p/Nucleus：选择累计概率达到 $p$ 的最小 token 集，候选数随上下文变化。
- Typical、min-p 等方法根据分布形状过滤候选。

实践中常组合 temperature 与 top-p，并使用 repetition penalty、frequency/presence penalty 或 n-gram 限制。采样参数应根据事实问答、代码或创作场景分别调节。

### 受约束与多样化生成

Constrained Beam Search、词法约束、JSON Schema、CFG、trie 或有限状态机可以保证输出满足格式和合法 ID。Diverse Beam Search、contrastive search 用于降低候选同质化。

### 加速方法

Speculative Decoding 由小 draft model 提议多个 token，再由目标模型并行验证，在不改变目标分布的条件下减少大模型 decode 次数。还有 Medusa、多 token prediction 等并行候选头方案。

无论采用哪种方式，最终都是基于 logits 选择离散 token，并循环直到 EOS、stop sequence 或长度上限。评价要同时看任务质量、多样性、事实性、延迟和可复现性。
