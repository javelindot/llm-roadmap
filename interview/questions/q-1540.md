# 推导 Contrastive Loss（CLIP）的梯度

## 答案

CLIP 在一个 batch 内把匹配图文对作为正样本，其他组合视为负样本，并对 image-to-text 与 text-to-image 两个方向做对称交叉熵。

### 1. 相似度

设归一化图像和文本表示为 `u_i、v_j`，温度为 `tau`：

```text
s_ij = u_i^T v_j / tau
```

图像到文本方向：

```text
p_ij = exp(s_ij) / sum_k exp(s_ik)
L_i2t = -(1/N) * sum_i log p_ii
```

文本到图像方向对每一列做 softmax：

```text
q_ij = exp(s_ij) / sum_k exp(s_kj)
L_t2i = -(1/N) * sum_j log q_jj
```

总损失：

```text
L = (L_i2t + L_t2i) / 2
```

### 2. 对 logits 的梯度

交叉熵加 softmax 的标准结果为：

```text
dL_i2t / ds_ij
= (p_ij - 1[i=j]) / N
```

反方向：

```text
dL_t2i / ds_ij
= (q_ij - 1[i=j]) / N
```

因此：

```text
g_ij = dL/ds_ij
= (p_ij + q_ij - 2*1[i=j]) / (2N)
```

对正样本 `i=j`，梯度通常为负，梯度下降会增大其相似度；对高概率负样本，梯度为正，会降低相似度。越“难”的负样本，推开力度越大。

### 3. 对归一化表示的梯度

因为：

```text
s_ij = u_i^T v_j / tau
```

所以：

```text
dL/du_i = (1/tau) * sum_j g_ij * v_j
dL/dv_j = (1/tau) * sum_i g_ij * u_i
```

温度越低，logits 和梯度尺度越大，分布更尖锐。

### 4. 对归一化前特征的梯度

若：

```text
u = z / ||z||
```

则：

```text
dL/dz
= (I - u*u^T) / ||z|| * dL/du
```

梯度被投影到单位球面的切空间，不会沿着 `u` 的径向任意改变归一化表示。

若使用可学习 `logit_scale a`：

```text
s_ij = exp(a) * u_i^T v_j
```

则：

```text
dL/da = sum_ij g_ij * s_ij
```

工程上常对 scale 做上限约束，防止温度过低导致数值和训练不稳定。

### 5. 分布式训练注意

CLIP 常把多卡特征 all-gather 后构造更大的 in-batch negatives。必须确保正样本在全局 logits 中的标签位置正确；是否让 all-gather 保留梯度，会影响其他 rank 特征接收的梯度路径。

### 6. 面试总结

CLIP 梯度的核心是两次 softmax 交叉熵：`预测概率 - one-hot`。它拉近匹配图文，按当前误判概率推远负样本，再通过相似度链式传到图像和文本编码器。
