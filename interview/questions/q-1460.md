# 在做对齐的时候，为什么 SFT 之后还要做 RLHF，只用 SFT 可以吗

## 答案

只用 SFT 可以完成很多任务，但 SFT 与 RLHF/偏好优化解决的问题不同。是否需要后训练取决于目标，不是所有项目都必须走完整 RLHF。

### 1. SFT 解决什么

SFT 使用专家示范做最大似然训练：

```text
L = -log p(reference response | prompt)
```

它擅长：

- 学会对话格式；
- 指令遵循；
- 领域术语；
- 工具 schema；
- 固定任务行为；
- 基础安全示范。

如果任务边界清晰、标准答案稳定、数据充足，只用 SFT 可能已经足够。

### 2. SFT 的局限

**只学习示范分布**

它让模型模仿给定答案，不能直接表达“回答 A 比 B 好多少”。

**序列级目标弱**

Token CE 不直接优化最终任务成功、用户偏好或工具执行结果。

**数据利用效率**

一个 prompt 的多个候选中，SFT 通常只使用参考答案，难充分利用排序反馈。

**探索能力**

SFT 不会主动探索新策略，复杂推理和 Agent 任务可能受限于示范质量。

### 3. RLHF/偏好优化的价值

- 使用 chosen/rejected 偏好；
- 优化序列级 reward；
- 改善帮助性、安全和风格；
- 对数学、代码、工具等使用可验证反馈；
- 通过采样发现优于初始示范的策略；
- 对多个目标做权衡。

PPO、GRPO 是在线 rollout 优化；DPO 等可直接使用离线偏好数据。

### 4. 后训练的代价

- 训练系统复杂；
- rollout 成本高；
- reward hacking；
- 模型可能过度拒答或模式塌缩；
- 需要 KL、entropy 和完整回归；
- 偏好标签可能有偏。

因此不能为了“流程完整”强行上 RLHF。

### 5. 决策方式

```text
Prompt/RAG baseline
-> SFT
-> 评测剩余问题
-> 若问题是偏好/序列决策/可验证探索，再用 DPO/RL
```

### 6. 跨境支付场景

工具格式和客服 SOP 可先用 SFT。若要优化复杂调查步骤、回答偏好和任务完成，可用 DPO/GRPO；资金、权限和合规硬规则仍放在系统门禁，不能期待 RL 学会后就取消。

### 7. 面试总结

SFT 学“应该怎么回答”，RLHF/偏好优化学“多个可行回答中哪个更好”并可直接优化结果。简单稳定任务只用 SFT 可以，复杂偏好、推理和 Agent 决策才更值得引入后训练。
