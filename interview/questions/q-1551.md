# Deepseek中蒸馏R1是什么？是否比从零RL训练更好？

## 答案

DeepSeek-R1 Distill 是把大型 R1 生成并筛选的推理样本用于微调较小的 Qwen/Llama 基座模型，让小模型学习 R1 的问题求解轨迹和答案。它主要是 sequence-level reasoning distillation，不是必须拿到教师全部 logits 的经典软标签蒸馏。

### 1. R1 蒸馏做了什么

DeepSeek-R1 报告公开了多个 `DeepSeek-R1-Distill-Qwen` 和 `DeepSeek-R1-Distill-Llama` 模型。典型流程是：

```text
题目
-> R1 生成推理与答案
-> 正确性/质量筛选
-> 构造 SFT 数据
-> 微调小型开源基座
```

学生学到的是教师表现出来的推理模式、格式和解题路径。最终能力同时取决于学生基座、数据覆盖、筛选器和训练配方。

### 2. 与从零 RL 比较

**蒸馏优势**

- 训练稳定，直接做 SFT；
- 不需要大规模在线 rollout；
- 不需要复杂 reward、reference 或 critic；
- 对小模型尤其高效；
- 教师已探索出的有效轨迹可快速迁移。

**蒸馏局限**

- 上限受教师样本与学生容量约束；
- 会复制教师错误、冗长和风格偏差；
- 只覆盖教师生成过的状态分布；
- 学会文本轨迹不等于获得同等搜索能力；
- 很难通过模仿主动发现超越教师的新策略。

**从零/冷启动 RL 优势**

- 可基于环境奖励探索新策略；
- 在可验证任务上可能发现教师示范之外的方法；
- 可直接优化最终任务 reward。

**RL 局限**

- rollout 成本高、训练不稳定；
- 小模型早期探索能力弱，奖励稀疏；
- 容易 reward hacking 或输出退化；
- 需要完整安全回归。

### 3. 是否一定更好

R1 报告中的实验表明，对若干较小基座，蒸馏强模型推理数据的效果优于直接在小模型上做同类 RL。但这不是普遍定理：

- verifier 很强、环境可探索时，RL 可能继续提高上限；
- 教师数据与目标领域不匹配时，蒸馏可能无效；
- 最常见的工程路线是高质量蒸馏/SFT 冷启动，再用 RL 做目标任务优化。

比较必须保持基座、计算预算、训练 token 和测试集可比。

### 4. 跨境支付场景

可以蒸馏调查 SOP、证据归纳和工具轨迹，但教师生成的数据必须由业务规则、沙箱执行和人工抽检验证。不能蒸馏教师虚构的订单状态或资金动作；高风险操作仍由权限和审批约束。

### 5. 面试总结

R1 Distill 用 R1 的高质量推理序列教小模型，成本低、稳定，特别适合冷启动；RL 更贵但有探索和突破示范上限的可能。现实中通常是“蒸馏/SFT 打底 + 可验证 RL 提升”，不是二选一。
