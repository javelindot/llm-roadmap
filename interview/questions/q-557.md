# 训练 LoRA 模型时，你是如何选择冻结层的？依据是什么？

## 答案

严格来说，标准 LoRA 会冻结全部基座参数，真正需要选择的是 **在哪些层、哪些线性模块插入 LoRA**，而不是逐层决定基座权重是否冻结。

### 1. 默认起点

对于 Decoder-only LLM，常见目标模块是：

- `q_proj`、`v_proj`：最小且常用的起点。
- `q/k/v/o_proj`：完整覆盖 Attention 投影。
- `gate/up/down_proj`：加入 FFN，增加容量但参数和显存更高。

### 2. 选择依据

#### 任务差异

- 风格、格式、轻量指令：Attention LoRA 往往足够。
- 领域知识、复杂推理或较大分布迁移：可加入 FFN。
- 分类任务：还可能训练 Classification Head。

#### 数据规模

数据少时注入范围过大容易过拟合；数据多且任务复杂时可以提高 Rank 或覆盖更多模块。

#### 资源预算

目标模块越多、Rank 越高，训练参数、优化器状态和通信成本越大。

#### 层敏感度

可通过梯度范数、Fisher 信息、激活差异或消融实验观察哪些层对任务更敏感。部分任务偏高层语义，领域术语或底层表征也可能需要更广覆盖。

### 3. 推荐实验顺序

```text
q/v
→ q/k/v/o
→ Attention + FFN
→ 调 Rank 与 Alpha
```

每一步比较验证集、通用能力回归、训练成本和过拟合情况。

### 4. 是否解冻原层

只有当 LoRA 容量明确不足、数据质量和规模足够时，才考虑解冻 Norm、顶部若干层或做全参微调，并使用更小学习率。

**面试总结**：选择依据应来自任务、数据、资源和消融结果，而不是固定说“最后几层最重要”。
