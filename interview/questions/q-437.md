# 🔥 DPO的原理，损失函数如何计算？

## 答案

**一句话**：DPO 把“带 KL 约束的奖励最大化”转化成一个直接优化偏好对的分类损失，不需要显式训练 Reward Model，也不需要 PPO 的 Critic 和在线采样循环。

### 1. 数据形式

每条数据是：

```text
(x, y_w, y_l)
```

- `x`：Prompt
- `y_w`：chosen / preferred answer
- `y_l`：rejected answer

### 2. 损失函数

```text
L_DPO = -E log σ(β [
  log πθ(y_w|x) - log πref(y_w|x)
  - log πθ(y_l|x) + log πref(y_l|x)
])
```

也可以写成：

```text
chosen_advantage =
  log πθ(y_w|x) - log πref(y_w|x)

rejected_advantage =
  log πθ(y_l|x) - log πref(y_l|x)

loss = -log sigmoid(
  β × (chosen_advantage - rejected_advantage)
)
```

### 3. 直观理解

DPO 不只是让 chosen 的概率变大，而是要求：

> 当前模型相对于参考模型，对 chosen 的提升，要大于对 rejected 的提升。

`πref` 通常是冻结的 SFT 模型，用于约束策略不要偏离原模型太远。序列概率一般由回答中各 Token 的 Log Probability 求和得到。

### 4. `β` 的含义

`β` 来自原始 KL 正则化目标，控制偏好优化与参考策略约束之间的尺度。理论上更大的 KL 系数意味着更强的参考约束，但实际训练效果还会与学习率、序列长度和数据 Margin 共同作用，不能简单理解为“β 越大更新越猛”。

### 5. 优缺点

优点：

- 不需要单独训练 Reward Model。
- 不需要 Critic，显存和工程复杂度更低。
- 使用离线偏好数据，训练通常比 PPO 稳定。

局限：

- 高度依赖偏好数据质量和覆盖范围。
- 容易学习长度、格式等伪偏好。
- 对多步决策和在线探索不如强化学习自然。

**面试总结**：DPO 的本质是用参考模型校准后的 chosen/rejected Log Probability 差，直接训练模型做偏好排序。
