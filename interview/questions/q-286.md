# 实体识别怎么做的？什么BERT模型？哪一年？

## 答案

**一句话**：实体识别一般可以用规则、词典、传统模型或 BERT 类预训练模型实现，项目里更常见的是 BERT + CRF 或大模型抽取结合规则校验。

### 1. 实体识别怎么做

实体识别的目标是从文本中抽取关键实体，比如人名、公司名、国家、金额、币种、订单号、银行卡号、错误码、时间等。

常见方案有：

- **规则/正则**：适合订单号、金额、币种、邮箱、手机号等格式稳定的实体。
- **词典匹配**：适合国家、通道名、银行名、产品名。
- **BERT + CRF**：适合上下文相关的实体识别，能理解语义。
- **大模型抽取**：适合复杂文本和弱结构化材料，但要做格式和事实校验。
- **混合方案**：规则保证确定性，模型处理复杂语义。

### 2. BERT 模型是什么

BERT 是 Google 在 2018 年提出的预训练语言模型，核心是 Transformer Encoder，通过 Masked Language Model 和 Next Sentence Prediction 预训练，学习双向上下文表示。

在实体识别里，常见做法是：

1. 文本输入 BERT。
2. BERT 输出每个 token 的上下文向量。
3. 接一个分类层或 CRF。
4. 输出 BIO 标签，比如 `B-ORG、I-ORG、B-LOC`。

CRF 的作用是约束标签序列更合理，比如 `I-ORG` 不应该随便出现在没有 `B-ORG` 的位置。

### 3. 项目注意点

- 中文分词和 token 对齐要处理好。
- 训练数据要标注一致。
- 业务实体最好有词典和规则兜底。
- 金额、币种、订单号这类实体不能只靠模型。
- 低置信度结果要人工复核或二次校验。

### 4. 结合跨境支付项目怎么说（面试口径）

> 在跨境支付场景里，实体识别很常见，比如从客服对话、KYC 材料、银行回执、通道错误信息里抽取订单号、金额、币种、国家、银行名、错误码、公司名、证件号。我的做法不会只依赖一种模型。
>
> 对订单号、金额、币种这种格式强的字段，我会用规则和正则优先，因为它们必须准确；对公司名、地址、银行名称、失败原因这类上下文相关实体，可以用 BERT + CRF 或大模型抽取；最后再用业务规则校验，比如币种是否合法、金额格式是否正确、国家是否在支持列表里。
>
> 如果面试官问 BERT 哪一年，我会回答 BERT 是 2018 年提出的，NER 里常见结构是 BERT + CRF，用 BERT 做上下文表示，用 CRF 做序列标签约束。
