# AMP 混合精度训练

## 答案

**一句话**：AMP（Automatic Mixed Precision）= **FP16/BF16 做前后向计算 + FP32 做主权重和优化器状态**，关键是用 loss scaling 防 FP16 下溢、master weights 保精度。

### 1. 为什么要混合精度
- FP16/BF16 占显存只是 FP32 的一半
- Tensor Core 上 FP16/BF16 算力是 FP32 的 2-8 倍
- 显存省、速度快、bandwidth 友好

### 2. 哪些用 FP16/BF16、哪些用 FP32
| 部分 | 精度 | 原因 |
|------|------|------|
| 前向激活 | FP16/BF16 | 占大头，省显存 |
| 反向梯度 | FP16/BF16 | 同上 |
| **Master Weights** | **FP32** | Adam 更新精度敏感 |
| **Optimizer State (m, v)** | **FP32** | 同上 |
| Loss 计算 | FP32 | 数值范围大 |
| Softmax / LayerNorm | FP32 内部 | 数值不稳 |

### 3. Loss Scaling（FP16 特有的麻烦）
- 问题：FP16 表示范围窄（最小 ~6e-5），小梯度直接变 0
- 解法：把 loss 乘以一个 scale（默认 65536）→ 梯度也跟着放大 → 反传不下溢 → 更新前再除回去
- **动态 loss scaling**：自动检测 overflow，是 inf/nan 就降低 scale，连续无溢出就提升

### 4. FP16 vs BF16（重要！）
| 维度 | FP16 | BF16 |
|------|------|------|
| 表示范围 | ±65504 | ±3.4e38（和 FP32 一样） |
| 精度 | 高（10 位尾数） | 低（7 位尾数） |
| Loss Scaling | 必需 | **不需要** |
| 硬件支持 | V100/A100/H100 | **A100/H100/TPU** 才有 |
| 训练稳定性 | 易 NaN | **稳得多** |
| 业界选择 | 推理多 | **训练首选** |

### 5. PyTorch AMP 用法
```python
from torch.cuda.amp import autocast, GradScaler
scaler = GradScaler()  # BF16 不用
for x, y in loader:
    with autocast(dtype=torch.bfloat16):  # 或 float16
        out = model(x)
        loss = criterion(out, y)
    scaler.scale(loss).backward()  # FP16 才需要 scaler
    scaler.step(optimizer)
    scaler.update()
```

### 6. 常见坑
- LayerNorm / RMSNorm 内部要 FP32 计算，不然 NaN
- 模型 forward 里手动 `.to(torch.float16)` 容易破坏 AMP 假设
- 混 FP16 和 BF16 的算子会爆错
- BF16 用 Adam 时学习率可能要调，精度差异

### 7. 一句话总结
**A100/H100 上一律 BF16**——免 loss scaling 麻烦、训练稳得多；**V100 没办法只能 FP16**，必须开 GradScaler。

## 结合项目谈（口语版 · 约 40 秒）

> AMP 是我们跨境支付训练的"默认设置"，没有任何疑问。具体是 FP16 还是 BF16 主要看卡。
>
> 我们集群是 8 卡 H100，**默认 BF16**——免 loss scaling 麻烦、训练稳得多。一开始有个同事用 V100 训客服模型用了 FP16，loss scaling 没调好，跑到第 5k 步梯度全 NaN，回滚浪费了几小时算力。后来集群升级到 A100/H100 之后我们全员切 BF16，再没出过 NaN。
>
> 一个细节坑：我们 SFT Qwen-14B 时发现 RMSNorm 在 BF16 下偶尔输出不稳，后来强制 RMSNorm 内部用 FP32 计算（HuggingFace 默认就是这么做的，但我们魔改过 forward 把这个改没了）。修回去之后 loss 曲线平滑很多。
>
> 推理侧我们部分模型用 **FP16 + Flash-Attention**，因为推理时 BF16 vs FP16 速度差不大，但显存优势 FP16 更好（无需 master weights）。
>
> 我对 AMP 的总结：**训练用 BF16（如果硬件支持），推理可以更激进**（INT8/AWQ）。所有 NaN 调过来调过去，第一招就是切 BF16，多数问题立刻消失。
