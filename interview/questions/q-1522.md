# 模型蒸馏主要的两种方式。硬标签和软标签。

## 答案

知识蒸馏中，硬标签只告诉学生最终目标是什么；软标签保留教师对各类别或 token 的完整概率分布，包含类别相似性和不确定性等“暗知识”。

### 1. 硬标签蒸馏

硬标签可以是：

- 原始人工标注；
- 教师模型的 argmax 类别；
- 教师生成的完整序列；
- 教师筛选后的伪标签。

学生使用普通交叉熵：

```text
L_hard = CE(y_hard, student_logits)
```

优点是实现简单、存储小，适用于 API 教师只返回文本的场景；缺点是丢失教师对其他候选的相对判断，并会把教师错误当成确定事实。

### 2. 软标签蒸馏

用温度 `T` 平滑教师与学生分布：

```text
p_teacher = softmax(z_teacher / T)
p_student = softmax(z_student / T)
```

蒸馏损失通常为：

```text
L_soft = T^2 * KL(p_teacher || p_student)
```

`T > 1` 会让分布更平滑，暴露非最大类别之间的关系；`T^2` 用于补偿温度造成的梯度缩放。

### 3. 联合目标

常见形式：

```text
L = alpha * L_hard + (1-alpha) * L_soft
```

既学习真实标签，又学习教师分布。还可加入中间层、Attention 或样本关系蒸馏。

### 4. 大模型中的对应关系

- Sequence-level KD：教师生成回答，学生做 SFT，属于硬序列标签；
- Logit KD：学生逐 token 拟合教师 logits，属于软标签；
- CoT distillation：学习教师推理轨迹，通常是硬文本轨迹；
- On-policy distillation：学生先生成，再由教师打分或提供分布，减少训练/推理分布差异。

教师和学生 tokenizer 不同会使 token-level logit 对齐困难，可改做序列级、分布投影或行为级蒸馏。

### 5. 跨境支付场景

教师生成的合规解释必须先经过规则和人工验证，不能把幻觉作为硬标签批量复制。金额、币种、国家和工具参数应单独做结构化校验。

### 6. 面试总结

硬标签便宜、简单但信息少；软标签保留教师分布信息但需要 logits、温度和更多存储。实际常把真实硬标签与软标签 KL 加权结合。
