# PDF扫描件支持吗？如果想要支持这种扫描件的话，有思路吗？

## 答案

**一句话**：PDF 扫描件可以支持，但不能按普通文本 PDF 解析，要走图片化、OCR、版面分析和质量控制流程。

### 核心回答
支持思路：
1. 判断 PDF 是否有文本层。
2. 扫描件转图片。
3. 图像预处理：去噪、纠偏、增强、旋转检测。
4. OCR 提取文字。
5. 版面分析：标题、段落、表格、图片区域。
6. 表格结构化。
7. 置信度低的内容标记或人工复核。
8. 保留页码、坐标、原图链接，方便追溯。

难点：OCR 错字、表格错位、印章/水印干扰、低清晰度和多语言。

### 1. 扫描件和文本 PDF 的区别

文本型 PDF 里面本身有文字层，可以直接抽文本；扫描件本质是图片，需要 OCR。扫描件还可能存在歪斜、模糊、阴影、水印、印章遮挡等问题。

### 2. 生产处理要加质量控制

支持扫描件不能只做 OCR，还要：

- OCR 置信度。
- 图像清晰度检测。
- 旋转和倾斜校正。
- 关键字段校验。
- 低置信转人工。
- 原图留存。

### 3. 面试总结

扫描件支持的关键是 OCR + 版面分析 + 置信度 + 可追溯，而不是简单"能识别文字"。

### 4. 结合跨境支付项目怎么说（面试口径）

> 支付里扫描件很常见，比如营业执照、银行证明、合同扫描件。如果要支持，我会先做 OCR 和版面分析，而不是直接丢给文本解析器。
>
> 关键是保留原图、页码和 OCR 置信度，低置信度字段不能直接进入风控结论。比如公司名、证件号或银行账号识别错了，后续审核会很危险。
>
> 所以这类字段要和人工或外部系统校验。RAG 可以辅助查阅和总结，但不能把低置信 OCR 当成事实。

**钩子**：扫描件 RAG 的核心是 OCR 质量和可追溯，不只是能不能抽出文字。
