# 6. 后面确实问了PPO的优化目标。。。

## 答案

PPO 的目标是在利用优势函数提高策略收益的同时，限制新策略相对旧策略的单次更新幅度，避免策略因一次高方差更新发生崩溃。

### 1. Clipped Surrogate Objective

定义新旧策略概率比：

```text
r_t(θ) =
π_θ(a_t | s_t)
────────────────
π_θ_old(a_t | s_t)
```

PPO-Clip 的核心目标为：

```text
L_clip(θ) =
E_t[min(
  r_t(θ) A_t,
  clip(r_t(θ), 1-ε, 1+ε) A_t
)]
```

其中：

- `A_t` 是优势函数，表示动作相对基线有多好；
- `ε` 是裁剪范围，常见起点约为 0.1 到 0.2，但需按任务调节；
- 当更新让概率比超出区间时，clip 会截断继续增益，抑制过大的策略更新。

当 `A_t > 0` 时希望提高该动作概率，但最多奖励到 `1+ε`；当 `A_t < 0` 时希望降低概率，但最多下降到 `1-ε` 附近。

### 2. 完整训练目标

实际 PPO 通常同时优化策略、价值函数和熵：

```text
L_total =
- L_clip
+ c_v L_value
- c_e H(π)
```

- `L_value` 训练 Value Model 估计回报；
- `H(π)` 是策略熵，鼓励探索并防止过早塌缩；
- 优势函数通常用 GAE 估计，降低方差并在偏差与方差间折中。

还会监控 empirical KL、clip fraction、value loss 和 explained variance。如果 KL 过大，可提前停止当前 epoch 或调整学习率。

### 3. RLHF 中的 PPO

在语言模型对齐中：

- `state` 是 prompt 和已经生成的 token；
- `action` 是下一个 token；
- 策略模型生成完整 response；
- Reward Model 对 response 给出序列级奖励；
- Value Model 估计 token 级价值；
- Reference Model 用于限制策略远离 SFT 模型。

常见奖励形态是：

```text
R_total =
R_reward_model
- β × KL(π_policy || π_reference)
```

序列末尾加入 Reward Model 分数，每个 token 位置加入 KL 惩罚，再计算 return 和 advantage。PPO-Clip 限制相对旧策略的更新，Reference KL 限制相对初始对齐策略的漂移，两者作用不同。

### 4. 为什么 PPO 容易不稳定

- Reward Model 可被策略钻漏洞；
- 序列级稀疏奖励的 credit assignment 困难；
- Advantage、reward scale 和 value 初始化敏感；
- 多轮采样和多 epoch 更新成本高；
- KL 太小会限制学习，太大则语言能力和安全性漂移；
- 长回复通过长度投机获得更高奖励；
- 训练数据分布外的 prompt 容易退化。

工程上需要做 reward normalization、advantage whitening、梯度裁剪、KL 控制、长度归一化和持续人工评测。

### 5. PPO 与 DPO 的区别

PPO 是在线策略优化，需要策略采样、Reward Model、Value Model 和 Reference Model，系统复杂但可以直接优化可计算奖励。

DPO 使用偏好对 `(chosen, rejected)` 直接优化策略与参考策略的相对概率，不需要在线 rollout 和 Value Model，训练更简单稳定。但 DPO 的能力受离线偏好数据覆盖限制，也不能自动解决奖励设计问题。

### 6. 跨境支付中的使用边界

PPO 或 DPO 可以优化客服回答的帮助性、引用风格和拒答策略，但不能把资金合规规则交给偏好模型学习。付款、退款、账户解冻和风控审批仍需确定性权限、规则、幂等和人工审核。

对支付客服的奖励还要包含 faithfulness、引用正确、敏感信息保护和错误承诺惩罚，不能只优化用户点赞或对话时长。
