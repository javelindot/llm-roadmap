# Transformer经典架构

## 答案

**一句话**：Transformer 经典架构由 Encoder 和 Decoder 组成，核心是自注意力机制、位置编码、前馈网络、残差连接和 LayerNorm。

### 1. Transformer 解决什么问题

传统 RNN 按序处理文本，长距离依赖和并行训练能力较差。Transformer 用 Attention 直接建模任意 token 之间的关系，因此更适合并行训练和捕捉长距离依赖。

### 2. 核心组件

- **Embedding**：把 token 转成向量。
- **Position Encoding**：补充位置信息，因为 Attention 本身不感知顺序。
- **Multi-Head Self-Attention**：从多个子空间学习 token 之间关系。
- **Feed Forward Network**：对每个位置做非线性变换。
- **Residual Connection**：缓解深层网络训练问题。
- **LayerNorm**：稳定训练。
- **Encoder-Decoder Attention**：Decoder 在生成时关注 Encoder 输出。

### 3. Encoder 和 Decoder

Encoder 适合理解任务，比如分类、检索、实体识别。BERT 就是典型 Encoder 架构。

Decoder 适合生成任务，比如 GPT 系列。它使用 causal mask，保证生成当前 token 时看不到未来 token。

Encoder-Decoder 适合翻译、摘要等输入输出转换任务，比如 T5。

### 4. Attention 直观理解

Attention 可以理解为：每个 token 根据 Query 去和其他 token 的 Key 计算相关性，再加权汇总 Value。这样模型能判断当前词应该重点关注上下文里的哪些词。

### 5. 结合跨境支付项目怎么说（面试口径）

> 如果结合跨境支付业务，Transformer 的 Encoder 可以用于实体识别和文本分类，比如从银行回执里识别错误码、币种、国家、收款人信息；Decoder 模型可以用于客服回答生成和异常解释；Encoder-Decoder 可以用于文档摘要或多语言翻译。
>
> 比如用户问“这笔 USD payout 为什么失败”，模型需要注意 USD、payout、失败原因、错误码、国家、通道这些词之间的关系。Self-Attention 的价值就在于能捕捉这种上下文关联，而不是只看某个关键词。
>
