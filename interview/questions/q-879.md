# 生成式推荐相较于传统推荐方法的优势是什么

## 答案

生成式推荐将 item 或 Semantic ID 视为 token，直接生成下一物品或推荐列表，而传统方法通常先召回候选，再逐项打分。

### 主要优势

- **统一任务**：召回、排序、解释和多场景可使用一个序列建模目标；
- **减少级联错位**：避免前级截断后正确 item 永久丢失，并可直接优化列表；
- **利用序列上下文**：自然建模用户行为顺序、session 和多步兴趣；
- **语义迁移**：Semantic ID 或内容 token 能改善新品、长尾和跨域泛化；
- **开放交互**：可结合自然语言需求、对话推荐和工具；
- **Scaling 潜力**：可借鉴语言模型的预训练、后训练和推理时搜索。

P5、TIGER、HSTU、OneRec 分别代表任务统一、Semantic ID 生成、规模化序列转导和召回排序统一等方向。

### 相比传统推荐的代价

- item 空间巨大，解码、beam 和无效 ID 约束困难；
- 新 item 加入与 codebook 更新复杂；
- 自回归列表延迟高于并行打分；
- 曝光偏差、热门偏差和 reward hacking 仍存在；
- 库存、广告和多样性规则需外部约束；
- 解释与生成结果更难做确定性归因。

因此工业系统常采用混合路线：生成式模型做语义召回、用户表征或离线 teacher，传统轻量排序承接高 QPS。

### 面试总结

生成式推荐的优势是统一建模、列表级优化和语义泛化，尤其适合序列、冷启动和交互场景；短板是大空间解码、实时更新、延迟和可控性，短期更可能与传统级联共存。
