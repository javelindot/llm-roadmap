# 怎么实现NL2SQL的？

## 答案

**一句话**：NL2SQL 的核心是让模型理解自然语言问题、表结构、指标口径和权限约束，生成安全可执行的 SQL，再对结果进行解释。

### 1. 实现流程

典型流程是：

1. 用户输入自然语言问题。
2. 识别时间、指标、维度、过滤条件。
3. 检索数据字典、表结构、指标定义。
4. 选择相关表和字段。
5. 生成 SQL。
6. 做 SQL 安全校验和权限过滤。
7. 执行 SQL。
8. 对结果生成解释或图表。

### 2. 关键模块

- **Schema Linking**：把用户说法映射到表字段。
- **指标口径库**：定义成功率、拒付率、到账时效等指标。
- **SQL 生成**：模型根据问题和 schema 生成 SQL。
- **SQL 校验**：禁止危险语句，检查字段、表、权限。
- **结果解释**：把查询结果转成业务可理解语言。

### 3. 难点

- 用户问题模糊。
- 指标口径复杂。
- 表关系多。
- 权限控制严格。
- SQL 幻觉字段。
- 查询性能和成本。

### 4. 结合跨境支付项目怎么说（面试口径）

> 在跨境支付问数项目里，NL2SQL 可以让业务同学直接问“上周巴西付款失败率是多少，主要失败原因是什么”。系统要识别时间是上周，国家是巴西，指标是付款失败率，维度是失败原因，然后去指标库找失败率口径，再关联交易表、通道回执表或错误码表生成 SQL。
>
> 这里最关键的是指标口径和权限。比如付款成功率到底按订单数、金额还是最终状态统计，必须提前定义，不能让模型自己猜。商户数据也要做行级权限过滤。
>
> 我会在生成 SQL 前检索表结构和指标定义，生成后再经过 SQL Parser 校验，只允许 select，限制扫描范围，最后再执行。这样能降低模型生成错误 SQL 的风险。

**钩子**：可以引到“Schema Linking、指标口径、SQL 安全校验、支付数据权限”。
