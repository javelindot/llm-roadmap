# ppo算法为什么有reward model 又有critic model

## 答案

Reward Model 和 Critic/Value Model 都输出标量，但回答的是不同问题。

### Reward Model

Reward Model 近似人类偏好：

\[
r_\phi(x,y)
\]

它通常在完整回答结束后给出序列级奖励，由 chosen/rejected 偏好对训练。它回答的是：“这条回答在帮助性、安全性或正确性上有多好？”

### Critic Model

Critic 估计当前策略从某个中间状态继续生成的期望累计回报：

\[
V_\psi(s_t)\approx
\mathbb E_{\pi_\theta}[G_t\mid s_t]
\]

它作为 baseline 计算 advantage：

\[
A_t=G_t-V_\psi(s_t)
\]

PPO 用 \(A_t\) 判断某个 token 动作比当前策略的平均预期好多少，从而降低 Policy Gradient 方差，并把终局 reward 分配到各生成步骤。

### 为什么不能只用 Reward Model

- RM 通常只对完整序列打分，不提供每个 token 状态的价值；
- 直接把同一个终局 reward 分配给所有 token，方差高且信用分配粗糙；
- critic 随当前 policy 的 rollout 训练，RM 则应相对独立地表示外部偏好；
- KL penalty、格式奖励等还会与 RM reward 组合成最终 return，critic 要估计的是这个总回报。

GRPO 等算法可以用同一 prompt 的多条回答组内相对分数替代独立 critic，从而省显存，但会增加 rollout 需求，并不意味着 reward 也被省掉。

### 跨境支付场景

客服 Agent 的 RM 可评价最终回答和任务成功，critic 估计每一步查询或工具选择的长期价值。资金操作仍应由可验证规则和审批约束，不能把 RM 当作安全边界。

### 面试总结

Reward Model 定义“目标好不好”，Critic 估计“当前状态未来能拿多少回报”。PPO 用 reward 构造 return，用 critic 形成低方差 advantage；二者职责不同，GRPO 只是用组内基线替代 critic。
