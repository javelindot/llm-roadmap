# 分类任务怎么处理类别不平衡

## 答案

**一句话**：类别不平衡治理四件套——**采样（过采/降采/SMOTE）、损失加权（Focal Loss / Class Weight）、数据增强、阈值调整**——金融风控这种极端不平衡场景往往要组合用。

### 1. 采样层
**过采样（Oversampling）**
- 复制少数类
- 简单但易过拟合
- SMOTE：插值生成新少数类样本
- ADASYN：自适应 SMOTE

**降采样（Undersampling）**
- 丢弃多数类
- 信息损失但训练快
- Random / Tomek Links / Cluster Centroid

**混合采样**
- 过采少数 + 降采多数
- 平衡训练分布

### 2. 损失加权
**Class Weight**
```python
loss = CrossEntropyLoss(weight=[1.0, 10.0])  # 少数类权重高
```

**Focal Loss**
$$L_{focal} = -(1-p_t)^\gamma \log p_t$$
- 容易样本权重低、难样本权重高
- γ 通常 2.0
- 经典用于 dense detection、长尾分类

### 3. 数据增强
- 文本：EDA、回译、LLM 改写
- 图像：MixUp、CutMix、AutoAug
- 表格：SMOTE、CTGAN

### 4. 阈值调整
- 模型默认 0.5 阈值
- 不平衡时调到符合业务的阈值
- Precision-Recall 曲线找最优点
- ROC 曲线找 Youden 指数

### 5. 评估指标改用
- 不能只看 Accuracy（90% 不平衡时全猜多数类 acc 90%）
- 用 **F1 / Precision / Recall**
- **AUC-PR**（不平衡场景比 AUC-ROC 更敏感）
- **Confusion Matrix**

### 6. 高级方法
- **Anomaly Detection**：把少数类当异常
- **One-class SVM**
- **Cost-Sensitive Learning**：每个误分类都有不同代价
- **Ensemble**：多个平衡子集训子模型集成

### 7. LLM 时代的类别不平衡
- LLM 通用能力强，少数类靠 few-shot prompt 可救
- SFT 时少数类样本可重复
- DPO 时长尾偏好用 reject sampling 增强

### 8. 工程经验
- 类别比 1:10 内 → class weight 够
- 1:10 到 1:100 → 加 SMOTE / 数据增强
- 1:100 到 1:1000 → 极端方法 + Anomaly Detection
- > 1:1000 → 规则 + 弱监督

### 9. 一句话总结
**类别不平衡治理 = 采样 + 加权 + 增强 + 阈值**——四件套按不平衡程度组合，金融风控这种 1:1000 场景往往要全上。

## 结合项目谈（口语版 · 约 30 秒）

> 类别不平衡在我们跨境支付里**是日常**，分享几个实战案例。
>
> **案例一：可疑交易检测（1:1000）**。正样本（确认洗钱）只有 5k 条，负样本（正常交易）500w 条。我们做了：
> - 负样本下采样到 5w（按时间均匀采样保多样性）
> - 正样本 SMOTE + LLM 改写扩到 2w
> - Focal Loss γ=2
> - 阈值从 0.5 调到 0.3（业务要 recall）
> - 指标看 PR AUC + recall@95% precision
>
> 最终 recall 提升 22 个点。
>
> **案例二：客服意图分类（1:50）**。有些罕见意图样本少。用 Qwen-7B + LoRA 微调，关键是**用 prompt few-shot 给少数类多个样例**——LLM 时代的不平衡治理多了 in-context learning 这条路。
>
> **案例三：商户违约预测（1:30）**。传统 ML（XGBoost）+ class_weight + AUC-PR 评估。这种场景没必要 LLM。
>
> **几个工业经验**：
> - **不要盲目 SMOTE**——SMOTE 在文本任务上效果不如 LLM 改写
> - **阈值调整是 ROI 最高的一步**——很多场景不用动数据，只调阈值就能拿到大半收益
> - **业务定 Precision 还是 Recall 优先**——风控要 Recall（漏一笔风险大）、广告推荐要 Precision（推错代价高）
> - **AUC-PR 比 AUC-ROC 在极不平衡场景更有意义**
>
> 一句话：**类别不平衡是工程问题不是算法问题**——四件套组合 + 阈值调整 + 选对评估指标，比单点神技重要。
