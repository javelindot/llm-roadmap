# T5 的结构和预训练目标是什么？

## 答案

**一句话**：T5 = **Encoder-Decoder Transformer + "Text-to-Text" 范式 + Span Corruption**——把所有 NLP 任务统一成"文本输入 → 文本输出"，预训练通过遮蔽连续 span 来学。

### 1. 结构（Encoder + Decoder）
- **Encoder**：双向 attention（无 mask）
- **Decoder**：causal attention（看不到未来）
- Decoder 通过 cross-attention 看 Encoder 输出
- 相比 GPT 多一个 Encoder

### 2. Text-to-Text 范式
- 输入任务前缀 + 数据 → 输出文本
- "translate English to German: Hello" → "Hallo"
- "summarize: ..." → "summary"
- 分类、回归、生成全部统一

### 3. 预训练目标：Span Corruption
```
原文：The cat sat on the mat
遮蔽：The cat <X> on the <Y>
目标：<X> sat <Y> mat <Z>
```
- 连续遮蔽多个 token（不像 BERT 单 token mask）
- 用 sentinel token (`<X>`, `<Y>`) 标记位置
- 目标是恢复被遮蔽 span

### 4. T5 的特殊设计
- **Relative Position Bias**：相对位置编码（非 sinusoidal）
- Pre-LN
- 全用 FC（无 bias）
- 共享 input/output embedding

### 5. T5 vs BERT vs GPT
| 维度 | T5 | BERT | GPT |
|------|-----|------|-----|
| 结构 | Encoder-Decoder | Encoder-only | Decoder-only |
| Attention | 双向+因果 | 双向 | 因果 |
| 预训练 | Span corruption | MLM + NSP | Next-token |
| 适合 | seq2seq、翻译、QA | 理解 | 生成 |

### 6. T5 的优势
- 任务统一：所有任务同套接口
- Encoder 看双向信息，理解强
- 翻译、摘要、问答很合适
- 中等规模下效果不错

### 7. T5 的劣势（被 GPT 风格替代的原因）
- 参数效率低（Encoder + Decoder 都要）
- Scaling 不如 decoder-only
- 推理两路计算
- 大规模 LLM 时代被边缘化

### 8. T5 后续
- T5.1.1、Flan-T5（指令微调）
- mT5（多语言）
- UL2 / PaLM 受 T5 启发
- 但 ChatGPT 后 decoder-only 完全主导

### 9. 仍然有用 T5 的场景
- 短输入 + 短输出（翻译、改写）
- 资源有限（中等规模 T5 比 GPT 风格更可控）
- 学术研究

### 10. 一句话总结
**T5 = Encoder-Decoder + Text-to-Text + Span Corruption**——曾是统一 NLP 的优雅尝试，但 decoder-only LLM 时代把它边缘化。

## 结合项目谈（口语版 · 约 20 秒）

> T5 在我们跨境支付里**早期用过、现在基本不用了**。
>
> **早期用 T5 的场景**：客服话术多语言翻译（中→英→中等等）、订单状态短摘要、政策文档关键信息抽取。这些都是"短输入 → 短输出"的 seq2seq 任务，**T5 在中等规模（base/large）下比同尺寸 GPT 风格性价比高**。
>
> **后来换掉的原因**：
> - 业务需求复杂化——客服需要对话式回复，T5 的 Encoder-Decoder 在多轮对话里不自然
> - 一个模型多任务——decoder-only 的 instruction-following 比 T5 的 task prefix 灵活
> - 算力上来后——直接 Qwen-14B 一个模型干所有事，T5 系列没必要单独维护
>
> 现在只在两个地方还有 T5 痕迹：
>
> **1. 短文本嵌入**：bge、e5 这些 embedding 模型有 T5 style 也有 BERT style，看场景。
>
> **2. mT5 / Flan-T5 多语言翻译**：东南亚小语种翻译我们试过，比通用 LLM 翻译更稳定（因为专为这任务训过）。
>
> 一句话：**T5 是 GPT 时代之前最优雅的统一方案**，工业仍有边缘用途，但主线 decoder-only 已经接管。
