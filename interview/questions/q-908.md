# 大模型的优缺点及个人体验

## 答案

大模型的价值在通用语义、生成和少样本迁移，但其输出是概率生成，不能直接视为数据库或确定性程序。

### 优点

- 一个模型覆盖问答、抽取、分类、代码和多语言；
- zero/few-shot 快速验证新任务；
- 能处理非结构化文本、图像、音频和复杂自然语言；
- 通过 RAG、工具和 Agent 进入业务流程；
- 可用 SFT、LoRA、DPO 和蒸馏适配；
- 交互门槛低，能显著提高知识和开发效率。

### 缺点

- 幻觉、知识过时和不确定性表达不足；
- prompt 敏感，长上下文不代表能可靠使用；
- 成本、延迟、KV 显存和并发压力；
- 训练数据偏见、版权、隐私和提示注入；
- 复杂 Agent 容易循环、工具错误和状态漂移；
- 评测容易污染，离线 benchmark 与业务不一致；
- 难以解释单次输出的完整因果依据。

### 个人体验参考

> 我的体验是，大模型最适合把模糊自然语言转换成候选方案、代码草稿或工具计划，但关键事实仍需检索、执行器和测试验证。我在[真实项目]中发现，效果瓶颈经常来自数据、检索和流程，而不只是模型大小。因此我会先建立基线和评测，再增加 RAG、工具或微调。

跨境支付尤其不能让模型自行决定余额、费率和放行；它适合理解、解释和辅助规划，确定性系统负责事实与动作。

### 面试总结

大模型强在通用迁移和自然交互，弱在事实可靠、确定性、成本和安全。成熟用法是把模型放进有证据、工具、验证和回滚的系统，而不是单独信任生成结果。
