# deepseek r1的训练流程 和r0 的训练流程的区别。

## 答案

题目中的 “R0” 通常指 DeepSeek-R1-Zero。二者都从 DeepSeek-V3-Base 出发使用强化学习激发推理，但 R1-Zero 不经过人工推理 SFT 冷启动，R1 增加冷启动、两阶段 RL 和两阶段 SFT，使可读性、语言一致性和通用能力更好。

### 1. DeepSeek-R1-Zero

简化流程：

```text
DeepSeek-V3-Base
-> 大规模 GRPO/RL
-> DeepSeek-R1-Zero
```

Reward 主要使用可验证的正确性和格式信号。它展示了不依赖 SFT 也能通过 RL 出现自反思、验证和更长推理等行为。

问题包括：

- 可读性较差；
- 语言混杂；
- 格式和表达不稳定；
- 通用问答与写作不够均衡。

### 2. DeepSeek-R1

官方报告中的主要阶段可概括为：

```text
Base
-> Cold-Start SFT
-> Reasoning-oriented RL
-> Rejection Sampling + SFT
-> Secondary RL for all-scenario alignment
-> R1
```

**Cold-Start SFT**

用少量高质量长 CoT 数据建立可读、稳定的初始策略。

**Reasoning RL**

在数学、代码、逻辑等可验证任务上继续提高推理。

**Rejection Sampling + SFT**

从 RL checkpoint 采样并筛选正确推理数据，同时混合写作、问答等通用 SFT 数据。

**Second RL**

兼顾推理正确、帮助性和安全性，改善各种场景表现。

### 3. 核心区别

| 维度 | R1-Zero | R1 |
|---|---|---|
| 冷启动 SFT | 无 | 有 |
| 主体方法 | 直接 RL | SFT + RL + SFT + RL |
| 推理涌现 | 强 | 强且更稳定 |
| 可读性 | 较差 | 更好 |
| 语言混杂 | 更明显 | 缓解 |
| 通用对齐 | 较弱 | 更完整 |

### 4. 蒸馏

DeepSeek 还使用 R1 生成的数据微调较小的 Qwen/Llama 基座，得到 Distill 模型。这不是把大模型权重直接压缩，而是用强模型推理数据做监督蒸馏。

### 5. 面试总结

R1-Zero 的关键实验是“Base 直接 GRPO/RL 也能涌现推理”；R1 则用 Cold-Start SFT、推理 RL、拒绝采样 SFT 和第二阶段 RL，把推理能力转化为更可读、更通用、更安全的模型。
