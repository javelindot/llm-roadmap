# 什么是拒绝采样？具体流程？

## 答案

**一句话**：拒绝采样是先生成多个候选结果，再用规则、模型或人工打分筛掉差的，保留质量更高的样本或答案。

### 核心流程
1. 对同一个 prompt 生成多个候选回答。
2. 用 reward model、规则、判别器或人工评估打分。
3. 拒绝低分样本，保留高分样本。
4. 高分样本可以直接返回，也可以作为 SFT/DPO 数据。

优点：
- 简单有效，不需要直接改模型参数。
- 可以提升数据质量。
- 适合构造偏好数据和蒸馏数据。

缺点：
- 生成多个候选成本高。
- 评分器有偏差会影响筛选。
- 只筛选不训练时，线上成本仍然较高。

### 1. 拒绝采样在对齐里的作用

拒绝采样经常用于构造高质量数据。模型先生成多个候选，评分器或人工选择更好的结果。被选中的可以作为 SFT 数据，未被选中的可以作为 DPO 的 rejected。

它的关键不是"多生成几个就完事"，而是要有可靠的筛选标准。

### 2. 筛选标准怎么设计

业务里可以从几个维度打分：

- 事实是否正确。
- 是否引用了正确证据。
- 格式是否符合要求。
- 是否过度拒绝或过度放行。
- 是否符合合规边界。
- 处理建议是否可执行。

如果评分器本身有偏差，拒绝采样会把偏差放大。

### 3. 面试总结

拒绝采样是简单有效的数据提质方法，适合构造 SFT/DPO 数据，但效果取决于评分标准和人工质检。

### 4. 结合跨境支付项目怎么说（面试口径）

> 在支付业务里，拒绝采样很适合做高质量训练数据。比如同一个商户审核问题，让模型生成 5 个审核意见，再用规则和人工挑出证据完整、合规边界清楚、话术合适的版本。
>
> 好的版本可以作为 SFT 的标准答案，差的版本可以作为 DPO 的 rejected。比如一个回答只说"风险高，建议拒绝"，另一个回答说明了国家风险、交易频次异常、缺少 KYB 材料和建议补充项，后者明显更适合 chosen。
>
> 这个方法的价值不是多玄学，而是把运营和风控专家的判断沉淀到训练样本里。关键是评分标准要明确，否则模型可能学到"越长越好"这种伪偏好。

**钩子**：拒绝采样可以把业务专家经验转成高质量 SFT/DPO 数据。
