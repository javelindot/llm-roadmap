# 算法题 mid链表题（1-2-3-4-5-6的链表变成1-6-2-5-3-4）

## 答案

这是经典“重排链表”：

```text
L0 -> L1 -> L2 -> ... -> Ln
变成
L0 -> Ln -> L1 -> Ln-1 -> L2 -> Ln-2 -> ...
```

例如：

```text
1 -> 2 -> 3 -> 4 -> 5 -> 6
1 -> 6 -> 2 -> 5 -> 3 -> 4
```

可以分三步完成：

1. 快慢指针找中点；
2. 反转后半段；
3. 前后两段交替合并。

### Python 实现

```python
class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next


def reorder_list(head):
    if not head or not head.next:
        return head

    # 1. 找前半段尾节点
    slow = fast = head
    while fast.next and fast.next.next:
        slow = slow.next
        fast = fast.next.next

    # 2. 断开并反转后半段
    second = slow.next
    slow.next = None

    prev = None
    cur = second
    while cur:
        nxt = cur.next
        cur.next = prev
        prev = cur
        cur = nxt
    second = prev

    # 3. 交替合并
    first = head
    while second:
        first_next = first.next
        second_next = second.next

        first.next = second
        second.next = first_next

        first = first_next
        second = second_next

    return head
```

### 过程示例

```text
原链表：1 -> 2 -> 3 -> 4 -> 5 -> 6

切分：
1 -> 2 -> 3
4 -> 5 -> 6

反转后半段：
6 -> 5 -> 4

交替合并：
1 -> 6 -> 2 -> 5 -> 3 -> 4
```

### 复杂度

```text
时间复杂度：O(n)
额外空间：O(1)
```

### 面试总结

这题是链表组合题，核心不是一次想出所有指针，而是拆成“找中点、反转、合并”三个熟悉模块。要先断开前后半段，避免合并时形成环。
