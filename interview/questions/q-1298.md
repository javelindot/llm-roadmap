# 拷打实习项目，每个环节的参数选择、一个transformer decoder的输出来回问了有十分钟，压力拉满了

## 答案

项目中的每个参数都应对应“业务目标、资源约束、实验结果”三类依据。Transformer Decoder 的输出也要从张量维度、训练目标和推理状态解释，而不是只说“输出下一个 token”。

### 1. 参数选择框架

以跨境支付 RAG/Agent 为例：

**切片参数**

- chunk size：由文档结构、Embedding 窗口和证据完整性决定；
- overlap：防止跨边界语义丢失，但过大会增加重复召回；
- 通过 Context Recall、NDCG、最终答案和成本消融。

**检索参数**

- Dense/BM25 Top-K；
- metadata filter；
- rerank 候选数；
- 最终送入 LLM 的证据数。

Top-K 太小会漏证据，太大会引入噪声和上下文成本。需要按 FAQ、长文档、错误码等场景分桶。

**生成参数**

- temperature：事实问答通常低温；
- max tokens：按答案结构和延迟预算；
- context length：按检索证据和并发显存；
- 模型尺寸：按任务难度、P99 和单位成本路由。

**训练参数**

- learning rate、warmup、batch；
- sequence length、packing；
- LoRA rank/alpha/dropout；
- epoch、weight decay 和 gradient clipping。

先用小样本检查数据和 loss，再做单变量消融，最后在冻结黄金集回归。参数不能只引用论文默认值。

### 2. Decoder 输入

对 decoder-only 模型，token ids：

```text
input_ids: [B, S]
```

Embedding 后：

```text
X_0: [B, S, H]
```

其中 `B` 是 batch，`S` 是序列长度，`H` 是 hidden size。

### 3. 一层 Decoder Block

以 Pre-Norm 结构为例：

```text
U = RMSNorm(X)
Q = U W_Q
K = U W_K
V = U W_V
```

MHA 中常见形状：

```text
Q: [B, N_q, S, D]
K: [B, N_kv, S, D]
V: [B, N_kv, S, D]
```

`D = H / N_q`。GQA 中 `N_kv < N_q`，多个 Query head 共享 KV head。

注意力：

```text
A = softmax(Q K^T / sqrt(D) + causal_mask)
O = A V
X_attn = X + Concat(O) W_O
```

再经过：

```text
Z = RMSNorm(X_attn)
F = (SiLU(Z W_gate) * (Z W_up)) W_down
X_next = X_attn + F
```

每层输出仍是 `[B, S, H]`，不是词表概率。它是下一层的上下文化 hidden states。

### 4. 最后一层输出

经过最终 Norm 和 LM Head：

```text
hidden: [B, S, H]
logits: [B, S, Vocab]
```

`logits[b,s,:]` 是位置 `s` 的词表未归一化分数。Softmax 后才是概率：

```text
p(token | prefix) = softmax(logits)
```

训练时 labels 通常右移一位：

```text
logits[:, 0:S-1] predicts labels[:, 1:S]
```

对 assistant-only SFT，Prompt 和 padding 位置设为 ignore index，不参与 loss。

### 5. 训练与推理的差别

**训练**

- Teacher forcing；
- 整段序列并行计算；
- causal mask 防止看到未来；
- 所有有效位置计算交叉熵；
- 反向传播保存 activation。

**Prefill**

- 并行处理整个 Prompt；
- 建立每层 K/V Cache；
- 通常主要使用最后一个有效位置 logits 生成首 token。

**Decode**

- 每步输入一个新 token；
- Query 长度为 1；
- 历史 K/V 从缓存读取；
- 输出 `[B, 1, Vocab]` logits；
- 经过 temperature、top-k/top-p 等采样得到下一个 token。

### 6. 常见追查点

- hidden size 是否能被 head 数整除；
- GQA 的 Q/KV head 映射；
- RoPE 作用于 Q/K，而不是 V；
- causal mask 与 padding mask；
- logits 不是概率；
- LM Head 是否与 embedding 权重共享；
- KV Cache 缓存每层 K/V，不缓存 Attention 权重；
- 训练全序列并行，生成仍逐 token 串行。

跨境支付输出中的金额、币种和订单状态即使 token 概率很高，也必须由工具和业务校验确认，模型置信度不能替代事实来源。
