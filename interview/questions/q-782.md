# 为啥分类任务用交叉熵，不用MSE?

## 答案

分类通常使用交叉熵，因为它与 Bernoulli/Categorical 分布的最大似然一致，并且配合 Sigmoid/Softmax 时梯度更适合分类。

多分类交叉熵为：

\[
\mathcal L=-\sum_k y_k\log p_k
\]

Softmax 与 CE 合并后，对 logit 的梯度是：

\[
\frac{\partial\mathcal L}{\partial z_k}=p_k-y_k
\]

即使模型高置信地预测错误，梯度仍能有效纠正。

MSE 更自然地对应带高斯噪声的连续值回归。若在 Softmax/Sigmoid 输出后使用 MSE，梯度还要乘激活函数 Jacobian，预测接近 0 或 1 时容易变小，通常收敛更慢。

MSE 并非绝对不能用于分类，它可用于 logit/表示蒸馏或特殊软标签目标；但普通单标签分类默认使用 CE，多标签分类使用逐标签 BCE。

跨境支付风险分类还需要类别权重、Focal Loss 或采样处理不平衡，并在校准后根据误杀与漏放成本设置阈值。损失更低不代表业务阈值自动最优。

### 面试总结

CE 来自分类分布的最大似然，梯度直接为 \(p-y\)；MSE 更适合连续回归，接 Sigmoid/Softmax 后可能因饱和使梯度变小。因此分类默认选 CE/BCE，但仍需单独处理不平衡和概率校准。
