# 有没有微调 rerank 模型/应该怎么微调

## 答案

Reranker 微调的核心不是堆积普通正负样本，而是让模型学会区分“语义相近但业务条件不适用”的候选。跨境支付知识库中，同一段规则可能只在国家、币种、产品、客户类型或生效版本上不同，这类困难负样本决定了微调是否真正有效。

### 1. 明确模型输入和目标

常用 Reranker 是 Cross-Encoder，将 `query` 和 `document` 拼成一对共同编码，输出相关性分数。由于每个候选都要重新计算，它通常只重排召回阶段的 Top-N，而不直接扫描全库。

训练样本可以表示为：

```text
pointwise: (query, document, relevance_label)
pairwise:  (query, positive_document, negative_document)
listwise:  (query, candidate_list, relevance_distribution)
```

- Pointwise 实现简单，可使用 BCE 或回归损失；
- Pairwise 直接学习正例应排在负例之前，适合 margin ranking、RankNet 类目标；
- Listwise 更接近最终排序任务，但数据构造和训练成本更高。

初期可先用 pointwise 或 pairwise 建立可靠基线，再依据数据量和收益决定是否上 listwise，不必一开始追求复杂损失。

### 2. 数据质量比模型尺寸更重要

正样本主要来自业务专家标注、已解决工单、人工确认的引用关系和高质量搜索日志。点击数据只能作为弱监督，因为点击会受到曝光位置、客服习惯和旧系统排序的影响。

负样本要分层构造：

- **随机负样本**：用于学习基本主题边界，但训练价值有限；
- **BM25 困难负样本**：关键词高度重合，实际答案不匹配；
- **Dense 困难负样本**：语义接近，但国家、币种、产品或流程不同；
- **版本负样本**：旧费率、旧限额和已失效通道规则；
- **对抗负样本**：只差否定词、数值、时间、客户类型或例外条件；
- **跨语言负样本**：语言不同但业务对象相似，防止模型只靠表面语言判断。

例如查询“巴西 BRL 本地付款支持个人收款人吗”，相关规则和“支持企业收款人”“墨西哥 MXN 支持个人收款人”“历史版本曾支持个人收款人”都很相似，后面三类正是高价值困难负样本。

挖掘困难负样本后必须排除 false negative。一个问题可能同时由正文、FAQ 和操作手册正确回答，不能因为只标了一个正例，就把其他有效证据当作负例训练。

### 3. 防止数据泄漏与标签污染

训练集、验证集和测试集不能只随机拆行。相同文档切出的相邻 chunk、同一工单的改写问题、同一规则的多语言版本如果跨集合分布，会造成虚高指标。更稳妥的拆分方式是：

- 按文档或规则组隔离；
- 按商户会话和工单隔离；
- 使用时间切分检验对新规则的泛化；
- 单独保留长尾国家、低频产品和新版本作为挑战集。

标注规范要明确“完全相关、部分相关、不相关、冲突或过期”等等级，并抽样计算标注一致性。涉及法规、资金状态和风控说明的数据应由业务专家复核。

### 4. 训练时处理长文档与多语言

Cross-Encoder 有最大输入长度，直接截断可能丢掉结论或例外条款。输入中应优先保留：

> 标题路径 + 适用范围 + 命中段落 + 必要的上下文。

对超长文档先切成结构化 chunk，再做 chunk 级排序；必要时采用“chunk 排序 → 父段聚合”的两级方案。训练批次要平衡语言、国家和业务线，避免高频中文 FAQ 淹没英文规则和长尾市场。

还可以用高质量教师模型生成软标签或 pairwise 偏好，再蒸馏到较小的 Reranker，但教师结果必须经过黄金集校准，不能把教师幻觉直接固化进模型。

### 5. 评测要覆盖排序质量和线上成本

离线至少比较：

- NDCG@K、MRR、MAP、Precision@K；
- 正确证据进入生成上下文的比例；
- 过期规则、错误国家或错误币种被排进 Top-K 的比例；
- 不同语言、国家、产品和难度分桶指标；
- P50/P95/P99、吞吐、显存和单请求成本。

还应做端到端评测：答案 faithfulness、引用正确率、问题解决率、人工改写率和错误承诺率。Reranker 的 NDCG 上升，如果没有改善最终证据质量或业务指标，就不能证明值得上线。

部署时限制重排候选数，采用 batch inference、FP16/BF16、量化或 ONNX/OpenVINO 等优化，并设置超时降级；超时时退回融合召回排序。模型、训练数据、召回器和索引版本要一起记录，因为召回分布变化后，原有 Reranker 的效果可能发生漂移。
