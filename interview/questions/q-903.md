# 长上下文压缩有哪些方法？

## 答案

长上下文压缩的目标是减少输入 token 或 KV 状态，同时保留当前任务需要的信息。应按“文本、检索、表示、KV”分层。

### 1. 文本级

- 规则去除模板、重复、日志噪声；
- extractive compression 保留关键原句；
- abstractive summary 生成摘要；
- 分层摘要：段落→章节→文档；
- 对话保留近期原文，旧消息压成事实、决策和待办；
- LLMLingua 类方法按 token/phrase 重要性压缩 prompt。

摘要可能引入事实错误，需保留来源、版本和可回溯原文。

### 2. 检索级

- 先路由到相关文档/章节；
- parent-child retrieval；
- hybrid recall + rerank；
- query-aware chunk selection；
- 去重、MMR 和 context packing；
- 多跳任务逐步检索，而非一次塞全库。

### 3. 表示级

- Memory token/Compressive Transformer；
- Autoencoder/latent compression；
- Perceiver Resampler、Q-Former 等把大量视觉/文本 token 压为少量 latent；
- 层次模型先局部编码再全局聚合。

### 4. Attention 与 KV

- Sliding Window、local/global sparse Attention；
- GQA/MQA、MLA 减少 KV 维度；
- KV Cache quantization、token eviction、importance-based retention；
- prefix sharing 和 paged cache 减少重复/碎片，但不压缩有效语义；
- recurrent memory 或 SSM 以固定状态承载历史。

### 5. 评估

比较压缩率、任务准确率、关键信息召回、位置鲁棒性、幻觉、TTFT、KV 和成本。跨境支付中金额、订单状态和规则版本不能被摘要丢失，应结构化保存并实时查询。

### 面试总结

长上下文压缩可在原文、检索、latent 和 KV 四层进行。生产系统通常使用“相关检索 + 分层摘要 + 结构化状态 + KV 优化”，并用任务准确率而非压缩率单独决策。
