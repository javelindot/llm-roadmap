# 5.为什么sft和rl在post-train中可能需要轮着来

## 答案

SFT 和 RL 在 post-training 中轮换，本质是利用两者不同的能力：SFT 用高质量示范提供稳定的行为先验，RL 根据奖励探索并优化超出示范上限的策略。RL 产生的新轨迹又可以筛选成下一轮 SFT 数据，形成“示范初始化、强化探索、数据蒸馏、再强化”的闭环。

### 1. SFT 解决什么问题

SFT 通过 teacher-forcing 学习：

- 指令格式和对话习惯；
- 基础推理范式；
- 工具调用 schema；
- 安全拒答和风格；
- 任务冷启动。

它稳定、样本效率高，但上限受示范数据约束。若数据只给出一种解法，模型很难通过纯模仿主动发现更优策略。

### 2. RL 解决什么问题

RL 可以基于可验证奖励进行探索，例如：

- 数学和代码答案是否正确；
- 工具调用是否成功；
- 结构化输出是否通过 schema；
- 任务是否完成；
- 安全规则是否满足。

它能从多个候选轨迹中强化更优行为，但也可能出现：

- reward hacking；
- 输出变得冗长或模板化；
- 可读性、语言一致性下降；
- 遗忘通用能力；
- 策略偏离训练分布；
- 奖励模型偏差被放大。

### 3. 为什么要再回到 SFT

RL 后筛选高奖励且质量合格的轨迹，加入 SFT：

- 将探索得到的新能力蒸馏成稳定策略；
- 修复语言混杂、格式和可读性；
- 混入通用、安全和多领域数据，缓解遗忘；
- 让下一轮 RL 从更好的初始策略开始；
- 把昂贵的采样结果转化为可重复训练数据。

循环可以表示为：

```text
冷启动 SFT
-> 可验证奖励 RL
-> 生成多条轨迹
-> 规则与人工筛选
-> rejection sampling
-> 新一轮 SFT
-> 综合目标 RL
```

DeepSeek-R1 公布的训练流程就包含 cold-start SFT、推理导向 RL、rejection sampling 加 SFT，以及后续覆盖更多场景的 RL，说明交替不是简单重复，而是每个阶段处理不同缺陷。

### 4. 为什么不能只做连续 RL

连续 RL 会不断优化当前 reward。只要 reward 不完整，策略就可能牺牲未被奖励覆盖的能力。SFT 数据相当于行为锚点，可以恢复：

- 人类可读性；
- 指令遵循；
- 通用知识表达；
- 多语言；
- 安全和边界行为。

反过来，如果只反复 SFT，模型只是模仿筛选后的答案，缺少在线探索和 credit assignment，也可能停留在已有数据上限。

### 5. 每轮如何验收

每个阶段都保留独立评测集，比较：

- 任务正确率和 pass@k；
- reward 与真实业务指标的一致性；
- 推理长度和单位任务成本；
- 格式、工具成功率；
- 语言一致性和可读性；
- 安全、越权和 reward hacking；
- 通用能力回退。

还应保存 reference policy、数据版本、reward 版本和 checkpoint，出现回退可以定位是哪一轮引入。

### 6. 跨境支付中的边界

RL 适合在沙箱或离线环境优化：

- 工单分类；
- 检索与工具选择；
- 有确定答案的规则问答；
- JSON 参数生成；
- 测试账户上的流程完成率。

真实付款、换汇、账户冻结或风控处置不能让策略自由探索。资金动作必须由确定性权限、限额、审批、规则和审计控制；RL 最多优化建议或调用计划，执行前仍需工具鉴权和业务校验。

SFT 与 RL 也不是必须固定轮换。若奖励不可验证、探索风险高或高质量示范已经足够，DPO、拒绝采样 SFT 或纯 SFT 可能更合适。阶段设计取决于任务、数据、reward 可信度和安全边界。
