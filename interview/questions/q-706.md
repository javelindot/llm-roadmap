# 在微调 Qwen 的时候，数据是怎么构造的，有用到什么数据清洗方法吗，数据配比是怎么做的

## 答案

微调 Qwen 的数据构造应围绕目标能力，而不是因为模型品牌不同就使用特殊方法。回答项目题时应给真实数据来源、数量、配比与消融结果，不能编造。

### 样本格式

统一为 Qwen 对应 tokenizer/chat template 的多轮消息格式，明确 system、user、assistant 和 tool 消息。只对需要学习的 assistant token 计算 loss，正确处理 EOS、截断和多轮边界。工具调用使用稳定 JSON schema。

### 清洗方法

- 规则清洗：空文本、乱码、重复标点、长度异常、截断和格式不合法。
- 精确与近重复：hash、MinHash/SimHash、embedding 聚类，避免 train/test 泄漏。
- 内容质量：事实核验、代码/SQL 执行、答案相关性与完整性评分。
- 安全合规：PII、版权、敏感信息和越权内容过滤。
- 模型辅助审核后抽样人工复核，避免 judge 偏差直接污染训练集。

对多轮数据检查角色交替和上下文依赖；对合成数据保留生成模型、prompt 和审核版本。

### 数据配比

按通用指令、目标领域、困难样本、拒答安全、工具调用和长上下文建立能力桶。先从通用数据占较高比例的基线开始，逐步提高目标领域权重，并限制模板化合成数据。可按数据量和质量加权采样：

$$
P(D_i)\propto n_i^\alpha w_i
$$

使用分桶验证集做消融，观察目标能力收益、通用能力退化和过拟合。若领域知识变化频繁，优先 RAG 而不是把所有事实都写入参数。最终配比应由质量曲线和业务权重决定，不存在固定最佳比例。
