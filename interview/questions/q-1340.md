# 微调使用的是什么框架？

## 答案

框架要按训练类型分层回答，并结合自己的真实经历。不能只报“用了 Transformers”，还要说明数据、并行、PEFT、监控和推理验证如何串起来。

### 1. 通用 SFT/PEFT

常见组合：

```text
PyTorch
+ Hugging Face Transformers
+ Datasets
+ PEFT
+ TRL
+ Accelerate
```

- Transformers：模型、Tokenizer、Trainer；
- PEFT：LoRA、QLoRA、DoRA 等；
- TRL：SFT、DPO、PPO/GRPO 类训练器；
- Accelerate：设备和分布式启动。

### 2. 训练平台

- LLaMA-Factory：配置化 SFT/PEFT/DPO，适合快速实验；
- ms-swift：对 Qwen/多模态和 PEFT 支持较完整；
- Axolotl：YAML 配置和多种训练配方；
- Unsloth：偏单机高效 LoRA/QLoRA；
- 自研 Trainer：复杂采样、loss 和多任务时使用。

框架能力和版本变化快，最终要读对应模型与框架的官方兼容矩阵。

### 3. 分布式与大模型训练

- DeepSpeed ZeRO；
- PyTorch FSDP/FSDP2；
- Megatron-LM/Core；
- Tensor/Sequence/Pipeline/Expert Parallel；
- FlashAttention、gradient checkpoint 和混合精度。

### 4. 对齐与 RL

- TRL；
- verl；
- OpenRLHF；
- NeMo-RL 等。

还需要 rollout engine（如 vLLM/SGLang）、Reward/Verifier、权重同步和可恢复的分布式状态。

### 5. 推荐的项目回答

> SFT/LoRA 我使用 Transformers + PEFT，数据用 Datasets 和自定义 chat template；单机通过 Accelerate，规模扩大后用 DeepSpeed ZeRO/FSDP。训练过程记录有效 token、loss、grad norm 和评测集。RL 阶段将 rollout 与训练分开，使用推理引擎采样，规则 verifier 打分。训练完成后用同一推理模板在 vLLM 上做回归，避免训练/部署模板不一致。

其中框架名要替换为自己的实际方案。

### 6. 选型原则

- 先看目标模型是否原生支持；
- 再看全参/LoRA、单机/多机、文本/多模态；
- 是否需要自定义 loss 和 rollout；
- checkpoint 能否恢复；
- 日志、评测和部署格式是否兼容；
- 团队是否能维护。

跨境支付训练还需要脱敏、数据权限、审计、版本和删除机制，框架方便不等于数据可以直接拿来训练。
