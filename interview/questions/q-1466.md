# ROPE在t维度是如何实现的？

## 答案

题目中的 `t` 可能指普通序列的 token 位置，也可能指视频多模态中的时间维度。面试时应先确认。两种场景本质都是根据坐标生成旋转角度，再作用于 Q/K 的二维通道对。

### 1. 标准 RoPE 中的序列位置 t

对位置 `t` 和第 `i` 个二维通道对，定义频率：

```text
theta_i = base^(-2i/d)
angle(t, i) = t * theta_i
```

将向量的两个维度旋转：

```text
[x'_(2i)  ]   [ cos(angle)  -sin(angle) ] [x_(2i)  ]
[x'_(2i+1)] = [ sin(angle)   cos(angle) ] [x_(2i+1)]
```

Q 和 K 使用同一位置旋转。它们的点积依赖相对位置：

```text
R(t_q)^T R(t_k) = R(t_k - t_q)
```

因此注意力自然包含相对距离信息。

### 2. 简化代码

```python
def apply_rope(x, cos, sin):
    # x: [batch, heads, seq, dim]
    x_even = x[..., 0::2]
    x_odd = x[..., 1::2]

    y_even = x_even * cos - x_odd * sin
    y_odd = x_even * sin + x_odd * cos

    return stack_interleaved(y_even, y_odd)
```

`cos/sin` 由位置 `t` 与各维频率预计算，并广播到 batch/head。

### 3. 视频中的时间维度 t

多模态 RoPE/mRoPE 会将位置拆成：

```text
temporal t
height h
width w
```

隐藏维的一部分通道对使用 `t` 旋转，另一些使用 `h/w`。同一视频帧内的 patch 共享或按设计映射到同一时间坐标，不同帧的 `t` 递增。

若视频以动态 FPS 采样，时间坐标最好反映真实时间间隔，而不是只用帧编号，否则不同帧率视频的运动速度语义不一致。

### 4. 长序列扩展

当 `t` 超过训练长度时，旋转相位分布发生变化。常见扩展包括 Position Interpolation、NTK-aware scaling、YaRN 和长上下文继续训练。只扩大 `max_position_embeddings` 不够。

### 5. 面试总结

标准 RoPE 在位置 `t` 上用 `t*theta_i` 对 Q/K 的二维通道做旋转，使点积依赖相对位置差；视频 mRoPE 则把一部分通道分配给时间轴 `t`，其余编码高和宽。
