# 什么是RNN？

## 答案

**一句话**：RNN 是一种处理序列数据的神经网络，会把前一时刻的隐藏状态传到下一时刻，从而建模时间或顺序依赖。

### 核心回答
RNN 的基本形式：

```text
h_t = f(Wx_t + Uh_{t-1} + b)
```

特点：
- 适合文本、语音、时间序列等顺序数据。
- 参数在不同时间步共享。
- 隐藏状态保存历史信息。

缺点：
- 难以并行。
- 长序列容易梯度消失或爆炸。
- 长距离依赖建模能力有限。

后来 LSTM/GRU 缓解长期依赖问题，Transformer 又进一步替代了很多 RNN 场景。

### 1. RNN 如何处理序列

RNN 每个时间步接收当前输入 `x_t` 和上一步隐藏状态 `h_{t-1}`，输出当前隐藏状态 `h_t`。这个隐藏状态可以理解成对历史信息的压缩。

因此它适合处理有顺序关系的数据，比如文本、语音、用户行为序列、交易时间序列。

### 2. RNN 的局限

- 计算必须按时间步顺序进行，并行性差。
- 长序列中早期信息容易被遗忘。
- 反向传播跨很多步，容易梯度消失或爆炸。

LSTM/GRU 用门控缓解这个问题，Transformer 则用自注意力直接建模 token 之间的关系。

### 3. 面试总结

RNN 的核心是隐藏状态递归传递，适合序列建模；缺点是长依赖和并行效率不如 Transformer。

### 4. 结合跨境支付项目怎么说（面试口径）

> 在支付业务里，RNN 可以用于交易序列建模，比如一个商户最近连续交易的金额、国家、失败率、退款率变化。它能把历史行为压到隐藏状态里，再辅助判断当前交易风险。
>
> 但现在如果做复杂风控，我更倾向用 Transformer、时序特征模型，或者直接构造历史统计特征，因为并行性和长依赖更好。
>
> 不过理解 RNN 仍然有价值，它说明了序列建模的基本思想：当前判断不能只看单笔交易，还要看商户或用户的历史行为轨迹。

**钩子**：风控不是单点判断，交易序列行为很关键。
