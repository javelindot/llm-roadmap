# 什么是大模型辅助召回？为什么需要？

## 答案

**一句话**：大模型辅助召回是用 LLM 对用户问题进行改写、扩展、拆解或生成假设答案，从而提升检索命中率。

### 核心回答
为什么需要：
- 用户问题口语化，和文档表达不一致。
- 一个问题可能包含多个子意图。
- 关键词缺失导致向量或 BM25 都召回不准。
- 业务别名、缩写、同义词很多。

常见做法：
- Query rewrite：改写成更标准的检索语句。
- Query expansion：扩展同义词、业务术语。
- Multi-query：生成多个角度问题。
- HyDE：先生成假设答案，再用答案向量检索。
- 意图识别后按知识库或元数据过滤。

### 1. 为什么原始 query 往往不好搜

用户提问经常不标准：

- 口语化："款怎么还没到？"
- 省略实体：没说国家、币种、通道。
- 多意图混杂：既问原因又问处理方案。
- 业务别名多：同一个通道或产品有多个叫法。

直接拿原始问题检索，召回容易偏。

### 2. LLM 可以怎么辅助

LLM 可以先做：

- 标准化问题。
- 抽取实体，如国家、币种、通道、错误码。
- 扩展同义词和业务别名。
- 拆分多个子问题。
- 判断应该查哪个知识库或工具。

但它只能辅助召回，不能替代证据。

### 3. 面试总结

大模型辅助召回的价值是把用户自然语言转成更适合检索的查询表达，提高召回率和路由准确率。

### 4. 结合跨境支付项目怎么说（面试口径）

> 支付里同一个问题可能有很多说法，比如"款没到""入账失败""通道没回执"可能都在问收款状态。大模型辅助召回可以先把用户口语改写成标准业务查询，再补充国家、币种、通道、场景这些关键词。
>
> 比如用户说"巴西这笔没到账"，模型可以改写成"巴西 BRL 本地收款入账延迟原因、通道回执状态、清算周期"，再去查对应知识库和订单工具。
>
> 但我会控制它只能改写查询，不能自己编规则。最终答案必须基于检索证据或工具返回结果，这样才适合支付场景。

**钩子**：LLM 辅助召回是提升检索，不是替代知识库证据。
