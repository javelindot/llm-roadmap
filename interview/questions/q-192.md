# 除了RAG以外，还学另外的一些AI相关的技术吗？

## 答案

**一句话**：除了 RAG，我还会关注 Agent、Function Calling/MCP、模型微调、评估体系、多模态和本地部署这些和业务落地直接相关的技术。

### 核心回答
可以按方向回答：
- **Agent**：任务规划、工具调用、记忆、多智能体协作。
- **Function Calling / MCP**：把模型接入内部系统和工具。
- **SFT/LoRA/DPO**：让模型适配垂直领域表达和偏好。
- **评估体系**：RAGAS、LLM-as-judge、人工评审、线上指标。
- **多模态**：OCR、CLIP、证照图片理解。
- **部署优化**：量化、vLLM、KV Cache、私有化部署。
- **安全治理**：权限、审计、提示注入防护、敏感信息脱敏。

重点是说明这些技术如何服务业务，而不是报技术名。

### 1. 怎么回答更自然

这个问题不要只罗列技术名，而要把技术和业务场景挂起来。可以说自己围绕大模型落地关注三层：

- 知识层：RAG、检索、评估。
- 行动层：Agent、Function Calling、MCP、工具权限。
- 模型层：SFT、LoRA、DPO、蒸馏、本地部署。

### 2. 展现能力的重点

面试官更想听你是否理解端到端落地：

- 数据怎么接。
- 工具怎么调。
- 权限怎么控。
- 效果怎么评估。
- 失败怎么回滚。
- 成本怎么优化。

### 3. 面试总结

除了 RAG，还要能讲 Agent、微调、评估、多模态和部署安全，体现你不是只会调知识库。

### 4. 结合跨境支付项目怎么说（面试口径）

> 除了 RAG，我比较关注 Agent 和模型对齐。因为跨境支付很多问题不是单纯查知识库，比如一笔款没到账，需要查订单、查通道、查账务、看风控状态，这就需要 Agent 调工具。
>
> 还有风控解释和商户审核话术，单靠 RAG 不一定够，需要 SFT 或 DPO 让模型更符合业务表达。比如同样是拒绝商户，好的回答要说明证据、规则和补材料建议，而不是一句"风险过高"。
>
> 同时支付数据敏感，所以本地部署、权限控制、审计日志、脱敏和提示注入防护也很重要。我的学习重点都是围绕真实落地，而不是只停留在模型调用。

**钩子**：RAG 解决知识，Agent 解决流程，微调解决表达和偏好，安全治理负责上线边界。
