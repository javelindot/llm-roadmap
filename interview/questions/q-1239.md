# 对于一个 base model 如何增强大模型的 reasoning 能力

## 答案

增强 Base Model 的 reasoning 能力应采用分阶段方案，而不是直接收集一批长思维链做 SFT。完整路径是：

> 基础能力诊断 → 高质量继续预训练 → reasoning cold-start SFT → 可验证奖励训练 → 蒸馏与拒绝采样 → 推理时搜索 → 全面回归。

### 1. 先建立基线

按数学、代码、逻辑、长上下文、工具调用和目标业务分桶，区分失败原因：

- 缺知识或术语；
- 不会分解问题；
- 中间计算错误；
- 不会调用工具；
- 有正确路径但解码未选中；
- 输出格式或安全边界错误。

不同原因需要不同训练，不应只看一个综合 benchmark。

### 2. 继续预训练

加入高质量数学、代码、科学、结构化数据和领域材料，保留题目、证明、程序与文档结构。严格去重和检查评测污染。继续预训练增强底层表示与知识，但不会自动得到稳定的指令跟随。

### 3. Cold-start SFT

使用少量高质量 reasoning 示例教会模型：

- 理解指令与输出格式；
- 分解和验证；
- 使用计算器、代码或检索工具；
- 在信息不足时澄清；
- 最终答案简洁、可读。

数据要包含正确短路径、错误修正和不同难度，不能全是冗长模板化 CoT。

### 4. 可验证奖励强化学习

在有可靠 verifier 的任务上使用 PPO、GRPO 等方法：

- 数学最终答案；
- 代码隐藏单测；
- SQL 执行结果；
- JSON Schema 和工具参数；
- 规则与状态机约束。

Reward 需分开记录结果、格式、安全和成本。训练在沙箱中进行，并防止修改测试、硬编码答案、重复调用和接口漏洞等 reward hacking。

### 5. Process supervision 与 verifier

最终答案 reward 稀疏时，可以训练过程奖励模型或 verifier 评价中间步骤。但过程标注昂贵，也可能把一种解法错误固化为唯一标准，因此应与 outcome reward 和人工抽检结合。

### 6. 蒸馏与拒绝采样

由强模型或 RL 模型为简单问题生成多条候选，使用执行器和 verifier 过滤，再蒸馏给较小模型。必须保留题目来源、去重、难度和通过原因，避免把教师幻觉直接复制。

### 7. 推理时扩展

- self-consistency；
- best-of-N + verifier；
- 搜索与回溯；
- 动态思考预算；
- 代码、检索和业务工具。

它能提升有多条候选路径的任务，但会增加延迟和成本，需要按难度路由。

### 8. 跨境支付应用

支付模型可以学习拆分“查交易事实、查规则、生成解释”，但订单金额、币种换算、账务和权限由工具验证。奖励可来自工具成功、字段一致性、引用和合规门禁，不能直接用投诉率或资金结果做单一 reward。

最终评测同时看准确率、pass@k、工具成功率、校准、幻觉、安全、输出长度和单位成功任务成本，避免 reasoning 分数上升却损害通用表达和生产稳定性。
