# deepspeed原理，如何实现zero1，2，3

## 答案

DeepSpeed 是大模型训练系统，ZeRO 的核心是消除 Data Parallel 中每张 GPU 重复保存完整训练状态的问题。它逐级分片优化器状态、梯度和参数，以通信换显存。

### 1. 训练显存组成

训练显存主要包括：

- 模型参数；
- 梯度；
- 优化器状态，例如 Adam 的一阶、二阶矩和可能的 FP32 master weights；
- 激活、Attention 中间量和临时 buffer。

普通 Data Parallel 中，每张卡都有完整参数、梯度和优化器状态。设数据并行度为 `N`，ZeRO 将可分片状态约降为每卡 `1/N`。

### 2. ZeRO-1

分片 Optimizer States：

- 参数仍在每张卡完整复制；
- 梯度仍在每张卡完整保存；
- 每张卡只更新自己负责的参数分片对应的优化器状态；
- 更新后的参数通过通信保持各卡一致。

Adam 状态通常占用较大，因此 Stage 1 已能明显节省显存，通信模式接近普通数据并行。

### 3. ZeRO-2

在 Stage 1 基础上再分片 Gradients：

- 反向传播后通过 Reduce-Scatter 将梯度归约并分到负责的 rank；
- 每张卡只保存和更新自己的梯度及优化器分片；
- 模型参数仍然完整复制。

适合模型参数能放入单卡，但完整梯度和优化器状态放不下的场景。它通常是全量微调中较容易取得显存与吞吐平衡的方案。

### 4. ZeRO-3

继续分片 Parameters：

- 静态状态中每张卡只持有部分参数；
- 前向计算某个模块前 All-Gather 所需参数；
- 计算后按策略释放或保留；
- 反向再次获取参数，并 Reduce-Scatter 梯度；
- 每张卡只更新自己的参数分片。

Stage 3 显存最省，但参数获取更频繁，对通信带宽、bucket、prefetch 和模块粒度更敏感。超大模型或单卡放不下参数时才体现必要性。

### 5. 三个 Stage 对比

| Stage | 分片内容 | 参数是否每卡完整 | 典型代价 |
|---|---|---|---|
| ZeRO-1 | Optimizer | 是 | 状态管理与同步 |
| ZeRO-2 | Optimizer + Gradient | 是 | Reduce-Scatter |
| ZeRO-3 | Optimizer + Gradient + Parameter | 否 | 频繁参数 All-Gather |

ZeRO 不会自动解决激活显存。长序列仍需要 Gradient Checkpointing、FlashAttention 或 Context Parallel。

### 6. DeepSpeed 配置

核心配置示例：

```json
{
  "bf16": {"enabled": true},
  "gradient_accumulation_steps": 8,
  "zero_optimization": {
    "stage": 3,
    "contiguous_gradients": true,
    "overlap_comm": true,
    "reduce_scatter": true,
    "stage3_prefetch_bucket_size": 50000000,
    "stage3_param_persistence_threshold": 100000
  }
}
```

将 `stage` 改为 1 或 2 即启用对应阶段；Stage 3 的 bucket 与 persistence 参数应通过模型和网络压测确定，不能盲目复制固定值。

训练初始化通常由 DeepSpeed 包装模型、优化器、DataLoader 和 scheduler。Stage 3 在超大模型初始化时还可使用 `deepspeed.zero.Init`，避免先在单卡构造完整权重。

### 7. Offload

可以进一步配置：

- `offload_optimizer`：把优化器状态和计算放到 CPU；
- `offload_param`：Stage 3 下把参数分片放 CPU/NVMe；
- ZeRO-Infinity：扩展到 NVMe 层级。

Offload 用 PCIe/NVMe 传输换 GPU 显存，必须测吞吐和尾延迟。若 CPU 内存、NUMA、pin memory 或 NVMe 带宽不足，训练可能显著变慢。

### 8. 选型和排障

- 参数能放单卡，主要缺优化器内存：ZeRO-1；
- 参数能放单卡，需要更大 batch：ZeRO-2；
- 参数本身放不下：ZeRO-3；
- 只有少量领域数据：先评估 LoRA/QLoRA；
- 跨节点带宽弱：谨慎使用通信频繁的 Stage 3；
- OOM 先用显存快照区分参数、状态还是激活；
- 对比 tokens/s、MFU、通信占比和 checkpoint 时间；
- 验证分片 checkpoint 的保存、合并和恢复。
