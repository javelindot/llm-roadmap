# 介绍transformer，编码器解码器的作用。介绍一下损失函数，介绍qlora，说说bert以及roberta和deberta区别和联系，介绍一下bert模型都有什么任务

## 答案

### 1. Transformer 与 Encoder/Decoder

Transformer 使用 Multi-Head Attention、FFN、残差连接、归一化和位置编码建模序列。

- **Encoder**：双向读取完整输入，输出每个 token 的上下文化表示，适合理解、分类、抽取和重排；
- **Decoder**：Self-Attention 使用 causal mask，只依赖已生成前缀；原始 Encoder-Decoder 中还通过 Cross-Attention 读取 Encoder 输出；
- **Encoder-only**：BERT、RoBERTa、DeBERTa；
- **Decoder-only**：GPT、LLaMA、Qwen；
- **Encoder-Decoder**：原始 Transformer、T5 等。

注意力核心计算：

```text
Q = X * W_Q
K = X * W_K
V = X * W_V
Attention(Q, K, V) = softmax(Q * K^T / sqrt(d_k) + Mask) * V
```

### 2. 常见损失函数

**多分类交叉熵**

```text
L_CE = -Σ_c y_c * log p_c
```

语言模型是对每个目标 token 做交叉熵；padding 和 Prompt token 可通过 mask 排除。

**二分类**

```text
L_BCE = -[y * log p + (1-y) * log(1-p)]
```

**回归**

- MSE 对大误差惩罚更强；
- MAE 对异常值更鲁棒，但零点不可导；
- Huber 在小误差用二次、大误差用线性。

**对比学习**

Embedding 和 reranker 常使用 in-batch negatives、InfoNCE 或 pairwise ranking loss，让相关 Query-Document 更近。

### 3. QLoRA

QLoRA 将冻结的基础模型权重量化为 4-bit，并把梯度通过反量化计算传到 LoRA Adapter。三个关键点是：

- NF4：针对近似正态分布权重设计的 4-bit 数据类型；
- Double Quantization：继续量化量化常数，减少存储；
- Paged Optimizer：使用统一内存缓解长序列和梯度检查点带来的显存峰值。

计算通常在 BF16/FP16 中进行，4-bit 主要用于存储基础权重；LoRA 参数和优化器状态保持较高精度。基础量化权重不更新，训练完成后保存 Adapter。QLoRA 省显存但不保证更快，反量化、kernel 和数据吞吐都可能成为瓶颈。

### 4. BERT、RoBERTa 与 DeBERTa

| 模型 | 共同点 | 关键差异 |
|---|---|---|
| BERT | Encoder-only、MLM | 原始方案还使用 NSP，静态 mask |
| RoBERTa | 保留 BERT 主体 | 去掉 NSP、动态 masking、更大数据和 batch、训练更充分 |
| DeBERTa | 仍是 Encoder/MLM 路线 | 内容与位置表示解耦的注意力，增强的 mask decoder |

**BERT**

- WordPiece 输入，加入 position 和 segment embedding；
- 使用 `[CLS]` 做句级任务，token 表示做序列标注；
- 原始预训练为 MLM + NSP。

**RoBERTa**

- 重点不是发明全新网络，而是重新优化 BERT 训练配方；
- 去掉 NSP，动态生成 mask；
- 使用更多数据、更大 batch 和更长训练；
- 采用 byte-level BPE。

**DeBERTa**

- 将每个 token 的内容和位置分别表示；
- 注意力分数同时考虑 content-to-content、content-to-position 和 position-to-content；
- enhanced mask decoder 在预测 masked token 时结合绝对位置信息。

### 5. BERT 可以做哪些任务

**句级任务**

- 文本分类、意图识别、情感分析；
- 句对匹配、自然语言推断；
- Cross-Encoder rerank。

**Token 级任务**

- NER、槽位填充、关键词和字段抽取；
- 序列标注。

**Span 级任务**

- 抽取式问答，预测 start/end；
- 实体片段和证据定位。

**向量检索**

原始 BERT 的 `[CLS]` 不一定是优质通用句向量，通常使用 Sentence-BERT、BGE 等经过对比学习的模型生成 embedding。

### 6. 跨境支付中的组合

- BERT/DeBERTa：工单分类、实体抽取、支付术语 NER、rerank；
- embedding 模型：召回产品规则、错误码和 SOP；
- Decoder LLM：结合 RAG 与工具结果生成解释；
- QLoRA：在资源受限时适配稳定的业务格式、术语与工具调用。

实时费率、订单状态和权限不通过微调记忆。模型只输出建议或调用受控工具，付款、退款、冻结和放行仍由确定性系统审批。
