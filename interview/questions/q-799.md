# 防抖和节流的原理及具体作用是什么？

## 答案

防抖和节流都用于限制高频事件执行，但时间语义不同。

### 防抖 Debounce

事件触发后等待一段时间；等待期间再次触发就重新计时。只有事件停止一段时间后才执行。

```javascript
function debounce(fn, wait) {
  let timer;
  return function (...args) {
    clearTimeout(timer);
    timer = setTimeout(() => fn.apply(this, args), wait);
  };
}
```

适合搜索框联想、表单校验、窗口 resize 后计算等“只关心最后一次”的场景。可扩展 leading/trailing、cancel 和 flush。

### 节流 Throttle

持续触发期间，保证固定时间窗口最多执行一次。

```javascript
function throttle(fn, wait) {
  let last = 0;
  return function (...args) {
    const now = Date.now();
    if (now - last >= wait) {
      last = now;
      fn.apply(this, args);
    }
  };
}
```

适合滚动、拖拽、鼠标移动和进度上报等需要持续反馈的场景。生产实现还应决定是否执行尾调用。

### 系统层类比

前端防抖/节流不等于服务端限流。支付查询或提交接口仍需 token bucket、幂等键和服务端校验；防抖只能改善体验，不能防止恶意或重复请求。

### 面试总结

防抖是“停止触发后执行最后一次”，节流是“持续触发时按固定频率执行”。前者适合搜索输入，后者适合滚动更新；关键业务接口还必须做服务端限流与幂等。
