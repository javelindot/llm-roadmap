# Bert如何训练的

## 答案

BERT 采用“两阶段”训练：先在大规模无标注文本上做自监督预训练，得到通用双向语言表示；再增加任务头，用少量标注数据端到端微调分类、序列标注、问答等下游任务。

### 1. 模型输入和结构

输入表示是三部分之和：

```text
input_embedding
= token_embedding
 + position_embedding
 + segment_embedding
```

- `[CLS]` 放在序列开头，常用于句级任务；
- `[SEP]` 分隔句子；
- segment A/B 标记句对；
- WordPiece 将词拆为子词。

主体是多层 Transformer Encoder。每个 token 可以双向关注左右上下文，不使用生成模型的 causal mask。

### 2. Masked Language Model

原始 BERT 随机选择约 15% token 作为预测目标。被选 token 中：

```text
80% -> [MASK]
10% -> random token
10% -> unchanged
```

只在被选位置计算交叉熵：

```text
L_MLM
= - sum_{i in masked_positions}
    log p(x_i | corrupted_input)
```

不全部替换成 `[MASK]`，是为了减小预训练与微调阶段的输入差异。

### 3. Next Sentence Prediction

原始 BERT 构造句对：

- 50%：B 是 A 在语料中的真实下一句；
- 50%：B 是随机句子。

用 `[CLS]` 表示做二分类：

```text
L_pretrain = L_MLM + L_NSP
```

后续 RoBERTa 移除了 NSP，并通过更多数据、动态 masking 和更充分训练取得更好结果，说明 NSP 不是 BERT 表示学习的必要组件。

### 4. 预训练数据与优化

原论文使用 BooksCorpus 和英文 Wikipedia，训练 BERT Base/Large。工程上需要：

- 清洗文档并保留句子/文档边界；
- 动态或静态 masking；
- 按长度组成 batch，减少 padding；
- AdamW、warmup 与学习率衰减；
- 混合精度和分布式训练；
- checkpoint、验证 MLM loss；
- 防止训练/评测数据泄漏。

中文 Whole Word Masking 会先按词边界选中完整词，再 mask 该词对应的所有 WordPiece/字符 token，但模型仍预测各 token。

### 5. 下游微调

**分类**

```text
[CLS] hidden -> Linear -> Softmax
```

**NER/槽位**

```text
token hidden -> Linear / CRF
```

**抽取式问答**

```text
token hidden -> start logits + end logits
```

**句对匹配/Rerank**

```text
[CLS] query [SEP] document [SEP]
-> relevance score
```

微调通常使用更小学习率和少量 epoch，并按随机种子重复。小数据需要 early stopping、类别权重和分层评测。

### 6. 适用边界

BERT 擅长输入理解、分类、抽取和 Cross-Encoder 重排，不适合长文本自回归生成。原始 `[CLS]` 也不是天然高质量向量，Dense Retrieval 通常使用经过对比学习的 Sentence-BERT、BGE 等。

跨境支付中可以用于工单意图、订单号/币种实体抽取、规则 rerank。金额和账户字段仍要用规则与业务系统校验，不能只依赖模型输出。
