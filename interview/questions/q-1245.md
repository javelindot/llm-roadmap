# 大模型智能体的工作原理和组成部分

## 答案

大模型智能体是在 LLM 外围加入状态、工具、执行环境和反馈闭环，使系统不仅生成文本，还能根据目标选择动作、观察结果并继续决策。核心循环是：

```text
接收目标与上下文
→ 理解和规划
→ 选择工具与参数
→ 权限检查
→ 执行动作
→ 读取 Observation
→ 更新状态并判断是否继续
→ 输出结果或转人工
```

### 1. 模型与决策策略

LLM 负责：

- 理解自然语言目标；
- 分解任务和选择下一步；
- 生成工具调用参数；
- 根据 Observation 调整计划；
- 汇总证据并生成最终回答。

常见模式包括 ReAct、Plan-and-Execute 和显式状态机。生产环境不应只依赖自由 ReAct，应对关键流程设置确定性节点、最大步数和停止条件。

### 2. 工具系统

工具将模型连接到外部世界，例如：

- 搜索、RAG 和数据库查询；
- 订单、账户、通道与汇率 API；
- 计算器、代码执行和文件解析；
- 工单创建和消息发送。

每个工具要有明确名称、用途、JSON Schema、超时、错误码和幂等语义。工具返回结构化结果，不能只给一段模糊自然语言。

### 3. 状态与记忆

需要区分：

- **短期状态**：当前对话、计划、工具结果和未完成步骤；
- **长期记忆**：经授权保存的用户偏好、历史任务摘要；
- **业务事实**：订单、余额和权限，必须实时从权威系统读取；
- **知识记忆**：产品规则、SOP 等由 RAG 提供。

长期记忆应可追溯、可删除、有时效和权限，不能把全部历史对话永久塞入 Prompt。

### 4. 规划与编排

编排层负责：

- 意图路由；
- 顺序、并行和条件分支；
- 子任务与子智能体调度；
- retry、timeout、circuit breaker；
- checkpoint、恢复和人工介入；
- token、步骤和费用预算。

简单固定流程优先使用工作流或状态机，只有下一步确实依赖开放式语义判断时才交给 LLM。

### 5. 安全与权限

模型提出动作不等于拥有权限。执行前需要：

- 用户、租户和角色鉴权；
- 参数白名单与 Schema 校验；
- 敏感字段脱敏；
- 写操作二次确认；
- 高风险动作人工审批；
- 防提示注入与工具结果污染；
- 完整审计日志。

读工具与写工具应隔离，默认最小权限。

### 6. 可观测与评测

记录每一步 Prompt、模型版本、工具参数、Observation、延迟、token、错误和最终状态。指标包括：

- 任务完成率；
- 工具选择与参数准确率；
- 平均步骤数和无效循环率；
- P95/P99、成本和超时率；
- 越权、敏感信息泄露和错误副作用；
- 人工接管率与接管原因。

评测要使用沙箱和可回放环境，不能让训练或测试 Agent 对真实资金系统自由探索。

### 7. 跨境支付示例

用户询问一笔 USD 付款未到账时，Agent 可以：

1. 校验用户是否有权查看该订单；
2. 查询付款状态、通道回执和清算批次；
3. 检索对应国家、币种和通道的到账规则；
4. 检查证据是否冲突；
5. 生成当前状态、依据和下一步；
6. 信息不足或涉及不可公开风控原因时转人工。

Agent 可以读状态并解释，但重新付款、退款、冻结和放行必须进入确定性状态机和审批流程。
