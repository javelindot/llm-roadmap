# SFT中的loss是怎么做计算的？

## 答案

**一句话**：SFT 的 loss 本质是 next-token prediction 的交叉熵损失，通常只对 assistant 回答部分计算 loss，不对用户 prompt 计算。

### 1. 计算方式

训练样本是一段对话或指令回答。模型在每个位置预测下一个 token，和真实 token 做交叉熵：

```text
Loss = - sum log P(y_t | y_<t, x)
```

在指令微调里，输入通常包括 system、user、assistant。一般会 mask 掉 system 和 user 部分，只让 assistant 输出部分参与 loss。

### 2. 为什么要 mask

如果对用户输入也算 loss，模型会学习复述用户问题，而不是学习回答。只对回答部分算 loss，可以让模型更专注学习如何根据指令生成正确回复。

### 3. 结合跨境支付项目怎么说（面试口径）

> 如果在跨境支付客服数据上做 SFT，比如用户问“付款为什么失败”，assistant 的标准回答是基于错误码解释原因和下一步处理建议。训练时会把用户问题作为上下文，但 loss 只算 assistant 回复部分。
>
> 另外要注意训练数据质量。支付场景不能把错误承诺、泄露风控规则或过期 SOP 放进训练集，否则模型会学坏。SFT 之后还要用真实工单评测准确率和合规性。

**钩子**：可以引到“next-token cross entropy、label mask、客服标准回复、数据质量”。
