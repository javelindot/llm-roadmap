# 关于 CPT（Continued Pre-Training）的工作？

## 答案

> **CPT（Continued Pre-Training，继续预训练）= 在通用 base 模型上用领域数据再做一轮预训练**——目的是让模型"吃透"领域语料、知识、术语，比 SFT 更深入但比从头预训练便宜。
>
> **CPT 的位置**：
>
> ```
> Pretrain（通用海量文本）
>    ↓
> CPT（领域大量文本）⭐
>    ↓
> SFT（指令对齐）
>    ↓
> RLHF / DPO（偏好对齐）
> ```
>
> CPT 在 Pretrain 和 SFT 之间，是"**领域适应**"专门阶段。
>
> **什么时候需要 CPT**：
>
> 1. **领域语料和通用差异大**——金融、医疗、法律、生物
> 2. **业务术语 base 模型不熟**——专业 jargon
> 3. **有大量领域无标注语料**（10G+ 文本）
> 4. **追求业务上限**
> 5. **SFT 后仍觉得"模型不懂行"**
>
> **不需要 CPT 的情况**：
>
> - 只是风格 / 格式定制 → 直接 SFT
> - 通用任务 → base + SFT 够
> - 数据 < 1GB → SFT + RAG 即可
> - 团队第一次做对齐 → 先 SFT 别折腾
>
> **CPT 的数据**：
>
> - **无标注领域文本**（关键！）
>   - 行业白皮书、监管文件、专业书
>   - 公司内部文档、SOP、对话历史
>   - 行业新闻、博客、论坛
> - **量级**：10GB-1TB 文本
> - **质量**：和 pretrain 一样要清洗（去重、去 PII、过滤低质）
>
> **CPT 的训练**：
>
> 和 pretrain 一样的 Next-Token Prediction loss：
> $$L = -\sum_t \log P_\theta(x_t | x_{<t})$$
>
> 但参数：
> - **学习率**：比 pretrain 小 10-100x（如 1e-5）
> - **epoch**：1-2 轮
> - **数据混合**：领域数据 + 一部分通用数据（防止灾难性遗忘）
> - **混合比例**：典型 50/50 或 70/30
>
> **典型用法（金融领域 CPT）**：
>
> ```python
> # 数据准备
> finance_corpus = load_finance_data(size="50GB")  # 金融文本
> general_corpus = load_general_data(size="50GB")  # 通用文本
>
> # 50/50 混合采样
> mixed_corpus = mix(finance_corpus, general_corpus, ratio=0.5)
>
> # CPT 训练
> training_args = TrainingArguments(
>     learning_rate=1e-5,
>     num_train_epochs=1,
>     per_device_train_batch_size=4,
>     gradient_accumulation_steps=32,
>     fp16=True,
> )
>
> trainer = Trainer(
>     model=qwen_base,
>     args=training_args,
>     train_dataset=mixed_corpus,
> )
> trainer.train()
> ```
>
> **CPT 的关键挑战**：
>
> **1. 灾难性遗忘**
> - 训领域数据时通用能力下降
> - 治理：混入通用数据、低学习率、少 epoch
>
> **2. 数据质量**
> - 噪声会被记住
> - 治理：严格清洗、去重
>
> **3. 算力成本**
> - CPT 几乎和 pretrain 一样贵
> - 7B + 50GB 数据 + 1 epoch ≈ 几十万元
>
> **4. 评测难**
> - PPL 下降不代表业务好
> - 必须配合下游任务评测
>
> **我们跨境支付的实际情况**：
>
> **没上 CPT**，原因：
>
> 1. 业务侧覆盖**SFT + RAG 已够**：5w 业务样本 SFT + 7w 文档 RAG，业务准确率 92%
>
> 2. **CPT 算力成本高**：14B + 50GB CPT 需要 8 张 H100 跑 2 周，成本 ~50w
>
> 3. **业务变化快**：CPT 训完业务规则又变了，不划算
>
> 4. **Qwen 通用能力够强**：base 已经"懂金融"
>
> **会上 CPT 的场景**：
>
> - 真正特殊领域（生物医学、量子物理）
> - 大公司有海量私域数据（千 GB+）
> - 业务相对稳定
> - 追求行业 SOTA
>
> 国内案例：
> - 蚂蚁金融大模型（CPT）
> - 法律大模型 ChatLaw（CPT + SFT）
> - 医学 HuatuoGPT（CPT）
>
> **几个工程经验**：
>
> 1. **CPT ≠ SFT**——很多人混用，CPT 是无监督 next-token，SFT 是有监督指令
>
> 2. **CPT 前先评估 ROI**——SFT 能搞定就别上 CPT
>
> 3. **混合通用数据防遗忘**——50/50 是经验值
>
> 4. **CPT 后必须 SFT**——只 CPT 模型不会对话
>
> 5. **效果难看出来**——业务评测才知道值不值
>
> 一句话：**CPT 是"让 base 模型扎根领域"的重武器**——成本高、收益深、适合大公司 + 特殊领域 + 稳定业务，工业 80% 场景 SFT + RAG 就够。

- "灾难性遗忘的治理方法可以展开。"
- "CPT 的成本 / 收益怎么算可以讲讲。"
