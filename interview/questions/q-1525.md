# 你在模型上线做 a b 实验有什么个人体会吗

## 答案

这是一道经验题，应只讲亲自参与过的实验；如果只做过离线评测或 shadow test，要明确边界。比“实验显著了”更有价值的是说明踩过什么坑、后来如何改进。

### 1. 个人体会

**离线提升不等于线上提升**

离线数据来自旧策略曝光，存在选择偏差。模型 AUC、NDCG 提升后，线上核心指标可能不变，甚至因时延、校准或流量分布变化而下降。因此上线前要建立“离线指标 -> 中间行为 -> 业务结果”的因果假设。

**随机化单位比流量比例更重要**

实验单位可能是用户、商户、会话、设备或请求。若同一商户同时进入两组，会产生污染；若按请求随机，用户体验也可能不一致。应根据干扰范围选择稳定分桶键。

**先保证实验可信**

- 检查 SRM，实际样本比例是否符合配置；
- 确认埋点、曝光和标签成熟窗口；
- 运行 A/A 或历史回放；
- 预先定义主指标、护栏、周期和停止规则；
- 不根据每天波动随意提前结束。

**尾部风险比平均值重要**

模型可能提升平均转化，却损害某个国家、语言、长尾商户或高风险人群。必须看分群、P95/P99、错误类型和严重事故，而不是只看总体均值。

**灰度与回滚是实验的一部分**

实验通过 feature flag 发布，先 shadow/canary，再逐步放量。模型、特征、Prompt、知识库和规则版本都要可追踪，并提前定义自动回滚阈值。

### 2. 面试示例

> 我最大的体会是，A/B 实验首先是测量系统，其次才是模型比较。有一次离线指标提升，但线上主指标没有显著变化。排查后发现新增模型增加了尾延迟，而且训练标签与线上成熟标签口径不一致。后来我们把服务 P95 和标签成熟率加入护栏，按稳定实验单位分桶，并在放量前做 shadow replay，实验结论才更可靠。

只能替换为真实案例和数字。

### 3. 跨境支付场景

支付路由实验应优先按商户或稳定业务实体分流，主指标可为支付成功率，护栏包括拒付率、成本、P99、重复扣款、通道集中度和资金损失。拒付等标签成熟慢，不能只跑几天就下结论；低频严重风险还需灰度上限和人工审批。

### 4. 面试总结

高质量体会包括：随机化和埋点决定可信度，离线线上存在差距，统计显著要结合业务显著，平均收益不能掩盖尾部风险，并且任何实验都必须可监控、可停止、可回滚。
