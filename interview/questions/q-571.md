# Bert结构

## 答案

BERT 是基于 Transformer Encoder 的双向预训练语言模型，核心是利用左右上下文学习 token 表示。

### 1. 输入

每个位置由三种 Embedding 相加：

\[
E=E_{\text{token}}+E_{\text{position}}+E_{\text{segment}}
\]

输入通常含 `[CLS]` 和 `[SEP]`。Segment Embedding 区分句子 A/B，Position Embedding 表示绝对位置。

### 2. Encoder

每层包含：

1. Multi-Head Self-Attention；
2. 残差连接与 LayerNorm；
3. 两层 FFN，原版使用 GELU；
4. 再次残差与 LayerNorm。

BERT Base 为 12 层、隐藏维 768、12 个头，约 110M 参数；BERT Large 为 24 层、隐藏维 1024、16 个头，约 340M 参数。

### 3. 预训练目标

- MLM：随机选择部分 token 做掩码预测，学习双向上下文；
- NSP：判断两句话是否相邻。后续 RoBERTa 等工作表明 NSP 并非必要，并改进了数据和训练策略。

### 4. 下游使用

- 句子分类：`[CLS]` 或池化表示接分类头；
- NER：每个 token 表示接分类头；
- 抽取式问答：预测答案起止位置；
- 语义检索：通常需要双塔对比学习，不能直接假设原生 `[CLS]` 是最佳向量。

### 局限与发展

BERT 是 Encoder-only，适合理解任务但不适合自回归长文本生成；绝对位置和 \(O(n^2)\) Attention 限制长度。现代工程常使用 DeBERTa、E5/BGE 等专门编码器，或长上下文 Encoder，但 BERT 仍是理解 Transformer 编码器的重要基线。

### 面试总结

BERT 由输入 Embedding、双向 Transformer Encoder 和任务头构成，使用 MLM 预训练。它擅长分类、抽取和编码任务，但生成、超长上下文及原生向量检索不是其强项。
