# rankmixer中token的交互机制是怎样的？

## 答案

RankMixer 是面向工业排序模型的硬件友好架构。它认为推荐特征 token 来自异构语义空间，未必需要像 NLP 一样计算 `QK^T` 相似度；其 Multi-Head Token Mixing 通过无参数的拆分与重组，让不同特征 token 交换信息。

### 1. 输入 Token

用户、物品、上下文、序列等特征先按语义分组和投影，形成：

```text
X in R^(T x D)
```

其中 `T` 是特征 token 数，`D` 是每个 token 的隐藏维度。

### 2. Multi-Head 拆分

每个 token 沿隐藏维拆成 `H` 个 head：

```text
x_t = [x_t^(1) || x_t^(2) || ... || x_t^(H)]

x_t^(h) in R^(D/H)
```

### 3. 跨 Token 重组

对同一个 head 编号，收集所有 token 的该子空间：

```text
s_h = Concat(
  x_1^(h),
  x_2^(h),
  ...,
  x_T^(h)
)
```

这样一个新的 shuffled token 同时包含所有原始 token 的部分信息，实现全局特征交互。

论文中可设置 `H=T`，使重组后的 token 数仍为 `T`，方便残差连接。这个过程本质是 reshape/transpose/concat，没有 `QK^T`、softmax 和可学习 Attention 权重。

### 4. 为什么能发生高阶交互

单次 Mixing 让每个新 token 获得多个原始特征 token 的子空间；随后 Per-Token FFN 对混合后的表示做非线性变换。堆叠多个 Block 后，信息反复重组和非线性组合，从而形成高阶交叉。

典型 Block：

```text
S = LN(TokenMixing(X) + X)
X' = LN(PFFN(S) + S)
```

PFFN 为不同 token 使用独立或更专门的 FFN 参数，以适配异构特征空间。

### 5. 与 Self-Attention 的区别

| 维度 | Self-Attention | RankMixer Token Mixing |
|---|---|---|
| 交互方式 | Q/K 相似度动态加权 | 固定拆分与重组 |
| 参数 | Q/K/V/O 投影 | Mixing 本身无参数 |
| Token 复杂度 | 通常 O(T²) | 近似线性重排 |
| 语义假设 | Token 可比较 | 适合异构特征 |
| 动态选择 | 强 | 主要依赖后续 PFFN |

代价是 Mixing 本身不根据样本动态决定“哪个 token 更重要”，表达力要由堆叠、PFFN 和上层任务共同补充。

### 6. 跨境支付场景

通道路由排序可把商户、国家、币种、支付方式、金额、通道健康度和历史成功率分成 token。Token Mixing 形成跨特征交互，但牌照、制裁和限额仍应作为硬过滤，不能只靠排序模型学习。

### 7. 面试总结

RankMixer 的 token 交互是：每个 token 拆成多个 head，再把所有 token 的同号 head 拼成新 token，最后通过 Per-Token FFN 做非线性交互。它用参数无关的线性重排替代二次 Self-Attention，以提高工业排序模型的 GPU 效率。
