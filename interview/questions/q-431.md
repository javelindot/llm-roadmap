# 🔥 提高模型的准确率核心是什么

## 答案

**核心观点**：提高模型准确率的关键不是先换更大的模型，而是建立“定义指标—拆解错误—定位瓶颈—针对优化—回归验证”的闭环。

### 1. 先定义“准确”

不同任务的准确率含义不同：

- 分类：Accuracy、Precision、Recall、F1。
- RAG：检索 Recall@K、答案正确性、忠实度、引用准确率。
- Agent：任务完成率、工具选择率、参数正确率。
- 生成：事实正确、格式遵循、安全性和业务可用性。

指标不清楚，优化方向就会混乱。

### 2. 把错误拆开

LLM 应用常见错误来源：

```text
输入 / 数据问题
→ 检索召回问题
→ 上下文排序与裁剪问题
→ Prompt / 工具描述问题
→ 模型能力问题
→ 解码与后处理问题
```

例如 RAG 回答错误，要先判断是“文档没召回”“召回了但排在后面”“上下文有答案但模型没用”，这三类问题的解法完全不同。

### 3. 优化优先级

1. **修数据和标签**：去重、纠错、统一口径、补足长尾。
2. **修上下文供给**：提高召回、Rerank、切片和元数据过滤。
3. **修任务表达**：Prompt、Few-shot、输出 Schema、工具说明。
4. **增加约束与校验**：规则、引用、结构化验证、二次确认。
5. **再考虑模型升级或微调**：当错误确实来自能力边界时再投入。

### 4. 必须建立的机制

- 固定离线评测集 + 持续收集线上 Bad Case。
- 按场景和错误类型看分数，不只看平均值。
- 每次变更做回归测试和 A/B。
- 监控数据漂移、模型版本和成本延迟。

**面试总结**：准确率优化本质是一个误差分析问题。先找到当前系统最大的误差项，再用最小成本解决它；盲目升级模型往往既贵，也不能修复错误数据和错误流程。
