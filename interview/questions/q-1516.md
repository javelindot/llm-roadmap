# PEFT的几种微调方法及其区别和优缺点

## 答案

PEFT（Parameter-Efficient Fine-Tuning）冻结大部分基础模型，只训练少量新增或选定参数。主要路线包括低秩更新、Adapter、软提示、缩放向量和少量参数选择。

### 1. LoRA 与 QLoRA

LoRA 把权重增量写为：

```text
W' = W + scale * B * A
```

其中 rank `r` 远小于原维度。

优点：

- 训练参数和 optimizer state 少；
- 易保存、切换和合并；
- 对生成任务通常效果稳定；
- 工具链成熟。

缺点：

- rank 和 target module 需要调优；
- 多 Adapter 在线切换有管理成本；
- 复杂能力变化可能受低秩容量限制。

QLoRA 在量化后的冻结基座上训练 LoRA，进一步省显存，但训练 kernel、量化误差和部署兼容性更复杂。

### 2. Adapter

在 Transformer 层中插入瓶颈网络：

```text
h' = h + W_up * activation(W_down * h)
```

表达力强、模块边界清楚，但每层增加串行计算，推理时可能比合并后的 LoRA 更慢。

### 3. Prompt/Prefix Tuning

- **Prompt Tuning**：训练输入侧的连续 soft prompt；
- **Prefix Tuning**：为多层 Attention 学习可训练的前缀 K/V；
- **P-Tuning**：通过可训练提示编码器构造连续提示。

参数最少，适合多任务快速切换；但小模型、复杂生成任务和强领域迁移中可能不如 LoRA，且会占用上下文或 KV Cache。

### 4. IA3、BitFit 与选择性微调

- **IA3**：学习向量缩放 Attention/FFN 激活，参数极少；
- **BitFit**：只训练 bias；
- **LayerNorm tuning**：只调归一化参数；
- **Selective tuning**：只解冻顶层、Embedding 或特定模块。

这些方法成本低，但容量通常更有限。

### 5. LoRA 变体

- **AdaLoRA**：训练中动态分配不同层的 rank 预算；
- **DoRA**：把权重更新拆为方向和幅度，提升低秩适配表达；
- **LoRA+**：为 A/B 矩阵使用不同学习率策略；
- **多 Adapter 融合/路由**：按任务组合多个适配器。

新方法不能只看论文精度，还要确认训练框架、合并、量化和推理引擎支持。

### 6. 如何选择

| 场景 | 优先方案 |
|---|---|
| 通用领域 SFT | LoRA |
| 显存非常紧张 | QLoRA |
| 大量轻量任务切换 | Prompt/Prefix、IA3 |
| 需要较强任务容量 | LoRA/DoRA、Adapter |
| LoRA 已证明容量不足 | 提高 rank、扩大 target modules 或全参 |

评测应保持数据、训练 token 和推理配置一致，比较质量、显存、吞吐、checkpoint 大小和能力回归。

### 7. 跨境支付场景

可为客服、合规文档和商户运营分别维护 Adapter，但必须做版本、权限和租户隔离。领域微调不能替代实时订单查询、确定性合规规则和人工审批。

### 8. 面试总结

PEFT 的核心差异在于“参数加在哪里、训练多少、推理是否增加开销”。LoRA 是当前通用首选，QLoRA 更省显存；Prompt/IA3 参数更少；Adapter 容量较强但可能增加推理延迟。
