# 什么是 3D并行？

## 答案

**一句话**：3D 并行是数据并行、张量并行、流水线并行三种方式组合起来训练超大模型。

### 核心回答
- **数据并行 DP**：不同 GPU 处理不同数据 batch，同步梯度。
- **张量并行 TP**：把单层里的大矩阵切到多张卡上，比如切 attention 或 MLP 权重。
- **流水线并行 PP**：把模型不同层切到不同 GPU，上游输出传给下游。

3D 并行适合单一并行方式不够用的超大模型：
- DP 提升数据吞吐。
- TP 解决单层参数太大。
- PP 解决模型层数太多。

难点是通信复杂、负载均衡、pipeline bubble、调参和排障成本高。

### 1. 三种并行分别解决什么问题

3D 并行不是一个单独算法，而是三类并行的组合：

- **DP 解决数据吞吐**：每张卡处理不同 batch，提高训练样本吞吐。
- **TP 解决单层太大**：一层里的矩阵乘法太大，就把矩阵切到多张卡。
- **PP 解决层数太多**：模型层太多时，把不同层放到不同设备。

如果只用 DP，每张卡仍然要放完整模型；如果只用 TP，通信频繁；如果只用 PP，会有 pipeline bubble。3D 并行就是在这些问题之间做平衡。

### 2. 工程难点

真正难的不是知道 DP/TP/PP 名字，而是：

- 卡间通信拓扑要匹配。
- batch、micro-batch、梯度累积要配好。
- pipeline bubble 要尽量减少。
- 不同层负载要均衡。
- checkpoint、恢复训练、排障复杂度都会上升。

### 3. 面试总结

3D 并行适合训练超大基础模型。对于普通业务微调，优先考虑 LoRA、QLoRA、FSDP 或更小模型，别为了技术复杂度而复杂。

### 4. 结合跨境支付项目怎么说（面试口径）

> 业务项目里我一般不会轻易上 3D 并行。像跨境支付里的风控助手、RAG 问答、商户审核模型，更多是基于已有模型做 LoRA、SFT 或 DPO，没必要用训练基座模型的复杂方案。
>
> 但如果公司真的要训练金融或支付领域基座模型，比如要覆盖多语言、多国家合规、多种支付产品，那模型和数据规模都很大，单卡或单一并行方式就不够了。这时 DP、TP、PP 组合起来才有意义。
>
> 我会重点看通信成本、pipeline bubble 和恢复训练能力。因为训练几天后挂掉，如果 checkpoint 和并行状态恢复不好，成本会非常高。

**钩子**：3D 并行适合基座模型训练，业务微调优先 LoRA/FSDP，别过度设计。
