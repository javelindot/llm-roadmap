# 🔥🔥 Transformer Decoder 的结构有哪些部分？

## 答案

回答这题要先区分 **原始 Encoder-Decoder Transformer 的 Decoder** 和 GPT/LLaMA 使用的 **Decoder-only Block**。

### 1. 原始 Transformer Decoder

每层包含三个子层：

1. **Masked Multi-Head Self-Attention**
   使用因果掩码，当前位置只能看前面的 Token。
2. **Encoder-Decoder Cross-Attention**
   Query 来自 Decoder，Key/Value 来自 Encoder 输出，用于读取源序列。
3. **Position-wise FFN**
   对每个位置独立执行两层前馈网络。

每个子层外还有：

- Residual Connection
- Layer Normalization
- Dropout

最后经过线性层和 Softmax 得到词表概率。

### 2. Decoder-only LLM

GPT、LLaMA 等没有独立 Encoder，因此移除了 Cross-Attention。典型 Block 是：

```text
输入
 → Norm
 → Causal Self-Attention
 → Residual
 → Norm
 → FFN / SwiGLU
 → Residual
```

多层堆叠后再经过最终 Norm 和 LM Head，预测下一个 Token。

现代 LLM 还常见：

- Pre-Norm 或 RMSNorm
- RoPE 位置编码
- GQA / MQA / MLA
- SwiGLU FFN
- KV Cache 加速自回归生成

### 3. 训练与推理差异

- 训练时使用 Teacher Forcing，可并行计算整个序列，但因果掩码阻止看到未来。
- 推理时逐 Token 生成，利用 KV Cache 避免重复计算历史 K/V。

**面试总结**：原始 Decoder 是“掩码自注意力 + 交叉注意力 + FFN”，而 Decoder-only LLM 是“因果自注意力 + FFN”的重复堆叠。
