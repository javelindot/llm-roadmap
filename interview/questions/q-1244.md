# 介绍一下了解的大模型有哪些，这些模型在结构上有什么差异

## 答案

大模型可以按 Transformer 形态、参数组织、注意力、位置编码和模态接入方式分类。闭源模型通常不披露完整结构，不能根据产品名臆测参数量或具体模块；结构比较应以公开技术报告为准。

### 1. Encoder-only

代表：BERT、RoBERTa、DeBERTa。

- 双向 Self-Attention；
- 擅长分类、NER、抽取、embedding 和 rerank；
- 不天然支持自回归长文本生成；
- 参数和延迟通常低于通用生成模型。

跨境支付中可用于工单分类、实体抽取和检索精排。

### 2. Encoder-Decoder

代表：原始 Transformer、T5。

- Encoder 双向理解输入；
- Decoder causal self-attention + cross-attention；
- 适合翻译、摘要和输入到输出的序列转换；
- Encoder 结果可复用，但架构比单塔更复杂。

### 3. Dense Decoder-only

代表：Llama、Qwen 的 Dense 版本，以及许多 GPT 类模型。

- causal language modeling；
- 统一完成对话、生成、代码和工具调用；
- 常用 RoPE、RMSNorm、SwiGLU 和 Pre-Norm；
- 现代模型常用 GQA 减少 KV Cache。

Llama 3 公开报告描述的是 Dense Transformer；Qwen2 同时提供多个 Dense 尺寸。

### 4. Sparse MoE Decoder

代表：Mixtral、Qwen2-57B-A14B、DeepSeek-V3。

- FFN 层包含多个 experts；
- router 为每个 token 选择少量专家；
- 总参数容量大，但单 token 只激活一部分；
- 训练和部署需要处理专家负载、all-to-all 通信和并行切分。

Mixtral 8x7B 每层从 8 个专家中选择 2 个；DeepSeek-V3 使用 DeepSeekMoE，并采用无辅助损失的负载均衡策略。不同 MoE 的“总参数”和“激活参数”不能混为一谈。

### 5. 注意力结构

**MHA**

```text
H_q = H_kv
```

表达能力强，KV Cache 最大。

**MQA**

```text
H_kv = 1
```

KV Cache 最小，但 K/V 共享最强。

**GQA**

```text
1 < H_kv < H_q
```

在质量与推理成本之间折中，Llama 3、Qwen2 等公开模型使用。

**MLA**

DeepSeek-V3 使用 Multi-head Latent Attention，把 K/V 信息压缩到低维 latent 表示以降低推理缓存，但实现与部署生态不同于标准 GQA。

### 6. 位置与长上下文

- 原始 Transformer 使用 sinusoidal position；
- BERT 使用 learned absolute position；
- Llama/Qwen 等常用 RoPE；
- 长上下文还会配合 RoPE scaling、YARN、局部/滑窗注意力或训练阶段长度扩展。

标称窗口不是唯一指标，还要评测目标长度的准确率、TTFT、KV Cache 和吞吐。

### 7. 多模态结构

- 图像 Encoder + Projector，将视觉 token 接入语言模型；
- Cross-Attention 让文本读取视觉特征；
- 统一 token/early fusion，在同一 Transformer 中建模多模态；
- 原生多模态与后接视觉适配器在训练成本和融合深度上不同。

### 8. 如何选型

| 任务 | 优先结构 |
|---|---|
| 分类、NER、rerank | Encoder-only |
| 通用生成、Agent | Decoder-only |
| 固定序列转换 | Encoder-Decoder |
| 高容量且可承担复杂部署 | MoE |
| 文档图片与截图理解 | VLM |

跨境支付落地还要比较多语言、工具调用、结构化输出、事实性、安全、延迟、成本、许可和数据部署边界。架构决定能力与成本的倾向，最终选择仍以真实业务黄金集和目标硬件压测为准。
