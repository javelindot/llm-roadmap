# SwiGLU激活函数的原理

## 答案

SwiGLU 是 GLU（Gated Linear Unit）的变体，在 Transformer FFN 中用 SiLU/Swish 分支作为门，控制另一条线性分支的信息通过。

### 1. 公式

SiLU：

```text
SiLU(x) = x * sigmoid(x)
```

SwiGLU FFN：

```text
gate = SiLU(x W_gate)
value = x W_up
hidden = gate ⊙ value
output = hidden W_down
```

即：

```text
SwiGLU(x)
= (SiLU(xW_gate) ⊙ xW_up) W_down
```

通常再与残差连接相加。

### 2. 与普通 FFN 区别

普通 GELU FFN：

```text
GELU(xW_1) W_2
```

SwiGLU 有 gate/up/down 三个矩阵。Gate 根据输入动态控制各隐藏维度，提供乘性交互和更灵活的条件计算。

### 3. 为什么有效

- SiLU 平滑，负区间仍有非零输出；
- 门控可选择性传递信息；
- 两分支逐元素乘法增加表达能力；
- 在大规模语言模型中通常比 ReLU/GELU FFN 有更好的质量-计算权衡；
- 与 Pre-Norm、RMSNorm 等现代架构配合成熟。

### 4. 参数与计算

若中间维度同为 `d_ff`：

```text
普通 FFN参数约 2*d_model*d_ff
SwiGLU 参数约 3*d_model*d_ff
```

因此实践会缩小 SwiGLU 的 `d_ff`。若普通 FFN 使用 `4*d_model`，为保持相近参数/FLOPs，SwiGLU 中间维度常取约：

```text
(8/3) * d_model
```

再按硬件友好倍数取整。不同模型配置不完全相同。

### 5. 局限

- 比单分支激活多一个投影；
- 中间张量和 kernel 更复杂；
- 最优中间维度需重新调整；
- 激活函数收益不能脱离数据和训练配方单独比较。

### 6. 面试总结

SwiGLU 用 `SiLU(gate) × value` 做输入相关门控，再经 down projection。它以额外分支换取更强表达，通常通过缩小中间维度保持参数和 FLOPs。
