# 大模型答非所问怎么办

## 答案

大模型答非所问可能发生在意图理解、上下文、检索、Prompt、模型能力、解码或评测环节。不能只靠“把 Prompt 写得更强”，应先分类 bad case 再处理。

### 1. 先定位原因

- 用户问题有歧义；
- 多轮指代解析错误；
- system/user/tool 指令冲突；
- 上下文过长，关键信息被截断；
- RAG 召回了错误内容；
- 工具返回未校验；
- 模型不具备所需知识或能力；
- 输出格式没有约束；
- temperature 太高；
- 安全策略误触发；
- 评测答案本身不唯一。

### 2. 输入和意图

- 意图分类和槽位抽取；
- 对缺失信息追问；
- query rewrite；
- 将历史压缩成结构化状态；
- 明确当前任务、约束和输出格式；
- 对复杂请求先规划。

### 3. RAG 和工具

- 提升召回与 Rerank；
- metadata/权限过滤；
- 证据不足时拒答；
- 工具结果 schema 校验；
- 只提供当前任务相关字段；
- 要求答案引用证据；
- 将实时状态从权威系统获取。

### 4. Prompt 和模型

- system 规则分层；
- 给出正反示例；
- 使用 JSON Schema/structured output；
- 降低无关背景；
- 调整 temperature/top-p；
- 对高频任务使用专用小模型；
- 必要时 SFT/DPO；
- 复杂任务路由到更强模型。

### 5. 输出后验证

- 检查是否回答用户意图；
- 关键实体和数值一致性；
- 引用是否支持结论；
- 业务规则验证；
- 不满足条件时重试、修复或转人工。

### 6. 建立评测闭环

为“答非所问”定义 taxonomy：

```text
意图错
实体错
证据错
遗漏子问题
格式错
过度拒答
无关扩展
```

记录每类占比和修复前后指标，避免只看主观案例。

### 7. 跨境支付场景

用户问“这笔欧元退款什么时候到账”，模型不能回答支付成功率或一般退款政策。应先识别退款订单，查实时状态、通道和 SLA，再回答；信息不足就追问订单号。

### 8. 面试总结

解决答非所问要沿“理解 -> 检索/工具 -> 生成 -> 验证”排查。先明确意图和证据，再用结构化输出与后验校验保证答案对题，最后用 bad case 数据闭环迭代。
