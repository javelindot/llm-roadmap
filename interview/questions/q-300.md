# 元组能否作为字典的key

## 答案

答：能。字典的key要求不可变。

### 补充说明

Python 字典的 key 要求是可哈希对象。元组本身是不可变类型，所以通常可以作为 key。但有一个前提：元组内部的元素也必须是可哈希的。

例如：

```python
{(1, "USD"): "ok"}  # 可以
{(1, [2, 3]): "bad"}  # 不可以，因为 list 不可哈希
```

所以更准确的回答是：元组能不能作为 key，取决于元组内部所有元素是否都可哈希。

### 结合跨境支付项目怎么说（面试口径）

> 在项目里我会用元组 key 表达组合维度，比如 `(country, currency, channel)` 对应某个通道路由配置，或者 `(merchant_id, currency)` 对应限额缓存。这样比字符串拼接更清晰。
>
> 但我会注意元组里的元素必须是不可变且可哈希的，比如字符串、数字、枚举都可以，不能放 list、dict 这种可变对象。支付系统里很多缓存 key 都是组合维度，理解这个点能避免一些很隐蔽的 bug。

**钩子**：可以引到“可哈希、不可变、组合 key、通道路由缓存”。
