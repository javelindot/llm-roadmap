# 10. static 关键字作用？

## 答案

结合相邻题目，这里按 Java 的 `static` 回答。`static` 表示成员属于类本身，而不是某个对象实例。

### 静态变量

类的所有实例共享同一份静态变量，可通过 `ClassName.field` 访问。它在类加载阶段初始化，常用于全局计数器、缓存和配合 `final` 定义常量：

```java
class User {
    static int count = 0;
    static final String TYPE = "normal";
}
```

静态可变变量会形成全局共享状态，多线程访问时要考虑可见性和原子性，也可能延长对象生命周期。

### 静态方法

静态方法无需创建对象即可调用。它没有 `this` 或 `super`，不能直接访问实例字段和实例方法，因为调用时可能不存在具体对象。它适合无实例状态的工具函数、工厂方法和程序入口：

```java
public static void main(String[] args) { }
```

静态方法可以被子类隐藏，但不是基于运行时对象的多态重写；调用哪个方法由引用处的类决定。

### 静态代码块和静态内部类

静态代码块在类初始化时执行一次，按源码顺序与静态字段初始化共同完成类级初始化。静态内部类不持有外部类实例，可独立创建；常见的 initialization-on-demand holder idiom 利用类加载机制实现懒加载单例。

Java 还支持静态导入，使调用静态成员时省略类名，但过度使用会降低可读性。要区分“类被加载”和“类被初始化”，只有首次主动使用等条件满足时才会执行静态初始化逻辑。
