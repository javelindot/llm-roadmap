# 自己的应用模型的经历，改进经历等

## 答案

**一句话**：讲应用模型经历，要说清楚你用了什么模型、解决什么问题、怎么改进、怎么评估，而不是只列模型名字。

### 1. 可以怎么讲

- 用过什么模型：LLM、Embedding、Rerank、OCR、多模态、分类模型。
- 用在什么场景。
- 遇到什么问题。
- 做了哪些改进。
- 效果如何评估。

### 2. 常见改进方向

- Prompt 优化。
- RAG 检索优化。
- Query rewrite。
- Chunk 和 metadata 优化。
- Rerank。
- 微调/LoRA。
- 工具调用和事实校验。
- 缓存和推理加速。

### 3. 结合跨境支付项目怎么说（面试口径）

> 我应用模型比较典型的经历是在跨境支付客服场景里。最开始只是用 RAG 回答 FAQ，但发现用户问“付款为什么失败”时，单纯检索知识库不够，因为还需要实时订单状态和通道回执。
>
> 后来我做了几类改进：第一，给知识库加国家、币种、通道、错误码 metadata；第二，引入 BM25 + 向量混合召回和 rerank；第三，通过 Function Call 查询订单和风控状态；第四，用真实工单做评测集，看引用命中率、人工改写率和转人工率。
>
> 这个经历让我更清楚模型应用不是单点优化，而是数据、检索、工具、生成和评估一起闭环。

**钩子**：可以引到“从简单 RAG 到工具增强 Agent、真实工单评测、badcase 优化”。
