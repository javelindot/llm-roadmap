# Agent和工作流有什么区别？

## 答案

Workflow 是预先定义的执行路径，Agent 是根据目标和环境反馈动态决定下一步动作的系统。二者不是互斥关系，生产系统通常用 Agent 决策、Workflow 可靠执行。

### 1. Workflow

特点：

- 步骤和分支预先定义；
- 输入输出明确；
- 可预测、可测试；
- 易审计和重试；
- 适合稳定、高频和高风险流程。

例如：

```text
收到退款申请
-> 校验订单
-> 校验金额
-> 风险检查
-> 人工审批
-> 执行退款
```

### 2. Agent

特点：

- 接受目标而非固定步骤；
- 动态规划；
- 自主选择工具；
- 根据 observation 重规划；
- 处理开放式和不确定任务；
- 需要终止、预算和安全机制。

例如研究“某国家支付成功率下降原因”，需要动态选择日志、通道、时间和错误码分析。

### 3. 对比

| 维度 | Workflow | Agent |
|---|---|---|
| 路径 | 预定义 | 动态 |
| 可预测性 | 高 | 较低 |
| 灵活性 | 低到中 | 高 |
| 测试审计 | 容易 | 更难 |
| 成本 | 可控 | 波动 |
| 适用任务 | 稳定流程 | 开放复杂任务 |

### 4. 什么时候不用 Agent

- 流程完全确定；
- 规则可直接编码；
- 对延迟极敏感；
- 错误代价极高；
- 没有动态工具选择；
- 简单分类或 CRUD。

把固定 if/else 包上 LLM 并不等于 Agent。

### 5. 混合架构

```text
Agent:
理解目标、规划、选择流程

Workflow:
执行工具、重试、审批、补偿、审计
```

Agent 输出受 schema 和权限约束，资金等不可逆动作交给确定性 Workflow。

### 6. 面试总结

Workflow 追求可预测和可靠，Agent 追求在不确定环境中自主决策。实际产品应把动态认知交给 Agent，把关键执行交给状态机和工作流。
