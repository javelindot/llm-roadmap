# TypeScript 中的泛型是什么?

## 答案

TypeScript 泛型是在定义函数、接口、类型或类时先保留类型参数，使用时再传入具体类型。它能让代码复用，同时保留静态类型安全，避免大量使用 `any`。

### 1. 泛型函数

```typescript
function identity<T>(value: T): T {
  return value;
}

const orderId = identity<string>("P123");
const amount = identity<number>(100);
```

`T` 是类型参数。TypeScript 通常也能自动推断：

```typescript
const currency = identity("USD"); // 推断为 string
```

### 2. 泛型接口

跨境支付接口常见统一返回结构：

```typescript
interface ApiResult<T> {
  code: string;
  message: string;
  data: T;
}

interface PaymentOrder {
  id: string;
  amount: number;
  currency: string;
  status: "pending" | "succeeded" | "failed";
}

type PaymentResult = ApiResult<PaymentOrder>;
```

这样 `data` 的类型由调用方决定，既复用结构，又能检查字段。

### 3. 泛型约束

可以用 `extends` 限制类型参数：

```typescript
function getId<T extends { id: string }>(value: T): string {
  return value.id;
}
```

只有包含 `id: string` 的对象才能传入。

结合 `keyof`：

```typescript
function getField<T, K extends keyof T>(obj: T, key: K): T[K] {
  return obj[key];
}

const status = getField(payment, "status");
```

如果写成不存在的字段，编译器会报错。

### 4. 泛型类

```typescript
class Repository<T extends { id: string }> {
  private items = new Map<string, T>();

  save(item: T): void {
    this.items.set(item.id, item);
  }

  findById(id: string): T | undefined {
    return this.items.get(id);
  }
}

const paymentRepo = new Repository<PaymentOrder>();
```

### 5. 泛型默认值

```typescript
interface Page<T, Meta = { total: number }> {
  items: T[];
  meta: Meta;
}
```

调用方不传第二个类型参数时，使用默认类型。

### 6. 和 any 的区别

`any` 会丢失类型检查：

```typescript
function unsafe(value: any): any {
  return value;
}
```

泛型会保留输入和输出关系：

```typescript
function safe<T>(value: T): T {
  return value;
}
```

需要注意：TypeScript 类型主要在编译期工作，编译成 JavaScript 后大部分泛型信息会被擦除。如果需要运行时校验支付回调、API 响应，仍要使用 schema validator。

### 7. 面试总结

泛型的价值是“复用实现，但不牺牲类型信息”。常见用法是泛型函数、`ApiResult<T>`、`Repository<T>`、`T extends ...` 和 `K extends keyof T`。在支付系统中，它适合约束订单、退款、出款等不同 API 数据结构，但不能替代运行时参数校验。
