# 混合检索的原理是什么？Sparse检索和Dense检索的原理及相似度度量是什么？

## 答案

混合检索并行使用 Sparse 和 Dense 召回，再做去重、分数/排名融合与 Rerank。Sparse 擅长精确词、编号和罕见实体，Dense 擅长语义、同义词和跨语言，两者覆盖的错误类型不同。

### 1. Sparse 检索

文档和查询表示为高维稀疏词项向量。经典 BM25：

```text
score(q,d)
= sum_{t in q}
  IDF(t)
  * tf(t,d) * (k1 + 1)
  / (
      tf(t,d)
      + k1 * (
          1 - b
          + b * |d| / avgdl
        )
    )
```

特点：

- 倒排索引，速度快；
- 错误码、SWIFT/SEPA/PIX、国家码和法规编号强；
- 依赖分词和同义词；
- 对语义改写较弱。

学习型 Sparse 如 SPLADE 仍输出词表维度稀疏权重，可用点积/倒排检索。

### 2. Dense 检索

双塔模型生成：

```text
q = E_q(query)
d = E_d(document)
```

常见相似度：

```text
dot(q,d) = q^T d

cos(q,d)
= q^T d / (||q|| ||d||)

L2(q,d) = ||q-d||_2
```

若向量已 L2 normalize，cosine 排名与 dot product 等价。ANN 索引可使用 HNSW、IVF、DiskANN 等。

### 3. 为什么不能直接加分

BM25 和 cosine 的范围、分布不同，直接：

```text
bm25_score + dense_score
```

没有统一意义。常用：

**RRF**

```text
RRF(d)
= sum_i 1 / (k + rank_i(d))
```

只依赖排名，鲁棒且无需校准。

**归一化加权**

对每路做 min-max、z-score 或学习校准后加权；有标注数据时可训练融合模型。

### 4. 完整链路

```text
Query/实体抽取
-> 权限与 metadata filter
-> BM25 TopK + Dense TopK
-> RRF/Weighted Fusion
-> 去重和 parent merge
-> Cross-Encoder Rerank
-> Context Packing
```

### 5. 评估

- 各路 Recall@K；
- 独占正确结果比例；
- 融合后 Recall/NDCG/MRR；
- Rerank 增益；
- 延迟和索引成本；
- 按编号类、语义类、多语言类分桶。
