# 实现一个完整的 Mini RAG 系统

## 答案

一个最小但完整的 RAG 应包含：文档切分、Embedding、向量索引、检索、上下文打包、生成和引用返回。下面用 NumPy 做内存索引，并把 Embedder/LLM 作为可替换接口。

### 1. Mini RAG 代码

```python
from dataclasses import dataclass
from typing import Callable, Iterable
import hashlib
import numpy as np


@dataclass
class Document:
    doc_id: str
    text: str
    source: str


@dataclass
class Chunk:
    chunk_id: str
    doc_id: str
    text: str
    source: str


def split_text(text: str, size: int = 500, overlap: int = 80):
    if not 0 <= overlap < size:
        raise ValueError("overlap must satisfy 0 <= overlap < size")

    start = 0
    while start < len(text):
        end = min(start + size, len(text))
        yield text[start:end]
        if end == len(text):
            break
        start = end - overlap


def l2_normalize(matrix: np.ndarray) -> np.ndarray:
    matrix = np.asarray(matrix, dtype=np.float32)
    norms = np.linalg.norm(matrix, axis=1, keepdims=True)
    return matrix / np.maximum(norms, 1e-12)


class MiniRAG:
    def __init__(
        self,
        embedder: Callable[[list[str]], np.ndarray],
        generator: Callable[[str], str],
    ):
        self.embedder = embedder
        self.generator = generator
        self.chunks: list[Chunk] = []
        self.vectors: np.ndarray | None = None

    def add_documents(self, documents: Iterable[Document]):
        new_chunks = []

        for document in documents:
            for index, text in enumerate(split_text(document.text)):
                digest = hashlib.sha256(
                    f"{document.doc_id}:{index}:{text}".encode()
                ).hexdigest()[:16]
                new_chunks.append(
                    Chunk(
                        chunk_id=digest,
                        doc_id=document.doc_id,
                        text=text,
                        source=document.source,
                    )
                )

        if not new_chunks:
            return

        new_vectors = l2_normalize(
            self.embedder([chunk.text for chunk in new_chunks])
        )
        self.chunks.extend(new_chunks)
        self.vectors = (
            new_vectors
            if self.vectors is None
            else np.vstack([self.vectors, new_vectors])
        )

    def retrieve(self, query: str, top_k: int = 4):
        if self.vectors is None or not self.chunks:
            return []

        query_vector = l2_normalize(self.embedder([query]))[0]
        scores = self.vectors @ query_vector
        indices = np.argsort(scores)[::-1][:top_k]

        return [
            {
                "chunk": self.chunks[index],
                "score": float(scores[index]),
            }
            for index in indices
        ]

    def answer(self, query: str, top_k: int = 4):
        hits = self.retrieve(query, top_k=top_k)
        if not hits:
            return {"answer": "知识库中没有足够信息。", "sources": []}

        context = "\n\n".join(
            f"[{i}] {hit['chunk'].text}"
            for i, hit in enumerate(hits, start=1)
        )
        prompt = f"""你只能根据给定证据回答。
证据不足时明确说不知道，不得补充未出现的事实。
每个关键结论使用 [编号] 引用。

证据：
{context}

问题：{query}
答案："""

        return {
            "answer": self.generator(prompt),
            "sources": [
                {
                    "id": i,
                    "source": hit["chunk"].source,
                    "chunk_id": hit["chunk"].chunk_id,
                    "score": hit["score"],
                }
                for i, hit in enumerate(hits, start=1)
            ],
        }
```

### 2. 完整链路

```text
文档
-> 解析/清洗
-> Chunk + metadata
-> Embedding/index
-> Query embedding
-> Top-K retrieval
-> 去重/rerank/打包
-> LLM
-> 引用与无答案
```

代码为面试版，生产还需 BM25 混合召回、ACL filter、reranker、持久化向量库、批量索引、增量删除、缓存、trace、评测和故障降级。

### 3. 如何评测

- Retrieval：Recall@K、MRR、NDCG；
- Context：precision、证据覆盖和重复率；
- Answer：正确率、faithfulness、引用支持率、无答案识别；
- System：P95、吞吐、成本和权限泄漏。

### 4. 跨境支付场景

知识库适合产品规则和 SOP，订单状态、余额、汇率等实时事实应走授权工具。检索前必须按商户、国家和角色过滤，不能先召回越权文档再要求模型忽略。

### 5. 面试总结

Mini RAG 的最小闭环是“切分、向量化、检索、打包、生成、引用”。手写时重点保证接口清楚、向量归一化、无知识兜底和来源可追踪。
