# 是否有微调经验及总结

## 答案

必须按真实经历回答。没有做过大规模全参训练，也可以介绍分类模型、Embedding、reranker 或 LoRA，但要讲完整闭环。

### 回答模板

> 我微调过[模型与版本]解决[任务]。数据来自[来源]，经过[清洗、去重、脱敏]后约[样本/token 数]，并按[用户/时间/实体]切分防止泄漏。最终选择[全参/LoRA/QLoRA/SFT/DPO]，目标模块和超参数是[真实配置]。
>
> 训练使用[卡型和数量]、[精度]、有效 batch [数值]，监控 loss、梯度、吞吐和[业务指标]。最大的训练问题是[真实 bad case]，我通过[排查过程]定位并修复。最终在[评测口径]上取得[真实结果]，上线经过[影子/灰度/回滚]。

### 可总结的经验

- 数据质量、模板和 label mask 往往比盲目增大 rank 更重要；
- 训练 loss 不能替代 free-running 生成和业务评测；
- LoRA 降低优化器与梯度显存，但基础权重和激活仍存在；
- 知识时效问题优先用 RAG，偏好问题用 DPO/RL，权限问题用系统约束；
- 保留基座与通用能力回归，防止过拟合和灾难性遗忘；
- 每个实验记录数据、代码、配置、seed 和 checkpoint。

跨境支付微调必须脱敏，并按国家、语言、币种和风险类型切片；资金事实来自工具，不能用训练数据记忆替代实时查询。

### 面试总结

微调经验要证明自己处理过数据、目标、资源、稳定性、评测和上线，而不是只会调用 Trainer。所有规模、指标和个人贡献都应真实可核对。
