# 大概问的都差不多，主要是简历的东西，剩下一点时间会问一些八股。

## 答案

这类面经描述说明该轮面试以简历深挖为主，少量基础题用于验证知识是否与项目匹配。准备重点应是让简历中的每一句话都能被连续追问。

### 项目准备

每个核心项目至少准备：

- 业务背景、目标、约束和自己的职责边界；
- 数据来源、清洗、切分和评测方式；
- 基线方案、技术选型与未选其他方案的原因；
- 关键实现、架构、性能和异常处理；
- 结果、A/B 测试或离线指标，以及指标可信度；
- 最大失败、排障过程和后续优化。

对“提升 20%”“支持百万数据”等数字，要能说明分母、口径、测试环境和自己贡献。对简历中的每个技术名词，要能讲原理、参数、优缺点和替代方案。

### 八股准备

基础题不要孤立背诵，应围绕项目展开。例如项目用了 RAG，就准备 chunk、embedding、ANN、rerank、评测和权限；用了微调，就准备 Transformer、LoRA、训练显存、数据与过拟合；用了 Java 服务，就准备集合、并发、JVM、数据库和网络。

### 面试表达

先用一到两分钟给出项目全貌，再根据追问逐层深入。不会的问题明确说已知边界，并给出排查或验证思路，不用模糊术语掩盖。面试前可让同伴对简历连续追问三层，检查是否存在“写了但讲不清”的内容。

这种面试的核心不是项目数量，而是候选人是否真正做过、能否解释决策，并能从结果中总结工程方法。
