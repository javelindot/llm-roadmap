# 显存占用和哪些因素有关?

## 答案

显存占用要区分训练和推理。通用拆分是：

```text
总显存 =
模型参数
+ 梯度
+ 优化器状态
+ 激活/KV Cache
+ 临时 workspace
+ 通信 buffer
+ 框架预留与碎片
```

### 1. 模型参数

近似为：

```text
参数量 * 每参数字节数
```

- FP32：4 Byte；
- FP16/BF16：2 Byte；
- INT8：约 1 Byte；
- INT4：约 0.5 Byte，另有 scale/zero-point 开销。

训练时混合精度还可能保留 FP32 master weights。

### 2. 梯度和优化器状态

全参训练需要梯度。AdamW 还为每个参数保存一阶、二阶矩，因此优化器状态常比参数本身更占显存。

影响因素：

- 全参训练还是 LoRA；
- AdamW、SGD、Adafactor 或 8-bit optimizer；
- ZeRO/FSDP 是否分片；
- optimizer/parameter offload。

### 3. 激活

激活与以下因素相关：

- micro-batch size；
- sequence length；
- hidden size；
- layer 数量；
- attention 实现；
- 是否保存中间张量；
- 图像分辨率和视觉 token 数。

长序列下激活可能成为主要瓶颈。可以使用 gradient checkpointing、FlashAttention、sequence packing 和减小 micro-batch。

### 4. 推理 KV Cache

自回归推理中：

```text
KV bytes ≈
2 * layers * batch * seq_len
* num_kv_heads * head_dim * bytes
```

并发数、上下文长度和输出长度越大，KV Cache 越大。GQA/MQA、KV 量化和 PagedAttention 可降低或改善管理成本。

### 5. 临时与系统开销

- GEMM/Attention workspace；
- CUDA graph；
- NCCL bucket；
- logits；
- allocator reserved memory；
- 内存碎片；
- 缓存未释放；
- 意外保留计算图。

将带梯度的 `loss` 或 hidden state 长期放进列表，会造成显存逐步增长。

### 6. 如何分析

不能只看 `nvidia-smi`，还应使用：

- `torch.cuda.max_memory_allocated()`；
- `max_memory_reserved()`；
- PyTorch memory snapshot/profiler；
- DeepSpeed/FSDP 内存估算；
- 按 step、算子和 rank 记录峰值。

优化时要同时报告显存、tokens/s、MFU 和训练时长。显存下降但重计算或通信翻倍，不一定更优。

### 7. 面试总结

训练显存主要看参数、梯度、优化器状态和激活；推理显存主要看权重与 KV Cache。batch、序列长度、精度、并行方式和框架临时 buffer 都会影响峰值，必须先 profile 再选择量化、checkpoint、ZeRO 或 offload。
