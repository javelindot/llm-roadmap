# 八股:解释BERT的模型结构和原理

## 答案

BERT 是 encoder-only Transformer，核心特点是双向上下文建模。它不像 GPT 那样从左到右生成文本，而是一次性看完整个输入序列，更适合理解类任务，例如分类、匹配、检索排序、序列标注和抽取式问答。

### 1. 模型结构

BERT 的输入通常由三部分 embedding 相加：

```text
Input Embedding = Token Embedding + Position Embedding + Segment Embedding
```

- **Token Embedding**：词或子词；
- **Position Embedding**：位置编码；
- **Segment Embedding**：区分句子 A/B，常用于句对任务；
- **[CLS]**：放在开头，常用于整句分类；
- **[SEP]**：分隔句子或段落。

主体是多层 Transformer Encoder，每层包括：

```text
Multi-Head Self-Attention
-> Add & LayerNorm
-> Feed Forward Network
-> Add & LayerNorm
```

### 2. Self-Attention 原理

每个 token 会生成 Q、K、V：

```text
Attention(Q, K, V) = softmax(QK^T / sqrt(d_k)) V
```

BERT encoder 里没有 causal mask，所以每个位置可以同时关注左边和右边的 token。例如理解“银行账户被冻结”时，模型可以同时看“银行账户”和“被冻结”，得到更好的语义表示。

### 3. 预训练任务

经典 BERT 使用两个预训练任务：

- **MLM**：随机 mask 一部分 token，让模型根据上下文预测被 mask 的词；
- **NSP**：判断两个句子是否连续，后来很多改进模型弱化或取消了 NSP。

MLM 让 BERT 能学习双向语义表示，但因为它不是自回归训练，所以不擅长直接长文本生成。

### 4. 下游使用方式

- 文本分类：取 `[CLS]` 表示接分类头；
- 序列标注：对每个 token 接分类头，例如 NER；
- 句子匹配：输入 `[CLS] sentenceA [SEP] sentenceB [SEP]`；
- 检索/Rerank：对 query-doc pair 打相关性分数；
- 抽取式问答：预测 start/end span。

跨境支付里，BERT 类模型可以用于：

- 工单意图分类；
- 支付错误码语义匹配；
- 商户问题和知识库文档相关性排序；
- 命名实体识别，例如订单号、币种、国家、通道；
- 风险文本分类。

### 5. BERT 和 GPT 的区别

- BERT 是 encoder-only，适合理解；GPT 是 decoder-only，适合生成；
- BERT 双向注意力，无 causal mask；GPT 使用 causal mask；
- BERT 预训练目标是 MLM，GPT 是 next-token prediction；
- BERT 通常要接任务头微调，GPT 可以直接指令生成。

### 6. 面试总结

BERT 的本质是用 Transformer Encoder 做双向语义表示学习。它通过 MLM 预训练获得上下文理解能力，适合分类、匹配、抽取和 Rerank。业务落地时，它可以作为跨境支付客服、风控和知识库检索中的理解模型，但不适合作为长文本生成主力。
