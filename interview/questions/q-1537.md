# 手撕MQA（Multi-Query Attention）

## 答案

MQA（Multi-Query Attention）让所有 Query heads 共享同一组 Key/Value head。Query 仍有多个头保持不同关注模式，但 KV Cache 从 `num_heads` 份降为 1 份。

### 1. PyTorch 实现

```python
import math
import torch
from torch import nn


class MultiQueryAttention(nn.Module):
    def __init__(self, hidden_size: int, num_heads: int):
        super().__init__()
        if hidden_size % num_heads != 0:
            raise ValueError("hidden_size must be divisible by num_heads")

        self.hidden_size = hidden_size
        self.num_heads = num_heads
        self.head_dim = hidden_size // num_heads

        self.q_proj = nn.Linear(hidden_size, hidden_size, bias=False)
        self.k_proj = nn.Linear(hidden_size, self.head_dim, bias=False)
        self.v_proj = nn.Linear(hidden_size, self.head_dim, bias=False)
        self.o_proj = nn.Linear(hidden_size, hidden_size, bias=False)

    def forward(
        self,
        x: torch.Tensor,
        attention_mask: torch.Tensor | None = None,
        past_key_value=None,
        use_cache: bool = False,
    ):
        batch, query_len, _ = x.shape

        q = self.q_proj(x).view(
            batch, query_len, self.num_heads, self.head_dim
        )
        q = q.transpose(1, 2)  # [B, H, Tq, D]

        # MQA 只有一个 KV head。
        k = self.k_proj(x).unsqueeze(1)  # [B, 1, Tq, D]
        v = self.v_proj(x).unsqueeze(1)  # [B, 1, Tq, D]

        past_len = 0
        if past_key_value is not None:
            past_k, past_v = past_key_value
            past_len = past_k.size(2)
            k = torch.cat([past_k, k], dim=2)
            v = torch.cat([past_v, v], dim=2)

        key_len = k.size(2)
        scores = torch.matmul(q, k.transpose(-2, -1))
        scores = scores / math.sqrt(self.head_dim)

        query_positions = past_len + torch.arange(
            query_len, device=x.device
        )
        key_positions = torch.arange(key_len, device=x.device)
        causal = key_positions.unsqueeze(0) <= query_positions.unsqueeze(1)
        scores = scores.masked_fill(
            ~causal.unsqueeze(0).unsqueeze(0),
            torch.finfo(scores.dtype).min,
        )

        if attention_mask is not None:
            # attention_mask: [B, key_len], True 表示可见。
            scores = scores.masked_fill(
                ~attention_mask[:, None, None, :].bool(),
                torch.finfo(scores.dtype).min,
            )

        probs = torch.softmax(scores.float(), dim=-1).to(q.dtype)
        output = torch.matmul(probs, v)  # v 自动广播到 H 个头
        output = output.transpose(1, 2).contiguous()
        output = output.view(batch, query_len, self.hidden_size)
        output = self.o_proj(output)

        if use_cache:
            return output, (k, v)
        return output
```

为突出 MQA，代码省略了 RoPE、dropout 和 fused kernel。

### 2. 张量形状

```text
Q: [B, H, Tq, D]
K: [B, 1, Tk, D]
V: [B, 1, Tk, D]
Attention: [B, H, Tq, Tk]
```

矩阵乘时 K/V 的单 head 沿 `H` 广播，不需要物理复制。

### 3. 与 MHA/GQA 比较

| 方法 | Query heads | KV heads | KV Cache |
|---|---:|---:|---:|
| MHA | H | H | 最大 |
| GQA | H | G，`1<G<H` | 折中 |
| MQA | H | 1 | 最小 |

MQA 的 KV Cache 约为同 head_dim MHA 的 `1/H`，能降低 Decode 显存和内存带宽；代价是所有 Query heads 共享 KV，模型容量和质量可能下降。GQA 是更常见的折中。

### 4. 面试总结

MQA 不是单头 Attention：Q 仍是多头，只有 K/V 共享。手写时最关键的是投影维度、广播、causal mask 和缓存拼接。
