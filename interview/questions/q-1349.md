# LoRA的原理是什么？LoRA是否只能在Linear层插入？为什么不能插入在LayerNorm之后？这会对训练稳定性造成什么影响？

## 答案

LoRA 通过低秩矩阵学习预训练权重的增量。它最常作用于 Linear/Conv 的二维权重，但“只能插 Linear”“不能放在 LayerNorm 后”都不是严格规则。关键是适配哪个参数，以及新增分支是否破坏归一化和残差尺度。

### 1. LoRA 原理

```text
W' = W + Delta W
Delta W = (alpha / r) * B A
```

冻结 `W`，只训练 `A/B`。它假设下游权重更新具有较低有效秩。

### 2. 为什么最常加在线性层

Attention 的 Q/K/V/O 和 FFN 都是大二维矩阵：

- 参数量大，节省显著；
- 低秩矩阵乘自然；
- 合并后仍是同形状权重；
- 高效 kernel 和框架支持成熟。

Conv2D 可通过展平 kernel 或专用 LoRA 适配；Embedding 也可使用专门实现。

### 3. LayerNorm 参数为什么不适合经典 LoRA

LayerNorm/RMSNorm 的主要可训练参数是：

```text
gamma: [d]
beta:  [d]
```

它们是一维缩放/偏置，不是大的二维矩阵，做低秩分解没有明显意义。若要适配，可以：

- 直接解冻 gamma/beta；
- 使用 IA3/scale tuning；
- BitFit 训练 bias；
- 在相邻 Linear 层加 LoRA。

### 4. “LayerNorm 后能否加”

现代 Pre-Norm Block 本来就是：

```text
h = Norm(x)
y = x + Linear_with_LoRA(h)
```

也就是说 LoRA 线性层天然接收 Norm 后的输入，这完全正常。

不推荐的是凭空在 Norm 后增加一个未经缩放的大残差 Adapter：

```text
h = Norm(x)
y = x + Sublayer(h) + Adapter(h)
```

若 Adapter 非零初始化、alpha 过大或缺少门控，可能改变残差分支尺度，造成激活放大、训练初期震荡和基座能力退化。

### 5. 稳定设计

- 一个低秩分支零初始化，使初始 `Delta W=0`；
- 合理 alpha/r 或 rsLoRA scaling；
- Pre-Norm 架构中保持原残差路径；
- 小学习率和梯度裁剪；
- 监控 activation/gradient norm；
- 对 Norm 参数通常不做 weight decay；
- 做插层位置和 target module 消融。

### 6. 面试总结

LoRA 是“适配权重矩阵”的方法，不是“在某层后插一块模块”的固定位置规则。LayerNorm 后接带 LoRA 的 Linear 很常见；直接改 Norm 或新增残差分支需单独设计尺度。
