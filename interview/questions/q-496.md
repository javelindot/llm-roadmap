# Lora的损失函数，优势和缺点

## 答案

**一句话**：LoRA 本身不改变任务损失函数，损失仍然取决于训练任务，比如 SFT 用交叉熵；LoRA 只是改变参数更新方式。

### 1. LoRA 的 loss

如果做 SFT，LoRA 的 loss 还是 next-token cross entropy；如果做分类，就是分类交叉熵；如果做 DPO，就是 DPO loss。LoRA 只是在模型层插入低秩可训练矩阵，冻结原模型参数。

### 2. 优势

- 显存低。
- 训练快。
- 适合多任务 adapter。
- 部署灵活。
- 不容易破坏原模型全部能力。

### 3. 缺点

- 表达能力有限。
- rank 选择需要调参。
- 对深层领域能力改造不如全参。
- 数据差同样会过拟合。

### 4. 结合跨境支付项目怎么说（面试口径）

> 在跨境支付客服话术微调里，LoRA 的 loss 还是 SFT 的交叉熵。我们训练的是让模型根据用户问题和上下文生成标准回复，比如付款失败解释、KYC 驳回说明。
>
> LoRA 的好处是成本低，可以针对客服、风控解释、合规问答分别训练 adapter。但我不会用 LoRA 记忆实时规则，规则变化快的部分还是放 RAG 和工具里。

**钩子**：可以引到“LoRA 不改变 loss、SFT 交叉熵、adapter 多场景、规则用 RAG”。
