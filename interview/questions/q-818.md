# LoRA 和 Prefix Tuning 的区别？在什么场景下选择 LoRA？

## 答案

LoRA 修改线性层的权重增量，Prefix Tuning 则在每层 Attention 中加入可学习的虚拟 K/V 前缀。

### 1. LoRA

\[
W'=W+\frac{\alpha}{r}BA
\]

冻结 \(W\)，训练低秩矩阵 \(A,B\)。可注入 Q/K/V/O 和 FFN，推理时可合并到原权重；参数较少，复杂任务上通常稳定。

### 2. Prefix Tuning

为每层生成长度为 \(m\) 的可学习 prefix：

\[
K'=[K_{\text{prefix}};K],\qquad
V'=[V_{\text{prefix}};V]
\]

主干冻结，只训练 prefix 或其重参数化网络。参数可非常少，但每个请求都会占用额外上下文/KV，prefix 过长会增加推理成本，而且它通过注意力上下文间接控制模型。

### 3. 如何选择

优先 LoRA：

- 任务和领域偏移较大；
- 需要改动 Attention/FFN 表征；
- 希望训练稳定、生态成熟；
- 单 adapter 部署且可合并权重；
- 复杂生成、工具调用或领域适配。

可考虑 Prefix：

- 参数预算极小；
- 多任务需快速切换软前缀；
- 不希望改动权重文件；
- 任务主要是行为引导，且可接受额外 KV 成本。

多租户动态 adapter 场景中，LoRA 若无法合并也会增加调度和小矩阵开销，需与 prefix 做真实压测。

### 面试总结

LoRA 学低秩权重增量，Prefix Tuning 学每层虚拟 K/V。LoRA 容量与通用性通常更强且可合并，Prefix 参数更少但占上下文和 KV，选型取决于任务偏移、部署与多租户需求。
