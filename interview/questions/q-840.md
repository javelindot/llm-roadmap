# 全参数微调 7B 需要多少显存

## 答案

没有单一答案，取决于精度、优化器、激活、序列长度和 ZeRO/FSDP 分片。先估算模型状态，再加激活和运行时空间。

### 1. AdamW 混合精度模型状态

以 7B 参数为例，常见每参数占用：

- BF16/FP16 参数：2 Bytes，约 14 GB；
- 梯度：2 Bytes，约 14 GB；
- FP32 master weights：4 Bytes，约 28 GB；
- Adam 一阶、二阶动量：8 Bytes，约 56 GB。

合计约：

\[
7\times10^9\times16\text{ Bytes}
\approx112\text{ GB}
\]

不同实现可能没有独立 FP32 master weight，或梯度保存 FP32，因此模型状态大约在 84～126 GB。

### 2. 激活与临时空间

激活随 micro-batch、序列长度、层数和隐藏维增长；Attention、logits、通信 bucket 和 allocator 还需额外空间。开启 gradient checkpointing 后，一个 7B 全参 SFT 的总峰值常仍可能在 **120～160 GB 甚至更高**，长序列和大 batch 会继续增加。

因此通常不能在单张 80GB GPU 上直接完成普通 AdamW 全参训练。

### 3. 多卡方案

- ZeRO-1 只分 optimizer；
- ZeRO-2 再分 gradient；
- ZeRO-3/FSDP 再分 parameter；
- activation checkpointing、FlashAttention、BF16 和 sequence parallel 继续省显存。

使用 ZeRO-3/FSDP 后，4 张 40/48GB 或 2 张 80GB 是否可行仍要看激活与吞吐；“能放下”不等于性能合理，应实际 profiler。

### 4. 与 LoRA 比较

LoRA 冻结基座，不保存全量梯度和 Adam 状态；7B BF16 基座约 14GB，再加激活和少量 adapter 状态，资源显著降低。QLoRA 进一步量化基座。

### 面试总结

7B 全参 AdamW 的模型状态约 112GB，连同激活通常需 120～160GB 以上。准确答案必须给出精度、序列、batch、checkpoint 和分片策略，不能只报“几张卡”。
