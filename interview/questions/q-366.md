# Neo4j的实体抽取怎么做的？

## 答案

**一句话**：Neo4j 的实体抽取一般是先从文本或业务表里抽实体和关系，再做归一、消歧、去重，最后写入 Neo4j 形成图谱。

### 1. 实体抽取来源

数据可以来自：

- 业务数据库。
- 客服工单。
- KYC/KYB 文档。
- 银行回执。
- 通道错误说明。
- 风控日志。
- 合同或制度文档。

其中结构化数据可以直接映射，非结构化文本需要模型或规则抽取。

### 2. 抽取方法

- 规则/正则：抽订单号、金额、币种、证件号。
- 词典匹配：国家、银行、通道、产品名。
- BERT/NER：抽公司名、人名、地址等。
- LLM 信息抽取：从复杂文本中抽实体和关系。
- 业务规则校验：校验实体合法性和关系可信度。

### 3. 写入 Neo4j

写入前要做：

- 实体归一，比如同一公司不同写法合并。
- 消歧，比如同名公司区分不同主体。
- 去重。
- 关系类型定义。
- 置信度记录。
- 来源记录。

Neo4j 里通常会有节点和关系，比如 `Merchant - OWNS -> Account`，`Account - USES -> Device`。

### 4. 结合跨境支付项目怎么说（面试口径）

> 在跨境支付风控场景里，Neo4j 可以用来构建商户、法人、账户、收款人、设备、IP、银行卡、国家和风险标签之间的关系图。实体抽取一部分来自结构化表，比如商户表、账户表、交易表；另一部分来自 KYC 材料、工单和银行回执，需要用 OCR、NER 或大模型抽取。
>
> 比如从 KYB 材料里抽公司名、注册号、法人、地址；从交易日志里抽收款账户、设备指纹、IP；再把这些关系写到 Neo4j。后续风控可以查询“某个收款账户是否和多个高风险商户共享设备或 IP”。
>
> 我会特别注意实体归一和来源追溯。公司名大小写、缩写、翻译都可能不同，如果不归一，图谱会碎片化；每条关系最好带来源和时间，方便审计。

**钩子**：可以引到“结构化 + 非结构化抽取、实体归一、关系路径、风控关联网络”。
