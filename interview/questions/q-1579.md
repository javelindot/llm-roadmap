# SFT的核心流程、数据集构建策略以及SFT后的Post-Training方法及其目的区别是什么？

## 答案

SFT 本身通常就是 Post-Training 的第一阶段。它负责教模型“如何按指令完成任务”；SFT 后的偏好优化、RL、安全训练和蒸馏则分别解决排序偏好、策略探索、边界和效率问题。

### 1. SFT 核心流程

```text
定义能力与评测
-> 收集/构造数据
-> 清洗、去重、分层切分
-> 统一 chat template/tokenizer
-> 选择全参或 PEFT
-> 训练与 checkpoint
-> 离线评测和能力回归
-> shadow/canary
```

训练目标通常为：

```text
L_SFT
= -sum assistant_tokens
  log p(y_t | prompt, y_<t)
```

system/user/padding 常设为 ignore，只在 assistant 输出计算 loss；工具轨迹则按目标决定是否训练 tool call 和 observation 后续回答。

### 2. 数据集构建

来源：

- 专家示范；
- 脱敏真实优质记录；
- 文档/SOP 转换；
- 可执行工具轨迹；
- 规则或强模型生成并验证；
- 有许可证的公开数据；
- 线上 bad case。

要求：

- 覆盖任务、语言、难度、长度和边界；
- 加入澄清、无答案、拒答和转人工；
- JSON/代码/工具参数程序校验；
- 按用户、文档簇和时间防泄漏切分；
- 记录来源、版本、质量和许可证；
- 通过 data scaling curve 决定规模。

### 3. 微调方式

- 全参数：容量大、成本高；
- LoRA：默认领域适配起点；
- QLoRA：显存更紧；
- Adapter/Prompt/Prefix/IA3；
- 继续预训练：领域语言知识缺口较大时使用。

知识频繁更新时优先 RAG/工具，不靠 SFT 记忆。

### 4. SFT 后的方法

**DPO/偏好优化**

使用 chosen/rejected，学习多个可行回答中哪个更好。适合风格、帮助性和安全偏好，链路比在线 RL 简单。

**PPO/GRPO/RLVR**

模型在线 rollout，通过 reward/verifier 优化序列级结果。适合数学、代码、工具和多步策略，但成本与风险更高。

**Reward Model**

将人类/规则偏好训练成评分器，可用于 PPO、best-of-N 或数据筛选。

**Distillation**

把强模型回答、logits 或轨迹迁移到小模型，降低部署成本。

**Safety Tuning**

训练拒答、越权、提示注入和高风险边界，并结合确定性 guardrail。

### 5. 目的区别

| 方法 | 主要学习信号 | 主要目的 |
|---|---|---|
| SFT | 黄金示范 | 学任务和格式 |
| DPO | 偏好对 | 学回答排序 |
| PPO/GRPO | 在线 reward | 探索和优化结果 |
| RM | 排序/评分 | 建模奖励 |
| Distill | 教师行为 | 压缩与迁移 |
| Safety | 对抗/边界 | 降低风险 |

### 6. 跨境支付场景

SFT 教模型调用订单工具和按 SOP 回答；DPO 偏好准确合规的表达；RL 可优化沙箱中的调查轨迹；资金动作、制裁和权限始终由系统硬约束。

### 7. 面试总结

SFT 建立基础行为，后续方法针对偏好、策略、安全和成本做增量优化。是否继续做 DPO/RL 应由 SFT 后的具体错误类型决定。
