# 🔥🔥 DPO的原理

## 答案

DPO 直接利用偏好对优化语言模型，不需要单独训练 Reward Model，也不需要在训练循环中进行 PPO rollout。

### 1. 从 KL 约束 RLHF 出发

标准目标可写为：

\[
\max_\pi\;
\mathbb E_{y\sim\pi}[r(x,y)]
-\beta D_{\mathrm{KL}}(\pi\Vert\pi_{\mathrm{ref}})
\]

其最优策略满足：

\[
r(x,y)=\beta\log
\frac{\pi^*(y|x)}{\pi_{\mathrm{ref}}(y|x)}
+C(x)
\]

将这个隐式 reward 代入 Bradley-Terry 偏好模型，得到 DPO loss：

\[
\mathcal L_{\mathrm{DPO}}
=-\log\sigma\left(
\beta\left[
\log\frac{\pi_\theta(y_w|x)}
{\pi_{\mathrm{ref}}(y_w|x)}
-
\log\frac{\pi_\theta(y_l|x)}
{\pi_{\mathrm{ref}}(y_l|x)}
\right]\right)
\]

它提高 chosen 相对 reference 的概率优势，降低 rejected 的相对优势。

### 2. 训练流程

1. 从同一 prompt 构造 chosen/rejected；
2. 冻结 reference model；
3. 分别计算 policy 与 reference 的序列 log probability；
4. 优化 pairwise logistic loss；
5. 用独立偏好、能力和安全集验证。

### 3. 优点与局限

优点是稳定、实现简单、不需要 value model 和在线采样。局限包括：

- 依赖离线数据覆盖，无法主动探索新轨迹；
- 对长度、标注者和生成模型偏差敏感；
- \(\beta\) 控制相对 reference 的偏移；
- reference 选择和 token 平均/求和口径影响结果；
- 多步可验证任务可能更适合在线 RL。

依据可参考 [DPO 原论文](https://arxiv.org/abs/2305.18290)。

### 跨境支付场景

可用偏好对优化客服表达、引用完整性与拒答边界，但资金权限、真实订单和制裁规则仍需工具与工作流硬约束。

### 面试总结

DPO 将 KL 正则 RLHF 的最优策略重参数化为隐式 reward，直接用 chosen/rejected 做分类式优化。它比 PPO 简单稳定，但受离线偏好覆盖限制，不等于能替代所有在线 RL。
