# 系统的状态有哪些指标

## 答案

“系统状态指标”应从请求、资源、依赖、业务和模型五层观察。常用框架是四个黄金信号：Latency、Traffic、Errors、Saturation。

### 1. 请求层

- QPS/RPS；
- 并发数；
- P50/P95/P99 延迟；
- 成功率和错误率；
- 超时率；
- 重试率；
- 请求大小和响应大小；
- 队列长度和等待时间。

平均延迟可能掩盖尾延迟，因此生产系统必须看 P95/P99。

### 2. 资源层

- CPU 使用率和 load；
- 内存使用、GC、OOM；
- GPU 利用率、显存和温度；
- 磁盘 IOPS、吞吐和空间；
- 网络带宽、丢包、连接数；
- 线程池、连接池、文件句柄；
- 容器重启和节点健康。

### 3. 存储与中间件

- 数据库连接、慢查询、锁等待；
- 主从延迟；
- 缓存命中率、淘汰率、热 key；
- MQ backlog、消费延迟和死信；
- 搜索索引延迟和新鲜度；
- 外部 API 成功率与响应时间。

### 4. 模型/RAG/Agent

- TTFT、TPOT、tokens/s；
- batch size 和排队时间；
- KV Cache 使用率；
- GPU OOM；
- 检索延迟和 Recall proxy；
- 工具调用成功率；
- Agent 步数、重试、超时和任务完成率；
- 模型版本和降级比例；
- Token 成本。

### 5. 业务状态

跨境支付可监控：

- 支付、退款、出款成功率；
- 各通道失败率和延迟；
- pending 订单积压；
- 账务对账差异；
- 拒付和风险命中；
- KYC 审核积压；
- 重复请求拦截；
- 工单转人工和重开率。

技术指标正常但支付成功率突然下降，系统仍然是不健康的。

### 6. 告警原则

- 使用 SLI/SLO 和 error budget；
- 告警面向用户影响，不只看单机 CPU；
- 静态阈值与同比/环比异常结合；
- 区分 warning、critical；
- 告警要能定位服务、版本、地域和依赖；
- 配套 runbook 和自动降级。

### 7. 面试总结

系统状态不能只答 CPU、内存。应覆盖流量、延迟、错误、饱和度、存储依赖、模型链路和业务结果，并用 tracing 和版本标签关联，才能判断问题发生在哪一层。
