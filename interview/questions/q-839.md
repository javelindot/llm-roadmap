# RAG遇到模型缺失电商知识一般怎么做

## 答案

先判断缺失的是“稳定商品知识”“实时业务状态”还是“领域推理能力”，三者修复方式不同。

### 1. 稳定商品知识

- 接入商品 PIM、标题、属性、类目、说明书、FAQ 和审核内容；
- 结构化字段与正文分别索引，建立商品 ID 与版本；
- BM25 保留 SKU、型号和精确属性，向量召回语义表达；
- 图像、OCR 和文本做多模态表示，解决描述不完整；
- 文档解析、去重、冲突和失效治理；
- 用 hard negative 和领域数据微调 embedding/reranker。

### 2. 实时状态

库存、价格、优惠、配送和上下架不应只进入静态 RAG。召回商品 ID 后，实时调用商品/库存服务校验；展示前过滤，下单时再次原子校验。超时则明确无法确认，不能让模型猜。

### 3. 模型不会使用知识

若正确文档已召回但回答仍错，应优化 prompt、上下文去重、引用约束和生成模型；可用 SFT 教领域表达，但易变事实仍不写进参数。

### 4. 无知识时降级

- 回退传统搜索结果；
- 追问型号、预算和规格；
- 提示知识不足并转人工；
- 将空结果和用户反馈进入知识补全队列，经审核后入库。

### 评估

分层看知识覆盖率、Recall@K、版本正确率、库存一致率、答案忠实度和无证据回答率，而不是只看最终 BLEU 或主观流畅度。

### 面试总结

电商知识缺失先补数据与混合索引，实时价格库存走权威工具，已召回却答错再优化生成。RAG 不应靠模型参数记忆易变商品事实，无证据时必须降级。
