# 请详细描述 Instruct-GPT RLHF 的整体流程及各阶段作用

## 答案

**一句话**：Instruct-GPT 把 base 模型变成"听话助手"的三段式流程——**SFT → RM 训练 → PPO 强化学习**，每段分别教模型"怎么说""怎么评""怎么改进"。

### 1. 阶段 1：SFT（监督微调）
- **数据**：人类标注员写 demonstration（任务指令 + 高质量回答），约 1.3w 条
- **训练目标**：标准 Causal LM loss
- **作用**：
  - 让 base 模型学会"对话格式"
  - 注入"听指令"的行为习惯
  - 为后续阶段提供合理起点
- **产物**：SFT 模型（如 SFT-GPT-3）

### 2. 阶段 2：RM（奖励模型训练）
- **数据**：同一 prompt 让 SFT 模型采样 K 个回复，人类标注员排序（约 3.3w 个 pair）
- **训练目标**：Bradley-Terry pairwise loss
$$L_{RM} = -\log \sigma(r(x, y_w) - r(x, y_l))$$
- **作用**：
  - 把人类偏好编码成可微分函数
  - 后续 PPO 阶段的"老师"
- **产物**：6B Reward Model（Instruct-GPT 用 6B RM，论文证明比 175B RM 更稳定）

### 3. 阶段 3：PPO（强化学习对齐）
- **数据**：纯 prompt（无回答），约 3.1w 条
- **训练流程**：
  1. Actor（待训）从 prompt 采样回复 y
  2. RM 对 (x, y) 打分得 reward r
  3. KL 项约束：r' = r - β × KL(π_actor || π_ref)
  4. Critic 估 value，GAE 算 advantage
  5. PPO clip loss 更新 actor + value loss 更新 critic
- **作用**：
  - 用 RM 信号超越 SFT 数据上限
  - KL 项防止偏离 SFT 太远
  - 学到"如何让 RM 给高分"的策略

### 4. 四个并存的模型
| 模型 | 角色 | 是否训练 |
|------|------|---------|
| Actor | 待训策略 | ✅ |
| Critic | 价值函数 | ✅ |
| Reward Model | 奖励信号 | ❌（冻结） |
| Reference | KL 约束基线 | ❌（冻结，= SFT 起点） |

### 5. 关键超参（论文经验）
- Adam lr = 9.65e-6
- KL 系数 β = 0.02
- PPO clip ε = 0.2
- 每 batch 4 PPO epoch
- 总训练量约 256k episodes

### 6. 各阶段的痛点
- **SFT 痛点**：数据成本高、质量难保证
- **RM 痛点**：标注主观、长度偏置、reward hacking
- **PPO 痛点**：训练不稳、4 模型显存爆、调参困难

### 7. 为什么这套流程"魔法般"有效
- SFT 让模型"知道形式"
- RM 把"主观偏好"量化
- PPO 在 RM 信号下持续探索
- KL 约束防止崩塌
- 三段式分工 → 工程可控

### 8. 一句话总结
**SFT 教格式、RM 教评分、PPO 教改进**——三段式 RLHF 是 LLM 对齐的开山方案，DPO 等新方法都是它的简化或替代。

## 结合项目谈（口语版 · 约 40 秒）

> 我们跨境支付**客服对齐**走过完整的 Instruct-GPT 风格三段式流程，分享下我们落地的版本。
>
> **SFT 阶段**——我们没用 1.3w 条人工写的，而是从历史一年高满意度（4-5 星）客服对话里挖了 8w 条高质量 (问题, 回答) pair 当 demonstration，Qwen-7B 上跑 SFT。这一步主要让模型学**我们品牌的客服语气和格式**。
>
> **RM 训练**——5w 条 (prompt, chosen, rejected)，标注来源 60% 历史评分（高分 vs 低分回复）+ 40% GPT-4 judge 蒸馏。RM 用 Qwen-7B 微调（论文用 6B，规模差不多），输出标量分数。这一步最痛苦的是**长度偏置**和**标注分歧**，我们花了两周对齐标注口径。
>
> **PPO 阶段**——这就是工程地狱。4 个模型同时占显存、KL 系数 β 难调（β=0.02 太松、0.1 又学不动）、reward hacking 严重（模型学到刷"尊敬的客户"开头）。我们调了一个月才把训练跑稳。
>
> 最终评测：PPO 版本比 SFT 版本满意度高 6 个点，但**比直接 DPO 高得不多**——我们综合工程成本之后下一版直接切到 DPO，节省 60% 训练资源。
>
> 我对 Instruct-GPT 流程的总结：**经典但贵**，理解它是底子，工业落地有更便宜的替代（DPO）。
