# Pre-LayerNorm vs Post-LayerNorm

## 答案

**一句话**：Post-LN 是原始 Transformer 的设计、Pre-LN 是后人为了**深层训练稳定**做的改进——**现代 LLM 几乎全用 Pre-LN（或 RMSNorm Pre 变体）**。

### 1. 两种位置示意
**Post-LN（原始 Transformer 2017）**：
```
x' = LayerNorm(x + Attention(x))
y  = LayerNorm(x' + FFN(x'))
```
Norm 在残差**之后**。

**Pre-LN（GPT-2 起的主流）**：
```
x' = x + Attention(LayerNorm(x))
y  = x' + FFN(LayerNorm(x'))
```
Norm 在子层**之前**。

### 2. 核心区别
| 维度 | Post-LN | Pre-LN |
|------|---------|--------|
| 训练稳定性 | 浅层稳，深层易爆 | 深层也稳 |
| 是否需要 warmup | 必须 | 可以省 |
| 对学习率敏感度 | 高 | 低 |
| 最终精度 | 调好略高 | 略低（论文经验） |
| 工业易用度 | 难调 | 简单 |

### 3. 为什么 Post-LN 在深层不稳？
- 残差路径上 norm 反复挤压输出
- 梯度反传时被 norm 缩放
- 信号难穿过几十层，梯度爆/消
- 需要小心 warmup 才能起步

### 4. 为什么 Pre-LN 稳？
- 残差路径是"恒等流"，norm 只作用于子层输入
- 梯度反传时能直接穿过残差
- 即使去掉 warmup 也能稳起步
- 缺点是**最终精度略低**，因为残差通道没经过 norm 调节，输出会越加越大

### 5. 现代 LLM 几乎都用 Pre-LN 变体
- **GPT-2/3/4、Llama、Qwen、Mistral**：Pre-LN
- 大多数 Pre-LN + **RMSNorm**（更轻量）
- 部分用 **Sandwich-LN**（Pre + Post 同时加，CogVLM）
- DeepNorm（Megatron）：post-LN 的稳定版，特殊残差系数

### 6. RMSNorm 顺带提一句
- 去掉 mean 中心化，只做 scale
- 比 LayerNorm 快 10-20%
- 效果几乎一样
- Llama、Qwen 全切了 RMSNorm

### 7. 一句话总结
**现代 LLM 默认 Pre-LN + RMSNorm**——稳、快、好调；只有少数追极致精度的场景才回 Post-LN，但要小心训练崩。

## 结合项目谈（口语版 · 约 40 秒）

> 这块我们跨境支付训练流水线没踩过 Post-LN 的坑——因为我们用的 base 都是 Llama / Qwen 系列，**默认就是 Pre-LN + RMSNorm**，开箱即用。
>
> 但我们做过一个相关的小实验，挺有意思——我们想给客服模型加一层"安全检测头"作为辅助输出，做"是否敏感问题"的二分类。第一版我们加了一个 **Post-LN 风格的分支**，从最后一层 hidden state 出来直接加 norm 再过分类头，训了半天 loss 不动。
>
> 检查发现就是**梯度在 Post-LN 这一层被挤压得很厉害**，分类头几乎学不动。改成 Pre-LN 风格——hidden state 先做 norm 再喂分类头、不再 norm 输出——loss 立刻正常收敛。这是个小细节但让我对 Pre vs Post 的差异有了实感。
>
> 我对这道题的总结：**Pre-LN 是工程默认选项，但要知道为什么**——遇到训练不稳第一反应去查归一化位置和 warmup，多数问题在这里。RMSNorm 我们也默认开，省 10% 算力，效果不掉。

**🪝 顺嘴带出去的钩子**：
- "RMSNorm vs LayerNorm 的速度对比我们有数据。"
- "DeepNorm 和 Sandwich-LN 在 100+ 层模型里的作用可以聊。"
- "Pre-LN 最终精度略低这事在我们业务里有没有体现，可以展开。"
