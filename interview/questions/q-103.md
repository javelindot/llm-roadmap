# 什么是词嵌入（Word Embedding）？

## 答案

**一句话**：词嵌入 = **把离散的词映射到连续的低维向量空间**，让"语义相近的词向量距离也相近"——是 NLP 从符号到向量的桥梁。

### 1. 为什么需要词嵌入
- 文本本身是离散符号（one-hot 几万维稀疏）
- 神经网络擅长处理连续向量
- 同义词在 one-hot 下没有相似性（cat 和 dog 完全正交）
- 嵌入让"语义"成为可计算的几何

### 2. 三代词嵌入

**第一代：one-hot / TF-IDF**
- 稀疏、无语义
- 仅用于早期统计 NLP

**第二代：静态词向量（Word2Vec / GloVe / FastText）**
- 每个词一个固定向量
- Word2Vec 用 Skip-gram / CBOW 学
- 缺点：一词多义无法区分（bank = 银行 / 河岸 同一向量）

**第三代：上下文相关嵌入（ELMo / BERT / LLM）**
- 同一个词在不同上下文有不同向量
- ELMo 用 BiLSTM
- BERT / GPT 用 Transformer
- 这是现代 NLP 默认

### 3. Word2Vec 原理速览
- **CBOW**：用周围词预测中心词
- **Skip-gram**：用中心词预测周围词
- 训练目标：让相关词向量内积大、无关词内积小
- **副产物：embedding 矩阵** = 词表 × dim

### 4. 词嵌入的经典性质
- vec("king") - vec("man") + vec("woman") ≈ vec("queen")
- 类比、聚类、相似度查询都靠这个性质

### 5. 现代 LLM 里的嵌入
- 第一层就是 Embedding 矩阵：`(vocab_size, hidden_dim)`
- Llama 152k × 4096 = 600M 参数
- 训练时和模型一起学
- Tied weights：embedding 和 LM head 共享参数（早期常用）

### 6. 在 RAG / 检索里的嵌入
- 现代 retrieval 用专门的 **embedding model**（bge、e5、gte、jina、Qwen-Embedding）
- 这些是双塔结构，把"query 和 doc 嵌到同一空间"
- 不再是单词级，而是**句子/段落级**
- 距离：cosine / dot product

### 7. Embedding 在工业里的核心用法
- **语义检索**：向量库 + ANN
- **相似度匹配**：客服意图、商品推荐
- **聚类**：客户分群、话题挖掘
- **冷启动**：少样本分类的特征提取

### 8. 一句话总结
**词嵌入 = 把词变成向量，让语义可计算**——从 Word2Vec 静态到 BERT/LLM 动态再到 RAG 双塔，**嵌入是 NLP 一切下游任务的地基**。

## 结合项目谈（口语版 · 约 30 秒）

> 嵌入在我们跨境支付里是**RAG 客服的核心**，落地经验我分享下。
>
> **嵌入模型选型**——一开始用 m3e-base 中文小模型，1.5 万条客服知识库做向量检索，准确率 67%。后来切到 **bge-large-zh-v1.5**，准确率涨到 78%。再后来用 **Qwen3-Embedding-8B**，金融领域召回率涨到 85%。模型规模和效果是正相关，但 8B 推理慢，最后我们用 **8B 离线索引 + bge-large 在线查询**的混合方案。
>
> **嵌入精调**——我们用业务对话挖了 5w 条 (问题, 标准答案) 正样本 + 同问题不同答案做负样本，对 bge-large 做对比学习微调（InfoNCE loss）。微调后业务准确率涨到 88%。
>
> **几个工业坑**：
> - 长文档要切块再嵌入，块太大向量稀释；
> - 嵌入要 normalize 后再做内积，等价 cosine；
> - 不同 embedding 模型不能混用，向量空间不通；
> - 业务术语（如"代付""跨境结算"）要在精调数据里覆盖到。
>
> 一句话：**Embedding 在 RAG 里就是检索的"语义指纹"**，选模型 + 业务精调 + 工程化（切块、normalize）三件事做好，准确率能压到 90%+。

- "Embedding 微调的 InfoNCE 损失和负样本采样有经验。"
- "RAG 长文档切块策略我们做过 ablation。"
