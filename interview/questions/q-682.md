# transformer计算的时间复杂度和空间复杂度，deepseek对transformer主要做了哪些改进?

## 答案

设序列长度为 $n$、隐藏维度为 $d$、FFN 中间维度为 $d_{ff}$。单层标准 Transformer 的主要计算包括线性投影、注意力和 FFN。

### 时间与空间复杂度

Q/K/V/O 投影约为 $O(nd^2)$，计算 $QK^T$ 和 $AV$ 为 $O(n^2d)$，FFN 为 $O(ndd_{ff})$。若 $d_{ff}\approx4d$，单层时间可概括为：

$$
O(nd^2+n^2d)
$$

训练时注意力矩阵占 $O(n^2)$，激活还包括 $O(nd+nd_{ff})$；参数为 $O(d^2+dd_{ff})$。自回归推理使用 KV Cache 后，每生成一个 token 的历史注意力约为 $O(nd)$，但每层缓存随上下文增长为 $O(nd)$。

### DeepSeek 的代表性改进

“DeepSeek”有多个版本，应明确所指。以 DeepSeek-V2 及后续延续的核心设计为例：

- Multi-head Latent Attention：将 K/V 联合压缩到低维 latent，并采用解耦 RoPE，显著减少 KV Cache 与 decode 内存带宽。它主要优化缓存常数和推理效率，并不自动把完整注意力的理论序列复杂度从二次降为线性。
- DeepSeekMoE：将 FFN 替换为稀疏 MoE，只激活部分 routed experts；通过更细粒度专家划分和 shared experts 提高专家专门化与知识共享。在总参数很大的同时控制每 token 的激活计算量。
- 后续版本还采用 Multi-Token Prediction、负载均衡策略和低精度训练等手段，但不能把所有版本特性混为一个固定架构。

MLA 解决的是注意力侧 KV Cache 瓶颈，MoE 解决的是以较低激活计算扩展参数容量。回答时还应区分理论 FLOPs、实际显存带宽和通信开销，MoE 的路由与 all-to-all 可能成为新瓶颈。
