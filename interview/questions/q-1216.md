# rlhf怎么做

## 答案

RLHF 是把难以直接写成规则的“人类偏好”转成可优化的奖励信号。经典 InstructGPT 流程可以概括为：

> 预训练模型 → SFT → 偏好比较数据 → Reward Model → PPO 对齐 → 离线与人工评测。

### 1. 先训练 SFT 模型

收集高质量的 `(prompt, response)` 示范数据，在基础模型上做监督微调。SFT 的作用是先让模型学会任务格式、基本指令跟随和对话行为，否则直接做强化学习的搜索空间太大，训练容易不稳定。

数据需要覆盖正常任务、边界任务、拒答、澄清、多轮上下文和工具调用，训练集与最终评测集按用户、时间和业务来源隔离。

### 2. 构造偏好数据

对同一个 Prompt 从一个或多个策略模型采样多个回答，让标注者根据明确 rubric 排序。常见维度包括：

- 正确性和证据一致性；
- 是否完整执行指令；
- 是否简洁、可操作；
- 是否泄露敏感信息或产生安全风险；
- 应该拒答、澄清或转人工时是否处理正确。

应保存标注者分歧和置信度，而不是把主观问题强行压成唯一答案。高风险领域还要由业务、合规或安全专家复核。

### 3. 训练 Reward Model

通常在语言模型顶部增加标量 reward head。对于偏好对 `(x, y_w, y_l)`，使用 Bradley-Terry 形式：

```text
P(y_w > y_l | x) = sigmoid(r_phi(x, y_w) - r_phi(x, y_l))
```

对应损失为：

```text
L_RM = -log sigmoid(r_phi(x, y_w) - r_phi(x, y_l))
```

Reward Model 的验证不能只看训练 loss，还要看 held-out pair accuracy、不同任务和语言分桶、长度偏差、拒答偏差以及对抗样本。否则模型可能学到“回答越长分越高”等伪规律。

### 4. 使用 PPO 优化策略

从 SFT 模型初始化 policy，同时保留冻结的 reference model。Policy 针对 Prompt 生成回答，Reward Model 给出序列级分数，并加入相对参考模型的 KL 惩罚：

```text
R(x, y) = r_phi(x, y)
          - beta * KL(pi_theta(. | x) || pi_ref(. | x))
```

随后使用 PPO clipped objective 更新策略，value model 估计 return，GAE 计算 advantage。KL 项用于防止策略为了迎合 Reward Model 而远离原模型，实际训练需要持续监控：

- reward、KL、entropy 和生成长度；
- clip fraction、policy loss、value loss；
- reward margin 和拒答率；
- 通用能力、安全能力和语言能力回归。

如果 reward 快速上升但人工胜率不升，通常是 reward hacking、长度偏差或训练分布泄漏，不能继续盲目加大 PPO 步数。

### 5. 评测和上线

RLHF 完成后仍需：

1. 与 SFT、旧版本和 reference model 做盲评；
2. 使用冻结的业务黄金集、安全集和对抗集；
3. 检查事实性、过度拒答、格式、工具成功率和多轮一致性；
4. 影子流量、小比例灰度并保留回滚；
5. 将线上失败样本经脱敏、去重和人工复核后进入下一轮数据闭环。

### 6. 跨境支付中的奖励设计

跨境支付助手不能只优化“用户喜欢”。例如付款状态解释的 reward 可以同时包含人工偏好、事实校验、工具调用成功、合规规则和敏感字段保护：

```text
总奖励 = 回答质量
       + 事实与工具结果一致性
       + 合规和格式通过
       - 越权操作
       - 虚构费率或到账承诺
       - 泄露账户与交易敏感信息
```

资金划转、冻结、放行等动作必须由权限系统、确定性规则和人工审批控制，RLHF 只优化解释、信息收集和辅助决策，不能成为资金状态机本身。
