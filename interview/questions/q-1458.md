# 7.你关于大模型理解的看法。

## 答案

我对大模型的理解是：它首先是通过大规模数据和计算训练出的概率序列模型，在预训练中学习语言、代码和多模态数据的统计结构，再通过 SFT、偏好优化、RL、RAG 和工具使用变成可完成任务的系统。

### 1. 能力从哪里来

- Transformer 提供可扩展的序列建模；
- 大规模预训练获得通用表示和知识；
- 指令微调让模型理解任务格式；
- 后训练改善推理、偏好和安全；
- 工具与环境反馈扩展可执行能力；
- 推理时计算让模型在复杂问题上投入更多搜索。

所谓“涌现”往往也受评测阈值、数据覆盖和规模共同影响，不能把模型能力神秘化。

### 2. 大模型擅长什么

- 非结构化信息理解和生成；
- 跨任务迁移；
- 多语言；
- 代码；
- 文档和多模态；
- 自然语言工具接口；
- 低样本适配。

它把大量过去需要专用模型的任务统一到一个接口上。

### 3. 局限

- 生成目标不等于事实正确；
- 知识有截止时间；
- 推理不保证稳定；
- 对 prompt 和上下文敏感；
- 长链路错误会累积；
- 权限和责任意识不能仅靠模型自觉；
- 成本、延迟和能耗高；
- 训练数据可能带来偏见、版权和隐私问题。

### 4. 工程上如何看

我不把大模型当数据库、规则引擎或最终决策者，而把它看成：

```text
理解器 + 生成器 + 规划器 + 工具接口
```

生产系统还需要：

- RAG；
- 权威 API；
- 状态机；
- 验证器；
- 权限和审批；
- 监控；
- 评测；
- 降级。

### 5. 未来方向

- 原生多模态和实时交互；
- MoE 与高效推理；
- Agent 和可验证环境；
- 长上下文与外部记忆；
- 小模型蒸馏和端侧；
- 训练/推理时计算融合；
- 数据、版权、安全和治理；
- 模型能力与确定性软件更深结合。

### 6. 跨境支付场景

大模型适合商户客服、调查摘要、材料理解、知识检索和运营分析；真实交易、余额、汇率、KYC 和合规结果必须由权威系统提供，高风险动作走规则和人工审批。

### 7. 面试总结

大模型是强大的通用概率模型，但不是全知系统。真正的价值来自模型、数据、检索、工具和治理的组合；工程目标不是让它“看起来聪明”，而是让结果可验证、可控、可运营。
