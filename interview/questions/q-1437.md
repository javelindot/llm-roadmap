# 4．有没有全量微调过

## 答案

这是经验核验题，必须如实回答。全量微调指基座模型的大部分或全部可训练参数都参与反向传播和优化，与只训练 LoRA Adapter 不同。

### 1. 如果做过

参考结构：

> 我对 X 参数规模的模型做过全量 SFT/继续预训练，使用 X 张某型号 GPU，采用 bf16、gradient checkpointing 和 DeepSpeed ZeRO-X/FSDP。训练数据为 X 类任务，最大长度、global batch、学习率和训练 token 分别是实际值。选择全量微调是因为任务需要较大能力迁移/领域表示更新，LoRA baseline 在关键指标上不足。

继续说明：

- Dataset/Collator；
- loss mask；
- optimizer/scheduler；
- ZeRO stage；
- 每卡显存；
- tokens/s；
- checkpoint 保存与恢复；
- 通用能力回归；
- 与 LoRA 的消融。

### 2. 如果没做过

可以诚实回答：

> 我目前主要做过 LoRA/QLoRA，没有完成过生产级全量微调。原因是模型规模和资源预算不匹配。不过我理解完整流程，也做过显存估算和训练方案设计：全参训练需要保存参数、梯度、Adam 状态和激活，通常要用 ZeRO/FSDP、checkpointing 和 bf16。我不会把 LoRA 说成全量微调。

### 3. 全量微调与 LoRA 对比

| 维度 | 全量微调 | LoRA |
|---|---|---|
| 可训练参数 | 全部或大部分 | 低秩 Adapter |
| 显存与通信 | 高 | 低 |
| 能力上限 | 通常更高 | 任务依赖 |
| 小数据过拟合 | 风险较高 | 相对受限 |
| checkpoint | 大 | 小 |
| 多任务切换 | 成本高 | 灵活 |

### 4. 什么时候值得全量微调

- 大规模高质量数据；
- 需要改变底层语言或领域表示；
- LoRA 明显欠拟合；
- 有足够算力和训练基础设施；
- 能做完整通用能力和安全回归。

只做格式、术语或工具调用适配时，LoRA 往往更经济。

### 5. 跨境支付场景

支付领域数据通常敏感且规模有限，优先评估 RAG、工具和 LoRA。只有在有大规模合规语料、明确收益和完善回归评测时，才考虑全量继续预训练或微调。

### 6. 面试总结

回答关键是给出可验证的模型、硬件、数据、配置和结果。没做过就明确边界，再展示自己理解资源估算和方案设计，可信度比虚构经验更重要。
