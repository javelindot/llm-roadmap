# Transformer中自注意层计算公式

## 答案

**一句话**：Transformer 自注意力的核心公式是 `Attention(Q,K,V)=softmax(QK^T/sqrt(d_k))V`。

### 核心回答
计算步骤：
1. 输入向量通过线性变换得到 `Q`、`K`、`V`。
2. 计算相似度：`QK^T`。
3. 缩放：除以 `sqrt(d_k)`，防止数值过大。
4. softmax 得到注意力权重。
5. 权重乘以 `V` 得到加权表示。

```text
Attention(Q, K, V) = softmax(QK^T / sqrt(d_k)) V
```

多头注意力是把 Q/K/V 切成多个 head 并行计算，再拼接投影。

### 1. Q、K、V 分别是什么

可以直观理解为：

- **Q（Query）**：当前位置想找什么信息。
- **K（Key）**：每个位置提供什么索引特征。
- **V（Value）**：真正被聚合的信息内容。

`QK^T` 用来计算相关性，softmax 后得到注意力权重，再用这些权重对 `V` 做加权求和。

### 2. 为什么要除以 `sqrt(d_k)`

如果维度很大，`QK^T` 的数值容易变大，softmax 会变得过于尖锐，梯度不稳定。除以 `sqrt(d_k)` 是为了缩放分数，让训练更稳定。

### 3. 面试总结

自注意力的本质是让每个 token 根据相关性动态聚合上下文信息。多头注意力则是从多个子空间学习不同关系。

### 4. 结合跨境支付项目怎么说（面试口径）

> 我会用业务话解释自注意力：一条支付问题里，模型生成答案时会判断哪些上下文更重要。比如用户问"这笔款为什么没到账"，它应该关注通道回执、清算时间、币种和风控状态，而不是无关的商户介绍。
>
> 自注意力的 QK 相似度就是在算当前 token 和上下文证据的相关性，最后用这些权重聚合信息。多头注意力则可以从不同角度看问题，比如一个 head 关注时间，一个 head 关注错误码，一个 head 关注风控状态。
>
> 但前提是 RAG 要先把证据召回准。否则 attention 再强，也是在无关上下文里找答案。

**钩子**：自注意力本质是动态选择上下文证据，RAG 里还要先把证据召回准。
