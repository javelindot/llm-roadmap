# 梯度累积等价于 batch 扩大的严格条件是什么？

## 答案

梯度累积与一次使用大 batch 等价，需要“同一参数点上，对所有样本梯度求相同加权平均，并只更新一次”。这是严格条件，不是任何训练配置下都自动成立。

### 数学条件

大 batch 被拆成 $K$ 个 micro-batch。若每个 micro-batch loss 是均值，则反向前需除以 $K$：

$$
g=\frac1K\sum_{k=1}^{K}g_k
$$

参数、优化器状态和学习率 scheduler 在 $K$ 次反向之间不能更新；最后只执行一次 optimizer step。样本权重和有效 token 数也必须一致。变长序列按 token 平均时，应按各 micro-batch 有效 token 总数加权，不能简单平均 micro-batch loss。

### 破坏严格等价的因素

- BatchNorm 使用每个 micro-batch 的统计量。
- Dropout、数据增强等随机性使前向 mask 与一次大 batch 不同；期望可能接近，但逐步不完全相同。
- 在每个 micro-step 做 gradient clipping、optimizer step、EMA 或 scheduler。
- 跨样本操作，如 in-batch negatives、batch contrastive loss、某些 MoE 负载均衡和 batch-level normalization。
- 混合精度溢出、浮点求和顺序和分布式规约带来数值差异。

### 实践

裁剪应在完成全部累积、unscale 后进行；DDP 可在非最后 micro-step 使用 `no_sync` 减少通信。有效 global batch 为 micro-batch × accumulation × data-parallel size。

因此梯度累积通常近似获得大 batch 的优化效果，但对依赖 batch 交互的任务，不能声称严格等价。
