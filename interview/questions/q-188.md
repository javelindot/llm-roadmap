# 有什么有效的方法能将准确率从 70% 提升到 90%？

## 答案

**一句话**：把准确率从 70% 提到 90%，要先定位错误来源，再从数据、检索、重排、prompt、模型和评估闭环逐层优化。

### 核心回答
优化路径：
- **错误分析**：把 bad case 分为解析错、召回错、排序错、生成错、知识缺失。
- **数据治理**：清洗文档、补充缺失知识、更新版本。
- **切片优化**：语义切分、表格保留、元数据增强。
- **检索优化**：混合检索、多 query、HyDE、业务过滤。
- **重排优化**：reranker、规则加权。
- **生成优化**：更强 prompt、引用约束、拒答机制。
- **模型优化**：换 embedding、微调 reranker 或 LLM。
- **评估闭环**：构建测试集，持续收集 bad case。

### 1. 先做错误归因

准确率从 70% 到 90%，不是靠一个动作完成的。首先要把错误拆开：

- 文档缺失。
- 解析错误。
- 切片不合理。
- 召回没命中。
- rerank 排错。
- prompt 没约束好。
- 模型生成幻觉。
- 标准答案本身不一致。

不同错误对应不同优化方法。

### 2. 优化优先级

我会优先处理高频、高风险、容易修的问题。比如解析表格错，修一次影响一批问题；知识缺失，补文档比换模型更有效；召回错，优先调切片和检索。

### 3. 面试总结

准确率提升要靠 bad case 驱动的系统优化，不是简单换大模型。

### 4. 结合跨境支付项目怎么说（面试口径）

> 我不会直接说换大模型。支付 RAG 准确率 70%，先看错在哪里：如果规则没召回，优化切片和检索；如果召回了但答案错，优化 prompt 和引用约束；如果文档本身没有，补知识库；如果表格抽乱了，修解析。
>
> 我们之前通道规则类问题很多错在表格和元数据，后来把国家、币种、通道作为过滤条件，再加 rerank，效果比单纯换模型提升更明显。
>
> 所以我的思路是先做 bad case 归因，再按收益排序优化。只有确认是模型理解能力不足，才考虑换模型或微调。

**钩子**：准确率提升要先做 bad case 归因，别一上来就换模型。
