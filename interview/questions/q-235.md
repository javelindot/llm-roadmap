# 用户意图识别怎么做的？

## 答案

> 我们跨境支付里用户意图识别分**3 层**——**关键词规则、小模型分类、LLM 兜底**，按延迟和准确率梯度组合。
>
> **背景**：
> 客服场景每天 10w+ 对话，意图大致 20 类（订单查询、退款、汇率、限额、KYC、投诉、商户咨询等）。要求 p95 延迟 < 200ms，准确率 > 90%。
>
> **三层架构**：
>
> **L1：关键词规则（10ms）**
> - 90% 高频意图（"我的订单"、"退款"、"汇率"）能直接命中关键词词典
> - 规则维护成本低，可解释
> - 覆盖率 ~60%，准确率 ~95%
>
> **L2：BERT 分类器（30ms）**
> - 规则没命中的走 BERT
> - bert-base-chinese + 微调，20 分类
> - 训练数据：8w 条人工标 + 历史日志聚类
> - 覆盖率 ~30%，准确率 ~92%
>
> **L3：LLM 兜底（500ms）**
> - 前两层都不确定（BERT 信心 < 0.7）
> - 调 Qwen-7B 做 zero-shot 分类 + 一句解释
> - 覆盖率 ~10%，准确率 ~85%（长尾难分类）
>
> **联合调度**：
> ```python
> async def detect_intent(query: str) -> Intent:
>     # L1
>     if intent := match_rules(query):
>         return intent
>     # L2
>     bert_intent, conf = bert_classify(query)
>     if conf > 0.7:
>         return bert_intent
>     # L3
>     return await llm_classify(query)
> ```
>
> **多意图怎么处理**：
> "我想查订单状态顺便问下汇率"——这种多意图我们让 LLM 拆分成两个独立意图，并行走 RAG 召回再聚合答案。
>
> **多轮上下文怎么处理**：
> "那退款呢"——需要历史上下文才知道指什么。Query 改写器（详见 q-211）先拼上下文，再走意图识别。
>
> **几个工程优化**：
>
> **1. 数据回流飞轮**
> - 每天 sample 100 条人工标
> - 失败 case 补到 BERT 训练集
> - 每月重训 BERT
>
> **2. 信心阈值动态调整**
> - 监控线上意图分布，发现某类意图分类不准 → 单独提高阈值降级到 LLM
>
> **3. 拒识机制**
> - LLM 也不确定（输出 "unknown" 或信心 < 0.5）→ 转人工
> - 比强答错好
>
> **4. 业务规则注入**
> - 高价值用户（VIP 商户）的意图直接走 LLM，不省成本
> - 简单用户尽量走规则 / BERT 省钱
>
> **5. 监控**
> - 各层覆盖率 + 准确率
> - L3 占比上升 → 说明长尾意图增加，需要扩 L1/L2 训练
>
> **整体效果**：
> - p95 延迟 80ms（L1 命中是大头）
> - 端到端意图准确率 92%
> - 单次调用平均成本 0.0001 元（90% 走 L1 不调 LLM）
>
> 一句话：**意图识别是性价比敏感的环节**——能用规则就别用模型、能用小模型就别用 LLM，但要给长尾留兜底。
