# 8．平常有用过 RLHF 吗，简单介绍一下

## 答案

RLHF 是 Reinforcement Learning from Human Feedback，用人类偏好信号让模型的输出更有帮助、更安全、更符合任务要求。经典流程包括 SFT、偏好数据/奖励模型和强化学习三个阶段。

### 1. SFT

先用高质量指令数据训练基础策略：

```text
prompt -> reference response
```

目标是让模型学会对话格式、任务行为和基本安全边界。

### 2. 偏好数据与 Reward Model

对同一 prompt 生成多个回答，由标注者选择或排序：

```text
chosen > rejected
```

Reward Model 学习输出标量分数，常用 pairwise preference loss，使 chosen 得分高于 rejected。

人工反馈也可以部分由 AI Feedback、规则或可执行 verifier 补充，但高风险任务仍需要专家校准。

### 3. PPO 优化

Policy 生成回答，Reward Model 打分，再用 PPO 更新策略。目标通常包含：

```text
reward
- beta * KL(policy || reference)
```

KL 限制策略不要偏离 SFT 模型太远。训练中还要监控 entropy、clip fraction、reward hacking 和安全回归。

### 4. 现代替代和扩展

- DPO/IPO：直接用偏好对优化，不显式训练在线 RM+PPO；
- RLAIF：使用 AI 反馈；
- GRPO：对同一 prompt 的一组回答做相对优势估计；
- Process Reward：监督中间过程；
- 可验证奖励：数学、代码、工具任务由程序判定。

这些方法不应全部混称为经典 RLHF，但都属于后训练和对齐体系。

### 5. 如何回答“用过吗”

必须按真实经历区分：

> 我实际做过的是偏好数据构造和 DPO/GRPO 实验，未完整训练过生产级 Reward Model + PPO。

或：

> 我参与过完整 RLHF，包括偏好标注、RM、rollout 和 PPO，并负责具体模块与指标。

需要给出真实模型数、硬件、数据量、reward、KL 和评测，不能只说“用过 TRL”。

### 6. 跨境支付场景

偏好可以奖励解释清晰、证据一致和正确转人工，但资金、隐私和合规应作为硬约束。不能为了用户满意奖励模型承诺到账、退款或绕过 KYC。

### 7. 面试总结

经典 RLHF 是 SFT -> 偏好数据/RM -> PPO，并通过 KL 保持稳定。回答时应说明自己真正做过哪一段，同时讲清 reward hacking、成本和高风险业务边界。
