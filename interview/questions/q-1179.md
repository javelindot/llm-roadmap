# 如何求第k大元素，使用线性复杂度的快速排序方法

## 答案

可以使用 Quickselect。它复用快速排序的 partition 思想，但每轮只递归进入包含目标位置的一侧，因此平均时间复杂度为 `O(n)`，不是完整排序的 `O(n log n)`。

第 `k` 大对应升序下标 `len(nums) - k`。

```python
import random


def find_kth_largest(nums: list[int], k: int) -> int:
    if not 1 <= k <= len(nums):
        raise ValueError("k is out of range")

    target = len(nums) - k
    left, right = 0, len(nums) - 1

    while left <= right:
        pivot_index = random.randint(left, right)
        nums[pivot_index], nums[right] = nums[right], nums[pivot_index]
        pivot = nums[right]

        store = left
        for i in range(left, right):
            if nums[i] < pivot:
                nums[store], nums[i] = nums[i], nums[store]
                store += 1

        nums[store], nums[right] = nums[right], nums[store]

        if store == target:
            return nums[store]
        if store < target:
            left = store + 1
        else:
            right = store - 1

    raise RuntimeError("unreachable")
```

### 运行过程

以 `[3, 2, 1, 5, 6, 4]` 的第 2 大为例，目标升序下标为 `4`。每次 partition 后：

- pivot 左侧都小于 pivot；
- pivot 已在最终排序位置；
- 只保留包含下标 4 的一侧继续处理。

无需保证左右两侧内部有序。

### 复杂度

- 随机 pivot 的期望时间：`O(n)`；
- 最坏时间：`O(n²)`，例如每次都选到极端 pivot；
- 迭代版本额外空间：`O(1)`，但会原地修改输入。

若不能修改输入，应先复制一份，空间变为 `O(n)`。如果面试官严格要求最坏 `O(n)`，需要使用 Median of Medians，也称 BFPRT，按 5 个元素分组选择质量有保证的 pivot；实现更复杂，工程中常用随机化 Quickselect。

### 重复元素优化

当重复值很多时，可以使用三路 partition，将数组分成：

```text
< pivot | == pivot | > pivot
```

如果目标落在等于区间可以立即返回，避免重复值导致多轮无效划分。
