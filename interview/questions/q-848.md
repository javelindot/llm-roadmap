# 大模型相关知识了解

## 答案

可以按“结构、训练、后训练、推理、应用和评估”形成完整知识地图。

### 1. 结构

主流 LLM 多为 Decoder-only Transformer，包括 causal Attention、RoPE、RMSNorm、SwiGLU 和残差。GQA/MQA、MLA 降低 KV Cache；MoE 用稀疏专家扩大总容量；多模态模型接入视觉和音频编码。

### 2. 训练

预训练做 next-token prediction，关键是数据治理、scaling、混合精度、并行与稳定性。持续预训练用于领域语言，SFT 学指令与格式，LoRA/QLoRA 降低微调资源。

### 3. 对齐与推理能力

DPO 用离线偏好对，PPO 使用 actor-critic 在线优化，GRPO/DAPO 等面向可验证推理和组内奖励。需要关注 reward hacking、likelihood displacement 和能力回退。

### 4. 推理部署

推理分 prefill/decode。常用量化、FlashAttention、PagedAttention、continuous batching、prefix cache、speculative decoding、张量/专家并行和蒸馏，并分别评估 TTFT、ITL、吞吐和成本。

### 5. 应用

RAG 提供可更新证据，Agent 调用工具和 workflow，推荐与搜索可用 LLM 做内容理解、改写、重排和生成。高风险系统必须使用权限、验证器、审计和人工审批。

### 6. 评估

基础 benchmark 之外，还要建立业务集，分别评估检索、生成、工具、安全、延迟、成本与漂移。LLM-as-Judge 需要人工校准和偏差控制。

### 面试总结

大模型知识不只是 Transformer 八股，还包括数据、后训练、推理系统、RAG/Agent 与评估。回答时应主动连接原理、工程指标和业务边界。
