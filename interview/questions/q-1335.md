# 向量维度说上了

## 答案

题目上下文不完整，可能是在追问 Embedding 维度、Attention 各矩阵维度，或“论文讲到多细”。面试时应先确认对象，再把 shape 写出来，不能只报一个 `768/1024/1536`。

### 1. Embedding 检索维度

文本经 Embedding 模型得到：

```text
E(query):    [d]
E(document): [d]
```

批量：

```text
Q: [Bq, d]
D: [Bd, d]
```

点积矩阵：

```text
S = Q D^T
S: [Bq, Bd]
```

若做 cosine similarity，通常先 L2 normalize：

```text
q_hat = q / ||q||
d_hat = d / ||d||
score = q_hat^T d_hat
```

向量维度由模型训练决定，索引、查询和文档必须使用同一版本与维度。

### 2. 维度越高越好吗

不一定。高维可能增加表达容量，但也会增加：

- 向量存储；
- 内存带宽；
- ANN 距离计算；
- 索引构建和网络传输；
- 量化与部署成本。

FP32 向量存储近似：

```text
N * d * 4 bytes
```

1 亿条、1024 维仅原始向量就约 409.6 GB，还未包括索引结构和副本。

### 3. 降维方式

- 选择原生较低维模型；
- Matryoshka Representation 截断前若干维；
- PCA/OPQ；
- FP16/INT8/PQ 量化；
- 多阶段检索：低维召回，高精度 Rerank。

降维必须重新测 Recall@K、NDCG、延迟和索引大小，不能只看压缩率。

### 4. Transformer 维度

若面试官问 Attention，应写：

```text
X: [B, N, d_model]
Q,K,V: [B, h, N, d_head]
d_head = d_model / h
QK^T: [B, h, N, N]
AttentionOutput: [B, N, d_model]
```

GQA 中 Query head 数大于 KV head 数；MLA 还会出现低维 latent，不能按标准 MHA 套 shape。

### 5. 跨境支付

向量模型要评估中英文、错误码、币种、通道和卡组织术语。对精确编号依赖 BM25/字段查询，不应靠提高向量维度解决；高维向量也不能替代租户和时间版本过滤。

**面试总结**：维度是容量与成本的工程旋钮。先写矩阵 shape，再讲存储、计算和效果权衡，比背模型默认维度更重要。
