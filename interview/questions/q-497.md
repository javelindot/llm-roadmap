# BERT的结构，比过去的文本编码器好在哪些地方？（这个确实需要好好总结一下）

## 答案

BERT 是由多层 Transformer Encoder 堆叠的双向文本编码器。它的输入由 Token、位置和句段三种 Embedding 相加，经过 Self-Attention 与 FFN 后，为每个 Token 输出上下文化表示。

### 1. 结构

```text
Token / Position / Segment Embedding
→ N 层 Transformer Encoder
→ 每个 Token 的上下文化向量
```

每层包括：

- Multi-Head Self-Attention
- FFN
- Residual Connection
- LayerNorm

`[CLS]` 常用于句级分类，Token 表示用于 NER、抽取式问答等任务。

### 2. 比静态词向量好在哪里

Word2Vec/GloVe 为一个词提供固定向量，多义词无法随上下文变化。BERT 的“苹果”在水果和公司语境中会得到不同表示。

### 3. 比 RNN/LSTM 编码器好在哪里

- Self-Attention 可以并行计算，训练效率更高。
- 任意两个位置的依赖路径更短，长距离关系更容易建模。
- 每层都能让 Token 聚合全局上下文。

### 4. 双向上下文

传统自回归语言模型只能使用左侧上下文；BERT 的 Encoder 在预训练和编码时可以同时使用左右信息，因此更适合分类、匹配、NER 和检索表征。

### 5. 预训练迁移

BERT 先在大规模无标注语料上学习语言知识，再用少量标注数据 Fine-tune。相比为每个任务从头训练编码器，数据效率和泛化能力更好。

### 6. 仍然存在的限制

- Attention 对长序列成本高。
- `[MASK]` 带来预训练与真实输入差异。
- 不适合直接做长文本自回归生成。
- 通用预训练不保证掌握特定业务口径。

**面试总结**：BERT 的核心提升是“上下文化、双向、可并行、可迁移”，它把过去任务专用的文本编码器变成了统一的预训练表示模型。
