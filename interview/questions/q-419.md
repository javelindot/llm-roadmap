# 🔥🔥 设计模式

## 答案

设计模式是对反复出现的软件设计问题的可复用解法。面试时不建议背完 23 种模式，而应说明：**它解决什么变化、如何解耦、代价是什么**。

### 1. 三大类别

- **创建型**：控制对象创建，如工厂、抽象工厂、建造者、原型、单例。
- **结构型**：组织类和对象，如适配器、装饰器、代理、外观、组合。
- **行为型**：组织协作和职责，如策略、观察者、责任链、状态、命令、模板方法。

### 2. 高频模式与使用场景

| 模式 | 解决的问题 | 示例 |
|------|------------|------|
| Factory | 隔离具体对象创建 | 根据配置创建不同模型客户端 |
| Strategy | 运行时切换算法 | 在向量检索、混合检索之间切换 |
| Adapter | 统一不兼容接口 | 统一 OpenAI、Anthropic、本地模型调用 |
| Decorator | 不改原类叠加能力 | 给模型调用增加重试、缓存、监控 |
| Observer | 一对多事件通知 | 任务状态变化触发日志、告警、回调 |
| Chain of Responsibility | 多处理器按序尝试 | 内容审核、鉴权、参数校验链 |
| State | 行为随状态变化 | Agent 任务在待执行、运行、审批、失败间流转 |
| Command | 把请求封装成对象 | 工具调用支持排队、重试、审计和撤销 |

### 3. 一个工程化示例

LLM 网关可以组合多种模式：

- 用 **Adapter** 统一不同模型供应商接口。
- 用 **Strategy** 根据成本、延迟和能力路由模型。
- 用 **Decorator** 增加缓存、限流、重试和 Trace。
- 用 **Factory** 根据配置装配最终客户端。

### 4. 使用原则

- 先确认变化点，再选择模式，不要为了“用了模式”而过度抽象。
- 模式通常通过增加一层间接性换取扩展性，也会增加类数量和理解成本。
- 如果只有一个实现且短期没有变化，简单代码可能更合适。

**面试总结**：设计模式的核心不是记名称，而是识别“哪里会变化”，然后把变化隔离在稳定接口之后。
