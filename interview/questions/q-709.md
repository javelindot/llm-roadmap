# SFT和DPO训练心得

## 答案

SFT 和 DPO 的训练心得可以从数据、模板、超参数、监控和回归五方面回答。二者都非常依赖数据质量，但故障信号不同。

### SFT

确保 chat template、BOS/EOS、padding 和 label mask 正确；通常只监督 assistant 输出。数据按能力桶去重和平衡，避免大量同模板样本支配。小数据优先 LoRA 和较少 epoch，并用 validation loss、分桶任务指标及生成样例共同判断过拟合。

学习率过高会破坏通用能力，过低则格式或领域能力学不进去。按 token 数而非文件数理解数据规模，使用 packing 时避免跨样本注意力或标签边界错误。

### DPO

每条数据必须是同一 prompt 的 chosen/rejected pair。两者差异过弱没有信号，差异只在长度或格式会产生捷径。按 prompt 去重切分，并检查模型初始 log-prob margin。

DPO 初始 policy 与 reference 相同时，loss 常接近 $\log2$。训练时不能只看 loss，要看 chosen/rejected reward margin、pairwise accuracy、KL、长度和人工偏好。$\beta$、学习率和有效 batch 是关键参数。

### 顺序与回归

通常先用 SFT 建立稳定指令策略，再用 DPO 做偏好对齐。DPO 数据分布应接近当前 policy，否则离线 pair 与模型输出差距过大。两阶段都要保留通用能力、安全和格式回归集，防止目标任务提升但基础能力退化。

排障时先用几十个高质量样本验证能否过拟合，再扩大到全量和分布式环境。回答中最好给出真实 bad case、指标和修复过程，而不是只列配置。
