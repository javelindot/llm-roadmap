# 训练流程？

## 答案

模型训练流程可以概括为“定义目标、治理数据、建立基线、训练验证、部署监控”，但面试中应讲出每一步的输入、输出和决策依据。

### 1. 完整流程

1. 定义任务、标签、约束和线上业务指标；
2. 采集、脱敏、清洗、去重并版本化数据；
3. 按用户或时间切分训练、验证、测试集，避免泄漏；
4. 建立简单基线和可复现训练配置；
5. 选择模型、损失、优化器、学习率、batch 和精度；
6. 训练中监控 loss、梯度、吞吐、显存及异常 batch；
7. 用验证集选 checkpoint 和调参，测试集只做最终评估；
8. 做消融、切片、稳定性、安全和成本评测；
9. 导出、量化或蒸馏，影子流量与灰度上线；
10. 监控漂移、bad case 和业务指标，形成数据回流与回滚机制。

### 2. 大模型训练要补充

- 预训练、持续预训练、SFT、偏好优化应明确属于哪一阶段；
- 使用 LoRA 还是全参训练由数据规模、任务差异和预算决定；
- 分布式训练说明 DP/FSDP/ZeRO/TP 与有效 batch；
- BF16、梯度累积、gradient checkpointing 和裁剪保证稳定；
- 每次实验记录代码、数据、模型、随机种子和环境版本。

### 3. 跨境支付场景

支付模型应按时间、国家、币种和通道切片，PII 脱敏后才能训练。训练集不能混入未来规则或同一用户的测试样本。上线不只看 F1，还要看高风险召回、正常交易误杀、人工审核量、延迟和可审计性。

### 面试总结

训练不是从 `trainer.train()` 开始，而是从目标和数据定义开始，到线上监控与回流结束。回答时要突出数据切分、防泄漏、可复现、业务切片、灰度和回滚。
