# 你做Prompt优化时，是如何判断优化后的Prompt在Agent推理链路中性能提升的?

## 答案

判断 Prompt 优化是否提升 Agent 性能，必须做端到端、可重复的对照评测，而不是只看几个回答“更像人话”。Prompt 版本之外的模型、工具、数据和采样参数应保持一致。

### 建立评测集

从真实任务和历史 bad case 构建分层集合，覆盖正常、长尾、歧义、缺参、工具失败和 prompt injection。为每条任务定义可验证成功条件、允许工具和预算，并冻结一部分测试集防止对 prompt 过拟合。

### 分层指标

- Router/Planner：意图、步骤和工具选择准确率。
- Tool call：参数 schema 通过率、执行成功率、不必要调用率。
- Retrieval：Recall@K、引用命中和证据支持率。
- Final answer：任务完成率、事实性、格式、安全和人工偏好。
- 系统：平均步骤、token、费用、TTFT/P95 延迟和循环率。

能用执行器验证的代码、SQL、预订流程优先使用确定性指标；LLM-as-a-Judge 需固定 rubric、做位置交换和人工抽检。

### 实验方法

离线对旧/新 Prompt 做 paired evaluation，报告每个能力桶的差值和置信区间，并分析退化样本。线上先 shadow，再小流量 A/B，设置安全与成本 guardrail。一次只改一个因素并版本化。

如果新 Prompt 最终成功率上升但步骤、成本或安全违规恶化，不能简单判定更好。还要确认收益来自真正改进推理，而不是输出更长、调用更多工具或迎合评审器。
