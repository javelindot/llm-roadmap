# Qwen 2.5 VL中的Transformer结构与普通Transformer结构有哪些不同？

## 答案

Qwen2.5-VL 不是单一普通 Transformer，而是“动态分辨率 Vision Transformer + 投影/合并模块 + Qwen2.5 LLM”的多模态架构。主要差异在视觉 token 化、时空位置和跨模态输入。

### 1. 视觉 Encoder

普通文本 Transformer 输入离散 token embedding；Qwen2.5-VL 使用从头训练的 native dynamic-resolution ViT：

- 不同尺寸图像产生不同数量视觉 token；
- 视频按动态 FPS 采样；
- 绝大多数 ViT 层使用最大约 \(8\times8\) 的 Window Attention；
- 只有少数层使用 Full Attention，降低视觉侧二次成本；
- ViT 采用与 LLM 更一致的 RMSNorm 和 SwiGLU，而不是传统 ViT 的 LayerNorm+GELU。

### 2. 视觉 Token 压缩

图像 patch 经 ViT 后通过 patch merger/投影映射到 LLM hidden size，再与文本 token 交错输入语言模型。视觉 token 数随原始分辨率变化，因此需要控制 `min_pixels/max_pixels` 和视频帧数。

### 3. mRoPE

普通文本 RoPE 主要编码一维位置；Qwen-VL 系列使用 Multimodal RoPE，将位置拆为：

- 时间轴；
- 图像高度；
- 图像宽度。

Qwen2.5-VL 进一步用动态 FPS 和绝对时间对齐 temporal position，使模型理解视频事件发生时刻；图像框和点可使用实际尺寸尺度，而非传统归一化坐标。

### 4. 能力影响

这种结构支持文档/表格、视觉定位、长视频和 GUI Agent，但视觉 token 会显著增加 prefill 和显存，不能按纯文本 7B 的成本估算。

依据可参考 [Qwen2.5-VL 技术报告](https://arxiv.org/abs/2502.13923)与[官方介绍](https://qwenlm.github.io/blog/qwen2.5-vl/)。

### 面试总结

Qwen2.5-VL 在普通 LLM 前增加动态分辨率 ViT，以 Window Attention 降低视觉计算，用 mRoPE 编码时间、高度和宽度，再把视觉 token 与文本共同送入 LLM。部署成本主要受视觉 token 数影响。
