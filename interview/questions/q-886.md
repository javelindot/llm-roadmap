# LORA为什么可以实现少量的参数完成全量微调的效果

## 答案

LoRA 冻结原权重，只学习低秩增量：

\[
W'=W+\frac{\alpha}{r}BA
\]

参数从 \(d_{out}d_{in}\) 降为 \(r(d_{in}+d_{out})\)，其中 \(r\ll d\)。

### 为什么有效

- 预训练模型已拥有大部分语言、知识和推理能力，下游任务主要是重新组合与偏移；
- 许多任务的有效更新位于低维子空间，完整权重变化常有少数主导奇异方向；
- 每层都加入低秩分支后，跨多层组合仍有较强表达能力；
- 低秩限制具有正则化作用，小数据时可减少过拟合和遗忘；
- LoRA 可覆盖 Attention 与 FFN，不只是修改最后一层。

LoRA 原论文观察到适配更新具有 rank deficiency，但这不是“所有全参更新都严格低秩”的数学保证。

### 为什么不一定等于全参

- 领域与基座差异很大；
- 需要改变底层视觉/语音或 embedding；
- rank、目标模块过小；
- 数据规模大且要学习新能力；
- LoRA 已饱和而全参仍持续提升。

此时可增大 rank、覆盖 FFN、使用 DoRA/AdaLoRA，或转部分/全参训练。

### 面试总结

LoRA 之所以常接近全参，是因为下游适配通常只需沿少数有效方向调整已有能力；它以低秩约束换取参数和显存效率，但任务偏移大时仍可能不够。
