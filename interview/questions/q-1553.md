# 请比较一下几种常见的 LLM 架构,例如 Encoder-Only, Decoder-Only, 和 Encoder-Decoder,并说明它们各自最擅长的任务类型。

## 答案

三类架构的主要区别是 Attention 可见范围、是否单独编码输入，以及预训练目标。没有绝对优劣，任务和部署决定选择。

### 1. Encoder-Only

代表：BERT、RoBERTa、DeBERTa，以及大量 Embedding/Reranker 模型。

特点：

- 输入 token 之间通常双向 Attention；
- 每个位置同时利用左右上下文；
- 常用 MLM、对比学习或分类目标；
- 一次前向得到整段表示，不擅长自由长文本生成。

擅长：

- 分类、NER、抽取；
- 语义匹配和 Embedding；
- Cross-Encoder rerank；
- 高吞吐理解任务。

### 2. Decoder-Only

代表：GPT、LLaMA、Qwen、DeepSeek 等主流生成模型。

特点：

- causal mask，只看当前及过去 token；
- next-token prediction；
- Prompt 与输出统一为一个序列；
- 易扩展到对话、工具、代码和多模态 token；
- 自回归生成灵活，但逐 token Decode 慢。

擅长：

- 开放式生成；
- 对话、代码、推理；
- few-shot/in-context learning；
- Agent 工具调用。

### 3. Encoder-Decoder

代表：原始 Transformer、T5、BART、mT5。

特点：

- Encoder 双向理解输入；
- Decoder 因果生成输出；
- Decoder 用 Cross-Attention 读取 Encoder 表示；
- 输入 K/V 可在生成过程中缓存；
- 两套堆栈使参数和服务更复杂。

擅长：

- 翻译、摘要、改写；
- 输入输出边界明确的 seq2seq；
- 文档到结构化文本；
- 条件生成。

### 4. 对比

| 维度 | Encoder-Only | Decoder-Only | Encoder-Decoder |
|---|---|---|---|
| 输入可见性 | 双向 | 因果 | Encoder 双向 |
| 生成 | 弱/需额外头 | 最灵活 | 强条件生成 |
| 理解效率 | 高 | 通常成本较高 | 强 |
| 结构 | 单 Encoder | 单 Decoder | Encoder + Decoder |
| 典型任务 | 分类、检索 | 对话、Agent | 翻译、摘要 |

现代模型还有 Prefix LM、UniLM、MoE、SSM/Attention 混合和原生多模态等变体，三分法是主干，不是全部。

### 5. 跨境支付场景

- Encoder：工单分类、实体抽取、Embedding/Rerank；
- Decoder：客服回答、调查 Agent、工具调用；
- Encoder-Decoder：多语言翻译、固定文档转换。

生产系统常组合使用，而不是一个大模型包办全部任务。

### 6. 面试总结

Encoder-Only 强理解，Decoder-Only 强开放生成，Encoder-Decoder 强输入到输出的条件转换。选择依据是任务、时延、生成需求与部署成本。
