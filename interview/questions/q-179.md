# 把图片转为文字描述是怎么做的？

## 答案

> 我们用 **Qwen2.5-VL 7B 本地部署**做图片转文字描述。流程是这样：
>
> **1. 图片预处理**
> - 缩放到 448×448（保留长宽比 + padding）
> - 太小的图（< 64px）跳过，多半是装饰图
> - 太大的图分块处理
>
> **2. Prompt 设计**（按图片类型区分）
>
> 流程图类：
> ```
> 这是一张业务流程图，请用 100 字以内描述图中的关键步骤和顺序，
> 用"1. xxx → 2. xxx"格式。不要描述视觉风格。
> ```
>
> 表格截图类：
> ```
> 这是一个表格图片，请提取表头和每行数据，输出 Markdown 表格格式。
> ```
>
> UI 截图类：
> ```
> 这是一个 App / 网页界面截图，请描述：1) 当前页面是什么；
> 2) 关键按钮和位置；3) 显示的核心信息。100 字以内。
> ```
>
> **3. 后处理**
> - 长度截断到 200 token
> - 关键词抽取（NER）补充元数据
> - 描述 + 原图路径一起入库
>
> **4. 工程优化**
> - **批处理**：一次 8 张图并行（vLLM）
> - **缓存**：相同 hash 的图复用描述
> - **失败重试**：VLM 偶尔输出乱码，自动重跑 1 次
> - **质量过滤**：描述里出现"我不知道"、长度 < 10 字 → 标记人工
>
> **为什么不用 GPT-4V 或 Claude**：
> - 成本：GPT-4V 0.05/张 vs Qwen2.5-VL 本地 0.01/张
> - 数据合规：客户上传内容不能出公网
> - 中文场景：Qwen2.5-VL 中文图理解比 GPT-4V 更好
>
> 上线后 1w 张图描述生成 + 入库总成本 ~100 元，准确率 92%（人工抽检 100 张）。
