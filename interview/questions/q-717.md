# 对比RLHF、PPO和DPO算法的技术差异、优缺点及适用场景

## 答案

RLHF 是利用人类偏好对齐模型的一套流程，PPO 是其中可使用的策略优化算法，DPO 是绕过显式 Reward Model 和在线 RL 循环的直接偏好优化方法。三者不在同一抽象层级。

### RLHF 与 PPO

典型 RLHF 包括 SFT、偏好数据、Reward Model 和策略优化。PPO 从当前策略生成回答，用 RM 奖励和 reference KL 构造回报，通过 clipped probability ratio 更新 policy，同时训练 value model。

优点是能在线探索当前策略分布、组合多个奖励，并直接优化序列级目标。缺点是需要 rollout、RM、reference、value 多个模型，显存和工程复杂度高；on-policy 样本效率低，容易 reward hacking 和训练不稳定。

### DPO

DPO 使用离线 chosen/rejected pair，将 RLHF 的最优策略形式转为分类式目标。它比较 policy 相对 reference 对 chosen 和 rejected 的 log-ratio，不显式训练 RM，也不需要在线 rollout 和 critic。

优点是实现简单、稳定、资源低；缺点是受离线偏好数据覆盖限制，难以主动探索，也不方便使用不可微环境奖励或复杂在线约束。$\beta$ 仍控制偏离 reference 的程度。

### 适用场景

- 有高质量静态 pair、希望快速稳定对齐：优先 DPO。
- 有可验证奖励、需要在线探索或持续从当前策略采样：PPO/GRPO 等更合适。
- 目标主要是格式和领域任务：可能高质量 SFT 已足够。

比较时应控制 SFT 起点和数据，观察人工偏好、KL、通用能力、安全、成本与稳定性，而不是只比较训练 loss。
