# 为什么分类用CE不用 MSE？

## 答案

分类任务通常用 Cross Entropy 而不是 MSE，因为分类输出表示概率分布，CE 与最大似然目标一致，并且与 Softmax/Sigmoid 组合时梯度更合适。

### 1. 概率建模

多分类假设标签服从 Categorical Distribution，最大化真实类别的似然等价于最小化：

```text
CE = -Σ y_k log p_k
```

二分类对应 Bernoulli Likelihood，得到 BCE。

MSE 更自然地来自高斯噪声假设，适合连续值回归。

### 2. 梯度差异

Softmax + CE 对 Logit 的梯度为：

```text
∂L / ∂z_k = p_k - y_k
```

即使模型非常自信地预测错了，仍能得到直接有效的纠正信号。

如果对 Softmax 概率使用 MSE，梯度还要乘 Softmax Jacobian。当概率接近 0 或 1 时容易变小，训练通常更慢。

### 3. 分布距离

CE 关注真实类别的对数概率，对“把正确类别概率压得极低”惩罚很强；MSE 只度量欧氏距离，不能同样自然地表达分类概率的对数似然。

### 4. MSE 是否完全不能做分类

不是。它可以用于：

- 知识蒸馏中的表示或 Logit 回归；
- 某些软标签任务；
- 特殊实验设置。

但普通分类的默认选择仍是 CE/BCE。

### 5. 实现注意

训练时使用框架内置 `CrossEntropyLoss`，输入原始 Logits，不要先手动 Softmax；这样数值更稳定。

**面试总结**：CE 不只是经验上更好，它来自分类分布的最大似然，并提供更直接的梯度。
