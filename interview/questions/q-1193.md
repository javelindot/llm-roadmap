# 介绍一个传统ML模型

## 答案

LightGBM 是基于 Gradient Boosting Decision Tree 的监督学习模型，适合表格数据上的分类、回归和排序任务。它通过串行训练多棵弱树，使每棵新树拟合当前模型的残差或负梯度。

### 1. 核心目标

第 `t` 轮模型为：

```text
F_t(x) = F_(t-1)(x) + η f_t(x)
```

其中 `f_t` 是新树，`η` 是学习率。训练目标可用二阶近似：

```text
Obj ≈ Σ_i [
  g_i f_t(x_i) + 1/2 h_i f_t(x_i)^2
] + Ω(f_t)
```

`g_i、h_i` 是损失对当前预测的一阶和二阶梯度，`Ω` 控制叶子数和叶子权重，防止模型过度复杂。

### 2. LightGBM 的特点

- **Histogram**：把连续特征离散到 bins，降低内存和分裂搜索成本；
- **Leaf-wise Growth**：每轮选择增益最大的叶子继续分裂，收敛快，但小数据容易过拟合；
- **GOSS**：保留大梯度样本并采样小梯度样本，减少计算；
- **EFB**：合并互斥的稀疏特征，降低有效特征维度；
- 原生处理缺失值；
- 支持分类、回归、LambdaRank 等目标；
- CPU 上对中等规模表格数据非常强。

### 3. 重要超参数

- `num_leaves`：模型表达能力核心，过大容易过拟合；
- `max_depth`：限制树深度；
- `min_data_in_leaf`：防止叶子样本过少；
- `learning_rate` 与 `n_estimators`：小学习率配更多树；
- `feature_fraction`、`bagging_fraction`：特征和样本随机化；
- `lambda_l1、lambda_l2`：叶子权重正则；
- `scale_pos_weight`：类别不平衡时调整正类权重。

参数应通过时间切分验证，而不是随机切分后反复调测试集。

### 4. 优点和局限

优点：

- 表格数据效果强，训练和推理快；
- 自动学习非线性和特征交互；
- 对特征缩放不敏感；
- 能处理缺失值和混合分布；
- 可用 feature importance、SHAP 做解释。

局限：

- 不擅长直接处理原始文本、图片和超长序列；
- 高基数类别通常需要编码或专门处理；
- 分布漂移后容易失效；
- 输出概率未必校准；
- SHAP 解释的是模型行为，不等于因果关系；
- Leaf-wise 策略在小数据上容易过拟合。

### 5. 跨境支付应用

可以用 LightGBM 预测支付成功或通道超时，特征包括：

> 国家、币种、金额分桶、商户类型、收款人类型、通道、银行工作时间、近期健康度。

需要严格保证：

- 特征只使用路由决策时已知的信息；
- 按时间切分，防止未来通道状态泄漏；
- 商户与租户数据隔离；
- 按国家、币种和通道分桶评测；
- 使用 Platt Scaling 或 Isotonic Regression 做概率校准；
- 牌照、制裁、KYB、额度等仍由硬规则处理。

指标除 AUC 外，还要看 PR-AUC、Recall、校准、成功率、成本、覆盖率和风险。模型可以排序合法通道，但不能替代合规准入和资金执行。
