# GRPO的原理，损失函数如何计算？

## 答案

GRPO（Group Relative Policy Optimization）对同一个 Prompt 采样一组回答，用组内相对奖励估计 Advantage，从而省去 PPO 中单独训练 Value/Critic 模型的成本。

### 1. 组内 Advantage

对 Prompt `q` 采样 `G` 个输出：

```text
o_1, o_2, ..., o_G
```

得到奖励 `r_i` 后做组内标准化：

```text
A_i = (r_i - mean(r)) / (std(r) + eps)
```

高于组平均的回答获得正 Advantage，低于平均的获得负 Advantage。

### 2. 策略目标

简化后的裁剪目标为：

```text
ratio_i = πθ(o_i|q) / πold(o_i|q)

L_policy = -E min(
  ratio_i A_i,
  clip(ratio_i, 1-ε, 1+ε) A_i
)
```

实际实现通常在 Token 级计算概率比，并加入对参考模型的 KL 约束：

```text
L = L_policy + β · KL(πθ || πref)
```

KL 项用于限制策略偏离基座模型过远。

### 3. 与 PPO 的区别

PPO 常用 Critic 估计状态价值，再通过 GAE 计算 Advantage；GRPO 直接使用同题多个候选的相对奖励作为 Baseline，因此：

- 少一个 Value Model，显存更低；
- 训练链路更简单；
- 但每个 Prompt 需要多次采样，生成成本更高。

### 4. 适用场景与风险

适合数学、代码等有明确验证器的任务。主要风险包括：

- 奖励设计错误导致 Reward Hacking；
- 组内奖励方差太小，学习信号不足；
- 组大小太小，标准化噪声大；
- 在线采样数据昂贵；
- 开放式任务难以得到可靠标量奖励。

**面试总结**：GRPO 用“组内比较”替代 Critic，但没有消除强化学习对采样质量、奖励可靠性和 KL 控制的依赖。
