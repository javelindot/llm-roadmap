# 图文笔记的多模态理解与检索方案

## 答案

图文笔记的多模态检索不能只把图片 OCR 成文本。完整方案应同时保留正文语义、图片视觉语义、OCR、版面和图文关系：

> 内容解析 → 多模态表征 → 多路索引 → 查询路由 → 混合召回 → 多模态重排 → 业务过滤与评测。

### 1. 内容解析与结构化

每篇笔记拆成以下信息：

- 标题、正文、话题、标签和作者提供的图片描述；
- 每张图片及其顺序、尺寸、质量和在笔记中的位置；
- 图片 OCR 文本、检测框和识别置信度；
- 商品、地点、品牌、颜色、人物、场景等视觉实体；
- VLM 生成的短 caption，但需标记为模型推断而非作者原文；
- 图文之间的引用关系，例如“图 3 是步骤 2 的操作结果”。

索引单元不宜只有整篇笔记。可以同时构建：

- note 级父文档；
- paragraph 级文本 chunk；
- image 级视觉单元；
- image-region 或 OCR block 级细粒度单元。

所有单元通过 `note_id、image_id、position、author、visibility、version` 关联，命中局部图片后可以回取完整笔记上下文。

### 2. 多路表征与索引

生产中可并行建设四类索引：

1. **文本倒排索引**：标题、正文、标签和 OCR，保证品牌名、型号、地点等精确匹配；
2. **文本 Dense Index**：处理口语、同义词和跨语言表达；
3. **图文共享向量索引**：使用 CLIP、SigLIP 2 等模型支持 text-to-image、image-to-text 和 image-to-image 检索；
4. **多向量视觉索引**：保存 patch 或区域级向量，用 Late Interaction 处理局部物体、版面和细节匹配。

SigLIP 2 增强了多语言图文理解和定位能力，适合多语言笔记；ColPali 直接将页面图片编码成多向量并进行 late interaction，更适合图文排版、截图和视觉文档。普通生活图片与文档截图的分布不同，应在内部数据上分别评测，不能只选一个模型覆盖所有内容。

### 3. 查询理解与路由

查询可能是文本、图片或图文组合：

- 文本找笔记：“红色复古连衣裙搭配”；
- 图片找相似内容：上传商品或场景图；
- 图文组合：“这张图里的包是什么型号”；
- 精确 OCR 查询：搜索截图中的错误码、店名或型号；
- 局部视觉查询：寻找相似纹理、版式或图片区域。

查询侧提取关键词、实体、语言、视觉 embedding 和意图，并决定各路预算。精确型号提高 BM25 和 OCR 权重；视觉相似提高图像向量权重；抽象风格和场景描述提高共享图文向量权重。

### 4. 混合召回与多模态重排

各路候选先按 rank 或校准分数融合，并保留召回来源。常用流程是：

```text
BM25/Text Dense/Image Dense/Multi-vector Top-K
 → 去重与父子聚合
 → 元数据过滤
 → 多模态 Cross-Encoder 或 VLM Rerank
 → 多样性与质量重排
```

重排模型输入 query、正文、图片缩略图、OCR 和关键区域，判断图文是否共同支持相关性。VLM 成本较高，只处理较小候选集，并设置 batch、缓存、超时和轻量模型降级。

不能把自动 caption 当成唯一证据。caption 可能遗漏颜色、数量、品牌细节，甚至生成图中不存在的信息；精确实体应结合 OCR、检测结果和原图验证。

### 5. 训练数据

正样本可以来自搜索点击、长停留、收藏和人工标注，但要处理位置与曝光偏差。困难负样本包括：

- 文本相似但图片不匹配；
- 图片相似但笔记主题不同；
- 同品牌不同型号；
- OCR 关键词相同但场景不同；
- 视觉上接近但被用户跳过的已曝光笔记。

一个 query 可能对应多篇有效笔记，挖掘负例时要避免 false negative。训练、验证和测试按用户、笔记及时间隔离，防止同一图片的裁剪、转发或近重复版本跨集合泄漏。

### 6. 工程与安全

- 图片 embedding 异步计算，新增笔记先用文本和轻量视觉特征入库；
- 原图、OCR、caption 和向量分别版本化；
- 使用感知哈希和向量聚类处理近重复图片；
- 删除或权限变化时同步清理所有子索引；
- 对人脸、证件、账号、二维码和隐私内容执行安全策略；
- 搜索结果仍需经过可见性、内容安全和版权过滤。

### 7. 评测

- 文本到图片、图片到图片、图文组合分别统计 Recall@K、MRR、NDCG；
- OCR 精确查询统计实体命中率；
- 局部细节查询统计 region-level recall；
- 评估图文一致性、近重复率、多样性和长尾覆盖；
- 按语言、图片类型、笔记类型和新旧内容分桶；
- 系统侧观察 P95/P99、索引体积、VLM 调用率和单位成本；
- 线上观察搜索成功率、点击、收藏、改写率和负反馈。

### 8. 跨境支付中的迁移应用

类似链路可用于检索付款截图、银行回单、KYB 材料、通道流程图和错误页面：

- OCR 与 BM25 命中订单号、错误码和银行名称；
- 文本向量匹配自然语言问题；
- 视觉向量识别相似页面或凭证版式；
- 多模态重排确认截图区域与文档说明是否对应。

支付材料含大量敏感信息，必须在入库前进行租户隔离、字段脱敏、权限控制和保留期管理。VLM 只能辅助定位和解释，金额、账户、交易状态仍要以权威业务系统或人工审核为准。
