# 用的模型参数量有多大？

## 答案

必须回答真实模型名称、参数口径和选择依据，不能只说“7B”或把 MoE 总参数当作每 token 计算量。

### 回答模板

> 项目使用的是[模型全名与版本]，总参数约[数值]。如果是 MoE，每个 token 激活参数约[数值]；如果使用量化，权重精度是[BF16/INT8/INT4]。训练方式为[全参/LoRA]，实际可训练参数为[数值或比例]。选择该尺寸是因为在[任务指标]达到要求的同时，能满足[显存、P99、QPS、成本]约束。

### 参数量如何核对

```python
total = sum(p.numel() for p in model.parameters())
trainable = sum(p.numel() for p in model.parameters() if p.requires_grad)
```

还应说明：

- Dense 还是 MoE，总参数与激活参数；
- 是否共享 embedding 与 LM head；
- LoRA rank、target modules 和 adapter 参数；
- 权重显存不等于推理总显存，还包括 KV Cache 和运行时开销；
- 量化减少字节数，不改变参数个数。

### 跨境支付场景

模型越大不一定越合适。意图分类和路由可能由小模型完成，复杂解释再升级大模型；订单事实由工具查询。应展示质量、延迟、成本和合规之间的选型实验，而不是以参数量证明项目价值。

### 面试总结

回答模型规模要给出模型版本、总参数、激活参数、可训练参数和精度，并解释为什么这个规模满足业务 SLO。参数量是资源口径，不是效果结论。
