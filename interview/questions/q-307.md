# 大数据、大模型，对于敏感信息的处理方案？

## 答案

> 金融场景敏感信息处理是**生死线**——我们跨境支付从**入口、训练、推理、输出、审计**五层做防护，组合拳治理。
>
> **敏感信息分类**：
>
> | 等级 | 类型 | 示例 |
> |------|------|------|
> | 极敏感 | 身份证号、银行卡、密码、生物特征 | 全程加密、零拷贝 |
> | 高敏感 | 姓名、手机、地址、邮箱 | 脱敏后入 LLM |
> | 中敏感 | 订单号、金额、IP | 业务上下文用脱敏版 |
> | 低敏感 | 商户类型、国家 | 可明文使用 |
>
> **五层防护**：
>
> **第一层：数据接入**
>
> - **PII 检测服务**（独立微服务）
>   - 正则 + NER + 业务字典
>   - 进入 LLM 前必经
>
> - **加密存储**
>   - 数据库字段级加密
>   - 密钥用 KMS 管理
>
> - **数据本地化**
>   - 国内业务数据不出境
>   - 跨境业务按地区落地
>
> **第二层：训练数据**
>
> - **训练前清洗**
>   - 历史日志先脱敏再训练
>   - 用 GPT-4 / Qwen-72B 二次检测漏网鱼
>
> - **数据红线**
>   - 客户隐私永远不进训练集
>   - 只用聚合统计、脱敏样本
>
> - **数据血缘**
>   - 每条训练样本记录来源
>   - 投诉时能撤回
>
> **第三层：推理时（最关键）**
>
> 用户输入 → LLM 之间必经的脱敏 pipeline：
>
> ```python
> async def safe_llm_call(user_query: str, context: dict):
>     # 1. PII 检测
>     pii = await pii_detector.scan(user_query)
>
>     # 2. 脱敏（保留 placeholder 映射）
>     masked_query, mapping = mask_pii(user_query, pii)
>     # 例："我的卡号 4111****1234" → "我的卡号 {{CARD_1}}"
>
>     # 3. context 也脱敏
>     masked_context = mask_pii_in_dict(context, mapping)
>
>     # 4. 调 LLM（用脱敏版）
>     llm_output = await llm.invoke(masked_query, masked_context)
>
>     # 5. 还原 placeholder
>     final = unmask(llm_output, mapping)
>
>     # 6. 二次检测输出是否泄漏
>     if pii_in_output(final, exclude=mapping):
>         return SAFE_FALLBACK  # 兜底回复
>
>     return final
> ```
>
> **第四层：输出**
>
> - **输出二次校验**：LLM 偶尔会"创造"敏感信息（编造卡号等）
> - **黑名单过滤**：内部账户、商业机密关键词
> - **审计日志**：每条 LLM 调用入库（含输入、输出、脱敏映射）
>
> **第五层：审计 + 合规**
>
> - **6 个月以上日志**留存
> - **定期红蓝对抗**：每月 jailbreak 测试
> - **数据访问权限**：按角色分级，运营不能直接查客户原始数据
> - **监管报送**：必要时提供完整可追溯链路
>
> **特别处理：训练数据脱敏**
>
> 训练数据可能含 PII，我们的处理：
> - **手机号** → `{{PHONE}}`
> - **身份证** → `{{ID_CARD}}`
> - **金额** → 离散化为区间（`< 1k`, `1k-10k`, ...）
> - **公司名** → 替换为虚构样例
>
> 这样训出的模型不会"记住"具体客户信息。
>
> **几个工程现实**：
>
> 1. **PII 检测要做 service**——单独高可用服务，所有 LLM 链路必经
>
> 2. **placeholder 映射不能丢**——用户的卡号要还原回去，不能模型说"您的卡号 {{CARD_1}}"
>
> 3. **prompt cache 兼容脱敏**——cache key 用脱敏后的，避免 cache 泄漏
>
> 4. **日志加密 + 访问审计**——保存 PII 日志比不保存还危险
>
> 5. **模型权重也是敏感的**——业务 SFT 后的模型不能外发
>
> **不能做的事**：
> - 用 OpenAI / Anthropic API 处理客户数据
> - 把脱敏不全的样本喂模型
> - 把 LLM trace 日志暴露给非授权人员
> - 训练时混入未授权数据
>
> 一句话：**敏感信息处理 = 检测 + 脱敏 + 加密 + 审计**——金融场景必须把"任何环节漏了都是事故"作为设计前提，五层防御任何一层都不能掉。
