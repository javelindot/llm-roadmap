# 什么是强-弱蒸馏（Teacher-Student Distillation）

## 答案

**一句话**：强-弱蒸馏是用能力更强的 Teacher 模型生成数据、打分或提供软标签，再训练一个更小更便宜的 Student 模型。

### 核心回答
常见方式：
- **logit 蒸馏**：学生学习 teacher 的概率分布。
- **答案蒸馏**：teacher 生成高质量回答，student 用这些数据做 SFT。
- **偏好蒸馏**：teacher 比较多个回答，构造 DPO/RLHF 数据。
- **过程蒸馏**：teacher 给出推理步骤，student 学习推理轨迹。

优点：
- 降低部署成本和延迟。
- 把强模型能力迁移到小模型。
- 可结合领域数据做垂直优化。

缺点：teacher 错误会被继承，蒸馏数据质量很关键。

### 1. 为什么要做强弱蒸馏

强模型效果好，但部署成本高、延迟高、可能不能私有化。弱模型便宜、快、可控，但能力不足。蒸馏就是用强模型的能力帮助弱模型提升。

常见目标：

- 降低线上推理成本。
- 提升小模型在垂直场景的表现。
- 把强模型的表达、格式和推理模式迁移下来。
- 在私有化场景中减少对外部 API 的依赖。

### 2. 蒸馏的风险

- teacher 也会幻觉。
- teacher 的风格可能不适合业务。
- 生成数据如果不质检，会污染 student。
- 小模型容量有限，不一定能学会复杂能力。

所以蒸馏通常要加人工抽检、规则校验和验证集评估。

### 3. 面试总结

蒸馏不是简单让大模型生成数据，而是用强模型辅助构造高质量训练信号，再让小模型在目标场景上低成本运行。

### 4. 结合跨境支付项目怎么说（面试口径）

> 在支付项目里，我会用强模型帮我们生成或筛选训练数据，再蒸馏到小模型。比如用强模型读商户审核材料，生成结构化审核意见，再让风控专家抽样校验。
>
> 通过这种方式，可以把强模型的表达能力和业务归纳能力迁移到一个更小的模型里。最后小模型部署在内网，延迟和成本都比直接调用大模型低，也更符合敏感数据私有化要求。
>
> 但 teacher 输出不能全信。支付场景里公司名、账号、国家限制、制裁名单这些都不能错，所以我会加规则校验、人工抽检和业务验证集，保证蒸馏数据质量。

**钩子**：蒸馏能降本，但支付领域要严控 teacher 数据质量。
