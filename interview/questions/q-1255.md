# vector的reserve和resize区别

## 答案

`std::vector` 的 `size` 表示当前已构造的元素数量，`capacity` 表示当前已分配、无需重新分配就能容纳的元素数量。`reserve` 只调整容量，`resize` 调整实际元素数量。

### 1. reserve

```cpp
std::vector<int> values;
values.reserve(100);
```

执行后通常：

```text
size = 0
capacity >= 100
```

特点：

- 不创建 100 个 `int` 元素；
- 不能通过 `values[0]` 访问，因为 size 仍为 0；
- 后续 `push_back/emplace_back` 在 capacity 内不会重新分配；
- 参数小于等于当前 capacity 时通常不做任何事；
- 不会缩小容量，缩容可考虑 `shrink_to_fit`，但它只是非强制请求。

### 2. resize

```cpp
std::vector<int> values;
values.resize(100);
```

执行后：

```text
size = 100
capacity >= 100
```

新增元素会被值初始化，`int` 为 0；也可以指定填充值：

```cpp
values.resize(100, 7);
```

如果缩小 size，尾部元素会被销毁，但 capacity 通常不变。

### 3. 主要区别

| 操作 | 改变 size | 可能改变 capacity | 构造/销毁元素 | 典型用途 |
|---|---|---|---|---|
| `reserve(n)` | 否 | 是，只扩不缩 | 否 | 已知将追加约 n 个元素 |
| `resize(n)` | 是 | 扩大时可能 | 是 | 需要 n 个可直接访问的元素 |

### 4. 迭代器失效

- `reserve` 若触发重新分配，所有 iterator、pointer 和 reference 失效；
- `reserve` 未重新分配时不失效；
- `resize` 扩容并重新分配时全部失效；
- `resize` 缩小时，被删除元素及其后的迭代器失效；
- 依赖旧指针前必须理解当前实现是否发生重分配。

### 5. 正确示例

```cpp
#include <iostream>
#include <vector>

int main() {
    std::vector<int> a;
    a.reserve(3);
    a.push_back(10);
    a.push_back(20);

    std::vector<int> b;
    b.resize(3, 7);
    b[0] = 10;

    std::cout << a.size() << " " << a.capacity() << "\n";
    std::cout << b.size() << " " << b[0] << " " << b[2] << "\n";
}
```

`reserve` 适合避免重复扩容和元素搬移；`resize` 适合先建立有效长度再按索引写入。对昂贵对象，误用 `resize` 会产生大量不必要的默认构造。
