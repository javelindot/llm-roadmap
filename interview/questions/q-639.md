# Java中常用的集合及其用途是什么？

## 答案

Java 集合框架主要围绕 `List`、`Set`、`Queue/Deque` 和 `Map` 四类接口，选型要看是否有序、是否允许重复、查询方式、并发需求和复杂度。

### List

- `ArrayList`：动态数组，随机访问 $O(1)$，尾部追加通常摊销 $O(1)$，中间插入删除需移动元素。最常用的顺序容器。
- `LinkedList`：双向链表，同时实现 `Deque`。已定位节点后的插删快，但按索引访问是 $O(n)$，对象开销和缓存局部性较差，通常不应只因“插入多”就盲选。
- `CopyOnWriteArrayList`：写时复制，适合读多写极少的并发场景；写入和内存成本高。

### Set 与 Map

- `HashSet`/`HashMap`：基于哈希，平均查询、插入为 $O(1)$；元素或 key 需要正确实现 `equals` 和 `hashCode`。
- `LinkedHashSet`/`LinkedHashMap`：在哈希结构上维护插入或访问顺序，后者可实现 LRU。
- `TreeSet`/`TreeMap`：红黑树，有序操作为 $O(\log n)$，支持范围查询；元素需可比较或提供 `Comparator`。
- `ConcurrentHashMap`：高并发 Map，支持并发读写，但复合操作应使用 `compute`、`putIfAbsent` 等原子 API。

### Queue 与 Deque

- `ArrayDeque`：高效双端队列，适合栈和普通队列，通常优于 `Stack` 和以 `LinkedList` 实现队列。
- `PriorityQueue`：堆结构，每次取最小或最大优先级元素，插入和删除堆顶为 $O(\log n)$。
- `BlockingQueue`：如 `ArrayBlockingQueue`、`LinkedBlockingQueue`，用于生产者消费者和线程池任务队列。

还要注意集合通常不保证线程安全；`Collections.synchronizedXxx` 是粗粒度包装。若只需要不可变视图，可使用 `List.of`、`Set.of`、`Map.of` 或防御性拷贝，减少误修改。
