# 你是如何利用多Agent协同来提高推理正确率的?

## 答案

多 Agent 提升正确率的前提是角色提供互补信息或独立验证，而不是让多个同质模型重复讨论。架构通常采用 Planner、并行 Specialist、Verifier 和 Aggregator。

### 协作流程

1. Planner 将任务拆成可验证子问题，并定义输出 schema。
2. 不同 Specialist 分别负责检索、计算、代码执行或领域分析，可并行获取独立证据。
3. Critic/Verifier 检查事实引用、约束、计算结果和冲突。
4. Aggregator 只基于通过验证的证据合成答案，分歧无法解决时追问或拒答。

对于高风险结论，可让两个 Agent 独立求解，再用可执行工具或外部数据裁决，而不是简单多数投票。多个 Agent 共享结构化任务板和证据引用，避免上下文失真。

### 保证可靠性

- 每个角色使用最小工具权限和明确成功标准。
- 对代码、数学、SQL 优先使用执行器验证。
- 设置最大轮数、超时、token 与费用预算，防止辩论循环。
- 状态持久化、步骤幂等，失败后可恢复。
- Aggregator 不得引入 Agent 未提供的新事实。

### 证明是否有效

在同一任务集上比较单 Agent、同等 token 预算的 self-consistency 和多 Agent。统计任务成功率、事实错误、成本、延迟和错误相关性。若多个 Agent 的错误高度相关，增加角色数量不会提升正确率。

生产中常只对困难或低置信任务启用多 Agent，简单任务走单 Agent 快速路径，以获得质量与成本平衡。
