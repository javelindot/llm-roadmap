# AUC，机器学习里样本不均衡怎么解决，二分类的话如果负样本过多对AUC有什么影响，sigmoid的平均值会怎么变

## 答案

这道题要区分训练集不均衡、评测集采样比例和指标本身的性质。

### AUC 与负样本过多

ROC-AUC 可以解释为随机取一个正样本和一个负样本，模型把正样本排在负样本前的概率：

$$
AUC=P(s(x^+)>s(x^-))
$$

因此当正负类的条件分数分布不变时，单纯重复或增加同分布负样本不会系统性改变理论 AUC，它对类别先验比例相对不敏感。但有限样本下估计方差会变化；如果新增的是更难的负样本，负类条件分布发生改变，AUC 会下降。

训练时负样本过多会让未加权 BCE 的梯度主要由负类贡献，模型可能偏向预测负类，从而影响排序能力和召回率，间接改变 AUC。

### 不均衡处理

- 分层划分训练/验证集，确保评测集接近真实业务分布。
- 对少数类过采样、对多数类欠采样，或使用 hard negative mining。
- 使用 class weight、focal loss、balanced softmax 等代价敏感方法。
- 调整决策阈值，并按业务成本优化 precision、recall、F1 或收益。
- 生成式过采样需防止合成噪声和数据泄漏。

严重不均衡时 ROC-AUC 可能看起来很高但正类 precision 很低，应同时看 PR-AUC、Recall@Precision、混淆矩阵和校准。

### Sigmoid 平均值

若模型经过良好概率校准，在所评估人群上平均预测概率应接近正类比例。负样本比例增加时，整体 sigmoid 输出均值通常下降。对一个固定模型，仅改变评测集采样比例，也会因为加入更多低分负样本而机械性下降；但这不代表每个样本的 sigmoid 值发生变化。

若用改变类别比例的数据重训，模型截距也会向负类移动。经过重采样或 class weighting 后，输出往往不再是原始业务分布下的校准概率，需要做先验修正或 Platt scaling、isotonic calibration。
