# 如何理解强化学习中熵的概念，如何保证训练过程中熵保持在较高的水平。

## 答案

强化学习中的熵衡量策略分布的不确定性或随机程度：

```text
H(pi(.|s)) = - sum_a pi(a|s) log pi(a|s)
```

- 熵高：多个动作概率接近，探索较多；
- 熵低：策略集中在少数动作，利用较多；
- 熵为 0：确定性地选择一个动作。

### 1. 为什么需要较高熵

训练早期如果熵快速下降，策略会过早锁定在局部最优动作，无法发现更好的策略。在大模型后训练中，表现为 rollout 高度重复、固定模板、工具调用单一或 reward hacking。

但目标不是让熵永远越高越好。训练后期应该逐渐从探索转向利用，否则策略会一直随机，任务效果无法收敛。

### 2. 熵正则

在目标函数中加入 entropy bonus：

```text
J = J_policy + alpha * H(pi)
```

`alpha` 控制探索强度。也可以设定 target entropy，根据当前熵动态调整 `alpha`。SAC 就使用了最大熵强化学习思想，并可自动调节温度参数。

### 3. 控制策略更新速度

- 降低学习率；
- 使用 PPO clip；
- 加 KL penalty；
- 减少同一批 rollout 的重复更新次数；
- 做梯度裁剪；
- reward normalization/clipping。

这些方法可以避免一次高奖励样本让策略概率突然集中。

### 4. 提高数据和采样多样性

- 提高 rollout sampling temperature；
- 调整 top-p；
- 增加每个 prompt 的采样数量；
- 对重复回答去重；
- 混合不同难度和不同领域数据；
- 加入 hard case、对抗样本和分布外样本；
- 必要时混入 SFT/replay 数据稳定策略。

### 5. 修复奖励函数

如果某个固定动作总能拿高分，单纯增加 entropy coefficient 只能缓解表象。还要检查：

- reward 是否偏爱长度或模板；
- 是否只奖励安全拒答而忽略任务完成；
- judge 是否被固定措辞欺骗；
- 工具调用和最终状态是否真的可验证；
- 不同任务奖励尺度是否失衡。

跨境支付 Agent 不能因为“转人工”安全就总是转人工，也不能因为“解决率”高就编造交易结果。奖励需要同时覆盖事实、流程、安全和任务完成。

### 6. 监控指标

- token/action entropy；
- KL divergence；
- unique response ratio；
- 工具使用分布；
- group 内答案相似度；
- reward 均值和方差；
- held-out 成功率；
- 拒答率和误拒率。

### 7. 面试总结

熵表示策略的不确定性，高熵促进探索，低熵促进利用。训练中可以通过 entropy bonus、target entropy、KL、较小学习率、多样化采样和修复奖励函数防止过早塌缩；最终目标是合理的熵退火，而不是始终保持最大随机性。
