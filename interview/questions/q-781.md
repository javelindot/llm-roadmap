# KV Cache显存计算方法

## 答案

对 Decoder-only Transformer，单条序列的 KV Cache 大小近似为：

\[
\text{Bytes}
=2\times L\times S\times H_{kv}\times D_h\times B_e
\]

其中：

- `2`：Key 和 Value 两份；
- \(L\)：Transformer 层数；
- \(S\)：已缓存 token 数，即 prompt 加已生成 token；
- \(H_{kv}\)：KV head 数；
- \(D_h\)：每个 head 的维度；
- \(B_e\)：每个元素字节数，如 FP16/BF16 为 2，FP8/INT8 为 1。

Batch 为 \(N\) 时再乘 \(N\)。MHA 中 \(H_{kv}=H_q\)，GQA/MQA 使用更少 KV head，因此 cache 显著下降。

### 计算示例

以 Qwen2-7B 的 28 层、4 个 KV head、head dimension 128、BF16 为例：

\[
2\times28\times4\times128\times2
=57{,}344\text{ Bytes/token}
\]

约为 56 KiB/token。单条 32K 序列约占：

\[
56\text{ KiB}\times32768\approx1.75\text{ GiB}
\]

这只是 KV Cache，不含模型权重、激活、CUDA graph、临时 workspace 和分配器碎片。

### 分布式与工程修正

- Tensor Parallel 若按 KV head 分片，每卡约除以有效 TP 数；
- KV head 数少于 TP 时可能复制或采用特殊分片，不能机械相除；
- PagedAttention 减少预留和碎片，不改变有效 KV 数据量；
- Sliding Window Attention 只保留窗口内 KV，可将 \(S\) 替换为窗口长度；
- prefix cache 可跨请求复用相同前缀，但需要额外缓存预算；
- 实际容量规划要加入 block 对齐、并发峰值和安全余量。

### 面试总结

KV Cache 核心公式是 `2 × 层数 × token 数 × KV头数 × head维度 × 元素字节`。GQA/MQA 和低精度直接降低每 token 大小，PagedAttention 主要减少碎片；最终还要结合并发、TP 分片和运行时开销。
