# 现有一个能力较弱的多模态模型和一个能力较强的文本模型（如 DeepSeek-R1），如何结合两者的能力来回答与多模态相关的问题？

## 答案

最稳妥的方案不是让弱多模态模型直接给最终答案，而是把它当“感知与证据抽取层”，强文本模型当“推理与表达层”，中间通过结构化证据和可验证工具连接。

### 1. 基本链路

```text
图片/PDF/视频
-> 多模态模型或 OCR/版面模型
-> 结构化事实 + 坐标 + 置信度
-> 强文本模型推理
-> 工具核验
-> 带引用的最终答案
```

多模态模型输出不应只是长描述，而应包含：

- 文本与 OCR box；
- 表格单元格和行列关系；
- 图中对象、属性、位置；
- 页码、区域坐标、时间戳；
- 每条事实的置信度；
- 无法辨认的字段。

### 2. 分阶段协作

1. 文本模型先分析问题，决定需要从视觉输入提取什么；
2. 多模态模型按目标做局部裁剪、OCR、表格或图表解析；
3. 文本模型检查字段完整性，必要时再次请求局部放大；
4. 对金额、日期、证件号等关键字段调用 OCR/规则双重校验；
5. 文本模型仅基于已提取证据推理，并引用页码或区域。

这比“一次把整张图描述给 R1”更可控，也减少弱视觉模型遗漏关键细节。

### 3. 训练型方案

若有数据和算力，可进一步：

- 用强文本模型生成推理轨迹，蒸馏给多模态模型；
- 训练 projector/Q-Former，将视觉 token 对齐到强文本模型；
- 对高质量图文指令做 SFT；
- 用可验证 VQA、OCR、表格任务做偏好或强化学习；
- 使用多 Agent：感知、核验、推理、审查分工。

但若强文本模型本身不支持视觉 token，不能仅靠 Prompt 直接输入图像，必须先做模态转换或训练连接模块。

### 4. 跨境支付场景

处理护照、营业执照、银行回单、拒付材料时：

- 多模态模型定位字段和图章；
- OCR 引擎读取原文；
- 规则校验日期、币种、金额和证件格式；
- 后端系统核对真实交易；
- 文本模型总结差异和生成审核草稿；
- 低置信或高风险命中转人工。

涉及 KYC/KYB 和资金凭证时，不能把多模态模型描述当最终事实，更不能让模型根据图片直接批准商户或放款。

### 5. 评估

- 感知层：OCR CER/WER、字段 F1、定位 IoU；
- 推理层：答案准确率、证据一致率；
- 系统层：端到端成功率、重试率、人工接管率；
- 安全层：伪造材料漏检、跨租户泄漏和越权率。

**面试总结**：弱视觉模型负责“看见并引用”，强文本模型负责“推理和组织”，确定性工具负责“核真”，三者职责不能混在一起。
