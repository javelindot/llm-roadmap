# transformer相比于其他模型架构的优势

## 答案

Transformer 相比 RNN/LSTM、传统 CNN 序列模型的核心优势是并行计算、全局依赖和规模扩展，但不是所有场景都更优。

### 主要优势

- **训练并行**：序列各位置可同时计算；RNN 必须按时间步递归；
- **依赖路径短**：任意两个 token 一层即可交互，长距离信息不必经过多个时间步；
- **适合加速器**：主体是大矩阵乘，GPU/TPU 利用率高；
- **多头表示**：不同子空间可学习语义、位置、指代和局部模式；
- **规模化成熟**：支持数据、张量、流水线、专家和序列并行；
- **架构通用**：文本、视觉、语音、多模态、推荐和强化学习可统一为 token 建模；
- **迁移能力强**：大规模预训练后可 SFT、PEFT、RAG 和 Agent 化。

### 相比 CNN

CNN 依赖局部卷积，多层才能覆盖远距离；Transformer 的感受野可立即全局，关系由内容动态决定。但 CNN 有更强局部归纳偏置，小数据和端侧视觉任务可能更高效。

### 相比 RNN/LSTM

Transformer 避免时间步串行和长路径梯度问题，但标准 Attention 为 \(O(n^2)\)，自回归生成仍逐 token 且需要 KV Cache。

### 当前补充

状态空间模型和 Transformer-SSM 混合架构尝试以近线性复杂度处理长序列。说明 Transformer 的优势在通用性和成熟生态，不代表它在所有长度与硬件上永远最优。

### 面试总结

Transformer 以并行矩阵计算、全局动态交互和强扩展性胜过传统 RNN/CNN，代价是长序列二次 Attention 和自回归 KV 成本。选型仍取决于数据、长度、实时性和硬件。
