# RAG架构有哪些模块？

## 答案

**一句话**：RAG 架构通常由数据接入、文档处理、向量化、索引存储、检索召回、重排、上下文构造、答案生成、评估监控这几大模块组成。

### 1. 数据处理模块

负责把原始知识变成可检索数据，包括：

- 文档采集。
- PDF/Word/网页解析。
- OCR 和表格抽取。
- 清洗去重。
- 敏感信息脱敏。
- chunk 切分。
- 元数据补充。

这是 RAG 的基础，文档质量差，后面模型再强也很难救。

### 2. 索引模块

负责把文本变成检索结构：

- embedding 模型生成向量。
- 向量库保存向量。
- ES/BM25 保存关键词索引。
- 元数据库保存来源、权限、版本、生效时间。

如果需要混合检索，向量索引和关键词索引都要有。

### 3. 查询和召回模块

用户提问后，会经过：

- 意图识别。
- query rewrite。
- 向量召回。
- 关键词召回。
- 元数据过滤。
- rerank 重排。

目标是把最相关、最可信、最符合权限的内容找出来。

### 4. 生成和校验模块

把检索结果构造成 prompt，让 LLM 生成答案。之后还要做：

- 引用来源。
- 幻觉检测。
- 敏感信息过滤。
- 答案格式校验。
- 低置信度转人工。

### 5. 结合跨境支付项目怎么说（面试口径）

> 在跨境支付客服项目里，RAG 架构可以分成四层。第一层是知识处理层，把 FAQ、通道规则、错误码、KYC 材料要求、退款 SOP 清洗切分；第二层是索引层，用 embedding 做向量索引，同时用 ES 做错误码和关键词索引；第三层是检索层，根据用户问题做 query rewrite、混合召回和 rerank；第四层是生成层，要求模型基于检索内容回答并给出引用。
>
> 支付场景里我会特别强调元数据和权限。每个 chunk 要带国家、币种、通道、产品线、生效时间和可见范围。用户问巴西 PIX，不能召回欧洲 SEPA；普通客服也不能看到完整风控规则。
>
> 最后还要有评估模块，用真实工单评估召回率、引用命中率、答案准确率和转人工率。RAG 不是一次搭完就结束，而是持续运营。

**钩子**：可以引到“数据处理、混合召回、元数据权限、评估闭环”。
