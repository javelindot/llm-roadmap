# 🔥 3. 知道 LoRA 的原理吗， A 和 B 两个矩阵怎么初始化，有了解过其他的初始化方法吗

## 答案

### 1. LoRA 原理

LoRA 冻结原权重 `W`，只学习一个低秩增量：

```text
W' = W + ΔW
ΔW = (α / r) · B A
```

其中：

- `A ∈ R^(r × d_in)`：降维矩阵
- `B ∈ R^(d_out × r)`：升维矩阵
- `r` 远小于输入、输出维度
- `α / r` 控制增量的缩放

这样训练参数从 `d_out × d_in` 降为 `r × (d_in + d_out)`。

### 2. A、B 如何初始化

原始 LoRA 的常见做法是：

- `A` 使用随机高斯或 Kaiming 初始化。
- `B` 初始化为 0。

因此训练开始时：

```text
ΔW = BA = 0
```

模型初始行为与基座模型完全一致。第一步主要更新 `B`；当 `B` 不再为 0 后，`A` 也会获得有效梯度。

也可以反过来令 `A=0、B` 随机，仍能保证初始增量为 0，但主流实现更常采用前一种。**A、B 同时初始化为 0 不可取**，因为两者会互相导致梯度为 0。

### 3. 其他初始化方法

- **PiSSA**：对原权重做 SVD，用主奇异子空间初始化低秩矩阵，通常收敛更快。
- **OLoRA**：利用 QR 分解初始化，使优化从更有意义的子空间开始。
- **EVA**：基于真实激活的 SVD 分配和初始化低秩适配器。
- **LoftQ**：在量化权重上联合考虑量化误差，适合 QLoRA 初始化。
- **DoRA**：不是单纯初始化方法，而是把权重幅值与方向分开学习，改善低秩适配表达。

### 4. 工程选型

普通指令微调使用默认初始化通常足够；数据少、训练步数短时可考虑 PiSSA/EVA；量化微调时可考虑 LoftQ。无论哪种方法，都应比较收敛速度、最终任务指标和额外预处理成本。

**面试总结**：LoRA 默认初始化的关键目标是让 `ΔW=0`，保证从原模型无扰动启动；改进初始化则希望找到更有信息量的低秩子空间。
