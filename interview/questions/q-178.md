# 文档里的图片处理后存到哪里去了？

## 答案

**一句话**：文档里的图片通常会单独抽取出来存到对象存储或文件系统，正文里保留图片引用、页码、位置和描述信息。

### 核心回答
常见做法：
- 图片原文件存对象存储，如 S3、MinIO、OSS。
- 数据库保存图片 metadata：文档 ID、页码、坐标、类型、hash。
- 文本 chunk 中保留图片占位符和引用链接。
- 对图片做 OCR、caption 或多模态 embedding。
- 检索时可以通过文本召回图片，也可以图文向量联合召回。

不要把图片直接塞进纯文本 chunk，否则会丢失视觉信息和追溯能力。

### 1. 为什么图片要单独存

图片本身不是纯文本，直接丢进文本 chunk 会丢失视觉信息。单独存储可以保留原始证据，方便后续 OCR、多模态检索、人工复核和审计。

### 2. 图片和文本怎么关联

一般会在文本里保留占位符或引用，例如：

- 图片 ID。
- 文档 ID。
- 页码。
- 坐标区域。
- 图片 caption。
- OCR 文本。

这样回答时既能引用图片，也能回到原文档位置。

### 3. 面试总结

图片处理要做到"原图留存、文本关联、多模态可扩展、证据可追溯"。

### 4. 结合跨境支付项目怎么说（面试口径）

> 在支付文档里，图片可能是流程图、费率截图、证照样例或表格截图。我会把图片单独存对象存储，chunk 里保留引用，比如第几页第几个图。
>
> 如果图片有文字，就 OCR；如果是流程图，可以生成简短 caption；如果是证照样例，可以做多模态索引。这样模型回答时能引用原始图片，业务也能回看证据。
>
> 支付和合规场景里，原始证据很重要。不能解析完只剩一段模型生成的描述，后面出了问题没法追溯。

**钩子**：图片要原始存储 + 文本引用 + 可追溯，不能解析完就丢。
