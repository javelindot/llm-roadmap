# Vision Transformer (ViT) 架构

## 答案

**一句话**：ViT = **把图像切成 patch + 当 token 喂 Transformer Encoder**——证明"纯 Transformer 不用 CNN 归纳偏置也能做视觉"，是多模态时代的视觉骨干。

### 1. 整体结构
```
图像 224×224×3
  → Patch Embedding（切成 14×14=196 个 16×16 patch）→ Linear 投影到 d 维
  → 加 [CLS] token + 位置编码
  → N × Transformer Encoder（Self-Attn + FFN + LN + Residual）
  → [CLS] 输出 → MLP head → 分类
```

### 2. Patch Embedding（关键设计）
- 输入：H×W×3
- patch_size = P（通常 14、16）
- 切成 (H/P) × (W/P) 个 patch
- 每个 patch 展平成 P²×3 向量
- 用 Linear 投到 d 维
- 等价于一个 stride=P、kernel=P 的 conv2d

### 3. [CLS] Token 和位置编码
- 在 token 序列开头加 [CLS]
- 学习的可训 embedding
- attention 后 [CLS] 输出当全局表征
- 位置编码：可学习 absolute（ViT 默认）/ 2D sinusoidal

### 4. ViT 系列
| 版本 | 参数 | patch | depth |
|------|------|-------|-------|
| ViT-S/16 | 22M | 16 | 12 |
| ViT-B/16 | 86M | 16 | 12 |
| ViT-L/16 | 307M | 16 | 24 |
| ViT-H/14 | 632M | 14 | 32 |

### 5. ViT 的优点
- **架构简洁**：和 LLM 同源，统一架构
- **可扩展**：scaling 红利显著
- **多模态友好**：Vision Encoder 接 LLM 投影层
- **可解释**：attention 可视化

### 6. ViT 的劣势（vs CNN）
- **小数据下不如 CNN**：缺归纳偏置
- **计算量大**：高分辨率 patch 多
- **训练敏感**：需要 warmup + 大 batch
- **对位置信息依赖强**

### 7. ViT 变体
- **DeiT**：数据高效 ViT
- **Swin Transformer**：滑窗 + 多尺度
- **DINOv2 / CLIP**：自监督预训练
- **SAM**：分割大模型
- **EVA / InternVL**：多模态 backbone

### 8. ViT 在多模态 LLM 中的角色
- 几乎所有多模态 LLM（LLaVA、Qwen-VL、InternVL、GPT-4V）的 vision encoder 都是 ViT
- 通过投影层接到 LLM
- 训练时常冻 ViT + 训投影层 + 微调 LLM

### 9. 一句话总结
**ViT = Patch + Transformer + [CLS]**——证明视觉不需要 CNN 也能 work，是多模态 LLM 时代的视觉基础设施。

## 结合项目谈（口语版 · 约 25 秒）

> 我们跨境支付**ViT 在 KYB 多模态识别**里是核心。
>
> 场景：商户上传营业执照、税单、合规证明等证件，要识别字段（公司名、统一社会信用代码、有效期、法人等）。早期我们用传统 OCR + 规则提取，错误率高、特别怕印章遮挡和手写体。
>
> 方案演化：
>
> **v1：CLIP-ViT-B/16 + 线性投影 + Qwen-7B**——CLIP vision encoder + 一层 Linear 投影到 LLM 空间，冻 Qwen 只训投影层。5w 张标注证件训 2 天，字段提取准确率 85%。
>
> **v2：升级到 ViT-L/14 + MLP 投影 + Qwen-14B**——更大 vision encoder、两层 MLP 投影。同样数据训出来准确率 91%。
>
> **v3：DINOv2 + LoRA 微调 ViT + Qwen-14B**——DINOv2 自监督预训练比 CLIP 在领域适应上更好。LoRA 微调 ViT 让它对印章、手写体更鲁棒。准确率到 95%。
>
> 几个工程坑：
> - **patch_size 选择**：16 vs 14 对细节识别影响大，证件细节多用 14
> - **输入分辨率**：224 不够，我们提到 448，patch 数翻 4 倍但能看清小字
> - **[CLS] vs Mean Pool**：投影到 LLM 时我们用所有 patch token（保留空间），不只用 [CLS]
> - **冻 vs 微调 ViT**：数据足时微调更好，少时冻 ViT 只训投影层
>
> 一句话：**ViT 在我们多模态场景是绝对主力**，CNN 时代彻底翻篇。
