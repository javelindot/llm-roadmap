# LoRA介绍，和全参sft怎么选择？小模型的sft和大模型的LoRA效果对比会是怎么样的？sft数据集如何构建？

## 答案

LoRA 和全参 SFT 的训练目标都可以是 assistant token 的交叉熵，区别在于更新哪些参数。

### 1. LoRA 原理

\[
W'=W+\frac{\alpha}{r}BA
\]

冻结基座，只训练低秩 \(A,B\)。它大幅减少梯度和优化器状态，适合资源有限、多任务 adapter 和中小数据适配。

### 2. 如何选择

优先 LoRA：

- 数据有限，任务主要是格式、风格或领域适配；
- 训练/存储预算有限；
- 需要多个业务 adapter；
- 希望降低遗忘风险。

考虑全参：

- 领域差异大且数据规模、质量足够；
- 需要获得新底层能力或修改所有层表示；
- LoRA 扩 rank、覆盖 FFN 后仍明显饱和；
- 有足够算力并能做通用能力回归。

### 3. 小模型全参 vs 大模型 LoRA

没有固定胜负：

- 大模型 LoRA 保留更强基座知识与推理，复杂任务常有优势；
- 小模型全参可更彻底适配，推理便宜、延迟低；
- 若任务窄且标签充分，小模型可能达到更好的成本收益；
- 若任务依赖模型本身没有的能力，全参也不一定补得回来；
- 应在相同数据上比较质量、P99、吞吐、显存和单位成本。

### 4. SFT 数据集

- 明确任务 taxonomy 和输出规范；
- 收集人工黄金数据、审核后的日志与合成困难样本；
- 去重、脱敏、去污染和质量分层；
- 覆盖正常、边界、拒答、工具错误恢复和长尾语言；
- 按用户/时间切分；
- 使用正确 chat template、BOS/EOS 与 assistant-only loss mask；
- 平衡数据源，避免模板化和过拟合；
- 版本化并将线上 bad case 审核后回流。

跨境支付中，知识更新用 RAG，资金动作由 workflow 控制；SFT 主要学习理解、表达和合法工具调用。

### 面试总结

LoRA 适合低成本适配，全参适合数据充分且任务偏移大的能力重塑；大模型 LoRA 与小模型全参必须按质量和部署成本实测。SFT 成败首先取决于数据治理与正确 loss mask。
