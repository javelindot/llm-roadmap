# 对推荐系统中的重排模块了解哪些?

## 答案

推荐系统的重排位于精排之后，对一个候选列表整体做最后调整。精排更关注单个 item 的预测分数，重排则关注整个展示列表的总效用、约束和用户体验。

### 1. 重排要解决的问题

- 多目标：点击、转化、时长、利润、满意度；
- 多样性：避免连续展示同类内容；
- 去重：相同商品、商家或主题不能重复；
- 新颖性与探索：给新品、长尾内容适当曝光；
- 公平性：控制不同供给方的曝光；
- 业务约束：库存、地域、频控、广告位、运营规则；
- 长期价值：避免只追求短期点击；
- 系统约束：控制模型延迟和候选规模。

### 2. 常见重排方法

**规则重排**

按类目、作者、商家、广告位等做打散、去重、置顶和频控。易解释、易回滚，是生产系统的重要基线。

**分数融合**

```text
score =
  w1 * pCTR
+ w2 * pCVR
+ w3 * value
- w4 * risk
```

需要对不同分数做校准，并根据场景设置权重。

**MMR**

在相关性和多样性之间权衡：

```text
MMR = lambda * relevance
    - (1-lambda) * max_similarity_to_selected
```

适合减少列表内相似内容。

**DPP**

通过行列式概率建模集合质量和多样性，理论优雅，但大候选集计算成本较高。

**Learning-to-Rank / Slate Model**

用 pointwise、pairwise、listwise loss 学习排序，或直接对整个 slate 建模，捕获 item 之间的上下文和竞争关系。

**约束优化与强化学习**

将点击、长期价值、多样性和供给约束写成目标与限制。RL/bandit 可优化长期反馈，但需要处理探索风险、反馈延迟和反事实评估。

### 3. 重排特征

- 精排分数和校准概率；
- item 之间相似度；
- 已选列表的类目、作者、价格带分布；
- 用户短期与长期兴趣；
- 曝光频次和疲劳度；
- 新品、长尾和业务优先级；
- 实时库存、距离和服务能力。

### 4. 评估指标

- NDCG、MAP、MRR；
- CTR、CVR、GMV、停留时长；
- ILD、类目覆盖率、重复率；
- 新品和长尾曝光；
- 用户留存、负反馈和投诉；
- P95/P99 延迟。

需要做整页离线回放和线上 A/B，不能只看单 item AUC。

### 5. 跨境支付类比

支付通道路由也可以看成重排：

```text
候选通道
-> 预测成功率、成本、时效和风险
-> 过滤不满足牌照/币种/限额的通道
-> 多目标排序
-> 保留备用通道
```

模型只能在合法且可执行的候选中排序，制裁、合规、限额和通道可用性属于硬约束，不能作为可被低权重抵消的软目标。

### 6. 面试总结

重排不是简单再排一次分数，而是在列表层面同时优化相关性、多样性、长期价值和业务约束。生产上通常是规则、分数融合、学习排序和约束优化的组合，最终通过整页指标和 A/B 实验验收。
