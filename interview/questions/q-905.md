# 八股：Qwen 或 DeepSeek 技术报告中提到的关键创新点有哪些？（如 RoPE 外推、MoE）

## 答案

可以按 Qwen 与 DeepSeek 两条技术路线回答，并区分每一代模型的贡献。

### 1. Qwen 系列

- **Qwen/Qwen1.5**：RoPE、RMSNorm、SwiGLU、byte-level BPE，逐步扩展上下文与模型尺寸；
- **Qwen2**：GQA 降低 KV Cache；DCA+YaRN 支持长上下文；提供 Dense 与 57B-A14B MoE，采用细粒度 routed experts 和 shared experts；增强多语言、代码和数学；
- **Qwen2.5**：预训练数据扩至最多 18T token，增强 JSON、结构化数据、长输出、代码和数学，并提供 Coder/Math/VL 专用模型；
- **Qwen3**：thinking/non-thinking 统一模式，Dense 与 MoE；MoE 如 30B-A3B、235B-A22B 使用 128 experts、每 token 激活 8 个；加强 Agent 与多语言。

### 2. DeepSeek 系列

- **DeepSeekMoE**：细粒度专家分割与 shared expert；
- **DeepSeek-V2**：MLA 将 K/V 压成 latent，减少 KV Cache；DeepSeekMoE 扩容量；
- **DeepSeek-V3**：671B 总参数、37B 激活；无辅助损失负载均衡、Multi-Token Prediction、FP8 训练、DualPipe、通信计算重叠；
- **DeepSeek-R1**：在 V3 基座上使用 GRPO 与多阶段 RL/SFT，R1-Zero 展示纯 RL 推理行为，R1 通过 cold start、rejection sampling 和蒸馏改善可读性与通用性。

### 3. 如何比较

RoPE 外推、MoE、GQA/MLA、训练精度和 RL 属于不同层次。面试中应说明它们分别优化位置、容量、KV、训练效率和后训练，而不是列成同类技巧。

### 面试总结

Qwen 强调 GQA、长上下文、多语言和多尺寸 Dense/MoE；DeepSeek 强调细粒度 MoE、MLA、FP8/并行训练及推理 RL。回答时要把架构、训练和后训练创新分层。
