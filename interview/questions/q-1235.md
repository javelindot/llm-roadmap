# DPO对齐训练的曲线是怎么样的，正例的概率会提升嘛？

## 答案

DPO 训练最重要的曲线不是 chosen 的绝对概率，而是 chosen 相对 rejected、同时相对 reference model 的 **偏好 margin**。标准目标可写为：

```text
chosen_reward =
    beta * (log pi_theta(y_w | x) - log pi_ref(y_w | x))

rejected_reward =
    beta * (log pi_theta(y_l | x) - log pi_ref(y_l | x))

margin = chosen_reward - rejected_reward
loss = -log sigmoid(margin)
```

### 1. 典型曲线

训练初期 policy 与 reference 相同，chosen/rejected reward 都接近 0，margin 接近 0，单样本 loss 接近 `log 2`。随着训练推进，通常看到：

- train loss 下降；
- preference accuracy，即 `margin > 0` 的样本比例上升；
- chosen reward 与 rejected reward 的差距扩大；
- reward margin 上升后逐渐趋稳；
- policy 与 reference 的 KL 或参数偏移增大；
- 生成长度、拒答率和风格可能随数据偏好变化。

验证集曲线比训练集更重要。Train margin 持续变大但 eval loss 回升、人工胜率下降，通常说明过拟合、标签噪声或策略利用了长度、模板等偏差。

### 2. chosen 概率是否一定提升

**不一定。** DPO 约束的是相对差：

```text
log pi_theta(chosen) - log pi_ref(chosen)
>
log pi_theta(rejected) - log pi_ref(rejected)
```

只要 rejected 的概率下降得更多，margin 就能增大；chosen 的绝对 sequence log-prob 可能上升、持平，甚至与 rejected 一起下降。这类现象常被称为 likelihood displacement。

此外，sequence probability 是 token 概率乘积，回答越长，log-prob 的绝对值通常越负，所以不能直接跨长度比较原始序列概率。

### 3. 应监控哪些指标

- train/eval DPO loss；
- preference accuracy 和 reward margin；
- chosen/rejected 的隐式 reward；
- chosen/rejected 的平均 token log-prob；
- policy-reference KL 或可替代的漂移指标；
- chosen/rejected 生成长度；
- SFT loss 或通用能力回归；
- 人工 pairwise win rate；
- 事实性、安全、工具成功率和拒答率。

如果希望 chosen 本身不能被整体压低，可使用 reward anchor、NLL 辅助项或与 SFT loss 混合，但会引入新的权重超参数，需要消融验证。

### 4. 跨境支付数据中的判断

支付客服偏好对中，模型可能仅通过“写得更长、更礼貌”扩大 margin，却没有提升状态和规则正确性。因此除曲线外，还要分别检查金额、币种、订单状态、引用、工具参数和越权行为。最终依据应是冻结业务集与人工盲评，而不是只看 DPO loss。
