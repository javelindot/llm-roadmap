# 如何通过重新排名（Rerank）提升 RAG 效果？

## 答案

**一句话**：rerank = **在向量召回的粗排结果上再用更强但更慢的模型精排**——典型组合是"**双塔召回 top-50 → cross-encoder rerank top-5**"，能让最终 RAG 准确率提升 5-15 个点。

### 1. 为什么需要 rerank
- 向量召回（双塔）快但**精度有限**
- 双塔把 query 和 doc 独立编码，**没有交互**
- top-1 可能不是最相关的（但通常在 top-20 里）
- rerank 用更精的模型从粗排里挑最好的

### 2. Rerank 的核心模型
**Cross-Encoder（最主流）**
- query + doc **拼接后**一起喂模型
- 输出相关性分数
- 比双塔慢 100-1000 倍，但精度高
- 代表：bge-reranker、Cohere Rerank、Jina Reranker

**LLM-as-Reranker**
- 让 LLM 直接判断"这条 doc 是否相关"
- 比 cross-encoder 更准但更贵
- 代表：RankGPT、RankZephyr

**ColBERT（晚交互）**
- 双塔 + token 级交互
- 精度接近 cross-encoder、速度接近双塔
- 折中选择

### 3. 经典 pipeline
```
Query
  → 向量召回 (bge-large, top-100)
  → BM25 召回 (top-100)
  → Hybrid 融合（RRF）→ top-50
  → Cross-Encoder Rerank（bge-reranker）→ top-5
  → LLM 生成
```

### 4. 选型对比
| 方案 | 速度 | 精度 | 用例 |
|------|------|------|------|
| 无 rerank | 快 | 一般 | 简单场景 |
| **bge-reranker-large** | 中 | 高 | **生产主流** |
| Cohere Rerank API | 中 | 高 | 调 API 方便 |
| LLM Rerank（GPT-4） | 慢 | 极高 | 关键场景 |
| ColBERT | 中快 | 中高 | 平衡选 |

### 5. Rerank 怎么提效
- 双塔召回的 top-50 中往往**真相关的混在 5-30 名**
- Rerank 把真相关的提到 top-3
- 端到端 RAG 准确率显著提升

### 6. 工程经验
- top-50 → top-5 是甜点
- rerank 太多（top-200+）成本翻倍但收益低
- batch rerank：把 query 和 50 个 doc 打包一次推理
- 离线评测 nDCG / Recall@5 验证

### 7. Rerank 不是银弹
- 召回里没有的，rerank 也救不回来
- 优先保证召回率 → 再 rerank 提精度
- 召回失败 = 必败

### 8. Rerank 的进化方向
- 跨语言 rerank（多语言场景）
- 多模态 rerank（图文混合）
- Long-context rerank（千 token 级 doc）

### 9. 一句话总结
**召回保 recall，rerank 保 precision——两者缺一不可，rerank 是 RAG 准确率的关键 5-15 个点**。

## 结合项目谈（口语版 · 约 30 秒）

> rerank 在我们跨境支付 RAG 里是**默认标配**，分享下落地。
>
> **基础 pipeline**：bge-large（双塔召回 top-50） + BM25（top-50） → RRF 融合到 top-30 → **bge-reranker-large rerank → top-5 → Qwen-14B 生成**。
>
> 收益数据：去掉 rerank 直接用 top-5 向量召回的准确率 78%；加 rerank 后 88%（**+10**），这是单点最大收益之一。
>
> **几个工程细节**：
>
> 1. **rerank 的 batch size**：bge-reranker-large 在 A100 上 batch=64 推理 600ms，能撑 20 QPS。我们用 vLLM-Reranker 优化后 batch=128 时延迟 400ms，吞吐翻倍。
>
> 2. **rerank 的 top-N 选择**：top-50 → top-5 是甜点，top-100 → top-5 精度只多 0.5%，但延迟翻倍。
>
> 3. **rerank 的 threshold**：rerank 分数低于 0.3 的 doc 我们直接丢，避免低质 doc 进 prompt 污染生成。
>
> 4. **多模型对比**：我们试过 Cohere Rerank API（贵且要出公网，金融合规不行）、Jina Reranker（开源、效果接近 bge）、LLM Rerank（GPT-4 太贵）。最后选 bge-reranker-large 本地部署。
>
> 5. **关键场景上 LLM Rerank**：风控调查这种高价值场景我们额外加一层 Qwen-14B LLM Rerank，准确率再提 3 个点，但成本高、只在 critical path 用。
>
> 一句话：**rerank = RAG 的"准确率放大器"**，不上是浪费召回数据。
