# 如果一个agent误判导致策略冲突，如何处理？

## 答案

**一句话**：Agent 误判导致策略冲突时，要靠优先级规则、冲突检测、人工确认、回滚机制和审计 trace 来处理。

### 处理步骤

1. 识别冲突：比如两个 Agent 给出相反动作。
2. 判断优先级：安全、合规、资金优先级最高。
3. 暂停执行高风险动作。
4. 回到保护节点或转人工。
5. 记录 trace，复盘是意图、工具还是知识问题。
6. 优化规则、prompt 或评估集。

### 结合跨境支付项目怎么说（面试口径）

> 在跨境支付里，如果客服 Agent 建议放行，但风控 Agent 判断高风险，这就是策略冲突。我的原则是安全优先，资金和合规动作不能由模型自动决策，必须进入人工审核或保护节点。
>
> 系统上会有冲突检测，比如风控状态为高风险时，任何自动放行建议都无效。最后记录每个 Agent 的输入、工具调用和输出，方便排查误判原因。

**钩子**：可以引到“安全优先级、保护节点、冲突 trace、人工确认”。
