# RAG检索召回的片段中，有一些质量差的，如果排除掉？

## 答案

排除低质量 RAG 片段不能只设一个相似度阈值，应在入库、召回后过滤、重排和生成前校验多个阶段处理，并保留无证据时拒答的能力。

### 入库前治理

- 去除重复、导航、广告、乱码、OCR 低置信文本和过短无语义片段。
- 保留标题、章节、来源、时间、作者、权限和质量评分。
- 按文档结构分块，避免表格被拆散、标题与正文分离。
- 对来源建立可信等级，对过期或无权限内容不入当前索引。

### 召回后过滤

先做权限、时间、语言、文档状态等硬过滤。然后综合：

- dense similarity 与 BM25 分数；
- query-chunk cross-encoder reranker；
- 文档质量、时效性、来源权威性；
- chunk 是否包含完整可回答信息。

使用绝对阈值前要按业务校准，因为不同 query 的相似度分布不同。还可比较 top-1 与后续候选分差，检测整体低相关查询。

### 去重与证据校验

对相邻或近重复 chunk 做 MMR/聚类，提升证据覆盖。生成前让轻量模型判断片段是否支持问题，或抽取关键 claim 与证据；无法找到支持内容时返回“证据不足”，而不是强行生成。

### 评估闭环

构建带相关文档标注的 query 集，分别评估 Recall@K、Precision@K、nDCG、低质量片段率和最终引用正确率。记录被过滤片段及原因，防止规则误杀。线上 bad case 要区分是解析、chunk、召回、重排还是生成问题，再针对性修复。
