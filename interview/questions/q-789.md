# final关键字的作用

## 答案

在 Java 中，`final` 可修饰变量、方法和类，含义都是“禁止再次改变某个结构关系”，但不等于对象完全不可变。

### 1. final 变量

- 基本类型赋值后不能修改值；
- 引用类型不能再指向其他对象，但对象内部状态仍可变；
- blank final 可在构造器中完成一次赋值；
- `static final` 常用于类级常量，但只有编译期常量才可能被内联。

```java
final List<String> names = new ArrayList<>();
names.add("A");          // 合法：对象内容可变
// names = new ArrayList<>(); // 非法：引用不能重绑定
```

### 2. final 方法

子类不能 override，但仍可继承和调用。`private` 方法本来就不可被真正 override，`static` 方法是隐藏而非 override。

### 3. final 类

不能被继承，例如 `String`。它可用于保持不变量或限制扩展，但类内部字段未必都是不可变的。

### 4. 并发语义

Java Memory Model 对构造阶段正确赋值的 final 字段提供额外初始化安全保证：对象被正确发布后，其他线程更可靠地看到 final 字段值。但这不替代同步，也不自动保护字段所引用对象的后续修改。

`final`、`finally`、`finalize` 是不同概念；`finalize()` 已不应作为资源释放机制，应使用 try-with-resources。

### 面试总结

final 变量不可重新赋值，final 方法不可重写，final 类不可继承。final 引用不代表对象不可变；它还具有一定安全发布语义，但不能替代并发控制。
