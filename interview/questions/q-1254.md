# 编码：在已排序数组中用 O(log n) 实现目标值的左右边界查找

## 答案

在非递减数组中分别查找：

- `lower_bound`：第一个 `>= target` 的位置；
- `upper_bound`：第一个 `> target` 的位置。

如果 `lower_bound` 指向 target，则左边界是它，右边界是 `upper_bound - 1`；否则目标不存在。

```python
def search_range(nums: list[int], target: int) -> tuple[int, int]:
    def lower_bound(value: int) -> int:
        left, right = 0, len(nums)
        while left < right:
            mid = left + (right - left) // 2
            if nums[mid] < value:
                left = mid + 1
            else:
                right = mid
        return left

    def upper_bound(value: int) -> int:
        left, right = 0, len(nums)
        while left < right:
            mid = left + (right - left) // 2
            if nums[mid] <= value:
                left = mid + 1
            else:
                right = mid
        return left

    first = lower_bound(target)
    if first == len(nums) or nums[first] != target:
        return -1, -1

    last = upper_bound(target) - 1
    return first, last
```

### 1. 区间定义

代码始终维护半开区间 `[left, right)`：

- 初始为 `[0, n)`；
- 当 `nums[mid]` 不可能成为答案时令 `left = mid + 1`；
- 当 `mid` 仍可能是答案时令 `right = mid`；
- 结束时 `left == right`，就是第一个满足条件的位置。

使用半开区间可以自然返回 `n`，表示所有元素都小于目标条件。

### 2. 为什么不能找到一个 target 后线性扩展

先二分找到任意 target，再向左右逐个扫描，最坏情况下数组全部等于 target，复杂度会退化为 `O(n)`。左右边界必须分别二分。

### 3. 正确性要点

`lower_bound` 的循环不变量：

- `[0, left)` 中所有值 `< target`；
- `[right, n)` 中所有值 `>= target`。

`upper_bound` 的循环不变量：

- `[0, left)` 中所有值 `<= target`；
- `[right, n)` 中所有值 `> target`。

循环结束时左右区间相接，因此得到第一个 `>=` 或第一个 `>` 的位置。

### 4. 测试

```python
cases = [
    ([], 1, (-1, -1)),
    ([1], 1, (0, 0)),
    ([1], 2, (-1, -1)),
    ([1, 2, 2, 2, 3], 2, (1, 3)),
    ([2, 2, 2], 2, (0, 2)),
    ([1, 2, 3], 0, (-1, -1)),
    ([1, 2, 3], 4, (-1, -1)),
]

for nums, target, expected in cases:
    assert search_range(nums, target) == expected
```

时间复杂度为 `O(log n)`，额外空间为 `O(1)`。若语言标准库提供 `bisect_left/bisect_right`，生产代码可以复用；手写题重点是边界条件和循环不变量。
