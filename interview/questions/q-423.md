# 🔥 Transformer 基础八股全问答

## 答案

下面是一套 Transformer 基础问题的高频回答框架。

### 1. 为什么需要 Attention

它允许每个 Token 直接聚合序列中其他位置的信息，长距离依赖路径长度为 1，并且训练时可以并行处理整个序列。

### 2. Self-Attention 怎么算

```text
Q = XW_Q, K = XW_K, V = XW_V
Attention(Q,K,V) = softmax(QK^T / sqrt(d_k)) V
```

`QK^T` 衡量相关性，Softmax 转成权重，再对 V 加权求和。

### 3. 为什么除以 `sqrt(d_k)`

维度增大时点积方差会变大，Softmax 容易进入饱和区，梯度变小。缩放后数值更稳定。

### 4. Multi-Head 有什么用

多个头使用不同投影，在不同子空间关注不同关系。最终拼接后通过输出矩阵融合。

### 5. 为什么需要位置编码

纯 Attention 对输入顺序本身不敏感。位置编码用于注入绝对或相对位置信息，现代 LLM 常用 RoPE。

### 6. Encoder 和 Decoder 的区别

- Encoder 使用双向注意力，适合理解和表征。
- Decoder 使用因果掩码，只看前文，适合自回归生成。
- 原始 Decoder 还包含读取 Encoder 输出的 Cross-Attention。

### 7. FFN 的作用

Attention 负责跨 Token 混合信息，FFN 对每个 Token 做非线性特征变换。LLM 中大量参数位于 FFN，常用 SwiGLU。

### 8. Residual 和 Normalization 的作用

残差连接帮助信息与梯度跨层传播；LayerNorm/RMSNorm 稳定激活尺度。现代深层 LLM 多采用 Pre-Norm。

### 9. 复杂度是多少

标准 Attention 的时间和注意力矩阵空间复杂度随序列长度约为 `O(n²)`；线性投影和 FFN 还包含与隐藏维度相关的计算。

### 10. 训练为什么能并行，生成为什么不能

训练时已知完整目标序列，可用因果 Mask 一次算完；生成时下一个 Token 依赖刚生成的 Token，只能自回归循环。

### 11. KV Cache 是什么

推理时缓存历史 Token 的 K/V，新一步只计算新 Token 的 Q/K/V，避免重复计算历史部分。代价是显存随批大小和上下文长度增长。

### 12. MHA、GQA、MQA 的区别

- MHA：每个 Query 头有独立 K/V。
- GQA：一组 Query 头共享 K/V。
- MQA：所有 Query 头共享一组 K/V。

共享越多，KV Cache 越小，但可能牺牲部分模型质量。

**面试建议**：先用“Attention + FFN + 残差归一化 + 位置编码”讲清主干，再根据追问展开公式、复杂度和推理优化，不要一开始堆满术语。
