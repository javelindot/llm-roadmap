# 微调是如何进行微调的？为什么LoRA能够work？除了LoRA外，还了解哪些微调方法？

## 答案

微调是从预训练模型出发，用目标领域或任务数据继续优化。应先明确目标是注入领域知识、学习输出格式、提升指令遵循还是偏好对齐，因为对应 CPT、SFT 和 DPO 等不同阶段。

### 基本流程

定义评测集后，完成数据清洗、去重、模板化和 train/validation/test 切分；加载基础模型与 tokenizer，选择全参或 PEFT，设置 label mask、学习率、global batch、序列长度和 scheduler；训练中监控 loss、梯度、吞吐和分桶指标，最终做通用能力与安全回归。

### LoRA 为什么有效

LoRA 冻结 $W_0$，学习：

$$
\Delta W=BA
$$

其中 rank $r$ 远小于原矩阵维度。经验上，任务适配的有效权重变化常集中在较低维子空间，因此低秩增量能用少量参数表达主要变化。它降低梯度、优化器状态和 checkpoint 成本，同时保留预训练权重。

LoRA 常用于 Q/K/V/O 和 FFN 投影，rank、alpha、dropout 和目标层需通过实验选择。它主要节省训练资源，并不意味着基础模型前向计算消失。

### 其他方法

- 全参数或部分层微调：容量最大，但资源和遗忘风险高。
- Adapter：插入瓶颈模块，模块化强但增加推理层。
- Prompt/Prefix/P-Tuning：训练软 token 或各层 prefix。
- IA3、BitFit：训练缩放向量或 bias，参数更少。
- QLoRA：量化冻结基础模型并训练 LoRA。
- AdaLoRA、DoRA：动态 rank 分配或分离权重方向与幅值。

选择时比较质量、显存、吞吐、推理延迟、多任务切换和部署复杂度。通常先以 LoRA 建立成本较低的基线，容量不足再扩大 rank、解冻部分层或做全参训练。
