# 介绍 DeepResearch 几篇工作？主 Agent 和子 Agent 有什么区别，主要哪些子 agent？

## 答案

Deep Research 不是单纯“搜索后总结”，而是围绕复杂问题反复规划、检索、阅读、验证、分析和写作，并保留来源。不同工作在单 Agent、多 Agent、训练方法和工具环境上有明显差异。

### 1. 几类代表工作

**OpenAI Deep Research**

- 使用面向真实任务训练的推理模型；
- 能搜索、阅读网页和用户文件，并使用 Python 分析；
- 通过强化学习学习多步浏览与分析；
- 更接近一个长时程研究 Agent，不要求所有步骤都拆成平行子 Agent；
- 最终报告包含来源，但仍需检查来源质量和结论可验证性。

**Anthropic Multi-Agent Research System**

- 使用 lead agent 作为 orchestrator；
- lead agent 根据问题创建多个 subagent 并行研究不同方向；
- subagent 各自在独立上下文中搜索、阅读并返回证据；
- lead agent 综合结果；
- citation agent 再处理引用；
- 特别适合信息宽度大、可以并行分解的研究问题。

**DeepResearcher**

- 将搜索和浏览视为真实 web environment 中的交互；
- 用端到端 RL 训练模型学习何时搜索、打开页面、继续追问和回答；
- 强调不能只依赖合成检索轨迹，要在真实搜索环境中学习；
- 代表“训练出研究策略”，而不只是手工编排 workflow。

**WebThinker / ReSearch 一类工作**

- 将推理与检索交错进行；
- 模型根据当前证据决定下一步 query，而不是一次召回后固定生成；
- 通过 RL 或可验证信号优化搜索策略和最终答案；
- 关注长时程 credit assignment、证据利用和复杂问答。

这些系统名称相近，但不能把产品能力、论文算法和多 Agent 架构混为一谈。

### 2. 主 Agent 的职责

主 Agent 对最终任务负责：

- 理解用户目标和交付格式；
- 制定研究计划；
- 判断哪些任务可并行；
- 分配时间、token、搜索次数和子 Agent 数；
- 合并结果、识别冲突和证据空白；
- 要求补充调查；
- 形成最终结构与结论；
- 对引用完整性和整体质量负责。

主 Agent 应只接收子任务摘要、关键证据和来源，不需要吞下每个子 Agent 的完整浏览轨迹。

### 3. 子 Agent 的职责

子 Agent 负责边界清晰的专项任务，例如：

- Search Agent：设计 query、网页检索；
- Internal Knowledge Agent：查企业内部文档；
- Document Agent：阅读长 PDF、表格和附件；
- Data/Code Agent：运行 Python、计算和画表；
- Domain Agent：按支付、合规、风控等领域调查；
- Verification Agent：交叉验证事实和时间；
- Citation Agent：检查来源是否支持对应陈述。

子 Agent 输出最好使用结构化 contract：

```text
subtask
findings
evidence
source_url
source_date
confidence
conflicts
open_questions
```

### 4. 什么时候多 Agent 有价值

适合：

- 任务可拆成互相独立的研究方向；
- 单上下文放不下大量资料；
- 需要不同权限、工具或专业模型；
- 并行搜索能显著缩短 wall-clock time；
- 需要独立验证减少单一路径偏差。

不适合：

- 子任务强依赖、必须顺序推进；
- 问题简单；
- 实时延迟要求高；
- 预算有限；
- 多 Agent 产生的重复搜索和冲突超过收益。

多 Agent 通常会显著增加 token 和工具成本，必须设置预算、最大深度、去重和早停。

### 5. 跨境支付研究场景

例如研究“某国家新监管变化对收款产品的影响”：

- 主 Agent 拆成监管原文、牌照要求、产品影响、运营流程、风险事项；
- 子 Agent 分别查监管机构、内部合规文档、产品配置和历史工单；
- Verification Agent 核对发布日期、生效日期、适用主体；
- 主 Agent 输出影响范围、证据、待确认项和行动建议。

外部搜索结果不能直接修改产品规则。最终结论需由合规或法务审核，内部敏感数据也不能发送给未授权的外部模型。
