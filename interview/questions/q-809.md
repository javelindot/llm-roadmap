# transformer

## 答案

Transformer 用 Attention 替代循环结构，通过全局信息交互和并行矩阵计算建模序列。

### 1. 输入

token embedding 与位置编码组合。原始模型采用正弦位置编码，现代 LLM 常用 RoPE。

### 2. Attention

\[
Q=XW_Q,\quad K=XW_K,\quad V=XW_V
\]

\[
\operatorname{Attention}(Q,K,V)
=\operatorname{softmax}
\left(\frac{QK^\top}{\sqrt{d_k}}+M\right)V
\]

多头 Attention 在不同子空间并行计算后拼接。Encoder 使用双向 mask，Decoder 使用 causal mask。

### 3. FFN、残差与归一化

Attention 负责 token 间通信，FFN 对每个位置做非线性变换。残差连接和 LayerNorm/RMSNorm 保证深层训练。现代 LLM 多使用 Pre-Norm 和 SwiGLU。

### 4. 三种架构

- Encoder-only：BERT，适合理解、分类和抽取；
- Decoder-only：GPT、LLaMA、Qwen，适合自回归生成；
- Encoder-Decoder：T5，适合条件生成和序列转换。

### 5. 复杂度与现代优化

一层主要复杂度为 \(O(nd^2+n^2d)\)，长序列受二次 Attention 限制。FlashAttention 优化 IO，GQA/MLA 减少 KV Cache，稀疏 Attention 和状态空间模型进一步探索长序列效率。

### 跨境支付场景

Transformer 可理解多语言支付工单和规则，但实时订单事实由工具提供；RAG 控制证据，workflow 控制资金动作。模型能力必须与权限、评测和审计结合。

### 面试总结

Transformer 的核心是缩放点积多头 Attention、FFN、残差、归一化和位置编码。优势是并行和全局依赖，主要成本是长序列二次计算与自回归 KV Cache。
