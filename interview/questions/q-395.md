# 设置数据权限，每个子 Agent 只能访问特定数据库

## 答案

> 数据权限隔离在金融 Agent 系统是**硬要求**——监管 / 合规 / 客户隐私都不容许 Agent 越界查数据。我们跨境支付落地了多层防护。
>
> **多层权限设计**：
>
> **1. 数据库层（最底层）**
>
> 每个 Agent 一个独立的数据库账号 + 只读视图：
>
> ```sql
> -- 客服 Agent 账号
> CREATE USER 'agent_cs'@'%' IDENTIFIED BY 'xxx';
> GRANT SELECT ON faq_db.* TO 'agent_cs'@'%';
> GRANT SELECT ON policy_db.public_policies TO 'agent_cs'@'%';
> -- 不给客户隐私表权限
>
> -- 风控 Agent 账号
> CREATE USER 'agent_risk'@'%' IDENTIFIED BY 'xxx';
> GRANT SELECT ON risk_db.* TO 'agent_risk'@'%';
> GRANT SELECT, INSERT ON audit_db.investigation_log TO 'agent_risk'@'%';
> -- 风控可以写审计日志，不能写业务库
>
> -- KYB Agent 账号
> CREATE USER 'agent_kyb'@'%' IDENTIFIED BY 'xxx';
> GRANT SELECT ON kyb_db.* TO 'agent_kyb'@'%';
> ```
>
> 优点：**数据库级硬约束**，应用层逻辑漏了底层也兜得住。
>
> **2. 中间件层（API Gateway / RBAC）**
>
> Agent 通过 API 网关访问数据：
>
> ```python
> @router.post("/api/query/{table}")
> async def query_table(
>     table: str,
>     agent_token: str = Header(...),
>     params: dict = Body(...),
> ):
>     agent = verify_agent_token(agent_token)
>
>     # 权限校验
>     if table not in agent.allowed_tables:
>         raise HTTPException(403, f"{agent.name} cannot access {table}")
>
>     # 字段级权限
>     allowed_fields = agent.get_allowed_fields(table)
>     query = build_query(table, allowed_fields, params)
>
>     # 行级权限（如商户 Agent 只能看自己商户）
>     if agent.tenant_id:
>         query += f" AND tenant_id = '{agent.tenant_id}'"
>
>     return await db.execute(query)
> ```
>
> **3. 向量库 / ES 层**
>
> Milvus / ES 也按 Agent 分隔：
>
> ```python
> # 客服 Agent 只能查客服 collection
> client = milvus_client(user="agent_cs")
> result = client.search(
>     collection="faq_kb",  # ✅ 客服库
>     query_vector=...,
> )
> # client.search(collection="risk_kb", ...)  # ❌ 拒绝
> ```
>
> ES 用 role-based access：
> ```yaml
> roles:
>   agent_cs:
>     indices:
>       - names: ["faq_*", "policy_public_*"]
>         privileges: ["read"]
> ```
>
> **4. Agent 内部工具白名单**
>
> 详见 q-391：
> ```python
> @register_agent("customer_service")
> class CustomerServiceAgent:
>     allowed_tools = ["query_order", "query_kyc_status", "query_rate"]
>     # 不能调 freeze_account（写操作）
>     # 不能调 query_other_customer（越租户）
> ```
>
> **5. 审计日志（兜底追溯）**
>
> 所有 Agent 调数据都入审计日志：
>
> ```python
> async def audit_data_access(agent_name, table, query, result_size):
>     await audit_db.insert({
>         "agent_name": agent_name,
>         "table": table,
>         "query": query,
>         "result_size": result_size,
>         "timestamp": now(),
>         "trace_id": current_trace()
>     })
> ```
>
> 用于：
> - 监管查询时提供证据
> - 异常访问告警
> - 季度审计复盘
>
> **典型示例**：
>
> | Agent | 可访问 | 不可访问 |
> |-------|--------|---------|
> | 客服 Agent | FAQ 库、公开政策、订单状态（脱敏）| 风控规则、商户合同、其他客户隐私 |
> | 风控 Agent | 风控规则、客户档案、交易历史、关联图 | 商户合同、内部财务 |
> | KYB Agent | 商户档案、KYB 证件 | 客户隐私、风控规则 |
> | 报表 Agent | 聚合统计、脱敏样本 | 任何原始 PII |
> | 客户自己（C 端 LLM）| 自己的订单、KYC 状态 | 他人数据、内部数据 |
>
> **跨 Agent 数据流转**：
>
> 子 Agent 把结果**摘要 + 脱敏**后返回主 Agent，不传原始数据：
>
> ```python
> # 风控子 Agent 返回主 Agent
> {
>     "risk_score": 0.85,       # ✅ 风险分
>     "category": "suspicious", # ✅ 分类
>     "evidence_count": 5,      # ✅ 证据数
>     "needs_human_review": True,
>     # 不返回客户身份证、卡号原文
> }
> ```
>
> **几个工程经验**：
>
> 1. **多层防护**：数据库 + 中间件 + Agent 内部 + 审计日志，任一层失守不致命
>
> 2. **最小权限原则**：Agent 默认无权限，按需开放
>
> 3. **定期审计**：每月审计权限配置 + 实际访问日志
>
> 4. **权限变更走流程**：不能随手改，要 review
>
> 5. **红蓝对抗**：每月模拟 Agent jailbreak 测试权限边界
>
> **不能做的事**：
>
> - 用 root / admin 账号跑 Agent（极危险）
> - 把所有 Agent 共享一个账号（无法追溯）
> - 只靠 prompt 约束（必被绕过）
> - 不做审计日志（出了事说不清）
>
> 一句话：**数据权限隔离 = 数据库账号 + 中间件 RBAC + 工具白名单 + 审计日志**——四层组合是金融 Agent 系统的最低要求。
