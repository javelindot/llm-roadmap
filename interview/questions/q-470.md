# 业务场景题

## 答案

**一句话**：业务场景题要先问清目标和约束，再拆流程、数据、模型、工具、评估和风险，不要一上来堆技术方案。

### 1. 回答框架

1. 明确业务目标：降本、提效、风控、体验还是合规。
2. 明确用户和流程：谁使用、在哪一步使用。
3. 明确数据来源：知识库、业务库、日志、工单。
4. 设计方案：RAG、工具调用、Agent、模型微调。
5. 风险控制：权限、脱敏、人工确认。
6. 评估指标：准确率、转人工率、成本、延迟、满意度。

### 2. 结合跨境支付项目怎么说（面试口径）

> 如果给我一个跨境支付业务场景，比如“降低付款失败问题的客服处理成本”，我会先梳理用户路径：用户发起咨询，系统识别意图，查订单状态、通道回执、风控结果，再检索对应 SOP，最后生成解释或转人工。
>
> 技术上用 RAG 管理规则和错误码，用 Function Calling 查实时交易，用 Agent 编排多步骤流程。高风险场景比如风控拦截、资金冻结必须转人工或审批。
>
> 指标上看问题解决率、转人工率、人工改写率、平均响应时间和投诉率。这样回答既有业务目标，也有技术落地和风险控制。

**钩子**：可以引到“先业务后技术、流程拆解、风险控制、指标闭环”。
