# 项目中PDF文件的上传流程是怎样的？如何处理PDF中的图片内容？

## 答案

**一句话**：PDF 上传流程一般包括文件接收、格式校验、存储、异步解析、文本抽取、图片/OCR 处理、切分入库和状态回写；图片内容不能简单丢掉，要按业务价值做 OCR 或多模态理解。

### 1. PDF 上传整体流程

一个比较完整的流程是：

1. 前端上传文件，后端校验文件类型、大小、页数、病毒扫描和权限。
2. 文件先落对象存储，数据库记录文件元数据和处理状态。
3. 通过 MQ 或任务队列触发异步解析，避免阻塞上传接口。
4. 解析 PDF 文本、标题、页码、表格、图片和版面结构。
5. 对文本做清洗、切分、embedding，写入向量库和元数据库。
6. 对解析失败或低置信度内容进入人工复核。
7. 最后回写处理状态，供用户查看是否入库成功。

### 2. PDF 中图片怎么处理

PDF 里的图片通常有几类：

- 扫描件：整页都是图片，需要 OCR。
- 截图：可能包含错误码、流程图、后台页面。
- 票据/证件：需要结构化抽取字段。
- 图表：需要表格识别或多模态模型理解。
- 装饰图片：没有业务价值，可以过滤。

处理方式一般是：先判断图片类型和清晰度，再决定是否 OCR、是否走多模态模型、是否只保存引用。对重要图片要保留页码、坐标和来源，方便答案引用和人工追溯。

### 3. 工程注意点

- PDF 解析要异步化，否则大文件会拖慢接口。
- OCR 结果要带置信度，低置信度不能直接进入知识库。
- 表格要尽量保留行列关系，不能切成碎片。
- 图片和文本要建立关联，比如“第 5 页流程图说明了退款审核路径”。
- 敏感文件要做权限控制和脱敏，尤其是身份证、银行卡、营业执照。

### 4. 结合跨境支付项目怎么说（面试口径）

> 在跨境支付项目里，PDF 可能是商户入驻材料、KYC/KYB 证明、银行回单、通道对账文件或合规制度文档。我会把上传和解析做成异步任务：上传接口只负责鉴权、校验和落存储，解析服务再去抽取文本、表格和图片。
>
> 图片处理要看场景。比如营业执照、护照、银行证明这种图片，不能只当附件保存，要做 OCR 和字段抽取，比如公司名、证件号、有效期、国家、银行账号。如果是通道报错截图，也要识别错误码和提示文案，后续客服问答才能检索到。
>
> 但我也不会把所有图片都无脑入库。OCR 置信度低的内容要标记出来，敏感字段要脱敏，重要材料要保留原图链接和页码，方便风控或合规人员复核。这样既能提高自动化能力，也能满足审计要求。
