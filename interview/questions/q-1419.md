# 有没有端侧部署经验，或者安卓部署用过没

## 答案

这是经验核验题。若有实践，应讲清模型格式、运行时、量化、Android 集成、性能数据和兼容性；若没有完整上线经验，应明确说做过原型或了解流程，不要编造设备和延迟。

### 1. 参考回答

> 我做过/了解 Android 端侧模型部署流程。通常先将 PyTorch、TensorFlow 或 ONNX 模型转换成端侧运行时支持的格式，再做算子兼容检查、量化和设备侧 benchmark，最后通过 Kotlin/Java 与 JNI/C++ 接入应用。运行时可选 LiteRT、ExecuTorch、ONNX Runtime Mobile、MNN 或 NCNN，具体看模型来源、硬件加速和包体要求。

### 2. 完整流程

```text
训练模型
-> export
-> graph optimization
-> quantization
-> operator compatibility
-> package model
-> Android runtime integration
-> CPU/GPU/NPU benchmark
-> accuracy regression
-> 灰度和监控
```

### 3. 端侧优化

- FP16、INT8、INT4 量化；
- PTQ 或 QAT；
- pruning、distillation；
- operator fusion；
- dynamic shape 控制；
- mmap/分片加载；
- 减少峰值内存和模型包体；
- 使用 NNAPI、GPU delegate、厂商 NPU；
- 多线程和大小核调度；
- 首次加载与 warm-up 优化。

量化后必须重新评估精度，不能只报告模型大小。

### 4. Android 集成关注点

- AAR/Native library 大小；
- ABI：arm64-v8a 等；
- Kotlin/Java 与 JNI 生命周期；
- Tensor 内存复用；
- 摄像头和图像预处理；
- 前后台切换；
- 电量、温度和降频；
- 不同 SoC/Android 版本兼容；
- 模型下载、签名、版本和回滚。

### 5. 性能指标

- cold start/warm start；
- P50/P95 推理延迟；
- 峰值 RAM；
- 模型大小；
- CPU/GPU/NPU 利用率；
- 功耗和温升；
- 准确率回归；
- 崩溃率和设备覆盖率。

### 6. 跨境支付场景

端侧可做证件图质量检测、OCR 前处理、活体检测、敏感信息遮挡和轻量风险特征提取。最终 KYC、制裁筛查和资金决策不应只依赖客户端模型，因为客户端可能被篡改，权威校验仍应在服务端完成。

### 7. 面试总结

端侧部署不是只把模型转成 ONNX，而是格式转换、算子适配、量化、硬件 delegate、Android 生命周期、功耗和精度回归的完整工程。没有生产经验时，诚实说明原型边界并讲清可执行方案即可。
