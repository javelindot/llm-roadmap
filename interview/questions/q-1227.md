# 文案生成为什么不总BLEU指标

## 答案

文案生成不应总用 BLEU，因为 BLEU 主要衡量候选文本与参考文本的 n-gram 重合度，而开放式文案通常存在大量“措辞不同但同样正确”的答案。BLEU 适合标准答案较稳定的机器翻译语料级评测，不适合单独衡量事实性、说服力、品牌风格和合规性。

### 1. BLEU 实际衡量什么

BLEU 对 1 到 N-gram 的 clipped precision 做几何平均，并增加 brevity penalty，避免候选句子过短：

```text
BLEU = BP * exp(Σ_n w_n * log p_n)
```

它回答的是“候选文本与参考文本在表面词序上有多像”，不是“文案是否完成业务目标”。

### 2. 同义改写会被低估

参考文案：

> 付款已提交银行处理，预计时间以收款行入账为准。

候选文案：

> 银行正在处理该笔付款，最终到账时间取决于收款银行。

二者语义接近，但 n-gram 重合可能不高。开放生成只有一条参考答案时，BLEU 对同义词、语序变化和多语言表达尤其敏感。

### 3. 高 BLEU 不代表高质量

候选文本复制参考句的大部分词，就可能获得较高分，但仍可能：

- 把币种、金额或日期写错；
- 漏掉限制条件和下一步；
- 违反品牌语气；
- 包含敏感信息；
- 对到账时间做不当承诺；
- 与当前交易状态冲突。

这些都是跨境支付文案的关键质量维度，BLEU 无法可靠识别。

### 4. 文案生成应分层评测

**确定性校验**

- 金额、币种、国家、状态和日期与工具结果一致；
- 必填字段、链接和免责声明存在；
- 不出现完整账号、证件号和禁用承诺；
- JSON、模板或字符长度符合渠道要求。

**语义与证据**

- BERTScore 或 embedding similarity 作为语义近似指标；
- NLI、claim-evidence 检查和引用验证评估事实一致性；
- 无证据时是否澄清、拒答或转人工。

**文案质量**

- 清晰度、完整性、简洁性；
- 品牌语气、语言自然度和本地化；
- CTA 是否明确；
- 是否适合邮件、站内信、Push 等具体渠道。

**人工与 LLM Judge**

- 业务专家盲评高风险样本；
- LLM Judge 使用固定 rubric、隐藏候选顺序和多次评审；
- 先用人工黄金集校准 judge 的一致性、位置偏差和长度偏差；
- 上线前采用 pairwise preference，通常比绝对打分稳定。

### 5. 不同任务使用不同指标

| 任务 | 更合适的组合 |
|---|---|
| 机器翻译 | SacreBLEU + chrF/COMET + 人工 |
| 摘要 | ROUGE + 事实一致性 + 人工 |
| 营销文案 | rubric judge + 人工偏好 + 转化 guardrail |
| 支付状态通知 | 字段规则 + faithfulness + 合规审核 |
| 客服改写 | 语义保持 + 风格 + 敏感信息检查 |

线上点击率或转化率也不能成为唯一指标，因为它们受渠道、用户、活动和发送时间影响，还可能鼓励夸张表达。应同时设置投诉率、误导率、退订率和合规违规率等 guardrail。

BLEU 可以保留为一个低成本回归信号，但文案生成的最终验收必须覆盖“事实正确、目标完成、表达质量和安全合规”。
