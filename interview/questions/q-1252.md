# 岗位主要做提升文心一言的问答和检索增强任务

## 答案

这个岗位的核心不是只调整生成模型，而是提升从用户问题到可信答案的完整链路：

> Query 理解 → 检索与排序 → 证据组织 → 模型生成 → 事实校验 → 线上反馈。

### 1. Query 理解

- 多轮指代消解；
- 意图分类和实体抽取；
- Query rewrite、扩展和问题分解；
- 判断查网页、知识库、结构化数据还是工具；
- 识别无答案、时效和权限要求。

这一步错误会导致后续检索方向完全偏离。

### 2. 召回与排序

建立 BM25、Dense、FAQ 和结构化检索多路召回，使用 RRF 或校准融合，再通过 Cross-Encoder reranker 精排。重点优化：

- Embedding 领域适配；
- hard negative 构造；
- 长尾实体和精确关键词；
- 多语言与口语问法；
- 文档权威性、时效性和去重；
- 用户、租户和内容权限。

### 3. 证据与知识治理

问答质量的上限受知识质量影响。需要建立：

- 文档解析、结构切块和父子块；
- 来源、作者、版本和生效时间；
- 内容冲突、过期和重复检测；
- 增量更新和删除；
- 引用粒度与证据覆盖；
- 训练集和评测集污染检查。

### 4. 生成模型优化

先做 Prompt/RAG baseline，再根据失败模式选择：

- SFT：指令格式、工具调用、领域表达；
- DPO：回答偏好、简洁性、拒答和安全；
- verifier：事实、引用、格式和数值；
- 模型路由：按难度选择不同模型；
- 长上下文：只在检索不足或全局阅读任务中使用。

模型应区分检索事实、推断和未知信息，无法确认时给出限制而不是补全。

### 5. 评测体系

黄金集覆盖事实问答、多跳、长文档、多轮、时效、无答案、安全和对抗样本。指标拆为：

- Query/路由准确率；
- Recall@K、MRR、NDCG；
- Context Precision/Recall；
- answer correctness、faithfulness、citation accuracy；
- 拒答和安全；
- P95/P99、成本和线上任务成功率。

必须做错误归因：是知识缺失、检索漏召回、排序错误、上下文噪声，还是模型不遵循证据。

### 6. 线上闭环

- 完整记录 Query、召回、排序、证据、模型和引用 trace；
- 收集人工改写、用户追问和投诉；
- 对 bad case 脱敏、去重并人工标注；
- 影子评测和小流量灰度；
- 知识、Embedding、Reranker、LLM 分别版本化和回滚。

### 7. 可迁移的跨境支付经验

跨境支付同样是知识密集型问答：产品规则和 KYC/KYB 走检索，订单、余额和汇率走工具，模型负责理解和解释。最重要的经验是把静态知识、实时事实、生成能力和权限边界拆开，并用分层指标定位问题。这套方法可以迁移到通用问答和搜索增强，但不能把支付数据或内部规则包装成文心一言项目经历。
