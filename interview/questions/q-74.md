# 什么是监督微调？它与预训练有什么区别？

## 答案

**一句话**：预训练是让模型从海量无标注数据里学通用语言能力，监督微调是用人工构造的指令数据教模型按任务和格式回答。

### 核心回答
- **预训练**：
  - 数据规模大，通常是网页、书籍、代码等。
  - 目标多是 next token prediction。
  - 学到通用知识、语言模式和基础推理能力。
- **SFT**：
  - 数据规模相对小，但质量要求高。
  - 格式是 instruction-response。
  - 教模型遵循指令、按业务格式输出、减少胡乱生成。

区别：预训练解决"会不会语言和知识"，SFT 解决"会不会按人类要求做任务"。

### 1. 预训练学什么，SFT 学什么

预训练阶段主要通过 next token prediction 学通用语言规律、世界知识、代码模式和基础推理能力。这个阶段数据规模巨大，但不一定教模型"怎么听人话办事"。

SFT 阶段用人工整理的指令数据，让模型学会：

- 理解用户指令。
- 按格式回答。
- 遵守业务流程。
- 给出更符合人类预期的表达。

### 2. SFT 的关键不是数据越多越好

SFT 更看重质量：

- 指令是否真实。
- 答案是否准确。
- 格式是否统一。
- 是否覆盖核心业务场景。
- 是否包含拒答、转人工、异常处理样本。

低质量 SFT 数据会把模型训得更差。

### 3. 面试总结

预训练给模型"底层能力"，SFT 给模型"任务习惯"。业务落地通常不是从零预训练，而是在基座模型上做 SFT/LoRA，再结合 RAG 和工具。

### 4. 结合跨境支付项目怎么说（面试口径）

> 在跨境支付里，我们一般不会从零预训练模型，成本太高。更现实的是拿已有基座模型做 SFT 或 LoRA，比如用风控审核记录、客服问答、拒付处理 SOP、商户准入材料来教模型怎么回答业务问题。
>
> 预训练给它基础语言和推理能力，SFT 让它学会支付领域的术语、流程和输出格式。比如遇到高风险商户，不是简单说"拒绝"，而是要说明命中的规则、缺少的材料和是否需要人工复核。
>
> 但规则经常变，所以我不会指望 SFT 记住所有最新规则。上线前还要结合 RAG，把通道文档、合规条款、国家限制实时接进来，这样知识更新不需要每次重训模型。

**钩子**：业务场景更常见是 SFT + RAG，而不是从零预训练。
