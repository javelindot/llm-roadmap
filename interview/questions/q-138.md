# One-hot 编码的原理？有什么缺点？

## 答案

**一句话**：One-hot 是用一个长度为词表大小的向量表示词，当前位置为 1，其余为 0。

### 核心回答
例如词表有 5 个词，"支付"可能表示为：

```text
[0, 1, 0, 0, 0]
```

优点：
- 简单直观。
- 不需要训练。
- 适合类别特征表示。

缺点：
- 维度很高，稀疏。
- 无法表示词语相似性。
- 不包含语义信息。
- 遇到未登录词难处理。

后续的 word embedding、BERT、LLM 都是在解决语义表示问题。

### 1. One-hot 的本质

One-hot 把离散类别映射成互相正交的向量。它只表达"是不是这个类别"，不表达类别之间的距离和语义关系。

比如 `USD`、`EUR`、`BRL` 都是不同维度上的 1，模型不会天然知道它们都是币种。

### 2. 缺点为什么明显

- 词表大时维度很高。
- 向量稀疏，计算和存储浪费。
- 任意两个词距离都一样，无法表示相似性。
- 新词或未登录词难处理。

所以文本语义任务会使用 embedding，让相似词在向量空间里更接近。

### 3. 面试总结

One-hot 适合简单离散类别，不适合语义表示。它是很多 embedding 方法的对照起点。

### 4. 结合跨境支付项目怎么说（面试口径）

> 在支付风控里，one-hot 适合一些离散类别特征，比如国家、币种、通道、商户行业。比如国家是 BR、US、CN，币种是 USD、BRL、EUR，这些可以作为类别特征。
>
> 但如果是文本语义，比如拒付原因、商户描述、客服对话，one-hot 就太弱了，因为它不知道"付款失败"和"入账未到账"语义接近，也不知道"chargeback"和"拒付"可能指同一类业务。
>
> 所以结构化类别可以 one-hot，文本理解更适合 embedding、BERT 或大模型。

**钩子**：one-hot 适合离散类别，不适合表达业务语义相似性。
