# Llama与原始Transformer架构的主要区别是什么？

## 答案

经典 Llama 1-3 与 2017 原始 Transformer 的主要区别是：从 Encoder-Decoder 改为 Decoder-only，并采用 Pre-RMSNorm、RoPE、SwiGLU、无/少 bias 和后续 GQA 等现代化设计。回答时要说明 Llama 4 又加入 MoE 与原生多模态，不能把整个家族永久概括成 Dense 文本模型。

### 1. 整体结构

**原始 Transformer**

```text
Encoder
+ Decoder
+ Cross-Attention
```

主要为机器翻译设计。

**经典 Llama**

```text
Token Embedding
-> N x Causal Decoder Block
-> RMSNorm
-> LM Head
```

用于 next-token prediction，没有独立 Encoder 和 Cross-Attention。

### 2. Normalization

原始 Transformer 常写作 Post-LayerNorm：

```text
LN(x + Sublayer(x))
```

Llama 使用 Pre-RMSNorm：

```text
x + Sublayer(RMSNorm(x))
```

Pre-Norm 更利于深层梯度传播；RMSNorm 不减均值，计算更简洁。

### 3. 位置编码

- 原始 Transformer：正弦绝对位置加到输入；
- Llama：RoPE 旋转 Q/K，在注意力内注入相对位置信息。

### 4. FFN

原始：

```text
ReLU(W1 x) W2
```

Llama：

```text
W_down(
  SiLU(W_gate x) * W_up x
)
```

即 SwiGLU 门控 FFN。

### 5. Attention 与参数细节

- Llama 使用 causal self-attention；
- 后续型号广泛采用 GQA，减少 KV Cache；
- 线性层通常不使用 bias；
- tokenizer、词表和上下文长度随代际变化；
- 推理使用 KV Cache，但这属于运行方式而非基础训练结构。

### 6. Llama 4

Llama 4 Scout/Maverick 加入：

- 原生多模态 early fusion；
- Mixture-of-Experts；
- 总参数与激活参数分离；
- 更复杂的专家路由和部署要求。

### 7. 跨境支付

文本客服和工具 Agent 可选经典 Instruct；证件、回单和截图需要视觉模型；生产选型还要看量化、许可证、数据驻留、工具准确率和审计能力。
