# 测试集如何选择？如何判断测试集和应用场景是一致的（无偏的）？模型的离在线效果差异如何解决？

## 答案

测试集的目标不是复现训练数据，而是估计模型在目标用户、目标流量和目标决策策略下的真实表现。需要同时保证代表性、独立性、覆盖度和时间有效性。

### 1. 先定义目标总体

明确模型将服务：

- 哪些用户、商户和地区；
- 哪些语言、产品和渠道；
- 哪些设备、入口和流量时段；
- 哪些任务与风险等级；
- 当前线上模型和业务策略是什么。

如果目标总体不明确，“无偏测试集”没有定义。测试集只能相对于某个目标分布评估是否具有代表性。

### 2. 从真实流量分层抽样

跨境支付场景可以按以下维度分层：

- 国家、币种、语言和时区；
- 全球收款、付款、换汇、收单、退款和拒付；
- 新老商户、规模和行业；
- 高频、长尾、复杂多轮和无答案问题；
- 正常、异常、投诉和高风险任务；
- 不同通道、错误码和工具状态。

主体样本按线上自然分布抽样，用于估计总体表现；长尾和高风险样本额外过采样，用于发现风险。报告指标时对过采样集按线上权重还原，不能直接把挑战集比例当成真实流量。

### 3. 避免数据泄漏和隐性过拟合

- 同一会话、商户、订单或模板变体不跨训练与测试；
- 按时间切分，测试集来自训练时间之后；
- 近似去重问题、答案和文档片段；
- 冻结最终测试集，调参只使用验证集；
- 检查预训练或合成数据是否包含测试答案；
- RAG 评测区分“知识已存在但没召回”和“知识在测试时尚未生效”。

频繁查看测试集并按结果修改系统，同样会造成测试集过拟合。

### 4. 判断与线上场景是否一致

对比测试集与线上流量的：

- 类别、国家、语言和长度分布；
- 意图、实体、工具调用和多轮深度；
- embedding 或主题聚类分布；
- 标签基准率和难度；
- 缺失字段、异常状态和错误码；
- 模型置信度与输出长度。

可使用 PSI、Jensen-Shannon divergence、KS 检验或分类器两样本检验作为分布差异信号，但统计接近不等于业务覆盖充分，还要由业务专家检查关键场景。

训练一个“来源分类器”区分测试样本与线上样本：若很容易区分，说明存在明显 covariate shift，需要重新采样或分桶报告。

### 5. 建立多套测试集

- **自然流量集**：估计整体线上效果；
- **分桶回归集**：稳定比较版本；
- **长尾挑战集**：复杂、多语言和少样本场景；
- **安全对抗集**：越权、注入、隐私和错误资金动作；
- **新鲜度集**：规则变更、新通道和新错误码；
- **事故回放集**：历史严重 bad case。

不能把所有样本混成一个平均分。高风险分桶即使数量少，也应设置独立上线门槛。

### 6. 离线与在线效果差异的来源

- 线上流量分布变化；
- 离线标签与真实业务目标不一致；
- 训练样本来自已曝光或已处理数据，存在选择偏差；
- Prompt、知识库、工具和模型版本不一致；
- 线上延迟、超时和降级未在离线模拟；
- 多轮上下文、缓存和并发行为不同；
- 用户会根据模型回复改变后续行为，离线数据无法完全复现；
- 退款、拒付和投诉等标签延迟成熟。

### 7. 缩小离在线差距

1. 离线回放使用与生产相同的 Prompt、检索、工具 mock 和版本；
2. 对超时、工具失败、知识冲突和降级做故障注入；
3. 使用 shadow traffic 在不影响用户的情况下运行新版本；
4. 小流量随机 A/B，按用户或商户分组防止污染；
5. 设置质量、安全、延迟、成本和投诉 guardrail；
6. 回收成熟标签，分析分桶差异而不是只看总分；
7. 对 covariate shift 做重采样、重要性加权或更新测试集；
8. 线上 bad case 进入回归集，但保留一份从未用于调参的审计集。

### 8. 评测结论

报告应包含样本来源、时间范围、分桶比例、置信区间和已知盲区。测试集不可能永久无偏，业务、用户和模型都会变化，因此需要持续监控数据漂移，并按固定治理流程更新版本。

最终判断模型是否可上线，依赖“离线通过 + 影子流量稳定 + 在线随机实验满足 guardrail”，而不是一个静态测试集分数。
