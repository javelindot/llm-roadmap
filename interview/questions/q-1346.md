# 询问大模型的基本概念和原理

## 答案

大模型通常指参数规模大、在海量数据上预训练、可通过 Prompt 或少量微调迁移到多任务的神经网络。当前通用文本模型多是 Decoder-only Transformer，本质目标仍是根据上下文预测下一个 token。

### 1. 输入与 Token

文本先经 tokenizer 转为 token id：

```text
text -> tokens -> embeddings
```

模型处理的是 token，不直接理解字符串。词表、切词效率会影响多语言、数字、代码和上下文成本。

### 2. Transformer

每层主要包含：

```text
Normalization
-> Self-Attention
-> Residual
-> FFN
-> Residual
```

Attention：

```text
Q = X W_Q
K = X W_K
V = X W_V

Attention
= softmax(QK^T / sqrt(d_k) + Mask) V
```

Decoder 使用 causal mask，只能看当前位置之前的 token。

### 3. 训练目标

自回归预训练：

```text
L
= -sum_t log p(x_t | x_<t)
```

通过大规模数据学习语言、知识、代码和模式。模型是在拟合条件分布，不是数据库，也不保证每个输出为事实。

### 4. 训练阶段

- Pre-training：通用 next-token 学习；
- Continued Pretraining：领域、语言或长上下文适配；
- SFT：学习指令、对话和工具格式；
- Preference/RL：DPO、PPO、GRPO 等优化偏好或可验证目标；
- Safety Tuning：拒答、红队和政策边界。

### 5. 推理

- Prefill：并行处理 Prompt，构建 KV Cache；
- Decode：逐 token 生成并复用历史 KV；
- temperature、top-p 等控制采样；
- Function Calling 让模型输出结构化工具请求，真正执行由外部程序完成。

### 6. 能力来源

能力不只由参数量决定，还包括：

- 数据质量和规模；
- 架构与 tokenizer；
- 训练算力和配方；
- Post-training；
- 检索、工具和 inference-time compute；
- 评测与工程约束。

### 7. 局限

- 幻觉和知识时效；
- Prompt Injection；
- 长上下文遗忘；
- 数字和精确状态不稳定；
- 成本、延迟和显存；
- 偏见、隐私和不可解释性。

### 8. 跨境支付

大模型适合理解多语言问题、检索规则、编排只读工具和生成解释。账务余额、汇率、制裁名单、交易状态和资金操作必须由确定性系统、权限及人工审批控制。

**面试总结**：大模型是概率生成与通用任务接口，不是事实源或业务决策引擎。
