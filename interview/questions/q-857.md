# 训练一个智能npc你觉得需要多少数据量，模型尺寸，训练的策略该怎么选

## 答案

数据量和模型尺寸不能脱离角色数量、世界观复杂度、并发和硬件直接给答案。应先用最小方案验证，再根据评测曲线扩大。

### 1. 初始模型选型

- 只做有限角色对话：3B～7B 指令模型加 RAG/状态机可作为基线；
- 复杂中文角色、多步任务与工具：可比较 7B～14B；
- 更大模型可作为 teacher 或云端困难请求后备，不一定直接承载全部 NPC；
- 端侧或高并发场景需要蒸馏、量化和小模型路由。

模型尺寸由角色一致性、任务成功、首 token 延迟、每 token 延迟、并发和单位成本共同决定。

### 2. 数据量起点

可分阶段准备，而不是追求一个数字：

- 数千条人工黄金对话建立 prompt/RAG 基线；
- 约 1 万～10 万条高质量、多角色 SFT 轨迹作为首轮实验范围；
- 数千到数万偏好对优化角色一致性、安全和趣味性；
- 大量世界观文本主要进入 RAG，稳定语言风格才考虑持续预训练；
- 玩家日志只能经同意、脱敏、过滤和人工抽检后回流。

这些是实验起点，不是通用最优值。应画 data scaling curve，观察新增数据的边际收益。

### 3. 训练策略

1. 基座 + 角色 prompt + RAG + 状态机；
2. LoRA SFT 学语气、状态格式和工具调用；
3. DPO 学偏好；
4. 可验证任务在沙箱中做 RL；
5. 蒸馏至服务模型；
6. 在线 bad case 审核后回灌。

不要用参数记忆实时世界状态；任务、背包、关系和经济数据由游戏引擎管理。

### 面试总结

智能 NPC 应先以 3B～7B 加 RAG/状态机验证，用万级高质量轨迹启动 SFT，再依据质量、延迟和成本曲线扩展数据与模型。世界事实和动作权限始终由游戏系统控制。
