# 简述监督学习和无监督学习的区别

## 答案

**一句话**：**监督学习有标签教模型该输出啥；无监督学习只有数据，模型自己挖结构**——LLM 时代有第三类**自监督学习**夹在中间，是预训练的本质。

### 1. 监督学习
- **输入**：(x, y) 配对样本
- **目标**：学一个 f(x) → y
- **任务**：分类、回归
- **算法**：SVM、决策树、神经网络
- **要求**：标签准确、覆盖全面

### 2. 无监督学习
- **输入**：只有 x，没有 y
- **目标**：发现数据内在结构
- **任务**：
  - 聚类（K-Means、DBSCAN、层次聚类）
  - 降维（PCA、t-SNE、UMAP）
  - 密度估计（GMM）
  - 关联规则（Apriori）

### 3. 自监督学习（LLM 的灵魂）
- 介于监督和无监督之间
- **从数据本身构造监督信号**
- 例：next-token prediction（用前文当输入，下一 token 当标签）
- BERT 的 MLM（mask 一些 token 再预测）
- 对比学习（同图增强 vs 不同图）

### 4. 三者对比
| 维度 | 监督 | 无监督 | 自监督 |
|------|------|--------|--------|
| 标签来源 | 人工 | 无 | 数据自身 |
| 数据规模 | 受标注约束 | 任意 | 任意 |
| 用途 | 直接做任务 | 探索 | 预训练 |
| 评估 | 准/召 | 内在指标 | 下游任务 |

### 5. 半监督和强化
- **半监督**：少量标签 + 大量无标签
- **强化学习**：从奖励信号学，介于监督和无监督之间
- **弱监督**：标签不准/不全（远程监督、众包）

### 6. LLM 时代的范式
- **预训练**：自监督（next-token）+ 海量纯文本
- **SFT**：监督（人写指令-答案对）
- **RLHF**：强化学习（用 RM 当奖励）

→ LLM 几乎用了所有学习范式

### 7. 实战经验
- 业务有充足标签 → 监督
- 业务标签少 + 大量无标注 → 自监督预训练 + 监督 finetune
- 业务无标签 + 想找结构 → 无监督聚类
- 业务要"对齐主观偏好" → 强化学习 / DPO

### 8. 一句话总结
**监督 = 有教师，无监督 = 自学，自监督 = 自己出题自己答**——LLM 时代靠"自监督预训练 + 监督微调 + RL 对齐"三段式打天下。

## 结合项目谈（口语版 · 约 25 秒）

> 这道题工业里我会用我们跨境支付的**三层架构**讲——监督、无监督、自监督在我们项目里都活着。
>
> **监督学习**：风控可疑交易分类、客户违约预测、客服意图分类，全是经典监督任务。XGBoost / 神经网络 + 业务标注数据。
>
> **无监督学习**：客户分群、异常交易聚类。我们用 K-Means + DBSCAN 对千万级客户做行为聚类，找出"小批量高频跨境"这种异常模式，**不需要标签**，模型自己发现。
>
> **自监督学习**：客服 LLM 的预训练用的就是 next-token prediction，对齐用的 DPO 本质是从偏好对构造监督信号。这是 LLM 时代的范式。
>
> 一个有趣案例：我们想做**长尾欺诈检测**——欺诈样本只有几百条，怎么训？做法是先用无监督聚类找异常簇（candidate），让人工标注确认是不是欺诈，再用监督学习训分类器。这是典型的"无监督 + 半监督"组合，**用无监督扩展监督的样本边界**。
>
> 面试这道题我会强调一句：**三种学习范式不是替代关系是组合关系**，业务场景里基本都是混着用。
