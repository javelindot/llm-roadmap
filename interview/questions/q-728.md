# 常用的对比学习有哪些方法？怎么构造数据的？

## 答案

对比学习通过拉近正样本表示、推远负样本表示学习 embedding。方法差异主要在正负样本构造、是否需要大批负样本和监督信号。

### 常见方法

- SimCLR：同一样本的两种增强为正对，batch 内其他样本为负对，使用 NT-Xent/InfoNCE。
- MoCo：使用 momentum encoder 和队列维护大量负样本。
- SupCon：同类别样本都作为正样本，利用标签改善类内聚合。
- CLIP：图像和文本双塔，匹配图文为正对，batch 内其他组合为负对。
- Triplet Loss：anchor、positive、negative，优化正负距离 margin。
- BYOL/SimSiam：不显式使用负样本，通过双分支、stop-gradient 和 predictor 避免坍缩。

### 数据构造

图像正对可用裁剪、颜色、模糊等语义保持增强；文本可用同义改写、回译或同一语义问答，但增强不能改变标签。多模态正对来自真实配对图文/音频。

负样本包括 batch 内随机负例、队列负例和 hard negatives。随机负例太简单，hard negative 过难或实际同义会形成 false negative。可按元数据、检索结果和人工标注构造，并对同类样本做 false-negative masking。

### 关键实践

embedding 通常 L2 归一化，相似度使用 cosine，并用 temperature 调节分布。大 batch 提供更多负例，但也增加 false negative。评估除训练 loss 外，还要看 Recall@K、线性探针、聚类和下游任务。

数据增强定义了模型应保持哪些不变性，因此往往比单纯更换 loss 更重要。
