# 大模型中常用的位置编码方案有哪些？请分别说明其优缺点及适用场景。

## 答案

Attention 本身对顺序不敏感，位置编码用于表示 token 的绝对位置、相对距离或顺序关系。

### 1. 固定正弦绝对位置

原始 Transformer 使用：

\[
PE_{pos,2i}=\sin(pos/10000^{2i/d}),\quad
PE_{pos,2i+1}=\cos(pos/10000^{2i/d})
\]

优点是无额外可训练参数、理论上可计算任意位置；缺点是训练外长度仍会分布失配，且位置通过加法间接进入 Attention。适合基础 Transformer 和固定中短序列。

### 2. 可学习绝对位置

BERT、GPT-2 等为每个位置学习 embedding。优点是训练长度内表达灵活；缺点是超过最大位置没有已训练参数，外推差，扩长常需插值和继续训练。

### 3. 相对位置编码

Transformer-XL、T5 relative bias、DeBERTa 等在 Attention score/value 中表示相对距离。优点是直接建模 token 间距离，模式可跨绝对位置复用；缺点是实现和 KV Cache 处理更复杂，bucket 化会损失精细距离。适合长文本、Encoder 和序列任务。

### 4. RoPE

对 Q/K 进行位置相关旋转，使点积依赖相对距离。优点是不增加序列 embedding、兼容自回归和 KV Cache，且可用 Position Interpolation、NTK-aware scaling、YaRN/LongRoPE 扩展。缺点是超训练长度仍有相位外推问题，高频维度可能混叠。现代 Decoder LLM 最常用。

### 5. ALiBi

按不同 head 给 Attention score 加随距离线性衰减的 bias。结构简单、无需位置表、长度外推相对方便；但位置表达形式较固定，在某些复杂任务上不如 RoPE 灵活。

### 6. NoPE 与混合方案

部分研究在某些层不显式加入位置，或混合 RoPE、局部 Attention、绝对/相对 bias，以改善长度外推。效果依赖训练配方，尚不是通用替代。

长文本能力还受训练长度、Attention 复杂度和数据影响，不能只改位置编码就宣称支持长上下文。

### 面试总结

绝对位置简单但外推弱，相对位置直接表达距离，RoPE 兼顾相对关系与 KV Cache，ALiBi 用线性 bias 简化外推。现代 LLM 多用 RoPE，但扩长仍需缩放、长序列训练和真实评测。
