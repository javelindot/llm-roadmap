# "涌现能力"是大型模型中一个备受关注的现象,请问你如何理解这个概念?它通常在模型规模达到什么程度时出现?

## 答案

涌现能力通常指某项能力在小模型上接近随机或不可见，模型规模、数据或计算达到某个区域后，测得性能突然明显提升。但“突然出现”不一定代表模型内部发生离散相变。

### 1. 为什么看起来会涌现

- 任务需要多个子能力同时超过门槛；
- exact match、pass/fail 等离散指标把平滑变化变成跳变；
- 小模型性能低于可观测阈值；
- Prompt、few-shot、解码和评测噪声随规模变化；
- 数据覆盖或训练配方在某一代同时改变；
- 使用更多 test-time compute 后跨过任务成功阈值。

例如模型每个推理步骤都平滑提高，但一道多步题只有全部步骤正确才记 1 分，最终准确率可能呈现突跃。

### 2. 争议

早期研究在 BIG-Bench 等任务中观察到随规模突然提升。后续研究指出，改用连续、部分信用或对数指标时，一些“涌现”会变成平滑可预测曲线，因此部分现象可能是评测度量造成的 mirage。

这不等于所有涌现都不存在。复杂系统可能确实出现新的可组合行为，但需要排除指标、数据和 Prompt 的混杂因素。

### 3. 在什么规模出现

没有统一参数阈值，也不能回答“达到 100B 就会涌现”。阈值取决于：

- 任务难度和评价指标；
- 参数、训练 token 与数据质量；
- 架构和 tokenizer；
- 后训练、Prompt 和工具；
- 推理预算与采样策略。

同一能力在不同模型家族中可能在完全不同的规模出现。

### 4. 如何严谨验证

- 训练多个中间尺寸，而不是只比较两端；
- 同时报告连续和离散指标；
- 控制数据、架构和训练预算；
- 多 Prompt、多随机种子与置信区间；
- 对任务分步骤给 partial credit；
- 检查数据污染和记忆；
- 使用 held-out 组合和 OOD 测试。

### 5. 对研发的意义

不能仅凭小模型某项能力为零就断言扩大无效，也不能押注“再扩十倍自然出现”。应通过 scaling curve、能力分解和小规模代理实验评估，并为突然能力或风险准备评测。

### 6. 跨境支付场景

模型规模增大可能突然提高多步调查成功率，也可能突然具备绕过软约束的能力。权限和资金边界不能依赖“当前小模型做不到”，必须由后端硬控制。

### 7. 面试总结

涌现描述的是观测性能的非线性变化，没有统一规模阈值。部分涌现来自离散指标，部分可能来自真实能力组合；严谨判断需要密集 scaling 实验、连续指标和污染控制。
