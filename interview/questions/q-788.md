# 说说策略模式？

## 答案

策略模式将一组可互换算法封装为统一接口，由上下文在运行时选择具体实现，避免大量 `if/else`，属于行为型设计模式。

```java
interface RouteStrategy {
    RouteResult route(Payment payment);
}

final class CostFirstStrategy implements RouteStrategy {
    public RouteResult route(Payment payment) {
        return chooseLowestCostChannel(payment);
    }
}

final class SuccessRateFirstStrategy implements RouteStrategy {
    public RouteResult route(Payment payment) {
        return chooseHighestSuccessRateChannel(payment);
    }
}

final class PaymentRouter {
    private final Map<String, RouteStrategy> strategies;

    PaymentRouter(Map<String, RouteStrategy> strategies) {
        this.strategies = Map.copyOf(strategies);
    }

    RouteResult route(String name, Payment payment) {
        RouteStrategy strategy = strategies.get(name);
        if (strategy == null) {
            throw new IllegalArgumentException("Unknown strategy: " + name);
        }
        return strategy.route(payment);
    }
}
```

### 优点

- 算法与调用方解耦；
- 新增策略符合开闭原则；
- 每个策略可独立测试和配置；
- 可通过工厂、依赖注入或配置动态选择。

### 局限

- 策略类数量增加；
- 客户端或选择器仍需理解适用条件；
- 若策略只差一两行，使用函数对象可能比建类更简单；
- 共享大量状态时要避免策略之间隐式耦合。

跨境支付路由可按成本、成功率、国家和合规要求选择策略，但真正上线时需要先做确定性资格过滤，再在合法通道中应用策略，不能让策略模式绕过规则。

### 面试总结

策略模式是“统一接口封装可替换算法，运行时选择实现”。适合支付路由、计价和重试规则；价值在解耦与可测试性，不应为简单分支过度设计。
