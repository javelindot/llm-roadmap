# 深挖多模态大模型论文用的video - llama ，讲一下大模型的结构

## 答案

Video-LLaMA 是 2023 年提出的音视频指令微调框架，核心思想是冻结预训练视觉/音频编码器和 LLM，用两个 Q-Former 分支把视频与音频压缩、对齐到语言模型的 embedding 空间。

### 视觉分支

视频采样为多帧，每帧经过预训练图像编码器得到视觉 token。Video Q-Former 由 image Q-Former 初始化，并加入位置编码，对跨帧特征进行时间建模。少量可学习 query 从大量帧 token 中提取固定数量的视觉表示，再通过线性投影映射到 LLM 的输入维度。

这样既复用图像预训练能力，又避免把所有帧 patch 直接送入 LLM。视频到文本生成任务帮助模型学习视频与语言的对应关系。

### 音频分支

音频先被转换为频谱等表示，由 ImageBind 音频编码器提取特征。Audio Q-Former 使用可学习 query 聚合音频 token，再通过投影层对齐 LLM。视觉 query、音频 query 与文本 prompt 一起作为 LLM 条件，实现音视频问答和对话。

### 训练方式

第一阶段使用大规模图像/视频描述数据进行跨模态对齐，主要训练 Q-Former 和投影层；第二阶段使用高质量视觉指令数据做 instruction tuning，使输出符合对话指令。冻结大模块降低了训练成本，但也限制了端到端适配能力。

### 局限与追问

均匀稀疏采帧和固定 query 数可能遗漏细粒度动作与长时序信息；音画同步、幻觉和复杂时间推理仍是难点。面试时要区分原始 Video-LLaMA 与后续 Video-LLaMA 2 等工作，不能把后续架构细节放回原论文。
