# 为什么在大模型训练中需要预热（Warmup）

## 答案

**一句话**：Warmup 是训练初期把学习率从很小逐步升到目标值，避免模型一开始梯度不稳定导致训练崩掉。

### 核心回答
大模型训练初期参数还没进入稳定区域，如果直接用较大学习率：
- loss 可能剧烈震荡。
- 梯度可能爆炸。
- optimizer 的动量估计还不稳定。
- 混合精度训练更容易溢出。

Warmup 的作用：
- 给模型和优化器一个适应过程。
- 让梯度和动量统计更稳定。
- 降低训练初期崩溃概率。

常见策略：linear warmup、cosine warmup、按总 step 的 1%-5% 做 warmup。

### 1. 为什么大模型训练初期不稳定

训练刚开始时，模型参数、梯度分布、优化器动量估计都还不稳定。如果一上来就用目标学习率，可能出现：

- loss 剧烈震荡。
- 梯度爆炸。
- mixed precision 溢出。
- 模型刚开始就偏离预训练能力。

Warmup 让学习率从小到大，给优化器一个缓冲期。

### 2. Warmup 怎么配

常见做法：

- 总 step 的前 1%-5% 做 warmup。
- warmup 后接 cosine decay 或 linear decay。
- 微调任务通常 warmup 不宜过长，否则有效学习步数变少。
- 数据少或学习率偏大时，可以适当增加 warmup。

### 3. 面试总结

Warmup 是训练稳定性的保护机制，尤其适合大模型、混合精度训练和 LoRA/DPO 这类对学习率敏感的微调。

### 4. 结合跨境支付项目怎么说（面试口径）

> 做支付领域微调时，我会把 warmup 当成稳定训练的保险。比如用 LoRA 或 DPO 训练风控解释模型，如果一开始学习率太大，模型很容易把原来的指令遵循能力冲坏，表现为回答变模板化、乱拒绝，或者对业务术语理解变差。
>
> 我的习惯是设置一小段 warmup，再配合 cosine decay。训练过程中不只看 loss，还会看验证集上的拒答率、格式遵循、风控证据引用是否准确。
>
> 支付场景里我宁愿模型慢一点学，也不希望它突然偏离原本稳定能力。因为一旦上线回答不稳，影响的是客服、风控和合规链路。

**钩子**：业务微调宁愿更新慢一点，也不要一开始把模型训崩。
