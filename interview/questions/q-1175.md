# transformer中QKV的作用是什么

## 答案

Q、K、V 是 Attention 对同一份输入做的三种可学习投影，分别承担“发起匹配、被匹配、提供内容”的职责：

```text
Q = XW_Q
K = XW_K
V = XW_V

Attention(Q, K, V)
= softmax(QKᵀ / √d_k + Mask)V
```

### 1. 三者分别做什么

- **Query**：当前位置希望从上下文中寻找什么信息；
- **Key**：每个位置用于参与匹配的索引表示；
- **Value**：匹配完成后真正被加权汇总的内容表示。

`QKᵀ` 只负责计算相关性，softmax 将其转成注意力权重，最后用该权重对 V 加权求和。把匹配表示和内容表示分开，使模型可以用一组特征判断“该关注谁”，再用另一组特征决定“取回什么内容”。

例如句子“银行拒绝了这笔付款，因为收款人账户已关闭”中，处理“拒绝原因”时，Q 可以与“账户已关闭”的 K 得到较高分数，再汇总对应 V 中的原因信息。

### 2. 为什么不能只使用一个向量

如果 Q、K、V 完全相同且没有独立投影，匹配关系和被聚合内容会被限制在同一表示空间。独立的 `W_Q、W_K、W_V` 允许模型学习非对称关系：

- 代词作为 Query，可以匹配实体的 Key；
- 当前 token 可以匹配历史条件，但读取另一种内容表示；
- Cross-Attention 中，目标序列与源序列来自不同空间；
- 多个 head 可以分别关注位置、实体、否定、时间等关系。

Q、K、V 的名称描述的是计算角色，不代表三套固定语义字段；具体学到什么由数据和损失决定。

### 3. Self-Attention 与 Cross-Attention

Self-Attention 中三者来自同一序列：

```text
Q = HW_Q, K = HW_K, V = HW_V
```

Decoder 的因果 Self-Attention 还会加入 causal mask，使位置 `t` 不能看到未来 token。

Encoder-Decoder 的 Cross-Attention 中：

```text
Q = decoder_hidden W_Q
K = encoder_output W_K
V = encoder_output W_V
```

Decoder 用当前生成状态查询 Encoder 编码的源文本。这里 Q 与 K/V 明确来自不同序列。

### 4. Multi-Head Attention

多头注意力把表示投影到多个子空间：

```text
head_i = Attention(QW_i^Q, KW_i^K, VW_i^V)
MHA = Concat(head_1, ..., head_h)W_O
```

每个 head 可以学习不同关系。最后拼接并通过 `W_O` 混合。标准实现通常令每头维度约为 `d_model / h`，因此增加 head 数不一定按比例增加主 Attention 矩阵的总计算量。

### 5. KV Cache、MQA 与 GQA

自回归推理时，历史 token 的 K/V 不再变化，可以缓存；每一步只计算新 token 的 Q/K/V：

- Q 只服务当前查询，不需要长期缓存；
- 历史 K 用来与新 Q 匹配；
- 历史 V 用来汇总内容。

KV Cache 的大小随层数、序列长度和 KV head 数增长。MQA 让多个 Q head 共用一组 K/V，GQA 让一组 Q head 共享一组 K/V，以较小的质量代价减少显存和内存带宽。

### 6. 跨境支付场景

在支付客服或规则抽取中，Attention 可以把“为什么失败”的查询位置与订单状态、错误码、国家、币种和通道规则建立联系。但 Attention 只能在给定上下文中分配权重；如果 RAG 召回了错误版本的规则，QKV 机制本身无法保证事实正确，因此仍需要权限、时效过滤和引用校验。
