# Agent 如何记忆？

## 答案

**一句话**：Agent 的记忆不是把所有聊天记录都塞回上下文，而是把有价值的信息抽取、结构化、存储，并在合适的时候检索出来辅助决策。

### 1. 记忆类型
- **短期记忆**：当前对话上下文，适合保存本轮任务的临时信息。
- **长期记忆**：跨会话保存的事实、偏好、历史结论，通常放在数据库或向量库里。
- **情景记忆**：某次任务的过程记录，比如一次风控排查、一次对账异常处理。
- **语义记忆**：稳定知识，比如业务规则、产品文档、合规条款。
- **程序记忆**：做事流程，比如如何处理拒付、如何生成商户审核报告。

### 2. 怎么实现

完整记忆链路一般包括：

1. **写入**：从对话、工具结果、业务事件中提取关键信息。
2. **归一化**：变成结构化字段，比如商户号、国家、币种、风险标签、处理结论。
3. **存储**：短期放上下文，长期放关系库、向量库或事件库。
4. **检索**：根据当前任务召回相关记忆，而不是全部加载。
5. **更新和遗忘**：过期、冲突、敏感信息要能更新或删除。

关键点：记忆要**可控、可解释、可清理**，不能把隐私数据、卡号、敏感资金信息随便写入长期记忆。

### 3. 面试总结

Agent 记忆的核心是"把历史经验变成可复用上下文"。好的记忆系统不是越多越好，而是要做到：该记的能记住，该忘的能忘掉，该用的时候能准确召回。

## 结合项目谈（口语版 · 面试现场约 50 秒）

> 在跨境支付场景里，我理解 Agent 的记忆特别重要，但不能简单理解成保存聊天记录。比如一个商户之前因为高频小额交易触发过风控，后来补过 KYB 材料；或者某个通道在某个国家经常回执延迟，这些都属于业务经验。如果 Agent 每次处理都从零开始，就很像新人值班，效率会很低。
>
> 我会把记忆分层：当前工单里的交易信息放短期记忆；商户历史风险标签、历史审核结论放结构化长期记忆；风控 SOP、合规规则放知识库。Agent 处理一笔异常收款时，不是把所有历史流水都塞进去，而是只召回和当前商户、国家、币种、交易模式相关的几条证据。
>
> 另外支付数据很敏感，所以记忆一定要做脱敏和权限控制。比如卡号、证件号不能直接进向量库，只存风险标签、事件摘要和原始数据链接。这样既能让 Agent 有经验，又不会把敏感信息到处扩散。

**自然带出去的钩子**：
- "我会把记忆做成短期上下文、结构化历史和业务知识库三层。"
- "支付场景里记忆一定要脱敏，否则 Agent 能力越强，数据风险越大。"
- "记忆召回要按任务精准召回，不然上下文会乱，模型也容易误判。"
