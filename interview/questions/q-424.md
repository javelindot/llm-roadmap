# 🔥 从数学角度分析L1、L2正则化区别

## 答案

**一句话**：L1 正则更容易产生稀疏权重，适合特征选择；L2 正则让权重整体变小更平滑，适合抑制过拟合。

### 1. 数学区别

L1 正则是参数绝对值之和：

```text
lambda * sum(|w|)
```

L2 正则是参数平方和：

```text
lambda * sum(w^2)
```

L1 在 0 附近不可导，优化时更容易把部分权重压到 0，形成稀疏解。L2 的梯度连续，会让权重变小，但通常不会直接变成 0。

### 2. 使用场景

- L1：特征很多、希望自动筛掉无用特征。
- L2：希望模型更平滑、更稳定，避免权重过大。
- Elastic Net：结合 L1 和 L2。

### 3. 结合跨境支付项目怎么说（面试口径）

> 在跨境支付风控模型里，如果有大量稀疏特征，比如国家、通道、设备、商户行业、历史行为标签，L1 可以帮助做特征选择，让无效特征权重变成 0。L2 更适合让模型参数不要过大，提高泛化能力。
>
> 例如风控里某些特征在训练集里偶然和欺诈相关，但线上不稳定，L2 可以抑制这种过大的权重。实际项目里我会结合验证集看 AUC、召回率、误杀率，而不是只从数学上决定。

**钩子**：可以引到“L1 稀疏特征选择、L2 平滑泛化、风控误杀率”。
