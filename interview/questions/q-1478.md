# private和protected区别，如何访问私有变量，不要get和set怎么访问私有属性

## 答案

`private` 和 `protected` 的精确规则依语言而异。共同点是它们都限制封装边界；外部代码“不用 getter/setter 直接访问 private”通常就是不被允许，否则 `private` 没有意义。

### 1. Java 中的区别

- `private`：仅声明它的类内部可访问；
- `protected`：同包可访问，且其他包的子类可通过继承语义访问；
- package-private：不写修饰符，同包可访问；
- `public`：任何可见位置。

注意：Java 子类也不能直接访问父类的 `private` 字段。

### 2. C++ 中的区别

- `private`：类成员和 friend 可访问；派生类不可直接访问；
- `protected`：类成员、friend 和派生类可访问；
- 继承方式还会影响基类 public/protected 成员在派生类中的可见性。

### 3. 不用 getter/setter，如何合法使用 private

**在类自己的业务方法中访问**

```java
class Account {
    private long balance;

    public void debit(long amount) {
        validate(amount);
        balance -= amount;
    }
}
```

这比暴露 `setBalance` 更符合封装，因为外部执行的是业务行为。

**构造器/工厂**

通过构造器或静态工厂初始化私有字段。

**Java 嵌套类**

同一顶层类中的嵌套类可通过编译器机制访问彼此私有成员。

**C++ friend**

可以声明 friend 函数/类，但会扩大耦合，应谨慎。

**序列化/ORM/反射**

框架可通过 reflection、method handle 或代码生成访问，但这属于框架能力，可能受模块系统、安全策略和运行时限制，不应作为普通业务代码绕过封装的方法。

### 4. 为什么不建议反射强行访问

- 破坏不变量；
- 升级易失败；
- 安全和模块限制；
- AOT/混淆兼容问题；
- 难测试和维护。

### 5. 设计建议

如果外部确实需要某种能力，应暴露语义化方法、只读视图、接口或受控构造器，而不是公开字段。

### 6. 跨境支付场景

账户余额应是 private，外部只能调用经过幂等、权限和账务校验的 `debit/credit`，不能通过 setter 任意修改。

### 7. 面试总结

private 只允许类内部访问，protected 扩展到子类或包等范围。外部不通过方法就不应直接访问 private；反射/friend 是特殊机制，不是常规封装设计。
