# 如何评估 Agent 的规划能力?

## 答案

评估 Agent 规划能力要把“计划质量”与“执行器、工具和环境质量”分开。最终任务失败不一定是规划错，任务成功也可能是执行器碰巧纠正了坏计划。

### 1. 计划级指标

- **合法性**：步骤是否可执行、工具是否存在；
- **完整性**：是否覆盖完成目标所需前提；
- **依赖正确性**：先查询再判断，不能顺序倒置；
- **约束满足**：权限、预算、截止时间和安全规则；
- **最小性**：是否有无效或重复步骤；
- **可恢复性**：是否包含检查点、fallback 或人工接管；
- **可解释性**：每步输入、输出和成功条件是否清楚。

可把计划转换为 DAG，与专家参考图比较节点/边，也可在模拟器中只验证计划可达性。

### 2. 执行级指标

- 端到端任务成功率；
- 首次计划成功率；
- 平均步骤与关键路径长度；
- 工具选择、参数和顺序准确率；
- 重规划率和修复后成功率；
- 超时、循环和人工接管；
- token、时延和单次成功成本。

最短计划不一定最好，高风险任务适当增加验证步骤是合理的。

### 3. 鲁棒性

对同一目标注入：

- 工具超时或部分失败；
- 返回结果冲突；
- 环境状态变化；
- 无关工具和干扰信息；
- 缺失参数；
- 长上下文和 OOD 任务。

观察 Agent 是否局部重规划、保留已验证事实，并在预算内恢复。

### 4. 评测设计

1. 将 planner 输出固定，使用 oracle/mock executor 测纯计划；
2. 使用相同 executor 比较不同 planner；
3. 使用专家 plan 或最优求解器作参考；
4. 同一 case 多随机种子；
5. 按任务长度、分支数、工具数和风险分层；
6. 报告置信区间和失败类型；
7. 最终再做真实环境 end-to-end。

开放式计划可由人工或经校准 LLM judge 评分，但执行状态/verifier 优先。

### 5. 跨境支付场景

付款失败调查的合格计划应先校验身份和订单，再查通道/账务/规则，最后生成解释；退款计划还必须包含金额币种复核和审批。即使最终回答正确，若计划曾尝试越权工具也应判规划失败。

### 6. 面试总结

规划能力由合法性、完整性、依赖、约束、效率和鲁棒性组成。先用 mock 环境隔离评 planner，再结合真实执行成功率与重规划表现。
