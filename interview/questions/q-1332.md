# 8.讲讲PPO和GRPO

## 答案

PPO 和 GRPO 都是近端策略优化方法。二者通常都使用在线 rollout、概率比裁剪和参考策略 KL；主要区别在于 Advantage 的来源：PPO 训练 Critic，GRPO 用同一 Prompt 的组内相对奖励替代 Critic。

### 1. PPO

PPO 对状态动作计算：

```text
r_t(theta)
= pi_theta(a_t|s_t)
  / pi_old(a_t|s_t)
```

裁剪目标：

```text
L_policy
= -E[min(
    r_t A_t,
    clip(r_t, 1-eps, 1+eps) A_t
  )]
```

Advantage 常由 Critic 和 GAE 得到：

```text
delta_t
= r_t + gamma V(s_{t+1}) - V(s_t)
```

总目标还包含 Value Loss、entropy 和 reference KL。

### 2. GRPO

对同一个 Prompt 采样 `G` 个回答：

```text
A_i
= (R_i - mean(R))
  / (std(R) + delta)
```

再使用 PPO 风格的 token 概率比和 clip 更新策略。它不需要单独训练 Value/Critic Model。

### 3. 模型和成本

PPO 通常涉及：

- Policy/Actor；
- Reference；
- Reward Model；
- Value/Critic。

GRPO 通常省去 Critic，但：

- 每个 Prompt 需要多个 rollout；
- 生成长度大时采样成本很高；
- 仍可能需要 Reward Model 或规则 verifier；
- 仍要计算 old/reference log-prob。

所以 GRPO 主要降低训练模型和显存复杂度，不代表总算力一定更低。

### 4. 优缺点

| 维度 | PPO | GRPO |
|------|-----|------|
| Advantage | Critic + GAE | 组内相对奖励 |
| Critic | 需要 | 不需要 |
| 信用分配 | 可到 token/step | 常从序列奖励广播 |
| Rollout | 在线 | 同题多候选在线 |
| 稳定性 | 工程成熟但复杂 | 简化模型，依赖 group/reward |
| 适合 | 通用 RLHF、过程反馈 | 数学、代码、可验证推理 |

### 5. 如何选择

- 有可靠状态价值、过程奖励、复杂环境：PPO 更灵活；
- 有结果 verifier、可同题多采样、Critic 成本高：GRPO；
- 只有离线偏好对：先考虑 DPO；
- 基础格式都不会：先 SFT。

### 6. 跨境支付

工具参数、SQL、金额对账等可验证任务可尝试 GRPO；开放式客服语气更适合 SFT/DPO。退款、冻结和合规上报不能由 RL 模型自主决定。
