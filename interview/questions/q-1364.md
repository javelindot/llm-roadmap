# Transformer在训练过程中更新哪些参数？

## 答案

Transformer 训练时更新哪些参数取决于训练方式。全参预训练/SFT 会更新所有 `requires_grad=True` 的模型参数；LoRA 只更新 Adapter；冻结层、量化基座和优化器配置会进一步改变更新集合。

### 1. Embedding

- Token Embedding；
- 可学习 Position Embedding（若使用）；
- Segment/Token Type Embedding（如 BERT）；
- RoPE 通常是固定公式，没有可训练位置表；
- 某些 RoPE scaling 参数也通常不是训练参数。

### 2. Attention

标准 MHA：

```text
W_Q, W_K, W_V, W_O
```

以及存在时的 bias。

GQA/MQA 的 K/V 投影形状不同；MLA 还包括 Q/KV down/up projection 和 decoupled RoPE 投影；MoE Attention 以具体架构为准。

### 3. FFN

原始 Transformer：

```text
W_1, b_1, W_2, b_2
```

SwiGLU：

```text
W_gate, W_up, W_down
```

及可选 bias。

### 4. Normalization

- LayerNorm 的 `gamma` 和 `beta`；
- RMSNorm 的 scale `gamma`；
- 现代模型常没有 Norm bias。

### 5. 输出层

- LM Head/分类头；
- LM Head 可能与 Token Embedding 权重共享；
- 权重共享时物理上是同一参数，不能重复计数。

### 6. MoE 和多模态

MoE：

- Router/Gate；
- 各 Expert FFN；
- Shared Expert；
- 可能的 expert bias。

多模态：

- Vision/Audio Encoder；
- Projector/Connector；
- Cross-Attention；
- 语言模型。

实际训练可能冻结视觉或语言主干，只更新连接层。

### 7. 不属于模型参数的状态

- Adam 的一阶/二阶矩；
- 梯度；
- 激活；
- KV Cache；
- Dropout mask。

它们占显存，但不是模型权重。

### 8. 不同训练方式

**全参训练**

所有权重更新。

**LoRA**

```text
A, B
```

更新，基座冻结；若启用 `modules_to_save`，LM Head 等也可更新。

**Prefix/Prompt Tuning**

只更新虚拟 token/前缀参数。

**BitFit**

只更新 bias。

**冻结部分层**

只更新顶层、任务头或指定模块。

### 9. 如何验证

```python
trainable = [
    (n, p.numel())
    for n, p in model.named_parameters()
    if p.requires_grad
]
```

还应检查 Optimizer 的 param groups，避免参数虽 `requires_grad=True` 却没交给优化器，或冻结参数被误加入。

**面试总结**：架构决定“有哪些参数”，训练策略决定“哪些会更新”，优化器 param group 决定“哪些真正被 step”。
