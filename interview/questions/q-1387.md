# 7.了解哪些强化学习算法？off-policy的举几个例子。

## 答案

强化学习算法可以按价值函数、策略梯度和 Actor-Critic 分类，也可以按 on-policy/off-policy 分类。

### 1. Value-based

- Dynamic Programming；
- Monte Carlo；
- SARSA；
- Q-learning；
- DQN、Double DQN、Dueling DQN；
- Distributional DQN。

这类方法主要学习状态价值 `V(s)` 或动作价值 `Q(s,a)`，再根据价值选择动作。

### 2. Policy-based

- REINFORCE；
- Policy Gradient；
- TRPO；
- PPO。

它们直接优化策略：

```text
pi_theta(a | s)
```

适合连续动作或随机策略，但方差可能较大。

### 3. Actor-Critic

- A2C/A3C；
- DDPG；
- TD3；
- SAC；
- PPO 也通常以 Actor-Critic 形式实现。

Actor 负责选动作，Critic 估计价值或优势。

### 4. 什么是 off-policy

off-policy 指训练目标策略和生成数据的行为策略可以不同：

```text
behavior policy: mu
target policy: pi
mu != pi
```

因此可以使用历史数据或 replay buffer，样本利用率通常更高，但需要处理分布偏差和训练稳定性。

典型 off-policy 算法：

- Q-learning；
- DQN；
- Double DQN；
- DDPG；
- TD3；
- SAC。

典型 on-policy 算法：

- SARSA；
- REINFORCE；
- A2C；
- TRPO；
- PPO。

严格说，一些现代实现会混合旧策略数据、重要性采样或异步采样，不能只凭算法名字判断，仍要看数据生成与更新方式。

### 5. on-policy 和 off-policy 对比

| 维度 | on-policy | off-policy |
|---|---|---|
| 数据 | 主要使用当前策略采样 | 可使用其他策略和历史数据 |
| 样本效率 | 较低 | 较高 |
| 稳定性 | 通常较稳定 | 分布偏差更难处理 |
| replay buffer | 通常不用或很少复用 | 常使用 |
| 典型算法 | PPO、A2C | DQN、SAC、TD3 |

### 6. 大模型后训练

大模型 RL 中常见 PPO、GRPO 等基于当前或近邻策略 rollout 的方法，工程上更接近 on-policy。DPO 是离线偏好优化，不应简单等同于传统环境交互式 RL。

跨境支付 Agent 如果使用历史工单做离线训练，要注意日志由旧策略和人工共同产生，存在选择偏差；不能直接把历史动作当最优策略。真实资金动作也不能作为自由探索环境，应在仿真、沙箱和可回放工具环境中训练。

### 7. 面试总结

常见 off-policy 算法有 Q-learning、DQN、DDPG、TD3 和 SAC；PPO、A2C、REINFORCE 更典型地属于 on-policy。核心区别是训练策略能否复用其他行为策略产生的数据，而不是有没有 replay buffer 这一单一表象。
