# 9.大模型了解多少，强化学习了解什么

## 答案

这是知识广度题，适合先给知识地图，再选择与岗位和项目最相关的两三个方向深入，避免堆术语。

### 大模型部分

可从以下链路回答：

- 架构：decoder-only Transformer、Attention、RoPE、RMSNorm、SwiGLU、GQA/MQA、MoE。
- 训练：tokenization、预训练、CPT、SFT、偏好优化、数据配比与分布式并行。
- 高效适配：LoRA/QLoRA、量化、蒸馏和评测。
- 推理：prefill/decode、KV Cache、continuous batching、speculative decoding、延迟与吞吐。
- 应用：RAG、Agent、工具调用、安全、可观测性和成本。

回答中应联系真实项目，说明自己在哪些部分有实现或实验经验，哪些只阅读过。

### 强化学习部分

先说明 MDP 中 state、action、transition、reward、policy 和 discounted return。常见方法包括：

- Value-based：Q-learning、DQN；
- Policy Gradient：REINFORCE；
- Actor-Critic：A2C/A3C、PPO；
- 连续控制：DDPG、TD3、SAC；
- 离线强化学习与 model-based RL。

重要概念还有 on-policy/off-policy、exploration-exploitation、credit assignment、value/advantage、importance sampling 和 GAE。

### 与 LLM 对齐的连接

RLHF 中 prompt/部分序列可视为 state，token 是 action，策略是 LLM，Reward Model 给序列级奖励。PPO 使用 clipped objective、value model 和 reference KL；GRPO 用同一 prompt 的组内奖励构造相对优势。DPO 则直接从偏好对优化策略，不需要在线 rollout 的 RL 循环。

高质量回答应明确传统 RL 理论与 LLM 对齐实践的共同点和差异，而不是把所有偏好训练都称为强化学习。
