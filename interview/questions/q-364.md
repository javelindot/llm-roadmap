# FastAPI vs Flask？

## 答案

答：Flask，老牌轻量Web框架，简单灵活，但性能一般、异步弱。
FastAPI，现代高性能Web框架，天生异步、自动文档、速度接近Go，是大模型部署首选。

### 补充说明

Flask 的优势是简单、生态成熟、自由度高，适合小服务、后台管理、简单 API。FastAPI 基于 Starlette 和 Pydantic，天然支持 async、类型校验、自动生成 OpenAPI 文档，更适合模型服务、RAG 服务、工具 API 这类需要高并发和清晰 schema 的场景。

但也不是所有场景都必须 FastAPI。如果团队已有 Flask 体系，简单服务用 Flask 也没问题；如果要做流式响应、WebSocket、异步调用模型和多个下游服务，FastAPI 更合适。

### 结合跨境支付项目怎么说（面试口径）

> 在跨境支付 AI 项目里，我更倾向用 FastAPI 封装模型服务、RAG 检索服务和工具服务。因为这些服务经常要异步调用向量库、模型 API、订单系统和风控系统，FastAPI 的 async 和类型校验会比较方便。
>
> 比如客服系统调用一个“付款失败解释”接口，入参里有订单号、商户 ID、用户问题，FastAPI 可以用 Pydantic 定义 schema，自动校验字段。接口文档也能自动生成，方便 Java 后端或前端对接。
>
> Flask 我也能用，但如果是大模型实时流式回复、WebSocket、并发工具调用，我会优先选 FastAPI。
