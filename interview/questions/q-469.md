# 🔥 跨模态对齐的方案有哪些？为什么选择该方案

## 答案

跨模态对齐的目标是让文本、图像、音频等不同模态表达的同一语义，在可比较或可交互的表示空间中对应起来。

### 1. 常见方案

#### 对比学习双塔

图像和文本分别编码，通过 InfoNCE 等对比损失拉近正样本、推远负样本。CLIP 是典型代表。

- 优点：检索效率高，可提前计算 Embedding。
- 缺点：模态交互较晚，细粒度推理能力有限。

#### 线性投影 / MLP Adapter

用视觉 Encoder 提取特征，再通过投影层映射到 LLM 的 Token 空间。

- 优点：结构简单、训练成本低、容易复用预训练模型。
- 缺点：投影层容量有限，复杂视觉关系可能损失。

#### Cross-Attention / Q-Former

引入可学习 Query 或跨模态注意力，让语言表示主动读取视觉特征。

- 优点：细粒度交互更强。
- 缺点：结构和训练更复杂，推理开销更高。

#### 统一 Token 与原生联合训练

把不同模态离散化或映射成统一 Token，在同一 Transformer 中联合建模。

- 优点：理解与生成接口统一，上限高。
- 缺点：数据、算力和训练稳定性要求高。

### 2. 常见训练目标

- 图文对比损失
- 图文匹配损失
- Caption / Next Token Generation
- Masked Modality Modeling
- 指令微调和偏好优化

### 3. 为什么选择某种方案

选型取决于任务：

- 大规模图文检索：优先双塔对比学习。
- 在已有 LLM 上快速增加看图能力：视觉 Encoder + Projector。
- 文档、图表等细粒度问答：Cross-Attention 或更强的连接模块。
- 统一理解和生成、追求能力上限：原生多模态联合训练。

### 4. 项目回答模板

> 我们选择“预训练视觉 Encoder + 投影层 + LLM”，因为数据和算力有限，重点是快速获得视觉问答能力。先冻结两端只训练投影层完成特征对齐，再用图文指令数据微调部分参数。评测不仅看回答正确率，还看视觉证据定位和纯文本能力是否退化。

**面试总结**：没有通用最优方案。关键是在对齐粒度、推理能力、训练成本和检索效率之间做权衡。
