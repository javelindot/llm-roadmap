# 偏好模型：优化用户偏好预测，主要关注用户个体对不同项的评分或偏好。

## 答案

偏好模型学习“在给定用户或上下文下，哪个候选更符合偏好”。它可以输出单项效用分数、成对胜率或完整排序，Pairwise 只是训练偏好模型的一种方式。

### 1. 输入与输出

推荐场景可以表示为：

```text
r(u, i) = PreferenceModel(user, context, item)
```

生成模型对齐场景可以表示为：

```text
r(x, y) = RewardModel(prompt, response)
```

两个候选的偏好概率常使用 Bradley-Terry 形式：

```text
P(a ≻ b | context) = σ(r(context, a) - r(context, b))
```

因此模型既可以用成对偏好训练，也可以使用评分、列表或隐式反馈。

### 2. 数据来源

- 显式评分、点赞、收藏和选择；
- 点击、停留、购买等隐式反馈；
- 同一 Prompt 下的 chosen/rejected；
- 多候选人工排序；
- 线上 A/B 或受控探索；
- 业务专家依据 rubric 的比较。

隐式反馈不等于真实偏好。未点击可能是未曝光，购买还受价格、库存和支付可用性影响，因此要记录曝光、位置和候选集。

### 3. 个性化与群体偏好

全局 Reward Model 学的是标注群体的平均偏好，可能掩盖用户差异。个性化偏好模型可以加入：

- 用户历史与当前 session；
- 国家、语言和场景；
- 长短期兴趣；
- 标注者或人群 embedding；
- 对长度、风格和风险容忍度的条件。

但敏感属性必须符合合法用途和最小化原则，不能用代理特征形成不透明歧视。

### 4. 常见问题

- 曝光和位置偏差；
- 热门 item 主导；
- 标注者标准不一致；
- 长度、礼貌或格式捷径；
- 偏好不满足传递性；
- 用户兴趣随时间漂移；
- Reward Hacking；
- 绝对分数未校准；
- 训练候选与线上候选分布不一致。

应使用随机流量或 propensity 校正、长度对照样本、多人标注、时间切分和持续校准。

### 5. 评测

- Pairwise Accuracy、Kendall's Tau；
- NDCG、MRR 和 Top-K 采纳率；
- 与人工偏好的相关性；
- 分用户、场景和长尾分桶；
- 长度、位置和风格偏差测试；
- 校准与不确定性；
- 下游业务收益和风险守护指标。

### 6. 跨境支付场景

偏好模型可以预测商户更偏向低成本、快到账还是本地付款体验，并在合法产品池中排序收款、付款或 FX 方案。它不能代替：

- KYB 和牌照准入；
- 制裁与反洗钱规则；
- 实时汇率和通道状态；
- 额度与资金执行。

在客服对齐中，可由专家比较两条回答的证据完整性、合规性和可执行性，但订单状态、金额和时效仍要用权威系统核验。
