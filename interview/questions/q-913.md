# 模型压缩这一块知道哪些具体的方法？

## 答案

模型压缩应从权重精度、参数数量、架构和知识迁移四类理解，目标是满足质量、延迟、内存、功耗和成本 SLO。

### 1. 量化

- PTQ：训练后量化，成本低；
- QAT：训练中模拟量化，精度通常更好；
- Weight-only INT8/INT4：LLM 常见；
- W8A8、FP8/FP4：同时降低激活/计算；
- GPTQ、AWQ、SmoothQuant、bitsandbytes；
- KV Cache 量化。

必须确认目标硬件有高效 kernel，否则位宽低不一定更快。

### 2. 剪枝与稀疏

- 非结构化权重稀疏；
- N:M 半结构化稀疏；
- 结构化剪 head、channel、FFN neuron、layer；
- Token pruning/early exit。

结构化更容易获得真实加速，非结构化需要硬件/运行时支持。

### 3. 蒸馏

- logits/soft label；
- hidden/attention feature；
- sequence-level 与 reasoning distillation；
- on-policy distillation；
- 大模型生成数据再训练小模型。

### 4. 低秩与参数共享

- SVD/低秩分解大矩阵；
- 权重共享、ALBERT；
- LoRA 主要是参数高效微调，不会自动压缩基础模型；合并后参数规模不变；
- 若用低秩近似替换原权重，才属于模型压缩。

### 5. 架构与系统

- 减层、减 hidden/FFN、GQA/MQA；
- Dense→更小 Dense 或端侧架构；
- MoE 降激活 FLOPs，但总权重内存可能更大；
- FlashAttention、PagedAttention 是推理优化，不等于压缩。

### 验证

在真实硬件测任务质量、校准、安全、P99、QPS、峰值内存、功耗和模型包大小，逐步做量化→蒸馏/剪枝→运行时优化。

### 面试总结

压缩方法包括量化、剪枝、蒸馏、低秩/共享和架构缩小。要区分“模型更小”和“服务更快”，最终以真实硬件上的质量、延迟、内存和成本共同评估。
