# 🔥🔥 怎么解决LLM幻觉问题

## 答案

LLM 幻觉不能只靠一个 Prompt 消除，应先区分知识缺失、检索失败、推理错误、引用不一致和过度自信，再从数据、模型、检索、生成和验证分层治理。

### 源头治理

- 提高预训练/SFT 数据的事实性、去重和时间标注，减少冲突与错误监督。
- 对稳定领域知识可 CPT/SFT；频繁变化事实优先外部检索。
- 偏好训练加入“证据不足时承认不知道”的样本，避免奖励迎合式回答。

### RAG 与工具

对时效或私有知识先检索，采用 hybrid retrieval、reranker、权限和时间过滤。上下文保留来源，要求关键 claim 引用证据。数学、代码、数据库和实时状态尽量调用可验证工具。

RAG 本身也会幻觉：召回错误、chunk 不完整或模型忽略证据。因此要评估检索和生成两个阶段。

### 生成与验证

- 降低不必要温度，使用结构化输出与约束解码。
- 将复杂任务拆分，先抽取证据再回答。
- 使用独立 verifier、NLI/claim checker 或执行器逐条核验。
- 多样采样与 self-consistency 只在可验证任务上有帮助，多个相同模型可能一致地错。
- 无可靠证据时拒答、追问或返回搜索结果。

### 评估

建立带来源和时间的事实集，统计 claim-level precision、引用支持率、拒答准确率和分桶 bad case。线上记录问题、证据、模型版本和用户纠错。

目标不是让模型永不出错，而是降低错误概率、让输出可核验，并把高风险错误拦在业务动作前。
