# 什么是agent

## 答案

**一句话**：Agent 是能感知环境、规划任务、调用工具并根据反馈继续决策的智能体，不只是一次性问答模型。

### 核心组成

- LLM：理解和规划。
- Tools：查询或执行外部能力。
- Memory/State：保存上下文和任务状态。
- Planner：拆解任务。
- Executor：执行步骤。
- Guardrail：权限、安全和校验。

### 结合跨境支付项目怎么说（面试口径）

> 在跨境支付客服里，Agent 不只是回答 FAQ。用户问“这笔付款为什么失败”，Agent 要先判断意图，再查订单、查通道回执、查风控结果、检索 SOP，最后生成解释。如果信息不够，还要追问订单号或转人工。
>
> 所以我理解 Agent 的关键是“模型 + 工具 + 状态 + 保护边界”，而不是让模型自由发挥。

**钩子**：可以引到“工具调用、状态管理、保护节点、支付异常排查”。
