# 自回归大语言模型的训练目标是什么？

## 答案

**一句话**：**最大化条件概率 P(x_t | x_<t)** ——给定前面的 token，预测下一个 token 是什么；用 token-level cross-entropy loss 实现。

### 1. 数学定义
对一段长度 T 的文本 x = (x_1, x_2, ..., x_T)：
$$L = -\sum_{t=1}^{T} \log P_\theta(x_t | x_{<t})$$

每个 token 单独计算 -log p，求和得序列 loss。这就是熟悉的 **causal language modeling**。

### 2. 实现细节
- **因果掩码（causal mask）**：attention 时屏蔽未来 token，保证 t 只能看 <t
- **teacher forcing**：训练时用真实历史而非模型预测
- **一次性算完整个序列**：训练时是并行的，不像推理串行

### 3. Cross-Entropy 视角
$$L_t = -\log p(x_t | x_{<t}) = -\sum_v y_v \log \hat{p}_v$$

其中 y 是 one-hot（真实下一 token），$\hat{p}$ 是模型预测分布——所以本质就是 V 类分类问题（V = 词表大小）。

### 4. 困惑度（PPL）与 loss 的关系
$$PPL = \exp(L)$$
PPL 越低，模型对数据预测越准。Llama-3 在 wikitext-2 上 PPL ~5，GPT-2 ~30。

### 5. 与其他训练目标对比
| 目标 | 代表模型 | 特点 |
|------|---------|------|
| **Causal LM**（next token） | GPT、Llama、Qwen | 自回归生成 |
| **Masked LM**（掩码恢复） | BERT、RoBERTa | 双向理解 |
| **Span Corruption** | T5 | 适合 seq2seq |
| **Prefix LM** | UL2 | 双模式融合 |

### 6. 三阶段训练里的目标变化
- **Pretrain**：Causal LM，海量纯文本
- **SFT**：Causal LM，但只对 assistant token 算 loss（input mask）
- **RLHF/DPO**：不再直接 cross-entropy，换成偏好 loss 或 PPO 奖励

### 7. Loss Masking 细节
- SFT 时只在 assistant 输出上算 loss，input 部分 mask 掉
- 长序列 packing 时不同样本之间要 mask
- pad token loss 必须 mask

### 8. 一句话总结
**自回归 = 自我条件预测**，训练目标是 next-token cross-entropy；简单、并行性好、适合 GPU 扩展，是 LLM 时代的基石训练范式。

## 结合项目谈（口语版 · 约 40 秒）

> 这个我会用一个具体细节切入——**我们跨境支付做 SFT 时 loss masking 的踩坑**。
>
> 我们刚做客服模型 SFT 时，写训练脚本没注意到 loss masking 这件事，整个 prompt+answer 全算 loss。跑了几轮发现模型出现了奇怪现象：它**学会了客户问什么**——你给它一段客服回复让它续写，它会续写出一个"客户问"。
>
> 复盘下来很傻——因为我们没 mask 掉用户输入部分，模型对"客户问"的下一 token 预测也在被训练，相当于教它"复述客户问题"。改成 only-assistant loss 之后立刻正常。
>
> 这个坑让我对"训练目标 = next-token CE"的理解更深一层——**理论上你算 loss 的所有 token 都在被它学**，所以训练数据怎么切、mask 怎么打，比模型架构还重要。
>
> 现在我们的 SFT 标准流程：input mask、pad mask、长样本 packing 之间互相 mask、turn boundary mask——把"我希望模型学什么"显式控制，效果稳很多。
>
> 一句话：**自回归训练目标看着简单，工程里 loss mask 这块决定了 1/3 的效果**。
