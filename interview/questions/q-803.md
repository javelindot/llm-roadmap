# 🔥🔥 了解大模型吗

## 答案

可以从训练、结构、后训练、推理和应用五层回答，体现体系而不是零散名词。

### 1. 基本原理

主流文本大模型多为 Decoder-only Transformer，通过 next-token prediction 在海量 token 上预训练。核心组件包括 causal Self-Attention、RoPE、RMSNorm、SwiGLU 和残差连接。

### 2. 规模化结构

- Dense 模型每个 token 使用全部参数；
- MoE 只激活少量专家，以通信和路由复杂度换取容量；
- GQA/MQA、MLA 减少 KV Cache；
- 多模态模型用视觉/音频编码器、投影器或统一 token 接入语言主干。

### 3. 后训练

- SFT 学指令、格式和领域行为；
- DPO 等方法学习离线偏好；
- PPO、GRPO、DAPO 等在 rollout 上优化奖励；
- 数学、代码和工具任务越来越多使用可验证奖励；
- 对齐不能替代知识更新、权限和运行时安全。

### 4. 推理与部署

推理分 prefill 与 decode，瓶颈包括算力、KV 显存和 HBM 带宽。常用量化、continuous batching、PagedAttention、prefix cache、speculative decoding、并行和蒸馏。

### 5. 应用与边界

RAG 提供可更新证据，Agent 连接工具和 workflow，评测需覆盖正确性、安全、延迟、成本和业务指标。截至 2026 年，方向重点已从“单纯扩大参数”扩展到推理 RL、Agent、多模态、长上下文、小模型与高效服务。

跨境支付中，大模型适合客服、知识检索和操作辅助，但订单状态、费率和资金动作必须由权威系统与权限流程控制。

### 面试总结

大模型是 Transformer 预训练、后训练和推理系统共同组成的产品，不只是一个权重文件。回答要能从结构、数据、对齐、部署、应用与风险完整展开。
