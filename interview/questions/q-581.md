# 训练模型如果loss不稳定，如何解决?

## 答案

Loss 不稳定要按“数据 → 数值 → 优化器 → 模型结构 → 分布式实现”的顺序排查，先找到异常从哪一步开始。

### 1. 先观察现象

- 是正常抖动、持续震荡、突然 Spike，还是 NaN/Inf；
- 训练集和验证集是否同时异常；
- 是否只在特定 Batch、长度或数据源出现；
- 梯度范数、学习率和显存是否同步异常。

### 2. 数据问题

- 标签错误、脏数据、重复样本；
- 极长序列或异常输入；
- 不同数据集混合比例突变；
- Batch 太小导致方差大；
- Padding、Mask 或 Loss Mask 写错。

应记录触发 Spike 的 Batch 并可复现。

### 3. 数值稳定性

- FP16 Overflow/Underflow，改 BF16 或动态 Loss Scaling；
- Softmax、Log、除法未做稳定实现；
- 混合精度中关键归一化和累加精度不足；
- 检查 NaN/Inf，必要时开启异常检测。

### 4. 优化器与超参数

- 降低学习率；
- 增加 Warmup；
- 使用 Gradient Clipping；
- 增大有效 Batch 或梯度累积；
- 检查 Adam 的 Betas、Epsilon 和 Weight Decay；
- 调整学习率调度器，避免恢复训练时 LR 跳变。

### 5. 模型与分布式

- Pre-LN/RMSNorm 和初始化是否合理；
- 残差尺度是否过大；
- Checkpoint 恢复是否包含优化器状态；
- 梯度同步、Loss Reduction 和数据并行缩放是否正确。

### 6. 推荐排查方式

先在单卡、小数据、FP32/BF16 下复现，再逐步打开混合精度、并行和大 Batch。一次只改变一个变量。

**面试总结**：不要看到 Loss 抖就直接调学习率。最有效的方法是定位首个异常 Batch 和首层异常梯度，再按数据与数值链路逐层缩小范围。
