# 介绍字节实习

## 答案

**一句话**：介绍实习要讲清楚业务背景、项目目标、个人职责、技术方案、结果和复盘，不要流水账。

### 参考结构

- 团队做什么。
- 项目解决什么问题。
- 你负责哪些模块。
- 用了什么技术。
- 遇到什么难点。
- 结果和收获是什么。

### 结合跨境支付项目怎么说（面试口径）

> 我实习期间主要参与的是跨境支付智能客服/风控解释相关项目。业务背景是一笔交易异常往往需要查订单、通道、风控、合规和知识库，人工处理成本比较高。
>
> 我负责的部分包括 RAG 知识库建设、错误码和通道规则检索优化、工具调用封装，以及 Agent 流程里的状态管理和保护节点。技术上用了向量检索、BM25、rerank、Function Calling 和 LangGraph 类流程编排。
>
> 项目让我最大的收获是，大模型落地不是只调 prompt，而是要和业务系统、权限、安全、评估指标一起设计。

**钩子**：可以引到“个人负责模块、RAG/Agent、真实业务约束”。
