# PPT 模板在项目中如何使用？适用于哪些业务场景？是否支持用户自定义模板？

## 答案

> **模板用法**：
>
> 我们用 **python-pptx** 操作模板 pptx 文件，模板里用占位符（如 `{title}`, `{chart_1}`, `{section_text}`）标记可填充位置，运行时用 LLM 生成的内容 + 渲染好的图表填充。
>
> 实现思路：
> ```python
> from pptx import Presentation
>
> def render_ppt(template_path: str, content: PPTContent) -> bytes:
>     prs = Presentation(template_path)
>     for slide in prs.slides:
>         for shape in slide.shapes:
>             if shape.has_text_frame:
>                 replace_placeholders(shape.text_frame, content)
>             if shape.shape_type == MSO_SHAPE_TYPE.PICTURE:
>                 maybe_swap_image(shape, content)
>     return save_to_bytes(prs)
> ```
>
> 模板按"章节模板 / 图表模板 / 表格模板"分类，Agent 生成完内容后根据章节类型选对应模板拼装。
>
> **业务场景**：
>
> 我们生产里跑了三个场景：
>
> **1. 月度业务报告（最重）**
> 给管理层看，30+ 页，包含交易趋势、风控指标、增长机会、下月计划。固定模板，月底自动跑。
>
> **2. 商户准入审核报告**
> 给商户和合规看，10-15 页，包含 KYB 审核结果、风险评分、限额建议、合规备注。每个商户一份。
>
> **3. 风控调查结案报告**
> 给监管和内审看，5-8 页，包含工单详情、调查链路、证据列表、最终结论。可疑交易调查完触发。
>
> **模板分类**：
> - **品牌模板**（公司 logo / 色调 / 字体）：固定，不可改
> - **业务模板**（章节结构 / 图表样式）：按场景预设
> - **自定义模板**（用户上传）：支持但有限制
>
> **是否支持用户自定义**：
>
> 部分支持。原因：
> - **完全自由**：用户上传任意 pptx 当模板。但风险大——版式不规范、占位符冲突、字体丢失，渲染崩。
> - **半结构化自定义**：我们做了"模板编辑器"，让用户在固定章节框架内调样式（颜色、字体、logo），但不能改章节结构。这是我们当前主流方案。
> - **完全模板化**：业务侧用户大多只用预设三套模板（标准 / 简洁 / 监管风格），自定义需求其实不高。
>
> **几个工程细节**：
> - **占位符规范**：用 `{{var_name}}` 双花括号语法，避免和正常文字混淆
> - **图表大小自适应**：用户改了 slide 尺寸，渲染时按比例缩放图表
> - **字体兜底**：模板字体缺失时 fallback 到"思源黑体"
> - **模板版本管理**：每个模板有 version，更新时旧 job 用旧模板继续跑
> - **预览功能**：模板更新后先生成示例 PPT 让用户看效果
>
> **没解决的问题**：
> - 排版自适应（章节文字过长会溢出）
> - 复杂的表格嵌入（嵌入 Excel 数据保留 vs 截图）
> - 模板互相引用 master slide 的兼容性
>
> 一句话：**模板化是 PPT 生成的关键**——既保证品牌一致性，又让 LLM 内容能稳定渲染；完全自定义听起来酷，但工业上"半结构化 + 预设几套"够用。
