# 一面，自我介绍，介绍推理框架ollama，流程化结构dify，rag原理，用到的开源闭源大模型效果如何

## 答案

您好，我主要关注大模型在跨境支付业务中的工程落地，业务范围包括全球收款、付款、换汇、商户入驻、客服和运营辅助。项目设计上，我不会让模型直接决定资金动作，而是把系统拆成知识检索、实时工具、模型生成和安全控制四层：RAG 提供产品与合规规则，业务 API 返回订单、余额和通道状态，模型负责理解与解释，权限、幂等、账务和审批由确定性系统控制。

技术上，我熟悉文档解析与切块、BM25 与向量混合检索、rerank、Prompt 与结构化输出，也使用 Ollama 做本地模型验证，用 Dify 快速搭建和观察工作流。模型选型不会只看公开榜单，而是用脱敏业务黄金集比较正确性、引用、工具调用、安全、延迟和单次成本。

### 1. Ollama 的定位

Ollama 是本地和服务化运行模型的工具，提供模型拉取、运行、管理和 HTTP API。官方 API 默认地址为：

```text
http://localhost:11434/api
```

它支持流式生成、embedding、结构化输出、tool calling，以及 OpenAI 兼容接口，适合：

- 本地开发和快速验证不同模型；
- 数据不能直接发送外部服务的原型；
- 小规模内部工具和离线实验；
- 统一模型启动与基础 API 调用。

但 Ollama 不等于完整的生产推理平台。正式上线还要验证：

- 目标模型和量化版本的准确率；
- 并发、continuous batching、KV Cache 和显存利用；
- P50/P95/P99、超时与取消；
- 多实例负载均衡、弹性和故障恢复；
- 模型版本、审计、限流、租户隔离和监控。

高并发或严格 SLA 下，应与 vLLM、TGI 或云模型服务做同硬件、同输入输出长度的压测，按结果选型。

### 2. Dify 的定位

Dify 是 LLM 应用与工作流编排平台，可以配置模型供应商，并通过节点组合：

- User Input、LLM、Knowledge Retrieval；
- 条件分支、变量、代码和 HTTP 请求；
- Agent/Tool、模板、Answer；
- 数据集、embedding、rerank 和元数据过滤。

一个支付客服 Chatflow 可以是：

```text
用户问题
→ 鉴权与意图识别
→ 知识 / 实时交易 / 人工路由
→ BM25 + Dense Retrieval + Rerank
→ 订单或通道 API
→ LLM 生成结构化答案
→ 敏感信息与事实校验
→ 回复或转人工
```

Dify 的价值是快速编排、可视化调试和验证流程，但生产系统不能把所有关键逻辑锁在低代码节点中。资金权限、签名、幂等、账务状态机和审计应由后端服务实现；工作流定义需要版本管理、测试、发布审批和回滚。

### 3. RAG 原理

RAG 在生成前检索外部证据：

```text
文档采集与治理
→ 解析、结构切块和元数据
→ BM25 / Vector 索引
→ Query 理解与 ACL 过滤
→ 混合召回、RRF 和 Rerank
→ 证据上下文
→ LLM 回答与引用
```

跨境支付知识库包括产品说明、国家与币种支持、KYC/KYB、错误码、退款/退汇/拒付 SOP 和 API 文档。订单状态、余额、实时汇率、通道健康和审核进度由工具读取，不能依赖静态 RAG。

文档更新时保留 `version、effective_time、tenant、ACL` 并删除旧索引。在线回答要求关键结论引用来源，证据不足或冲突时澄清或转人工。

### 4. 开源与闭源模型如何比较

不能笼统说开源一定差或闭源一定好，应在同一任务、同一 Prompt、同一工具和同一评测集上比较。

**开源或开放权重模型**

- 可私有化、固定版本、量化和 LoRA；
- 数据路径与推理成本更可控；
- 需要承担 GPU、容量、推理优化、升级和安全运维；
- 效果受模型尺寸、量化、语言和部署配置影响。

**闭源 API 模型**

- 接入快，复杂推理、长上下文或多模态候选通常更丰富；
- 需要评估数据出境、保留策略、版本漂移、限流、可用性和供应商锁定；
- token 成本在高并发、长上下文下可能显著。

实际可以分层路由：

- 高频意图分类、Query 改写和简单客服走小模型；
- 敏感数据与稳定业务走私有模型；
- 合规允许的复杂低频任务再路由到更强服务；
- 所有路径统一鉴权、脱敏、审计、fallback 和评测。

### 5. 效果评测

按国家、币种、产品、语言和风险等级构建黄金集，指标包括：

- RAG 的 Recall@K、NDCG、Context Precision/Recall；
- 答案正确性、faithfulness、引用准确率；
- 工具选择、参数和任务完成率；
- PII 泄露、越权、错误资金承诺和过度拒答；
- TTFT、P95/P99、吞吐、超时和单个成功任务成本；
- 人工改写率、首次解决率和转人工准确率。

最终选择不是某个模型的总榜单分数最高，而是它在目标业务和合规边界内达到可验收的质量、稳定性与成本。
