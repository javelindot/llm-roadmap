# 强化学习中马尔可夫（MDP）中的五元组是什么？

## 答案

马尔可夫决策过程 MDP 常用五元组表示：

```text
(S, A, P, R, gamma)
```

### 1. S：State Space

状态空间，描述智能体当前所处环境。

例如跨境支付客服 Agent 的状态可以包括：

```text
当前用户问题
已确认的订单号
订单状态
已调用工具
已获得证据
当前风险等级
```

### 2. A：Action Space

动作空间，智能体可以执行的动作：

```text
回复用户
查询支付订单
查询出款通道
获取 KYC 状态
要求补材料
转人工
结束任务
```

### 3. P：Transition Probability

状态转移概率：

```text
P(s' | s, a)
```

表示在状态 `s` 执行动作 `a` 后转移到状态 `s'` 的概率。确定性工具调用也可以看成概率集中在一个结果上的特殊情况；真实支付环境中，通道超时、审核结果等可能是随机的。

### 4. R：Reward Function

奖励函数：

```text
R(s, a, s')
```

衡量动作好坏。例如：

- 正确查到订单并解释原因：正奖励；
- 完成用户问题：正奖励；
- 编造状态：大负奖励；
- 泄露敏感信息：大负奖励；
- 不必要的重复工具调用：小负奖励；
- 高风险问题正确转人工：正奖励。

### 5. gamma：Discount Factor

折扣因子：

```text
0 <= gamma <= 1
```

控制未来奖励的重要程度：

- `gamma` 小，更重视即时收益；
- `gamma` 大，更重视长期累计收益；
- episodic 任务中也可以接近 1。

### 6. 马尔可夫性质

MDP 假设：

```text
P(s_{t+1} | s_t, a_t)
```

只依赖当前状态和动作，不依赖完整历史。因此状态设计必须包含决策所需信息。如果支付 Agent 的状态里没有订单号、权限和之前工具结果，它就不满足有效决策所需的信息条件。

### 7. Policy 是否属于五元组

策略：

```text
pi(a | s)
```

描述在状态 `s` 下选择动作 `a` 的概率。策略是我们要学习或优化的对象，但通常不包含在 MDP 环境定义的五元组中。

### 8. 面试总结

MDP 五元组是 `(S, A, P, R, gamma)`：状态、动作、状态转移、奖励和折扣因子。策略 `pi(a|s)` 是基于这个环境学习出来的。落到 Agent 场景，最关键的是状态是否包含足够信息、动作是否受权限约束、奖励是否体现真实业务目标。
