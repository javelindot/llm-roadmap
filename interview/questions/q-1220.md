# QWEN2的架构和训练方式

## 答案

Qwen2 是基于 Transformer 的自回归因果语言模型系列，包含 0.5B、1.5B、7B、72B 四个 Dense 模型，以及总参数 57B、每个 token 激活约 14B 参数的 `57B-A14B` MoE 模型。回答时要以 Qwen2 技术报告为准，避免把 Qwen2.5 或后续 Qwen 系列的配置混入其中。

### 1. Dense 模型架构

Qwen2 使用 Decoder-only Transformer 和 causal mask，训练目标仍是 next-token prediction。核心组件包括：

- **GQA**：Query 头多于 KV 头，减少推理阶段 KV Cache 和内存带宽；
- **SwiGLU**：作为 FFN 激活结构；
- **RoPE**：提供相对位置信息；
- **RMSNorm + Pre-Norm**：提高深层训练稳定性；
- **QKV bias**：注意力投影中保留 Q/K/V bias；
- **DCA + YARN**：用于长上下文外推与跨 chunk 的相对位置建模。

以 Qwen2-7B 为例，报告给出的配置是 28 层、hidden size 3584、28 个 Query head、4 个 KV head，体现了 GQA 对 KV Cache 的压缩。

Tokenizer 延续 byte-level BPE，包含 151,643 个常规 token 和 3 个控制 token，所有尺寸共用词表，有利于多语言和代码数据统一处理。

### 2. MoE 模型

Qwen2-57B-A14B 用 MoE FFN 替代 Dense FFN：

- 每层包含细粒度 routed experts；
- 每个 token 选择部分专家计算；
- 同时设置 shared experts 和 routed experts；
- 从 Dense 模型 upcycle，并通过参数重排与部分重初始化增加专家差异。

它的目标是在较高总容量下控制单 token 激活计算，但路由均衡、专家通信和部署复杂度会高于 Dense 模型。

### 3. 预训练

训练数据强调质量、规模和分布三方面：

- 使用启发式与模型过滤清理低质量内容；
- 增加代码、数学和多语言数据，覆盖约 30 种语言；
- 在小规模模型上实验不同数据源配比；
- 部分高质量指令数据也进入预训练，以增强 in-context learning。

报告说明，除 0.5B 外的 Dense 模型使用超过 7T token 的高质量数据；0.5B 使用 12T token，MoE 模型在 upcycling 后继续训练 4.5T token。团队曾比较更大但质量阈值更低的 12T 数据，较大模型上并未比 7T 高质量数据获得明显收益，说明数据质量和配比不能被 token 数量替代。

### 4. 长上下文训练

预训练末期将上下文从 4,096 扩展到 32,768，并加入更多长文本数据，同时把 RoPE base frequency 从 10,000 调整为 1,000,000。结合 YARN 和 Dual Chunk Attention，报告评测扩展到 131,072 token。

这里要区分：

- 训练时实际上下文长度；
- 位置编码可外推长度；
- 模型在目标长度上的真实任务准确率；
- 推理服务能否承受对应 KV Cache 和延迟。

标称上下文很长，不代表所有长度下都具有相同质量和吞吐。

### 5. 后训练

Qwen2 的后训练分为 SFT 和偏好对齐：

1. 构造 demonstration 与 preference 数据，结合人工标注、拒绝采样、代码执行反馈、约束验证和 Constitutional Feedback。
2. 使用超过 50 万条指令样本做两轮 SFT，报告序列长度为 32,768。
3. 离线阶段使用预先构造的偏好对做 DPO。
4. 在线阶段从当前策略采样多个回答，由 Reward Model 选择最好与最差回答形成新偏好对，再继续 DPO。
5. 使用 Online Merging Optimizer 缓解对齐后通用能力下降。

因此，Qwen2 并不是“预训练后直接拿来聊天”，Base 模型和 Instruct 模型对应不同训练阶段与使用方式。

### 6. 跨境支付应用判断

Qwen2 的中英多语言、代码能力、GQA 和长上下文适合支付客服、商户材料解释、API 文档问答和多国家规则检索。但工程落地仍需：

- 用 RAG 和工具提供最新费率、路由、余额与交易状态；
- 对 KYC/KYB、制裁和资金操作设置确定性权限门禁；
- 按语言、国家、币种和产品建立业务评测；
- 压测长上下文下的 TTFT、KV Cache、吞吐和成本；
- Base、Instruct、量化版或微调版分别验证，不能只看公开榜单。
