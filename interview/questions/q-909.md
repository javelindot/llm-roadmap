# QLoRA是如何实现的

## 答案

QLoRA 将冻结的基础模型量化到 4-bit，只训练 BF16/FP16 LoRA adapter，从而显著降低微调显存，同时在计算时按需反量化权重。

### 1. 4-bit NF4

原始 QLoRA 使用 NormalFloat 4-bit（NF4），其量化取值针对近似正态分布权重设计。基础权重以 4-bit 存储，但矩阵计算通常反量化到 BF16/FP16 compute dtype。

### 2. Double Quantization

不仅量化权重，还量化每个 block 的量化常数，进一步减少 scale 元数据平均占用。

### 3. Paged Optimizer

使用统一内存分页处理长序列或 gradient checkpointing 引起的显存峰值，降低偶发 OOM。它主要管理 adapter optimizer state 的峰值，不意味着没有性能代价。

### 4. LoRA 训练

\[
W_q\approx\operatorname{dequant}(W_{4bit})
\]

\[
y=W_qx+\frac{\alpha}{r}BAx
\]

量化基座冻结，梯度只更新 \(A,B\)。常见配置：

- `load_in_4bit=True`；
- `bnb_4bit_quant_type="nf4"`；
- `bnb_4bit_use_double_quant=True`；
- `bnb_4bit_compute_dtype=bf16`。

训练前通常将 Norm 等模块准备到稳定精度，并开启 gradient checkpointing。

### 5. 局限

- 省显存不等于一定更快，反量化有开销；
- 量化误差会影响高敏感任务；
- optimizer、激活和 KV/临时张量仍占显存；
- adapter 合并与部署需要选择合适精度；
- 不同硬件对 4-bit kernel 支持不同。

原始方法参考 [QLoRA](https://arxiv.org/abs/2305.14314)。

### 面试总结

QLoRA 的三项关键技术是 NF4、Double Quantization 和 Paged Optimizer；4-bit 基座保持冻结，计算时反量化，只训练 LoRA。它主要省权重与优化状态显存，不保证训练同比加速。
