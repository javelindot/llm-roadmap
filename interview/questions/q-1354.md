# ChatGLM的结构是什么？

## 答案

“ChatGLM”需要先区分原始 GLM、ChatGLM-6B/2/3，还是后续 GLM-4.x。早期 ChatGLM 的核心是 Prefix Language Model 与 GLM 的自回归空白填充思想；新一代 GLM 已演进到标准生成式、多模态和 MoE/推理模型，不能用一套结构概括全部代际。

### 1. 原始 GLM

GLM 使用 Autoregressive Blank Infilling：

```text
原文: A [MASK] B
目标: 自回归生成被遮盖 span
```

可见上下文采用双向注意力，待生成 span 内采用因果注意力，试图统一理解与生成。

它使用二维位置信息表达：

- 空白在原序列的位置；
- token 在空白内部的生成顺序。

### 2. ChatGLM-6B

ChatGLM-6B 是面向中英对话的约 6B 参数模型，使用单个 Transformer 主干和 Prefix LM 风格注意力：

- Prompt/上下文部分可双向建模；
- 生成部分保持自回归 causal；
- 使用 GLM 空白填充预训练思想；
- 量化和较低部署门槛是其早期特点。

回答具体 head 数、层数和位置编码前应确认 checkpoint，避免把 ChatGLM2/3 配置混入。

### 3. 后续演进

ChatGLM2/3 对上下文、训练数据、工具调用和工程效率进行了升级。GLM-4 系列进一步扩展：

- 更长上下文和更强工具能力；
- 视觉语言模型；
- 推理/Agent 训练；
- GLM-4.5 等型号引入 MoE，并融合 reasoning 与 non-reasoning 模式。

因此“ChatGLM 是 Encoder-Decoder”并不准确；它是单主干生成模型，早期 GLM 通过特殊 mask 统一双向理解和自回归生成。

### 4. 与 BERT/GPT

- BERT：Encoder-only，双向 MLM，适合理解；
- GPT/Llama：Decoder-only causal LM；
- 早期 GLM/ChatGLM：Prefix/blank-infilling mask，让前缀双向、生成区因果；
- 新 GLM：更接近现代通用生成模型与 MoE/多模态路线。

### 5. 跨境支付选型

不能因为中文效果好就直接上线。需要评测多语言支付术语、工具调用、量化质量、许可证、数据驻留和推理框架兼容性，实时账务事实仍由工具提供。
