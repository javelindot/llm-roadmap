# 在高并发查询 Agent 系统中，你会如何优化召回和生成阶段的延迟？

## 答案

应把端到端延迟拆成召回、重排、首 token、逐 token 生成和 Agent 多步循环，并同时优化 P95/P99、吞吐和任务成功率。

### 1. 召回阶段

- 先做意图与租户路由，只检索相关分区；
- 权限和 metadata 前置过滤，缩小候选空间；
- BM25 与 ANN 并行，使用 RRF/学习融合；
- 调优 HNSW `efSearch`、IVF `nprobe` 等参数，在 recall 与延迟间取舍；
- query embedding、热点结果和文档片段缓存；
- 轻量模型做第一阶段，昂贵 reranker 只处理小 top-k；
- 分片副本、异步 I/O、连接池和超时降级；
- 索引增量更新与蓝绿切换，避免缓存旧规则。

### 2. 生成阶段

- 压缩 prompt、去重片段，控制输入和最大输出；
- continuous batching、PagedAttention、FlashAttention 和优化 kernel；
- prefix caching 复用 system prompt 与工具 schema；
- 量化权重/KV，使用 GQA/MQA 模型；
- chunked prefill 避免长请求阻塞 decode；
- speculative decoding 降低 ITL，按接受率实测；
- 小模型处理简单请求，大模型只处理复杂问题；
- 流式输出降低感知等待。

### 3. Agent 与并发治理

- 能并行的检索和只读工具并行执行；
- 固定流程使用 workflow，限制最大步数与 token；
- deadline-aware 调度、租户配额、背压和熔断；
- 预留高优先级容量，长任务进入异步队列；
- 分别监控召回 P99、rerank P99、TTFT、ITL、队列时间、cache hit 和空结果率。

跨境支付中，缓存键必须包含租户、国家、权限和规则版本。订单状态、汇率、费率不可使用过期缓存；超时应返回可解释降级或转人工，而不是跳过权威查询。

### 面试总结

高并发 Agent 先通过路由、过滤、缓存和小 top-k 降低召回成本，再用连续批处理、KV 管理、前缀缓存和 speculative decoding 优化生成。最终以端到端 P99 和任务正确率评估，不能用吞吐提升掩盖单请求变慢。
