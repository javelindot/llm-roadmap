# DPO的原始数据构建，如果我有一批数据格式遵循不好但是推理能力OK，另一批数据推理不行格式遵循OK，怎么构建偏好对做RLHF/DPO

## 答案

这两批数据分别在“推理正确性”和“格式遵循”上占优，不能粗暴规定其中一批始终 chosen，否则模型会学到错误的目标权衡。

### 1. 先拆成多维标签

对每条回答分别标注：

- 最终答案正确；
- 推理过程有效；
- 格式约束满足；
- 安全与事实；
- 长度和风格。

用规则或解析器验证格式，用执行器/答案校验推理，主观维度再用人工或 Judge。

### 2. 构造 2×2 数据

理想上补齐：

| 推理 | 格式 | 用途 |
|---|---|---|
| 对 | 对 | 最佳 chosen/SFT |
| 对 | 错 | 格式改写来源 |
| 错 | 对 | 推理 hard negative |
| 错 | 错 | 明确 rejected |

可让教师模型只修复正确推理答案的格式，得到“推理对+格式对”；对格式正确但推理错误的回答保留为正确性负例。

### 3. 偏好对要控制变量

- 学格式时：chosen/rejected 推理内容相同，只改变格式；
- 学推理时：格式保持一致，只改变推理正确性；
- 避免一对样本同时在两个维度一好一坏，否则偏好信号矛盾；
- 若无法控制，可使用多目标 Reward Model、条件偏好或 Pareto 优化，而非普通单标量 DPO。

### 4. 训练顺序

先用“正确且格式合规”的数据 SFT，建立稳定输出；再分阶段 DPO：先高权重正确性，再在正确样本内优化格式。训练中分别监控答案正确率和格式通过率，防止一项提升掩盖另一项退化。

跨境支付中，正确但格式不合规的工具调用不能执行；格式正确但金额/订单错误更危险。两者都要作为硬校验，偏好训练只在合法输出间排序。

### 面试总结

先把推理和格式拆成独立维度，补齐“二者都好”的 chosen，并用控制变量构造偏好对。不要让 DPO 在“推理好但格式差”和“格式好但推理错”之间猜业务权重。
