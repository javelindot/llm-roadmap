# 介绍RAG流程;介绍对编码模型的了解、原理、优缺点;如何评估编码模型的能力

## 答案

这道题可分为 RAG 全链路、编码模型原理和评测三个部分。编码模型通常指把 query/document 映射为向量的 Encoder/Bi-Encoder。

### 1. RAG 流程

离线：

```text
文档采集
-> 解析/OCR/清洗
-> 语义切分
-> metadata/ACL
-> embedding
-> BM25/向量索引
```

在线：

```text
query 理解/改写
-> 权限与版本过滤
-> BM25 + Dense 多路召回
-> 融合/去重
-> Cross-Encoder Rerank
-> 上下文打包
-> LLM 生成
-> 引用/校验/拒答
```

最后用 trace、反馈和增量索引形成闭环。

### 2. Bi-Encoder 原理

Query 和 Document 分别编码：

```text
q = f_query(query)
d = f_doc(document)
score = cosine(q, d) 或 q^T d
```

训练常用对比损失：

```text
L =
-log exp(sim(q, d+) / tau)
/ sum_j exp(sim(q, d_j) / tau)
```

负样本可来自 in-batch、BM25 hard negatives 或模型挖掘。Query/Document 可共享或不共享参数，输入常加任务前缀。

### 3. 优缺点

优点：

- 文档向量可离线计算；
- ANN 支持百万/十亿规模；
- 语义匹配和跨语言能力强；
- 在线只需一次 query 编码。

缺点：

- Query 与文档在最终打分前没有 token 级交互；
- 数字、错误码、否定和精确短语可能弱；
- 向量压缩造成信息瓶颈；
- hard negative、领域和长度分布影响很大；
- 更换模型通常要重建索引。

Cross-Encoder 将 query/document 拼接后共同编码，精度通常高但无法全库计算，适合 rerank。Late Interaction（如 ColBERT）介于二者之间。

### 4. 如何评估编码模型

**检索质量**

- Recall@K、HitRate@K；
- MRR、MAP、NDCG；
- 分语言、意图、长度和难度；
- hard negative、无答案和跨域；
- ANN exact-vs-approximate recall。

**表示质量**

- STS/Spearman；
- 分类、聚类或语义匹配迁移；
- 各向异性和向量塌缩；
- Query/Document 长度与截断敏感性。

**系统指标**

- 编码 QPS、P95；
- 向量维度、索引大小；
- ANN 查询延迟与内存；
- 更新成本和缓存；
- 最终 RAG answer accuracy/faithfulness。

公开 MTEB/C-MTEB 适合初筛，最终必须使用目标语料和真实 query-document 标注。

### 5. 跨境支付场景

测试集应覆盖错误码、国家、币种、支付方式、多语言和版本规则。Dense 处理语义改写，BM25 保住 `SWIFT、IBAN、R03` 等精确字段，ACL 在召回前执行。

### 6. 面试总结

Bi-Encoder 的核心优势是文档可预编码、检索可扩展；弱点是缺少深度交互。评估要同时看离线 Recall/NDCG、ANN/延迟和最终 RAG 事实正确性。
