# Qwen 与传统的Transformer模型相比，有什么结构上的改进？

## 答案

需要先说明代际。Qwen1/2/2.5/3 的主干大多仍是现代 Decoder-only Transformer；Qwen3-Next、Qwen3.5/3.6 又引入混合注意力、稀疏 MoE 和原生多模态等变化。因此不能给整个 Qwen 家族只列一套结构。

### 1. 相对原始 Transformer 的常见改动

原始 Transformer 是 Encoder-Decoder、Post-LN、绝对位置编码、ReLU FFN。经典 Qwen 文本模型通常采用：

- Decoder-only 自回归结构；
- Pre-Norm/RMSNorm；
- RoPE，而不是绝对位置 embedding；
- SwiGLU gated FFN；
- 去除或减少线性层 bias；
- GQA，减少 KV Cache；
- FlashAttention 等高效实现；
- 更大的 tokenizer 和多语言词表；
- 长上下文 RoPE scaling。

这些变化与现代 Llama 系模型有共性，不都是 Qwen 独有创新。

### 2. Qwen 的工程与训练特征

- 中文、多语言、代码和数学数据覆盖；
- ChatML/对话模板与工具调用格式；
- Dense 与 MoE 多种尺寸；
- 长上下文训练及百万 token 工程；
- Thinking/non-thinking 模式统一；
- Coder、Math、VL、Omni、Embedding/Reranker 专项模型。

模型能力提升很大一部分来自数据和 post-training，而不是只改网络层。

### 3. 新一代混合架构

Qwen3-Next 以及后续 Qwen3.5/3.6 的部分型号进一步使用：

- 全注意力与线性/门控注意力交替的混合结构；
- 极稀疏 MoE，提高总容量但控制每 token 激活参数；
- 更高效的长上下文训练和推理；
- 原生视觉语言建模与多层视觉特征；
- 更强的 Agent、工具和代码训练。

应以具体模型 `config.json` 和模型卡为准。例如 Dense、MoE、文本和多模态版本的层结构并不相同。

### 4. 与“传统 Transformer”比较时的注意点

不要只说：

- “Qwen 使用 Attention”；
- “参数更多”；
- “支持中文”。

更好的回答是从四层展开：

1. 基础块：Norm、Attention、FFN、位置编码；
2. 效率：GQA/MoE/混合注意力/KV Cache；
3. 训练：数据、上下文、SFT/RL；
4. 产品能力：多模态、工具调用、结构化输出。

### 5. 跨境支付选型

多语言客服和 RAG 可使用文本 Instruct；回单、证件和截图使用 VL；高频服务重点测试量化后吞吐、长上下文退化和工具调用准确率。模型结构再新，也不能替代权限、审计、幂等和确定性账务系统。
