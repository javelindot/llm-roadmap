# 了解的召回推荐模型

## 答案

推荐系统召回的目标，是从百万或亿级候选中快速选出数百到数千个可能相关的候选，优先保证覆盖，再交给粗排和精排。常见方法可分为非参数、Embedding、序列、图和生成式召回。

### 1. 协同过滤

**ItemCF**

根据共同交互用户计算物品相似度，从用户历史物品扩展相似物品。可解释、上线简单，但热门偏置和冷启动明显。

**UserCF**

寻找相似用户并召回其交互物品。大规模用户场景计算和存储成本较高。

### 2. 矩阵分解

MF、BPR-MF 将 User/Item 映射到低维向量，使用点积评分。BPR 用 pairwise loss 优化隐式反馈排序。相比 ItemCF 泛化更强，但仍难充分利用复杂上下文。

### 3. 双塔召回

User tower 编码用户、历史和上下文，Item tower 编码物品属性：

```text
score(u, i) = user_embedding(u) · item_embedding(i)
```

Item 向量可离线建立 ANN 索引，在线只计算 User 向量，适合大规模召回。DSSM、YouTube Candidate Generation 都属于代表路线。

### 4. 多兴趣召回

MIND、ComiRec 等为一个用户生成多个兴趣向量，分别召回不同主题候选，缓解单向量把多个兴趣平均掉的问题。需要处理兴趣数量、路由、候选去重和分数校准。

### 5. 序列推荐

GRU4Rec、SASRec、BERT4Rec 等根据行为顺序建模短期意图和兴趣迁移，适合 Session 与 next-item prediction。长序列会增加计算，也容易受曝光策略和噪声行为影响。

### 6. 图召回

Random Walk、PinSage、LightGCN 等在 User-Item、Item-Item 或知识图谱上聚合邻居。适合高阶协同关系，但图更新、采样和在线新鲜度更复杂。

### 7. 内容与规则召回

- BM25/全文：标题、类目和 Query；
- 内容向量：文本、图片和多模态；
- 热门、新品、地域和运营；
- 类目、品牌和价格带；
- 安全、库存和资格过滤。

这些通道是冷启动和兜底的重要组成。

### 8. 生成式召回

将物品映射为 Semantic ID，使用 Transformer 根据用户序列自回归生成候选 ID。TIGER 是代表工作。它可统一序列建模和召回，但面临 ID 设计、约束解码、无效 ID、索引更新和推理延迟问题。

### 9. 多路融合

生产中通常并行 ItemCF、双塔、序列、图、内容和热门通道，按 quota/RRF/学习融合合并，再去重、过滤、粗排和精排。评测 Recall@K、覆盖、长尾、新内容、延迟和独占贡献。

跨境支付并非电商推荐，但相似技术可用于商户与产品/通道候选匹配。支付候选必须先满足国家、币种、牌照和风险硬约束，不能只按相似度召回。
