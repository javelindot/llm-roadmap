# HuggingFace 下载不了模型问题

## 答案

**一句话**：国内访问 HF 的核心方案有四个——**HF-Mirror 镜像、ModelScope（魔搭）、手动 wget、内网代理**——99% 场景用 `HF_ENDPOINT=https://hf-mirror.com` 一行解决。

### 1. 解决方案速查
| 方案 | 难度 | 速度 | 推荐 |
|------|------|------|------|
| **HF-Mirror（镜像）** ⭐ | 简单 | 快 | 99% 场景 |
| **ModelScope** | 简单 | 极快（国内） | 中文模型首选 |
| **wget + 镜像直链** | 中 | 看节点 | 大文件 |
| **代理 / VPN** | 复杂 | 不稳定 | 兜底 |
| **离线包传输** | 简单 | 一次性 | 跨网段集群 |

### 2. HF-Mirror 用法（最简单）

**方式一：环境变量**
```bash
export HF_ENDPOINT=https://hf-mirror.com
huggingface-cli download Qwen/Qwen3-7B
```

**方式二：代码里**
```python
import os
os.environ["HF_ENDPOINT"] = "https://hf-mirror.com"
from transformers import AutoModel
model = AutoModel.from_pretrained("Qwen/Qwen3-7B")
```

**方式三：hf_transfer 加速**
```bash
pip install -U hf-transfer
export HF_HUB_ENABLE_HF_TRANSFER=1
export HF_ENDPOINT=https://hf-mirror.com
huggingface-cli download Qwen/Qwen3-7B
```

### 3. ModelScope 用法（中文模型推荐）
```python
from modelscope import snapshot_download
model_dir = snapshot_download('Qwen/Qwen3-7B')
```
- 阿里魔搭，国内访问极快
- 中文模型（Qwen、ChatGLM、InternLM、Baichuan）首选
- 下载下来路径直接可被 transformers 加载

### 4. 大文件断点续传
```bash
huggingface-cli download \
  --resume-download \
  --local-dir ./models/qwen3-7b \
  Qwen/Qwen3-7B
```

### 5. 完全离线场景
- 步骤 1：在能访问的环境下载完整 repo
- 步骤 2：tar 打包 → 复制到离线集群
- 步骤 3：`from_pretrained("/local/path", local_files_only=True)`

### 6. 验证完整性
- 下完检查 `sha256` 与 HF 提供的一致
- 大模型常出现下载不完整（safetensors 文件少几个 shard）
- HF-Mirror 偶尔会出现 metadata 跟主站不一致

### 7. 常见报错对应解
| 错误 | 解 |
|------|-----|
| `requests.exceptions.SSLError` | 关 SSL 验证 / 换镜像 |
| `404 Repo Not Found` | 模型名错 / 权限缺 |
| `403 Gated Model` | 需要登录 HF + 接受协议 |
| `connection timeout` | 切镜像 / 加代理 |
| `disk space error` | 清 `~/.cache/huggingface` |

### 8. 一句话总结
**HF-Mirror + hf-transfer = 90% 场景最佳解**，中文模型首选 ModelScope，离线集群打 tar 包。

## 结合项目谈（口语版 · 约 25 秒）

> 这道题我们跨境支付里**实打实踩过坑**——团队刚做 LLM 时第一周全在跟下载问题战斗。
>
> 几个具体的方案演化：
>
> **第一阶段**——直接 transformers from_pretrained，访问 huggingface.co 经常 timeout、retry 失败。
>
> **第二阶段**——加镜像 `HF_ENDPOINT=https://hf-mirror.com`，速度从龟速变可用，但下 70B 这种大模型还是慢。
>
> **第三阶段**——`hf_transfer` 加 Rust 加速下载，70B 模型从 4 小时降到 40 分钟，可以接受。
>
> **第四阶段**——给整个团队搭了**内部模型仓库**（基于 MinIO 的 S3 兼容存储），常用模型一次性 mirror 到内网，团队任何人 `from_pretrained` 自动从内网拉，**几乎瞬间到位**。生产集群也直接从这里拉，不再依赖外网。
>
> 还有一个细节：**安全合规要求**——金融业务不能让生产机直接出公网。所以我们的标准 SOP 是"研发用 HF-Mirror、生产用内部仓库"，两套链路严格分开。
>
> 一句话：**HF-Mirror 是个人快速解、内部模型仓库才是工业方案**。

**🪝 顺嘴带出去的钩子**：
- "内部模型仓库怎么搭可以聊聊。"
- "金融合规对 LLM 下载链路的要求可以展开。"
- "ModelScope vs HF-Mirror 在国内的速度对比有数据。"
