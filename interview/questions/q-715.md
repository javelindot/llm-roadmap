# 绝对位置编码和相对位置编码的区别是什么？

## 答案

绝对位置编码表示“token 在序列的第几个位置”，相对位置编码表示“query 与 key 相距多远或方向如何”。它们都用于打破 Self-Attention 对输入排列的对称性。

### 绝对位置

将位置向量与 token embedding 相加：

$$
h_i=e_i+p_i
$$

$p_i$ 可以是固定正弦余弦或可学习表。实现简单，每个位置有明确表示，但注意力需要间接从两个绝对向量推断距离。可学习位置表通常受训练最大长度限制，外推较弱。

### 相对位置

注意力分数或 value 中直接加入与 $i-j$ 有关的项：

$$
score_{ij}=q_i^Tk_j+b_{i-j}
$$

Transformer-XL、T5 relative bias、DeBERTa 等采用不同相对设计。相同距离可跨位置共享，更符合许多序列模式，也常有更好的长度泛化，但实现和高效 kernel 更复杂。

### RoPE 与 ALiBi

RoPE 对 Q/K 按绝对位置旋转，但点积结果显式依赖相对位移，因此常归为相对位置建模方法。ALiBi 直接给注意力分数加入与距离成比例的 head-specific bias。

### 选型

短且固定长度的分类任务使用可学习绝对位置足够；长上下文、自回归和长度外推更常使用 RoPE、relative bias 或 ALiBi。任何方法的外推都不能只看“能运行”，还要评估长距离检索、困惑度和 lost-in-the-middle。
