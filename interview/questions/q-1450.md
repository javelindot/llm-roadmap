# 有了解Qwen3的embedding和Reranker模型吗

## 答案

Qwen3 Embedding 系列是 Qwen 团队在 2025 年发布的文本 Embedding 和 Reranker 模型，基于 Qwen3 Dense 基座，面向文本/代码检索、排序、分类、聚类和跨语言匹配。

### 1. 模型系列

文本 Embedding：

```text
Qwen3-Embedding-0.6B
Qwen3-Embedding-4B
Qwen3-Embedding-8B
```

文本 Reranker：

```text
Qwen3-Reranker-0.6B
Qwen3-Reranker-4B
Qwen3-Reranker-8B
```

官方仓库标注上下文长度为 32K，覆盖 100 多种语言及多种编程语言。

### 2. Embedding 特点

- 将 query/document 编码成向量；
- 支持 instruction-aware 输入；
- 支持 Matryoshka Representation Learning，可截取不同向量维度；
- 适合 ANN 大规模召回；
- 多语言、跨语言和代码检索能力较强。

官方示例对 query 加任务指令，而 document 通常不必加同样指令：

```text
Instruct: Given a query, retrieve relevant passages
Query: ...
```

实际使用必须保持训练与服务的模板、pooling、归一化一致。

### 3. Reranker 特点

Reranker 联合读取 query 和 document，输出相关性分数。它能建模 token 级交互，通常比向量相似度准确，但不能对百万文档逐一计算，所以放在召回后：

```text
Embedding retrieve top-100
-> Reranker score
-> top-5/10
```

### 4. 选型

- 0.6B：延迟和成本优先；
- 4B：效果与成本平衡；
- 8B：追求上限、GPU 资源充足；
- 向量维度可按 Recall、索引大小和延迟评测；
- Reranker 候选数要按 P95 和吞吐预算确定。

不能只看公开榜单，必须在自己的 query、语言和文档分布上评测。

### 5. 跨境支付场景

可用于：

- 中英文/多语言商户问题召回；
- 错误码和政策文档检索；
- 工单相似案例；
- 代码和技术文档搜索。

需要按国家、币种、产品、版本和商户权限做 metadata filter，Reranker 不能替代硬过滤。

### 6. 评测

- Recall@K、HitRate@K；
- MRR、NDCG；
- 多语言与长文本；
- hard negative；
- 无答案；
- P95、显存、索引大小；
- 最终回答正确率和引用支持率。

### 7. 最新扩展

截至 2026 年，Qwen 团队还发布了 Qwen3-VL-Embedding/Reranker，将检索扩展到文字、图片、截图和视频。文本知识库与多模态材料应分别评估，不要混用模型结论。

### 8. 面试总结

Qwen3 Embedding 用于高效召回，Qwen3 Reranker 用于精排；系列覆盖 0.6B/4B/8B、32K、多语言、指令感知和可变向量维度。生产选型要联合看召回、排序、延迟和索引成本。
