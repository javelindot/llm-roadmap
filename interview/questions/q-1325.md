# GRPO为什么要做clip，直接用SFT后的模型还会不稳定吗？优势度可以怎样改进？

## 答案

GRPO 仍然是策略梯度方法。即使从 SFT 模型开始，在线采样、稀疏奖励和长序列概率比也会让策略在少数更新内偏移，因此仍需要 clip 和 KL 等约束。SFT 是较好的初始化，不等于 RL 天然稳定。

### 1. 为什么要 Clip

对旧策略采样的 token，概率比为：

```text
r_t(theta)
= pi_theta(a_t | s_t)
  / pi_old(a_t | s_t)
```

GRPO 沿用 PPO 风格目标：

```text
L
= -E[min(
    r_t * A,
    clip(r_t, 1-eps, 1+eps) * A
  )]
```

若不裁剪，一个高奖励样本就可能让其全部 token 概率被大幅提高；低奖励样本也可能被过度压低，导致：

- 策略更新过大；
- KL 快速上升；
- 输出格式或语言能力退化；
- 奖励模型漏洞被迅速放大；
- 训练震荡甚至崩溃。

Clip 限制单批数据对策略的影响，但它不是严格 trust region，也不能替代学习率、KL 和数据质量。

### 2. 为什么 SFT 后仍会不稳定

- RL 的 rollout 来自当前策略，分布持续变化；
- 一个序列包含大量 token，概率比误差会累积；
- 结果奖励通常稀疏，信用难分到具体推理步骤；
- 同组奖励接近时，标准化会放大噪声；
- verifier 或 Reward Model 可能存在漏洞；
- 训练过度偏向数学、代码等窄分布会造成遗忘；
- 长回答可能因聚合方式产生额外奖励或惩罚偏差。

### 3. 基础优势估计

同一 Prompt 采样 `G` 个输出，奖励为 `R_i`：

```text
A_i
= (R_i - mean(R))
  / (std(R) + delta)
```

它用组均值替代 Critic baseline，节省 Value Model，但会依赖组大小和组内方差。

### 4. 优势度如何改进

- **Leave-one-out baseline**：比较其他候选均值，降低自身进入 baseline 的耦合；
- **过滤无信号组**：全对或全错时不更新，或动态补采样；
- **增大/动态 group size**：困难题多采样，简单题少采样；
- **Reward whitening/clipping**：限制异常奖励和数值放大；
- **过程奖励**：给关键中间步骤反馈，改善长程信用分配；
- **多维奖励**：正确性、格式、安全、长度分别归一化后组合；
- **去长度偏差**：谨慎选择 token/sequence 聚合和长度归一化；
- **分桶 baseline**：按题型和难度比较，避免简单题与难题尺度混杂。

DAPO 等后续工作还探索了非对称 clip、动态采样、token-level policy gradient 和超长回答惩罚，以改善探索与稳定性。

### 5. 跨境支付

支付 Agent 的奖励应优先来自可验证结果：工具参数、订单状态一致性、币种金额校验、权限和审批流程。不能只奖励“最终话术看起来正确”，否则模型可能跳过真实查询，直接猜测交易状态。

**面试总结**：SFT 解决起点，clip 控制单次步幅，可靠优势估计决定更新方向，三者职责不同。
