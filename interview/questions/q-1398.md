# 召回离线评估指标体系

## 答案

召回阶段的目标是在有限候选规模和延迟预算内，尽可能覆盖后续排序需要的相关 item。离线评估不能只看 Recall@K，还要看覆盖、增量、质量、效率和分布公平性。

### 1. 核心相关性指标

**Recall@K**

```text
Recall@K =
前 K 个候选中的相关 item 数 / 全部相关 item 数
```

**HitRate@K**

前 K 个候选中是否至少命中一个相关 item，适合每个 query 只有少量正例的场景。

**Precision@K**

前 K 个候选中相关 item 的比例。召回阶段 K 较大时它通常不是唯一主指标，但可以衡量候选噪声。

**MRR/NDCG**

如果召回结果也有顺序，可看首个正例位置或位置敏感相关性。

### 2. 候选上限指标

后续排序模型不可能找回召回阶段遗漏的正例，因此要看：

- Ground-truth coverage；
- Oracle Recall；
- 精排正样本在召回候选中的覆盖率；
- 不同 K 下的 Recall 曲线；
- 候选规模增加带来的边际收益。

### 3. 多路召回指标

对每一路分别统计：

- 独立 Recall@K；
- 独占命中率；
- 与其他通道的重合率；
- 合并后的边际 Recall 增益；
- 移除该通道后的指标下降；
- 多路命中比例；
- 不同用户和场景的覆盖。

通道召回很多重复候选，即使单路 Recall 高，也可能没有增量价值。

### 4. 覆盖与多样性

- item/文档覆盖率；
- 用户覆盖率；
- 类目、商户、国家或主题覆盖；
- 长尾和新品覆盖；
- 候选内多样性；
- 热门集中度；
- 新颖性。

### 5. 工程指标

- P50/P95/P99 延迟；
- QPS 和吞吐；
- 索引大小和内存；
- 索引新鲜度；
- 超时率和空结果率；
- 单请求成本；
- ANN 的近似召回率。

### 6. 数据和评测陷阱

- 日志只包含旧策略曝光过的 item，有曝光偏差；
- 将未点击全部当负样本会引入伪负例；
- 随机切分可能泄漏未来行为；
- 只用热门样本会高估效果；
- 标签窗口未成熟会漏掉延迟转化；
- 离线正例不一定代表用户唯一满意的候选。

应采用时间切分、去重、受控探索日志、hard negative 和分人群切片。

### 7. 跨境支付 RAG

支付知识库召回应看：

- 正确政策/错误码文档 Recall@K；
- 当前有效版本命中率；
- 国家、币种、产品和语言覆盖；
- 无答案查询的空召回识别；
- 权限过滤准确率；
- 检索延迟和索引更新时效。

### 8. 面试总结

召回离线评估体系是“相关性 + 上限 + 增量 + 覆盖多样性 + 系统成本”。主指标通常是 Recall@K/HitRate@K，但必须结合多路独占贡献、长尾覆盖、延迟和数据偏差，才能判断真实价值。
