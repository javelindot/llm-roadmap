# 排序模型离线指标和线上效果不一致如何处理?

## 答案

排序模型离线变好但线上无收益，通常不是一句“线上有噪声”能解释，需要沿着数据、特征、候选、指标、系统和实验链路逐层排查。

### 1. 离线指标与业务目标错位

- 优化整体 AUC，但线上只展示 top-K；
- 点击提升不代表转化或长期满意；
- 离线没有计算位置敏感的 NDCG；
- 忽略多样性、退款、投诉和成本；
- 标签定义与线上 KPI 不一致。

解决：重新建立目标树，增加 NDCG、校准、长期价值和护栏指标。

### 2. 训练数据存在偏差

- 曝光和位置偏差；
- 旧策略选择偏差；
- 未曝光 item 被当负样本；
- 负采样与线上候选分布不同；
- 延迟转化标签未成熟；
- 随机切分导致时间泄漏。

解决：时间切分、成熟标签窗口、随机探索流量、IPS/SNIPS/DR 校正和 hard negative。

### 3. Train-Serving Skew

- 线上线下特征实现不同；
- 特征时间点穿越；
- 默认值或缺失率不同；
- 模型、词典、归一化版本不一致；
- 序列截断和预处理不同。

解决：统一特征定义，做样本回放、特征 diff、线上日志复算和版本审计。

### 4. 候选分布变化

排序模型离线评测使用固定候选集，但新模型上线后：

- 上游召回发生变化；
- 过滤规则不同；
- 热门内容和库存变化；
- 新模型改变曝光后又影响后续反馈。

需要在真实候选上回放，并联合评估召回、排序和重排。

### 5. 模型分数未正确使用

- 分数未校准却直接做多目标加权；
- 阈值未重新选择；
- 新分数被重排规则覆盖；
- 缓存仍返回旧结果；
- 服务降级比例过高；
- P99 增加导致请求超时。

解决：记录从模型分数到最终展示的完整决策链和覆盖率。

### 6. A/B 实验问题

- 分流不随机或样本污染；
- 实验周期太短；
- 指标方差大、样本量不足；
- 新奇效应或节假日影响；
- 用户跨端、跨桶；
- 多个实验互相干扰。

需要做 SRM 检查、功效分析、置信区间、A/A 实验和分层分析。

### 7. 排查顺序

```text
确认实验可信
-> 确认线上模型实际生效
-> 对齐线上线下特征
-> 回放真实候选和最终展示
-> 分析指标与标签偏差
-> 做人群/场景切片
-> 调整目标并重新实验
```

### 8. 跨境支付场景

例如通道成功率模型离线 AUC 上升，线上成功率不变，可能是：

- 合规规则过滤掉了模型最优通道；
- 实时通道健康度没有进入离线数据；
- 模型分数未校准；
- 路由规则覆盖了模型；
- 成功标签有清算延迟；
- 成本或风险护栏改变了最终选择。

### 9. 面试总结

离线线上不一致要排查“评测目标、日志偏差、特征一致性、候选分布、规则覆盖、服务生效和实验设计”。先证明线上确实使用了正确模型，再讨论模型本身是否需要改。
