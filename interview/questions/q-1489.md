# 如何将知识图谱融入推荐系统

## 答案

知识图谱通过“实体和关系”给推荐系统增加结构化语义，常用于召回、特征增强、表示学习和可解释推荐。

### 1. 建图与实体对齐

先定义用户、物品、品牌、类目、属性、地点等实体，以及点击、购买、属于、相似、关联等关系。日志和内容需要做实体链接、去重、置信度与时间版本管理。

图中应区分事实关系和行为关系，并保留事件时间，避免把未来交易或后生成属性泄漏到训练样本。

### 2. 融入方式

1. **路径和规则召回**：从用户交互物品沿关系扩散，例如“购买过 A -> A 属于某类目 -> 召回同类物品”；
2. **图嵌入**：用 TransE、RotatE 等学习实体向量，再与推荐模型表示融合；
3. **GNN 聚合**：用 GraphSAGE、GAT、LightGCN 等聚合邻居信息；
4. **特征增强**：把品牌、类目、关系路径和邻域统计作为精排特征；
5. **联合训练**：推荐损失与知识图谱补全或对比学习损失一起训练；
6. **可解释性**：展示经过验证的关系路径，但不能把相关路径误写成因果解释。

### 3. 主要难点

- 图谱稀疏、错误边和热门节点偏置；
- 多跳扩散带来的噪声与计算成本；
- 动态行为图的实时更新；
- 新用户、新实体和跨域对齐；
- 训练时的信息泄漏；
- 离线图表示与线上特征版本不一致。

实践中可使用邻居采样、关系类型门控、时间衰减、置信度过滤和增量更新，并通过去掉图特征的消融确认真实收益。

### 4. 跨境支付场景

可构建：

```text
商户 -> 行业/MCC -> 国家/地区 -> 币种
支付方式 -> 收单机构 -> 通道 -> 支持区域
交易 -> 设备/IP/账户 -> 风险关系
```

图可用于相似商户冷启动、通道候选召回和团伙风险发现。例如，新商户可参考同地区、同行业、相似客单价商户的成功通道。

但牌照、制裁名单、受益所有人和限额等事实必须来自权威数据并走确定性规则与人工治理，不能由图嵌入的相似性推断代替。

### 5. 面试总结

知识图谱融入推荐主要有路径召回、图嵌入、GNN、特征增强和联合训练五条路线。关键不只是“用了图”，而是实体对齐、时序防泄漏、图噪声控制、线上更新以及消融验证。
