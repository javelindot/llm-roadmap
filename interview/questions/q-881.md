# llm如何处理长文本记忆功能

## 答案

LLM 本身没有无限、可靠的持久记忆。长文本处理应把“上下文窗口、会话记忆、外部知识和模型参数记忆”分开设计。

### 1. 扩展上下文窗口

使用 RoPE scaling、长序列继续训练、FlashAttention、GQA/MLA、序列并行等支持更长输入。优点是保留原文，缺点是计算、KV Cache 和 lost-in-the-middle，不适合无限累积。

### 2. 滑动窗口与摘要

保留最近对话，将旧内容分层摘要：

- 短期窗口保存原始最近消息；
- 中期摘要记录决策和未完成事项；
- 结构化状态保存用户偏好、实体和任务；
- 摘要必须带来源和更新时间，避免错误不断压缩放大。

### 3. 外部检索记忆

将历史事件、文档和对话 chunk 写入向量/关键词索引，查询时按当前问题召回。metadata 包含用户、时间、权限、实体和版本。重要事实优先存结构化数据库，不只存 embedding。

### 4. Agent Memory

- Episodic：发生过什么；
- Semantic：稳定事实；
- Procedural：如何完成任务；
- Working memory：当前计划和工具结果。

写入前判断价值、置信度和隐私；支持更新、冲突、过期和遗忘。模型不能直接信任检索到的历史用户文本，以防提示注入。

### 5. 参数记忆

持续预训练/SFT 可内化稳定模式，但更新慢、难删除、不适合用户私有和易变事实。生产记忆通常放在外部系统。

跨境支付中，用户偏好可保存，订单、余额和 KYC 状态必须实时查询；记忆系统应租户隔离、加密、审计并支持删除。

### 面试总结

长文本记忆通常是“近期原文 + 分层摘要 + 结构化状态 + 外部检索”，而不是无限扩 context。参数只保存稳定能力，实时与敏感事实留在受控系统。
