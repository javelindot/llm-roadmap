# 如果新增一个收藏的指标，应该如何加在PLE 模型中

## 答案

在 PLE 中新增“收藏”任务，不能只在模型末尾随意加一个输出节点。需要同时修改标签定义、任务分支、损失、采样和线上打分，并验证新任务是否给原有任务带来负迁移。

### 1. 先定义收藏标签空间

假设原模型预测点击、停留和转化，收藏任务可以定义为：

```text
y_fav = 1：本次曝光后，在归因窗口内发生有效收藏
y_fav = 0：发生曝光，但归因窗口内没有收藏
```

需要明确：

- 收藏发生在哪个曝光或 session 上；
- 收藏后立即取消是否仍算正样本；
- 重复收藏如何去重；
- 归因窗口和标签成熟时间；
- 未曝光样本不能直接当负例；
- 特征只能使用曝光时已知信息，不能泄漏收藏后的行为。

如果只有点击后才能收藏，只在点击样本上训练会产生样本选择偏差。更稳妥的做法是在全曝光空间定义 `pFavorite`，或者同时建模：

```text
pClick
pFavoriteAfterClick
pClickAndFavorite = pClick × pFavoriteAfterClick
```

具体选择取决于线上排序直接使用哪个概率以及标签是否可观测。

### 2. 修改 PLE 的任务结构

PLE 的每层 CGC 通常包含 shared experts 和 task-specific experts。新增收藏任务后：

1. 增加一组 `favorite-specific experts`；
2. 增加 Favorite Gate，在共享专家与收藏专属专家之间分配权重；
3. 增加 Favorite Tower，输出 `pFavorite`；
4. Shared Gate 按 PLE 结构决定是否聚合新增任务专家；
5. 原有任务 Gate 仍只读取共享专家和自己的专属专家，避免直接引入无关噪声。

可以表示为：

```text
h_fav = Gate_fav(x) ·
        [SharedExperts(x), FavoriteExperts(x)]

p_fav = sigmoid(Tower_fav(h_fav))
```

如果收藏与停留、点赞关系强，共享专家会学习共同兴趣；收藏专属专家负责学习“强兴趣但未必立即转化”的特征。

### 3. 调整多任务损失

总损失可以写成：

```text
L = λ_click L_click
  + λ_watch L_watch
  + λ_convert L_convert
  + λ_fav L_fav
```

收藏通常比点击稀疏，直接设置相同权重可能导致梯度过小；权重过大又可能损害主任务。可以先根据正样本率和业务价值设置基线权重，再尝试 uncertainty weighting、GradNorm 或动态权重，但必须观察每个任务的梯度方向和线上收益。

对于类别极不平衡，可使用正例加权、Focal Loss 或分层采样。采样后要保留或校正真实先验，否则输出概率会失去校准性。

### 4. 防止负迁移

上线前要比较新增任务前后的：

- 每个任务 AUC、GAUC、LogLoss 和校准误差；
- 收藏任务 PR-AUC、Recall 和不同人群分桶表现；
- shared/task-specific expert 的利用率与 Gate 熵；
- 任务梯度 cosine similarity；
- 原有点击、时长和转化任务是否下降。

如果收藏与某些任务冲突，可以增加收藏专属专家数量、减少共享层数、调整 loss 权重，或者使用梯度冲突处理方法。不能只看收藏指标提升就认为改造成功。

### 5. 线上排序与实验

模型输出新增 `pFavorite` 后，最终分数可采用多目标融合：

```text
Score = w1 × f(pClick)
      + w2 × f(pWatch)
      + w3 × f(pFavorite)
      + w4 × f(pConvert)
```

各任务概率应先校准，再通过网格实验、约束优化或可学习融合确定权重。收藏是中长期价值信号，线上实验除收藏率外还应观察停留、回访、取消收藏、转化、多样性和负反馈，防止系统用标题党内容换取短期收藏。

### 6. 跨境支付中的类似多任务建模

在向商户推荐收款、付款、换汇或卡产品时，“收藏”可以对应“查看详情、订阅、加入对比或表达开通意向”。新增意向任务的方式与 PLE 类似，但最终准入仍由 KYB、牌照、额度和风险服务决定。

多任务可预测：

> `pView、pInterested、pApply、pApproved、pUse、pPaySuccess`

其中申请、审批和实际使用存在明显的序列关系和延迟反馈。模型负责估计商户兴趣与使用概率，不能把“高兴趣”直接等价为“允许开通”。
