# PLE之后的多任务模型还有了解吗

## 答案

PLE 之后的多任务推荐研究并不是沿单一路线演进，主要在四个方向继续解决问题：

> 更显式的任务关系建模、Embedding 层隔离、漏斗偏差修正、梯度与损失优化。

### 1. AdaTT：显式任务到任务融合

PLE 通过 shared experts 和 task-specific experts 控制共享，但任务间知识如何传递仍较间接。AdaTT 使用多层 task-specific fusion units，并结合 Gate 和残差连接，自适应学习一个任务应该从其他任务接收多少信息。

适合任务关系复杂、不完全遵循统一共享模式的场景。代价是结构和专家路由更复杂，需要监控任务间传递是否引入负迁移。

### 2. STEM：把隔离下沉到 Embedding 层

许多 MMoE/PLE 实现虽然上层有任务专属专家，但底层仍共享同一套 Embedding，任务冲突可能在进入专家前已经发生。

STEM 引入 shared embedding 与 task-specific embedding，并使用门控控制前向知识共享和反向梯度更新，使不同任务可以学习不同的用户与 item 表示。它重点处理“表示层负迁移”，而不只是 Tower 或 Expert 层冲突。

### 3. AITM：建模有序转化漏斗

对于曝光、点击、加购、购买等有先后依赖的任务，简单并行 Tower 没有显式使用顺序关系。AITM 使用自适应信息传递模块，将上游任务表示传到下游任务，并通过注意力控制传递。

适合有明确漏斗关系的任务，但如果任务不是严格单向依赖，强制传递结构可能产生错误归纳偏置。

### 4. ESCM²：修正 ESMM 类模型的估计偏差

ESMM 利用：

```text
pCTCVR = pCTR × pCVR
```

缓解 CVR 数据稀疏和样本选择偏差，但后续研究指出它仍可能存在 CVR 高估和因果关系弱化。ESCM² 在 entire-space 多任务框架中加入反事实风险最小化，针对这些偏差做校正。

这类方法解决的是样本可观测性和因果偏差，不是 PLE 的 Expert 结构问题，可以与多专家模型组合。

### 5. 梯度与任务权重方法

架构之外还可以使用：

- **Uncertainty Weighting**：依据任务不确定性学习 loss 权重；
- **GradNorm**：平衡不同任务训练速度；
- **PCGrad**：任务梯度冲突时做投影；
- **MGDA**：寻找多目标共同下降方向；
- **动态采样**：控制高频任务对 batch 的主导。

这些方法解决“如何优化共享参数”，与 PLE/AdaTT 的“共享什么、如何路由”互补。

### 6. 如何选型

- 任务高度相关、资源有限：Shared-Bottom 或 MMoE 先做基线；
- 既有共享又有明显任务差异：PLE；
- 任务之间需要显式、非对称传递：AdaTT；
- 冲突已发生在 Embedding：STEM；
- 有明确的曝光到转化漏斗：AITM、ESMM/ESCM²；
- 架构合理但训练仍冲突：动态权重或梯度方法。

没有必要因为论文更新就替换稳定的 PLE。应先通过任务相关性、梯度夹角、Gate 利用率、样本空间和分桶 bad case 确认瓶颈，再选择对应方法。

### 7. 跨境支付场景

支付产品推荐可以联合预测：

> `查看 → 申请 → 审批 → 开通 → 使用 → 支付成功`

申请、审批和使用有漏斗关系，可考虑 AITM 或 entire-space 建模；成功率、成本和风险则可能互相冲突，更适合 PLE/AdaTT 的专属表示。不同国家、币种和通道的底层特征语义差异很大时，可以评估 STEM 类任务专属 Embedding。

无论使用哪种模型，KYB、牌照、制裁、额度和实时通道可用性都是硬约束，不能由多任务概率替代。
