# 对一份数据在某个场景下进行分类/识别/检测，用大模型有什么方法可以做？

## 答案

先判断数据模态、标签量、延迟和错误成本，再决定是否真的需要大模型。可按从低成本到高成本的顺序建立基线。

### 1. Prompt/ICL

把类别定义、few-shot 示例和输出 schema 给 LLM，做 zero/few-shot 分类。适合快速验证、标签少和类别语义明确的任务，但成本高且稳定性需 constrained output。

### 2. Embedding + 传统分类器

使用预训练 Embedding，训练 Logistic Regression、XGBoost 或小 MLP。训练快、延迟低，适合中等标签量和高 QPS；可用原型/最近邻支持新增类。

### 3. Encoder 微调

BERT/DeBERTa/ViT 等接分类头，全参或 LoRA 微调。固定类别、高吞吐分类通常比生成式 LLM 更合适。

### 4. 生成式 SFT

让 LLM 输出类别、证据和结构化字段，适合标签体系复杂、多任务、开放集与解释需求。训练只计算 assistant 输出，并对 schema 做校验。

### 5. 弱监督与蒸馏

用大模型给无标签数据产弱标签、解释或困难样本，经规则和人工抽检后蒸馏到小模型。可用 active learning 优先标注不确定样本。

### 6. RAG/工具与多模态

类别依赖动态规则时，先检索规则再判断；图像、音频或表格使用多模态模型。高风险检测加入规则和人工复核。

评估除 Accuracy/F1/AUC 外，还要看类别不均衡、开放集拒识、校准、漂移、P99 和成本。跨境支付风险分类尤其关注召回、误杀和可解释证据。

### 面试总结

大模型可通过 prompt、embedding、Encoder 微调、生成式 SFT、弱监督蒸馏和 RAG 做分类。应先用最小可行模型，只有复杂语义、开放集或多模态确有收益时才升级。
