# 询问之前实习中Agent的设计思路，以及所提创新方法的实现方式。

## 答案

这是项目深挖题，应按真实实习经历回答，清楚区分已有组件、团队工作和个人创新，不能把框架调用包装成算法创新。

### Agent 设计思路

先说明任务为何需要 Agent，而不是固定 workflow 或单轮 RAG。画出状态流：

1. 输入理解与风险分类；
2. Planner 拆解任务；
3. Tool/Retrieval 节点执行；
4. Verifier 核验结果；
5. Aggregator 生成最终答案；
6. 失败、追问和人工确认分支。

说明状态如何持久化、工具 schema、超时重试、幂等、权限、最大循环和降级方案。

### 创新方法怎么讲

用“问题、基线、改动、机制、消融、收益和局限”展开。例如：

> 基线容易选错工具。我增加候选工具检索和 schema-aware reranker，只向 LLM 暴露 top-k 工具；再用执行成功和参数正确性构造评测。

随后说明训练/规则数据、算法、接口和线上流程，以及去掉该模块后的结果。创新可以是工程机制、数据闭环或评测，不一定是新网络。

### 评测

报告任务完成率、工具选择/参数准确率、引用正确率、循环率、P95 延迟和成本。对 Prompt injection、工具失败和无证据问题做分桶。

若没有实际上线，应明确是原型或离线实验。面试官更看重对失败和取舍的理解，而不是“创新”标签本身。
