# 常见位置编码

## 答案

**一句话**：位置编码分**绝对**（Sinusoidal / Learned）和**相对**（RoPE / ALiBi / T5 Relative Bias）两大派——**现代 LLM 几乎全用 RoPE**（Llama/Qwen/Mistral），少数长上下文场景上 ALiBi。

### 1. 绝对位置编码
**Sinusoidal（原始 Transformer）**
$$PE(pos, 2i) = \sin(pos / 10000^{2i/d})$$
$$PE(pos, 2i+1) = \cos(pos / 10000^{2i/d})$$
- 加在 embedding 上
- 不需训练，可外推但效果一般

**Learned Absolute（BERT/GPT-2）**
- 每个位置一个可训 embedding
- 上限固定（max_len 写死）
- 无法外推

### 2. 相对位置编码（主流）
**T5 Relative Bias**
- 在 attention score 上加一个相对距离的 bias
- 简单但慢

**RoPE（Rotary Position Embedding）⭐**
- 对 Q/K 做**旋转**：每对维度按位置旋转一定角度
- Attention(q, k) 自然包含相对位置信息：`<R_m q, R_n k> = <q, R_{n-m} k>`
- **优点**：实现简单、效果好、支持外推（YaRN/NTK 扩展）
- **缺点**：长度外推需要技巧

**ALiBi（Attention with Linear Biases）**
- 在 attention score 上加一个**线性衰减 bias**：score -= |i - j| × slope
- 不用训练、自动外推
- **优点**：长上下文外推稳
- 用于 BLOOM、MPT

### 3. 主流 LLM 的选择
| 模型 | 位置编码 |
|------|---------|
| BERT | Learned Absolute |
| GPT-2/3 | Learned Absolute / Sinusoidal |
| T5 | Relative Bias |
| **Llama-2/3** | **RoPE** |
| **Qwen-2/3** | **RoPE + YaRN** |
| **Mistral** | **RoPE** |
| BLOOM / MPT | ALiBi |
| GLM-4 | RoPE |

### 4. RoPE 外推技巧
- **PI（Position Interpolation）**：把位置坐标缩放到训练长度内
- **NTK-aware**：调整 base 频率，长上下文更平滑
- **YaRN**：NTK + temperature scaling，Qwen 用这套支持 128k
- **LongRoPE**：搜索最优外推参数

### 5. 选型经验
- 常规 LLM 默认 RoPE
- 极长上下文（>32k）追外推 → YaRN / ALiBi
- 推理场景显存有限 → RoPE 配 ring attention

### 6. 一句话总结
**RoPE 是现代 LLM 的位置编码标配**——简洁、效果好、可外推；YaRN/PI 是把它拉到 128k 的关键技巧。

## 结合项目谈（口语版 · 约 40 秒）

> 我们跨境支付里位置编码这块**直接用 base 自带的 RoPE**，没自己魔改。但**长上下文外推**这事我们专门研究过。
>
> 业务场景：风控调查 Agent 跑长链路时，prompt + 多轮工具结果常常超过 8k，最长上到 32k。Qwen-7B 原生支持 32k，理论够用，但我们想用 14B 版本做更精细的判断，14B 训练上限是 8k——直接超长效果会断崖式下降。
>
> 我们试了三个外推方案：
> - **PI（Position Interpolation）**：线性插值，长度 8k→32k 比较糙，质量丢得多
> - **NTK-aware**：调 RoPE base 频率，比 PI 好一截
> - **YaRN**：Qwen 推荐方案，加温度缩放，**32k 长度下质量基本和原生 32k 模型持平**
>
> 最后我们用 YaRN 把 14B 模型外推到 32k，作为风控调查的核心 LLM。
>
> 还有一个细节：**RoPE 的 base 频率（默认 10000）调高（如 1M）对长上下文有奇效**，这其实就是 NTK 的思路。Llama-3 默认 RoPE base = 500000，本身就是为长上下文优化的。
>
> 我对位置编码的总结：**RoPE 已经是事实标准**，业务里要不要外推、用哪种外推方案，比"用哪种编码"更值得花时间。
