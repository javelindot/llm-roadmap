# Agent的组成部分及实现方法

## 答案

一个可生产化的 Agent 通常由模型、状态、规划/策略、工具、记忆、执行循环、护栏和评测观测组成。

### 核心组件

- LLM：理解任务并生成结构化决策。
- State：保存目标、消息、当前步骤、工具结果和预算。
- Planner/Policy：决定下一步行动，可用 ReAct、状态机或显式 DAG。
- Tools：搜索、数据库、代码执行和业务 API，使用 schema 定义。
- Memory：working memory 管当前任务，long-term memory 持久化事实与经验。
- Executor：校验参数、调用工具、处理超时重试和状态转移。
- Reflection/Verifier：检查证据、结果与约束，决定修正或结束。
- Guardrails：权限、安全、人工确认、最大循环和成本限制。
- Observability/Evaluation：trace、回放、离线任务集和线上指标。

### 实现方式

简单流程优先用代码状态机：每个节点输入/输出为 typed schema，边由确定性条件控制。需要循环和长任务时使用支持 checkpoint 的图式工作流；I/O 工具异步执行，GPU 模型集中服务。

工具描述提供给模型，但代码层必须再次做 schema、权限与业务校验。有副作用的操作使用幂等键和人工确认。状态外置到数据库或事件日志，避免进程失败丢失。

### 结束条件

Agent 在任务通过验证、需要用户补充、预算耗尽或出现不可恢复错误时结束。不要把“模型输出 final”作为唯一停止条件。

评估要拆分工具选择、参数、检索、任务完成率、循环率、延迟和成本。若固定工作流能完成任务，就不必引入自由循环 Agent。
