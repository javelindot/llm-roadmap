# rerank重排是不是有比准确率更好的指标？

## 答案

有。Rerank 是排序问题，普通 Accuracy 只判断预测标签是否正确，无法充分反映“相关文档是否排在更前面”。通常使用位置敏感的排序指标。

### 1. MRR

关注第一个相关结果出现的位置：

```text
MRR = mean(1 / rank_first_relevant)
```

适合每个 query 通常只需要一个正确答案的场景，例如 FAQ 问答。

### 2. NDCG@K

NDCG 同时考虑：

- 相关文档是否排得靠前；
- 不同文档可以有多级相关性；
- 不同 query 的理想排序不同。

```text
DCG@K = sum_i (2^rel_i - 1) / log2(i + 1)
NDCG@K = DCG@K / IDCG@K
```

它是 Rerank 最常用的指标之一，适合 `0/1/2/3` 等分级相关标签。

### 3. MAP

对每次出现相关文档的位置计算 Precision，再求平均：

```text
AP = average(Precision@position_of_each_relevant_doc)
MAP = mean(AP)
```

适合一个 query 有多个相关文档的场景。

### 4. Recall@K、HitRate@K、Precision@K

- `Recall@K`：前 K 个结果覆盖了多少相关文档；
- `HitRate@K`：前 K 个结果是否至少命中一个相关文档；
- `Precision@K`：前 K 个结果中相关文档的比例。

Rerank 不会召回召回阶段完全遗漏的文档，所以最好同时看“召回上限”和“重排后的 NDCG/MRR”。

### 5. Pairwise 指标

如果模型训练目标是判断文档 A 是否比 B 更相关，可以看：

- Pairwise Accuracy；
- AUC；
- 正负文档 margin；
- 正例被排在负例前的比例。

但 pairwise 指标仍不完全等于最终 top-K 体验。

### 6. 业务和系统指标

还要加入：

- P95/P99 延迟；
- 每 query 成本；
- top-K 多样性和重复率；
- 引用正确率；
- 下游回答正确率；
- 无答案问题的拒答率。

跨境支付 RAG 中，最终要看正确政策、错误码说明和操作手册是否排在前面，以及回答是否引用了正确版本。不能只看 Reranker 自己的分类分数。

### 7. 面试总结

Rerank 比 Accuracy 更合适的指标有 NDCG@K、MRR、MAP、Recall@K 和 HitRate@K。选择取决于相关文档数量和标签粒度；生产中还要关联下游任务成功率、延迟、成本和引用正确率。
