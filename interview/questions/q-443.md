# 🔥 阐述GQA、MQA、MLA的原理分别是什么。

## 答案

三者都在解决同一个问题：标准 Multi-Head Attention 的 KV Cache 太大，长上下文和高并发推理成本高。

### 1. MQA：Multi-Query Attention

- 保留多个 Query 头。
- 所有 Query 头共享 **一组 K 和 V**。

```text
Q heads = h
KV heads = 1
```

优点是 KV Cache 最小、读取带宽低、Decode 快；缺点是共享过强，模型质量可能下降。

### 2. GQA：Grouped-Query Attention

- 多个 Query 头被分成若干组。
- 每组共享一组 K/V。

```text
Q heads = h
KV heads = g，且 1 < g < h
```

GQA 位于 MHA 和 MQA 之间，通常能保留接近 MHA 的质量，同时显著减少 KV Cache，因此被很多开源 LLM 采用。

### 3. MLA：Multi-Head Latent Attention

MLA 不只是减少 KV 头数量，而是把每个 Token 的 K/V 信息压缩成低维潜变量：

```text
c_KV = W_DKV · h
K、V 在需要时由 c_KV 投影恢复
```

推理时缓存的是低维 `c_KV`，而不是完整的多头 K/V，因此 KV Cache 可以进一步下降。为兼容 RoPE，MLA 通常会把与位置相关的部分单独处理，再与内容部分组合。

### 4. 对比

| 机制 | KV 方式 | 缓存大小 | 质量与复杂度 |
|------|---------|----------|--------------|
| MHA | 每个 Q 头独立 K/V | 最大 | 质量基线，简单 |
| GQA | 每组 Q 共享 K/V | 中等 | 常用折中 |
| MQA | 所有 Q 共享一组 K/V | 很小 | 最快，但可能损失质量 |
| MLA | 缓存低维 KV 潜变量 | 很小 | 压缩强，结构和实现更复杂 |

### 5. KV Cache 的直观关系

MHA/GQA/MQA 的缓存大致与 `num_kv_heads` 成正比；MLA 则主要与潜变量维度成正比。

**面试总结**：MQA 靠“全共享”、GQA 靠“分组共享”、MLA 靠“低秩潜变量压缩”。它们都用更少的 KV 表示换取更高的推理吞吐。
