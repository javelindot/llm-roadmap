# 用户塔输入特征列表中，哪些对跨类目效果拉动最大？上线后除 AUC 还重点监控哪些业务指标？

## 答案

对跨类目效果拉动最大的特征不能凭经验下结论，必须用消融、permutation importance 或 SHAP，并在时间切分测试集与线上 A/B 中确认。通常比静态画像更重要的是“近期跨类目行为、兴趣迁移和当前意图”。

### 1. 用户塔特征分组

**长期偏好**

- 长周期点击、购买与收藏的类目分布；
- 品牌、价格带和内容主题；
- 活跃频率、复购周期；
- 类目 embedding 聚合。

**短期行为序列**

- 最近 N 次曝光、点击、停留、加购和购买；
- 行为时间间隔与衰减；
- Session 内类目切换；
- 搜索词、当前入口和上下文。

短期序列通常对当前跨类目意图最敏感。

**跨类目迁移特征**

- 类目转移矩阵或序列模型表示；
- 同一 Session 的前后类目关系；
- 互补品、替代品和共同购买；
- 用户最近探索过但未转化的类目；
- 用户兴趣分布的熵和变化速度。

**用户约束**

- 价格敏感度；
- 地域、语言、设备和时间；
- 库存、配送或资格限制；
- 新老用户与冷启动状态。

人口属性应谨慎使用，需考虑隐私、合规与公平，不应成为跨类目推荐的主要捷径。

### 2. 如何判断拉动最大

在固定模型和数据下：

1. 逐组移除特征做 ablation；
2. 比较总体和跨类目 Recall@K/NDCG；
3. 检查头部、长尾、新用户和新类目分桶；
4. 使用时间外测试避免未来泄漏；
5. 做线上 A/B 验证真实增量；
6. 监控特征漂移、缺失率和实时延迟。

特征重要性不等于因果贡献。SHAP 高可能只是与热门类目或曝光策略强相关。

### 3. 除 AUC 外的离线指标

AUC 衡量全局正负排序，负样本大量且简单时可能很好看，却不能反映 Top-K 推荐质量。还应看：

- Recall@K、HitRate@K；
- NDCG@K、MRR；
- PR-AUC 和 LogLoss；
- 校准误差；
- 跨类目 Recall@K；
- Coverage、Novelty、Diversity；
- 长尾和冷启动分桶；
- 各召回路独占贡献。

### 4. 线上业务指标

- CTR、长点击率；
- 加购率、CVR、GMV；
- 跨类目点击/转化率；
- 新类目渗透率；
- Session 深度和留存；
- 不感兴趣、跳失、投诉与退订；
- 曝光集中度和类目公平；
- P95 延迟、超时和单请求成本。

必须同时设置 guardrail。跨类目 CTR 上升但总体 CVR、用户满意度或延迟恶化，不代表方案成功。

### 5. 冷启动与偏差

- 新用户依赖当前 Query、入口和上下文；
- 新类目依赖内容、商品属性和类目图；
- 点击日志受旧排序策略和位置影响；
- 未曝光样本不能直接当负例；
- 热门类目需要采样、去偏或重加权。

### 6. 跨境支付类比

在商户服务匹配中，可用商户历史币种、国家走廊、收付款频次、季节性和当前业务意图建模，但牌照、地域、限额和风险资格是硬过滤。线上除模型 AUC，还应监控成功率、成本、时效、覆盖、投诉和风险指标，不能只优化点击。
