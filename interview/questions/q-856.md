# 如何计算LLM参数量

## 答案

参数量等于所有可训练张量元素个数之和。对 Decoder-only Transformer，可按 Embedding、每层 Attention/FFN、Norm 和输出头计算。

设词表 \(V\)、隐藏维 \(d\)、层数 \(L\)、head dimension \(d_h\)、KV head 数 \(h_{kv}\)、FFN 中间维 \(d_{ff}\)。

### 1. Embedding 与输出头

\[
P_{\rm emb}=Vd
\]

LM Head 也是 \(Vd\)。若与输入 embedding 权重共享，只计算一次；不共享则再加 \(Vd\)。

### 2. 每层 Attention

Query 和输出投影各为 \(d^2\)，K/V 输出维为 \(h_{kv}d_h\)：

\[
P_{\rm attn}
=2d^2+2d(h_{kv}d_h)
\]

MHA 中 \(h_{kv}d_h=d\)，因此约 \(4d^2\)；GQA/MQA 的 K/V 参数更少。

### 3. FFN

普通两层 FFN：

\[
P_{\rm FFN}\approx2dd_{ff}
\]

SwiGLU 有 gate、up、down 三个矩阵：

\[
P_{\rm SwiGLU}\approx3dd_{ff}
\]

Norm、bias 和 router 参数通常相对较小，但精确计算不能忽略。

### 4. 总参数

\[
P\approx
Vd+
L(P_{\rm attn}+P_{\rm FFN}+P_{\rm norm})
+P_{\rm head}
\]

MoE 将 FFN 乘专家数 \(E\)，再加 router：

\[
P_{\rm MoE,layer}
\approx E\cdot3dd_{ff}+dE
\]

要区分总参数和每 token 激活参数。

代码中可直接核对：

```python
total = sum(p.numel() for p in model.parameters())
trainable = sum(p.numel() for p in model.parameters()
                if p.requires_grad)
```

量化改变每参数字节数，不改变参数个数；LoRA 增加 adapter 参数，不等于基座参数消失。

### 面试总结

按 `Embedding + L×(Attention+FFN+Norm) + LM Head` 计算。MHA Attention 约 \(4d^2\)，SwiGLU 约 \(3dd_{ff}\)；GQA 和 MoE 要分别处理 KV head、总专家与激活专家。
