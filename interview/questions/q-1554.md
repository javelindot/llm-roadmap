# 在LLM的推理阶段,有哪些常见的解码策略?请解释 Greedy Search, Beam Search, Top-K Sampling 和 Nucleus Sampling (Top-P) 的原理和优缺点。

## 答案

解码策略将模型每一步的词表 logits 转成下一个 token。它影响确定性、多样性、质量、延迟和 KV Cache，但不改变模型参数。

### 1. Greedy Search

每步选择概率最大的 token：

```text
token_t = argmax p(token | prefix)
```

优点：

- 简单、快、确定性较强；
- 只维护一条序列；
- 适合分类式回答、工具 JSON 和低随机任务。

缺点：

- 局部最优不保证整句概率最高；
- 易产生重复或平庸表达；
- 无法探索其他候选。

### 2. Beam Search

每步保留累计 log-prob 最高的 `B` 条部分序列，再扩展和全局剪枝。

优点：

- 比 Greedy 更接近高概率序列；
- 适合翻译、语音识别和受约束生成；
- 可结合 length penalty 与词表约束。

缺点：

- 时间和 KV Cache 随 beam size 增长；
- 开放式对话常缺乏多样性；
- 概率最高的句子不一定最有帮助；
- 需处理 EOS、长度偏好和 cache 重排。

### 3. Top-K Sampling

只保留概率最高的 `K` 个 token，重新归一化后随机采样。

优点：

- 排除极低概率 token；
- 可控制候选规模；
- 比 Greedy 更多样。

缺点：

- 固定 K 不适应分布宽窄；
- 分布很确定时可能保留多余候选；
- 分布很平时又可能截掉合理 token。

### 4. Nucleus / Top-P Sampling

按概率从高到低选择累计概率达到 `P` 的最小 token 集，再采样。

优点：

- 候选数随模型不确定性动态变化；
- 通常比固定 Top-K 更自然；
- 适合开放式生成。

缺点：

- 有随机性，复现和评测需多次运行；
- P 过高可能引入低质量 token；
- P 过低会退化为保守输出。

### 5. Temperature 与组合

采样前通常先做：

```text
p = softmax(logits / temperature)
```

低温更确定，高温更多样。Top-K、Top-P 可同时使用，实际保留二者共同允许的集合。具体过滤顺序以框架实现为准。

### 6. 选型

- 工具调用/结构化输出：Greedy 或低温 + constrained decoding；
- 翻译/受约束任务：Beam Search；
- 创意写作/多样候选：Top-P 或 Top-K；
- 推理/代码：低温并配合 verifier、测试或多样本选择。

不能用低温代替事实校验。

### 7. 跨境支付场景

金额、币种、订单号和工具参数使用低随机性与 JSON schema；客服措辞可有限采样。实时状态必须来自授权工具，不因 Beam 或 Greedy 就变成可信事实。

### 8. 面试总结

Greedy 快但局部，Beam 保留多条高概率路径但贵，Top-K 固定候选数，Top-P 按累计概率动态选候选。选择时同时考虑任务正确性、多样性、时延和验证机制。
