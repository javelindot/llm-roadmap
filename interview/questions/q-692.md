# 10.手撕 transformer 并计算 flops 和内存开销

## 答案

手写 Transformer 时先说明采用 Pre-Norm decoder block，并明确 batch $B$、序列长度 $S$、隐藏维度 $D$、头数 $H$、FFN 维度 $F$。

### 核心伪代码

```python
def block(x, causal_mask):
    h = rmsnorm(x)
    q, k, v = q_proj(h), k_proj(h), v_proj(h)
    q, k, v = split_heads(q), split_heads(k), split_heads(v)
    score = (q @ k.transpose(-1, -2)) / sqrt(head_dim)
    score = score.masked_fill(causal_mask, -inf)
    attn = softmax(score, dim=-1)
    x = x + o_proj(merge_heads(attn @ v))

    h = rmsnorm(x)
    x = x + down_proj(silu(gate_proj(h)) * up_proj(h))
    return x
```

实际实现还需处理 RoPE、padding mask、dropout 和 KV Cache。

### 前向 FLOPs

按一次乘加约 2 FLOPs：

- Q/K/V/O 投影约 $8BSD^2$；
- $QK^T$ 与 $AV$ 约 $4BS^2D$；
- SwiGLU 三个线性层约 $6BSDF$。

单层前向近似：

$$
FLOPs\approx8BSD^2+4BS^2D+6BSDF
$$

若普通两层 FFN，则 FFN 约 $4BSDF$。训练还包含反向传播，粗略常按前向的约 3 倍估算，但具体取决于算子、重计算和是否统计 embedding/head。

### 内存开销

参数量单层约为：

$$
4D^2+3DF
$$

SwiGLU 使用三个 FFN 矩阵。训练还需梯度和 Adam 一、二阶状态；混合精度下可能保留 fp32 master weights。

激活主要有 $O(BSD)$、FFN 中间 $O(BSF)$ 和朴素注意力矩阵 $O(BHS^2)$。FlashAttention 降低注意力激活存储，gradient checkpointing 以重算换显存。推理时还要计算每层 KV Cache：

$$
2BSH_{kv}d_h\times bytes
$$

最终估算必须注明 dtype、层数、是否 GQA、训练或推理，以及是否包含框架临时 buffer。
