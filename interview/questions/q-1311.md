# 了解大模型的解码策略吗，简要说一说吧

## 答案

解码策略决定如何从模型每一步输出的词表 logits 中选择下一个 token。它不改变模型参数，但会影响确定性、质量、多样性、延迟和可复现性。

### 1. Greedy Search

```text
token_t = argmax P(token | prefix)
```

- 快、确定、适合结构化输出和低随机任务；
- 容易陷入局部最优和重复；
- 温度设为 0 的 API 通常近似 greedy，但底层并不一定完全位级确定。

### 2. Beam Search

每步保留概率最高的 `B` 条序列：

- 适合翻译、语音识别等目标较明确的序列任务；
- 计算和显存随 beam width 增长；
- 开放式对话中可能生成相似、保守的候选；
- 常结合 length penalty。

### 3. Temperature

```text
P_i = softmax(logit_i / T)
```

- `T < 1`：分布更尖锐，更稳定；
- `T > 1`：分布更平，更有创造性；
- temperature 本身不是截断策略。

### 4. Top-k 与 Top-p

**Top-k**：只在概率最高的 `k` 个 token 中采样。

**Top-p / nucleus sampling**：选择累计概率达到 `p` 的最小候选集合后采样，候选数可随分布动态变化。

二者可组合，但约束过强会让输出僵化。

### 5. 其他策略

- Typical/min-p：按信息量或相对概率过滤低质量 token；
- repetition/frequency penalty：减少重复；
- constrained decoding：用 JSON Schema、CFG、正则或有限状态机限制输出；
- speculative decoding：小模型先草拟，大模型并行验证，目标是在不改变目标分布的前提下降低延迟；
- contrastive search：兼顾模型概率和表示退化；
- self-consistency：对推理题采样多条路径，再投票或验证，不是单条 token 解码算法。

### 6. 跨境支付建议

- 金额、币种、订单号、工具参数：低温度 + Schema 约束 + 字段校验；
- 客服措辞：适度 top-p，提高自然度；
- 合规调查：多候选生成后由规则和证据 verifier 校验；
- 资金操作：模型只生成调用建议，最终执行必须经过权限、幂等、限额和审批。

生产参数要用业务评测集调优，不能固定认为 `temperature=0.7` 或 `top_p=0.9` 对所有任务最好。
