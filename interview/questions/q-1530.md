# 大模型相关岗位的典型工作场景是怎样的?

## 答案

“大模型岗位”不是单一岗位，典型工作场景取决于团队更偏基础模型、算法应用、平台基础设施、数据评测还是产品落地。

### 1. 基础模型与后训练

- 预训练数据治理、tokenizer 和训练配方；
- SFT、偏好优化、RLHF/RLVR；
- 多模态、长上下文和 MoE；
- 分布式训练、稳定性和 checkpoint；
- benchmark、能力回归与安全评测。

日常工作大量是数据分析、实验设计、训练故障和消融，不只是阅读论文。

### 2. 应用算法

- RAG、搜索、Rerank 和知识治理；
- Agent 规划、工具调用、记忆和工作流；
- 文档理解、代码生成、NL2SQL；
- Prompt、SFT/PEFT、蒸馏和小模型；
- 离线评测、线上 A/B 与 bad case 闭环。

该方向最需要把模型能力转化为可验收业务指标。

### 3. 推理与训练平台

- FSDP/ZeRO/Megatron 并行；
- vLLM/SGLang/TensorRT-LLM；
- KV Cache、量化、批处理和调度；
- 模型网关、路由、弹性和多租户；
- TTFT、TPOT、goodput、成本和可观测性。

该方向更接近分布式系统、CUDA/kernel 和性能工程。

### 4. 数据与评测

- 采集、清洗、去重、标注和合成；
- 黄金集、自动 verifier 和 LLM judge 校准；
- 红队、安全、偏见和隐私；
- 线上反馈与数据飞轮；
- 版本、血缘和污染检测。

模型效果上限往往由数据与评测定义，而不是只由架构定义。

### 5. 产品与交付

- 与业务定义任务、SLO 和错误成本；
- 原型验证、灰度发布和培训运营；
- 权限、审计、合规与人机协作；
- 供应商、成本和版本管理；
- 复盘是否真的节省人力或提升收入。

### 6. 一次典型迭代

```text
业务问题
-> baseline 与评测集
-> 错误分类
-> 数据/检索/模型/系统改动
-> 离线消融
-> shadow/canary/A-B
-> 监控和数据回流
```

当前值得持续关注的方向包括可验证 RL、多模态 Agent、长时程记忆、推理效率、模型路由和高质量评测，但岗位价值最终仍以可靠交付为准。

### 7. 跨境支付场景

可能包括多语言客服、合规文档 RAG、KYC 文档理解、调查 Agent、通道路由解释和模型服务平台。实时资金状态、权限与审批必须保留在确定性业务系统。

### 8. 面试总结

大模型岗位通常是“模型、数据、系统、评测、业务”中的一到两个主方向。回答时应先明确目标岗位，再说明日常产出和成功指标，而不是笼统地说训练模型、调 Prompt。
