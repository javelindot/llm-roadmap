# 上传的文档中包含图片吗？处理图片了吗？

## 答案

> 是的，**有图片，而且我们专门处理**，分三类：
>
> **1. 流程图 / 架构图（最常见）**
> 比如"跨境支付清算流程图"——客户客服里经常问"我的款在哪个环节"。这种图必须处理，不然 RAG 召回不到关键信息。
>
> 做法：**用 Qwen2.5-VL 把图转成结构化描述**——"流程图：1. 商户提交订单 → 2. 风控审核 → 3. 清算渠道路由 → 4. 收款方银行 → 5. 完成结算"。这段文字和原图一起入库，检索时用文字 embedding，展示时附原图。
>
> **2. 表格截图**
> Word/PDF 里偶尔有"表格做成图片"的情况。我们用 **PaddleOCR + 表格结构识别**重建为 Markdown 表格，再入库。
>
> **3. 装饰图 / 截屏示例**
> 比如"App 操作截图：点击右上角设置 → 我的订单"。这类用 Qwen2.5-VL 生成场景描述："App 界面截图，显示订单列表页面，右上角有设置按钮"。
>
> **几个工程细节**：
> - **VLM 描述要短**（< 100 字），太长会稀释 embedding 信号
> - **图 + 描述一起入库**，元数据带 image_path，召回后能展示原图
> - **批处理 + 缓存**：同一张图描述生成后缓存，避免重复 VLM 调用
> - **成本控制**：VLM 单图 0.02 元，1w 张图 200 块，比 GPT-4V 便宜
>
> 没处理的：纯装饰图（logo、分割线、品牌色块）直接跳过，省成本。
>
> 上线后**含图文档的 RAG 召回率提升 14 个点**——这是个挺大的收益。

- "怎么判断哪些图要处理哪些跳过有规则。"
- "原图和文字描述的联动展示方案可以展开。"
