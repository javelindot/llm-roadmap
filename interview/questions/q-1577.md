# 后训练有哪些方式;微调有哪些方式;LoRA原理及参数量

## 答案

后训练是预训练之后让模型获得指令、偏好、推理、安全和领域能力的总称；微调是其中的参数更新方式。两者不是同义词。

### 1. 后训练方式

- **Continued Pretraining**：继续做语言模型目标，学习领域语料；
- **SFT/Instruction Tuning**：模仿高质量示范；
- **Reward Model**：学习人类或规则偏好；
- **Preference Optimization**：DPO、IPO、KTO、SimPO 等；
- **RLHF/RLVR**：PPO、GRPO 等在线 rollout 优化；
- **Distillation**：学习教师回答、logits、特征或轨迹；
- **Safety Tuning/Red Team**：拒答、边界和对抗训练；
- **Model Merging**：合并多个能力 checkpoint，但需完整回归。

目的分别是补知识、学行为、学偏好、探索策略、压缩模型和提高安全，不应为了流程完整全部使用。

### 2. 微调方式

**全参数微调**

更新全部权重，容量最大，但显存、通信、checkpoint 和遗忘风险高。

**选择性微调**

只训练顶部层、Embedding、Norm、bias 或指定模块，成本较低但容量有限。

**PEFT**

- LoRA/QLoRA；
- Adapter；
- Prompt/Prefix/P-Tuning；
- IA3、BitFit；
- DoRA、AdaLoRA 等变体。

### 3. LoRA 原理

冻结原矩阵：

```text
W in R^(d_out x d_in)
```

训练低秩增量：

```text
W' = W + (alpha / r) * B * A
A in R^(r x d_in)
B in R^(d_out x r)
```

新增参数量：

```text
r * (d_in + d_out)
```

原矩阵参数为：

```text
d_in * d_out
```

当 `r` 很小时，节省明显。`alpha` 是缩放超参数，不增加参数。

### 4. 参数量示例

若一个线性层：

```text
d_in = d_out = 4096
r = 16
```

LoRA 参数：

```text
16 * (4096 + 4096)
= 131,072
```

原权重：

```text
4096 * 4096
= 16,777,216
```

单层增量约为原矩阵的 `0.78%`。若 32 层只给 Q/V 两个投影加 LoRA，总参数约：

```text
32 * 2 * 131,072
= 8,388,608
```

实际还取决于 target modules、非方阵维度和是否训练 bias。

### 5. LoRA 优缺点

优点：

- 训练参数、梯度和 optimizer state 少；
- Adapter 小、易版本化和多任务切换；
- 可合并到基座，通常不增加推理层数；
- 适合领域 SFT 与偏好优化。

缺点：

- 受 rank 和 target modules 限制；
- 基座能力和错误仍保留；
- 多 Adapter 管理与 base 升级复杂；
- 高 rank/全模块后成本优势缩小；
- 不适合用来存储频繁变化的事实。

QLoRA 是冻结 4-bit 基座并训练高精度 LoRA，不是把 LoRA 本身量化成 4-bit。

### 6. 跨境支付场景

客服格式、工具 schema 和领域表达可用 LoRA；规则、汇率、订单和制裁名单使用 RAG/工具。高风险行为还需要后端权限与审批。

### 7. 面试总结

后训练回答“预训练后还做什么”，微调回答“更新哪些参数”。LoRA 用低秩 `BA` 近似权重增量，参数量为 `r(d_in+d_out)`，是企业领域适配的常用起点。
