# R1在SFT阶段进行冷启动的目的是什么？

## 答案

DeepSeek-R1 的冷启动 SFT 主要不是给 Base Model 灌更多知识，而是给后续推理 RL 提供可读、稳定且符合格式的起点，解决 R1-Zero 虽能涌现推理但可读性和语言一致性不足的问题。

### 1. R1-Zero 暴露的问题

从 Base Model 直接做 RL 可以出现：

- 更长推理；
- 自我反思；
- 验证和策略切换。

但也出现：

- 中英文等语言混杂；
- 推理格式混乱；
- 可读性差；
- 最终答案边界不清；
- 用户体验不稳定。

### 2. 冷启动 SFT 的目的

**建立推理格式**

让模型学会清晰地区分推理与答案，并按可解析格式输出。

**改善可读性**

高质量样本展示连贯、完整、适度的推理表达，而不是无组织 token。

**降低 RL 初始方差**

初始策略已经能生成基本有效候选，组内奖励更有区分度，减少大量无效 rollout。

**注入人类偏好**

在只看结果正确性的奖励之外，先加入语言一致、解释清晰和安全边界。

**提供更好的探索起点**

RL 在已有推理模式上扩大和优化，而不是从随机格式中寻找。

### 3. 数据特征

论文描述的冷启动数据强调：

- 长 CoT；
- 反思与验证；
- 可读格式；
- 正确最终答案；
- 少量高质量，而非海量低质复制。

具体数据量和配方未完全公开，面试中不应把非官方猜测说成事实。

### 4. 后续流程

```text
DeepSeek-V3-Base
-> Cold-start SFT
-> Reasoning RL
-> Rejection Sampling + General SFT
-> Final RL
-> DeepSeek-R1
```

冷启动后仍需 RL 提升推理上限，也需要后续通用 SFT 和安全对齐恢复非推理能力。

### 5. 跨境支付边界

可以用专家轨迹教调查报告的证据组织和工具顺序，但不应训练模型输出未经核验的隐藏 CoT 或直接做 AML 最终定性。生产更适合输出可审计的“证据、规则、结论草稿”，高风险结论转人工。
