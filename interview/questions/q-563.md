# 用bert做分类任务时，输出是怎么处理的？

## 答案

用 BERT 做分类时，先得到序列的上下文化表示，再根据任务类型选择 Pooling 和分类头。

### 1. 单句/句对分类

输入通常为：

```text
[CLS] sentence A [SEP] sentence B [SEP]
```

BERT 输出：

```text
last_hidden_state: [batch, seq_len, hidden]
```

常见做法是取 `[CLS]` 对应向量：

```text
h_cls = H[:, 0, :]
logits = Linear(Dropout(h_cls))
```

也可以使用 Mean Pooling、Max Pooling 或 Attention Pooling。

### 2. Loss

- 单标签多分类：`CrossEntropyLoss(logits, label)`，模型输出不需要先手动 Softmax。
- 多标签分类：每类独立，使用 `BCEWithLogitsLoss`。
- 回归任务：使用 MSE 等回归损失。

推理时：

- 多分类对 Logits 做 Softmax 后取最大类别。
- 多标签对每类做 Sigmoid，再按阈值判断。

### 3. Token 级分类

NER 等任务不是取 `[CLS]`，而是对每个 Token 的向量接分类层：

```text
[batch, seq_len, hidden]
→ Linear
→ [batch, seq_len, num_labels]
```

需要忽略 Padding 和 Subword 中不参与标注的位置，必要时接 CRF 建模标签转移。

### 4. 实践注意

- 类别不平衡可用类别权重、重采样或 Focal Loss。
- 长文本要处理截断、滑窗或层级编码。
- 只用 `[CLS]` 不一定最优，应在验证集比较 Pooling。
- 预训练模型和分类头可使用不同学习率。

**面试总结**：BERT 负责把文本编码成上下文向量，分类头负责把句级或 Token 级表示映射成任务 Logits。
