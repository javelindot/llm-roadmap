# 🔥 bn和ln

## 答案

BN（Batch Normalization）和 LN（Layer Normalization）都通过归一化改善优化，但**统计维度不同**。

### 1. BN

对形状为 `[B, C, H, W]` 的卷积特征，BN 通常按通道统计，在 `B、H、W` 维计算均值和方差：

\[
\hat{x}=\frac{x-\mu_B}{\sqrt{\sigma_B^2+\epsilon}},
\qquad y=\gamma\hat{x}+\beta
\]

训练时使用当前 batch 统计量并更新滑动均值、方差；推理时使用滑动统计量。优点是 CNN 中效果成熟，缺点是依赖 batch，大分布差异、小 batch、变长序列和分布式训练时处理较麻烦。

### 2. LN

LN 对单个样本的特征维归一化，不依赖其他样本。例如 Transformer 中对每个 token 的隐藏维 \(d\) 计算统计量。训练与推理行为一致，适合小 batch、变长序列和自回归模型。

### 3. 核心区别

- BN 让同一特征在一批样本间标准化，统计量依赖 batch；
- LN 让同一样本内部的特征标准化，不依赖 batch；
- CNN 常见 BN，Transformer 通常使用 LN 或 RMSNorm；
- BN 推理需要运行统计量，LN 不需要；
- LN 不等于解决所有数值问题，Pre-Norm、残差缩放和优化器配置同样重要。

RMSNorm 可视为 LN 的简化变体，只按均方根缩放而不减均值，LLaMA 等模型广泛使用。

### 跨境支付场景

支付模型常面对小批量、多语言、变长文本和在线单请求推理，Transformer 使用 LN/RMSNorm 更自然。若用 CNN 处理票据图像且 batch 足够稳定，BN 仍可能合适；跨机构数据分布差异大时要特别检查 BN 运行统计量漂移。

### 面试总结

BN 按 batch 维统计，依赖批次且训练、推理统计方式不同；LN 按单样本特征维统计，不依赖 batch，因而更适合 Transformer。选择的关键是张量维度、batch 大小和线上分布，而不是简单判断谁更好。
