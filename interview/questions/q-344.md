# LangChain和LangGraph的区别？

## 答案

**一句话**：LangChain 更像大模型应用开发工具箱，LangGraph 更像面向复杂 Agent 流程的图编排和状态管理框架。

### 1. LangChain 是什么

LangChain 提供很多大模型应用组件，比如：

- Prompt 模板。
- 模型封装。
- 文档加载器。
- Retriever。
- Tool。
- Chain。
- Agent。
- Memory。

它适合快速搭建 RAG、工具调用、简单 Agent 和各种 LLM 应用。

### 2. LangGraph 是什么

LangGraph 更关注复杂流程编排。它用图结构表达：

- 节点。
- 边。
- 条件路由。
- 状态传递。
- 循环。
- checkpoint。
- 人工介入。

它适合多步骤、多分支、可恢复、可观测的 Agent 工作流。

### 3. 核心区别

- LangChain 偏组件化开发。
- LangGraph 偏流程和状态编排。
- LangChain 的 Chain 更适合线性流程。
- LangGraph 更适合有分支、循环、工具、人工节点的复杂流程。
- LangGraph 的状态更显式，便于恢复和调试。

### 4. 结合跨境支付项目怎么说（面试口径）

> 在跨境支付客服项目里，如果只是做一个 FAQ RAG，用 LangChain 的文档加载、向量检索、prompt、model chain 就够了。但如果要处理“付款失败原因分析”这种多步骤任务，LangGraph 更合适。
>
> 因为付款失败可能要先查订单，再根据状态查通道回执、风控原因或合规限制。不同结果走不同分支，有些情况要追问用户订单号，有些情况要转人工。这种流程用图表达会更清晰。
>
> 所以我的理解是：LangChain 提供基础积木，LangGraph 负责把这些积木组织成稳定可控的业务流程。
