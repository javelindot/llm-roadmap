# RAG 知识查询、Agent 工具执行过程中出现错误该如何处理？

## 答案

> 错误分两类——**可恢复（重试 / 降级）** 和**不可恢复（转人工）**，对应不同处理策略。
>
> **RAG 查询常见错误**：
>
> | 错误类型 | 表现 | 处理 |
> |---------|------|------|
> | 召回为空 | top-K 都 < threshold | LLM 回 "不知道" / 转人工 |
> | embedding 超时 | 模型服务挂 | 重试 + 降级到关键词 |
> | 向量库不可用 | Milvus 5xx | 切备份集群 / 走纯 BM25 |
> | rerank 失败 | 模型 OOM | 跳过 rerank 用原序 |
> | 生成幻觉 | LLM 编内容 | 引用强约束 + verification |
>
> **Agent 工具调用常见错误**：
>
> | 错误类型 | 表现 | 处理 |
> |---------|------|------|
> | 工具调用超时 | API 504 | 重试 1 次 → 切备用 |
> | 参数错误 | 422 / 校验失败 | LLM 重新生成参数 |
> | 权限错 | 403 | 不重试，告警 |
> | 业务空结果 | 200 但 data=[] | 告知 LLM "无数据"，让它换工具 |
> | 工具结果异常 | 数据明显错 | LLM 自检 + 备用工具 |
>
> **我们的统一错误处理 pipeline**：
>
> **1. 工具调用层封装**
> ```python
> async def call_tool_with_retry(tool, args, max_retry=2):
>     for attempt in range(max_retry + 1):
>         try:
>             result = await tool.invoke(args)
>             if not result_looks_valid(result):
>                 raise InvalidResultError()
>             return result
>         except (TimeoutError, NetworkError) as e:
>             if attempt < max_retry:
>                 await asyncio.sleep(2 ** attempt)  # 指数退避
>                 continue
>             return ToolFailureResult(error=str(e), fallback=True)
> ```
>
> **2. Agent 层的错误反馈**
> ```python
> # 工具返回错误时，把 error 作为 observation 喂给 Agent
> # 让 Agent 自己决定换工具 / 调整参数 / 转人工
> observation = f"工具调用失败: {error}. 请重新规划。"
> ```
>
> **3. 全局兜底**
> - max_iterations 触顶 → 强转人工
> - 总耗时超 30s → 强终止 + 返回部分结果
> - 关键工具失败超 3 次 → 直接返回标准答辞
>
> **优化策略**：
>
> **1. 召回为空的优化**
> - 降低 rerank threshold
> - 启用 RAG Fusion 多 query 增召回
> - 触发 step-back 抽象查询
> - 都不行 → "请人工咨询"
>
> **2. 工具参数错误优化**
> - 给 LLM 返回 schema validation error 详情
> - "参数 amount 必须是数字，你传了 'abc'，请重新生成"
> - LLM 通常一次就能改对
>
> **3. 幻觉防护**
> - 生成 prompt 强制要求"必须从 [context] 中引用，否则说不知道"
> - 后置用 NLI 模型校验答案是否被检索支持
> - 不支持的句子标红 + 转人工
>
> **4. 降级链**
> ```
> 主模型 → 备模型 → 模板回答 → 转人工
> 向量检索 → BM25 → 元数据规则匹配 → "不知道"
> ```
>
> **5. 监控 + 告警**
> - 所有错误进 OpenTelemetry trace
> - 失败率超 5% → 企微告警
> - 每日复盘 top 10 失败类型
>
> 一句话：**错误不可避免，但要做到"业务无损 + 用户体验降级"**——宁可慢一点、宁可说不知道，也不能让用户拿到错答案。

**🪝 顺嘴带出去的钩子**：
- "工具调用幂等设计可以聊。"
- "NLI 校验幻觉的具体实现可以展开。"
- "失败 case 回流飞轮我们怎么做可以讲讲。"
