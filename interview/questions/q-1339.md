# bert和t5的区别

## 答案

BERT 是 Encoder-only 的双向表示模型，T5 是 Encoder-Decoder 的 Text-to-Text 生成模型。BERT 更适合理解、分类和抽取；T5 更适合输入到输出的序列转换，如翻译、摘要和生成式问答。

### 1. 架构

| 维度 | BERT | T5 |
|------|------|----|
| 主体 | Transformer Encoder | Encoder + Decoder |
| Encoder Attention | 双向 | 双向 |
| Decoder | 无 | Causal Self-Attention |
| Cross-Attention | 无 | Decoder 读取 Encoder |
| 输出 | hidden representation/task head | 自回归文本 |

### 2. 预训练目标

**BERT**

- Masked Language Modeling；
- 原版还有 Next Sentence Prediction；
- 预测被遮盖 token。

**T5**

- Span Corruption；
- 连续片段替换为 sentinel token；
- Decoder 生成被删除的 span；
- 所有任务统一成文本到文本。

### 3. 下游任务

**BERT**

- 分类、NER、抽取式 QA；
- 句对匹配；
- Embedding/Reranker；
- 一次前向，延迟低。

**T5**

- 翻译、摘要、改写；
- 生成式 QA；
- 结构化文本转换；
- 需要 Decoder 逐 token 生成。

### 4. 位置和结构细节

- BERT 原版使用可学习绝对位置和 Post-LayerNorm；
- T5 使用相对位置 bias，采用 Encoder-Decoder；
- T5 常共享词嵌入与 LM Head；
- 两者都不是当前通用聊天模型的主流 Decoder-only 架构，但仍有明确用途。

### 5. 成本与选型

同规模下，BERT 做分类/抽取通常更快；T5 为生成任务付出 Encoder + Decoder 成本，但输入可双向编码，适合 seq2seq。

跨境支付中：

- BERT：工单路由、实体抽取、召回/Rerank；
- T5：早期多语言翻译、短摘要、字段到文本转换；
- 复杂多轮客服和 Agent：通常选择现代 Decoder-only Instruct 模型。

**面试总结**：BERT 输出“表示”，T5 输出“序列”；选择取决于任务是否需要生成。
