# 实习部分继续预训练的数据大小？

## 答案

这道题必须给真实数据，不能用行业规模替代自己的项目。回答应同时包含文档数、原始字节、清洗后 token、训练步数和数据配比。

### 回答模板

> 继续预训练使用了约[文档数]篇、清洗后[GB]、经目标 tokenizer 统计约[token 数]。其中[领域数据]占[比例]，[通用回放数据]占[比例]，用于降低灾难性遗忘。去重、质量过滤和 PII 清洗后保留率为[比例]。模型实际训练了[步数/epoch]，因此总训练 token 约为[数值]，不是简单等于语料 token 数。

总训练 token 可按下式核对：

\[
\text{tokens}
=\text{global batch size}
\times \text{sequence length}
\times \text{training steps}
\]

若使用 sequence packing，应报告有效 token 利用率；若同一数据重复多轮，要区分 unique tokens 和 seen tokens。

### 如何判断数据量是否合理

- 先做小规模 scaling experiment，观察 held-out perplexity 和下游任务增益；
- 质量、覆盖和混合比例通常比盲目增加低质 token 更重要；
- 用通用能力回归集监控遗忘；
- 领域语料不足时可使用合成数据，但要去重并人工抽检；
- 训练预算应结合模型参数量、任务偏移和收益饱和点。

跨境支付继续预训练可学习稳定的领域语言与流程，但易变化的费率、牌照和通道规则更适合 RAG。数据还必须脱敏、按地区合法处理，并防止把测试期未来文档混入训练集。

### 面试总结

回答数据大小时给出“清洗后 token、领域/通用配比、训练步数、unique/seen tokens”完整口径。数据量不是越大越好，应以领域验证集增益、通用能力回归和成本曲线决定。
