# transformer的normalization，为什么用layernorm不用其他的；介绍RMSnorm

## 答案

Transformer 通常用 LayerNorm/RMSNorm，因为它们在单个 token 的 hidden 维上归一化，不依赖 batch 统计，适合变长序列、小 batch、自回归 Decode 和分布式训练。BatchNorm 在这些条件下统计不稳定且训练/推理行为不同。

### 1. LayerNorm

对 token hidden vector `x ∈ R^d`：

```text
mu = mean(x)
var = mean((x - mu)^2)

LN(x)
= gamma * (x - mu)
  / sqrt(var + eps)
  + beta
```

每个 token 独立计算，不受同 batch 其他样本和 padding 长度影响。

### 2. 为什么不用 BatchNorm

BatchNorm 跨 batch 统计：

- NLP batch 和序列长度动态；
- 大模型训练每卡 micro-batch 往往很小；
- padding 会干扰统计；
- 多卡需要同步统计，通信昂贵；
- 自回归推理一次可能只有一个 token；
- train/eval 使用不同统计，增加分布偏差。

InstanceNorm/GroupNorm 更适合视觉张量，对通用 token hidden state 没有 LayerNorm 自然。

### 3. RMSNorm

RMSNorm 不减均值：

```text
RMS(x)
= sqrt(mean(x^2) + eps)

RMSNorm(x)
= gamma * x / RMS(x)
```

通常没有 bias。它只控制尺度，计算和实现更简单；Llama、Qwen 等现代 LLM 广泛使用。

### 4. Pre-Norm 与 Post-Norm

原始 Transformer：

```text
y = LN(x + Sublayer(x))
```

现代 LLM 常用：

```text
y = x + Sublayer(RMSNorm(x))
```

Pre-Norm 给残差和梯度提供更直接路径，深层训练更稳定；Post-Norm 可能需要更严格的初始化、warmup 或 DeepNorm 类方案。

### 5. 工程注意

- Norm 的 reduction 常保留 FP32 累积，防止低精度误差；
- `eps` 过小可能导致数值问题；
- weight decay 通常不作用于 norm scale/bias；
- RMSNorm 节省的只是局部计算，端到端收益要压测；
- 不要在预训练模型中随意替换 Norm，权重分布不兼容。

**面试总结**：Transformer 选 LayerNorm/RMSNorm 的关键不是“效果一定最高”，而是它们与 batch 无关，训练和逐 token 推理一致。
