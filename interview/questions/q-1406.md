# 深度学习相关：transformer结构，lstm和rnn区别

## 答案

这道题包含 Transformer 结构，以及普通 RNN 与 LSTM 的区别。

### 1. Transformer 结构

原始 Transformer 是 Encoder-Decoder 架构。

Encoder 每层包括：

```text
Multi-Head Self-Attention
-> Residual + LayerNorm
-> Feed Forward Network
-> Residual + LayerNorm
```

Decoder 每层包括：

```text
Masked Self-Attention
-> Cross-Attention
-> Feed Forward Network
```

现代 GPT、LLaMA、Qwen 等生成模型通常使用 Decoder-only 架构，使用 causal mask 做 next-token prediction。

注意力公式：

```text
Attention(Q,K,V) =
softmax(QK^T / sqrt(d_k)) V
```

Multi-Head Attention 让模型在不同子空间建模语义、位置和依赖关系。FFN 对每个 token 独立做非线性变换，残差和归一化帮助深层训练。

### 2. RNN

普通 RNN 按时间递归更新隐藏状态：

```text
h_t = tanh(W_x*x_t + W_h*h_(t-1) + b)
```

优点是结构简单、天然处理序列；缺点是：

- 必须按时间顺序计算，难并行；
- 长序列反向传播容易梯度消失或爆炸；
- 长期依赖能力弱。

### 3. LSTM

LSTM 在 RNN 基础上增加 cell state 和门控机制：

```text
forget gate
input gate
output gate
```

Cell state 提供较直接的信息与梯度通路，门控决定保留、写入和输出哪些信息，因此比普通 RNN 更能建模长期依赖。

### 4. LSTM 和 RNN 对比

| 维度 | RNN | LSTM |
|---|---|---|
| 状态 | hidden state | hidden + cell state |
| 门控 | 无 | 遗忘、输入、输出门 |
| 长期依赖 | 较弱 | 更强 |
| 参数与计算 | 少 | 更多 |
| 梯度稳定性 | 较差 | 较好 |

LSTM 只是缓解梯度问题，并非彻底消除。

### 5. Transformer 与 RNN/LSTM

- Transformer 训练时可对序列并行计算；
- Attention 更容易建模长距离依赖；
- 复杂度对序列长度通常是二次的，长上下文成本高；
- RNN/LSTM 的时间复杂度线性，但递归依赖限制并行；
- 流式、小模型和时序任务中 LSTM 仍有价值。

### 6. 跨境支付场景

- LSTM 可用于连续交易序列、设备行为和时序风险建模；
- Transformer 可处理长交易历史、客服对话、文档和多模态材料；
- 在线风控要结合延迟和增量更新需求，不能因为 Transformer 更新就无条件替换轻量时序模型。

### 7. 面试总结

普通 RNN 用单一隐藏状态递归，LSTM 通过 cell state 和门控增强长期记忆；Transformer 用自注意力直接连接任意位置并提高训练并行度。选型要结合序列长度、实时性、数据量和部署成本。
