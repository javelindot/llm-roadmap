# 为什么Transformer使用Layer Normalization而不是Batch Normalization？

## 答案

Transformer 更常使用 LayerNorm，核心原因是它在 **单个 Token 的隐藏维度内部**归一化，不依赖 Batch 中其他样本，更适合变长序列和自回归推理。

### 1. 归一化范围不同

假设输入为 `[batch, sequence, hidden]`：

- LayerNorm 对每个 Token 的 `hidden` 维计算均值和方差。
- BatchNorm 对 Batch 中同一特征收集统计量。

### 2. Transformer 不适合依赖 Batch 统计

- 不同样本序列长度不同，Padding 比例也不同。
- Batch Size 会因显存和长度动态变化。
- 自回归在线推理时 Batch 可能只有 1。
- 训练和推理的 Batch 分布可能完全不同。

LayerNorm 对每个样本独立计算，训练和推理行为一致。

### 3. 序列语义更匹配 LayerNorm

Transformer 的一个 Token 表示是多个隐藏特征组成的向量。LayerNorm 对这些特征统一缩放，有助于稳定每层残差流；BatchNorm 则会把不同句子、不同位置的 Token 统计混在一起，容易受 Padding 和样本组成影响。

### 4. 与训练稳定性的关系

现代 Transformer 常使用 Pre-LN：

```text
x = x + Attention(Norm(x))
x = x + FFN(Norm(x))
```

它能改善深层网络中的梯度传播。很多 LLM 进一步使用 RMSNorm，只按均方根缩放以减少计算。

### 5. 是否绝对不能用 BatchNorm

不是。某些视觉或特定结构可以使用 BatchNorm，但对通用变长序列和生成式 Transformer，LayerNorm/RMSNorm 更稳定、部署更简单。

**面试总结**：Transformer 选择 LayerNorm，不是因为 BN 本身不好，而是因为序列任务的 Batch 和长度不稳定，而 LN 与 Batch 无关。
