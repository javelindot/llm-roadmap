# 介绍下vllm的技术点。

## 答案

vLLM 是面向 LLM 推理与服务的高吞吐引擎，核心不是某一个 kernel，而是 KV Cache 管理、动态调度、模型执行和分布式服务的组合。

### 1. PagedAttention / Paged KV Cache

把每个请求的 KV Cache 切成固定 token block，用 block table 映射到非连续物理显存。它减少预留和外部碎片，支持请求动态增长、并行采样和前缀块复用。它不减少有效 KV 元素本身。

### 2. Continuous Batching

请求可在每个调度步动态进入和退出 batch，不必等待同批最长请求完成。这样提高 GPU 利用率和吞吐，但 scheduler 仍需权衡长 prefill 与 decode 的公平性。

### 3. Prefill/Decode 优化

- chunked prefill 将长 prompt 拆块，与 decode 混合调度；
- automatic prefix caching 复用相同 system prompt、工具 schema 或文档前缀；
- speculative decoding 使用 n-gram、EAGLE、MTP 等草稿方案，但收益依赖接受率和负载；
- disaggregated prefill/decode 可分别扩容计算密集与带宽密集阶段。

### 4. 执行与模型能力

- FlashAttention、FlashInfer、FlashMLA 和优化 GEMM/MoE kernel；
- CUDA/HIP Graph、`torch.compile` 和算子融合；
- FP8、INT8、INT4、GPTQ、AWQ 与量化 KV；
- Tensor/Pipeline/Data/Expert/Context Parallel；
- Multi-LoRA、多模态、Embedding、rerank/reward 和 structured output；
- OpenAI-compatible API、流式输出与监控。

截至 2026 年的能力可参考 [vLLM 官方文档](https://docs.vllm.ai/en/stable/)。功能存在不代表默认配置最优，必须按模型、GPU、prompt/output 长度和并发压测 TTFT、ITL、吞吐与显存。

跨境支付服务还需在 vLLM 外层实现鉴权、租户隔离、PII 清洗、限流、超时和审计。

### 面试总结

vLLM 通过分页 KV 管理和 continuous batching 提升并发，再结合 chunked prefill、prefix cache、量化、优化 kernel 和分布式并行构建完整服务。选型要看端到端 SLO，而不是只看离线 tokens/s。
