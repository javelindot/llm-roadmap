# 为什么Transformer选用LayerNorm而不是BatchNorm？

## 答案

**一句话**：Transformer 用 LayerNorm 而不是 BatchNorm，主要是因为 NLP 序列长度和 batch 不稳定，LayerNorm 对单个样本内部归一化，更适合序列模型和自回归生成。

### 核心区别

- BatchNorm：跨 batch 维度统计均值方差，依赖 batch 大小。
- LayerNorm：对单个样本的 hidden 维度归一化，不依赖 batch。

Transformer 常用于变长序列、小 batch、推理逐 token 生成，BatchNorm 的 batch 统计不稳定；LayerNorm 在训练和推理时行为一致，更适合。

### 结合跨境支付项目怎么说（面试口径）

> 在跨境支付客服推理里，请求是一个个来的，batch 大小和序列长度都不稳定。LayerNorm 不依赖 batch 统计，因此更适合线上生成式模型。
>
> 这类底层设计保证了模型在不同请求长度、不同并发情况下更稳定，但业务上我们仍然要控制上下文长度和推理延迟。

**钩子**：可以引到“变长序列、小 batch、训练推理一致、自回归生成”。
