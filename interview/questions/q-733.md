# sparse特征和dense特征的区别

## 答案

Sparse 和 dense 特征既可描述原始输入，也可描述存储表示。核心区别是维度、非零比例和语义结构。

### Sparse 特征

高维但大部分为 0，例如 one-hot/multi-hot 的用户 ID、商品 ID、词袋、类别交叉特征。它们通常没有连续距离语义，“ID 10”和“ID 11”并不更相似。

优点是可解释、精确记忆和处理大词表；缺点是维度巨大、泛化到未见组合弱。实现上常用 CSR/COO、哈希和 embedding lookup，避免构造完整稠密向量。

### Dense 特征

低维且大多数位置有值，例如年龄、价格等连续数值，或由 embedding/神经网络学得的用户、文本和图像向量。维度间可能共同编码语义，适合相似度与深度模型计算。

Dense 表示内存连续、矩阵乘法高效，但可解释性低，也可能把不同对象压得过近。

### 推荐系统中的结合

类别 sparse 特征通常先通过 embedding table 转成 dense vector；连续特征经归一化、分桶或小网络处理，再与 embedding 拼接进入 Wide&Deep、DeepFM 等模型。Wide 部分记忆高频交叉，Deep 部分学习泛化。

还要区分“稀疏特征”与“稀疏 embedding”：原输入 one-hot 很稀疏，但 lookup 后的 embedding 是 dense。选型需考虑词表、OOV、哈希冲突、更新频率、存储和线上延迟。
