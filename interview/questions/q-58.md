# LLMs归一化方式

## 答案

**一句话**：LLM 常用的归一化方式主要是 LayerNorm 和 RMSNorm，位置上又分 Pre-LN 和 Post-LN。

### 核心回答
- **LayerNorm**：对每个 token 的 hidden 维度做归一化，稳定训练。
- **RMSNorm**：只用均方根做缩放，不减均值，计算更简单，很多 LLaMA 系模型采用。
- **Post-LN**：归一化放在残差之后，早期 Transformer 常用，但深层训练不稳定。
- **Pre-LN**：归一化放在 attention/MLP 之前，梯度更稳定，现代 LLM 常用。

归一化的作用：
- 稳定激活分布。
- 缓解梯度不稳定。
- 允许更深网络训练。
- 提高收敛速度。

### 1. LayerNorm 和 RMSNorm 的区别

LayerNorm 会对 hidden 维度做均值和方差归一化，公式上会减均值再除标准差。RMSNorm 简化了这个过程，只使用均方根做缩放，不减均值。

RMSNorm 的优势是：

- 计算更简单。
- 训练更高效。
- 在很多现代 LLM 中效果稳定。

因此 LLaMA、Qwen 等模型常见 RMSNorm，而 BERT/GPT 早期模型更多使用 LayerNorm。

### 2. Pre-LN 和 Post-LN

Post-LN 是把归一化放在残差之后，早期 Transformer 常用；Pre-LN 是先归一化再进入 attention/MLP，深层网络中梯度更稳定。

现代 LLM 通常偏向 Pre-LN 或类似结构，因为模型层数很深，训练稳定性非常重要。

### 3. 面试总结

归一化的核心作用是稳定训练，让深层模型更容易收敛。业务微调时通常不改 Norm 结构，但理解它有助于判断模型架构和训练稳定性。

### 4. 结合跨境支付项目怎么说（面试口径）

> 这类问题我会回答到工程感：归一化不是业务侧经常直接改的点，但它影响模型训练稳定性。比如我们做支付领域 LoRA 或 DPO 时，一般不会去动底层 Norm 结构，而是选择一个已经稳定验证过的基座模型。
>
> 真正调优时，我更关注学习率、batch、LoRA rank、warmup 和数据质量。因为如果这些没控制好，模型可能出现回答模板化、拒答率升高、业务术语理解变差。
>
> 所以我的理解是：Norm 是基座模型训练稳定性的底层设计，业务微调时要尊重原架构，尽量用小步增量去适配支付场景。

**钩子**：微调业务模型时通常不改 Norm，重点保证训练数据和超参稳定。
