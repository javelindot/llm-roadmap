# R1的 MLA是如何节约 KV cache的?

## 答案

DeepSeek-R1 的基础架构来自 DeepSeek-V3，MLA（Multi-head Latent Attention）最早在 DeepSeek-V2 中系统公开。它通过缓存低维 latent，而不是每个 head 的完整 K/V，降低 KV Cache。

### 1. 标准 MHA

每个 token、每层需缓存：

\[
K\in\mathbb R^{h\times d_h},
\qquad
V\in\mathbb R^{h\times d_h}
\]

缓存随 \(2h d_h\) 增长。

### 2. MLA 压缩

先下投影：

\[
c_t^{KV}=W^{DKV}h_t
\]

其中 latent 维度 \(d_c\ll h d_h\)。K/V 由它上投影得到：

\[
k_t^C=W^{UK}c_t^{KV},\qquad
v_t=W^{UV}c_t^{KV}
\]

推理时缓存 \(c_t^{KV}\)，并通过矩阵吸收将 query 投影与 K 的上投影合并，避免显式恢复完整 K。

### 3. RoPE 解耦

RoPE 与低秩矩阵吸收不完全兼容，因此 Q/K 被分成：

- 内容部分：由 latent 生成并可吸收；
- 较小的 decoupled RoPE key/query：单独缓存/计算位置关系。

每 token 缓存大致变为：

\[
d_c+d_{\rm rope}
\]

而非标准 MHA 的 \(2h d_h\)，显著降低显存与 decode HBM 读取。

### 4. 边界

MLA 是模型架构能力，不能给普通 MHA 模型靠服务配置直接开启。其训练、kernel 和 TP 分片更复杂；低并发或计算瓶颈场景收益也可能不同。

### 面试总结

MLA 把各头 K/V 压成共享低维 latent，只缓存 latent 与小型 RoPE key，并用矩阵吸收完成注意力计算。它主要通过减少 KV 容量和带宽提升长上下文并发。
