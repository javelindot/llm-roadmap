# 谈谈 ViT 的图像是怎么输入模型的，class token 的作用是什么？

## 答案

**一句话**：图像先**切 patch → Linear 投影 → 加 [CLS] + 位置编码 → 喂 Transformer Encoder**；**[CLS] token 是"汇聚全局信息的虚拟 token"，最终用它的输出做分类**。

### 1. 图像输入流程
```
图像 (H, W, 3)
  → 切 N = (H/P) × (W/P) 个 patch，每个 P×P×3
  → 展平 + Linear 投影到 d 维 → N 个 patch token
  → 前面加 [CLS] token → N+1 个 token
  → 加 1D 位置编码（learnable）
  → Transformer Encoder
```

### 2. Patch Embedding 等价实现
本质等价于一个 conv2d：
```python
self.proj = nn.Conv2d(3, d, kernel_size=P, stride=P)
# 输入 (B, 3, H, W) → 输出 (B, d, H/P, W/P)
# 再 reshape 成 (B, N, d)
```

### 3. [CLS] Token 是什么
- 一个**可学习的 d 维向量**
- 在 token 序列**最前面**插入
- 不对应任何图像区域，是"虚拟 token"
- 经过 attention 后，**汇聚全图信息**

### 4. [CLS] 的作用
- **全局表征**：经过 N 层 attention，[CLS] attend 所有 patch token，提取全局语义
- **分类输出**：最后一层 [CLS] 的 hidden state 接 MLP 做分类
- **预训练目标**：BERT 风格，[CLS] 是任务接口

### 5. 为什么不用 patch token 平均池化
- 平均池化 OK 但有信息丢失
- [CLS] 可以**学习关注哪些 patch**，比固定平均更灵活
- 实验表明 ViT 用 [CLS] 比 mean pool 略好
- 但 DeiT 等也证明 mean pool 不差太多

### 6. 位置编码细节
- ViT 用**1D learnable position embedding**（不是 2D）
- 长度 = N+1（含 [CLS]）
- 训练时学到自然的 2D 结构
- 长宽不固定时要做 bicubic interpolation 适配

### 7. 推理时 [CLS] 怎么用
| 任务 | 取 token |
|------|---------|
| 分类 | [CLS] |
| 分割 | 所有 patch token |
| 检测 | 所有 patch token |
| 多模态接 LLM | **所有 patch token 当 image tokens** |

### 8. ViT 变体里的 [CLS] 演化
- **DINOv2**：保留 [CLS]，用作全局表征
- **MAE**：训练时遮蔽，重建用 patch
- **SigLIP**：去掉 [CLS]，用 mean pool
- **LLaVA / Qwen-VL**：不用 [CLS]，把所有 patch 当 token 喂 LLM

### 9. 一句话总结
**图像 = 切 patch → 投影 → 加 [CLS] → 加位置 → 进 Encoder**；**[CLS] 是"虚拟汇聚 token"做分类输出**；多模态场景常不用 [CLS]，把所有 patch 喂 LLM。

## 结合项目谈（口语版 · 约 25 秒）

> 这道题我们 KYB 多模态识别里实打实涉及。分享几个工程细节。
>
> **patch 输入**：营业执照原图 1024×768，缩放到 448×448 喂 ViT-L/14，切成 32×32 = **1024 个 patch token**。这就是为什么多模态 LLM 上下文很容易吃满——一张图等于 1000 个 token。
>
> **[CLS] 的去留**：分类任务我们用 [CLS]，多模态接 LLM 时我们**直接用所有 patch tokens 当 image tokens 喂 LLM**——这是 LLaVA 风格的做法。原因是 [CLS] 只有一个 token 信息量太少，LLM 看不清细节；用所有 patch 让 LLM 自己 attend 重要区域。
>
> **位置编码外推**：ViT 默认 224×224，196 个 patch 学到 196 个位置 embedding。我们要用 448×448，1024 个 patch，position 不够用。解法是 **bicubic interpolation 把 14×14 positions 放大到 32×32**——这是 ViT 输入分辨率改变时的标准操作。
>
> **冻不冻 ViT**：早期我们冻 ViT 只训投影层 + LLM——成本低但天花板低。后来用 LoRA 微调 ViT，特别是对印章遮挡、手写体的识别提升明显（5 个点）。
>
> 一个有意思的发现：**ViT 学到的 [CLS] attention 真的在"看脸"** —— 对人脸图，[CLS] 的注意力几乎集中在面部；对证件图，集中在文字密集区。可视化 attention 是个挺好的 debug 手段。
>
> 一句话：**[CLS] 是 ViT 的"全局视角"，但多模态场景里 LLM 更适合直接看所有 patch**。
