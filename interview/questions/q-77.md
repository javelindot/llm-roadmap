# 什么是 DPO

## 答案

**一句话**：DPO（Direct Preference Optimization）= **跳过 RM 和 PPO，用闭式 loss 直接把偏好数据训进模型**——把 RLHF 三段式压成两段式，是当前工业对齐的事实标准。

### 1. 核心动机
传统 RLHF：
```
SFT → RM → PPO（4 模型 + 在线采样 + 调参地狱）
```

DPO：
```
SFT → DPO（2 模型 + 离线数据 + 闭式 loss）
```

### 2. 关键数学（DPO 的灵魂）
PPO 在 KL 约束下的最优策略：
$$\pi^*(y|x) \propto \pi_{ref}(y|x) \cdot \exp(r(x,y)/\beta)$$

反推 reward：
$$r(x,y) = \beta \log \frac{\pi^*(y|x)}{\pi_{ref}(y|x)} + \beta \log Z(x)$$

代入 Bradley-Terry 偏好模型，**Z(x) 项相消**，得 DPO loss：
$$L_{DPO} = -\log \sigma\left(\beta \left[\log\frac{\pi_\theta(y_w|x)}{\pi_{ref}(y_w|x)} - \log\frac{\pi_\theta(y_l|x)}{\pi_{ref}(y_l|x)}\right]\right)$$

### 3. 训练流程
1. SFT 得 base 策略 π_sft
2. 准备偏好对 (x, y_w, y_l)
3. 训练 π_θ（初始化 = π_sft），ref = π_sft（冻结）
4. 直接对 (y_w, y_l) 算 loss，反传更新 π_θ
5. 没有 RM、没有 critic、没有在线采样

### 4. 关键超参
- **β（温度）**：典型 0.1
  - 大：约束强，模型不偏离 ref
  - 小：偏离自由，但易崩
- **learning rate**：通常比 SFT 小 5-10 倍

### 5. 优点
- **简单**：只要 2 个模型
- **稳定**：没有 PPO 的方差爆炸
- **便宜**：显存少、训练快
- **离线**：不用在线采样

### 6. 缺点
- 受**离线数据覆盖**限制（不会探索 OOD）
- log(π/π_ref) 容易整体漂移（Z(x) 消失副作用）
- 长尾偏好覆盖差
- 对 SFT 起点敏感

### 7. DPO 的变种
- **IPO**：解决过拟合，加 anchor
- **KTO**：不需要 pair，只要单条偏好标签
- **SimPO**：去掉 ref model
- **Online DPO**：在线采样 + DPO loss
- **cDPO**：考虑标注噪声

### 8. 第 0 步 loss
π_θ = π_ref 时，log ratio = 0，σ(0) = 0.5，loss = **-log(0.5) = log(2) ≈ 0.693**——是著名的 sanity check。

### 9. 一句话总结
**DPO = 把 PPO 的最优解直接写成离线 loss**——简单、稳定、便宜，工业 80% 对齐场景的首选。

## 结合项目谈（口语版 · 约 30 秒）

> 我们跨境支付**客服回复对齐**全套就是 DPO，理由很直接——比 PPO 便宜 60%、效果差不多、训练稳。
>
> 关键工程经验三条：
>
> 第一是**偏好数据准备**——5w 条 (prompt, chosen, rejected)，30% 人工标 + 70% GPT-4 judge 蒸馏，length-balanced sampling 避免长度偏置。
>
> 第二是 **β 调参**。一开始用论文默认 0.1，效果还行但有点偏离 SFT。后来调到 0.05，模型更激进对齐，但有点 reward hacking 迹象。最后定 0.1 是甜点。
>
> 第三是 **sanity check**——每次启动 DPO 训练，**第一个 batch loss 必须是 0.6931 ± 1%**，否则就是 ref model 没冻好或者 chosen/rejected 弄反了。这个简单 check 救过我们几次。
>
> 上线之后客服满意度提升 8 个点，训练成本只有 PPO 的 40%，迭代速度也快——业务规则改了 1 天重训。
>
> 一句话：**DPO 不是"次优 PPO"，是工程现实里更聪明的对齐方案**。
