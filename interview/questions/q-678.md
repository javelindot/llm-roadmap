# PPO 的核心目标函数是什么？

## 答案

PPO 的核心是限制每次策略更新幅度，同时利用优势函数提高高回报动作的概率。最常见的是 clipped surrogate objective。

### 策略目标

定义新旧策略概率比：

$$
r_t(\theta)=\frac{\pi_\theta(a_t\mid s_t)}
{\pi_{\theta_{old}}(a_t\mid s_t)}
$$

优势估计为 $\hat A_t$，则最大化：

$$
L^{CLIP}(\theta)=
\mathbb{E}_t\left[
\min\left(
r_t(\theta)\hat A_t,
clip(r_t(\theta),1-\epsilon,1+\epsilon)\hat A_t
\right)
\right]
$$

当概率比超出 $[1-\epsilon,1+\epsilon]$ 时，裁剪项阻止目标继续因过大策略变化获益。正优势动作不会被无限提高概率，负优势动作也不会被一次性压得过低。

### 完整损失

实现中通常还包括 value loss 和 entropy bonus：

$$
L=L_{policy}-c_vL_{value}+c_eH(\pi)
$$

若代码按最小化 loss 实现，各项符号会相应取反。Value network 拟合 return，优势常由 GAE 计算；entropy 鼓励探索。

### RLHF 中的补充

语言模型的 action 是生成 token，reward 通常在序列末端由 Reward Model 给出，并加入相对 reference model 的 KL 惩罚。PPO 只使用当前 rollout 策略附近的数据，属于 on-policy，采样成本较高。

训练时监控实际 KL、clip fraction、policy/value loss、entropy、reward 和生成长度。$\epsilon$ 过大更新可能不稳定，过小则学习缓慢；裁剪只是近似约束，通常还要通过 KL controller 或 early stop 控制策略漂移。
