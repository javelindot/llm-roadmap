# 如何进行监督微调（SFT）

## 答案

SFT 使用高质量“输入到期望输出”数据，通过 teacher forcing 最小化目标 token 的交叉熵，让预训练模型学会指令、格式和领域行为。

### 1. 定义目标与数据

- 明确要提升知识、格式、工具调用还是安全行为；
- 收集人工、业务日志或经验证的合成答案；
- 去重、脱敏、质量分层并覆盖困难负例；
- 按用户、任务或时间切分，避免泄漏；
- 固定 system/user/assistant chat template。

### 2. Token 与 Loss

序列通常拼成完整对话，但 loss 只计算需要模型生成的 assistant token：

\[
\mathcal L_{\rm SFT}
=-\sum_{t\in\text{assistant}}
\log p_\theta(y_t|x,y_{<t})
\]

system、user、padding 通常设为 `-100` mask。必须检查 BOS/EOS、截断、packing 边界和多轮角色。

### 3. 训练方式

- 数据充足、预算高可全参；
- 常见业务适配先用 LoRA/QLoRA；
- BF16、warmup、AdamW、gradient clipping 和 checkpoint；
- 按 token 而非仅样本数统计 batch；
- 监控 train/validation loss、梯度、吞吐和任务指标。

### 4. 评估与上线

比较基座、prompt-only 和 SFT；评估指令遵循、事实、格式、安全、通用能力回归、延迟与成本。分析 free-running 生成而非只看 teacher-forcing loss，再做影子与灰度。

### 5. 局限

SFT 主要模仿示范，无法自动解决知识时效、偏好排序、暴露偏差和高风险权限。知识用 RAG，偏好可用 DPO/RL，操作由工具与 workflow 约束。

### 面试总结

SFT 的关键是高质量对话数据、正确 chat template 和 assistant-only loss mask。训练结束必须做自由生成与业务回归，不能把低 loss 等同于模型可靠。
