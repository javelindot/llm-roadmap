# 知不知道序列中每一个token的embedding在经过很多层self attention之后会变得相似的问题以及对应的原因？

## 答案

多层 Self-Attention 后不同 token 表示越来越相似，通常称为 over-smoothing、representation collapse 或 rank collapse。它不是必然发生，但在网络很深、注意力分布接近均匀或残差比例不合适时更明显。

### 原因

忽略非线性和投影，一层注意力可近似看成：

$$
H^{l+1}=A^lH^l
$$

其中 $A^l$ 是按行归一化的注意力矩阵。连续乘多个混合矩阵，会反复对各 token 做加权平均；若这些矩阵具有较强混合性，表示会逐渐投向少数主方向，不同位置的差异被平滑掉。

实际 Transformer 还有残差、归一化、FFN 和位置编码，但以下因素仍可能加剧趋同：

- 注意力长期接近均匀或多个头学到相似模式。
- 深度增加后低秩主成分不断被强化。
- LayerNorm/RMSNorm 与各向异性的隐藏空间使表示集中在少数方向。
- 位置或局部信息不足，模型更依赖全局平均语义。

### 如何判断

可统计同一序列不同 token 的平均余弦相似度、隐藏状态协方差的有效秩、奇异值谱，以及不同注意力头之间的相似度。只看余弦相似度不够，还要观察任务性能和线性探针，因为高层 token 共享全局语义在一定程度上是合理现象。

### 缓解方法

- 使用稳定的残差路径和合适的 Pre-Norm、残差缩放或门控。
- 保留强位置编码、局部窗口或多尺度连接，避免每层都做全局平均。
- 促进注意力头多样性，或对表示相关性、有效秩加入正则。
- 采用跨层跳连、特征拼接，或在深层减少无必要的混合。
- 调整初始化、学习率和网络深度，并通过消融定位是注意力还是 FFN 导致。

目标不是让 token 永远彼此正交，而是在共享上下文语义的同时保留完成任务所需的位置和内容差异。
