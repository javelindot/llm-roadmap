# 有养龙虾吗？自己本地部署了吗？

## 答案

**一句话**：这里的"龙虾"一般是指 Ollama，本地部署我了解并实际跑过，适合快速拉起本地大模型做开发、验证和私有化原型。

### 核心回答
Ollama 的作用：
- 本地拉取和运行开源模型。
- 提供简单 CLI 和 HTTP API。
- 支持多模型管理。
- 适合开发测试、离线验证、私有化 demo。

常见命令：

```bash
ollama pull qwen2.5:7b
ollama run qwen2.5:7b
```

优点：
- 上手快。
- 本地数据不出机器。
- 适合调 prompt、测 RAG、做小型 Agent 原型。

局限：
- 高并发和生产级调度能力有限。
- 性能不一定比 vLLM/TGI 这类推理服务强。
- 仍要考虑显存、量化和模型许可。

### 1. Ollama 适合什么

Ollama 的定位更偏本地开发和原型验证。它把模型下载、运行、API 调用封装得很简单，适合快速测试模型能力、prompt、RAG 原型和本地 demo。

### 2. 和生产推理服务的区别

如果是生产高并发，通常还要考虑：

- 动态 batch。
- KV Cache 管理。
- 多卡调度。
- 请求队列。
- 监控告警。
- 权限和审计。

这时 vLLM、TGI 或专门推理服务会更合适。

### 3. 面试总结

Ollama 适合本地验证和轻量部署，生产要看并发、延迟、监控和安全。

### 4. 结合跨境支付项目怎么说（面试口径）

> 我本地用过 Ollama，主要是做原型验证。比如支付知识库 RAG 的 prompt、工具调用流程、模型对中文支付术语的理解，我会先在本地小模型上快速试。
>
> 它的好处是简单、数据不出本机，适合内网或敏感数据场景。但如果要上生产高并发，我不会只靠 Ollama，会考虑 vLLM、TGI 或云上推理服务。
>
> 对支付场景来说，模型跑起来只是第一步，后面还要加鉴权、脱敏、日志审计、资源限制和高危工具审批。

**钩子**：Ollama 适合本地原型，生产高并发要看 vLLM/TGI 和安全治理。
