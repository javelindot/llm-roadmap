# 什么是强化学习的熵崩溃？怎么优化或者解决这个问题？

## 答案

熵崩溃是指强化学习训练中策略分布过早变得非常尖锐，模型几乎总选择少数动作或生成少数模式，策略熵快速下降。结果是探索能力消失，模型容易卡在局部最优、重复模板或 reward hacking 策略上。

策略熵可以写成：

```text
H(pi(.|s)) = - sum_a pi(a|s) log pi(a|s)
```

熵越低，策略越确定；熵越高，动作更分散。低熵本身不一定错，问题在于训练早期或任务尚未学好时过早塌缩。

### 1. 在大模型 RL 中的表现

- 不同 prompt 输出几乎相同模板；
- 同一 prompt 的多次 rollout 高度重复；
- 推理链变短或只剩固定套路；
- 某个高奖励短语被频繁复用；
- 工具调用总选择同一个工具；
- reward 上升，但真实任务成功率下降；
- 新国家、新币种、新错误码泛化明显变差。

跨境支付中可能表现为：模型不管问题是什么，都先回复“请补充 KYC 材料”，因为这个动作在训练数据里经常得到安全奖励，但它并没有真正识别订单问题。

### 2. 常见原因

- reward 过强、过稀疏或有漏洞；
- 学习率过大；
- PPO/GRPO clip 设置不合理；
- entropy coefficient 太小；
- KL 约束太弱或参考模型质量差；
- 采样温度过低，rollout 缺乏多样性；
- group 内答案几乎相同，优势估计退化；
- 训练数据分布过窄；
- 过度惩罚错误，模型只学会最保守的固定回答。

### 3. 训练侧解决方法

**增加熵正则**

在目标中加入 entropy bonus：

```text
objective = policy_objective
          - beta * KL
          + alpha * entropy
```

也可以设目标熵，根据实际熵动态调整 `alpha`。

**加强 KL 约束**

限制 RL policy 不要快速偏离 SFT/reference model。要监控 token-level KL 和 response-level 行为变化，而不是只看平均值。

**降低更新强度**

- 降低学习率；
- 缩小 clip range；
- 减少每批数据的更新 epoch；
- 做 gradient clipping；
- reward clipping/normalization；
- 在熵快速下降时 early stop。

**提高 rollout 多样性**

- 适当提高 sampling temperature；
- 调整 top-p；
- 增加 group size；
- 对重复答案去重；
- 增加困难 prompt 和分布外任务；
- 混入一定比例 SFT/replay 数据。

### 4. 奖励侧解决方法

- 拆分事实、格式、安全、过程、结果奖励；
- 避免单一 judge；
- 对模板重复和长度投机做惩罚；
- 使用可执行 verifier；
- 增加反事实和 adversarial case；
- 对“总是拒答”与“总是调用同一工具”设置任务完成约束。

支付业务里不能只奖励“安全拒答”，否则模型会塌缩成全部转人工；也不能只奖励“解决率”，否则模型会编造结果。应同时约束事实正确、工具正确、安全、任务完成和用户体验。

### 5. 如何监控

训练时至少观察：

- policy entropy；
- KL divergence；
- unique response ratio；
- group 内答案相似度；
- reward mean/std；
- clip fraction；
- 生成长度分布；
- 工具调用分布；
- held-out 任务成功率；
- 安全拒答率和误拒率。

如果 reward 上升、entropy 和多样性快速下降、held-out 指标变差，通常就是危险信号。

### 6. 面试总结

熵崩溃是策略过早确定化导致探索和泛化丢失。解决思路是熵正则、KL 约束、降低更新强度、提高采样与数据多样性、修复奖励漏洞并持续监控。目标不是让熵永远高，而是在探索和稳定收敛之间保持平衡。
