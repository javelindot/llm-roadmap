# Pairwise 排序模型：优化成对文档的相对顺序，主要关注文档之间的关系。

## 答案

Pairwise 排序将同一 Query 下两个候选的相对顺序作为监督信号，而不是独立预测每个文档的绝对相关分数。

### 1. 数据形式

训练样本通常是：

```text
(query, document_positive, document_negative)
```

目标满足：

```text
score(q, d_positive) > score(q, d_negative)
```

正负文档可以来自人工相关性标注、点击日志、旧模型候选和困难负样本挖掘。

### 2. 常见损失

RankNet 使用 Logistic Pairwise Loss：

```text
L = -log σ(s_positive - s_negative)
```

Margin Ranking Loss：

```text
L = max(0, margin - s_positive + s_negative)
```

LambdaRank/LambdaMART 进一步按一次交换对 NDCG 等排序指标的影响调整梯度，使高排名位置和高价值错序获得更大权重。

### 3. 为什么使用 Pairwise

- 相对判断通常比绝对打 0 到 4 分更容易；
- 直接学习“谁应该排前面”；
- 可重点训练语义相似但不相关的困难负样本；
- 对不同标注者的绝对打分尺度不那么敏感；
- 适合搜索、推荐、RAG Rerank 和 Reward Model。

### 4. 数据构造是核心

随机负样本通常太简单，应增加：

- BM25 Top 结果中的误召回；
- Dense Retrieval 中语义接近但条件不符的文档；
- 相同国家、币种、产品但版本不同的规则；
- 只差否定词、额度、日期或客户类型的候选；
- 旧线上模型排得高但被专家判错的候选。

要排除 false negative。一个问题可能有多份有效证据，不能因为只标注一份，就把其他正确文档作为负例。

### 5. 局限

- 候选数大时可能产生 `O(n²)` 对；
- 只优化局部成对关系，不一定得到最优列表；
- Pairwise 概率不等于绝对相关概率，通常需要校准；
- 点击对存在曝光、位置和热门偏差；
- Pair 采样分布与线上候选不一致时会失效；
- 可能出现非传递偏好，例如 A>B、B>C、C>A。

需要结合 Pointwise/Listwise 基线、位置偏差校正和真实候选回放。

### 6. 评测

- Pairwise Accuracy；
- MRR、NDCG@K、MAP、Precision@K；
- 困难负样本分桶；
- 不同 Query、语言和业务场景；
- P95/P99 和单位候选打分成本；
- 下游答案引用正确率或推荐转化。

跨境支付 RAG 中，Pairwise Reranker 可以学习“巴西 BRL 个人付款新规则”应排在关键词相似的旧规则或企业收款人规则之前，但权限、有效期和租户过滤应在排序前后由确定性逻辑保证。
