# 大模型在短视频推荐中的应用与实践

## 答案

大模型在短视频推荐中的价值主要不是直接替换整条推荐链路，而是增强内容理解、用户意图、候选生成和列表优化。生产方案通常采用：

> 多模态内容理解 → 用户与 Query 理解 → 多路召回 → 轻量粗排 → 多任务精排 → 生成式重排/解释 → 安全与业务约束。

### 1. 多模态内容理解

短视频同时包含画面、字幕、语音、音乐、标题、评论和作者信息。可以使用视觉语言模型、ASR、OCR 和音频模型生成：

- 视频主题、场景、人物、物体、地点和动作标签；
- ASR 文本、字幕和 OCR；
- 内容 embedding、章节摘要和高光片段；
- 安全、广告、低质和版权风险标签；
- 多语言 caption 和搜索关键词。

这些特征可以用于冷启动、相似视频召回、搜索和审核。模型生成的标签必须带置信度和版本，不能把 caption 中的推断当成内容事实。

### 2. 用户与 Query 理解

LLM 可以把搜索词、评论和近期行为序列归一化为结构化兴趣：

```text
长期兴趣：跨境电商、AI 工具
短期意图：正在搜索 RAG 面试题
负向偏好：不看入门内容
场景：通勤、短 session
```

它适合解决多语言、口语、省略和长尾表达，但在线高 QPS 场景通常不会每次调用大参数模型。更常见的是离线生成用户兴趣标签、蒸馏到小模型，或只对复杂 query 路由到 LLM。

### 3. 增强召回

可以将 LLM 能力用于：

- 生成或改写搜索 query，扩展长尾召回；
- 生成视频 semantic ID，进行生成式召回；
- 用长序列模型编码用户历史；
- 根据自然语言兴趣从合法 item 集合中生成候选；
- 为双塔训练构造困难负样本和蒸馏标签。

HSTU 强调超长行为序列建模，OneRec 尝试用生成式模型统一召回与排序。它们代表能力上限方向，但自回归解码、semantic ID 更新、有效 item 约束和在线成本仍是工业落地重点。

### 4. 精排与列表级优化

传统精排预测 CTR、观看时长、完播、点赞、评论、关注和负反馈。LLM 或生成式模型可以补充：

- 高维语义特征；
- 用户与内容的细粒度匹配分；
- 列表级重排，减少主题重复；
- 对长期价值或自然语言偏好进行对齐；
- 生成可解释但不泄露内部策略的推荐理由。

列表生成不能绕过下架、权限、内容安全、频控、广告和多样性规则。更稳妥的方式是在经过过滤的候选白名单上做受约束选择，而不是自由生成视频 ID。

### 5. 数据和训练

训练数据来自曝光、播放、完播、点赞、关注、评论、分享和负反馈，需要处理：

- 曝光与位置偏差；
- 未曝光 item 不能直接当负例；
- 完播、留存等延迟反馈；
- 热门内容和重度用户主导；
- 多目标之间的冲突；
- 内容标签和用户隐私边界。

可使用 SFT 学习候选或列表生成，再用 reward model、DPO 或其他偏好优化学习列表偏好。但奖励必须包含负反馈、内容安全和长期指标，不能只优化短期时长。

### 6. 工程架构

短视频推荐具有高 QPS 和严格尾延迟，通常采用分层部署：

- 大模型离线生成内容、用户和训练特征；
- 中等模型用于低频复杂 query 或候选重排；
- 蒸馏后的小模型承担主流在线流量；
- 特征和 embedding 缓存，按内容或用户变化增量刷新；
- 大模型超时不阻塞主链路，自动回退传统召回和排序。

所有模型调用记录输入版本、候选、输出、延迟和降级原因，支持回放和 A/B。

### 7. 评测

- 内容理解：标签 F1、caption 事实性、跨语言一致性；
- 召回：Recall@K、HitRate、覆盖率和长尾召回；
- 排序：GAUC、NDCG、LogLoss 和校准；
- 业务：观看时长、留存、互动、关注和负反馈；
- 列表：多样性、重复率、新颖性和安全违规率；
- 系统：P95/P99、GPU 成本、缓存命中和降级率。

线上实验要同时观察内容生态指标，防止只提升时长却增加低质、重复或极化内容。

### 8. 跨境支付中的可迁移部分

相同能力可以用于理解商户的多语言需求、解析付款材料、召回支付产品和生成解释，但业务边界不同：

- LLM 负责意图、文档和候选理解；
- 规则服务负责 KYB、牌照、额度和制裁限制；
- 路由模型比较成功率、成本和时效；
- 确定性系统执行报价、幂等和资金动作。

推荐技术可以迁移，内容平台的用户行为数据和支付数据治理方式不能照搬。支付场景需要更严格的租户隔离、审计和人工审批。
