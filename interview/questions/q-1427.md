# subject agent 的设计思维考察（意图、环境、function）

## 答案

Subject Agent 是对某一业务领域负责的受限 Agent。设计时可以从 Intent、Environment、Function 三层展开，再补充状态、策略、安全和评测。

### 1. Intent：它要完成什么

先定义单一、可验收的职责。例如广告素材 Agent：

```text
输入商品、品牌、渠道和受众约束
输出可投放素材、生成依据和质量检查结果
```

需要明确：

- 支持哪些意图；
- 哪些请求应拒绝或交给其他 Agent；
- 成功标准；
- 是否需要用户确认；
- 最大成本和执行时长。

意图识别除了分类，还要抽取槽位，例如渠道、尺寸、语言、品牌风格和截止时间。

### 2. Environment：它能观察什么

环境包括：

- 当前任务状态；
- 用户和账户权限；
- 品牌资产；
- 历史素材及效果；
- 渠道规格；
- 预算和截止时间；
- 工具返回；
- 其他 Agent 产出的 artifact；
- 实时系统状态。

观察要最小权限、结构化、带版本和时间戳。不能把整个企业数据库无条件塞进上下文。

### 3. Function：它能做什么

工具应按业务原子能力设计：

```text
search_brand_asset
generate_copy
generate_image
resize_creative
check_policy
save_draft
request_approval
```

每个工具需要：

- 清晰描述和 JSON Schema；
- 权限范围；
- 幂等性；
- timeout/retry；
- 错误码；
- 审计日志；
- dry-run；
- 不可逆动作的确认机制。

### 4. Agent Policy

```text
observe
-> plan
-> select function
-> execute
-> inspect result
-> replan or finish
```

简单固定流程直接用 Workflow；只有存在动态决策、反馈和重规划时才需要 Agent。

### 5. 状态与输出

内部状态应存储在任务系统中：

```text
task_id
current_stage
confirmed_constraints
artifacts
tool_history
budget_used
pending_approval
```

输出不仅有结果，还应包含证据、版本、置信度、风险和下一步。

### 6. 评测

- 意图和槽位准确率；
- 工具选择和参数准确率；
- 任务完成率；
- 无效调用次数；
- Artifact 采纳率；
- 策略合规率；
- P95 和成本；
- 人工修改和回退率。

### 7. 跨境支付场景

“出款调查 Subject Agent”可查询订单、通道回执和银行状态，输出原因与补材料建议；它不能直接执行出款或修改账务。Intent 定义任务边界，Environment 提供授权状态，Function 限定可执行动作。

### 8. 面试总结

Subject Agent 设计不是先选框架，而是先定义意图边界，再设计可观察环境和最小工具集合，最后用状态机、权限、审计和评测保证它只在职责范围内自主决策。
