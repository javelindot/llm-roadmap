# 讲讲RAG项目中从数据清洗到检索服务上线的整个链路搭建以及chunk切分的方法

## 答案

完整 RAG 项目应覆盖数据治理、解析切分、索引、在线检索、生成校验、服务化和评测监控。上线不是“向量入库完成”，还要支持权限、版本、删除、灰度和故障降级。

### 1. 数据接入

来源：

- PDF/Word/Excel/网页；
- FAQ、工单和 SOP；
- 数据库元数据；
- 图片和扫描件；
- 外部规则文档。

接入时记录 source、owner、tenant、版本、生效时间、权限和 checksum。

### 2. 清洗与解析

- 文件去重和近重复；
- 编码、乱码和异常页；
- OCR 与版面恢复；
- 页眉页脚清理；
- 多栏阅读顺序；
- 表格和图片说明；
- 敏感信息脱敏；
- 过期与冲突文档标记。

解析结果保留页码、bbox 和原文件引用。

### 3. Chunk 切分

优先按：

```text
标题层级
-> 语义段落
-> FAQ/条款
-> 表格/列表
-> 固定 token 兜底
```

策略：

- 小 child 召回，大 parent 回填；
- 标题路径注入；
- 表格保持表头与行关系；
- overlap 防边界断裂；
- chunk size/overlap 用评测选择；
- 国家、币种、通道、版本、权限作为 metadata。

### 4. 索引构建

- Dense Embedding；
- BM25/倒排；
- 可选结构化/图索引；
- HNSW/IVF 等 ANN；
- 向量模型版本写入 metadata；
- 批量构建、增量更新、删除和重建；
- 索引完成后做抽样与 count/checksum 校验。

### 5. 在线检索

```text
Auth
-> Query Rewrite/Entity Extraction
-> Knowledge/Tool Routing
-> Metadata Filter
-> BM25 + Dense
-> RRF Fusion
-> Rerank
-> Dedup/Parent Expansion
-> Context Packing
```

实时状态和数值走工具，不只走知识库。

### 6. 生成与校验

- Prompt 要求只基于证据；
- 输出引用 chunk/page；
- 证据冲突按版本和权威性处理；
- 无证据拒答；
- Schema 校验；
- 高风险回答转人工；
- 防 Prompt Injection 和工具越权。

### 7. 服务上线

- API/SSE；
- 超时、重试、熔断和降级；
- Embedding/Rerank/LLM 独立扩缩；
- 缓存和连接池；
- tracing 保存 query、候选、分数、引用和模型版本；
- 蓝绿/灰度发布；
- 索引与模型可回滚；
- 多租户隔离和审计。

### 8. 评测与监控

- 解析/OCR 准确率；
- Recall@K、MRR、NDCG；
- Faithfulness、Correctness、Citation；
- 首次解决率和转人工率；
- P95/P99 和成本；
- 越权、泄漏和注入；
- bad case 归因与回流。

### 9. 跨境支付

通道规则应按国家、币种、产品、生效日期切分；费率、订单和余额使用工具；索引删除和权限变化需要及时生效，不能让旧缓存继续暴露内容。

**面试总结**：RAG 上线的核心是“证据生命周期”，从数据进入到引用、更新和删除都要可追踪。
