# 🔥 PPO原理

## 答案

**一句话**：PPO 是一种稳定的策略梯度算法，通过限制新旧策略差异，避免强化学习更新步子太大导致模型崩掉。

### 1. PPO 核心思想

PPO 优化策略时，会比较新策略和旧策略的概率比：

```text
r_t(theta) = pi_theta(a|s) / pi_old(a|s)
```

然后用 clip 限制这个比值不要偏离太多：

```text
min(r_t A_t, clip(r_t, 1-eps, 1+eps) A_t)
```

这样既能让策略朝高奖励方向更新，又不会一次更新过猛。

### 2. 在 RLHF 中

PPO 常见流程：

1. SFT 模型生成回答。
2. Reward Model 给回答打分。
3. Critic 估计价值。
4. PPO 更新策略模型。
5. KL 约束防止偏离 reference model 太远。

### 3. 结合跨境支付项目怎么说（面试口径）

> 在跨境支付场景里，我会谨慎使用 PPO。它可以用于优化客服模型偏好，比如更准确、更合规、更少幻觉，但不适合直接优化资金决策。
>
> 比如奖励可以鼓励模型回答有依据、表达清晰、不泄露风控规则；KL 约束可以防止模型为了追求奖励而过度迎合用户，比如承诺一定到账。
>
> 所以 PPO 的价值是做模型对齐和表达优化，核心交易规则仍然由业务系统控制。
