# 大模型的量化方法有哪些

## 答案

大模型量化是用低比特近似权重、激活或 KV Cache，主要目标是降低存储、显存带宽、通信和计算成本。可按“量化对象、是否训练、数值格式和粒度”分类。

### 1. 基本公式

```text
q
= clip(
    round(x / scale) + zero_point
  )

x_hat
= scale * (q - zero_point)
```

误差来自舍入、截断和 scale 粒度。

### 2. 按训练阶段

**PTQ**

- 训练后量化；
- 只需少量校准数据；
- 成本低、上线快；
- GPTQ、AWQ、SmoothQuant。

**QAT**

- 训练中模拟量化；
- 用 STE 近似离散操作梯度；
- 通常精度更好；
- 成本和实现复杂。

**Quantization-aware Fine-tuning**

- 量化模型上继续微调；
- QLoRA 用 4-bit NF4 冻结基座并训练 LoRA；
- 它主要是低显存训练方法，不等同于最终推理格式。

### 3. 按量化对象

- Weight-only：W8/W4，激活仍 BF16/FP16；
- W8A8：权重和激活 8-bit；
- FP8：E4M3/E5M2 等浮点格式；
- KV Cache：FP8/INT8/更低位；
- Embedding/LM Head：通常更敏感，可保留高精度；
- Mixed Precision：不同层/通道使用不同位宽。

### 4. 常见方法

**GPTQ**

- 逐层/逐列量化；
- 使用近似二阶信息补偿误差；
- 常用于 INT4 weight-only。

**AWQ**

- 根据激活统计保护显著权重通道；
- 减少低比特权重量化误差。

**SmoothQuant**

- 将激活 outlier 难度迁移到权重；
- 便于 W8A8。

**FP8**

- 动态范围比 INT8 灵活；
- 依赖对应 GPU 和 fused kernel。

**GGUF**

- llama.cpp/CPU/边缘生态常用的多种混合位宽格式；
- 与 GPU AWQ/GPTQ 是不同执行生态。

### 5. 粒度

- Per-tensor；
- Per-channel；
- Per-group；
- Per-token 动态激活；
- group 越小精度通常越好，但 scale 元数据和 kernel 复杂度增加。

### 6. 量化不一定提速

需要硬件低精度指令和 fused dequant-GEMM。否则可能只省显存，每次反量化反而增加开销。收益还受 batch、Prefill/Decode、并行和通信影响。

### 7. 评测

- Perplexity 和通用能力；
- 业务事实、金额、币种、JSON、工具参数；
- 长上下文和小语种；
- TTFT、TPOT、吞吐；
- 峰值显存、功耗和成本。

跨境支付中平均分接近不够，只要高风险数字或状态错误率明显上升，就不能上线。
