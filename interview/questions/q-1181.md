# 指标对比baseline提升主要来自于什么

## 答案

解释“相对 baseline 的提升来自哪里”，不能只给出直觉描述，应通过受控实验把收益归因到数据、特征、模型、训练、候选分布和工程变化。

### 1. 先保证比较公平

实验组与 baseline 应保持：

- 相同训练、验证和测试时间窗口；
- 相同样本、标签定义和成熟窗口；
- 相同候选集和特征可用时间点；
- 相同线上流量、人群和业务约束；
- 相同延迟或明确报告额外资源；
- 相同指标实现和置信区间。

若实验组同时使用更新数据、更多特征和更大模型，不能把全部提升归因于新结构。

### 2. 做增量消融

从 baseline 开始逐项加入：

```text
B0：原始 baseline
B1：B0 + 新数据/清洗
B2：B1 + 新特征
B3：B2 + 新模型结构
B4：B3 + 新损失/采样
B5：B4 + 新召回或候选
B6：B5 + 在线工程优化
```

同时做 reverse ablation：从最终模型中移除某一模块。如果模块加入和移除的结论一致，归因更可信。多个模块存在交互时，需要二因素实验或 Shapley-style 分解，不能简单相加单项 uplift。

### 3. 常见提升来源

- **数据**：去重、标签纠错、时间穿越修复、困难负样本；
- **特征**：更实时的用户行为、内容表示、国家币种等交叉信息；
- **模型**：更好的序列建模、任务共享或特征交互；
- **训练**：采样、loss weighting、蒸馏、正则和学习率；
- **候选**：召回覆盖提升，使精排有更好的上限；
- **校准与融合**：概率更准，多目标权重更符合业务价值；
- **工程**：缓存和 batch 使更大模型在相同 SLA 下可部署。

模型参数增加本身不是有效归因。应比较参数量、FLOPs、延迟相近的强 baseline，排除“只是算力更多”。

### 4. 分桶分析

总体指标可能由少数大流量分桶主导。需要按以下维度拆解：

- 新老用户、活跃度和历史长度；
- 热门与长尾 item；
- 不同召回来源和候选难度；
- 国家、语言、币种、产品和渠道；
- 正负样本率、金额区间和时间段；
- 延迟、特征缺失和降级路径。

如果提升只来自热门用户，长尾场景明显下降，应说明它是流量重分配，而非全面提升。

### 5. 统计和线上验证

离线报告绝对值、相对提升、多随机种子均值和置信区间。线上采用互斥 A/B，提前确定主指标、守护指标、样本量和停止规则，避免反复查看后挑选显著结果。

还要检查：

- 指标提升是否超过随机波动；
- 是否以延迟、成本或风险恶化为代价；
- 短期 CTR 提升能否转化为长期留存或净收益；
- 上线后数据分布变化是否使收益衰减。

### 6. 跨境支付案例

假设新的通道路由模型相对 baseline 将支付成功率提高 0.5 个百分点，归因至少要拆成：

1. 通道健康度从小时级更新为分钟级带来的收益；
2. 新增国家、币种、金额区间特征的收益；
3. 排序模型结构的收益；
4. 超时降级和候选过滤修复的收益；
5. 流量是否从高成本通道迁移，以及净收益和风险是否变化。

最终结论应类似：

> 主要收益来自更实时的通道状态和困难样本训练，模型结构贡献次之；在同等 P95 下成功率和净收益提升，拒付与合规守护指标无显著恶化。

这种结论比“新模型表达能力更强”更可验证，也能指导下一轮资源投入。
