# 同样是排序指标，NDCG 与 MRR 的评估视角差异在哪？

## 答案

MRR 关注“第一个相关结果出现得有多早”，NDCG 关注“整个 Top-K 排序是否把高相关结果放在前面”。二者都考虑位置，但评估视角不同。

### 1. MRR

对每个 Query，取第一个相关结果的排名 `rank_i`：

```text
RR_i = 1 / rank_i
MRR = mean(RR_i)
```

例如第一个正确结果在第 1、2、5 位，对应 RR 为 `1、0.5、0.2`。如果 Top-K 内没有正确结果，通常记 0。

特点：

- 只关心第一个相关结果；
- 适合每个 Query 只需要一个答案的任务；
- 后续相关结果排序变化不会影响 MRR；
- 通常使用二元相关标签。

### 2. NDCG

NDCG 先计算带位置折损的累计增益：

```text
DCG@K = Σ_(i=1..K) (2^rel_i - 1) / log2(i + 1)
NDCG@K = DCG@K / IDCG@K
```

`rel_i` 可以是 0、1、2、3 等分级相关性；`IDCG` 是同一批结果按理想相关度排序得到的最大 DCG。

特点：

- 评估 Top-K 全部结果；
- 高相关结果排得越靠前越好；
- 支持 graded relevance；
- 通过理想排序归一化，不同 Query 之间更可比。

### 3. 例子

假设两个列表的相关度为：

```text
A: [3, 0, 0, 0]
B: [3, 2, 1, 0]
```

二者第一条都相关，MRR 都是 1；但 B 后续还有高质量结果，因此 NDCG 更高。

再看：

```text
C: [1, 3, 2, 0]
```

C 的 MRR 仍是 1，但 NDCG 会惩罚最相关结果没有排在第一位。

### 4. 如何选择

| 场景 | 更关注的指标 |
|---|---|
| FAQ 只返回一个标准答案 | MRR、HitRate@K |
| 搜索结果页 | NDCG@K、MAP |
| RAG 需要多条互补证据 | NDCG、Recall@K、Context Recall |
| 第一个正确结果最关键 | MRR |
| 相关性有多个等级 | NDCG |

### 5. RAG 中的使用

跨境支付问题可能需要订单状态、通道规则和异常处理三条证据。只看 MRR 会认为第一条正确就够，但后续关键证据缺失仍会导致答案不完整，因此更适合同时看：

- MRR：首条权威证据是否靠前；
- NDCG：Top-K 整体质量；
- Recall@K：必要证据是否全部覆盖；
- Context Precision：进入模型的噪声比例。

指标计算前必须明确相关性标注和无答案 Query 的处理，否则不同实验的 MRR/NDCG 不可直接比较。
