# 2．实习微调过Qwen2，你说说Qwen2的模型结构吧，Qwen2相比Qwen1做了哪些改进

## 答案

Qwen2 是 Decoder-only、Causal Transformer，使用 next-token prediction。回答时要说明具体微调的是 Qwen2 哪个尺寸，因为 0.5B、1.5B、7B、72B 和 57B-A14B MoE 的层数、头数及参数量不同。

### 1. Dense 模型结构

每层主要包括：

- Pre-RMSNorm；
- 带 causal mask 的 Self-Attention；
- 残差连接；
- SwiGLU FFN；
- RoPE 位置编码和 QKV bias。

Qwen2 使用 GQA，而不是 Qwen 早期版本的普通 MHA。多个 Query Head 共享较少的 K/V Head，可显著减少 KV Cache。例如 Qwen2-7B 有 28 个 Query Head、4 个 KV Head，head dimension 为 128。

Qwen2 还包含 57B-A14B MoE：总参数约 57B，每个 token 激活约 14B，FFN 使用细粒度路由专家和共享专家。

### 2. 相比 Qwen/Qwen1.5 的主要改进

- **GQA**：降低每 token 的 KV Cache，提升长上下文推理吞吐；
- **长上下文**：训练末期从 4K 扩展到 32K，并结合 DCA、YaRN 与更高 RoPE base 支持更长外推；
- **训练数据**：大模型使用约 7T 高质量 token，增强代码、数学和约 30 种语言的数据；
- **模型谱系**：同时提供从 0.5B 到 72B 的 Dense 模型和 57B-A14B MoE；
- **后训练**：公开报告使用 50 万以上 SFT 样本，并组合离线、在线 DPO；
- **能力与部署**：多语言、代码、数学能力更强，同时小尺寸模型与更小 KV Cache 改善部署。

需要注意，Qwen2 沿用了 Qwen 的 byte-level BPE tokenizer、SwiGLU、RoPE、RMSNorm 和 Pre-Norm，并非所有组件都是 Qwen2 首创。依据可参考 [Qwen2 Technical Report](https://arxiv.org/abs/2407.10671)。

### 3. 微调时如何结合结构

LoRA 通常注入 `q_proj/k_proj/v_proj/o_proj`，任务偏移较大时再覆盖 `gate/up/down_proj`。GQA 使 K/V 投影尺寸与 Q 不同，选择 target modules 和 rank 时要检查真实权重形状。训练还需匹配官方 chat template、EOS、padding side 和上下文窗口。

### 跨境支付场景

多语言能力适合跨境客服和规则理解，GQA 有利于长对话服务。但费率、订单状态和合规规则仍应来自实时系统与版本化知识库，不能依赖继续预训练把易变事实固化进参数。

### 面试总结

Qwen2 是使用 RMSNorm、RoPE、SwiGLU 和 GQA 的 Decoder-only Transformer，并提供 Dense 与 MoE 模型。相对前代，重点改进是 GQA 降低 KV Cache、DCA/YaRN 扩展长上下文，以及更高质量的多语言、代码和数学数据与后训练流程。
