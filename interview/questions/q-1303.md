# 解释一下TRPO，PPO

## 答案

TRPO 和 PPO 都是 on-policy policy-gradient 算法，目标是在提高期望回报的同时限制新策略相对旧策略的变化，避免一次更新过大导致性能崩溃。TRPO 使用显式 KL 信赖域约束，PPO 用更易实现的一阶近似。

### 1. Policy Gradient 基础

策略目标：

```text
J(theta) = E_{tau ~ pi_theta}[return(tau)]
```

用旧策略采样后，常构造 surrogate objective：

```text
r_t(theta)
= pi_theta(a_t|s_t)
  / pi_old(a_t|s_t)

L(theta) = E[r_t(theta) * A_t]
```

`A_t` 表示动作相对当前基线的好坏。若不限制 `r_t`，一次梯度更新可能让策略分布变化过大。

### 2. TRPO

TRPO 最大化 surrogate objective，同时约束平均 KL：

```text
maximize_theta
  E[r_t(theta) * A_t]

subject to
  E[KL(
    pi_old(.|s_t)
    || pi_theta(.|s_t)
  )] <= delta
```

实现上：

- 对目标做一阶近似；
- 对 KL 约束做二阶近似；
- 用 Fisher Information Matrix；
- conjugate gradient 近似求自然梯度方向；
- line search 确保满足 KL 和性能改进条件。

**优点**

- 对策略更新有明确的信赖域约束；
- 理论动机强，训练相对稳健。

**缺点**

- 二阶近似和共轭梯度实现复杂；
- 计算、分布式和调试成本高；
- 对现代大网络不够方便。

### 3. PPO-Clip

PPO 定义同样的概率比，使用裁剪目标：

```text
L_clip(theta)
= E[min(
  r_t(theta) * A_t,
  clip(r_t(theta), 1-epsilon, 1+epsilon) * A_t
)]
```

当 `A_t > 0`，提高该动作概率的收益在 `1+epsilon` 后被截断；当 `A_t < 0`，过度降低概率的收益在 `1-epsilon` 后被限制。

完整训练通常还有：

```text
loss
= -policy_objective
 + c_v * value_loss
 - c_e * entropy
```

优势常用 GAE：

```text
delta_t
= r_t + gamma * V(s_{t+1}) - V(s_t)

A_t^GAE
= sum_l (gamma * lambda)^l * delta_{t+l}
```

### 4. PPO-Penalty

PPO 也可以在目标中直接加入 KL penalty，并根据实际 KL 自适应系数：

```text
L = E[r_t A_t - beta * KL(pi_old || pi_theta)]
```

PPO-Clip 更常见。Clip 并不保证实际 KL 一定小，所以仍要监控 empirical KL，必要时 early stop。

### 5. 对比

| 维度 | TRPO | PPO |
|---|---|---|
| 更新约束 | 显式 KL constraint | Clip 或 KL penalty |
| 优化 | 近似二阶 | 一阶 SGD/Adam |
| 实现 | 复杂 | 简单 |
| 数据 | On-policy | On-policy |
| 同批数据 | 通常一次信赖域更新 | 可做多个 mini-batch epoch |
| 稳定性 | 约束更直接 | 工程上更灵活，但需监控 KL |

### 6. LLM RLHF 中的含义

- state：Prompt 和已生成前缀；
- action：下一个 token；
- policy：语言模型；
- reward：Reward Model 或可验证规则；
- value：估计 token 状态价值；
- reference model：限制远离 SFT 策略。

PPO 相对 `old policy` 的 clip 控制单轮更新；相对 `reference model` 的 KL 控制长期能力漂移，两者不是同一约束。
