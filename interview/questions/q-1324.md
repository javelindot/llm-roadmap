# LLM的损失函数是什么？给你一个10w的词表，计算出事的损失值

## 答案

LLM 预训练和 SFT 最常用的是 token-level Cross Entropy，也就是负对数似然。词表大小为 10 万时，仅知道词表大小不能算出某个样本的真实 loss，还必须知道模型给正确 token 的概率。

### 1. 从 logits 到概率

对词表 logits `z_1 ... z_V`：

```text
p_i = exp(z_i) / sum_j exp(z_j)
```

真实 token 为 `y` 时：

```text
L = -log p_y
  = -z_y + logsumexp(z)
```

实现中直接使用 `CrossEntropyLoss(logits, label)`，不要先手动 softmax，以避免数值不稳定。

### 2. 10 万词表的初始 Loss

如果模型刚初始化、对 10 万个 token 完全均匀：

```text
p_y = 1 / 100000
L = -ln(1 / 100000)
  = ln(100000)
  ≈ 11.5129 nats
```

若以 2 为底：

```text
log2(100000) ≈ 16.61 bits
```

深度学习框架默认使用自然对数，所以通常回答约 `11.51`。

对应 perplexity：

```text
PPL = exp(L) = 100000
```

### 3. 给定正确 token 概率

例如：

```text
p_y = 0.8  -> loss ≈ 0.223
p_y = 0.1  -> loss ≈ 2.303
p_y = 0.01 -> loss ≈ 4.605
```

Loss 与词表大小没有简单线性关系，关键是正确 token 的归一化概率。

### 4. 序列 Loss

自回归模型：

```text
L_seq
= -1/N * sum_t m_t
  log p(x_t | x_<t)
```

`m_t` 用于屏蔽 padding 或 SFT 中不需要监督的 Prompt token。训练时 labels 通常相对输入右移一位。

### 5. 工程注意

- 使用 `logsumexp` 保证稳定；
- padding 设 `ignore_index=-100`；
- 统计有效 token 数，避免不同 batch 长度影响平均；
- label smoothing 会改变最低可达 loss；
- 比较不同 tokenizer/词表模型的 loss 和 PPL 要谨慎；
- 业务效果不能只看 loss，还要看事实、工具调用和安全指标。

**面试总结**：均匀预测 10 万词表时初始 CE 约为 `ln(100000)=11.51`；真实 loss 必须由正确 token 的概率决定。
