# 怎么选取大模型sft的样本

## 答案

SFT 样本选择的目标不是越多越好，而是以尽量少的高质量数据覆盖目标行为、长尾场景和安全边界。应先建立任务 taxonomy，再按质量、覆盖、难度和风险选样。

### 1. 从目标能力出发

将能力拆成：

- 指令遵循；
- 领域问答；
- 工具调用；
- 多轮状态；
- 格式输出；
- 推理/规划；
- 拒答和安全；
- 多语言；
- 异常与恢复。

每类设定目标占比和验收指标，避免数据被容易样本占满。

### 2. 数据来源

- 专家示范；
- 脱敏真实日志；
- 业务 SOP 和知识文档改写；
- 强模型合成；
- 线上 bad case 修正；
- tool execution 成功轨迹；
- 对抗和安全样本；
- 多语言平行数据。

### 3. 质量标准

每条样本检查：

- 问题是否真实、清晰；
- 答案是否正确且有来源；
- 是否遵守系统规则；
- 工具名和参数是否可执行；
- 是否包含过期知识；
- 是否泄露 PII；
- 表达是否符合目标风格；
- 是否与其他样本重复或冲突。

高风险样本由领域专家复核，不能只依赖 LLM judge。

### 4. 多样性和难度

- 不同意图、语言、长度和表达；
- 正常、异常、边界条件；
- 容易、中等、困难；
- 单轮和多轮；
- 高频和关键长尾；
- 正确执行与合理拒答；
- 多种正确解法。

可使用 embedding 聚类、近重复检测和分层采样控制分布。

### 5. Hard Case 与主动学习

从以下来源优先补样：

- 模型低置信度；
- 多模型分歧；
- 线上失败；
- 规则命中但模型答错；
- 新国家、新产品和新错误码；
- 高风险切片。

训练后再评测，形成：

```text
评测 -> bad case -> 标注/合成 -> 训练 -> 回归
```

### 6. 数据切分

- 按时间、用户、商户或任务实体切分；
- 去除 train/test 近重复；
- 固定独立安全集和通用能力集；
- 防止教师模型看到评测答案；
- 保留 OOD 集。

### 7. 跨境支付场景

重点覆盖支付失败、退款、出款、拒付、KYC、费率和多语言。实时订单、费率和监管变化不应作为静态答案硬记，而应训练模型正确调用工具或检索。

### 8. 面试总结

SFT 选样应以能力地图为起点，优先高质量、覆盖广、可验证的样本，再通过 bad case 和主动学习迭代。样本数量只是一个变量，正确性、分布和去重通常更重要。
