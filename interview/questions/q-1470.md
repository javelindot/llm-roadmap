# 使用LangGraph框架与纯Prompt方法相比有哪些核心优势？

## 答案

LangGraph 与“纯 Prompt”不是同一层次：Prompt 决定模型如何完成单次认知任务，LangGraph 将多步 Agent 的状态、路由、持久化、恢复和人工审批显式工程化。它不会自动提升模型智力，优势主要是可靠性和可运营性。

### 1. 显式状态

纯 Prompt 常把历史、工具结果和任务进度混在 messages 中。LangGraph 可定义：

```text
intent
order_id
retrieved_docs
tool_results
retry_count
pending_approval
artifacts
messages
```

节点只读取和更新明确字段，减少靠自然语言猜状态。

### 2. 显式控制流

- 节点；
- 条件边；
- 循环；
- 并行；
- 子图；
- 最大步数；
- fallback；
- 终止条件。

确定性环节可以固定，只有真正需要判断的位置才让 LLM 路由，降低随机性和 Token。

### 3. 持久化与恢复

通过 checkpointer 保存 graph state：

- 多轮会话恢复；
- 长任务暂停；
- 进程失败后继续；
- time travel/debug；
- 人工审批后恢复；
- bad case 重放。

外部写操作仍需幂等，框架 checkpoint 不能自动撤销重复支付。

### 4. Human-in-the-Loop

可以在高风险节点 interrupt：

```text
生成退款方案
-> 暂停
-> 人工确认
-> 执行工作流
```

比在 Prompt 中写“请谨慎”更可靠。

### 5. 可观测和测试

- 单节点测试；
- 路由测试；
- 状态快照；
- tool trace；
- 节点延迟和失败率；
- 使用 LangSmith 等追踪；
- 控制模型、Prompt、工具版本。

### 6. 与手写代码比较

这些能力也能手写，LangGraph 的价值是提供统一抽象和生态；代价包括框架学习、状态 reducer、版本升级和调试复杂度。单轮问答或一个工具调用不需要上图框架。

### 7. 跨境支付场景

支付调查可建：

```text
鉴权
-> 意图
-> 查订单
-> 并行查通道/账务
-> 检索 SOP
-> 生成解释
-> 风险检查
-> 回复/转人工
```

退款和出款进入独立审批子图并使用幂等键。

### 8. 面试总结

LangGraph 的核心优势是把隐式 Prompt 循环变成可持久化、可恢复、可审批、可测试的状态图。效果改善来自约束流程和管理上下文，不是框架让基座模型变强。
