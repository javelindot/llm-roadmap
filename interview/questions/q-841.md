# DPO、PPO、GROP之间的区别是什么？

## 答案

题目中的 “GROP” 通常是 **GRPO（Group Relative Policy Optimization）**。

### 1. DPO

- 数据：固定 chosen/rejected 偏好对；
- 方式：离线直接优化 policy 相对 reference 的概率 margin；
- 组件：policy + frozen reference；
- 不需要显式 Reward Model、critic 和在线 rollout；
- 优点：简单稳定；局限：受离线数据覆盖限制，不能主动探索。

### 2. PPO

- 当前 policy 在线生成 rollout；
- Reward Model/规则给 reward；
- critic/value model 估计回报并计算 advantage；
- clipped policy objective 限制更新；
- 通常加入 reference KL、value loss 和 entropy。

它能优化序列级和环境反馈，但需同时维护 actor、critic、reference、reward 等组件，显存和训练复杂度高。

### 3. GRPO

[DeepSeekMath](https://arxiv.org/abs/2402.03300)提出 GRPO。对同一 prompt 采样一组回答，用组内 reward 均值和方差标准化：

\[
A_i=\frac{r_i-\operatorname{mean}(r)}
{\operatorname{std}(r)+\epsilon}
\]

再进行带 clip 和 KL 的策略优化。它省去独立 critic，降低显存，特别适合数学、代码等可批量验证的任务。

### 对比

| 方法 | 在线采样 | Reward | Critic | 适合场景 |
|---|---|---|---|---|
| DPO | 否 | 偏好对隐式表示 | 否 | 离线行为/风格对齐 |
| PPO | 是 | RM、规则或环境 | 是 | 通用序列级 RL |
| GRPO | 是 | 组内结果奖励 | 否 | 可验证推理与多采样 |

GRPO 需要每个 prompt 多次 rollout，组内全对或全错时信号弱；没有 critic 也不代表训练一定稳定。

跨境支付客服风格可用 DPO，沙箱工具任务可用 GRPO/PPO；真实资金操作不允许开放式探索。

### 面试总结

DPO 是离线偏好分类式优化；PPO 是带 critic 的在线策略优化；GRPO 用同 prompt 多答案的组内相对奖励替代 critic。选型取决于是否有可验证奖励、是否需要探索和能否承担 rollout 成本。
