# 如何评估两个文本的相似度

## 答案

评估两个文本的相似度，先明确比较的是词面、语义、主题、蕴含关系还是业务等价。不同目标对应不同方法，不能只报一个余弦相似度。

### 1. 词面相似

**Jaccard**

```text
J(A, B) = |A ∩ B| / |A ∪ B|
```

适合关键词集合、标签和去重，不能识别同义改写。

**编辑距离**

衡量字符插入、删除和替换次数，适合 OCR 纠错、名称近似和拼写，不适合长文本语义。

**TF-IDF/BM25**

强调共同关键词和稀有词。错误码、币种、API 字段等精确术语效果好。

### 2. 向量语义相似

使用 Sentence-BERT、BGE 等模型将文本编码为向量。常用指标：

**余弦相似度**

```text
cos(a, b) = (a · b) / (||a|| * ||b||)
```

比较方向，最常用。

**点积**

```text
score(a, b) = a · b
```

会受向量范数影响。向量 L2 归一化后，点积与余弦排序等价。

**欧氏距离**

```text
distance(a, b) = ||a - b||_2
```

单位向量下与余弦具有单调关系。

必须使用与训练目标匹配的度量。不同 embedding 模型的分数分布不可直接共用阈值。

### 3. Cross-Encoder

将两个文本拼接后共同编码，输出相关性分数：

```text
[CLS] text_a [SEP] text_b [SEP]
```

它能细粒度建模否定、数字和词间交互，通常比双塔准确，但不能提前离线编码，计算成本高。常用于先用 embedding 召回 Top-N，再 rerank。

### 4. 相似不等于蕴含

“支持 USD 付款”和“不支持 USD 付款”词面与向量都可能很接近，但含义相反。需要 NLI 或任务分类器区分：

- entailment；
- contradiction；
- neutral。

对事实问答还要比较实体、数字、日期和关系，而不是只看整体语义。

### 5. LLM Judge

复杂长文本可以让 LLM 按 rubric 判断主题、事实和意图是否等价，但要提供结构化标准、随机顺序，并用人工集校准。LLM Judge 成本高且存在位置、长度和自偏好，不适合作为唯一 ground truth。

### 6. 如何构建评测

先收集人工标注的正负 pair，包含：

- 同义改写；
- 词面相似但语义相反；
- 相同主题但答案不同；
- 数字、时间和主体变化；
- 多语言与拼写错误；
- 困难负样本。

检索场景用 Recall@K、MRR、NDCG；二分类用 Precision、Recall、F1、ROC-AUC/PR-AUC；连续相似度用 Pearson/Spearman 与人工分数相关性。阈值在验证集选择，并按业务分桶检查。

### 7. 跨境支付示例

“退款多久到”和“退汇多久到”词面相似，但对应不同流程；“USD payout pending”与“美元付款处理中”跨语言语义相近。可以采用：

```text
BM25 + Dense 召回
→ Cross-Encoder rerank
→ 业务实体与否定校验
```

金额、币种、国家、产品和状态不同的文本不能仅因整体向量接近就判为等价。
