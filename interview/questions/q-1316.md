# 多模态大模型的vision encoder一般使用vit的第几层输出?

## 答案

没有统一答案。经典 LLaVA 路线常取冻结 ViT 的倒数第二层 patch features，而不是最后分类层；现代多模态模型越来越多地融合多个中间层或多尺度特征。选择哪一层取决于视觉编码器、任务和 projector 的训练方式。

### 1. 为什么常用倒数第二层

ViT 越靠后：

- 语义越抽象；
- 越适合全局分类；
- 局部位置、纹理和细粒度信息可能减少。

最后一层还可能更受预训练分类/对比目标和 final norm 影响。倒数第二层通常在高级语义与局部细节之间折中，因此很多 LLaVA 配置使用 `-2` 层。

### 2. 不同任务的选择

- 通用图像问答：倒数第一或第二层常可作为基线；
- OCR、文档、图表：需要更强的低层细节和高分辨率特征；
- 目标定位、细粒度识别：可融合中间层；
- 视频：还要处理时间维度和帧级压缩；
- 原生多模态模型：视觉主干和语言模型可能联合训练，不再是简单固定取某一层。

### 3. 多层融合趋势

现代方案可能使用：

- 多层 feature concat/sum；
- feature pyramid；
- layer attention 或可学习加权；
- DeepStack，将不同视觉层注入语言模型的不同深度；
- 动态分辨率和 token compression。

Qwen3-VL 公布的 DeepStack 路线就是将多级视觉特征与语言模型不同层对齐，以改善细粒度视觉理解。

### 4. 如何实验选择

固定其他变量，比较：

- `last`、`-2`、中间层；
- 单层与多层融合；
- 是否经过 final norm；
- CLS token 与 patch token；
- 不同分辨率和 token 数。

评估不能只看通用 VQA，还要看 OCR、DocVQA、ChartQA、定位和真实业务数据，并记录显存和延迟。

### 5. 跨境支付

营业执照、护照、回单和拒付凭证依赖小字、表格和版面位置，单取最后高层语义通常不够。应使用高分辨率 OCR/文档模型、多层视觉特征和坐标引用，并对金额、日期、币种、证件号做规则及后端系统核验。

**面试总结**：经典答案是“常用倒数第二层”，更完整的答案是“没有固定层，细粒度任务更适合多层和多尺度融合”。
