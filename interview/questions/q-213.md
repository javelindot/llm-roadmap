# 项目采用的向量库是什么？如何实现混合检索功能？

## 答案

> 我们用 **Milvus**（生产）+ **Elasticsearch**（关键词检索），混合检索靠 RRF 融合。
>
> **为什么选 Milvus**：
> - 国内活跃、中文社区成熟
> - HNSW、IVF、DiskANN 多种索引可选
> - 元数据 filtering 强（支持 expr 表达式）
> - K8s 部署友好
> - 性能稳定：单机 7w 向量 p95 < 50ms
>
> **对比过的方案**：
> - Faiss：内嵌库，没有持久化/多副本，被淘汰
> - Qdrant：性能也好，但中文生态弱
> - Pinecone：SaaS，金融合规不行
> - PG-Vector：方便但 7w+ 时性能跟不上
> - **Weaviate**：备选，最终选 Milvus 因生态更成熟
>
> **整体架构**：
> ```
> 用户 query
>   ↓ (Query 重写 - q-211)
> ┌─────────────┐     ┌────────────────┐
> │ 向量检索     │     │ BM25 检索       │
> │ (Milvus)    │     │ (Elasticsearch) │
> │ top-30      │     │ top-30          │
> └─────────────┘     └────────────────┘
>          ↓                  ↓
>          └────RRF 融合───────┘
>                ↓ top-30
>          ┌──────────┐
>          │ Rerank   │
>          │ (bge-r)  │
>          │ top-5    │
>          └──────────┘
>                ↓
>          LLM 生成
> ```
>
> **混合检索（Hybrid Search）的核心逻辑**：
>
> **1. 并行召回**（async 同时跑）
> ```python
> async def hybrid_search(query: str):
>     vec_task = asyncio.create_task(milvus_search(query, top_k=30))
>     bm25_task = asyncio.create_task(es_search(query, top_k=30))
>     vec_results, bm25_results = await asyncio.gather(vec_task, bm25_task)
>     return rrf_fusion(vec_results, bm25_results)
> ```
>
> **2. RRF 融合**
> ```python
> def rrf_fusion(*ranklists, k=60):
>     scores = {}
>     for rl in ranklists:
>         for rank, doc_id in enumerate(rl, 1):
>             scores[doc_id] = scores.get(doc_id, 0) + 1 / (k + rank)
>     return sorted(scores.items(), key=lambda x: -x[1])
> ```
>
> **3. 为什么 RRF 而不是分数加权**：
> - 向量分数和 BM25 分数**不在同一量纲**，归一化困难
> - RRF 只用 rank，无需归一化
> - 跨检索器友好
> - 工业最常用
>
> **Milvus 配置**：
> - 索引：HNSW（M=16, efConstruction=200）
> - 距离：cosine（向量都做了 L2 normalize）
> - 分片：3 个 query node
> - 元数据：source / section / language / created_at / tenant_id
> - filtering：query 时按用户租户 / 业务线先过滤
>
> **ES 配置**：
> - 中文分词：ik_max_word
> - 字段：content + title + path
> - 同义词：业务术语同义词表
>
> **几个工程坑**：
> - **向量必须 normalize**——否则 cosine 退化成内积，结果不可控
> - **元数据 filter 要建索引**——否则全表扫描慢
> - **embed 模型要和入库时一致**——版本不一样向量空间不通
> - **删除 + 重建索引**比改更新更稳定
>
> 上线后混合检索准确率比纯向量 +8 个点、比纯 BM25 +12 个点。
