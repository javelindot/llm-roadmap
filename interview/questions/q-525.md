# 3、DIN和Transformer 的注意力机制区别

## 答案

DIN（Deep Interest Network）和 Transformer 都使用注意力，但目标和计算方式不同。

### 1. DIN 的注意力

DIN 用于 CTR 推荐，核心是 **Target-aware Attention**：

- Query 是当前候选商品。
- Key/Value 是用户历史行为商品。
- 计算每个历史行为与候选商品的相关性。
- 加权汇总成“针对当前候选”的用户兴趣表示。

同一个用户面对不同候选商品，会得到不同的兴趣向量。

DIN 的局部激活单元常将候选与历史 Embedding 的拼接、差值或乘积输入 MLP，权重不一定要求归一化为和为 1。

### 2. Transformer 的注意力

Transformer 通常由线性投影得到 Q/K/V：

```text
Attention(Q,K,V) = softmax(QK^T / sqrt(d_k)) V
```

Self-Attention 让序列中每个位置都与其他位置交互，学习上下文化表示，并可使用 Multi-Head 捕捉多种关系。

### 3. 对比

| 维度 | DIN | Transformer |
|------|-----|-------------|
| 目标 | 候选商品相关的兴趣激活 | 通用序列上下文建模 |
| Query | 候选商品 | 每个序列位置 |
| 交互范围 | 候选对历史行为 | 序列位置两两交互 |
| 顺序建模 | 原始 DIN 较弱 | 配合位置编码自然支持 |
| 复杂度 | 候选对历史长度近似线性 | 标准 Self-Attention 对序列平方 |
| 典型场景 | CTR 排序 | NLP、序列推荐、多模态 |

### 4. 实践关系

DIN 更强调“这个候选激活了用户哪部分兴趣”；Transformer 更强调“历史行为之间如何互相影响”。推荐系统可以先用 Transformer 编码行为序列，再用 Target Attention 读取与候选最相关的部分。

**面试总结**：DIN 是候选感知的局部兴趣注意力，Transformer 是通用的全序列关系建模。
