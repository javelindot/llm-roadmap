# Lora 相比全参数微调可以节省多少计算量和显存？

## 答案

LoRA 能大幅减少**可训练参数、梯度和优化器状态**，但总计算量与激活显存不会按同样比例下降，因此不能给一个通用百分比。

### 1. 参数量

对 \(d_{out}\times d_{in}\) 线性层：

\[
\text{Full params}=d_{out}d_{in}
\]

\[
\text{LoRA params}=r(d_{in}+d_{out})
\]

若为 \(d\times d\)，比例约为：

\[
\frac{2r}{d}
\]

例如 \(d=4096,r=8\)，该层 LoRA 参数约占 0.39%。覆盖多个模块后，整模型可训练参数常仍低于 1%～数个百分点。

### 2. 显存

全参 AdamW 每参数常需参数、梯度、FP32 master weight 和两个 FP32 moment，约 12～18 Bytes/param，具体依实现。LoRA 只为少量 adapter 保存梯度和 optimizer states，基座仅保留冻结权重。

因此 7B BF16 基座权重约 14GB；LoRA 再加少量 adapter 状态，而全参模型状态约百 GB。激活、Attention 临时空间和 CUDA workspace 两者都要保留，所以实际峰值节省取决于 batch、序列和 checkpointing。原始 [LoRA 论文](https://arxiv.org/abs/2106.09685)在 GPT-3 175B 场景报告约 3 倍 GPU 显存降低，不应机械外推到所有模型。

### 3. 计算量

冻结权重后无需计算和保存全量 \(dW\)，但前向 \(Wx\) 和反向传递到下层的 \(W^\top dY\) 仍存在；LoRA 还增加两个小矩阵乘。因此：

- 可训练参数可减少 99% 以上；
- 训练 FLOPs 往往只降低一部分，不会减少 99%；
- 速度收益取决于激活、kernel、通信和数据管线；
- QLoRA 主要进一步省权重显存，量化反量化未必让训练更快。

### 面试总结

LoRA 的最大收益是去掉全量梯度和 Adam 状态，可训练参数通常低于 1%，显存可从百 GB 级模型状态降到以基座权重和激活为主；但仍需完整基座前向与输入梯度，计算量不会同比下降。
