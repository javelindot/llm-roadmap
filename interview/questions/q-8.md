# Agent 如何知道自己是谁？

## 答案

**一句话**：Agent 知道"自己是谁"，不是因为它真的有自我意识，而是系统通过角色提示、工具权限、任务状态、记忆和安全策略定义了它的身份边界。

### 1. Agent 身份来自哪里
- **System Prompt**：定义角色、职责、语气、禁止事项。
- **工具权限**：它能调用哪些工具，决定了它能做什么、不能做什么。
- **任务目标**：当前用户让它完成什么任务。
- **业务状态**：当前处于审核、执行、确认、失败还是转人工阶段。
- **长期记忆**：记录它服务的用户、业务偏好、历史经验。
- **安全策略**：哪些话不能说，哪些动作必须审批，哪些场景必须拒绝。

所以 Agent 的"自我认知"本质是**角色约束 + 能力边界 + 当前状态**。

### 2. 为什么重要

如果 Agent 不知道自己是谁，会出现几个问题：
- 把建议当成决策，越权执行。
- 混淆角色，比如客服 Agent 去做风控审批。
- 忘记当前任务边界，回答很多无关内容。
- 在高风险场景里没有转人工意识。

尤其在生产系统里，Agent 必须清楚：我是助手、审核员、路由器，还是执行器。不同身份对应不同权限。

### 3. 面试总结

Agent 的身份不是玄学，它是工程上配置出来的。一个成熟 Agent 要明确三句话：**我是谁、我能做什么、我什么时候必须停下来找人**。

## 结合项目谈（口语版 · 面试现场约 45 秒）

> 我会把 Agent 的"知道自己是谁"理解成角色和权限边界。比如在跨境支付里，如果是风控 Agent，它的身份应该是风险分析助手，不是资金操作员。它可以查交易、分析命中规则、给出放行或转人工建议，但不能直接冻结账户、退款或者改商户状态。
>
> 这个身份一般通过 system prompt、工具权限和工作流状态一起定义。比如商户准入场景里，Agent 当前处于 KYB 初审阶段，它就只能做资料完整性检查、主体一致性校验、风险摘要生成；如果发现制裁名单或高风险国家命中，就必须输出原因并转人工，而不是自己继续往下批。
>
> 所以我觉得 Agent 的身份设计，本质是在告诉它：你在这条业务链路里扮演什么角色，能用哪些工具，哪些事情必须交给人。这个边界越清楚，Agent 越不容易越权。

**自然带出去的钩子**：
- "我会把 Agent 身份和工具权限绑定，而不是只写在 prompt 里。"
- "支付系统里建议权和执行权要分开，不然风险很大。"
- "我更倾向让 Agent 做分析和编排，高危资金动作由规则或人工确认。"
