# 大模型的核心特点有哪些？请举例说明。

## 答案

**一句话**：大模型核心特点可以浓缩成 **6 字**——**大、通、涌、对、多、贵**：参数大、通用性强、能力涌现、可对齐、多模态、推理贵。

### 1. 大（Scale）
- 参数从亿级到万亿级（GPT-4 万亿、Llama-3 70B、Qwen3 235B）
- 数据从 TB 到 PB（Llama-3 用 15T tokens）
- 算力从单卡到万卡（GPT-4 训练成本估算上亿美金）

### 2. 通（Generalization）
- 一个模型解决多任务（翻译、问答、推理、代码、对话）
- 零样本（zero-shot）/ 少样本（few-shot）泛化
- 例：GPT-4 一个模型考过律师资格、SAT、AP

### 3. 涌（Emergent Abilities）
- 规模到某个临界点出现的能力，小模型完全做不到
- 典型：CoT 推理、Instruction Following、In-Context Learning、Tool Use
- 例：13B 以下做数学题靠记忆，70B+ 才开始真的"推理"

### 4. 对（Alignable）
- 可以通过 SFT / DPO / RLHF 把通用能力对齐到具体场景和价值观
- 例：通用 Qwen → 客服 Qwen 只需 5w 偏好对就能做到

### 5. 多（Multi-modal）
- 不再只处理文本，能吃图、声、视频
- 例：GPT-4V 看图回答、Sora 文生视频、Gemini 原生多模态

### 6. 贵（Expensive）
- 训练贵：百万到亿美金
- 推理贵：100 tokens 几分钱也算不便宜
- 维护贵：迭代、对齐、评测都烧钱
- 例：业务侧上 LLM 第一件事就是算 ROI

### 7. 配套副作用（坦诚说出来）
- **幻觉**：自信地胡说
- **延迟**：实时业务受限
- **数据敏感**：训练数据偏见会带到下游
- **可解释性差**：黑盒、难复盘

### 8. 一句话总结
**大、通、涌、对、多、贵**——前四点是它的能力优势，后两点是工业落地必须啃的代价。

## 结合项目谈（口语版 · 约 40 秒）

> 这道题我会结合我们跨境支付的落地体验讲。
>
> **"通"和"涌"**——我们一开始觉得 LLM 就是"高级 NLP"，后来才发现完全不一样。比如风控调查场景，传统模型每种异常都得单独训一个分类器，LLM 一个模型能同时处理"分类 + 解释 + 推理"，而且只要 prompt 写好它就能干，这就是"通"。CoT 推理只在 7B+ 才稳定出现，6B 以下你怎么 prompt 它都不会主动分步——这就是"涌"。
>
> **"对"**——这是我们觉得 LLM 真正给业务带来杠杆的地方。Qwen-7B base 模型对客服一无所知，但我们 5w 条偏好对喂下去 DPO 一晚上，它就能像我们品牌的客服那样说话。这种"低成本注入领域"传统模型给不了。
>
> **"贵"是真贵**——我们风控如果用 70B 跑实时调查，单工单成本能到 0.3 元，一天几十万工单根本顶不住，最后我们用 7B + 蒸馏把成本压到 0.001 元才上线。
>
> 所以我面试讲这题会强调一句：**LLM 最被低估的不是"通"，是"对"——能用小成本把通用能力捏成自己想要的形状，这是工业价值最大的特点**。

**🪝 顺嘴带出去的钩子**：
- "我们用 7B 蒸馏怎么压成本可以聊聊。"
- "涌现能力的临界点在我们业务里有过验证。"
- "LLM 上线的 ROI 怎么算我们有一套方法。"
