# Self-Attention和Cross-Attention中Q、K、V的来源有何不同？

## 答案

区别的核心是 Q 与 K/V 是否来自同一序列。

### 1. Self-Attention

输入为同一序列 `X`：

```text
Q = XW_Q
K = XW_K
V = XW_V
```

因此每个 token 查询同一序列中的其他 token。

- Encoder Self-Attention：通常双向；
- Decoder Self-Attention：加入 causal mask；
- 输出长度与输入序列长度一致。

### 2. Cross-Attention

有 Query 序列 `X_q` 和上下文序列 `X_c`：

```text
Q = X_q W_Q
K = X_c W_K
V = X_c W_V
```

Query 用自身状态去读取另一模态或另一序列的内容。

标准 Encoder-Decoder 中：

```text
Q <- Decoder hidden states
K,V <- Encoder outputs
```

输出长度等于 Query 长度 `L_q`，Attention 权重形状为：

```text
[B, heads, L_q, L_context]
```

### 3. Mask 差异

- Decoder Self-Attention 使用 causal mask；
- Cross-Attention 通常不需要对 context 做 causal mask，只屏蔽 padding 或业务不可见位置；
- 多模态 Cross-Attention 还可能按图像区域、时间或模态加 mask。

### 4. 推理缓存

- Decoder Self-Attention 的 K/V 随生成逐步增长；
- Encoder-Decoder Cross-Attention 的 Encoder K/V 在一次生成内固定，可预计算；
- 多模态模型若将视觉 token 直接拼入序列，则使用的可能仍是 Self-Attention，而不是 Cross-Attention。

### 5. 跨境支付场景

Self-Attention 可理解一段工单内部关系；Cross-Attention 可让工单 Query 读取文档或票据视觉特征。但权限过滤必须在 Context 进入 Attention 前完成。

### 6. 面试总结

Self-Attention 的 Q/K/V 来自同一序列；Cross-Attention 的 Q 来自目标序列，K/V 来自外部上下文。输出始终跟随 Q 的长度。
