# 对 RLHF 了解的多吗

## 答案

RLHF 是利用人类反馈优化语言模型行为的一类方法。经典公开流程包括 SFT、奖励模型和强化学习三个阶段，但现代偏好对齐还包括 DPO、可验证奖励和规则监督等路线。

### 1. 经典流程

1. **SFT**：用高质量指令回答数据训练参考策略 \(\pi_{\text{ref}}\)。
2. **偏好数据**：对同一 prompt 生成多个回答，由标注者排序。
3. **奖励模型**：对偏好对 \((y_w,y_l)\) 优化：

\[
\mathcal{L}_{RM}
=-\log\sigma(r_\phi(x,y_w)-r_\phi(x,y_l))
\]

4. **PPO 优化**：最大化奖励，同时用 KL 防止策略偏离参考模型：

\[
\max_\theta\;
\mathbb{E}\left[r_\phi(x,y)
-\beta D_{\mathrm{KL}}
(\pi_\theta\Vert\pi_{\text{ref}})\right]
\]

PPO 的 clipped objective 还限制单次策略更新幅度，value model 用于估计 advantage。

### 2. 难点

- 人类偏好主观且标注一致性有限；
- 奖励模型在分布外容易失真，策略可能 reward hacking；
- PPO 需要 rollout、奖励、价值和参考模型，训练成本高；
- KL 太大限制学习，太小可能造成能力漂移；
- 优化代理奖励不等于真实安全或业务收益。

### 3. 替代与发展方向

DPO 直接从偏好对优化策略，省去显式奖励模型和在线 PPO；RLAIF 使用 AI 反馈降低标注成本；数学、代码、工具执行等任务可使用可验证结果奖励。当前趋势是把离线偏好优化、过程监督、验证器和运行时约束组合，而不是所有问题都用 PPO。

### 4. 跨境支付场景

RLHF/DPO 可优化客服回答的帮助性、合规表达和拒答边界，但不能让模型直接凭奖励决定资金放行。退款、制裁筛查和账户变更必须经过确定性规则、权限、审批和审计。若优化支付 Agent，可在沙箱里用“参数合法、调用顺序正确、任务完成”作为可验证奖励，再经过影子流量和灰度上线。

### 面试总结

RLHF 的经典核心是“偏好排序训练奖励模型，再用带 KL 约束的 PPO 优化策略”。它能优化没有唯一标准答案的序列级偏好，但成本高且有奖励投机风险。现代工程通常根据任务选择 SFT、DPO、RL、验证器和运行时规则的组合。
