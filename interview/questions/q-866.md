# 介绍一下 beam search原理，与直接sample的区别？

## 答案

Beam Search 在每一步保留累计分数最高的 \(B\) 条部分序列，而不是只保留一个 token 或随机采样。

### 1. 流程

1. 用 BOS 初始化一个 beam；
2. 对每条 beam 计算下一 token log probability；
3. 将累计分数加上新 token logp；
4. 从所有扩展候选中选 top-\(B\)；
5. EOS 序列进入完成集合；
6. 达到终止条件后选得分最高序列。

常使用长度归一化：

\[
\operatorname{score}(y)
=\frac{\sum_t\log p(y_t|y_{<t},x)}
{\operatorname{lengthPenalty}(|y|)}
\]

否则 log probability 累加倾向短序列。

### 2. 与 Sample 的区别

| 维度 | Beam Search | Sampling |
|---|---|---|
| 目标 | 近似找高概率序列 | 从分布随机生成 |
| 多样性 | 通常较低 | 可通过 temperature/top-p 调节 |
| 确定性 | 相同输入通常固定 | 随机 |
| 成本 | 约随 beam size 增长 | 单路径较低 |
| 场景 | 翻译、ASR、结构化生成 | 对话、创作、多候选探索 |

Beam Search 容易重复和生成“安全但平庸”文本，beam 越大也不保证人类偏好越好。对 LLM 对话通常使用 greedy 或 top-p；对严格结构可结合 constrained beam search。

### 面试总结

Beam Search 保留多个累计高分路径，近似最大似然解码；sampling 按概率随机探索。前者适合确定性序列转换，后者更适合开放生成，多样性与成本不同。
