# Warmup和Annealing通常如何配合使用？

## 答案

**一句话**：Warmup 负责训练初期慢慢升学习率，Annealing 负责中后期逐步降学习率，一个保稳定，一个保收敛。

### 核心回答
常见组合：
1. **Warmup 阶段**：学习率从 0 或很小值线性升到峰值。
2. **稳定训练阶段**：短暂保持或直接进入衰减。
3. **Annealing 阶段**：按 cosine、linear 或 exponential 逐步降低学习率。

为什么要配合：
- 初期模型不稳定，需要小步走。
- 中期需要较大学习率快速学习。
- 后期需要小学习率细调，避免在最优点附近震荡。

常见策略是 linear warmup + cosine decay。

### 1. 为什么要先升后降

训练早期模型和优化器状态都不稳定，直接用大学习率容易把模型推到很差的区域，所以用 warmup 慢慢升。训练中后期模型已经接近较优区域，如果学习率一直很大，就会在最优点附近震荡，所以用 annealing 慢慢降。

### 2. 常见组合

- **Linear warmup + cosine decay**：大模型训练和微调里很常见。
- **Linear warmup + linear decay**：简单稳定，容易解释。
- **Warmup + constant + decay**：中间保持一段最大学习率。

调参时要结合总 step：数据少、训练步数少时，warmup 太长会浪费有效训练；学习率偏大或 DPO/PPO 这类不稳定训练，可以适当增加 warmup。

### 3. 面试总结

Warmup 解决"一开始别炸"，annealing 解决"后面要收敛"。它们配合起来，本质是控制优化过程的节奏。

### 4. 结合跨境支付项目怎么说（面试口径）

> 在支付领域微调里，我会把它理解成"先别冲太猛，后面慢慢收口"。比如训练一个合规问答或风控解释模型，一开始学习率太大容易破坏基座能力，模型可能突然变得过度拒答或者输出很模板化。
>
> 训练后期如果学习率不降，又容易在偏好数据上来回震荡，表现为同一类风控问题有时答得稳、有时答得很飘。所以我一般会用 warmup 加 cosine decay。
>
> 评估时我不会只看训练 loss，还会看验证集上的格式遵循、拒答率、证据引用准确率和业务准确率。支付场景里稳定比短期 loss 降得快更重要。

**钩子**：微调不是 loss 降越快越好，稳定保留基座能力更重要。
