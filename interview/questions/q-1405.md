# 优化算法知道哪些?讲-下AdamW

## 答案

常见优化算法可以分为 SGD 系列和自适应学习率系列：

- SGD；
- Momentum、Nesterov Momentum；
- AdaGrad；
- RMSProp；
- Adam、AdamW；
- Adafactor；
- LAMB/LARS；
- Lion；
- 8-bit optimizer。

大模型训练中 AdamW 是最常见基线，因为它结合一阶、二阶动量实现参数级自适应学习率，并把 weight decay 与梯度更新解耦。

### 1. Adam 的核心

给定梯度 `g_t`：

```text
m_t = beta1 * m_(t-1) + (1-beta1) * g_t
v_t = beta2 * v_(t-1) + (1-beta2) * g_t^2
```

做 bias correction：

```text
m_hat = m_t / (1-beta1^t)
v_hat = v_t / (1-beta2^t)
```

参数更新：

```text
theta = theta - lr * m_hat / (sqrt(v_hat) + epsilon)
```

`m` 平滑梯度方向，`v` 根据历史梯度幅度调整每个参数的实际步长。

### 2. AdamW 为什么比 Adam 加 L2 更合理

若把 L2 正则直接加进 Adam 的梯度：

```text
g' = g + lambda * theta
```

正则项也会被 `sqrt(v)` 自适应缩放，不同参数受到的实际衰减不一致。

AdamW 把 weight decay 独立出来：

```text
theta =
(1 - lr * lambda) * theta
- lr * m_hat / (sqrt(v_hat) + epsilon)
```

这样梯度自适应和权重衰减可以分别调节。

### 3. 工程使用

```python
decay = []
no_decay = []

for name, param in model.named_parameters():
    if not param.requires_grad:
        continue
    if param.ndim == 1 or name.endswith(".bias"):
        no_decay.append(param)
    else:
        decay.append(param)

optimizer = AdamW(
    [
        {"params": decay, "weight_decay": 0.1},
        {"params": no_decay, "weight_decay": 0.0},
    ],
    lr=learning_rate,
    betas=(0.9, 0.95),
    eps=1e-8,
)
```

具体超参数必须通过实验确定，不能把示例值当成所有模型的最优值。训练中通常还会配合：

- warmup；
- cosine/linear learning-rate decay；
- gradient clipping；
- bf16；
- 梯度累积；
- loss scaling 或数值异常监控。

### 4. AdamW 的代价

每个参数通常要维护一阶矩和二阶矩，优化器状态显存较高。超大模型可以使用：

- ZeRO/FSDP 分片；
- 8-bit Adam；
- Adafactor；
- CPU offload；
- LoRA/QLoRA。

### 5. 面试总结

AdamW 的核心是 Adam 自适应更新加解耦 weight decay。它比简单的 Adam+L2 更容易控制正则效果，是 Transformer 训练常用基线；工程上要同时说明参数分组、学习率调度、显存开销和数值稳定性。
