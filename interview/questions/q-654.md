# 做了哪些显存优化？

## 答案

回答“做了哪些显存优化”应先给出显存构成，再说明针对哪个瓶颈采用了什么方法。训练显存主要包括参数、梯度、优化器状态、激活、临时 buffer 和通信 buffer。

### 模型与数值精度

- 使用 bf16/fp16 混合精度，降低参数、激活和部分梯度占用；fp16 需配合 loss scaling。
- 对冻结基础模型使用 INT8/INT4，例如 QLoRA，但可训练参数和敏感计算仍保留较高精度。
- 使用 LoRA/Adapter 减少可训练参数、梯度和优化器状态；它不会消除基础权重和激活占用。

### 激活与序列

- Gradient checkpointing 不保存全部中间激活，反向时重算，以计算换显存。
- FlashAttention 避免显式保存完整 $n\times n$ 注意力矩阵。
- 按长度分桶、sequence packing 和动态 padding，减少 padding token。
- 降低 micro-batch，通过 gradient accumulation 保持有效 batch size。
- 控制上下文长度，并检查异常超长样本。

### 分布式与优化器

使用 ZeRO/FSDP 分片优化器状态、梯度和参数；模型仍放不下时再考虑 tensor/pipeline parallel 或 CPU/NVMe offload。8-bit optimizer 可减少 Adam 状态，但要验证收敛。

### 工程排查

使用 memory snapshot 或 profiler 区分 allocated、reserved、峰值临时张量和碎片。及时释放无用引用，验证 `zero_grad(set_to_none=True)`，避免把带计算图的 loss 或 hidden state长期存入列表。OOM 不一定是模型太大，也可能是数据长度异常或内存泄漏。

面试中最好量化优化前后的峰值显存、tokens/s、训练时间和指标变化，并说明最终方案的计算、通信与精度代价。
