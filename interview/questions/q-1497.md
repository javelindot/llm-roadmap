# PD分离相比传统的部署方法有什么优势？

## 答案

PD 分离即把大模型推理的 Prefill 与 Decode 放到不同实例或资源池中。传统共置部署由同一组 GPU 同时处理两阶段，容易互相干扰。

### 1. 两阶段特征不同

- **Prefill**：一次处理全部输入 token，矩阵计算密集，直接影响 TTFT；
- **Decode**：逐 token 生成，通常更受 KV Cache 容量和内存带宽限制，影响 TPOT/ITL。

长提示词的 Prefill 插入 Decode 批次时，可能抬高尾部 token 延迟。

### 2. 主要优势

1. **独立优化 SLO**：分别调优 TTFT 与 TPOT/ITL；
2. **隔离干扰**：长 Prefill 不阻塞持续 Decode；
3. **独立扩缩容**：按输入长度、输出长度和流量比例调整 P/D 资源；
4. **异构配置**：两侧可采用不同并行度、批处理策略甚至硬件；
5. **调度更清晰**：支持按 KV Cache、请求长度和队列状态路由；
6. **提高资源匹配度**：计算型 Prefill 与带宽/容量型 Decode 分别优化。

但 PD 分离不天然提升吞吐。vLLM 官方文档明确提醒，其主要价值是分别调节 TTFT/ITL 和控制尾延迟；KV Cache 传输、网络等待和额外调度可能抵消收益。

### 3. 额外成本

- Prefill 端要把 KV Cache 传给 Decode 端；
- 需要高带宽、低时延网络和 KV 生命周期管理；
- 故障恢复、重试、背压和可观测性更复杂；
- 两个资源池失衡时会出现排队或空闲；
- 跨节点传输可能增加首 token 延迟。

### 4. 跨境支付场景

支付客服或合规 RAG 往往输入长政策、交易证据和历史对话，输出相对较短。可以让 Prefill 池负责长上下文计算，Decode 池稳定生成，并对高优先级调查请求设置独立 SLO。

这只是推理基础设施优化，不能改变权限校验、数据驻留、审计和资金操作审批。

### 5. 面试总结

PD 分离的核心价值是隔离 Prefill/Decode 干扰、分别优化 TTFT 和 TPOT，并独立扩缩容。代价是 KV 传输与系统复杂度，所以必须用真实请求分布做端到端压测。
