# SFT和DPO的区别

## 答案

SFT 和 DPO 都会更新语言模型，但学习信号不同：

- **SFT** 学习“一个问题应该怎样回答”。
- **DPO** 学习“两个可用回答中更偏好哪一个”。

### 1. SFT

数据形式：

```text
(prompt, target_answer)
```

使用标准 Token-level Cross Entropy，最大化目标答案概率。

适合：

- 学习新任务格式；
- 注入领域表达和工作流；
- 建立基本指令遵循能力；
- 让模型先生成可用答案。

### 2. DPO

数据形式：

```text
(prompt, chosen, rejected)
```

DPO 比较当前模型相对 Reference Model 对 chosen/rejected 的偏好差，直接优化偏好排序。

适合：

- 调整回答风格、帮助性和安全性；
- 让模型在多个“都能回答”的候选中选择更优行为；
- 避免显式训练 Reward Model 和 PPO Critic。

### 3. 核心对比

| 维度 | SFT | DPO |
|------|-----|-----|
| 监督信号 | 标准答案 | 偏好对 |
| Loss | Cross Entropy | Pairwise Preference Loss |
| Reference Model | 不需要 | 通常需要 |
| 主要目标 | 学会任务 | 对齐偏好 |
| 数据构造 | 相对简单 | 要有高质量 chosen/rejected |
| 风险 | 模仿数据错误 | 学到长度、格式等伪偏好 |

### 4. 为什么通常先 SFT 再 DPO

如果模型连基础任务都不会，偏好优化缺少可靠候选。SFT 先建立能力分布，DPO 再在这个分布上调整选择倾向。

但如果已有强指令模型，也可以直接在其上做 DPO，不必为每个业务重复大规模 SFT。

**面试总结**：SFT 负责“能力和格式”，DPO 负责“选择和偏好”；二者不是互相替代，而是解决不同阶段的问题。
