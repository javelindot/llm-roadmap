# 4. 问Transformer encoder decoder结构上的不同

## 答案

原始 Transformer 是 Encoder-Decoder 结构。两者都由多层 Attention、残差连接、LayerNorm 和 FFN 组成，但注意力可见范围和职责不同。

### 1. Encoder

每个 Encoder Layer 主要包含：

```text
输入
 → Multi-Head Self-Attention
 → Add & Norm
 → Feed-Forward Network
 → Add & Norm
```

Encoder 的 self-attention 通常是双向的，每个 token 可以看到输入序列中的所有 token：

```text
Attention(Q,K,V) =
softmax(QKᵀ / √d_k + Mask)V
```

Padding Mask 用于屏蔽补齐位置，但没有因果约束。Encoder 的目标是将整段输入编码成包含上下文的 hidden states，适合理解、分类、抽取和检索表征。

### 2. Decoder

原始 Decoder Layer 主要包含：

```text
目标序列输入
 → Masked Multi-Head Self-Attention
 → Add & Norm
 → Cross-Attention
 → Add & Norm
 → Feed-Forward Network
 → Add & Norm
```

它比 Encoder 多两个关键差异：

1. **Causal Mask**：位置 `t` 只能看 `≤t` 的 token，不能看到未来目标；
2. **Cross-Attention**：Query 来自 Decoder 当前表示，Key/Value 来自 Encoder 输出，用于读取源序列信息。

可以写成：

```text
Q = decoder hidden states
K,V = encoder outputs
```

因此 Decoder 一边利用已生成的目标前缀，一边读取完整输入，逐 token 生成结果。

### 3. 训练与推理差异

训练时使用 teacher forcing，将右移后的真实目标序列一次性输入 Decoder。由于 causal mask，不同位置仍可并行计算。

推理时没有真实目标，需要自回归生成：

```text
y1 → y2 → y3 → ...
```

每一步只新增一个 token，通常缓存历史 Key/Value 减少重复计算。自回归依赖是 Decoder 推理延迟高于 Encoder 编码的主要原因。

### 4. 三种常见架构

- **Encoder-only**：BERT 类，双向理解，适合分类、NER、Embedding 和 Rerank；
- **Decoder-only**：GPT 类，因果 self-attention，统一做续写、对话和工具调用；没有独立 Cross-Attention；
- **Encoder-Decoder**：T5/BART 类，输入理解与输出生成分离，适合翻译、摘要和受条件约束的生成。

Decoder-only 模型虽然没有独立 Encoder，但会把 prompt 和已生成 token 放在同一因果序列中处理。Encoder-Decoder 则可以只编码一次长输入，Decoder 在生成时通过 Cross-Attention 重复读取。

### 5. 跨境支付场景的选型

- 付款意图分类、实体抽取：Encoder-only 或小型分类模型；
- 文档 embedding 和 rerank：双塔 Encoder 与 Cross-Encoder；
- 多语言规则摘要和客服回答：Decoder-only 或 Encoder-Decoder；
- 输入文档很长、输出较短的结构化抽取：Encoder-Decoder 可能更高效，但需要在目标硬件上实测。

架构选择最终由任务、延迟、上下文长度和部署生态决定，不能只根据“生成模型更通用”判断。
