# 我的RM怎么训练的？

## 答案

Reward Model 的任务是给定 prompt 和完整回答，输出一个标量分数，使人类偏好的回答得分更高。

### 数据构造

对同一 prompt 从一个或多个策略模型采样多个回答，由标注者进行成对比较或排序，形成 $(x,y_w,y_l)$，其中 $y_w$ 是 preferred answer，$y_l$ 是 rejected answer。数据要覆盖正常任务、安全边界、长短回答和困难样本，并按 prompt 划分训练与验证集，防止同一问题的改写泄漏。

### 模型与目标函数

常用做法是在预训练语言模型或 SFT 模型顶部增加一个 scalar value head，取 EOS 位置或最后一个有效 token 的隐藏状态输出奖励：

$$
r_\phi(x,y)=w^Th_{EOS}
$$

采用 Bradley-Terry 偏好模型，胜者优于败者的概率为：

$$
P(y_w\succ y_l\mid x)=\sigma(r_\phi(x,y_w)-r_\phi(x,y_l))
$$

损失函数为：

$$
\mathcal{L}_{RM}=-\log\sigma(r_\phi(x,y_w)-r_\phi(x,y_l))
$$

一个 prompt 有多个候选时，可构造多组 pair，也可使用 listwise ranking loss。训练时通常只比较同一 prompt 下的回答，绝对 reward 数值本身没有统一语义。

### 质量控制

- 建立明确标注规范，对分歧样本复审，记录标注者一致性。
- 平衡回答长度、格式和来源，避免模型只学到“越长越好”等捷径。
- 混入对抗样本，检查事实错误、奉承、关键词投机和 reward hacking。
- 可对多个 Reward Model 做集成或估计不确定性，对分歧大的样本追加标注。

### 评估与使用

离线至少看 held-out prompt 上的 pairwise accuracy，并按领域、安全类别和长度分桶；还要检查 reward 与长度、格式、模型来源的相关性。上线前用高奖励样本做人工盲测。

训练完成后，PPO 等算法将它作为序列级奖励，同时加入相对 SFT 或 reference model 的 KL 约束。Reward Model 只在训练分布附近可靠，策略持续变化会造成分布漂移，因此需要周期性收集新比较数据并重训。
