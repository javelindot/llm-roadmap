# DPO的损失函数是什么？

## 答案

**一句话**：DPO 的损失函数是一个二分类形式的偏好损失，让模型相对 reference model 更偏向 chosen，而不是 rejected。

### 1. 数据格式

DPO 训练数据通常是：

```text
(x, y_chosen, y_rejected)
```

其中：
- `x` 是问题或指令。
- `y_chosen` 是更好的回答。
- `y_rejected` 是更差的回答。

### 2. 损失函数

常见写法是：

```text
L_DPO = - E[ log sigmoid( beta * (
  (log pi_theta(y_chosen|x) - log pi_ref(y_chosen|x))
  - (log pi_theta(y_rejected|x) - log pi_ref(y_rejected|x))
) ) ]
```

也可以理解成：

```text
margin =
  [当前模型对 chosen 的偏好 - 参考模型对 chosen 的偏好]
  - [当前模型对 rejected 的偏好 - 参考模型对 rejected 的偏好]

loss = -log sigmoid(beta * margin)
```

### 3. 参数含义
- `pi_theta`：当前要训练的模型。
- `pi_ref`：参考模型，通常是 SFT 模型冻结得到。
- `beta`：控制偏好优化强度，也可以理解成 KL 约束强度相关的温度参数。
- `log pi(y|x)`：模型生成某个回答序列的 log probability。

### 4. 直观理解

DPO 不需要单独训练 reward model。它直接从偏好对里推导出目标：如果 chosen 比 rejected 好，那么当前模型相对 reference model，应该更提高 chosen 的概率、更降低 rejected 的概率。

## 结合项目谈（口语版 · 面试现场约 50 秒）

> 我口述 DPO loss 的时候，会先说它训练的是偏好对。比如在跨境支付里，同一个商户审核问题，chosen 回答能说明命中了哪些风控规则、需要补哪些材料、为什么暂时不能放行；rejected 回答可能只是简单说"拒绝"或者给了错误原因。
>
> DPO 做的事情就是比较这两个回答。它不是单纯让 chosen 概率变大，而是看当前模型相对参考模型，对 chosen 和 rejected 的相对偏好有没有拉开。公式里就是 chosen 的 logprob 减 reference 的 logprob，再减去 rejected 的同样差值，最后过一个 sigmoid 做偏好分类损失。
>
> 这里 beta 很关键，beta 大说明偏好拉得更猛，容易训偏；beta 小则更保守。支付场景我会比较保守，因为我们更关心回答稳定、证据准确、合规边界清楚，而不是让模型为了偏好数据过度迎合某种表达风格。

**自然带出去的钩子**：
- "DPO 优化的是相对偏好，不是简单让 chosen reward 绝对上升。"
- "reference model 的作用是防止模型偏离 SFT 能力太远。"
- "支付场景里 beta 会保守一点，避免模型为了偏好过度模板化。"
