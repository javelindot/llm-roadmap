# 手撕题hot100,中等题,合并 k 个升序链表

## 答案

合并 `k` 个升序链表的经典做法是维护大小不超过 `k` 的最小堆。堆中始终保存每条非空链表当前最小的头节点。

### 1. C++ 最小堆实现

```cpp
#include <queue>
#include <vector>

struct ListNode {
    int val;
    ListNode* next;
    explicit ListNode(int x) : val(x), next(nullptr) {}
};

struct Compare {
    bool operator()(const ListNode* a, const ListNode* b) const {
        return a->val > b->val;
    }
};

ListNode* mergeKLists(std::vector<ListNode*>& lists) {
    std::priority_queue<
        ListNode*,
        std::vector<ListNode*>,
        Compare
    > minHeap;

    for (ListNode* head : lists) {
        if (head != nullptr) {
            minHeap.push(head);
        }
    }

    ListNode dummy(0);
    ListNode* tail = &dummy;

    while (!minHeap.empty()) {
        ListNode* node = minHeap.top();
        minHeap.pop();

        if (node->next != nullptr) {
            minHeap.push(node->next);
        }

        tail->next = node;
        tail = node;
    }

    tail->next = nullptr;
    return dummy.next;
}
```

### 2. 为什么正确

每条链表内部有序，因此一条链表尚未合并部分的最小元素就是其当前头节点。所有链表当前头节点的最小值，也是全局剩余元素的最小值。弹出后只需把该节点的后继加入堆。

### 3. 复杂度

设所有链表节点总数为 `N`：

```text
时间：O(N log k)
额外空间：O(k)
```

每个节点最多入堆和出堆一次，堆大小最多为 `k`。

### 4. 其他方法

- 逐条合并：最坏可到 `O(Nk)`；
- 分治两两合并：`O(N log k)`，额外堆空间更少，也很常见；
- 收集全部节点再排序：`O(N log N)`，没有利用链表已有顺序。

### 5. 边界条件

- `lists` 为空；
- 所有链表为空；
- 只有一条链表；
- 节点值重复或为负；
- 若输入链表可能共享节点或有环，需要先定义行为，常规题默认互不相交且无环。

### 6. 面试总结

最小堆方案的核心不变量是“堆中保存每条链表当前最小节点”。总复杂度 `O(N log k)`，代码要正确处理空链表和 dummy head。
