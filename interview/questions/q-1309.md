# 介绍一下 Qwen布的这几版模型，都做了哪些贡献，包括数据、模型和训练

## 答案

Qwen 不是一条只改参数规模的模型线，它经历了“通用文本模型、多模态、代码/数学专项、MoE、推理模式、超长上下文、原生多模态和混合架构”的连续演进。回答时应按代际说贡献，不要背所有尺寸。

### 1. Qwen / Qwen1.5

- 建立中文、英文和多语言通用基座；
- 使用 Decoder-only Transformer、RoPE、SwiGLU、RMSNorm；
- 早期通过 NTK-aware interpolation 和 attention scaling 扩展上下文；
- Qwen-VL、Qwen-Audio 将能力扩展到视觉和音频；
- Qwen1.5 完善开源尺寸、GQA 和部署生态。

### 2. Qwen2

- 扩大高质量多语言、多领域训练数据；
- 改进 tokenizer 和多语言覆盖；
- 提供 Dense 与 MoE 型号；
- 普遍采用 GQA，降低 KV Cache；
- 强化代码、数学、长上下文和工具调用能力。

### 3. Qwen2.5

- 训练规模和数据质量继续提升；
- 通用、Coder、Math、VL、Omni 等专项系列更完整；
- 结构化输出、JSON、长文本、代码生成和 Agent 能力增强；
- Qwen2.5-1M 将长上下文扩展到百万 token 级，并配套稀疏注意力和推理优化。

### 4. Qwen3

- 在 Dense 和 MoE 模型上继续扩展；
- 将 thinking/non-thinking 统一在同一模型中，可按任务切换；
- 强化推理、代码、数学、Agent 和 MCP/工具使用；
- 扩大语言与方言覆盖；
- 后续 Qwen3-VL、Qwen3-Omni、Coder、Embedding/Reranker 形成完整模型族。

### 5. Qwen3-Next、Qwen3.5 与 Qwen3.6

截至 2026 年 6 月，官方公开模型线已继续演进：

- **Qwen3-Next**：探索更高效的混合注意力、极稀疏 MoE 和训练/推理效率；
- **Qwen3.5**：原生视觉语言、混合架构和更强 Agent 能力，强调效率与多模态统一；
- **Qwen3.6**：继续强化长上下文、Agent、代码与视觉推理，并提供 Dense/MoE、文本和多模态变体。

具体结构应以对应模型卡为准，不能把某一代的 GQA、全注意力或 MoE 配置套到所有 Qwen。

### 6. 数据和训练层面的共同趋势

- 数据从“更多 token”转向去重、过滤、合成、难度分层和可验证数据；
- Post-training 从 SFT + 偏好优化扩展到推理强化学习、工具环境和 Agent 轨迹；
- 长上下文从 RoPE 缩放扩展到专项训练、稀疏/混合注意力和系统优化；
- 多模态从外挂视觉编码器逐步走向原生联合训练与多层视觉特征融合。

### 7. 跨境支付选型

- 高频分类/召回不必用最大模型；
- 多语言商户客服可选中等尺寸 Instruct；
- KYC 单据、回单和截图理解需要 VL；
- 合规调查和代码/SQL Agent 要看工具调用、长上下文和权限控制；
- 生产选型还要评估许可证、数据驻留、量化后质量、吞吐和审计能力。
