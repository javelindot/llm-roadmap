# 如何评价Agent解决问题的质量？

## 答案

评价 Agent 不能只看最终回复是否流畅，而要同时评目标结果、执行过程、安全性、效率和真实业务价值。

### 1. 结果质量

- 任务成功率；
- 目标状态是否由环境/verifier 确认；
- 答案正确性、完整性和引用支持；
- 结构化输出或代码能否执行；
- 无答案时是否正确追问、拒答或转人工。

模型自评不能作为唯一成功标准。

### 2. 轨迹质量

- 计划是否覆盖必要步骤；
- 工具选择、参数和顺序；
- 是否使用最新、权威证据；
- 是否有重复调用、循环或无效反思；
- 失败后能否恢复；
- 是否在合理步骤内结束。

最终结果偶然正确但过程包含越权或错误工具调用，仍然是不合格。

### 3. 安全与治理

- 权限、隐私和数据隔离；
- 提示注入与工具结果污染；
- 不可逆动作是否确认和审批；
- 幻觉、错误承诺和有害输出；
- 审计、版本和可追溯性；
- 对抗与 OOD 场景。

高风险违规通常应作为硬失败，不能被其他得分抵消。

### 4. 效率与体验

- P50/P95/P99；
- 模型/工具调用次数、token 和成本；
- 首次响应和长任务进度反馈；
- 人工接管率、用户改写率和满意度；
- 单次成功解决成本；
- 稳定性和故障恢复。

### 5. 评测方法

```text
单元测试工具与节点
-> 固定轨迹/组件评测
-> 端到端黄金集
-> 仿真和故障注入
-> shadow traffic
-> canary/A-B
-> 线上事故回流
```

可执行 verifier 优先；人工评审用于开放式任务；LLM judge 需用人工黄金集校准，并检查位置偏差、长度偏差和同源模型偏差。

版本对比使用同一 case 配对、多随机种子和置信区间，按任务、难度、语言、工具数和风险切片。

### 6. 跨境支付场景

支付客服 Agent 的成功不仅是“用户满意”，还要订单状态正确、引用规则有效、金额币种一致、没有越权数据，并在需要时正确转人工。退款或出款动作必须以业务系统最终状态和审计记录作为 verifier。

### 7. 面试总结

Agent 质量是多维的：结果是否完成、轨迹是否正确、安全是否守住、成本是否合理、线上是否创造价值。最可靠的方法是环境 verifier + 轨迹诊断 + 人工校准 + 线上灰度。
