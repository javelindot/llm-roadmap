# 🔥🔥 多分类损失函数 ——交叉熵；交叉熵里每一项的意义是什么？

## 答案

多分类任务通常使用 Softmax 交叉熵。对一个样本，损失为：

```text
L = - Σ(k=1..C) y_k · log(p_k)
p_k = exp(z_k) / Σ(j=1..C) exp(z_j)
```

### 每一项的意义

- `C`：类别数量。
- `z_k`：模型对第 `k` 类输出的 logit，尚未归一化。
- `p_k`：Softmax 后第 `k` 类的预测概率，所有类别概率之和为 1。
- `y_k`：真实标签分布。普通分类中通常是 one-hot，正确类别为 1，其余为 0。
- `log(p_k)`：模型给真实类别分配概率的对数。
- 负号：由于概率在 0 到 1 之间，对数不大于 0，加负号后损失为非负。
- 求和：计算真实分布与预测分布在全部类别上的差异。

如果真实类别是 `t`，one-hot 标签会让公式化简为：

```text
L = -log(p_t)
```

模型给正确类别的概率越接近 1，损失越接近 0；概率越接近 0，损失越大。

### 为什么交叉熵适合分类

它等价于最大化训练数据真实标签的似然。在 Softmax + Cross Entropy 组合下，对 logit 的梯度非常简洁：

```text
∂L / ∂z_k = p_k - y_k
```

这表示预测概率高于目标就向下调，低于目标就向上调。

### 常见扩展与注意点

- **Label Smoothing**：把 one-hot 改成略平滑的分布，缓解过度自信。
- **类别不平衡**：可以给不同类别增加权重，或使用 Focal Loss。
- **软标签 / 蒸馏**：`y_k` 可以是教师模型给出的概率分布。
- 实现时应使用 `log_softmax` 或框架内置 `CrossEntropyLoss`，避免先算 Softmax 再取 log 导致数值不稳定。

**面试总结**：交叉熵衡量的是“真实分布希望概率放在哪里”与“模型实际把概率放在哪里”的差异；one-hot 情况下，本质就是惩罚模型没有给正确类别足够高的概率。
