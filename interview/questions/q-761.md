# 大模型的并行方式，多卡并行，张量并行，数据并行，流水线并行

## 答案

大模型多卡训练通常组合数据并行、张量并行、流水线并行和序列/上下文并行。选择依据是模型能否单卡放下、序列长度、节点拓扑和通信带宽。

### 数据并行

每卡持有完整模型，处理不同 mini-batch，反向后 All-Reduce 梯度。简单且计算扩展好，但参数、梯度和优化器状态在每卡重复。

ZeRO/FSDP 分片这些模型状态：ZeRO-1 分优化器，ZeRO-2 再分梯度，ZeRO-3/FSDP 连参数也分片，通过运行时 All-Gather/Reduce-Scatter 降低单卡显存。

### 张量并行

将单层大矩阵按列或行切到多卡，例如 MLP 与 Attention head 分片。每层内需要 All-Reduce/All-Gather，通信频繁，适合 NVLink/NVSwitch 域内。优点是单层放不下时仍能训练，缺点是跨节点效率敏感。

### 流水线并行

把不同层放到不同 stage，将 global batch 切成 micro-batch 流过各 stage。可容纳超深模型，但存在 pipeline bubble、调度复杂和 stage 负载不均。1F1B、interleaved schedule 可降低空泡。

### 序列/上下文并行

沿序列维分片激活、Norm 或 Attention，适合长上下文；需要通信 Q/K/V 或注意力中间量。Sequence Parallel 常与 Tensor Parallel 配合，Context Parallel 进一步拆分长序列 Attention。

### 组合

数据 × 张量 × 流水线构成 3D parallel，长上下文再加 context parallel。通常将高通信的 TP 放节点内，DP/PP 跨节点。评估需看 MFU、tokens/s、显存、通信占比、bubble 和容错，不是并行度越高越好。
