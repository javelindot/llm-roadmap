# 训练参数？

## 答案

“训练参数”较宽泛，可以先反问是模型参数量还是训练超参数。若需要直接回答，可按数据、优化器、批大小、精度和停止策略介绍，并说明参数来自基线与实验，而不是背一个固定数值。

### 核心超参数

- `learning_rate`：通常最敏感；全量微调一般比 LoRA 使用更小学习率。
- `batch_size`：应说明 micro-batch、gradient accumulation 和 global batch 的关系。
- `epochs/max_steps`：由数据量、重复率和验证集收敛决定，防止小数据过拟合。
- `warmup_ratio/steps` 与 scheduler：常用 linear 或 cosine decay。
- `weight_decay`、`betas`、`eps`：AdamW 的正则与动量参数。
- `max_grad_norm`：梯度裁剪，缓解异常 batch 导致的梯度爆炸。
- `max_seq_length`：直接影响样本截断、计算量和显存。

全局 batch 为：

$$
B_{global}=B_{micro}\times accumulation\times data\ parallel\ size
$$

改变 GPU 数或累积步数时，应保持全局 batch 与学习率策略的一致性。

### LoRA 与精度参数

LoRA 还需说明 rank、alpha、dropout 和 target modules；量化训练需说明 4-bit 类型、计算 dtype 和量化方案。混合精度通常优先 bf16，硬件不支持时再用 fp16 配合动态 loss scaling。

### 如何确定

先用公开配置或历史实验建立基线，做小规模 learning-rate range test 和少量组合实验，再扩大训练。持续观察 train/validation loss、梯度范数、有效 token 吞吐、显存、下游指标和过拟合。保存模型、优化器、scheduler、随机数和数据游标，以保证可恢复和可复现。

高质量回答应给出自己真实项目中的模型、数据规模、global batch、学习率、序列长度和选择依据；没有做过的数值不要虚构。
