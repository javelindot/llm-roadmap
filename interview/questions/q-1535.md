# 实现 Agent 任务成功率统计

## 答案

Agent 任务成功率必须先定义“一个任务”“进入分母的条件”和“成功判据”。不能简单把模型自报 `success` 的次数除以请求数。

### 1. 指标定义

以一次用户目标对应一个 episode：

```text
Task Success Rate
= 成功完成且通过 verifier 的有效任务数
/ 所有进入评测的有效任务数
```

成功通常要求同时满足：

- 最终目标状态达成；
- 自动 verifier 或人工标签通过；
- 必需工具结果正确；
- 没有安全、权限或不可逆副作用违规；
- 在最大步骤、时延和成本预算内。

超时、崩溃和无法恢复的工具错误通常计失败；测试环境故障可单独标为 invalid，但剔除规则必须预先定义。

### 2. Python 实现

```python
from collections import defaultdict
from math import sqrt


def wilson_interval(successes, total, z=1.96):
    if total == 0:
        return (0.0, 0.0)

    p = successes / total
    denominator = 1.0 + z * z / total
    center = (p + z * z / (2.0 * total)) / denominator
    margin = (
        z
        * sqrt(
            p * (1.0 - p) / total
            + z * z / (4.0 * total * total)
        )
        / denominator
    )
    return (max(0.0, center - margin), min(1.0, center + margin))


def is_success(record):
    return (
        record.get("goal_achieved") is True
        and record.get("verifier_pass") is True
        and record.get("safety_violation", False) is False
        and record.get("within_budget", True) is True
    )


def summarize_agent_runs(records):
    # infrastructure_invalid 必须由预先约定的规则产生。
    valid = [
        record
        for record in records
        if not record.get("infrastructure_invalid", False)
    ]

    def summarize(rows):
        total = len(rows)
        successes = sum(is_success(row) for row in rows)
        low, high = wilson_interval(successes, total)
        return {
            "total": total,
            "successes": successes,
            "success_rate": successes / total if total else 0.0,
            "ci95": [low, high],
            "tool_success_rate": (
                sum(row.get("all_required_tools_ok", False) for row in rows)
                / total
                if total
                else 0.0
            ),
            "safety_violation_rate": (
                sum(row.get("safety_violation", False) for row in rows)
                / total
                if total
                else 0.0
            ),
        }

    groups = defaultdict(list)
    for record in valid:
        groups[record.get("task_type", "unknown")].append(record)

    return {
        "overall": summarize(valid),
        "by_task_type": {
            name: summarize(rows)
            for name, rows in sorted(groups.items())
        },
    }
```

### 3. 还需要记录什么

- 首次尝试成功率与最终成功率；
- 工具选择、参数、顺序和执行成功率；
- 步骤数、重试、循环和人工接管；
- P50/P95/P99、token、工具费用；
- 按任务类型、难度、语言和风险分桶；
- 失败原因：规划、检索、工具、权限、生成或环境。

只有总体成功率会掩盖“简单题变多”造成的假提升。版本对比应使用同一 case 配对评测，并报告置信区间。

### 4. 跨境支付场景

“回答已处理”不算成功。退款调查任务需验证订单匹配、工具事实、币种金额、处理建议和审批状态；出现越权调用、虚构资金状态或重复退款，即使用户得到回复也判失败。

### 5. 面试总结

任务成功率的难点是可执行成功定义和分母治理。生产上应由 verifier/环境状态判定，并同时报告置信区间、任务切片、轨迹指标和安全失败。
