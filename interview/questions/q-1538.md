# 手撕RoPE（旋转位置编码）

## 答案

RoPE（Rotary Position Embedding）把 Q/K 的每两个维度视为二维向量，根据 token 位置旋转不同角度，使 Attention 内积自然包含相对位置信息。

### 1. 数学形式

对第 `i` 个二维子空间：

```text
theta_i = base^(-2i / d)
angle = position * theta_i
```

旋转：

```text
[x_2i']     [ cos(angle)  -sin(angle)] [x_2i]
[x_2i+1'] = [ sin(angle)   cos(angle)] [x_2i+1]
```

RoPE 应用于 Q 和 K，不作用于 V。

### 2. PyTorch 实现

```python
import torch


def apply_rope(
    q: torch.Tensor,
    k: torch.Tensor,
    position_ids: torch.Tensor | None = None,
    base: float = 10000.0,
):
    """
    q: [B, Hq, T, D]
    k: [B, Hk, T, D]
    position_ids: [B, T]
    """
    if q.size(0) != k.size(0) or q.size(2) != k.size(2):
        raise ValueError("q and k must share batch and sequence length")
    if q.size(-1) != k.size(-1) or q.size(-1) % 2 != 0:
        raise ValueError("q/k head_dim must be equal and even")

    batch, _, seq_len, head_dim = q.shape
    device = q.device

    if position_ids is None:
        position_ids = torch.arange(
            seq_len, device=device
        ).unsqueeze(0).expand(batch, -1)

    inv_freq = 1.0 / (
        base
        ** (
            torch.arange(0, head_dim, 2, device=device).float()
            / head_dim
        )
    )
    angles = position_ids.float().unsqueeze(-1) * inv_freq
    cos = angles.cos().unsqueeze(1)  # [B, 1, T, D/2]
    sin = angles.sin().unsqueeze(1)

    def rotate(x):
        original_dtype = x.dtype
        pairs = x.float().reshape(
            x.size(0), x.size(1), x.size(2), head_dim // 2, 2
        )
        even = pairs[..., 0]
        odd = pairs[..., 1]
        rotated = torch.stack(
            [even * cos - odd * sin, even * sin + odd * cos],
            dim=-1,
        )
        return rotated.flatten(-2).to(original_dtype)

    return rotate(q), rotate(k)
```

### 3. 为什么表达相对位置

设位置 `m、n` 的旋转矩阵为 `R_m、R_n`：

```text
(R_m q)^T (R_n k)
= q^T R_m^T R_n k
= q^T R_(n-m) k
```

内积只依赖位置差 `n-m`，因此在绝对旋转中注入了相对位置信息。

### 4. 工程注意

- head_dim 必须为偶数；
- 增量解码使用绝对 position id，不能每步从 0 开始；
- cos/sin 通常预计算和缓存；
- GQA/MQA 的 Q/K head 数可不同，但 head_dim 相同；
- 长上下文扩展涉及 position interpolation、NTK/YaRN 等 scaling 与再训练，不能只扩大 `max_position_embeddings`；
- 不同项目可能使用 interleaved 或 split-half 旋转布局，权重与实现必须一致。

### 5. 面试总结

RoPE 的核心是按位置旋转 Q/K 的二维维度对，并利用 `R_m^T R_n = R_(n-m)` 让 Attention 内积感知相对位置。手写时要处理位置、广播、精度和布局。
