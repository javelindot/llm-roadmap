# 8、从模型角度介绍特征交叉

## 答案

特征交叉用于表示“多个特征组合后才有意义”的关系，例如“某国家 + 某币种 + 某支付方式”。可按显式、隐式和注意力交叉介绍。

### 1. 线性与人工交叉

Wide/线性模型直接加入 \(x_ix_j\) 或哈希组合，可解释、稳定，但依赖人工经验且组合爆炸。

### 2. FM 家族

FM 用 embedding 内积表示二阶交叉：

\[
\sum_{i<j}\langle v_i,v_j\rangle x_ix_j
\]

DeepFM 将 FM 低阶交叉与 MLP 高阶交互结合；FFM 为不同 field 学不同 embedding，表达更强但参数大。

### 3. 显式高阶网络

- DCN/DCNv2：用 Cross Network 逐层构造有界阶数的显式交叉；
- xDeepFM：CIN 在向量级显式生成高阶交互；
- PNN：将内积/外积作为 MLP 输入。

### 4. 隐式与注意力

MLP 可隐式学习高阶关系，但难解释且数据效率不一定高。AutoInt 用 Self-Attention 建模 field 间交互；FiBiNET 使用 SENet 和双线性交互；DIN 则做候选与历史行为的目标相关交互。

### 5. 选型

比较 AUC/LogLoss 之外，还要看参数、延迟、稀疏特征、新 ID 和线上增量收益。强模型不能补救泄漏、错误交叉和不一致特征。

跨境支付风控可显式建模国家×币种×通道、设备×账户等组合，但高风险规则应保持可解释，深度交叉用于补充而非取代规则。

### 面试总结

特征交叉从人工/Wide、FM 二阶，到 DCN/xDeepFM 显式高阶，再到 MLP/Attention 隐式交互。选择取决于数据稀疏性、可解释性、模型容量和线上延迟。
