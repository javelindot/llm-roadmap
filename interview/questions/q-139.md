# Word2Vec 的核心思想？两种模型是什么？

## 答案

**一句话**：Word2Vec 的核心思想是"**词的语义由它的上下文决定**"（分布式假说），两种模型 **CBOW 和 Skip-gram**——前者用上下文预测中心词，后者反过来用中心词预测上下文。

### 1. 核心思想（Mikolov 2013）
- **分布式假说**：相似词出现在相似上下文里
- 学习目标：让"语义相近词"的向量也相近
- 不再用 one-hot 稀疏表达，而是低维稠密向量

### 2. CBOW（Continuous Bag-of-Words）
**思路**：用上下文词预测中心词
```
[the, cat, ___, on, the] → 预测 "sits"
```
- 上下文词 embedding 求平均/求和
- 投影到词表大小，softmax 选中心词
- **训练快**，对小数据集友好

### 3. Skip-gram
**思路**：用中心词预测上下文
```
"sits" → 预测 [the, cat, on, the]
```
- 中心词 embedding → softmax → 词表
- 对每个上下文词独立预测
- **训练慢但对稀有词友好**

### 4. 两个加速技巧
**Hierarchical Softmax**
- 用 Huffman 树代替全词表 softmax
- 复杂度 O(V) → O(log V)

**Negative Sampling**
- 不算全词表 softmax，只对正样本 + K 个负样本算 logistic
- K = 5-20
- 更快，质量也好

### 5. CBOW vs Skip-gram 对比
| 维度 | CBOW | Skip-gram |
|------|------|-----------|
| 训练速度 | 快 | 慢（每词多次） |
| 高频词效果 | 好 | 差 |
| 低频词效果 | 差 | **好** |
| 适用 | 大数据 | 小数据 + 罕见词 |

### 6. Word2Vec 的著名性质
$$\vec{king} - \vec{man} + \vec{woman} \approx \vec{queen}$$
- 类比关系（capital-of, gender, ...）
- 聚类（相似词聚在一起）

### 7. 局限（被现代取代的原因）
- **一词多义无法处理**：bank（银行/河岸）共用一个向量
- **静态向量**：上下文无关
- **OOV 问题**：训练时没见过的词无法处理
- → 被 ELMo / BERT / LLM 的上下文 embedding 取代

### 8. 历史地位
- 让 embedding 思想普及
- 启发 GloVe、FastText、ELMo
- 是 NLP 神经网络时代的开山之作

### 9. 一句话总结
**CBOW 上下文猜中心、Skip-gram 中心猜上下文**——核心是"用上下文学语义"，是 NLP embedding 的开山祖师。

## 结合项目谈（口语版 · 约 20 秒）

> Word2Vec 在我们跨境支付里**直接用不上**——LLM 时代的 embedding 全是上下文相关的（BERT 风格双塔），早就替代了静态词向量。
>
> 但有一个**遗产场景**仍在用——**业务术语聚类**。我们想找"同一意图的不同说法"，比如"退款 / 退付 / 退单 / refund"这种近义业务术语。一开始用 BERT embedding 算余弦相似度，但因为我们要的是"词级"语义而不是"句级"，BERT 句向量反而不准。后来用历史一年客服对话语料训了一个**领域 Word2Vec**（Skip-gram + 负采样），术语相似度准多了，发现的近义业务术语扩了 30%。
>
> 这个有点反直觉——**Word2Vec 已经被淘汰，但在"纯词级聚类"任务上反而比 BERT 更适合**，因为它没有"上下文歧义"的负担。
>
> 还有一个借鉴：**Word2Vec 的对比学习思想**（正样本 vs 负样本）在现代 embedding 训练里仍然用——bge-large 微调时的 InfoNCE 损失本质就是 Word2Vec 负采样的现代变体。
>
> 一句话：**面试要会讲 CBOW/Skip-gram 区别和分布式假说**，但工业落地里 Word2Vec 已经是博物馆藏品——除了术语聚类这种少数场景。

**🪝 顺嘴带出去的钩子**：
- "业务术语聚类用 Word2Vec 反而比 BERT 好的原因可以聊。"
- "InfoNCE 和 Word2Vec 负采样的演化关系可以展开。"
- "Skip-gram 对低频词友好这点在我们业务里有体现。"
