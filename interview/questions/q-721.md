# 为什么大模型有few-shot能力?

## 答案

大模型的 few-shot 能力通常指不更新参数，只在上下文中给几个输入输出示例，模型就能推断任务规则并完成新样本。这来自预训练规模、Transformer 的上下文计算和数据中的任务结构共同作用。

### 预训练提供了任务先验

大规模语料包含问答、翻译、分类、代码、表格和示例演示等模式。next-token prediction 迫使模型学习语言规律、概念和“根据前文继续规则”的能力。模型越大、数据越丰富，越可能在参数中形成可组合的任务表示。

### In-Context Learning

示例放入 prompt 后，Self-Attention 可以比较新 query 与示例的输入、标签和格式，在隐藏状态中构造临时任务表示。有研究将其解释为模型在前向过程中进行隐式贝叶斯推断、检索匹配或近似梯度下降，但这些是互补解释，并非所有模型都严格执行某一种算法。

### 为什么规模重要

更大的模型有更强容量识别示例中的抽象模式；更长、更稳定的预训练也增加了包含任务演示的数据概率。能力通常随规模平滑提升，某些基准上看似突然出现还可能受到指标阈值影响。

### 局限

few-shot 对示例顺序、标签词、格式和分布非常敏感，可能只模仿表面模式；它不等于真正学会新知识，也不会把能力永久写入参数。示例有误会直接诱导错误。

实践中应选择覆盖边界且格式一致的示例，控制长度，并与 zero-shot、检索示例和微调基线比较。若任务稳定且数据充足，SFT 往往比长期依赖昂贵上下文更可靠。
