# 用bert做下游任务时，输入有哪些embedding？

## 答案

BERT 的输入表示是三种 embedding 的逐元素相加：

$$
E_i=E_{token,i}+E_{segment,i}+E_{position,i}
$$

相加后的表示再经过 LayerNorm 和 dropout，送入 Transformer Encoder。

### Token Embedding

WordPiece tokenizer 将文本切成 subword token，每个 token id 查词嵌入表。序列通常包含 `[CLS]`、`[SEP]`、`[PAD]` 等特殊 token。`[CLS]` 的最终隐藏状态常用于句级分类，每个 token 的隐藏状态用于序列标注。

### Segment/Token Type Embedding

用于区分句子 A 与句子 B。原始 BERT 预训练的句对输入中，第一段使用 type 0，第二段使用 type 1，并由 `[SEP]` 分隔。单句任务通常全部为 0。

它不是 attention mask。Attention mask 用于告诉模型哪些位置是真实 token、哪些是 padding，通常不会作为一项 embedding 相加。

### Position Embedding

原始 BERT 使用可学习的绝对位置 embedding，每个位置查表。它让没有递归结构的 Self-Attention 感知 token 顺序。最大序列长度受位置表大小限制，超出时需要扩展并重新训练或采用其他位置方案。

### 下游任务扩展

基础 BERT 只有上述三类输入 embedding。具体任务可以额外加入实体、领域、视觉或布局 embedding，但这属于模型扩展，不是原始 BERT 的固定组成。面试中还应说明 `input_ids`、`token_type_ids`、`attention_mask` 的区别。
