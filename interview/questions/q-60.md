# LLMs都使用哪种激活函数？

## 答案

**一句话**：LLM 常见激活函数包括 GELU、SiLU/Swish，以及现代大模型里常见的 SwiGLU、GeGLU 等门控激活。

### 核心回答
- **ReLU**：简单高效，但负区间直接为 0，表达有限。
- **GELU**：BERT/GPT 早期常用，平滑，效果稳定。
- **SiLU/Swish**：`x * sigmoid(x)`，平滑且保留负区间信息。
- **GLU 系列**：引入门控机制，例如 GeGLU、SwiGLU。
- **SwiGLU**：LLaMA 等模型常用，通常在 FFN 中效果好，但参数和计算略有变化。

现代 LLM 越来越偏向用门控激活，因为表达能力更强，训练效果更好。

### 1. 为什么 LLM 常用门控激活

传统 FFN 可以理解成对每个 token 做非线性变换。门控激活如 SwiGLU 会多一条门控分支，让模型决定哪些信息应该通过、哪些应该抑制。

相比普通 GELU，SwiGLU 在很多大模型中表现更好，原因可以直观理解为：

- 表达能力更强。
- 信息选择更灵活。
- 在大规模训练中经验效果好。

### 2. 常见模型里的激活函数

- BERT：常见 GELU。
- GPT 早期：常见 GELU。
- LLaMA 系：常见 SwiGLU。
- 一些现代模型：GeGLU、SwiGLU、SiLU 变种。

### 3. 面试总结

激活函数是模型架构的一部分，业务微调一般不会单独替换。回答时要讲出 GELU 到 SwiGLU 的演进，以及门控激活为什么更适合现代 LLM。

### 4. 结合跨境支付项目怎么说（面试口径）

> 面试里我会把它和模型选型联系起来。比如我们拿 LLaMA 或 Qwen 类模型做支付领域微调，它里面常见 RMSNorm、RoPE、SwiGLU，这些结构本身就是为了稳定大规模训练。
>
> 业务侧不太会单独替换激活函数，但理解这些结构能帮助我判断不同基座模型的差异。比如同样是 7B，有的模型中文金融术语更好，有的工具调用更好，有的长上下文更稳。
>
> 所以在项目里我不会为了激活函数本身做改造，而是把它作为模型选型背景，真正评估还是看风控问答、商户审核、客服回复这些业务测试集表现。

**钩子**：激活函数是架构基础，业务微调更关注基座能力和数据质量。
