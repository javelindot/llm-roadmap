# 不同类型的Attention机制之间有什么区别？

## 答案

Attention 可按 Q/K/V 来源、可见范围、KV 共享、连接稀疏性和计算实现分类。不同名称常处于不同层次，不能把 FlashAttention、GQA 和 Cross-Attention直接当作互斥方案。

### 1. 按 Q/K/V 来源

**Self-Attention**

```text
Q,K,V <- 同一序列
```

用于序列内部建模。

**Cross-Attention**

```text
Q <- 目标序列
K,V <- 外部上下文
```

用于 Encoder-Decoder、多模态或 Query-Document 交互。

### 2. 按可见范围

- Bidirectional：每个 token 看整段，BERT/Encoder 常用；
- Causal：只能看过去，Decoder LLM 常用；
- Prefix LM：前缀双向、生成段因果；
- Local/Sliding Window：只看附近窗口；
- Block Sparse/Dilated/Global Token：按规则稀疏连接；
- Full Global：所有位置两两交互。

Full Attention 质量灵活但复杂度 `O(S^2)`；局部/稀疏降低长序列成本，但可能丢远程关系。

### 3. 按 Head 与 KV 组织

| 类型 | Query heads | KV heads | 特点 |
|---|---:|---:|---|
| MHA | H | H | 表达强，Cache 最大 |
| MQA | H | 1 | Cache 最小，共享最强 |
| GQA | H | G | 质量/成本折中 |
| MLA | H | latent 压缩 | 低秩压缩 KV |

Multi-Head 描述多套 Attention；MQA/GQA/MLA 主要优化 KV 表示和推理。

### 4. 按打分机制

- Scaled Dot-Product：`softmax(QK^T/sqrt(d))V`；
- Additive Attention：小网络计算 Q/K 相容性；
- Linear/Kernel Attention：重排或近似计算以降低复杂度；
- Retrieval/Memory Attention：访问外部或压缩记忆；
- Deformable Attention：只采样少量位置，视觉任务常见。

线性/近似 Attention 需要验证质量和 kernel，不能只看渐进复杂度。

### 5. 按实现优化

**FlashAttention**

是精确 Attention 的 IO-aware kernel，通过分块和 online softmax 减少 HBM 读写，不改变可见关系。

**PagedAttention**

是 KV Cache 内存管理方法，不是新的 Attention 数学机制。

**Ring/Context Parallel**

是在多 GPU 间分布长序列计算。

这些可以与 causal、GQA 或 sliding-window 组合。

### 6. 多模态 Attention

- Early fusion：视觉/文本 token 在同一 Self-Attention；
- Cross-Attention：文本 Query 读取视觉特征；
- Co-Attention：两种模态相互查询；
- Perceiver/Resampler：用少量 latent Query 压缩媒体 token。

### 7. 如何选择

- Encoder 理解：双向 Self-Attention；
- 自回归生成：causal Self-Attention；
- 条件生成/多模态：Cross-Attention 或 early fusion；
- 高并发 Decode：GQA/MQA/MLA；
- 长上下文：FlashAttention + local/sparse/context parallel；
- 选择最终由质量、长度、硬件和 SLO 决定。

### 8. 面试总结

先区分语义结构、KV 组织与 kernel 实现：Self/Cross 讲来源，Causal/Local 讲可见范围，MHA/GQA 讲 heads，FlashAttention 讲 IO 优化，PagedAttention 讲缓存管理。
