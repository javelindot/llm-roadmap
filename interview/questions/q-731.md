# 手撕对比学习的loss计算

## 答案

下面手写一个 SimCLR 风格的对称 InfoNCE loss。输入是同一批样本的两种增强表示 `z1`、`z2`。

```python
import torch
import torch.nn.functional as F


def info_nce_loss(z1, z2, temperature=0.07):
    if z1.shape != z2.shape:
        raise ValueError("z1 and z2 must have the same shape")

    batch = z1.size(0)
    z1 = F.normalize(z1, dim=-1)
    z2 = F.normalize(z2, dim=-1)
    z = torch.cat([z1, z2], dim=0)              # [2B, D]

    logits = z @ z.T / temperature              # [2B, 2B]
    self_mask = torch.eye(
        2 * batch, dtype=torch.bool, device=z.device
    )
    logits = logits.masked_fill(self_mask, float("-inf"))

    # i 的正样本是另一视图中的同一个样本
    labels = torch.arange(2 * batch, device=z.device)
    labels = (labels + batch) % (2 * batch)
    return F.cross_entropy(logits, labels)
```

### 对应公式

对 anchor $i$ 和正样本 $j$：

$$
\ell_{i,j}=-\log
\frac{\exp(sim(z_i,z_j)/\tau)}
{\sum_{k\ne i}\exp(sim(z_i,z_k)/\tau)}
$$

代码把两个方向都作为 anchor，因此是对称 loss。自身相似度被 mask，另一视图同索引是唯一正样本，其余 $2B-2$ 个向量为负样本。

### 注意点

embedding 要归一化，否则模型可通过增大范数降低 loss。temperature 太小会使分布过尖、对 hard negative 敏感；batch 太小负样本不足。分布式训练时应 all-gather 其他卡 embedding，并确保梯度策略正确。若同类样本也可为正样本，应改为 supervised contrastive loss，不能强行当负例。
