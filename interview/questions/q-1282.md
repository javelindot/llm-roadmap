# diffusion过程

## 答案

Diffusion 模型包含两个相反过程：前向扩散逐步向真实数据加噪，直到接近标准高斯噪声；反向过程学习逐步去噪，从随机噪声生成数据。

### 1. 前向扩散

从真实样本 `x_0` 开始，每一步加入少量高斯噪声：

```text
q(x_t | x_{t-1})
= Normal(
    sqrt(1 - beta_t) * x_{t-1},
    beta_t * I
)
```

定义：

```text
alpha_t = 1 - beta_t
alpha_bar_t = product_{s=1..t}(alpha_s)
```

可以直接从 `x_0` 采样任意时间步：

```text
x_t = sqrt(alpha_bar_t) * x_0
      + sqrt(1 - alpha_bar_t) * epsilon

epsilon ~ Normal(0, I)
```

这个闭式表达避免训练时真的逐步加噪 `t` 次。

### 2. 训练目标

随机采样：

1. 真实样本 `x_0`；
2. 时间步 `t`；
3. 高斯噪声 `epsilon`；
4. 计算带噪样本 `x_t`；
5. 网络预测噪声。

经典 DDPM 的简化损失：

```text
L = E[
    ||epsilon - epsilon_theta(x_t, t, condition)||^2
]
```

模型通常采用 U-Net 或 DiT，并注入时间步 embedding。条件生成还会加入文本、类别或其他条件。除了预测 `epsilon`，也可以预测 `x_0` 或使用 `v-prediction`，三者对应不同参数化与训练稳定性。

### 3. 反向去噪

生成时从：

```text
x_T ~ Normal(0, I)
```

开始，模型在每个时间步估计噪声或 score，再根据 scheduler 计算 `x_{t-1}`：

```text
p_theta(x_{t-1} | x_t)
= Normal(mu_theta(x_t, t), sigma_t^2 * I)
```

不断执行：

```text
x_T -> x_{T-1} -> ... -> x_1 -> x_0
```

最终得到生成样本。训练可以随机抽一个时间步并行进行，但采样通常依赖逐步迭代，因此推理比一次前向的生成模型慢。

### 4. DDPM、DDIM 和 Latent Diffusion

**DDPM**

- 随机采样；
- 步数通常较多；
- 理论与训练目标清晰。

**DDIM**

- 可使用确定性或低随机性的轨迹；
- 可以减少采样步数；
- 在质量和速度之间折中。

**Latent Diffusion**

- 先用 VAE 把图像压缩到 latent；
- 在 latent 空间扩散和去噪；
- 最后由 VAE decoder 还原；
- 显著降低高分辨率训练与推理成本。

### 5. Classifier-Free Guidance

训练时随机丢弃条件，使同一个模型同时学到 conditional 和 unconditional 预测。推理时组合：

```text
epsilon_guided
= epsilon_uncond
  + w * (epsilon_cond - epsilon_uncond)
```

较大的 `w` 通常增强条件一致性，但过大会降低多样性、放大伪影或导致过饱和。

### 6. 工程重点

- noise schedule 和 sampler；
- 训练分辨率、数据清洗与 caption 质量；
- EMA、混合精度和梯度检查点；
- attention 显存；
- 采样步数、guidance scale；
- FID、CLIP score 与人工评审；
- 生成安全、版权和敏感信息。

跨境支付中更适合把 Diffusion 用于非真实业务数据的视觉设计或合成数据研究，不应生成或篡改真实回单、身份证件和交易凭证，也不能把合成材料混入生产审核链路。
