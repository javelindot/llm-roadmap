# 如何利用大模型本身相关的方法，指标去判断数据，筛选数据

## 答案

可以让大模型参与质量评分、标签、去重和困难样本发现，但不能让单个 Judge 决定全部数据。推荐“规则粗筛、小模型/统计筛选、LLM 精筛、人工抽检”的级联。

### 1. 基础与统计指标

- 长度、语言、乱码、HTML/代码比例；
- exact/MinHash/Embedding 近重复；
- PII、毒性、版权和来源许可；
- n-gram 重复、模板占比、答案泄漏；
- 基础模型 perplexity 或 loss：过高可能是噪声，过低可能是重复/背诵；
- embedding 聚类检查覆盖、多样性和异常点。

### 2. LLM 评分

按明确 rubric 逐维评：

- 正确性、相关性、完整性；
- 指令与答案是否匹配；
- 推理是否自洽，结论能否验证；
- 风格、格式和安全；
- 难度、能力标签和潜在数据污染。

要求输出结构化分数、理由和引用证据；随机抽换顺序，减少位置/长度偏差。

### 3. 一致性与可验证性

- 多次采样或多 Judge 看一致性；
- 数学、代码和工具任务用执行器验证；
- 用 teacher-student disagreement 找困难样本；
- loss influence、gradient similarity 或 data attribution 可估计训练价值；
- 按能力和数据源配额采样，避免只留下“容易、同质”的高分数据。

### 4. 校准与闭环

建立人工黄金集，测 Judge precision/recall 和分桶误差；只自动删除高置信低质数据，边界样本人工复核。记录原始数据、评分模型、prompt 和版本，支持回滚。

跨境支付数据还要优先做 PII、地域存储和权限治理；事实与合规标签由权威系统验证，不交给 LLM 猜。

### 面试总结

LLM 适合做高语义质量筛选和能力标注，但应与去重、perplexity、执行验证、多 Judge 和人工黄金集结合。数据筛选目标是高质量与覆盖平衡，而不是只保留 Judge 最喜欢的风格。
