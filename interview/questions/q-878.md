# 商品塔接入多模态 encoder 后为何离线指标下滑？定位到哪个环节？

## 答案

多模态并不必然提升商品塔。如果图像/文本 encoder 的噪声、对齐或融合破坏了原有协同 ID 表示，离线指标会下降。应按数据、表示、融合、训练和评估逐层定位。

### 1. 先做可归因消融

保持数据、负样本和训练步数一致，比较：

1. 原 ID/结构化商品塔；
2. 只加文本；
3. 只加图像；
4. 图文融合；
5. 冻结 encoder；
6. 解冻顶部/全量 encoder。

再对 head/tail、冷启动、类目、缺失模态分桶。若总体下降但新品提升，可能是目标权衡而非完全失败。

### 2. 数据问题

- 图片与 SKU 错配、主图变更或一图多商品；
- 标题营销噪声、属性缺失、OCR 乱码；
- 训练和推理预处理不一致；
- 多模态特征缓存陈旧；
- 缺失模态用零向量导致模型学到伪特征；
- 时间切分后新品比例改变。

### 3. 表示与融合

- encoder 来自通用图文任务，与推荐目标不一致；
- embedding 尺度、Norm 或维度不匹配；
- 简单 concat 后参数激增，压过协同特征；
- 融合 gate 饱和，只使用某一模态；
- 投影层随机初始化破坏已有商品空间；
- L2 normalize、temperature 或 ANN 距离配置改变。

### 4. 训练问题

- 多模态 encoder 学习率过大导致灾难性漂移；
- 冻结后又无法适配电商语义；
- batch 变小、负样本减少或训练 token 不公平；
- false negative 增多，同款商品被当负例；
- 多任务 loss 权重冲突；
- 在线 item embedding 与用户塔版本不一致。

### 5. 诊断指标

检查对齐/均匀性、embedding norm、近邻可视化、模态缺失鲁棒性、梯度范数、gate 分布、ANN Recall 和端到端 Recall/NDCG。必要时用 teacher distillation 保留旧塔能力。

### 面试总结

先做单模态和冻结/解冻消融，定位数据、表示、融合、训练还是索引问题。多模态常先提升冷启动而损害热门协同信号，需用门控、蒸馏和分桶指标平衡。
