# 召回损失函数设计介绍一下？

## 答案

召回模型通常学习 query/user 与 item/document 的向量相似度。损失选择取决于标签形式、负样本来源和最终召回目标。

### 1. Pointwise

对每个 pair 做 BCE：

\[
\mathcal L=-y\log\sigma(s)
-(1-y)\log(1-\sigma(s))
\]

实现简单，但大量负例独立处理，不能直接强调正负候选间排序。

### 2. Pairwise

BPR：

\[
\mathcal L=-\log\sigma(s^+-s^-)
\]

也可用 hinge/triplet：

\[
\max(0,m-s^++s^-)
\]

适合偏好排序，效果高度依赖负样本。

### 3. Sampled Softmax / InfoNCE

\[
\mathcal L_i
=-\log
\frac{\exp(s(q_i,d_i^+)/\tau)}
{\sum_j\exp(s(q_i,d_j)/\tau)}
\]

双塔常使用 batch 内其他文档作负例，计算高效；但同一 batch 可能存在 false negative，热门 item 出现频率也会造成采样偏差。

### 4. Listwise 与多任务

对曝光候选列表做 softmax/listwise loss，更贴近排序；可同时优化点击、购买、停留等目标。召回阶段还可加入蒸馏 loss，让双塔学习 cross-encoder/reranker 分数。

### 5. 负样本设计

- 随机负例用于覆盖空间；
- 同类目或 ANN hard negative 提升区分度；
- 从真实曝光未点击中取负例，但要考虑位置偏差；
- 去除同义文档、重复商品等 false negative；
- 使用 popularity correction 或 sampled-softmax 校正采样概率。

跨境支付知识检索应特别加入错误码相似但国家/版本不同的 hard negative，并确保过期规则不被当正例。

### 面试总结

召回常用 BCE、BPR/triplet 和 InfoNCE/sampled softmax。实际效果往往更受负样本、false negative、采样偏差和温度影响，最终应按 Recall@K、长尾切片与线上候选收益评估。
