# 是否做过意图识别？如果要做意图识别，可以怎么实现？

## 答案

意图识别的目标是把用户表达映射到业务动作，例如“查询订单”“申请退款”“修改地址”，并识别无法处理的新意图。

### 1. 先定义任务

- 单标签还是多标签；
- 是否需要层级意图；
- 是否依赖多轮上下文；
- 是否同时抽取槽位；
- 是否需要识别 OOS（Out-of-Scope）。

标签体系应互斥、可解释，并给出正例、反例和边界说明。

### 2. 常见实现方案

#### 规则与关键词

适合冷启动、高风险和强确定性场景。优点是可控，缺点是召回和维护成本有限。

#### 监督分类模型

使用 BERT/轻量 Encoder 做分类。数据充足时延迟低、结果稳定，是生产系统常见主力。

#### Embedding 检索

把输入与每个意图的描述和示例做相似度匹配。适合少样本和意图快速扩展。

#### LLM Few-shot / Structured Output

适合复杂语义、层级分类和低频意图，但要控制成本、延迟和输出稳定性。

#### 混合方案

```text
高置信规则
→ 分类模型 / Embedding
→ 低置信样本交给 LLM
→ 仍不确定则澄清或转人工
```

### 3. 数据与训练

- 收集真实表达，而不只是标准问句。
- 做难负样本，特别是相似意图之间的边界。
- 对类别不平衡使用重采样、类别权重或 Focal Loss。
- 新增 OOS 和模糊意图样本。
- 多轮场景将历史摘要、当前状态和缺失槽位作为特征。

### 4. 评测与上线

- Macro-F1、各意图 Recall、混淆矩阵。
- OOS Recall 和误接收率。
- 置信度校准与拒识阈值。
- 线上澄清率、转人工率、任务完成率。

**面试总结**：意图识别不是单纯的文本分类，还要处理标签治理、低置信拒识、上下文和业务动作路由。
