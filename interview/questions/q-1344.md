# transformer中softmax的作用、问题及解决方法

## 答案

Softmax 在 Transformer 中主要有两个位置：Attention 中把相似度变成归一化权重；输出层把词表 logits 变成概率分布。它提供可微竞争和概率解释，但也有数值、梯度、长序列和计算方面的问题。

### 1. Attention 中的作用

```text
S = Q K^T / sqrt(d_k) + Mask
A_ij = exp(S_ij) / sum_j exp(S_ij)
O = A V
```

对每个 Query，权重沿 Key 维和为 1，使模型动态聚合不同 token 的 Value。

### 2. 输出层的作用

对词表 logits：

```text
p_i = softmax(z)_i
L = -log p_y
```

实际训练通常直接使用 fused Cross Entropy，不先显式计算完整概率，以提高稳定性和效率。

### 3. 数值溢出/下溢

`exp(z)` 对大数会溢出。使用稳定形式：

```text
softmax(z)_i
= exp(z_i - max(z))
  / sum_j exp(z_j - max(z))
```

Cross Entropy 使用 `logsumexp`。低精度下 reduction、max 和累加常提升到 FP32。

### 4. 饱和与梯度

logit 差异过大时，Softmax 接近 one-hot，非最大项梯度很小。Attention 因此使用：

```text
QK^T / sqrt(d_k)
```

还可通过初始化、Norm、temperature、logit clipping 和合适精度控制尺度。

### 5. Mask 问题

- causal/padding mask 必须在 softmax 前加；
- 被 mask 项应得到接近 0 的概率；
- 全部被 mask 的一行会产生 NaN；
- FP16 中 `-inf`/极小值使用要符合 kernel 语义；
- packed sequence 必须隔离不同样本。

### 6. 长序列和稀疏性

标准 Softmax Attention 需要 `N x N` 分数矩阵，时间和显存是 `O(N^2)`。这主要是全连接 Attention 的问题，不是 `exp` 单算子本身。

解决路线：

- FlashAttention：分块计算精确 softmax，减少 HBM IO；
- Sliding Window/Block Sparse；
- Top-k/Sparse Attention；
- Linear Attention，用核函数改写；
- 混合注意力和状态空间模型；
- GQA/MLA 主要降低 KV Cache，不直接消除 Prefill 的全部二次计算。

### 7. Softmax 过度扩散或集中

长序列可能出现：

- 权重过分平坦，关键信息被稀释；
- attention sink；
- 模型只盯少数 token；
- 长度外推时 logit 分布变化。

可通过位置编码、局部/全局混合、temperature、检索、上下文压缩和专门长文本训练改善。Sparsemax/Entmax 可产生稀疏概率，但生态和 kernel 不如 Softmax 普遍。

### 8. 跨境支付

订单号、金额和错误码不能指望 Attention 自动准确定位。应通过结构化工具、字段检索和引用校验提供事实；模型 Attention 只负责组织上下文。

**面试总结**：Softmax 的优势是可微归一化竞争；稳定实现靠减最大值和 logsumexp，长上下文效率靠 Flash/Sparse/Linear 等系统与架构方案。
