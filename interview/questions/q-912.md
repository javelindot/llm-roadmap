# 了解并介绍推荐系统中除重排之外的其他模型原理

## 答案

推荐系统通常是“候选生成→粗排→精排→重排”，此外还有用户/商品表示和多任务目标。可按链路介绍。

### 1. 召回

- 热门、规则、ItemCF/UserCF；
- 矩阵分解；
- 双塔/DSSM；
- 多兴趣 MIND/ComiRec；
- 图模型 PinSage/LightGCN；
- Semantic ID/生成式召回。

目标是在低延迟下从百万候选缩到数百/数千。

### 2. 粗排

比召回使用更多特征，但比精排轻：

- LR/GBDT、小 MLP；
- 轻量双塔或浅层 DeepFM；
- distillation 模型模仿精排；
- 多阶段 cascade，根据请求动态预算。

### 3. 精排

- Wide&Deep、DeepFM、DCN；
- DIN/DIEN/SIM 序列兴趣；
- MMoE/PLE 多任务；
- ESMM/AITM 转化建模；
- Transformer/HSTU 长序列。

### 4. 最终列表优化

严格说这是重排，但还可从决策角度讲：

- MMR、DPP 多样性；
- 约束优化处理库存、广告、公平和频控；
- Bandit/RL 平衡探索与长期收益。

### 5. 生成式与 LLM

LLM 可做内容标签、query 理解、冷启动、解释和少量候选重排；TIGER/OneRec 等进一步生成 item/列表。高 QPS 主链仍需评估延迟与可控性。

### 面试总结

除重排外，推荐系统还包括召回、粗排、精排、表示学习、多任务和探索。面试应讲每层候选规模、目标、特征和延迟，而不是只列模型名。
