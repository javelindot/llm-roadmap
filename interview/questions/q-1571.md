# 最近读过哪些Agent方向的论文？

## 答案

这是阅读真实性题。建议选择自己确实精读过的 2 至 3 篇，分别讲问题、方法、实验和局限。下面是截至 2026 年 6 月可选的阅读组合，不代表个人已经读过。

### 1. Agent Lightning

**问题**

现有 Agent 往往与具体编排框架耦合，复杂轨迹的奖励难以分配到关键步骤。

**方法**

- 将 Agent 执行与 RL 训练解耦；
- 把任意 Agent 运行建模为包含状态、动作和奖励的 MDP；
- 使用 LightningRL 做分层 credit assignment；
- 通过统一数据接口支持多种 Agent 框架和任务。

**实验关注**

论文在 text-to-SQL、RAG、数学工具等任务上比较训练前后效果，并讨论几乎不改 Agent 代码的训练方式。阅读时应检查 baseline、训练预算、轨迹采集和 reward 定义。

**局限**

奖励和状态抽象仍由任务设计；高风险真实环境不能直接在线探索。

### 2. Agentic Context Engineering

**问题**

Agent 长期运行时，固定 Prompt 或简单摘要难以持续积累有效经验，也容易上下文膨胀。

**方法**

- 把上下文视为持续演化的 playbook；
- 通过生成、反思和整理从轨迹中提取策略；
- 增量更新而非每次重写全部记忆；
- 让跨任务经验复用。

**实验关注**

看其在 Agent benchmark 上相对静态 context、总结和记忆基线的提升，以及 token 成本、跨任务泛化和消融。

**局限**

错误经验可能被写入并放大；playbook 需要来源、版本、置信度和删除机制。

### 3. τ²-bench

**问题**

真实客服/业务 Agent 不只是调用工具，还要与用户协作获取信息；用户也可能不完整、反复或改变目标。

**方法**

- 同时模拟用户与 Agent 对共享环境的操作；
- 新增电信领域的双边控制任务与组合式任务生成；
- 用环境状态和规则验证任务完成；
- 评估 Agent 是否能引导用户配合并正确执行。

**实验关注**

比较不同模型/Agent 在多轮协作、工具和策略约束下的成功率，并分析错误来自用户沟通、策略还是工具执行。

**局限**

模拟用户与真实人类仍有差距；领域规则和任务分布会影响结论。

### 4. 可补充的经典脉络

- ReAct：reasoning 与 acting 交替；
- Reflexion：用语言反馈改进后续轨迹；
- Voyager：开放世界技能库与自动课程；
- SWE-agent：面向代码仓库环境的 Agent-computer interface；
- WebArena/AgentBench/GAIA：通用交互与工具评测。

经典论文用于说明演进，近期论文用于说明当前问题：训练、上下文演化、协作评测和真实环境。

### 5. 面试表达

> 我会重点讲一篇训练、一篇记忆、一篇评测论文。每篇准备一张架构图、一个关键消融和一个我不认同或尚未解决的地方，而不是只复述摘要。

只保留真实阅读内容。

### 6. 跨境支付启发

Agent 训练可使用脱敏历史回放，记忆必须可审计，评测要模拟用户补充信息与工具状态；真实资金系统不能作为自由探索环境。

### 7. 面试总结

近期 Agent 研究重点正从“会调用工具”转向“如何训练、如何积累经验、如何在多轮真实环境中可靠评测”。回答时方法与实验设计比论文数量重要。
