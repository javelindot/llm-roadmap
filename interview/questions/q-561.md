# ControlNet是如何控制生成图像的？零卷积的作用？

## 答案

ControlNet 是给预训练扩散模型增加空间条件控制的结构，例如边缘、深度、人体姿态、分割图和草图。

### 1. 核心结构

ControlNet 保留原扩散模型主干参数，并复制一份部分 Encoder Block 作为可训练控制分支：

```text
条件图
→ 条件编码
→ 可训练控制分支
→ 控制特征注入冻结的扩散模型
```

主干保持冻结，可以保留原模型的生成能力；控制分支学习如何把空间条件作用到去噪过程。

### 2. 零卷积的作用

“零卷积”通常指权重和偏置初始化为 0 的 `1×1` 卷积。

训练开始时：

```text
ZeroConv(control_feature) = 0
```

因此控制分支初始不会扰动预训练模型，整个系统行为与原模型近似一致。随着训练进行，零卷积逐渐学习非零映射，把控制信号稳定地注入主干。

### 3. 为什么这样设计

- 避免随机初始化控制分支一开始破坏预训练能力。
- 允许从“无控制”平滑过渡到“有控制”。
- 冻结大模型主干，降低训练数据和算力需求。
- 不同条件可以训练独立 ControlNet，复用同一个基础模型。

### 4. 训练目标

仍然使用扩散模型的噪声预测或等价目标，只是模型额外接收条件图。训练数据需要文本、目标图像和控制条件三者对应。

### 5. 局限

- 控制图质量会直接影响生成结果。
- 控制过强可能限制多样性，过弱则结构不稳定。
- 多 ControlNet 叠加会增加显存与推理时间。
- 不同基础模型或领域之间不一定直接迁移。

**面试总结**：ControlNet 用冻结主干保留生成能力，用可训练副本学习条件控制，用零卷积保证训练从“不扰动原模型”开始。
