# 如何使用 transformers 加载 BERT 模型？

## 答案

**一句话**：用 `AutoModel` + `AutoTokenizer` 两行就能加载 BERT——选对**任务类（AutoModelForSequenceClassification 等）、设备（cuda）、精度（fp16/bf16）** 三个参数是工程关键。

### 1. 最简单加载
```python
from transformers import AutoTokenizer, AutoModel
tokenizer = AutoTokenizer.from_pretrained("bert-base-chinese")
model = AutoModel.from_pretrained("bert-base-chinese")
```

### 2. 按任务选择模型类
| 任务 | 模型类 |
|------|--------|
| 通用特征 | `AutoModel` |
| 分类 | `AutoModelForSequenceClassification` |
| Token 标注（NER） | `AutoModelForTokenClassification` |
| QA | `AutoModelForQuestionAnswering` |
| 掩码预测 | `AutoModelForMaskedLM` |

例：
```python
from transformers import AutoModelForSequenceClassification
model = AutoModelForSequenceClassification.from_pretrained(
    "bert-base-chinese", num_labels=3
)
```

### 3. 推理流程
```python
import torch
inputs = tokenizer("跨境支付如何申请退款", return_tensors="pt")
with torch.no_grad():
    outputs = model(**inputs)
# outputs.last_hidden_state: (1, seq_len, hidden_dim)
# outputs.pooler_output: (1, hidden_dim) - CLS pool
```

### 4. 工程化（设备 + 精度）
```python
model = AutoModelForSequenceClassification.from_pretrained(
    "bert-base-chinese",
    num_labels=3,
    torch_dtype=torch.bfloat16,  # 或 float16
    device_map="cuda",  # 多卡用 "auto"
)
model.eval()
```

### 5. Batch 推理
```python
texts = ["问题1", "问题2", "问题3"]
inputs = tokenizer(
    texts, return_tensors="pt", padding=True, truncation=True, max_length=512
).to("cuda")
with torch.no_grad():
    outputs = model(**inputs)
```

### 6. 微调（Trainer 一行）
```python
from transformers import Trainer, TrainingArguments
args = TrainingArguments(
    output_dir="./out", num_train_epochs=3,
    per_device_train_batch_size=16, learning_rate=2e-5,
    bf16=True, gradient_checkpointing=True,
)
trainer = Trainer(model=model, args=args, train_dataset=ds, tokenizer=tokenizer)
trainer.train()
```

### 7. 国内访问技巧
```python
import os
os.environ["HF_ENDPOINT"] = "https://hf-mirror.com"
# 或用 ModelScope
from modelscope import snapshot_download
path = snapshot_download("damo/nlp_bert_backbone_base_zh")
model = AutoModel.from_pretrained(path)
```

### 8. 常见坑
- BERT 默认 max_length=512，长文本要截断或滑窗
- `tokenizer.pad_token` 有时缺失（GPT 系列用 EOS 当 pad）
- `device_map="auto"` 在单卡环境可能产生问题
- BF16 在 V100 不可用，要 FP16 + GradScaler

### 9. 一句话总结
**`AutoTokenizer` + `AutoModelForXxx` 两行加载，配 `torch_dtype` + `device_map` 上 GPU**——HuggingFace transformers 是工业 LLM 的事实标准接口。

## 结合项目谈（口语版 · 约 20 秒）

> 这道题就是 SOP 题，我们跨境支付**90% 模型都通过 HF transformers 加载**。分享几个实战技巧。
>
> **技巧一：BF16 默认开**。H100 / A100 默认 `torch_dtype=torch.bfloat16`，免 GradScaler 麻烦、训推稳定。早期我们用 FP16 偶尔 NaN，切 BF16 后基本无忧。
>
> **技巧二：客服分类 BERT 流水线**。我们意图分类模型用 `bert-base-chinese` + LoRA 微调，加载代码不到 10 行，配 Trainer 一晚上能训出来。生产推理用 `vLLM` 或 ONNX 加速。
>
> **技巧三：内部模型仓库**。生产链路不让访问 HuggingFace，我们把常用模型 mirror 到内部 MinIO，`from_pretrained` 直接走内网路径，**几乎瞬间加载**。
>
> **技巧四：embedding 模型加载**。bge-large 当 embedding model 时，用 `AutoModel` 加载（不是 `ForSequenceClassification`），手动做 mean pooling，性能比 `SentenceTransformers` 略好但更可控。
>
> **踩过的坑**：
> - 一开始忘记 `model.eval()`，推理用了 dropout，输出每次不一样；
> - 长文本忘记 truncation，整个 batch OOM；
> - `padding=True` 没指定 max_length，padding 到最长样本，浪费算力。
>
> 一句话：**两行加载是入门，工程化关键在精度 / 设备 / 内部仓库 / batch 优化四件事**。
