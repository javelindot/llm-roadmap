# 讲一下你对BERT模型的理解？它相较于传统深度学习模型的它的突出点是什么？

## 答案

BERT 是 Encoder-only Transformer 预训练模型，通过双向 Self-Attention 为每个 token 生成同时融合左右上下文的表示，再通过少量任务数据微调到分类、抽取和匹配任务。

### 1. 结构

输入表示由三部分相加：

```text
Token Embedding
+ Position Embedding
+ Segment Embedding
```

经过多层 Transformer Encoder：

```text
Self-Attention -> Add & Norm -> FFN -> Add & Norm
```

注意力没有 causal mask，因此每个位置可同时关注左右上下文。

### 2. 预训练任务

**Masked Language Modeling**

随机遮盖部分 token，让模型利用双向上下文恢复原词。

**Next Sentence Prediction**

原始 BERT 用 NSP 学句间关系，后续 RoBERTa 等研究表明 NSP 不是必需，训练数据、动态 mask 和训练配方同样重要。

### 3. 相比传统模型的突出点

**上下文化表示**

Word2Vec 给一个词固定向量；BERT 会根据语境区分“苹果公司”和“苹果水果”。

**真正的双向建模**

相比单向语言模型或简单拼接双向 RNN，Self-Attention 在每层都融合两侧上下文。

**并行训练**

RNN 必须按时间步递推，Transformer 可并行处理整段序列，训练效率更高。

**长距离依赖**

任意两个 token 可通过一次注意力交互，不需要跨很多 RNN 步传播。

**预训练迁移**

同一个基座只需替换任务头或少量微调，就能覆盖分类、NER、阅读理解和匹配。

### 4. 局限

- 原始 Self-Attention 对序列长度是 `O(n^2)`；
- 不擅长开放式长文本生成；
- MLM 预训练与下游输入存在 `[MASK]` 差异；
- 小模型虽便宜，但复杂零样本推理和工具调用弱于现代生成式大模型。

### 5. 跨境支付

BERT 类模型适合高吞吐的工单意图、拒付原因分类、商户行业识别、实体抽取和第一阶段召回。对于规则解释、长文档问答和 Agent，则可与生成式模型组合。
