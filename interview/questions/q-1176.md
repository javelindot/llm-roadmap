# 列表和元组的区别

## 答案

Python 中 List 和 Tuple 都是有序序列，支持索引、切片、迭代和保存不同类型的元素，核心区别是 List 可变，Tuple 不可变。

| 维度 | List | Tuple |
|---|---|---|
| 语法 | `[1, 2, 3]` | `(1, 2, 3)` |
| 可变性 | 可增删改 | 创建后不能替换、增删元素 |
| 常用方法 | `append、extend、insert、pop、sort` | `count、index` |
| Hash | 不可 hash | 所有元素均可 hash 时才可 hash |
| 适用场景 | 动态集合 | 固定结构、不可变记录、字典键 |
| 内存与构造 | 通常略大 | 通常略小、构造略快 |

### 1. 可变性

```python
items = [1, 2]
items.append(3)
items[0] = 10

point = (1, 2)
point[0] = 10  # 执行时抛出 TypeError
```

Tuple 的不可变是“不能替换它保存的元素引用”。如果元素本身是可变对象，该对象仍可被修改：

```python
t = ([1, 2], "USD")
t[0].append(3)       # 合法
t[0] = [4, 5]        # 执行时抛出 TypeError
```

所以 Tuple 不等于深度不可变。

### 2. Hashability

List 永远不能作为字典键或 set 元素。Tuple 只有在内部所有元素都可 hash 时才可以：

```python
rate_key = ("US", "USD", "payout")
cache = {rate_key: 1.0}

bad_key = ("US", ["USD"])
hash(bad_key)  # 执行时抛出 TypeError
```

如果需要稳定、语义明确的记录结构，`dataclass(frozen=True)`、`NamedTuple` 往往比依赖位置的普通 Tuple 更易维护。

### 3. 复制与参数传递

List 和 Tuple 中保存的都是对象引用，赋值不会自动深拷贝：

```python
a = [1, 2]
b = a
b.append(3)
assert a == [1, 2, 3]
```

需要独立的浅拷贝可使用 `a.copy()` 或 `list(a)`；存在嵌套可变对象时才考虑 `copy.deepcopy`。

### 4. 选择原则

- 元素数量或内容需要变化：使用 List；
- 返回固定数量的多个值、坐标、不可变复合键：使用 Tuple；
- 需要字段名和类型约束：使用 `NamedTuple` 或 `dataclass`；
- 不应仅为了微小的性能差异选择 Tuple，可读性和数据语义更重要。

跨境支付代码中，动态收款通道列表适合 List；`(country, currency, product)` 这类缓存键适合 Tuple，但金额、汇率和业务状态仍应使用含字段名与校验的领域对象，避免位置传错。
