# 学习率退火（Annealing）对模型收敛和泛化的作用是什么？

## 答案

**一句话**：学习率退火 = **训练后期把 lr 慢慢调小**，让模型从"大步探索"切到"小步精细"，**既加速收敛、又改善泛化**——是 LLM 训练必备的调度策略。

### 1. 为什么需要退火
- 训练前期：参数远离最优，需要大 lr 快速接近
- 训练后期：参数已经接近最优，大 lr 会震荡
- 退火让 lr 跟随训练进度，**从粗调到精调**

### 2. 主流退火曲线
| 策略 | 形状 | 用例 |
|------|------|------|
| **Linear Decay** | 线性下降到 0 | 简单任务 |
| **Cosine Decay** ⭐ | 余弦平滑下降 | LLM 训练首选 |
| **Cosine with Warmup** | 先升后降 | 标准 LLM 配方 |
| **Polynomial Decay** | 多项式衰减 | 灵活控制 |
| **Step Decay** | 阶梯式下降 | CV 老套路 |
| **Inverse Sqrt** | 1/√t | 原始 Transformer |

### 3. Warmup + Cosine 是 LLM 标配
```
lr(t) = 
  lr_max × t/warmup_steps,           if t < warmup_steps
  lr_min + 0.5×(lr_max - lr_min) × (1 + cos(π × (t-warmup)/(T-warmup))),  else
```
- 前 ~3% 步数 warmup（线性从 0 升到 lr_max）
- 之后余弦衰减到 lr_min（通常 lr_max 的 10%）

### 4. 退火的两个作用
- **收敛**：后期小 lr 让 loss 稳定下降不震荡
- **泛化**：找到 **flat minima**（平坦极小），泛化更好（参考 SWA、Stochastic Weight Averaging）

### 5. 常见经验值（LLM 训练）
- lr_max：1e-5 ~ 5e-5（全参 SFT）/ 1e-4 ~ 3e-4（LoRA）
- lr_min：lr_max × 0.1
- warmup：总步数的 1-5%
- 退火曲线：cosine

### 6. WSD（Warmup-Stable-Decay）新趋势
- 大部分时间保持 lr 不变（stable phase）
- 最后 10% 才快速 decay
- MiniCPM / Qwen 用过
- 优点：可以"训得到一半丢退火继续训"

### 7. 一句话总结
**Warmup + Cosine 是 LLM 训练的退火黄金组合**——能加速收敛、改善泛化，工程上不调几乎不会出大错。

## 结合项目谈（口语版 · 约 30 秒）

> 我们跨境支付训练里学习率调度**默认 Warmup + Cosine**，HuggingFace Trainer 调好就跑，从没出过事。
>
> 几个具体配置：**全参 SFT** 用 lr_max=2e-5，warmup_steps=总步数×3%，min_lr=2e-6（10% lr_max）；**LoRA** lr_max=1e-4，warmup 同上。
>
> 踩过的坑有两个：第一是**没退火直接 constant lr** 训客服模型，后期 loss 上下震荡，最后 ckpt 比中间 ckpt 还差，证明退火的"精调"作用是真实的；第二是 **warmup 太短**（一开始只设了 100 步），梯度爆过几次，调到 3% 总步数就稳了。
>
> 我们最近也在试 **WSD 调度**——MiniCPM 用过这套，前 90% 训练保持 lr 不变，最后 10% 退火。好处是**前 90% 的 checkpoint 可以无损接着训**，不像 cosine 一开始就在退火没法接续。我们做风控模型迭代时这点很有价值——业务规则变了，从上一版 checkpoint 接着训 10% 数据就能上线。
>
> 我的总结：**退火不是为了好看的 loss 曲线，是为了泛化和工程灵活性**——选对调度比选对优化器还重要。
