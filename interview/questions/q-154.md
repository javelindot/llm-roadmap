# 预训练模型的"预训练 + 微调"范式是什么？

## 答案

**一句话**："预训练 + 微调"是先在大规模通用数据上学基础能力，再用少量任务数据适配具体场景。

### 核心回答
流程：
1. **预训练**：用海量无标注或自监督数据学习通用表示。
2. **微调**：用标注数据或指令数据适配具体任务。
3. **评估和部署**：在目标业务场景验证效果。

优点：
- 降低下游任务数据需求。
- 迁移通用语言和知识能力。
- 比从零训练成本低很多。

大模型时代常见变体：
- SFT。
- LoRA/QLoRA。
- DPO/RLHF。
- RAG 不改参数，靠外部知识增强。

### 1. 为什么这个范式有效

预训练模型已经从大规模数据中学到通用语言、知识和表示能力。下游任务通常数据量小，如果从零训练成本高且容易过拟合。微调是在通用能力基础上做任务适配。

### 2. 大模型时代的扩展

现在不只有传统 fine-tuning，还包括：

- **SFT**：教模型按指令和格式回答。
- **LoRA/QLoRA**：低成本参数高效微调。
- **DPO/RLHF**：让模型更符合人类或业务偏好。
- **RAG**：不改参数，接入外部知识。
- **Prompt/Agent**：通过上下文和工具调用适配任务。

### 3. 面试总结

预训练提供通用能力，微调负责业务适配，RAG 负责知识更新，Agent 负责流程执行。

### 4. 结合跨境支付项目怎么说（面试口径）

> 跨境支付里最现实的路线就是拿通用基座模型，再做领域微调和 RAG。预训练让模型会中文、英文、代码和通用推理；微调让它懂支付术语、风控流程、客服话术。
>
> 但支付规则更新很快，所以我不会把所有知识都靠微调记住。RAG 负责接最新通道规则、合规文档、拒付 SOP；微调更多负责表达风格、流程规范和业务偏好。
>
> 这样比从零训练金融模型更可控，也更容易随业务规则变化更新。

**钩子**：业务落地通常是基座模型 + LoRA/SFT + RAG，而不是从零训练。
