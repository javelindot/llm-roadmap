# 在RAG实践中选择BGE作为嵌入模型的原因及FAISS索引的构建方法

## 答案

选择 BGE 通常是因为其中文/多语言检索表现、开源可私有化、模型尺寸选择、Embedding/Reranker 生态和较成熟的工程支持。但不能只因为公开榜单高就选，最终应在业务语料上与其他模型做 Recall、延迟和成本对比。

### 1. 为什么选择 BGE

- 中文和跨语言能力较强；
- 有不同尺寸，便于效果/延迟权衡；
- 支持查询指令或领域提示；
- BGE-M3 可提供 dense、sparse 和 multi-vector 能力；
- 可本地部署，便于数据隔离；
- 有配套 Reranker；
- Hugging Face/Sentence Transformers 生态成熟。

不同 BGE 版本的 pooling、query instruction、最大长度和归一化方式不同，必须按对应 model card 使用。

### 2. 文档处理

```text
解析
-> 语义切分
-> 标题注入
-> metadata
-> embedding
-> FAISS
```

metadata 至少保存：

```text
doc_id, chunk_id, title, source,
version, country, product, permission,
updated_at, raw_text
```

FAISS 只存向量和数值 id，正文与权限通常保存在外部文档库/数据库，通过 id 映射。

### 3. 向量预处理

若使用 cosine similarity：

```python
vectors = model.encode(
    texts,
    normalize_embeddings=True
).astype("float32")
```

归一化后 inner product 等价于 cosine：

```python
index = faiss.IndexFlatIP(dim)
index.add(vectors)
```

查询也必须使用相同模板和归一化。

### 4. FAISS 索引选择

**IndexFlatIP/L2**

- 精确检索；
- 无训练；
- 内存和查询随数据量增长；
- 适合中小数据或基准。

**HNSW**

- 高 Recall、低延迟；
- 内存较高；
- 通过 `M`、`efConstruction`、`efSearch` 调节。

**IVF**

- 先聚类再搜索部分倒排桶；
- 需训练；
- `nlist` 和 `nprobe` 权衡 Recall/延迟。

**IVF-PQ**

- 对向量压缩；
- 适合超大数据；
- 节省内存但损失精度。

### 5. 示例

```python
import faiss
import numpy as np

xb = np.asarray(doc_vectors, dtype="float32")
xq = np.asarray(query_vectors, dtype="float32")

faiss.normalize_L2(xb)
faiss.normalize_L2(xq)

dim = xb.shape[1]
index = faiss.IndexHNSWFlat(dim, 32)
index.hnsw.efConstruction = 200
index.hnsw.efSearch = 64
index.add(xb)

scores, ids = index.search(xq, k=50)
```

召回后按 id 做权限/版本校验并 Rerank。更稳妥的是能前置的权限和结构化过滤尽量前置，避免越权候选进入模型。

### 6. 构建与更新

- 训练 IVF/PQ 时使用代表性样本；
- 保存 index 和 id mapping；
- 记录 embedding 模型/模板版本；
- 文档更新做增量 add/delete 或构建新索引后原子切换；
- 做副本、校验和和回滚；
- 防止新旧 embedding 混在同一索引。

### 7. 评测

- Recall@K；
- MRR/NDCG；
- ANN Recall 对比 Flat；
- 不同语言和长尾；
- P95/QPS；
- 内存和构建时间；
- 索引新鲜度；
- 最终回答和引用正确率。

### 8. 跨境支付场景

BGE+FAISS 可检索政策、错误码和操作文档，但要按国家、币种、产品、规则版本和商户权限过滤。订单和账务状态应查实时 API，不进入静态向量索引。

### 9. 面试总结

BGE 负责语义表示，FAISS 负责高效 ANN。选型流程是先用 Flat 建质量基线，再按数据规模选 HNSW/IVF/PQ，并联合优化 Recall、延迟、内存、权限和版本更新。
