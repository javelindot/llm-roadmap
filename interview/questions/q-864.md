# 八股:DeepSeek R1的创新点和优化点?

## 答案

DeepSeek-R1 的核心创新在后训练推理能力，而基础架构继承 DeepSeek-V3。MLA、DeepSeekMoE、FP8 训练和 DualPipe 属于 V3 基座贡献，不应全部算作 R1 新发明。

### 1. R1-Zero：纯 RL 激励推理

R1-Zero 不先做 SFT，直接在基座上使用大规模 RL。论文报告模型自发出现更长思考、自我验证、反思和策略调整等行为，说明可验证奖励能激励推理。

它也暴露了可读性差、语言混杂等问题，说明“会推理”不等于“适合用户使用”。

### 2. GRPO

对同一问题采样多条回答，用组内相对奖励形成 advantage，省去 PPO 的独立 critic，降低训练显存。奖励主要利用数学/代码等可验证正确性与格式信号，减少依赖主观 Reward Model。

### 3. R1 的多阶段训练

公开流程可概括为：

1. 少量高质量 cold-start 推理数据 SFT；
2. reasoning-oriented RL；
3. rejection sampling 生成高质量推理数据，并加入写作、事实问答等非推理数据做 SFT；
4. 第二阶段 RL 同时改善帮助性、安全和推理能力。

它用 cold start 修复 R1-Zero 的表达问题，并通过多阶段数据覆盖通用场景。

### 4. 蒸馏

将 R1 产生的推理数据蒸馏到 Qwen/Llama 的 1.5B～70B Dense 模型，表明大模型推理轨迹可显著提升小模型。但蒸馏不是简单复制长 CoT，需要验证最终答案、过滤错误和防止风格过长。

### 5. 边界

- 可验证数学奖励不直接迁移到开放业务；
- 长输出增加推理成本；
- reward hacking、语言混合和过度思考仍需控制；
- 论文并未证明纯 RL 是所有模型和任务的最佳路线。

依据可参考 [DeepSeek-R1 论文](https://arxiv.org/abs/2501.12948)与[官方仓库](https://github.com/deepseek-ai/DeepSeek-R1)。

### 面试总结

R1 的关键是 R1-Zero 展示纯 RL 涌现推理、GRPO 省去 critic、R1 用 cold start 与多阶段 RL/SFT 改善可读性和通用性，并通过蒸馏把推理能力迁移到小模型。
