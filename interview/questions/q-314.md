# 上机操作，2道：创建删除文件夹；显卡查看Qwen模型进程并用Docker关闭。

## 答案

答：nvidia-smi查看Qwen3进程，docker stop 容器ID。

### 补充说明

上机题要回答得具体一点：

```bash
mkdir test_dir
rmdir test_dir
rm -rf test_dir
nvidia-smi
docker ps
docker stop <container_id>
```

如果是查 Qwen 模型进程，可以先 `nvidia-smi` 看占用 GPU 的 PID，再用 `ps -fp <pid>` 看进程详情。如果模型在 Docker 里，还要 `docker ps` 找容器，再 `docker stop` 停掉容器。不要直接乱 kill 进程，生产环境要先确认是不是线上服务。

### 结合跨境支付项目怎么说（面试口径）

> 在项目现场部署大模型时，这类命令很常用。比如客服模型响应变慢，我会先用 `nvidia-smi` 看 GPU 显存和进程占用，再查是哪一个 Qwen 或 vLLM 容器。如果确认是测试容器异常占满资源，再通过 `docker stop` 停掉。
>
> 但在支付公司生产环境，我不会随便停服务。会先确认服务归属、是否有流量、是否有备份实例，再按发布或运维流程操作。因为模型服务虽然是旁路，也可能影响客服和风控解释。
