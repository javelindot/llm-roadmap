# 权重初始化的重要性是什么？为什么不能将权重初始化为 0？

## 答案

**一句话**：权重初始化决定**梯度能否正常传播 + 神经元能否打破对称**——初始化为 0 会导致**对称性问题**（同层神经元学到一样的东西），等价于"一层只有 1 个神经元"。

### 1. 为什么初始化重要
- 决定前向激活的尺度（防爆/防消失）
- 决定反向梯度的尺度（防梯度爆/消失）
- 决定 loss 起点和优化路径
- 错误初始化能让模型**完全训不动**

### 2. 为什么不能全 0
- 所有神经元的输入 z = 0
- 经过激活后输出全 0（或全相同常数）
- 反传时所有权重梯度相同
- 每步更新后所有神经元仍然相同
- → **永远对称**，等价于一个神经元

### 3. 为什么不能全相同非零常数
- 同样存在对称性问题
- 所有神经元学到相同特征
- 无法表达多样化模式

### 4. 必须用随机初始化
- 打破对称性
- 每个神经元起点不同
- 训练后学到不同特征

### 5. 但随机也要有方差控制
- 太大 → 激活/梯度爆炸（NaN）
- 太小 → 激活/梯度消失（不学）
- 需要根据网络深度 + 激活函数选合适方差

### 6. 经典方案
| 方法 | 方差 | 适用 |
|------|------|------|
| **Xavier/Glorot** | Var = 2/(n_in + n_out) | tanh / sigmoid |
| **He/Kaiming** | Var = 2/n_in | ReLU 系 |
| **截断正态 N(0, σ)** | σ 由经验定 | Transformer 常用 |

### 7. LLM 的初始化经验
- Embedding：N(0, 0.02)
- Linear / FFN：N(0, 0.02) 或 He
- LayerNorm γ=1, β=0
- 输出层有时缩小 1/√(2L) 防深层爆

### 8. 一句话总结
**全 0 初始化 = 模型只有一个神经元**——必须随机 + 方差控制，Xavier/He 是经典方案，LLM 常用截断正态。

## 结合项目谈（口语版 · 约 30 秒）

> 这道题在工业里基本不会自己撞——预训练 base 都帮你初始化好了，微调时也是冻 base 训 LoRA 或者 SFT，没人手动初始化。
>
> 但我们跨境支付**给客服模型加辅助头**时实打实碰到过这个坑。
>
> 场景：我们想在 Qwen-7B 顶上接一个**满意度回归头**和**意图分类头**，做多任务。第一版我用了 `nn.Linear` 默认初始化（Kaiming uniform），训了一天 loss 不动；后来发现是 `nn.Linear` 默认对 bias 也做 uniform，对大 hidden dim 不友好。改成 **truncated normal(0, 0.02)** 后立刻收敛。
>
> 还有一个相关的坑：我们试过给 LoRA 层手动初始化，**B 矩阵全 0、A 矩阵 Kaiming uniform**——这是 LoRA 论文的标准做法，目的是让初始 ΔW = A·B = 0，**模型起点等于 base，不破坏原能力**。如果两个矩阵都随机初始化，base 立刻被打乱，训练前几步会崩。
>
> 我对这道题的总结：**LLM 时代权重初始化看起来不重要，但加新模块时还是会撞到**——LoRA、辅助头、Adapter 都涉及，初始化错了能让训练完全失败。
