# 当相关文本库里有和用户查询相关的内容片段，如果提高检索召回率。

## 答案

如果知识库中确实有相关片段但没有召回，说明问题出在索引、切分、query、检索模型、过滤或候选预算。应先用标注 query-document 对定位漏召回发生在哪一层，再针对优化。

### 1. 检查文档是否真正进入索引

- 解析是否失败；
- OCR 是否正确；
- chunk 是否为空或被过滤；
- embedding 是否生成；
- 索引版本是否最新；
- 权限和 metadata 是否误过滤；
- 文档是否被错误去重；
- 当前查询是否命中旧索引。

### 2. 优化 Chunk

- 按标题和语义切分；
- 保留标题、章节和父文档上下文；
- 调整 chunk size/overlap；
- 表格、FAQ、错误码使用专用解析；
- 使用 parent-child retrieval；
- 将国家、币种、产品和版本写入 metadata 或可检索文本；
- 删除重复和过期片段。

### 3. 优化 Query

- query rewrite；
- multi-query；
- 同义词和缩写扩展；
- 纠错；
- HyDE；
- 意图路由；
- 将对话中的指代补全；
- 提取错误码、国家、币种和产品字段。

例如“钱没到”可扩展为“payout pending、出款未到账、beneficiary bank processing”。

### 4. 混合召回

结合：

- BM25：错误码、专有名词和精确字段；
- Dense retrieval：语义匹配；
- 结构化过滤：国家、版本、权限；
- 图谱或 SQL：实体关系；
- RRF 或可学习融合。

纯向量检索容易漏掉短错误码和数字。

### 5. 调整模型和候选

- 使用领域 embedding；
- 用 hard negative 微调；
- 增大 retrieval top-K；
- 调整 ANN `efSearch/nprobe`；
- 降低过严相似度阈值；
- 召回后使用 Reranker，而不是过早过滤；
- 分不同意图设置不同索引和预算。

增大 K 会增加噪声和成本，需要联合评估。

### 6. 评测

- Recall@K；
- HitRate@K；
- 分意图和语言召回；
- 独占召回；
- ANN 近似损失；
- 过滤前后召回；
- 下游回答正确率；
- P95 和成本。

### 7. 跨境支付边界

先按商户权限、国家、币种和规则版本过滤，再做混合召回。不能为了 Recall 取消权限过滤，也不能把过期监管文档和当前文档一起无差别召回。

### 8. 面试总结

库中有知识却召不回时，应沿“入库 -> chunk -> query -> retrieval -> filter -> ANN -> top-K”定位。常见有效组合是语义切分、query rewrite、BM25+dense、结构化过滤和 Rerank。
