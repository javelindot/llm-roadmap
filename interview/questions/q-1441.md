# 当文本库中不存在相关内容片段，如何提升检索召回率

## 答案

如果文本库中不存在相关内容，就不存在可被召回的正确片段，单纯调 embedding、top-K 或相似度阈值无法提升真实 Recall。此时问题是知识覆盖不足，而不是检索算法不足。

### 1. 先识别“无答案”

- top-K 分数整体低；
- BM25 和 dense 都无稳定命中；
- Reranker 判定不相关；
- 多次 query rewrite 仍无证据；
- 文档版本或权限范围内不存在答案；
- 模型无法给出可支持引用。

应输出：

```text
当前知识库没有足够证据
```

而不是从相似但无关的片段拼答案。

### 2. 扩展知识源

- 接入新的内部文档；
- 同步产品、规则和公告；
- 查询结构化数据库；
- 调用实时业务 API；
- 在允许时检索可信外部网页；
- 请求用户上传材料；
- 转领域专家或人工客服。

每种来源都要有权限、版本、可信度和更新时间。

### 3. 建立知识缺口闭环

记录：

```text
query
用户/场景
当前检索结果
缺失主题
最终人工答案
证据来源
```

聚类高频无答案 query，确定是否需要新增文档、FAQ、工具或产品能力。知识审核通过后再入库和重建索引。

### 4. 不要用低阈值伪造召回

无限降低阈值或增大 top-K 会召回更多无关文档，表面有结果但实际 Precision 和回答正确率下降。这属于“有返回”而不是“召回正确知识”。

### 5. 跨境支付场景

如果知识库没有某国家最新监管规定，应查询监管源或转合规团队，不能用其他国家的相似规则回答。订单状态本来也不应在文本库中，应该调用交易和账务系统。

### 6. 面试总结

库里没有知识时，正确策略是无答案检测、扩展权威知识源、调用工具或转人工，并将缺口回流知识建设。检索系统不能召回不存在的内容，诚实拒答比制造“高召回”更重要。
