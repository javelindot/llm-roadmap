# Transformer 的编码器和解码器结构分别是什么？

## 答案

**一句话**：Transformer Encoder 负责编码输入表示，Decoder 负责在自回归生成时结合已生成内容和 Encoder 输出生成目标序列。

### 核心回答
Encoder 层通常包括：
- Multi-Head Self-Attention。
- Feed Forward Network。
- Add & Norm。

Decoder 层通常包括：
- Masked Multi-Head Self-Attention。
- Cross-Attention，关注 Encoder 输出。
- Feed Forward Network。
- Add & Norm。

结构区别：
- Encoder 可以双向看完整输入。
- Decoder 的 self-attention 用 causal mask，只能看已生成 token。
- Encoder-Decoder 适合翻译、摘要等输入输出都明确的任务。

GPT 类 LLM 多用 Decoder-only；BERT 是 Encoder-only；T5 是 Encoder-Decoder。

### 1. Encoder 和 Decoder 的职责

Encoder 主要做理解：它可以双向看完整输入，所以适合提取句子表示、做分类、匹配、抽取和重排。

Decoder 主要做生成：它用 causal mask 保证只能看已生成内容，适合自回归生成。Encoder-Decoder 里的 Decoder 还会通过 cross-attention 关注 Encoder 输出。

### 2. 三类结构怎么选

- Encoder-only：理解类任务，成本低、速度快。
- Decoder-only：通用生成、对话、Agent、工具调用。
- Encoder-Decoder：输入输出结构清晰的生成任务，如翻译、摘要。

### 3. 面试总结

不同 Transformer 结构不是谁更高级，而是任务目标不同。理解、生成、输入输出转换各有合适架构。

### 4. 结合跨境支付项目怎么说（面试口径）

> 如果做支付知识理解，比如分类一条工单、判断问题属于收款还是付款、做检索重排，Encoder-only 模型就够，成本低、速度快。
>
> 如果要生成审核意见、客服回复、风控解释，Decoder-only 或 Encoder-Decoder 更合适。现在 LLM Agent 多用 Decoder-only，是因为它能统一成 prompt 到答案，还方便接工具调用。
>
> 所以我不会所有任务都上大模型生成。支付项目里，分类、抽取、重排可以用 Encoder；生成和流程编排再用 LLM，这样成本和稳定性更好。

**钩子**：不同 Transformer 结构适合不同业务任务，不是所有任务都要 LLM 生成。
