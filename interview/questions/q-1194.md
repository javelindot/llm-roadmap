# 偏好模型和对比模型（Pairwise）有什么区别？

## 答案

“偏好模型”和“Pairwise 模型”不是严格互斥的两种模型。偏好模型描述的是业务目标，Pairwise 描述的是训练数据和损失形式；很多偏好模型本身就是用 Pairwise 方法训练的。

### 1. Pairwise 是一种学习范式

Pairwise Ranking 输入同一上下文下的一对候选：

```text
(context, item_a, item_b, label)
```

目标是学习：

```text
score(context, item_a) > score(context, item_b)
```

常见损失包括：

```text
RankNet:
L = -log σ(s_a - s_b)

Margin Ranking:
L = max(0, margin - s_a + s_b)
```

它关注相对顺序，不要求每个候选有精确绝对分数。搜索排序、推荐和 Reward Model 都可使用 Pairwise。

### 2. 偏好模型描述“预测谁更受偏好”

偏好模型通常学习用户或人类在多个候选中的选择。输入可以包含：

- 用户或 prompt；
- chosen/rejected 候选；
- 场景、历史和约束；
- 标注者或人群信息。

输出可以是：

- 单个候选的 utility/reward；
- A 胜过 B 的概率；
- 多候选排序；
- 用户个性化偏好分布。

Bradley-Terry Reward Model 就是典型 Pairwise 偏好模型：

```text
P(a ≻ b) = σ(r(a) - r(b))
```

因此“Reward/Preference Model”和“Pairwise Model”经常是包含关系。

### 3. 主要区别

| 维度 | 偏好模型 | Pairwise 方法 |
|---|---|---|
| 概念 | 业务任务或输出含义 | 训练范式 |
| 目标 | 预测用户/人类偏好 | 学习任意候选相对顺序 |
| 数据 | 点击、评分、chosen/rejected、排序 | 成对样本 |
| 输出 | reward、偏好概率或排序 | 分数差或 pair 概率 |
| 应用 | RLHF、个性化推荐、选择策略 | 搜索、推荐、AUC 排序、偏好学习 |

偏好模型也可以用 Pointwise 评分或 Listwise 排序训练；Pairwise 方法也可以训练与“人类偏好”无关的文档相关性模型。

### 4. Pairwise 的优点和问题

优点：

- 相对比较通常比绝对打分更容易标注；
- 直接优化顺序；
- 不同标注者的打分尺度差异较小；
- 可构造困难候选对。

问题：

- `n` 个候选可能产生 `O(n²)` 对；
- 数据可能不满足传递性；
- 只学相对关系，绝对分数未必校准；
- 曝光、位置和热门偏差会污染选择；
- 长度、风格等捷径可能被当作偏好；
- pair 分布与线上候选分布不一致时泛化差。

### 5. 与 DPO 的关系

Reward Model 先通过 Pairwise 偏好学习得到显式 reward，再由 PPO 等算法优化策略。DPO 不单独训练 Reward Model，而是直接用 `(prompt, chosen, rejected)` 对优化策略与参考策略的相对 log probability。

二者都使用偏好对，但输出不同：

- Pairwise Reward Model 输出可复用的 reward；
- DPO 直接更新生成策略；
- Reward Model 可用于排序、拒绝采样和在线 RL；
- DPO 链路简单，但对离线偏好数据覆盖和偏差敏感。

### 6. 跨境支付场景

可以让业务专家比较两条客服答案，依据事实、证据、合规和可执行性选择更好的一条。若要构建可复用 Judge，训练 Pairwise Reward Model；若目标是直接改善客服生成，可以使用 DPO。

但偏好不能覆盖确定性事实。订单状态、金额、币种、权限和资金动作必须由业务系统校验，不能因为某答案“更受偏好”就认为它事实正确。
