# dpo 出现 chosen 和 rejected的概率都下降是为什么,怎么解决

## 答案

DPO 优化的是 chosen 相对 rejected 的**概率比差距**，不直接要求 chosen 的绝对 log-probability 上升。因此两者都下降，但 rejected 下降得更多，DPO loss 仍可能改善。

目标中的核心 margin 为：

\[
m=
\log\frac{\pi_\theta(y_w|x)}{\pi_{\rm ref}(y_w|x)}
-
\log\frac{\pi_\theta(y_l|x)}{\pi_{\rm ref}(y_l|x)}
\]

只要 \(m\) 增大，目标就变好。概率质量可能被移动到数据中未出现的第三种回答，这称为 likelihood displacement。ICLR 2025 的[研究](https://openreview.net/forum?id=uaMSBJDnRv)表明，chosen/rejected 语义表示过于相似时尤其可能发生，甚至把质量转移到相反含义。

### 先判断是否有害

- 同时记录 chosen/rejected 的 sequence 与 per-token logp；
- 记录 reward margin、相对 reference KL 和生成样本质量；
- 检查长度差异、截断、padding 和 label mask；
- 对 held-out prompt 实际采样，不只看 pairwise accuracy；
- 检查概率转移到了哪些回答，以及安全/拒答率是否下降。

chosen 小幅下降不一定有害；若生成质量、安全或 chosen likelihood 明显恶化，则需处理。

### 修复方向

- 清洗矛盾、低质量和 chosen/rejected 过于相似的偏好对；
- 增加更具语义区分度的负例，并按 prompt 去重；
- 将 DPO 与 chosen 的 NLL/SFT loss 混合，提供绝对正向锚点；
- 降低学习率、训练轮数并根据 KL 调整 \(\beta\)，通过验证集选 checkpoint；
- 使用 label smoothing/conservative DPO、IPO 或专门处理 displacement 的 DPO-Shift 等变体；
- 保持 reference、chat template 和 tokenizer 一致；
- 对安全数据独立做生成式回归。

### 面试总结

DPO 只扩大相对 margin，所以 chosen 和 rejected 可以同时降，关键看 rejected 是否降得更多以及概率去了哪里。用绝对 logp、KL 和实际生成诊断，再通过数据去歧义、SFT 锚定和保守训练缓解。
