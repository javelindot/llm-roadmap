# 用过Map嘛,怎么用的,底层是如何实现的,什么结构,不同版本之间的差别,如何保证并发安全,为什么1.7有环链,为什么2的n次方容量,扩容位置变化

## 答案

这里的 Map 一般指 Java `HashMap`。回答时可以从使用方式、底层结构、JDK 7/8 差异、扩容和并发安全五部分展开。

### 1. 基本用法

```java
Map<String, PaymentOrder> orders = new HashMap<>();

orders.put(order.getId(), order);
PaymentOrder value = orders.get("P123");
orders.remove("P123");

orders.computeIfAbsent(
    merchantId,
    key -> loadMerchantOrders(key)
);
```

`HashMap` 允许一个 `null` key 和多个 `null` value，不保证遍历顺序。

### 2. 底层结构

JDK 7：

```text
数组 + 单向链表
```

JDK 8：

```text
数组 + 链表 + 红黑树
```

发生 hash 冲突时，元素进入同一个桶。JDK 8 中链表长度达到树化阈值 8，且数组容量至少为 64 时，会转成红黑树；否则优先扩容。元素减少到一定程度时可退化为链表。

平均查找、插入复杂度是 O(1)，冲突严重时链表最坏 O(n)，树化后最坏 O(logn)。

### 3. 为什么容量通常是 2 的幂

HashMap 定位桶时使用：

```java
index = (n - 1) & hash;
```

当 `n` 是 2 的幂时，`n - 1` 的低位全是 1，可以用位运算代替取模，并让 hash 的低位均匀映射到桶。

JDK 8 还会做 hash 扰动：

```java
h ^ (h >>> 16)
```

让高位信息参与低位索引，降低碰撞。

### 4. 扩容过程和位置变化

默认负载因子通常是 0.75：

```text
threshold = capacity * loadFactor
```

元素超过阈值后，容量一般扩为 2 倍。

因为容量翻倍，元素的新位置只有两种：

```text
原索引 j
或
j + oldCapacity
```

判断依据是：

```java
(hash & oldCapacity) == 0
```

JDK 8 因此可以把原链表拆成 low/high 两条链，不需要重新计算完整 hash。

### 5. JDK 7 为什么可能出现环链

JDK 7 扩容迁移使用头插法，会反转链表顺序。在多线程同时执行 resize 时，线程看到的旧链表和被另一个线程修改后的指针可能交错，导致节点互相指向，形成环。之后 `get` 遍历链表时可能死循环。

这不是说 JDK 8 的 `HashMap` 就线程安全。JDK 8 改进了迁移方式，降低了这一特定问题，但并发 `put`、resize、遍历仍可能数据丢失或结果不一致。

### 6. 如何保证并发安全

优先使用：

```java
ConcurrentHashMap<K, V>
```

也可以使用：

```java
Collections.synchronizedMap(new HashMap<>());
```

但后者通常是整表锁，吞吐较低。

JDK 7 的 `ConcurrentHashMap` 主要使用 Segment 分段锁；JDK 8 改为数组桶上的 CAS、`synchronized` 和红黑树结构，锁粒度更细。复合操作应使用原子 API：

```java
map.putIfAbsent(key, value);
map.compute(key, (k, oldValue) -> update(oldValue));
map.merge(key, value, mergeFunction);
```

不要写成：

```java
if (!map.containsKey(key)) {
    map.put(key, value);
}
```

因为检查和写入不是原子的。

### 7. 业务使用注意

跨境支付系统中可以用 Map 做请求内聚合、配置快照、短期缓存，但不能把单机 `HashMap` 当分布式账本或幂等系统。订单幂等、余额和清算状态应由数据库、分布式缓存或一致性存储保证。

### 8. 面试总结

HashMap 在 JDK 7 是数组加链表，JDK 8 增加红黑树；容量取 2 的幂是为了用 `(n-1)&hash` 高效定位并简化扩容。JDK 7 并发 resize 的头插迁移可能形成环，但 JDK 8 HashMap 仍不线程安全；并发场景应使用 `ConcurrentHashMap` 和原子复合操作。
