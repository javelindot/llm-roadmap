# dpo训练对于系统问答的优化在哪里

## 答案

DPO 对系统问答的主要优化是**回答偏好和行为边界**，不是凭空补充模型知识。

### 可以优化的方面

- 正确回答优于事实错误或无依据回答；
- 引用证据完整、忠于检索内容；
- 简洁直接，减少复读和无关展开；
- 遵循格式、语气和多轮上下文；
- 证据不足时合理拒答或追问；
- 对提示注入、敏感请求和越权操作选择安全行为；
- 工具调用前后给出合适解释。

偏好数据可来自同一问题的多个候选，由人工、规则或可靠 Judge 选 chosen/rejected。应按错误类型构造负例，例如“内容流畅但引用错误”，避免模型只学习长度与风格。

### 不能替代的部分

- 新商品、库存、费率和规则更新应使用 RAG/工具；
- 检索漏召回应改 query、索引和 reranker；
- JSON schema 与权限应由 constrained decoding 和服务端校验；
- 复杂多步结果可验证时，在线 RL 可能比静态 DPO 更适合；
- 高风险安全不能只靠偏好模型。

### 评估

除 pairwise win rate，还要看答案正确率、faithfulness、引用准确率、拒答精确率、格式通过率、长度偏差和通用能力回归。

跨境支付问答中，可偏好“引用当前国家与通道规则、明确时效边界”的回答，拒绝承诺未经查询的到账时间。订单状态仍由实时系统提供。

### 面试总结

DPO 能把系统问答从“会回答”优化到“更符合偏好和边界”，尤其适合证据忠实、拒答、风格和格式；它不解决知识缺失、检索失败和权限控制。
