# 常用的不同模态数据在embedding层面对齐的方法有哪些？

## 答案

不同模态 embedding 对齐的目标，是让语义匹配的图像、文本、音频等表示在共享空间中接近，不匹配样本远离，同时保留各模态完成任务所需的信息。

### 对比学习

最常见是 CLIP 类双塔结构。各模态经过独立 encoder 和 projection head，做 L2 归一化，通过批内负样本优化对称 InfoNCE：

$$
s_{ij}=\frac{v_i^Tt_j}{\tau}
$$

图到文和文到图分别做交叉熵。大 batch、优质负样本和可学习温度会显著影响效果。要避免把语义相近样本误当作强负例。

### 匹配、回归与蒸馏

- Pair matching：预测一对样本是否匹配，可用 cross-encoder 学习更细交互。
- 距离回归：直接最小化配对 embedding 的 MSE、cosine loss 或 triplet loss。
- Knowledge distillation：用强模态或教师模型的 embedding、相似度矩阵和 soft target 监督弱模态。
- Center/prototype alignment：让不同模态同类别样本靠近共享原型。

### Token 级对齐

全局向量只能表达整体语义。细粒度任务可用 cross-attention、optimal transport、区域与词对齐、音频帧与视频帧对齐，或查询 token 从另一模态抽取信息。若配对时间不精确，可采用多实例学习和局部窗口。

### 分布级对齐

可用 MMD、CORAL 或 adversarial domain alignment 缩小不同模态的整体分布差异，但单纯让分布不可区分可能损失语义，通常要与实例级或类别级目标联合。

评估应包括跨模态 Recall@K、零样本分类、聚类和下游任务，并检查不同语言、噪声和缺失模态。投影到同一维度不等于完成语义对齐，关键仍是配对数据、负样本和训练目标。
