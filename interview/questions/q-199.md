# Python 熟练度怎样？

## 答案

> Python 是我的主力语言，写了 6 年了，AI 项目从数据处理到训练到部署全栈都用。
>
> **常用的库和场景**：
>
> - **数据处理**：pandas、numpy、polars（大数据时用）
> - **训练**：PyTorch、transformers、accelerate、deepspeed、peft（LoRA）
> - **推理部署**：vLLM、TensorRT-LLM、ONNX、Triton Inference Server
> - **RAG**：LangChain → LangGraph、LlamaIndex（部分场景）
> - **向量库**：pymilvus、qdrant-client
> - **OCR / 多模态**：PaddleOCR、Qwen-VL
> - **Web 后端**：FastAPI、Pydantic
> - **任务调度**：Celery、APScheduler、Airflow
> - **测试**：pytest、hypothesis（property testing）
> - **类型注解**：标准用 type hints + mypy
>
> **工程实践**：
> - 项目用 **poetry / uv** 管依赖
> - 代码用 **ruff + black + mypy** 静态检查
> - **logging**（结构化日志 structlog）+ **OpenTelemetry trace**
> - 单元 + 集成测试 + e2e 测试
> - CI / CD 用 GitHub Actions / GitLab CI
> - 容器化 Docker + K8s
>
> **熟悉的语言机制**：
> - asyncio / aiohttp（异步 IO）
> - typing（Protocol、TypedDict、Generic）
> - context manager（含 `__enter__` / `__exit__`）
> - 装饰器、元类（少用但懂原理）
> - GIL 限制和绕过（multiprocessing、threading 适用场景）
>
> **写过的非典型 Python 项目**：
> - 自研 RAG 评测平台（基于 RAGAS 魔改）
> - 自研 Agent 监控大盘（trace + token 消耗）
> - 自研模型仓库代理服务（HF API 兼容）
>
> **不熟的方向**：
> - 纯算法竞赛（leetcode 中等以下 OK，hard 看运气）
> - 桌面 GUI / Tkinter（业务用不到）
>
> 如果要做编程题，**最舒服的是 LLM 工程化、数据处理、API 设计、并发**这几块。

**🪝 顺嘴带出去的钩子**：

- "FastAPI 怎么和 vLLM 做高并发部署可以展开。"
- "自研评测平台的设计可以讲讲。"
