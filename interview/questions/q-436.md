# 🔥 介绍RAG项目

## 答案

**一句话**：RAG 项目可以介绍成一个企业知识问答系统，核心是把私有知识处理成可检索知识库，再让大模型基于检索证据回答。

### 1. 项目背景

业务问题通常是：

- 知识分散在文档、FAQ、工单和系统里。
- 规则更新快，模型参数知识跟不上。
- 人工查询成本高。
- 回答需要有依据，不能胡编。

### 2. 项目架构

- 文档采集和解析。
- 清洗、切分、标题注入、metadata。
- embedding 入库。
- query rewrite。
- 向量 + 关键词混合召回。
- rerank。
- prompt 构造和答案生成。
- 引用、评估、badcase 回流。

### 3. 结合跨境支付项目怎么说（面试口径）

> 我做的 RAG 项目可以放在跨境支付客服知识库里理解。知识来源包括 FAQ、通道规则、错误码说明、KYC/KYB 材料要求、退款拒付 SOP。用户问“为什么这笔付款没到账”，系统会根据国家、币种、通道和错误码检索相关文档，再生成有依据的解释。
>
> 我负责的重点是文档切分、元数据设计和检索优化。比如错误码要和处理建议放在同一个 chunk，国家和通道作为 metadata 做过滤，避免拿错规则。
>
> 上线后用真实工单评估召回率、引用命中率、人工改写率和转人工率，持续优化 badcase。

**钩子**：可以引到“文档治理、混合召回、metadata、真实工单评测”。
