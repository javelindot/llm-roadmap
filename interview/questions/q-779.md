# LoRA 初始化怎么做的，用的秩是多少，为什么不选其他的数

## 答案

LoRA 将权重增量写为：

$$
\Delta W=\frac{\alpha}{r}BA
$$

初始化要保证训练开始时 $\Delta W=0$，使模型初始行为与基础模型一致，同时至少有一条分支能收到梯度。

### 常见初始化

一种做法是 $A$ 使用小随机数或 Kaiming 初始化，$B=0$。开始时对 $B$ 有梯度，$B$ 更新后 $A$ 也获得有效梯度。也可反过来让一个矩阵为零，但不能两者都为零，否则二者初始梯度都可能为零。

LoRA dropout 作用在输入分支，$\alpha/r$ 控制更新尺度。某些变体使用 PiSSA、LoftQ 等基于原权重 SVD 或量化误差的初始化，但应与标准 LoRA 区分。

### Rank 如何选择

rank 表示增量最多包含的独立方向数。选择依据包括任务偏移、数据量、目标层和资源：

- 格式/风格轻微适配可从较小 rank 开始；
- 领域偏移大、FFN 层也需适配时可能需要更大 rank；
- 相同 rank 在 Q/V 和 FFN 上的容量意义不同。

实际应扫描如 4、8、16、32 等候选，保持训练 token 和其他配置一致，比较目标指标、通用回归、峰值显存和适配器大小。若增大 rank 已无收益，说明容量不再是瓶颈；若持续提升，可扩大 rank、目标模块或改全参。

项目回答应给真实 rank、alpha、target modules 和消融依据，不能只说“行业常用 8”。
