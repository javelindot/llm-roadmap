# 常见的量化方法有哪些及其原理

## 答案

量化是用低比特数值近似高精度权重、激活或 KV Cache，以减少存储、内存带宽和计算成本。核心线性量化可写为：

```text
q = clip(round(x / scale) + zero_point, q_min, q_max)
x_hat = scale * (q - zero_point)
```

误差来自 rounding、clipping 和 scale 粒度。

### 1. 对称与非对称

**对称量化**

- `zero_point = 0`；
- 范围围绕 0；
- 实现和矩阵乘更简单；
- 权重常用。

**非对称量化**

- 使用非零 zero point；
- 更好覆盖偏移分布；
- 适合非对称激活，但计算处理更复杂。

### 2. 标定粒度

- Per-tensor：整个张量一个 scale，快但误差大；
- Per-channel：每个输出通道一个 scale，权重精度更好；
- Per-group：每组若干权重一个 scale，INT4 常用；
- Per-token：每个 token 动态标定激活；
- Mixed precision：敏感层或通道保留更高精度。

粒度越细，scale 元数据和 Kernel 复杂度越高。

### 3. PTQ 与 QAT

**PTQ**

训练后用少量校准数据量化，无需完整训练，成本低。GPTQ、AWQ、SmoothQuant 属于代表方法。

**QAT**

训练时模拟量化误差，通过 Straight-Through Estimator 更新参数，通常精度更好，但训练成本、数据和实现复杂度更高。

**QLoRA**

冻结 4-bit NF4 基座，在较高计算精度下训练 LoRA Adapter，目标是低显存微调，不等于通用推理量化算法。

### 4. 常见 LLM 方法

**Weight-only INT8/INT4**

- 权重低比特，激活保持 FP16/BF16；
- 主要减少模型存储和权重带宽；
- GPTQ 使用近似二阶信息逐层校正；
- AWQ 根据激活统计保护显著权重通道。

**W8A8**

- 权重和激活都 8-bit；
- 可利用整数或 FP8 Tensor Core；
- SmoothQuant 将激活 outlier 的量化难度等价迁移到权重。

**FP8**

- 浮点 8-bit，动态范围通常优于 INT8；
- 是否提速高度依赖 Hopper、Ada、AMD 等硬件与 Kernel；
- 需要处理 E4M3/E5M2 等格式和 scale。

**KV Cache 量化**

- 将 K/V 缓存量化到 FP8/INT8/更低；
- 长上下文和高并发节省显存显著；
- 需要评测长距离注意力和累积误差。

**GGUF/CPU 量化**

- 常用于 llama.cpp 等 CPU/边缘推理；
- 多种混合位宽格式；
- 与 GPU GPTQ/AWQ 的 Kernel 和文件格式不同。

### 5. 量化不一定加速

权重文件变小不代表端到端更快。收益依赖：

- 硬件是否有对应低精度指令；
- 是否有 fused dequant + GEMM Kernel；
- group size、packing 和内存布局；
- batch、prefill/decode 比例；
- tensor parallel 通信；
- CPU/GPU 数据搬运；
- 推理框架实际支持。

如果每次都先反量化到 FP16，可能只省显存而没有明显提速。

### 6. 评测

- Perplexity 和通用 benchmark；
- 业务黄金集、工具调用和结构化输出；
- 小语种、数学、代码和长上下文；
- TTFT、TPOT、吞吐和 P95/P99；
- 峰值显存、功耗和单位请求成本；
- 不同 batch 与上下文长度。

跨境支付还要回归金额、币种、状态、引用和安全拒答。量化模型若在平均分上接近，但高风险字段错误率上升，不能上线。
