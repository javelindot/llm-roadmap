# SQL可执行率是怎么提升的？

## 答案

**一句话**：提升 SQL 可执行率，关键是让模型少猜，多依赖 schema、指标口径、示例 SQL 和校验修复闭环。

### 1. 提升方法

- 提供准确的数据字典和表结构。
- 做 schema linking，把自然语言映射到表、字段、指标。
- 提供 few-shot SQL 示例。
- 限制模型只生成 `select`。
- 用 SQL Parser 做语法校验。
- 执行前检查字段、表、权限、时间范围和 limit。
- 执行失败后把错误信息反馈给模型做一次修复。
- 高频问题沉淀成模板 SQL。

### 2. 关键不是只调 prompt

NL2SQL 最容易失败的原因是模型幻觉字段、表关联错、指标口径错、权限漏过滤。所以要把 SQL 生成做成“生成 + 校验 + 修复 + 执行”的闭环，而不是一次生成就执行。

### 3. 结合跨境支付项目怎么说（面试口径）

> 在跨境支付问数项目里，我会先建设指标口径库和数据字典。比如付款成功率到底按订单数、金额还是最终状态算，必须提前定义，不能让模型自由发挥。
>
> 生成 SQL 时，模型先检索相关表结构和指标说明，再生成 SQL。执行前用 Parser 校验只允许查询语句，检查字段是否存在、商户权限是否加上、时间范围是否合理。如果 SQL 执行失败，再把数据库错误反馈给模型做一次修复。
>
> 对高频问题，比如“按国家统计昨日付款失败率”，可以沉淀模板，减少模型自由生成，提高可执行率和稳定性。

**钩子**：可以引到“schema linking、SQL Parser、指标口径库、失败自修复”。
