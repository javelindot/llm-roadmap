# 什么是生成式大模型？

## 答案

**一句话**：生成式大模型是能够根据输入上下文生成新内容的模型，比如文本、代码、图片、语音或多模态内容。

### 核心回答
生成式模型和判别式模型的区别：
- 判别式模型回答"这个输入属于哪类"。
- 生成式模型学习数据分布，可以生成新的样本或回答。

在 LLM 中，生成通常是自回归过程：
1. 输入 prompt。
2. 模型预测下一个 token 的概率分布。
3. 采样或贪心选择 token。
4. 把新 token 拼回上下文继续生成。

常见能力：问答、摘要、翻译、代码生成、报告生成、对话、图像生成等。

### 1. 生成式模型和判别式模型的区别

判别式模型更像"判断器"，例如判断一笔交易是否高风险、一个工单属于哪个分类。生成式模型更像"内容生产器"，它能根据上下文生成解释、摘要、方案、代码或多模态内容。

在大语言模型里，生成过程通常是自回归的：

1. 给定 prompt 和上下文。
2. 模型预测下一个 token 的概率。
3. 根据 decoding 策略选择 token。
4. 把新 token 拼回上下文继续预测。

### 2. 生成式大模型的风险

生成能力强，也带来几个问题：

- **幻觉**：可能生成看似合理但不存在的事实。
- **不可控**：输出格式、长度、语气可能不稳定。
- **成本和延迟**：长文本生成会消耗更多 token。
- **安全问题**：可能泄露敏感信息或被提示注入诱导。

所以生产系统常配合 RAG、工具调用、输出格式约束和人工审核。

### 3. 面试总结

生成式大模型不是只会聊天，它的核心价值是把输入信息转化为新的、有用的表达。但在高风险业务里必须让它基于证据生成。

### 4. 结合跨境支付项目怎么说（面试口径）

> 在跨境支付里，生成式大模型的价值不只是聊天，而是把复杂业务信息生成可读、可执行的结果。比如一笔交易被风控拦截，系统里有规则命中、设备指纹、历史交易、国家风险、商户等级等很多字段，运营不可能每次都自己拼报告。
>
> 大模型可以生成一段审核摘要：为什么命中、证据是什么、建议补什么材料、是否需要人工复核。客服场景也类似，模型可以根据通道规则和订单状态生成用户能理解的解释。
>
> 但支付场景里我会要求它基于真实证据生成。比如交易状态必须来自工具查询，合规规则必须来自 RAG 文档，模型不能自己编原因。这样生成能力才是提效，而不是制造风险。

**钩子**：生成式模型适合把结构化风控证据转成可读、可审计的业务解释。
