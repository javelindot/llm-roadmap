# BERT模型相较于Transformer架构的优化是什么？

## 答案

严格说，BERT 不是对 Transformer 基础算子的全面“优化”，而是把原始 Encoder-Decoder Transformer 改造成适合语言理解预训练的 Encoder-only 架构，并设计了新的输入和训练目标。

### 1. 架构取舍

原始 Transformer 用于机器翻译：

```text
Encoder + autoregressive Decoder + Cross-Attention
```

BERT 只保留 Encoder：

```text
Bidirectional Transformer Encoder
```

因此没有 Decoder 和 Cross-Attention，也不做逐 token 自回归生成。

### 2. 双向注意力

BERT Encoder 不使用 causal mask，每个 token 可看完整输入：

```text
P(x_i | x_left, x_right)
```

这对分类、抽取和阅读理解更合适。GPT 类模型则预测：

```text
P(x_i | x_<i)
```

二者目标不同，不应简单说谁“更先进”。

### 3. MLM 预训练

BERT 随机选择 token：

- 80% 替换为 `[MASK]`；
- 10% 替换为随机 token；
- 10% 保持原词；

然后预测原 token。这样可以在不泄漏答案的前提下学习双向表示。

### 4. 句对输入

BERT 增加：

- `[CLS]`：聚合分类表示；
- `[SEP]`：分隔句子；
- Segment/Token Type Embedding：区分句 A、句 B；
- NSP：原版用于句间关系预训练。

### 5. 下游适配

- 分类：使用 `[CLS]`；
- NER：逐 token 分类；
- 抽取式 QA：预测答案起止位置；
- 句对匹配：联合编码两段文本。

这使“预训练一次、微调多任务”成为统一范式。

### 6. BERT 自身并非最终形态

后续改进包括：

- RoBERTa：更大数据、动态 mask、去掉 NSP；
- ALBERT：参数共享与 embedding factorization；
- DeBERTa：解耦内容和位置表示；
- Longformer 等：稀疏注意力处理长文本。

**面试总结**：BERT 的贡献重点是“Encoder-only + 双向 MLM + 通用预训练迁移”，不是修改了 Self-Attention 的数学公式。
