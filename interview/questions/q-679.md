# 相比 PPO，GRPO 的“Group”体现在哪一步？

## 答案

GRPO 中的 “Group” 体现在：对同一个 prompt 一次采样一组回答，用组内奖励的相对位置构造优势，而不是主要依赖单独训练的 critic 估计 value。

### 组内优势

对 prompt $q$ 采样 $G$ 个回答 $o_1,\ldots,o_G$，得到奖励 $r_1,\ldots,r_G$。常见标准化优势为：

$$
\hat A_i=\frac{r_i-\operatorname{mean}(r_1,\ldots,r_G)}
{\operatorname{std}(r_1,\ldots,r_G)+\epsilon}
$$

因此一个回答是否“好”是相对于同一 prompt 的其他回答判断。所有回答都很高时，组内仍区分相对优劣；若奖励完全相同，优势接近 0，该组几乎不提供学习信号。

### 与 PPO 的关系

GRPO 仍使用新旧策略概率比和 PPO 风格的 clipping，并通常加入 reference policy 的 KL 约束。主要差别是省去或弱化 value model，通过 group-relative reward 得到优势，降低 critic 的训练和显存成本。

### 为什么按同一 Prompt 分组

不同 prompt 的奖励尺度和难度可能差异很大。组内比较能消除部分 prompt 难度基线，让模型关注“同一道题哪种回答更好”。可验证奖励场景中，例如数学和代码，一次生成多条解答后按正确性评分尤其合适。

### 局限

每个 prompt 需要多次采样，rollout 成本较高；group size 太小会使优势估计噪声大，太大则吞吐下降。奖励稀疏或组内全同会缺少信号，标准差很小时还要防数值不稳定。GRPO 不等于没有基线，而是把组均值作为相对基线。
