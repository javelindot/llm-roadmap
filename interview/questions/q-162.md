# Vit图像补丁嵌入是什么？

## 答案

**一句话**：ViT 的图像补丁嵌入是把图片切成固定大小的 patch，把每个 patch 展平成向量后线性投影成 token embedding，再送入 Transformer。

### 核心回答
流程：
1. 输入图片，例如 224x224。
2. 切成 16x16 的 patch。
3. 每个 patch 展平成一维向量。
4. 通过线性层映射到 embedding 维度。
5. 加位置编码。
6. 像文本 token 一样送入 Transformer。

如果 224x224 图片切 16x16 patch，会得到 14x14=196 个图像 token。

ViT 把图像问题转成 token 序列建模问题。

### 1. Patch Embedding 为什么重要

Transformer 原本处理的是 token 序列，图片不是天然序列。ViT 的做法是把图片切成 patch，每个 patch 当成一个视觉 token，再通过线性投影进入 Transformer。

这一步相当于把二维图像转成一维 token 序列，让图像可以使用和文本类似的 self-attention 机制。

### 2. Patch 大小影响什么

- patch 越小，视觉 token 越多，细节保留更好，但计算更贵。
- patch 越大，token 更少，计算更省，但细节可能丢失。

所以 patch size 是细节和成本的平衡。

### 3. 面试总结

ViT 的 patch embedding 是图像进入 Transformer 的入口，本质是把图像局部区域 token 化。

### 4. 结合跨境支付项目怎么说（面试口径）

> 在商户资质审核里，ViT 可以用于证照、店铺截图、商品图这类图片理解。patch embedding 本质上是把图片切成一个个视觉 token，让 Transformer 学习不同区域之间的关系。
>
> 比如营业执照上公司名、统一社会信用代码、印章位置分布在不同区域，ViT 能建模这些区域关系。店铺截图和商品图也可以通过视觉 token 判断是否和商户描述一致。
>
> 但支付资质审核里很多关键信息是文字，所以我不会只靠 ViT，还会结合 OCR、规则校验和人工复核。

**钩子**：ViT 适合图像区域关系建模，证照审核还要结合 OCR。
