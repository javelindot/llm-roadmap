# 谈谈大模型召回的理解？

## 答案

大模型应用中的“召回”通常指在进入精排和生成前，从海量候选中高召回地取回一个较小集合。召回目标首先是不漏掉正确证据，再由 Reranker 和生成器提高精度。

### 1. 召回与精排

```text
百万/亿级文档
-> 召回 Top 50~200
-> Rerank Top 5~20
-> Context Packing
-> LLM
```

召回强调覆盖和速度，精排强调准确顺序。若正确文档没有进入候选集，后续模型再强也无法恢复。

### 2. 常见召回路线

**Sparse/BM25**

- 基于词频和倒排索引；
- 错误码、订单字段、专有名词精确匹配强；
- 同义表达和跨语言较弱。

**Dense Embedding**

- 把 query/document 映射到向量空间；
- 通过 cosine/dot product + ANN 检索；
- 语义和多语言强；
- 可能忽略数字、版本和罕见词。

**Hybrid**

- BM25 + Dense；
- 使用 RRF 或归一化加权融合；
- 企业 RAG 常用默认方案。

**Multi-vector/Late Interaction**

- 文档保留多个 token/segment 向量；
- 细粒度匹配更强；
- 存储与计算更高。

**结构化/图召回**

- SQL、知识图谱、metadata filter；
- 适合实体、关系、时间和权限强约束。

### 3. 查询侧

- 意图路由；
- 拼写和术语归一化；
- Query Rewrite/Expansion；
- 多查询并行；
- 多轮指代消解；
- 国家、币种、通道、时间等过滤条件抽取。

改写必须保留订单号、错误码和实体，不能为了语义流畅改坏关键字段。

### 4. 评估

- Recall@K、Hit Rate；
- MRR/NDCG；
- 每路召回贡献和重叠率；
- 延迟、索引大小、更新时效；
- 按语言、业务线和难度分桶；
- 最终答案正确率与引用率。

TopK 不应拍脑袋，要根据 recall-latency-context 曲线选择。

### 5. 跨境支付

对错误码和卡组织规则使用 BM25；对自然语言客服问题使用 Dense；用 metadata 限制国家、币种、支付通道、生效时间和租户；实时交易信息走订单/账务工具。

**面试总结**：好的召回系统不是“只用一个最强向量模型”，而是让不同检索方式覆盖不同错误类型，再用精排和权限过滤收敛。
