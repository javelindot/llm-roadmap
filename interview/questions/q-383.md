# 大数据工作

## 答案

> 我大数据背景比较深，几年下来做过**数仓、实时计算、特征工程、BI 报表、数据治理**几大块。在跨境支付公司也一直和大数据团队协作，分享下我的经验。
>
> **熟悉的技术栈**：
>
> **离线数仓**
> - Hadoop / HDFS：底层存储
> - Hive / HiveQL：数仓建模
> - Spark（PySpark 主）：大规模批处理
> - Iceberg / Delta Lake：现代数据湖
> - dbt：数据建模 + 文档
>
> **实时计算**
> - Flink：状态流计算
> - Kafka：消息中间件
> - Kafka Streams / Spark Streaming：早期用过
>
> **存储 + 查询**
> - MySQL / TiDB：业务库
> - ClickHouse：OLAP
> - Doris：OLAP（替代 ClickHouse 调研中）
> - Elasticsearch：日志检索 + 全文
> - Redis：缓存
> - MinIO / OSS：对象存储
>
> **调度 + 编排**
> - Airflow：DAG 调度
> - DolphinScheduler：国内方案
> - Argo Workflow：K8s 原生
>
> **数据治理**
> - Apache Atlas：元数据
> - DataHub：现代数据目录
> - Great Expectations：数据质量
>
> **BI**
> - Superset：开源 BI
> - QuickBI / FineBI：商业 BI
>
> **几个具体项目经历**：
>
> **1. 跨境支付实时风控数仓**
> - Flink + Kafka 实时聚合每个用户 / 商户 / 设备的近 N 分钟交易特征
> - 给风控规则引擎实时打分
> - 状态后端用 RocksDB，单节点支撑 1w+ QPS
>
> **2. 离线数仓建模**
> - 分 ODS / DWD / DWM / DWS / ADS 五层
> - Hive + Iceberg 替换老 Parquet 表
> - dbt 管 SQL + 自动生成文档
>
> **3. 客户画像系统**
> - 每天计算每个客户 200+ 维特征
> - Spark 离线 + Flink 实时双流计算
> - HBase / Redis 存特征供线上查询
>
> **4. 数据质量治理**
> - 关键指标对账日报
> - Great Expectations 监控关键表
> - 失败告警 + 数据修复
>
> **大数据和 LLM 的结合点**：
>
> 这是我大数据背景能加分的地方：
>
> **1. RAG 数据 pipeline = ETL**
> - 文档采集、清洗、切块、入库
> - 用 Airflow 调度
> - 用 dbt 做配置化清洗规则
>
> **2. LLM 评测 = BI 报表**
> - 黄金集结果聚合
> - 失败 case 分析
> - 各业务线指标对比
>
> **3. 模型训练数据 = 数仓产出**
> - 用 Spark 从数仓抽样
> - 标注数据回流飞轮
> - 增量训练数据准备
>
> **4. Agent trace = 日志分析**
> - OpenTelemetry trace 进 ES / ClickHouse
> - Agent 行为分析、瓶颈定位
> - 失败模式聚类
>
> **5. 业务监控 = 实时指标**
> - LLM 调用 QPS、token 消耗、p95 延迟实时计算
> - Flink + Grafana
>
> **大数据 vs LLM 工程的差异**：
>
> | 维度 | 大数据 | LLM 工程 |
> |------|--------|----------|
> | 数据量 | TB-PB | GB-TB |
> | 时延要求 | 秒-分钟 | 毫秒-秒 |
> | 算力 | CPU 集群 | GPU 集群 |
> | 工具栈 | Hadoop/Spark/Flink | HuggingFace/vLLM |
> | 输出 | 报表 / 指标 | 文本 / 结构化 |
> | 评测 | 数值精确 | 概率分布 |
>
> **几个心得**：
>
> 1. **大数据基础在 LLM 里复用率 80%**——ETL、调度、监控、治理几乎一样
>
> 2. **数据质量比模型重要**——LLM 落地我有 1/3 时间在做数据准备
>
> 3. **流批一体在 LLM 落地很有用**——离线训练 + 实时推理
>
> 4. **数据治理意识对 LLM 合规帮助大**——PII、血缘、审计概念都通用
>
> 5. **Spark / Flink 处理 embedding 也好用**——大规模向量化计算
>
> **我自我评价**：
> - 大数据：8/10（独立设计 + 实现 + 优化）
> - LLM 工程：7/10（生产落地 + 调优）
> - 算法理论：5/10（够用但不深）
>
> 一句话：**大数据 + LLM 是黄金组合**——LLM 工程本质是"数据 + 模型 + 服务"的拼装，大数据背景让我在数据这一面有天然优势。
