# 分布式部署？vllm和TensorRT有什么不同？

## 答案

答：常用框架：
TensorRT-LLM，英伟达官方，速度最快，高吞吐低延迟，企业级大模型推理首选。
CTranslate2，轻量级、超小体积、速度快，适合边缘计算。
（TGI，哈根菲斯官方，部署简单，适合快速上线、云原生。
LightLLM，国产，比vllm更快、更省显存，适合Qwen、Llama、百川。
ONNX Runtime，通用模，可跨平台，适合小模型。）
选择标准：
高并发、低延迟场景用TensorRT-LLM；
快速迭代、多模型兼容、高吞吐量场景用vLLM；
小模型用CTranslate2。

### 补充说明

分布式部署要先明确目标：是提高吞吐、降低延迟，还是支持更大模型。常见方式包括多副本部署、张量并行、流水线并行、数据并行和模型网关路由。

vLLM 更偏通用大模型在线服务，模型兼容性和动态批处理能力好，适合快速支撑多模型推理。TensorRT-LLM 更偏 NVIDIA 生态下的高性能推理优化，性能强，但构建和适配成本更高。

### 结合跨境支付项目怎么说（面试口径）

> 在跨境支付项目里，如果是客服问答和风控解释这类线上服务，我会优先考虑 vLLM，因为它接入快、OpenAI API 兼容好、动态批处理适合高并发。对于特别追求低延迟、高吞吐、模型相对固定的场景，可以评估 TensorRT-LLM。
>
> 分布式上我会做模型网关，把不同业务路由到不同模型实例。比如客服实时问答走一组实例，离线 PDF 抽取走另一组实例，避免离线任务影响实时服务。
>
> 选择标准不是谁性能最高，而是看团队能不能维护、模型更新频率、部署环境和业务 SLA。支付场景稳定性和可降级比极限跑分更重要。

**钩子**：可以引到“vLLM 兼容性、TensorRT-LLM 性能、模型网关、实时离线隔离”。
