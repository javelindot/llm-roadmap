# 大模型的实现原理是什么

## 答案

大模型的实现可以从 Tokenization、Transformer 前向、预训练、后训练和推理五层理解。它本质上是一个根据已有 token 预测下一个 token 概率的参数化模型。

### 1. Tokenization

文本先通过 BPE/SentencePiece 等变成 token id：

```text
text -> token ids
```

Embedding 表将离散 id 映射为向量，并加入 RoPE 等位置信息。

### 2. Transformer

每层主要包括：

```text
x = x + Attention(Norm(x))
x = x + FFN(Norm(x))
```

Attention：

```text
Q=XW_Q, K=XW_K, V=XW_V
softmax(QK^T/sqrt(d))V
```

Causal mask 保证当前位置看不到未来 token。FFN 用 SwiGLU 等非线性变换提升表示能力。

### 3. 预训练

Decoder-only LLM 优化 next-token cross entropy：

```text
L = -sum_t log p(x_t | x_<t)
```

大规模语料、参数和算力让模型学习语法、知识、代码和任务模式。训练通过前向、反向传播和 AdamW 更新参数，并使用数据/张量/流水线/专家并行扩展。

### 4. 后训练

- Continue Pretraining：补领域或语言能力；
- SFT：学习指令、格式和工具调用；
- DPO/RLHF/GRPO：优化偏好、推理和可验证结果；
- 安全对齐：拒答、红队和政策行为；
- 蒸馏/量化：降低部署成本。

### 5. 推理

```text
prompt
-> tokenize
-> prefill
-> logits
-> sampling/greedy
-> append token
-> decode loop
```

Prefill 并行处理输入，Decode 逐 token 生成。KV Cache 保存历史 K/V，避免重复计算；服务端还使用 continuous batching、PagedAttention、量化和 speculative decoding。

### 6. 为什么能完成多任务

训练目标虽然是预测 token，但大量任务都能表达成文本或多模态序列。模型学习了条件分布和可复用表示，再通过指令和上下文将能力组合出来。

但生成概率高不代表事实正确，模型没有天然数据库事务、权限和业务责任能力。

### 7. 跨境支付场景

模型负责理解、检索、解释和工具编排；订单、账务、汇率、风控和合规结果来自权威系统。高风险动作由确定性状态机和人工审批执行。

### 8. 面试总结

大模型是 Tokenizer + Transformer + next-token training + 后训练 + 高效推理组成的系统。理解实现原理既要会 Attention 和 loss，也要能解释数据、分布式训练、KV Cache、工具和安全边界。
