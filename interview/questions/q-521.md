# 如果量化后理解能力下降怎么办？怎么做精度补偿？

## 答案

量化后能力下降，先不要直接做微调补救，而要确认损失来自 **量化位宽、异常通道、校准数据、量化粒度还是算子实现**。

### 1. 先定位退化范围

- 比较 FP16 与量化模型在通用、数学、代码、长文本等分项指标。
- 做逐层误差或输出重构误差分析。
- 检查是权重量化、激活量化还是 KV Cache 量化导致。
- 确认推理 Kernel 与量化配置一致，排除实现错误。

### 2. 提高量化配置

- 从 INT4 升到 INT8，或使用权重 4bit、激活 16bit。
- 使用 Per-channel / Per-group，而不是粗粒度 Per-tensor。
- 减小 Group Size，提高 Scale 精度。
- 对 Embedding、LM Head、首尾层或敏感层保留高精度。
- 使用混合精度保护异常值和关键通道。

### 3. 使用数据感知的 PTQ

- GPTQ：根据校准数据和二阶信息近似减少层输出误差。
- AWQ：根据激活识别重要权重通道并进行缩放保护。
- SmoothQuant：把激活异常值的量化难度迁移到权重，适合 W8A8。

校准集必须覆盖真实任务分布、序列长度和语言，否则离线误差小也可能线上退化。

### 4. 训练式补偿

- QAT：在训练中模拟量化误差，让模型适应低精度。
- 在量化基座上做 LoRA/QLoRA 恢复任务能力。
- 使用知识蒸馏，让量化模型对齐高精度教师的 Logits 或中间表示。
- 对极低位量化，可做层级重构或 Block-wise Fine-tuning。

### 5. 系统层权衡

有时更高精度但吞吐略低，比低位宽导致任务失败更划算。应同时评估：

- 任务正确率
- 延迟与吞吐
- 显存
- Kernel 是否真正加速
- 不同硬件的支持情况

**面试总结**：精度补偿的顺序通常是“定位敏感项 → 改量化粒度和校准 → 保护关键层 → 必要时 QAT/LoRA/蒸馏”，而不是盲目把全部模型重新训练。
