# Multi-Agent 场景里，Reflection 模块与 Memory 模块的输入输出各自是什么？

## 答案

Reflection 和 Memory 都会读取 Agent 轨迹，但职责不同：Reflection 负责评价与改进，Memory 负责持久化、检索和提供历史信息。

### Reflection 模块

输入通常包括：

- 当前任务目标、约束和成功标准；
- 本轮计划、行动、工具调用、观察结果与最终回答；
- 环境反馈、验证器结果、用户评价或失败原因；
- 可选的历史反思与预算。

输出不是简单的“成功/失败”，而应是结构化结果，例如错误类型、证据、根因、可复用经验、修正计划和是否重试：

```json
{
  "status": "needs_revision",
  "issues": ["检索证据不足"],
  "lessons": ["先核验时间范围"],
  "next_action": "补充检索并重写结论"
}
```

反思结果可立即反馈给 Planner/Executor 开启下一轮，也可经过筛选后写入长期记忆。

### Memory 模块

Memory 的写入输入包括对话、事件、工具结果、成功或失败轨迹、Reflection 提炼的经验，以及时间、来源、权限、置信度等元数据。写入阶段应去重、摘要、分块，并区分：

- episodic memory：发生过什么；
- semantic memory：稳定事实和知识；
- procedural memory：如何完成某类任务；
- working memory：当前任务的临时状态。

检索输入是当前目标、query、Agent 身份、时间范围和权限；输出是经过相关性、时效性和可信度排序的少量记忆片段及引用，而不是把全部历史塞入上下文。

### Multi-Agent 中的边界

共享 Memory 应带 Agent、租户和任务作用域，防止信息串扰；Reflection 可以由执行 Agent 自评，也可由独立 Critic 交叉评审。只有经过验证、具有复用价值的反思才应进入长期 Memory，否则错误结论会被后续 Agent 反复放大。
