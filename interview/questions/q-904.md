# RAG常用的检索方式

## 答案

RAG 检索方式可按词法、语义、结构、图和迭代检索分类。

### 1. 词法检索

- 倒排索引、TF-IDF、BM25；
- 适合错误码、SKU、专有名词、数字和新实体；
- 不擅长同义改写和跨语言语义。

### 2. 稠密向量检索

用双塔 Embedding 编码 query/document，通过 HNSW、IVF、PQ、DiskANN 等 ANN 搜索。适合语义匹配，但受 embedding 领域适配、长尾和 false neighbor 影响。

### 3. 稀疏神经检索

SPLADE 等学习词项扩展与权重，保留倒排可解释性，同时加入语义匹配，但索引稀疏度和训练成本需控制。

### 4. 混合与重排

BM25、dense、规则、图谱多路召回，使用 RRF 或学习融合，再用 Cross-Encoder/BGE/Qwen reranker 精排。这是生产 RAG 常用方案。

### 5. 结构化检索

- metadata filter；
- SQL/OLAP；
- 知识图谱和 GraphRAG；
- parent-child/hierarchical retrieval；
- 表格、图片、代码的多模态/专用索引。

### 6. Agentic/Iterative Retrieval

query rewrite、multi-query、HyDE、问题分解和多跳检索；模型根据已有证据判断是否继续搜索。需要限制步骤、成本和查询漂移。

跨境支付错误码用 BM25，语义工单用向量，国家/币种/通道/版本先过滤；订单、余额和费率查实时工具。

### 面试总结

RAG 常用 BM25、向量、稀疏神经、图/结构化和迭代检索。生产系统通常多路召回加 rerank，并将权限、版本和实时事实放在检索链路中。
