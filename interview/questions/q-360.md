# 怎么校验？

## 答案

**一句话**：校验要分层做，既要校验模型生成的 SQL 或工具参数，也要校验业务结果、权限、安全和最终回答是否可信。

### 1. 如果是 NL2SQL，怎么校验

常见校验包括：

- 语法校验：SQL 能否解析。
- 白名单校验：只允许 select，不允许 update/delete/drop。
- 表字段校验：字段必须存在，不能幻觉字段。
- 权限校验：用户是否有权查这张表、这些字段、这些数据范围。
- 性能校验：限制时间范围、limit、扫描量。
- 指标口径校验：是否使用正确的指标定义。
- 结果校验：异常值、空结果、极端值要提示或二次确认。

### 2. 如果是 Agent 工具调用，怎么校验

- 工具是否在白名单内。
- 参数是否符合 schema。
- 用户是否有权限。
- 业务状态是否允许操作。
- 是否需要人工确认。
- 调用结果是否和上下文一致。

### 3. 如果是最终回答，怎么校验

- 是否基于检索内容或工具结果。
- 是否包含敏感信息。
- 是否做了不确定承诺。
- 是否遗漏关键限制条件。
- 是否需要引用来源。

### 4. 结合跨境支付项目怎么说（面试口径）

> 在跨境支付问数项目里，校验非常重要。比如模型生成一条 SQL 查询付款失败率，首先要校验它是不是只读查询，有没有访问未经授权的商户数据，有没有使用正确的时间字段和状态字段。支付指标很容易因为口径不同得出不同结果，所以指标定义必须从指标库里取。
>
> 如果是客服 Agent 调工具，也要校验。用户查订单时，不能只要提供订单号就返回所有信息，还要校验这个用户是否属于该商户、能不能看这笔交易、敏感字段是否脱敏。
>
> 最终回答也要校验，不能把“处理中”说成“成功”，不能承诺“今天一定到账”，不能泄露完整风控规则。我的理解是，校验层是大模型系统能不能上生产的关键。
