# 项目性能测试关注哪些指标?

## 答案

性能测试应从业务负载和 SLO 出发，不只是报告平均响应时间。先定义请求类型、并发模型、数据规模、硬件、软件版本和预热条件，再测容量与尾部表现。

### 1. 通用指标

- 延迟：平均值以及 P50、P95、P99、最大值；
- 吞吐：QPS、TPS、任务数/秒或 tokens/s；
- 错误：超时率、失败率、重试率和降级率；
- 并发：在途请求数、队列长度和等待时间；
- 资源：CPU、GPU、显存、内存、磁盘与网络；
- 饱和度：线程池、连接池、队列和下游依赖容量；
- 稳定性：长时间 soak test 的泄漏、抖动和故障恢复；
- 弹性：扩缩容耗时、冷启动和突发流量；
- 成本：单请求、单交易或每百万 token 成本。

压测结果必须绑定负载条件。例如只说 P95 为 200 ms 没有意义，还要给出并发、QPS、输入大小、缓存命中率和错误率。

### 2. 大模型/RAG 指标

- TTFT、TPOT/ITL、端到端延迟；
- Prefill 和 Decode tokens/s；
- KV Cache 使用率和命中率；
- 满足 SLO 的 goodput；
- 检索 P95、召回率和重排时延；
- 首次工具调用、总工具调用和 Agent 完成时延；
- 不同上下文长度、输出长度和 batch 下的曲线。

性能不能脱离质量测试。量化、截断、缓存或并发提高后，还要检查正确率、幻觉率、检索质量和任务成功率。

### 3. 测试类型

1. 基线测试：固定环境下建立可复现结果；
2. 负载测试：逐步提高并发，找到拐点；
3. 压力测试：超过容量，验证限流和降级；
4. 突发测试：模拟短时间流量峰值；
5. 稳定性测试：长时间运行；
6. 故障测试：下游超时、节点退出和网络抖动；
7. 容量规划：根据峰值、冗余和增长估算资源。

### 4. 跨境支付场景

核心链路关注授权/支付成功率、端到端 P99、重复扣款、幂等冲突、账务一致性和通道超时。应按国家、币种、支付方式、通道和峰值时段构造流量，并测试下游机构慢响应、回调乱序和重试风暴。

涉及资金的校验、审计和幂等不能为了降低延迟而跳过。

### 5. 面试总结

项目性能测试要覆盖延迟、吞吐、错误、资源、饱和度、稳定性、弹性和成本，并始终说明工作负载。LLM 项目额外关注 TTFT、TPOT、KV Cache、goodput 及质量退化。
