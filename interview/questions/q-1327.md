# Post-Training 的工作机制，为什么要做三阶段训练？什么情况下应该用GRPO？为什么DeepSeek用了GRPO？如何从V3到R1？

## 答案

Post-Training 的目标是把 Base Model 的通用预测能力转成可用的指令、推理、工具和安全行为。所谓“三阶段”通常可概括为冷启动 SFT、强化学习、再对齐/蒸馏；DeepSeek-R1 的公开流程更细，实际包含两轮 SFT/RL 交替。

### 1. 为什么不能只做一个阶段

- Base Model 会续写，但不一定遵循指令；
- SFT 能教格式和示范路径，但主要模仿数据，探索受限；
- RL 可围绕结果奖励探索新策略，但容易可读性下降、奖励投机和能力遗忘；
- 最终通用/安全对齐负责恢复可读性、帮助性和多场景能力。

各阶段解决不同问题，不能把 RL 当成更昂贵的 SFT。

### 2. DeepSeek-R1-Zero

R1-Zero 从 DeepSeek-V3-Base 直接进行大规模 RL，不先做人工 CoT SFT，观察到自反思、验证和更长推理等行为涌现。

但它也暴露了：

- 可读性差；
- 中英文混杂；
- 输出格式不稳定；
- 通用和非推理能力不足。

这说明纯 RL 能激发推理，但未必直接得到好产品。

### 3. 从 V3-Base 到 R1

公开流程可概括为：

1. **Cold-start SFT**：少量高质量长 CoT，让模型先学会清晰、可读的推理格式；
2. **Reasoning-oriented RL**：使用规则/可验证奖励提升数学、代码和逻辑推理；
3. **Rejection Sampling + SFT**：从 RL checkpoint 采样，筛选正确高质量推理，再混入写作、事实问答、翻译等非推理数据做 SFT；
4. **Final RL**：同时优化帮助性、安全性与推理能力，得到最终 R1。

所以“三阶段”回答可以把第 3、4 步合为“通用与安全再对齐”，但最好说明论文实际是交替流程。

### 4. 什么时候用 GRPO

- 最终结果可用程序或规则验证；
- 需要探索多步推理；
- 同题能采样多个候选；
- 有足够 rollout 算力；
- SFT 已建立基本语言和格式能力。

不适合把 GRPO 直接用于无法可靠判分的开放式业务，或用模型自评代替全部真值。

### 5. 为什么 DeepSeek 使用 GRPO

- 组内相对奖励可替代单独 Critic；
- 降低 PPO 的 Value Model 显存和训练复杂度；
- 数学、代码等任务有可验证结果奖励；
- 大规模采样可提供同题候选间的比较信号。

代价是 rollout 昂贵、组内无方差时没有信号，仍需 clip、KL、数据和 reward 治理。

### 6. 跨境支付的适用边界

可对 SQL 执行、工具参数、对账金额匹配、流程合规等可验证任务做 RL；监管判断、商户风险定性和冻结资金不能只依赖模型奖励，必须由规则、证据和人工审批完成。
