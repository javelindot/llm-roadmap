# 如何处理 prompt 和生成的图像不对齐的问题？

## 答案

**一句话**：解决图文不对齐有五条路——**更强的文本编码器（T5/CLIP-L 替代 CLIP-B）、Classifier-Free Guidance、Prompt 工程、Attention 控制（IP-Adapter/ControlNet）、对齐微调（DPO 风格）**。

### 1. 什么叫"不对齐"
- prompt 说"3 个红苹果"，生成 2 个
- prompt 说"一只穿西装的猫"，生成的没穿
- 颜色、数量、空间关系不准
- 复杂组合（多对象 + 属性绑定）失败

### 2. 五大解决思路

**1) 升级文本编码器**
- CLIP-B → CLIP-L / OpenCLIP / **T5-XXL**
- T5 编码器能理解更长、更复杂的文本
- SD3、FLUX 已经把 T5 当主力

**2) Classifier-Free Guidance（CFG）**
- 关键技巧
- 训练时随机 drop 文本条件
- 推理时 noise = uncond + scale × (cond - uncond)
- scale 越大越听 prompt，但过大失真

**3) Prompt 工程**
- 结构化描述
- 加质量词、负 prompt
- 顺序：主体 → 属性 → 风格 → 质量
- LLM rewrite prompt（OpenAI DALLE-3 思路）

**4) 显式控制（Attention 干预）**
- **Attend-and-Excite**：增强对齐属性的 attention
- **Prompt-to-Prompt**：编辑特定 token 的 attention map
- **ControlNet**：用边缘 / 深度 / 姿态额外控制
- **IP-Adapter**：用参考图控制

**5) 对齐微调**
- **Diffusion-DPO**：用 (prompt, chosen_image, rejected_image) 偏好对训练
- 类似 LLM 的 DPO 流程
- 让模型偏好"对齐 prompt"的输出

### 3. 业界案例
- **DALL-E 3**：用 LLM rewrite prompt + 大量对齐微调
- **SD3 / FLUX**：T5 编码 + 双流 DiT
- **Midjourney**：黑盒，但显然有 RLHF
- **Imagen 2**：T5 + 大模型 cascade

### 4. 评估对齐
- **CLIP Score**：图文 embedding 相似度
- **TIFA**：用 VQA 模型问"图里有 X 吗"
- **DPG-Bench**：复杂指令对齐
- 人工评（最准但贵）

### 5. 工程经验
- 文本编码器是上限决定因素
- CFG scale 3.5-7.5 是甜点
- 复杂 prompt 用 LLM rewrite 拆分
- 关键场景配 ControlNet
- 业务可控用偏好微调

### 6. 一句话总结
**图文不对齐 = 文本理解差 + 生成空间大 + 训练目标弱**——升级文本编码器是上限、CFG 是基线、LLM rewrite 是便宜大招、ControlNet 是精细控制、DPO 是终极对齐。

## 结合项目谈（口语版 · 约 25 秒）

> 我们跨境支付**图像生成场景很少**——业务上几乎用不到生成图，更多是识别/理解（多模态 VLM）。但我们有一个**营销素材生成**项目用过文生图，分享下踩坑。
>
> 业务场景：客服营销活动需要生成"祝贺商户成功完成第一笔跨境收款"这种活动 banner。我们用 SD3 做生成。
>
> 一开始 prompt 全英文，结果生成的图经常"3 个图标变 2 个"、"绿色背景变蓝色"。改进路径：
>
> **改进一：升级文本编码器**——SD2.1 用的是 CLIP-G，到 SD3 切到 T5-XXL，复杂 prompt 的对齐度明显好转（CLIP Score +8）。
>
> **改进二：CFG scale 调优**——默认 7.5 我们调到 5，太高生成会过饱和、太低不听话。
>
> **改进三：LLM rewrite**——把"祝贺商户首笔收款，要喜庆、专业"用 GPT-4 改写成结构化 prompt（主体 / 属性 / 风格 / 质量），对齐度再涨。
>
> **改进四：ControlNet 控关键元素**——业务硬要求"banner 里必须有公司 logo 在左上角"，用 ControlNet + 模板图位置约束。
>
> 我们没上 Diffusion-DPO（数据成本高、ROI 不够）。
>
> 一句话：**图文不对齐治理是"上限靠编码器、效果靠 CFG、灵活靠 prompt 工程、精细靠 ControlNet"四件套**——一般业务 prompt 工程 + CFG 调优能解 70% 问题。

**🪝 顺嘴带出去的钩子**：
- "T5 vs CLIP 当文本编码器的对齐效果对比可以聊。"
- "ControlNet 在业务约束场景的应用可以展开。"
- "Diffusion-DPO 的偏好数据怎么收集是个挑战。"
