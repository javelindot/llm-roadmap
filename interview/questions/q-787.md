# 如何提高模型的推理速度

## 答案

推理速度要先区分 TTFT、每 token 延迟、吞吐和端到端 P99。优化吞吐可能增加单请求等待，不能只报 tokens/s。

### 1. 模型与计算

- 选满足质量要求的最小模型，或蒸馏、剪层、结构化稀疏；
- INT8/FP8/INT4 量化权重，必要时量化 KV Cache；
- 使用 FlashAttention/FlashInfer、融合算子和优化 GEMM/MoE kernel；
- `torch.compile`、CUDA Graph 和静态 shape 减少 launch 开销；
- speculative decoding 在接受率高时减少目标模型 decode 次数。

### 2. KV 与上下文

- GQA/MQA 减少 KV head；
- PagedAttention 降低 KV 内存碎片；
- prefix caching 复用公共前缀；
- 控制 prompt 和输出长度，RAG 只提供相关证据；
- sliding window 或上下文压缩限制历史规模。

### 3. 调度与并行

- continuous batching 提高 GPU 利用率；
- chunked prefill 避免长 prompt 长时间阻塞 decode；
- 按长度分桶和 deadline 调度；
- Tensor Parallel 降低单模型延迟，Pipeline Parallel 常增加流水线延迟，主要用于放下模型；
- 高负载下可做 Prefill/Decode 分离和副本扩缩容。

### 4. 业务系统

缓存相同结果、异步并行外部工具、连接池、流式返回和小模型路由。优化前用 profiler 确认是算力、显存带宽、KV 容量、排队还是网络瓶颈。

跨境支付模型还需保证数值、权限和审计正确。不能为了低延迟跳过订单状态和合规查询；可设置超时后降级，不可生成未经验证的资金结论。

### 面试总结

提高推理速度要从模型压缩、kernel、KV Cache、批处理、并行和业务上下文共同优化。先确定目标是 TTFT、ITL 还是吞吐，再用真实流量压测质量、P99、显存和成本。
