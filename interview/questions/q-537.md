# 开发一个Agent时最大的难点是什么？

## 答案

开发 Agent 最大的难点通常不是让模型调用一次工具，而是让它在不确定环境下 **长期、稳定、可控地完成任务**。

### 1. 可靠性

- 模型可能选错工具或填错参数。
- 工具返回异常、超时或部分成功。
- 多步执行中一个早期错误会被后续放大。
- 重试可能产生重复写入等副作用。

需要参数校验、幂等、重试策略、超时和补偿机制。

### 2. 状态管理

对话历史不等于任务状态。生产 Agent 应显式保存：

- 当前目标和步骤；
- 已完成动作与证据；
- 待确认信息；
- 错误、重试和审批状态。

这样任务中断后才能恢复，也便于审计。

### 3. 边界和权限

- 不同工具使用最小权限。
- 高风险动作需要人工确认。
- 对外发消息、写数据库、支付等操作要有审批点。
- 防止 Prompt Injection 借外部内容触发越权工具。

### 4. 评测

Agent 不能只看最终回答好不好，还要评估：

- 任务完成率；
- 工具选择和参数正确率；
- 路径长度与成本；
- 错误恢复率；
- 高风险动作误触发率。

### 5. 可观测性

必须记录每步输入、决策、工具调用、结果、耗时和 Token 成本，支持 Trace、回放和 Bad Case 复现。

### 6. 实践原则

确定性流程用状态机，开放式判断交给模型；能用普通代码完成的部分不要强行 Agent 化。

**面试总结**：Agent 工程的核心矛盾是模型概率性与业务确定性之间的冲突，真正的工作在状态、工具、权限、评测和兜底。
