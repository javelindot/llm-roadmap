# 介绍lora，为什么lora是有效的（只需要训练更少的参数而不是训练全参数）

## 答案

LoRA 冻结预训练权重，只学习低秩权重增量：

\[
W'=W+\Delta W,\qquad
\Delta W=\frac{\alpha}{r}BA
\]

其中 \(A\in\mathbb R^{r\times d_{in}}\)、\(B\in\mathbb R^{d_{out}\times r}\)，且 \(r\ll d\)。可训练参数从 \(d_{out}d_{in}\) 降为 \(r(d_{in}+d_{out})\)。

### 为什么低秩有效

- 预训练模型已经具备大部分语言和任务能力，下游适配主要是在已有表示上调整；
- 许多任务所需的权重变化具有较低“内在维度”，不需要独立改变所有参数；
- 在过参数化网络中，全参更新矩阵常有能量集中于少数奇异方向的现象；
- 低秩约束也是一种正则化，小数据时可减少过拟合和灾难性遗忘；
- 将 LoRA 注入多个层后，每层少量低秩方向组合起来仍有较强表达能力。

[LoRA 原论文](https://arxiv.org/abs/2106.09685)对适配更新的 rank deficiency 做了经验分析，但不能把“所有任务更新都严格低秩”当作定理。

### 何时可能不够

- 领域与基座差异极大；
- 数据充足且需要获得新底层能力；
- rank、目标模块或模型容量太小；
- 需要修改 tokenizer/embedding 或完整视觉 encoder；
- LoRA 的验证集收益已饱和但全参仍提升。

此时可提高 rank、覆盖 FFN、使用 DoRA/AdaLoRA，或考虑部分/全参微调。

### 面试总结

LoRA 有效是因为下游适配通常只需在预训练能力上沿少数方向调整，低秩增量能以更少参数表达这些变化并提供正则化。但它是经验假设，任务偏移过大时仍需扩大容量或全参训练。
