# 要求转go语言

## 答案

若面试官要求把已有算法改写为 Go，先保持算法不变，再处理语言差异，不要边翻译边修改思路。

### 常见映射

- Python `list` 对应 Go slice，注意长度与容量；
- `dict/set` 对应 `map[K]V`，set 常用 `map[T]struct{}`；
- 队列可用 slice 加头指针，避免频繁删除首元素；
- 优先队列实现 `container/heap` 接口；
- 排序使用 `sort.Slice` 或 `slices.Sort`；
- 字符串按字节还是 Unicode 字符处理要先确认，中文应遍历 `[]rune`；
- 整数溢出风险高时使用 `int64`，并明确平台 `int` 宽度；
- Go 没有 Python 式负索引、列表推导和隐式真值判断。

### 面试表达

> 我先复述输入、输出和边界条件，保留原算法复杂度。实现时会确定数据结构和整数范围，再写核心逻辑，最后用空输入、单元素、重复值和最大规模样例检查。若需要并发，我只在任务可独立且收益大于调度成本时使用 goroutine，不会为了展示语法强行并发。

跨境支付服务中还应注意 `context.Context` 超时取消、错误包装、幂等键、金额使用最小货币单位或十进制定点类型，不能直接用 `float64` 表示资金。

### 面试总结

转 Go 的重点是保持算法正确和复杂度不变，同时处理 slice、map、heap、Unicode、整数范围与错误返回。业务代码还要体现超时、幂等和金额精度意识。
