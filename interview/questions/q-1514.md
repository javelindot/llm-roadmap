# 模型量化如何做的。 gptq , qat 等等，并说明为什么选择了w8a16的量化？

## 答案

模型量化把权重或激活从 FP32/BF16/FP16 映射到 INT8、INT4、FP8 等低精度表示，以降低显存、带宽和推理成本。选择 W8A16 表示权重约为 8 bit，激活仍保持 FP16/BF16。

### 1. 基本量化

线性量化可写为：

```text
q = clamp(round(x / scale) + zero_point)
x_hat = scale * (q - zero_point)
```

需要选择：

- 对称或非对称；
- per-tensor、per-channel 或 per-group；
- 静态或动态 scale；
- 权重位宽和激活位宽；
- 校准数据与异常值处理。

### 2. PTQ、GPTQ 与 QAT

**PTQ**

训练后直接量化，成本低、上线快。普通 round-to-nearest 简单，但低比特容易掉点。

**GPTQ**

训练后 weight-only 量化，按层/块利用二阶信息近似补偿量化误差，常用于 W4A16/W8A16。优点是不必重新训练；缺点是需要代表性校准数据，速度依赖专用 kernel。

**AWQ**

根据激活统计保护少量重要权重通道，属于 activation-aware 的 weight-only PTQ。

**SmoothQuant**

把激活异常值的一部分尺度迁移到权重，更适合 W8A8。

**QAT**

训练中插入 fake quantization，让模型适应量化误差。精度潜力高，但需要数据、算力和完整训练链路，对大模型成本更高。

### 3. 为什么选 W8A16

- 权重显存约比 FP16 减半；
- 降低权重读取带宽，尤其有利于 memory-bound Decode；
- 激活保留 FP16/BF16，避免 activation outlier 带来的 W8A8 精度风险；
- 校准和部署比 QAT 简单；
- 通常比 W4A16 精度更稳，适合业务先追求低风险压缩；
- INT8 weight-only kernel 和生态相对成熟。

但 W8A16 不保证一定加速。小 batch、硬件无高效 kernel、反量化开销大或 Prefill 计算占主导时，速度收益可能有限。

### 4. 如何做项目验证

1. 选择覆盖语言、长度和业务长尾的校准集；
2. 分别测试 FP16、W8A16、W4A16 和可行的 W8A8；
3. 比较任务准确率、perplexity、长文本、结构化输出和安全集；
4. 测显存、TTFT、TPOT、吞吐、功耗和单位成本；
5. 对敏感层保留高精度或调整 group size；
6. 按真实硬件和推理框架做端到端压测。

### 5. 跨境支付场景

支付客服或合规模型不能只看平均准确率，还应单独验证金额、币种、国家、否定词、多语言和工具参数。量化导致的一个数字错误可能比普通文本退化严重，因此高风险输出必须经过 schema 和权威系统校验。

### 6. 面试总结

W8A16 是精度与显存之间偏稳健的 weight-only 方案：权重降到 8 bit，激活保留 16 bit。是否选择它必须由模型质量、目标硬件 kernel、端到端时延和业务风险共同决定。
