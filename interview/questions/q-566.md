# 为什么要用reference model? 为了解决什么问题？

## 答案

**一句话**：reference model 用来给当前策略一个参照，限制微调或强化学习后的模型不要偏离原始模型太远。

### 作用

在 RLHF/PPO 或 DPO 中，reference model 通常是冻结的 SFT 模型。它的作用是：

- 提供基准分布。
- 计算 KL 约束。
- 防止模型过度优化奖励。
- 减少 reward hacking。
- 保留原模型通用能力。

### 结合跨境支付项目怎么说（面试口径）

> 在跨境支付客服对齐里，如果我们用偏好数据优化模型，希望它更合规、更准确。但不能让模型为了追求奖励，开始过度拒答或过度承诺。reference model 可以约束新模型不要偏离原来的语言能力太远。
>
> 比如模型不能为了用户满意就说“资金一定今天到账”，也不能因为安全奖励过强就什么都拒答。reference model 和 KL 约束能帮助保持平衡。

**钩子**：可以引到“KL 约束、reward hacking、保留通用能力、支付合规表达”。
