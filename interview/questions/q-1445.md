# 问小米那边的工作，encoder的预训练，怎样做拼接

## 答案

题干没有说明小米项目的具体模型和“拼接”对象，不能假定内部实现。面试时应先澄清：是句对输入拼接、多个样本 packing、多个 Encoder 表征拼接，还是多模态特征拼接。下面给出常见 Encoder 预训练口径。

### 1. 句对输入拼接

BERT 类 Encoder 常使用：

```text
[CLS] sentence_A [SEP] sentence_B [SEP]
```

输入表示为：

```text
token embedding
+ position embedding
+ token type/segment embedding
```

`attention_mask` 屏蔽 padding，`token_type_ids` 区分 A/B。可用于句对匹配、NLI、检索重排等任务。

### 2. 单塔检索拼接

Cross-Encoder 将 query 和 document 拼成一个序列：

```text
[CLS] query [SEP] document [SEP]
```

两者可做全局注意力，相关性准确但计算贵，适合 Rerank。

双塔一般不在 token 级拼接：

```text
q = Encoder_q(query)
d = Encoder_d(document)
score = sim(q, d)
```

它适合大规模向量召回。

### 3. 多字段拼接

商品、文档或支付知识可按字段组织：

```text
[TITLE] 标题
[CATEGORY] 类别
[BODY] 正文
[COUNTRY] 国家
[PRODUCT] 产品
```

可以直接文本拼接，也可以分别编码后做 concat、attention/gating fusion。直接拼接简单，但长字段可能挤掉关键信息；分支编码更灵活，但模型和部署更复杂。

### 4. 多样本 Packing

为了提高训练吞吐，可以把多个短样本放入一个长序列，但要防止样本间互相注意：

- 使用 block-diagonal attention mask；
- 保留每个样本边界；
- loss 只在对应 token 上计算；
- 不让正负样本因为 packing 泄漏标签。

Encoder MLM 预训练也可拼接同一文档的连续片段，但不应随意把无关文档拼成一个语义上下文。

### 5. 预训练目标

常见目标：

- MLM；
- 对比学习；
- query-document matching；
- pairwise/listwise ranking；
- 多视图一致性；
- hard negative discrimination。

拼接方式必须服务训练目标。例如 Reranker 需要 query-doc 交互，Embedding 召回则更强调可独立编码。

### 6. 跨境支付场景

支付知识 Rerank 可拼接：

```text
用户问题 + 国家/币种/产品 + 候选文档
```

但商户权限、规则版本和实时订单状态应先结构化过滤，不能只作为普通文本期待模型自己识别。

### 7. 面试总结

回答时先明确拼接层级。句对任务常用 `[CLS] A [SEP] B [SEP]`，双塔召回分别编码，Cross-Encoder 重排联合编码，多字段可文本拼接或特征融合，训练 packing 则必须隔离 attention 和 loss。
