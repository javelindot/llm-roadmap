# RAG有哪些分类;多模态RAG有哪些实现框架;伪多模态RAG和多模态RAG分别怎么实现,有什么区别;CLIP可以用于哪一类多模态RAG

## 答案

RAG 可以按数据模态、检索表示、流程复杂度和知识结构分类。多模态 RAG 的关键区别是：检索和生成阶段是否真正保留并使用原始视觉/音频信息。

### 1. RAG 分类

按流程：

- Naive RAG：单次切分、召回、拼接、生成；
- Advanced RAG：query rewrite、hybrid、rerank、压缩与校验；
- Modular/Agentic RAG：动态选择数据源、多跳检索、反思和工具；
- Self-corrective RAG：评价召回与答案，必要时重检索。

按知识与检索：

- 文本 sparse/dense/hybrid；
- 结构化 SQL/API；
- Knowledge Graph/GraphRAG；
- Late Interaction；
- 多模态图像、音频、视频与文档页面。

### 2. 伪多模态 RAG

先将非文本内容转成文本：

```text
图片/页面
-> OCR、caption、表格转 Markdown
-> 文本 chunk
-> 文本 embedding/BM25
-> 文本 LLM
```

优点：

- 可复用成熟文本 RAG；
- 成本低、易检索和引用；
- 对 OCR 清晰文档很实用。

缺点：

- 视觉布局、颜色、空间关系和细节有损；
- caption 可能幻觉；
- 生成模型看不到原图，无法复核；
- 表格/图表转文本可能破坏结构。

“伪”不代表无价值，很多企业文档场景中 OCR + 结构化解析是最可靠基线。

### 3. 真多模态 RAG

检索或生成至少一端直接使用原始模态：

```text
文本/图片 query
-> 跨模态 embedding 或视觉 late interaction
-> 检索图片/页面/视频片段
-> 原始媒体 + 文本证据
-> VLM/Omni 模型生成
```

实现包括：

- CLIP/SigLIP 双塔跨模态向量；
- ColPali/ColQwen 类文档页面 late interaction；
- 文本与图片分别索引后融合；
- 视频按镜头/时间段索引；
- VLM 对召回原图做 rerank 或回答。

它保留视觉证据，但图像 token、存储、rerank 和 VLM 推理成本更高。

### 4. CLIP 属于哪一类

CLIP 把图像和文本映射到共享向量空间，可实现：

- 文本查询图片；
- 图片查询文本/图片；
- 多模态向量召回。

因此它属于真多模态 RAG 的跨模态检索组件。若只用 VLM 生成 caption 后做文本检索，则是伪多模态，不是 CLIP 路线。

CLIP 擅长全局语义，不擅长细粒度 OCR、数字、版面和小目标，文档场景常需 OCR、layout model、ColPali 或 VLM 补充。

### 5. 实现框架

- **LlamaIndex**：MultiModal LLM、图文索引和 RAG 数据组件；
- **LangChain/LangGraph**：编排多模态模型、Retriever 和工作流；
- **Haystack**：组件化检索与多模态 pipeline；
- **向量库**：Milvus、Qdrant、Weaviate 等存储不同模态向量；
- **模型组件**：CLIP/SigLIP、ColPali、OCR、Qwen-VL 等。

框架负责集成，检索质量仍取决于解析、表示、数据和评测。

### 6. 评测

- 文本到图像/页面 Recall@K、NDCG；
- OCR、图表和视觉 grounding；
- 原图证据支持率；
- 多模态幻觉与无答案；
- 图像 token、存储、P95 和成本；
- 伪多模态与真多模态的消融。

### 7. 跨境支付场景

票据和证件可先用 OCR/版面解析建立伪多模态基线，再对印章、照片、复杂表格和截图加入原图检索与 VLM。金额、证件号和身份字段必须由结构化校验与人工复核。

### 8. 面试总结

伪多模态把媒体转文本后走文本 RAG，便宜但有损；真多模态直接检索或生成原始媒体，信息完整但成本高。CLIP 是真多模态中的跨模态双塔检索器。
