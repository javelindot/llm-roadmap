# 预训练一般分为哪些阶段,每个阶段的特点有何区别?

## 答案

现代大模型的“预训练”通常不是一次性跑完，而是由数据与模型准备、通用预训练、退火阶段和可选的继续预训练组成。SFT、偏好优化和安全对齐属于 Post-training，不应混入预训练概念。

### 1. 数据、Tokenizer 与小规模验证

正式训练前需要：

- 确定语言、代码、数学、领域和多模态数据配比；
- 清洗、去重、质量过滤、版权与隐私治理；
- 做 benchmark contamination 检测；
- 训练和评估 tokenizer；
- 用小模型或短程 run 验证架构、loss、并行和数据管线；
- 通过 scaling experiment 选择参数量、token 数和算力配比。

Tokenizer 会影响序列长度、语言效率、代码缩进和数字表示，训练后很难低成本替换。

### 2. 通用预训练

使用大规模多来源语料训练 next-token prediction：

```text
L = -Σ_t log p(x_t | x_<t)
```

这一阶段：

- token 规模最大；
- 学习率经过 warmup 后处于主训练区间；
- 数据覆盖广，强调语言、知识、代码和基础推理；
- 主要监控训练 loss、梯度、吞吐、数据分桶和中间 benchmark；
- 定期保存完整 checkpoint 和数据游标，支持故障恢复。

核心目标是建立通用表示和生成能力，不追求对话格式。

### 3. 退火与高质量阶段

训练后段通常逐步降低学习率，并提高高质量、数学、代码、推理等数据的比例：

- 在较小学习率下稳定收敛；
- 强化高价值能力；
- 避免最后阶段被低质量网页噪声主导；
- 使用留出的高质量数据，防止重复过拟合。

“退火”主要描述学习率和数据配比在末期的变化，不等同于指令微调。具体阶段名称在不同团队中可能叫 annealing、cooldown 或 high-quality phase。

### 4. 继续预训练

基础模型之后可进行 Continued Pretraining：

- **领域继续预训练**：金融、法律、医学、代码等无标注领域语料；
- **语言适配**：提高低资源语言或目标市场语言能力；
- **长上下文继续预训练**：扩大 context length，并调整位置编码与长文数据；
- **知识刷新**：加入新时间窗口数据，但需处理遗忘和时效治理；
- **多模态适配**：连接视觉、音频编码器并进行跨模态训练。

继续预训练仍然多使用自监督目标。学习率通常低于从零预训练，并混入通用数据防止 catastrophic forgetting。

### 5. Post-training

预训练之后才进入：

1. SFT：学习指令、对话和工具格式；
2. Preference Optimization：DPO、PPO、GRPO 等；
3. Safety Tuning：拒答、红队和政策边界；
4. Distillation/Deployment Tuning：蒸馏、量化感知或特定服务优化。

Post-training 改变模型“如何使用能力”，但无法可靠补回预训练中完全缺失的大规模语言或知识能力。

### 6. 跨境支付中的选择

多数支付团队不需要从零预训练模型。更合理的是：

- 选择合规可部署的基础模型；
- 用脱敏的规则、术语和多语言文档做小规模领域继续预训练；
- 频繁变化的费率、国家规则和通道状态放在 RAG 或工具中；
- 用 SFT 学习客服格式和工具调用；
- 用偏好数据优化证据完整、拒答和转人工行为。

商户交易、银行卡和 KYC/KYB 材料必须做权限、最小化和脱敏，不能因“预训练数据越多越好”就无边界使用。

### 7. 各阶段评测

- 通用预训练：perplexity、知识、语言、数学、代码；
- 继续预训练：领域集提升与通用能力遗忘；
- 长上下文：不同位置和长度的检索、推理与延迟；
- Post-training：指令遵循、工具、faithfulness、安全和格式；
- 全流程：数据污染、训练稳定性、成本和可复现性。
