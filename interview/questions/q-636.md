# 多个agent如何协作工作？

## 答案

多 Agent 协作的本质是任务分解、角色分工、共享状态和冲突解决。Agent 数量越多不一定越好，只有角色确实需要不同工具、上下文或权限时才值得拆分。

### 常见协作模式

- Supervisor：一个主管 Agent 分解任务、派发子任务并汇总结果，流程清晰，适合大多数业务。
- Planner-Executor：规划者生成步骤，执行者逐步调用工具，审查者检查结果并触发修正。
- Pipeline：研究、抽取、分析、写作等 Agent 按固定顺序传递结果，稳定且容易观测。
- Debate/Reflection：多个 Agent 独立给出方案，再由裁判选择或综合，适合高价值决策，但成本较高。
- Blackboard：Agent 围绕共享任务板发布发现和领取工作，适合动态并行任务。

### 协作协议

每个任务应有唯一 ID、目标、输入 schema、期望输出、依赖、截止时间和预算。消息使用结构化格式，不依赖模糊自然语言。共享状态放在数据库、事件日志或工作流引擎中，并记录版本，避免各 Agent 在私有上下文中形成不一致事实。

调度器根据依赖关系并行执行可独立任务，对同一资源设置锁或乐观并发控制。最终结果由明确的聚合节点处理，冲突时按证据质量、置信度、角色权限或人工审核解决。

### 生产保障

- 每个 Agent 设置最大循环、超时、重试和 token/费用配额。
- 工具权限遵循最小授权，高风险动作必须确认。
- 步骤保持幂等，失败后可从 checkpoint 恢复。
- 使用分布式 trace 记录消息、工具调用、模型版本和状态变化。
- 对单 Agent、协作流程和最终任务分别评测，确认拆分确实提升成功率。

如果任务可以由一个带工具的状态机稳定完成，就不应为了“多 Agent”增加通信、延迟和错误传播。协作架构应由任务边界驱动。
