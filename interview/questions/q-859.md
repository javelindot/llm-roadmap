# 为什么moe结构的RM，能力点没有标签

## 答案

如果 MoE Reward Model 只有总体 chosen/rejected 偏好，而没有“正确性、安全、风格”等能力标签，专家仍可能在训练中形成隐式分工，但不能保证每个专家对应某个可解释能力点。

### 为什么没有显式标签也能训练

对偏好对使用：

\[
\mathcal L
=-\log\sigma(r(x,y_w)-r(x,y_l))
\]

梯度会从标量 reward 经过被 Router 选中的专家传播。不同输入模式、领域和语言产生不同梯度，Router 和专家可能自发专业化。MoE 的目标是扩大条件计算容量，不要求预先给专家命名。

### 存在的问题

- 专家可能按语言、长度、格式或数据来源分工，而非真正能力；
- 标量偏好把多个目标混在一起，无法知道得分高是因为安全还是风格；
- 负载均衡 loss 只保证使用率，不保证语义分工；
- 稀疏路由会让长尾能力专家数据不足；
- Router 可能学到标注偏差和捷径。

### 如果需要能力可解释性

- 为正确性、安全、帮助性等增加多头 reward 与显式标签；
- 使用能力标签监督 router，或将专家分组并做软路由；
- 加入 process reward、规则/验证器结果；
- 统计每个专家在能力、语言、长度切片上的激活；
- 做专家屏蔽、替换和 counterfactual routing 消融；
- 仍保留无标签共享专家，避免过度僵化。

跨境支付 RM 中，事实正确、合规、隐私和语气不能只压成一个分数。金额与规则应由验证器给硬信号，主观表达再用偏好模型。

### 面试总结

MoE RM 不需要能力标签也能通过总体偏好训练，专家专业化可由路由和数据分布自发产生；但这种分工未必可解释。若业务需要能力级控制，应使用多目标标签、验证器和路由分析。
