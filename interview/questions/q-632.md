# 其他PEFT方法

## 答案

PEFT 的目标是在冻结大部分预训练参数的情况下，用少量可训练参数适配新任务。除 LoRA 外，常见方法可按参数放置位置分类。

### Prompt 类方法

- Prompt Tuning：在输入 embedding 前拼接可学习的 soft prompt，只训练这些虚拟 token。
- Prefix Tuning：为每层注意力注入可学习的 K/V prefix，比只改输入层的 prompt 表达能力更强。
- P-Tuning v2：将深层 prompt 扩展到多层，并配合重参数化，适合不同规模模型和理解任务。

这类方法参数极少，便于多任务切换，但小模型或复杂生成任务上可能不如 LoRA 稳定。

### Adapter 类方法

在 Transformer 层中插入小型瓶颈网络，例如先降维、非线性变换再升维，并通过残差连接回主干。Houlsby Adapter、Pfeiffer Adapter、AdapterFusion 等属于此类。它们模块化强，但推理时会增加额外串行算子和延迟。

IA3 不增加完整瓶颈网络，而是学习少量缩放向量，对注意力 K/V 和 FFN 激活进行逐通道缩放，参数比 LoRA 更少。

### 参数选择与稀疏更新

- BitFit：只训练 bias，成本很低，但容量有限。
- LayerNorm Tuning：只更新归一化中的缩放或偏置。
- Selective Fine-tuning：只训练部分层、注意力头或基于重要性选出的参数。
- Diff Pruning：学习一个稀疏的权重增量，并对增量加稀疏约束。

### LoRA 相关扩展

AdaLoRA 动态给重要矩阵分配 rank；DoRA 将权重的方向和幅值分开适配；QLoRA 将冻结的基础模型量化到低位宽并训练 LoRA；LoHa/LoKr 使用 Hadamard 或 Kronecker 结构提高参数表达效率。

选型时应比较质量、训练显存、推理延迟、可合并性和多租户切换成本。Prompt 类适合极低参数预算，Adapter 适合模块化组合，LoRA 因效果稳定且可合并到权重中，通常是通用场景的首选基线。
