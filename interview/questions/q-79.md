# 长思维链冷启动

## 答案

**一句话**：长 CoT 冷启动 = **用少量高质量长 CoT 数据 SFT，让 base 模型先学会"分步思考"的格式和节奏**，再用 RL 把推理深度训上去——是 DeepSeek-R1 流程的关键第一步。

### 1. 为什么需要冷启动
- base 模型不会"主动写长 CoT"，prompt 让它写也只是浅推理
- 直接 RL（R1-Zero）能学会推理但格式 ugly，可读性差
- 冷启动给模型一个"长 CoT 格式起点"，后续 RL 更稳

### 2. 冷启动数据来源
| 来源 | 优点 | 缺点 |
|------|------|------|
| **R1 蒸馏** | 长 CoT 风格、格式清楚 | 风格固化 |
| **GPT-4o / Claude 蒸馏** | 质量高 | 贵 |
| **人工标注** | 业务可控 | 量小 |
| **MathPile / OpenR1** | 公开、量大 | 通用偏 |
| **拒绝采样** | 自有模型产 | 需要不弱的 base |

### 3. 数据特征（重点！）
- **长**：通常 2k-8k token（远超普通 SFT）
- **结构化**：用 `<think>...</think><answer>...</answer>` 显式分段
- **多样**：覆盖数学、代码、逻辑、规划等
- **有反思**：CoT 里包含 "wait", "let me check", "actually..." 等 self-correction 标记
- **可验证**：尽量是有标准答案的任务

### 4. 训练设置
- 数据量：**1k - 10k** 高质量样本（不需要太多）
- epoch：**2-3**（多了过拟合）
- lr：比常规 SFT 略小（1e-5）
- 上下文长度：≥ 8k（CoT 容易超长）
- Loss：可以全 CoT 算，也可以只对 answer 算，效果差别有限

### 5. DeepSeek-R1 的完整流程（参考）
```
DeepSeek-V3 base
   → 冷启动 SFT（少量高质量长 CoT）
   → RL（GRPO + 可验证奖励）
   → 拒绝采样 + SFT（稳格式）
   → 通用对齐 RLHF
   → DeepSeek-R1
```

### 6. 冷启动的关键作用
- **格式注入**：模型学会"看到题就写 think 再写 answer"
- **降低 RL 起点方差**：RL 不需要从 base 那么散的分布开始
- **可读性**：避免 R1-Zero 那种乱码式 CoT
- **业务适配**：可以在冷启动阶段注入业务规则风格

### 7. 常见坑
- 数据太多反而塌陷，1-10k 是甜点
- CoT 太长（>10k）训练显存爆
- 没有反思标记的 CoT，RL 学不出 self-correction
- 答案直接夹在 think 里，loss masking 失效

### 8. 一句话总结
**长 CoT 冷启动 = 用少量精品 SFT 教模型"怎么思考"的格式**——是 R1-style 推理模型训练的钥匙起点，少即是多。

## 结合项目谈（口语版 · 约 40 秒）

> 我们跨境支付里**风控调查 Agent** 走过完整的长 CoT 冷启动流程，分享下细节。
>
> 业务背景：风控调查不是简单分类，而是要看一段交易+商户+用户信息，推断"这是不是洗钱、为什么这么判、证据链是什么"。通用 LLM 推理浅，2-3 步就给结论，证据不全。我们想训一个长推理风控模型。
>
> **冷启动数据**怎么准备的：找 4 个资深风控专家，每人标 200 条调查链路，每条是完整的 `<think>...</think><answer>...</answer>`——think 部分包含"看到这个商户类型 → 联想到 X 行业洗钱模式 → 检查交易金额对不对得上 → 检查关联账户 → 综合判断"。8w 条原始历史工单里挖出 800 条带完整推理的，扩到 5k 条做冷启动 SFT。
>
> **训练设置**：Qwen-7B base、SFT 2 epoch、lr=1e-5、上下文 16k、loss 全 CoT 算。
>
> **关键观察**——冷启动训完，模型已经会"写长 think 再给 answer"了，但推理深度还不够。后面才接 RL（GRPO + 历史结案金标），把推理深度训上去。
>
> 上线后**证据召回率提升 40%**，比直接 prompt 通用 LLM 高一大截。
>
> 我对这道题的总结：**冷启动数据"少而精"比"多而杂"重要**——5k 条专家级 CoT 比 5w 条蒸馏 CoT 效果好得多。

**🪝 顺嘴带出去的钩子**：
- "怎么挖 800 条带完整推理的历史工单可以聊。"
- "GRPO 的奖励设计我们有具体配置。"
- "self-correction 在风控推理里能不能提升准确率有实验数据。"
