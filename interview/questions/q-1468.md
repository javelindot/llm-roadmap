# 视频生成模型的SFT微调经验是什么？

## 答案

这是经验核验题，应按真实模型、数据和硬件回答。视频生成微调通常不是语言模型式的纯文本 SFT，而是在预训练 Video Diffusion/DiT/Flow 模型上，用文本-视频或图像-视频数据继续优化去噪/流匹配目标，并可叠加条件和时序损失。

### 1. 数据准备

每条样本至少包含：

```text
video
caption/prompt
fps
duration
resolution/aspect ratio
optional first frame/reference image/control signal
```

清洗重点：

- 去重和近重复；
- 清晰度、运动量、镜头切换；
- 水印和字幕；
- NSFW、版权和人物授权；
- caption 与视频一致性；
- 帧率和时长；
- 极端静态或抖动样本；
- identity 泄漏；
- train/test 按素材源或人物切分。

### 2. Caption

Caption 应描述：

- 主体；
- 动作；
- 场景；
- 镜头运动；
- 光照和风格；
- 时间顺序。

只写静态物体名会让模型学不到动作。可以用多模态教师生成，再由规则和人工抽检，避免幻觉描述。

### 3. 训练结构

典型流程：

```text
video frames
-> VAE encode latent
-> add noise / sample flow timestep
-> text encoder condition
-> Video DiT/Transformer predict noise, velocity or flow
-> loss
```

资源有限时：

- 冻结 VAE 和 text encoder；
- 对 DiT/Attention 加 LoRA；
- 只训练 temporal block 或 motion module；
- gradient checkpointing；
- bf16；
- sequence/context parallel。

### 4. Loss

主损失取决于基础模型：

```text
noise prediction MSE
velocity prediction
flow matching loss
```

可选辅助约束：

- text-video alignment；
- temporal consistency；
- identity/reference consistency；
- optical-flow consistency；
- perceptual loss；
- first-frame reconstruction；
- motion strength regularization。

辅助 loss 不能与基座训练目标不兼容，先复现原始 loss 再做增量。

### 5. Resolution Bucket

视频长度、分辨率和比例差异大，应按：

```text
frames x height x width
```

分桶，控制每 batch 的 token 预算。随机裁剪要避免切掉主体；帧采样需保持动作连续。

### 6. 常见问题

- 闪烁：加强时序数据和 temporal 模块；
- 动作弱：提高动态样本和动作 caption；
- 身份漂移：reference/identity 条件和一致性评测；
- 过拟合素材：更强去重、低学习率、正则和 replay；
- 显存高：latent、LoRA、checkpoint、短帧训练；
- prompt 不跟随：改善 caption 和条件 dropout。

### 7. 评测

- 文本一致性；
- 视频质量；
- temporal consistency/flicker；
- motion quality；
- identity consistency；
- 人工偏好；
- 不同人物、动作、镜头和分辨率切片；
- 推理速度和显存；
- 安全和水印。

FVD 等单一指标不能完整代表真实体验，应结合人工和任务评测。

### 8. 安全

只使用授权人物和素材，输出添加来源标记/水印，限制冒充、欺诈和未授权换脸。支付凭证、证件等不得通过生成模型伪造。

### 9. 面试总结

视频生成 SFT 的关键是高质量动作 caption、时空分桶、匹配基座的 diffusion/flow loss、LoRA/冻结策略和时序评测。没有真实经验时应明确说完成到原型还是线上，不编造显卡和指标。
