# Agent由哪些部分组成

## 答案

**一句话**：Agent 一般由大模型、角色提示、规划能力、记忆、工具、执行循环、状态管理和安全控制组成。

### 1. 核心组成
- **LLM**：负责理解、推理、生成计划和输出。
- **Prompt / Role**：定义 Agent 的角色、目标、约束和输出格式。
- **Planning**：把复杂任务拆成步骤，决定先做什么后做什么。
- **Tools**：连接外部系统，比如数据库、搜索、API、代码执行、工单系统。
- **Memory**：保存短期上下文和长期经验。
- **Action Loop**：执行动作、观察结果、继续决策。
- **State**：记录任务进度、执行结果、失败原因和下一步。
- **Guardrails**：权限、审批、敏感信息过滤、审计、停止条件。

### 2. 简单流程

一个 Agent 的运行可以简化成：

1. 接收目标。
2. 理解上下文和身份。
3. 制定计划。
4. 选择工具执行。
5. 观察工具返回。
6. 更新状态和记忆。
7. 继续执行或给出最终结果。

### 3. 面试总结

Agent 不是"大模型 + prompt"这么简单。真正能落地的 Agent，必须能和外部系统交互，有状态、有记忆、有工具、有安全边界。

## 结合项目谈（口语版 · 面试现场约 50 秒）

> 如果结合跨境支付，我会把 Agent 拆成几块来看。第一是模型和 prompt，告诉它现在是风控助手、对账助手还是商户审核助手；第二是工具，比如查交易流水、查商户资料、查通道回执、查风控规则；第三是记忆，比如这个商户历史上有没有拒付、有没有高风险国家交易；第四是状态管理，比如这笔异常单现在是待排查、待补材料、待人工还是已关闭。
>
> 比如做一笔异常付款排查，Agent 不是直接回答一句"可能失败了"，而是先查订单、再查通道回执、再看账务流水，最后判断是通道延迟、金额不一致还是汇率差异。涉及退款或冻结这类动作时，它只能生成建议和证据，不能直接执行。
>
> 所以我理解 Agent 的组成，重点不是堆模型能力，而是把模型接进业务系统之后，规划、工具、记忆、状态和权限这几块能不能闭环。
