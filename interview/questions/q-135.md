# RAG 有哪些关键评估指标和能力？

## 答案

**一句话**：RAG 评估分**召回质量、生成质量、端到端正确性、运维能力** 四类——主流框架 RAGAS 给出五大指标：**Context Relevance / Faithfulness / Answer Relevance / Context Precision / Context Recall**。

### 1. 召回阶段指标
| 指标 | 含义 |
|------|------|
| **Recall@K** | top-K 里有多少正确文档 |
| **Precision@K** | top-K 里有多少是正确的 |
| **MRR** | 第一个正确文档的倒数排名 |
| **NDCG@K** | 考虑排序的相关性 |
| **Context Relevance** | 召回内容和 query 相关度（RAGAS） |
| **Context Recall** | 是否召回了所有相关信息（RAGAS） |
| **Context Precision** | 召回里有多少是真用上的（RAGAS） |

### 2. 生成阶段指标
| 指标 | 含义 |
|------|------|
| **Faithfulness** | 答案是否忠于检索到的内容（无幻觉） |
| **Answer Relevance** | 答案是否真的回答了 query |
| **Hallucination Rate** | 幻觉率 |
| **Citation Accuracy** | 引用的正确性 |

### 3. 端到端指标
- **正确率（Accuracy）**：黄金集人工评定
- **完整性（Completeness）**：答案是否覆盖关键点
- **可用性（Usefulness）**：业务侧能否直接使用
- **用户满意度 / 点击率**：线上行为信号

### 4. 运维能力指标
- **延迟**：p50、p95、p99
- **成本**：单 query 平均 token
- **可用性**：成功率
- **可观测性**：trace、日志、回放
- **可解释性**：citation 完整度

### 5. RAGAS 框架（最主流）
五大指标：
1. Faithfulness（生成 vs 召回的一致性）
2. Answer Relevance（答案 vs query 相关性）
3. Context Relevance（召回 vs query 相关性）
4. Context Precision（召回的"信号:噪声"）
5. Context Recall（召回完整性）

**实现**：用 LLM-as-judge 给每个维度打分

### 6. 评测数据集构造
- **黄金集**：100-1000 条人工标 (query, 正确答案, 相关文档)
- **合成集**：让 LLM 从文档反向生成 query
- **线上对话**：定期 sample 真实流量人标
- **失败 case**：用户投诉、人工反馈

### 7. 评测流水线（生产标配）
```
代码改动 → 离线评测（RAGAS + 黄金集）
  → 灰度上线 → 线上指标监控
  → 失败 case 回流 → 数据更新 → 重训
```

### 8. 进阶能力评估
- 多语言能力
- 长上下文召回
- 多跳推理
- 拒绝回答（不该答的不能答）
- 时效性（知识更新延迟）
- 安全性（jailbreak 抵抗）

### 9. 一句话总结
**RAG 评估必须分两段**：召回看 Recall@K + Context Precision/Recall、生成看 Faithfulness + Answer Relevance；端到端用黄金集人工评 + 线上指标监控。

## 结合项目谈（口语版 · 约 30 秒）

> 我们跨境支付 RAG 评估体系**线上线下两套并行**，分享下我们的实践。
>
> **离线评测**：
> - 黄金集 500 条人工标 (query, 正确答案, 相关 chunk_id)，覆盖 5 大类场景
> - 关键指标：**Recall@5、Context Precision、Faithfulness、Answer Relevance**
> - 跑 RAGAS，每周一次基线对比
> - 任何代码改动必须先过离线评测，不掉点才能合并
>
> **线上监控**：
> - 客服 RAG：用户点击率、转人工率、点踩率
> - 风控 RAG：人工复核通过率、误报率、漏报率
> - 实时 trace：每条 query 留 (检索结果, 生成内容, 用户行为)
>
> **失败 case 回流**：
> - 客服每天 sample 100 条人工标
> - 失败 case 定位是召回失败还是生成失败
> - 召回失败 → 补知识库 / 重训 embedding
> - 生成失败 → 补 prompt / DPO 增强
>
> **几个工业经验**：
> - RAGAS 用 GPT-4 当 judge 太贵，我们用 Qwen-72B 当 judge，一致性 92%；
> - Faithfulness 是金融场景最重要指标——宁可"我不知道"也不能"我编"；
> - Context Precision 低意味着噪声多，要回头优化分块和 rerank；
> - 线上指标比离线指标 lag 一天，但更真实，最终听线上。
>
> 一句话：**没有评估的 RAG 等于盲飞**——离线 RAGAS 保稳定、线上指标定方向、失败 case 回流飞轮。

- "金融场景 Faithfulness 怎么硬性约束可以展开。"
- "失败 case 定位是召回还是生成的具体方法。"
