# 🔥 attention的计算时间复杂度

## 答案

设序列长度为 `n`，隐藏维度为 `d`。标准 Self-Attention 的主要计算包括：

### 1. Q/K/V 投影

```text
Q = XW_Q, K = XW_K, V = XW_V
```

三个矩阵乘法的时间复杂度约为 `O(n d²)`。

### 2. 计算注意力分数

```text
QK^T
```

得到 `n × n` 的注意力矩阵，复杂度为 `O(n² d)`。

### 3. 对 V 加权

```text
softmax(QK^T) V
```

复杂度也是 `O(n² d)`。

因此完整 Attention 层常写为：

```text
时间复杂度：O(n d² + n² d)
```

如果只强调序列长度的影响，通常简称 `O(n²)`。

### 4. 空间复杂度

- 注意力矩阵需要 `O(n²)` 空间。
- Q/K/V 和输出需要 `O(n d)`。

所以长上下文时，`n²` 的注意力矩阵常成为主要瓶颈。FlashAttention 不改变理论计算复杂度，但通过分块和减少 HBM 读写显著降低实际显存与时间。

### 5. Multi-Head 的影响

若总隐藏维度 `d` 不变，拆成多个头不会改变总体渐近复杂度；每个头的维度约为 `d/h`，所有头的计算量加总仍是同一量级。

### 6. Cross-Attention

如果 Query 长度为 `n`，Key/Value 长度为 `m`，注意力部分复杂度约为：

```text
O(n m d)
```

### 7. 自回归 Decode

有 KV Cache 时，每一步只有一个新 Query 与 `n` 个历史 K/V 计算，单步注意力约为 `O(n d)`，但缓存空间随上下文线性增长。

**面试总结**：标准 Attention 的核心瓶颈是序列维度上的平方项，而完整 Transformer 层还要同时考虑 `O(n d²)` 的投影和 FFN 计算。
