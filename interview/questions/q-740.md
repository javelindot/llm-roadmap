# 双塔可以用于哪些环节？你觉得目前对双塔模型的研究还存在哪些问题？怎么优化？

## 答案

双塔模型用两个 encoder 分别编码 query/user 和 document/item，在离线预计算候选向量后用点积或 cosine ANN 检索，核心优势是大规模低延迟匹配。

### 可用环节

- 搜索与 RAG 的语义召回；
- 推荐系统 user-item 召回；
- 广告候选匹配；
- 图文、音文等跨模态检索；
- 去重、相似推荐和 hard negative mining；
- 粗排或作为精排特征。

它不适合直接替代高精度 cross-encoder 精排，因为两侧编码时没有 token/item 级交互。

### 当前问题

- 表达瓶颈：所有匹配信息压缩到单个向量，复杂约束和细粒度交互丢失。
- 负样本：随机负例太简单，batch 内存在 false negative，热门 item 被过度采样。
- 训练服务偏差：训练负例、ANN 索引和线上候选分布不同。
- 各向异性与 hubness：少数向量成为很多 query 的近邻。
- 多兴趣压缩：一个用户向量难表示相互独立的兴趣。
- 新鲜度、库存和权限等动态信息不适合仅编码进静态向量。

### 优化方向

使用 ANN 挖掘 hard negative，并对 false negative 去噪；采用多向量/late interaction、multi-interest tower 或 ColBERT 式 token 表示增强交互；用 cross-encoder teacher 蒸馏；引入对比学习、温度和去偏采样；联合稀疏 BM25 与 dense 召回。

索引侧做量化、分片和增量更新，动态业务特征在过滤或后续排序处理。评估除 Recall@K 外还要看覆盖、热门偏置、索引延迟、更新时效和端到端 NDCG。
