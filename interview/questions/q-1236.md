# Deepseek-R1里面不仅推理能力很好，而且文采能力也很好，这个是个开放问题，如何让模型的文采能力也很好呢？

## 答案

推理能力与文采不是同一个目标。DeepSeek-R1 的公开报告主要证明多阶段训练和强化学习能够增强可验证推理；报告同时指出 R1-Zero 存在可读性差、语言混杂等问题，R1 通过 cold-start 数据和后续 SFT/RL 改善表达。因此，不能把观察到的文风简单归因于 reasoning RL。

要让模型既会推理又有良好表达，需要把“内容正确”和“最终呈现”拆开设计。

### 1. 预训练数据决定语言上限

- 提高高质量文学、新闻、说明文和多语言本地化文本的比例；
- 去除机器批量改写、SEO 堆词、重复模板和低质量翻译；
- 保留作者、体裁、时代、语言和许可证元数据；
- 防止某种单一网文风格占据数据分布；
- 用小模型消融确定数据源的真实贡献。

文采首先来自丰富、准确和多样的语言分布，不能只靠后训练凭空创造。

### 2. 用 SFT 建立表达范式

构造同一事实的多种高质量表达：

- 简洁说明、专业报告、故事化叙述、品牌文案；
- 不同受众、语气、长度和语言；
- 保持事实不变，只改变表达；
- 加入编辑前后样本，学习删冗余、改节奏和修辞。

对于推理模型，可以让内部推理过程服务于正确性，而最终答案按用户要求重新组织，避免把冗长探索直接暴露成正文。

### 3. 偏好优化要使用多维 rubric

偏好数据至少分开评价：

- 事实与逻辑；
- 清晰度和结构；
- 语言自然度；
- 风格一致性；
- 新颖性而非套话；
- 长度是否匹配任务；
- 安全与合规。

不能把“更长、更华丽”直接标成 chosen，否则模型会堆形容词。应加入长度匹配的 hard negative、事实正确但文风差的样本，以及文风好但事实错误的样本。

### 4. 保持能力解耦

可采用：

- style instruction 或显式风格标签；
- 通用推理 SFT 与写作 SFT 混合采样；
- 多目标 loss 和分任务评测；
- 独立写作 Adapter，按场景加载；
- 先生成事实骨架，再做受约束改写；
- 对改写前后做实体、数字和引用一致性校验。

这样能减少“提升文风却损害数学、代码和事实能力”的风险。

### 5. 评测

自动指标只做辅助，核心是盲评和成对比较：

- 事实性、逻辑和引用；
- 语言流畅、节奏、用词和结构；
- 风格遵循与跨段一致性；
- 套话率、重复率和多样性；
- 同一 Prompt 多次采样的稳定性；
- 不同语言和体裁分桶。

LLM Judge 需要用人工编辑黄金集校准，并控制位置、长度和模型自偏好。

跨境支付客服更强调准确、克制和可执行，不追求文学化。营销内容可以增加风格目标，但费率、时效、风险和监管表述必须由事实与合规校验优先约束。
