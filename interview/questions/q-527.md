# 用了多少卡

## 答案

必须按真实资源回答，不能为了显得规模大而虚报。建议同时说明卡型、数量、训练方式、时长和为什么需要这些资源。

### 回答模板

> 最终实验使用了[数量]张[型号/显存]，采用[数据并行、FSDP、ZeRO 或张量并行]，训练[时长/步数]。单卡放不下的主要原因是[参数、优化器状态、激活或长序列]。我们通过[混合精度、gradient checkpointing、LoRA、序列打包]把显存控制在[数值]，有效 batch size 为[数值]。我还做过[较小卡数]的消融或调参，最终配置是在效果、时间和成本间的权衡。

### 资源估算思路

全参数 AdamW 训练除权重外，还包含梯度、主权重和一二阶动量，粗略显存远高于仅保存模型参数；激活又与 batch、序列长度、层数相关。不能只用“参数量乘 2 字节”估算训练卡数。

若是 LoRA/QLoRA，要明确基座量化精度、可训练参数比例和是否做梯度检查点。若通过云服务训练，也应知道实际 GPU，而不是只说“平台自动分配”。

### 跨境支付项目

很多支付 RAG 或分类任务不需要大规模预训练。若小模型、LoRA 或 API 已满足准确率、延迟和合规要求，少用卡反而体现成本意识。应同时报告每次训练成本、推理 QPS 与单位请求成本。

### 面试总结

回答卡数时给出“卡型、数量、并行策略、训练时长、显存优化、有效 batch”完整口径，并解释资源瓶颈。规模不是成绩，能用实验和成本证明配置合理才是重点。
