# 找算法实习是专注于推荐领域吗？还是说有投大模型的岗位

## 答案

求职方向不是只投推荐，也会投大模型应用算法、RAG、Agent、检索排序和模型工程岗位。主线是“模型能力如何进入真实业务系统”，推荐与大模型在表示学习、召回排序、评测和在线实验上有不少共同基础，但岗位匹配会根据实际项目深度区分。

### 1. 核心方向

更匹配的岗位是：

- LLM 应用算法：RAG、Agent、工具调用和评测；
- 模型后训练：SFT、偏好优化和数据构造；
- 检索/排序：Embedding、Reranker、Learning to Rank；
- AI 平台与模型工程：训练、推理、服务治理；
- 多模态文档理解：KYC、票据、截图和文档抽取。

已有项目经验集中在跨境支付商户客服、支付异常解释、企业知识库和受控工具编排，重点是准确性、权限、审计和生产稳定性。

### 2. 为什么也考虑推荐或搜索

可迁移能力包括：

- 向量召回、负样本和 hard negative；
- Pointwise/Pairwise/Listwise 排序；
- 多目标优化与校准；
- 离线指标、A/B 和分桶分析；
- 高并发在线服务；
- 数据闭环和 bad case 挖掘。

例如支付产品匹配或通道路由也会经历：

```text
硬约束过滤
-> 多路候选生成
-> 成功率/成本/时效预测
-> 多目标重排
-> 在线实验
```

但跨境支付项目不等同于做过电商 CTR/CVR 推荐。若岗位要求多年广告竞价、Feed 排序或超大规模推荐经验，应明确能力差距，不能用相似概念替代直接经历。

### 3. 岗位选择标准

优先判断：

- 是否有真实业务和可量化目标；
- 工作是核心算法、平台还是长期数据清洗支持；
- 是否能接触数据、训练、评测和上线闭环；
- 团队是否有算力、数据与工程基础；
- 模型能力是否真正进入产品；
- 导师和团队是否能提供技术反馈；
- 实习周期内能否独立交付一个模块。

不会只根据“推荐”或“大模型”标签判断。一个数据和评测扎实的检索排序岗位，可能比只有 Demo 的大模型岗位更有成长价值。

### 4. 能力准备

推荐/搜索方向重点准备：

- 双塔、Cross-Encoder、召回与排序；
- CTR/CVR、多任务和去偏；
- Pointwise/Pairwise/Listwise；
- AUC、NDCG、Recall、校准与 A/B；
- 特征、样本和在线服务。

大模型方向重点准备：

- Transformer、训练与推理；
- SFT/DPO/PPO/GRPO；
- RAG、Agent 和工具安全；
- 评测、数据构造；
- vLLM、量化和分布式训练。

长期方向以 LLM 应用与检索排序融合为主：既能理解模型和训练，又能把系统做成可评测、可部署、可治理的生产能力。
