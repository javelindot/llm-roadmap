# 你在项目里担任什么角色？

## 答案

**一句话**：回答项目角色时，要讲清楚你负责的模块、做过的技术决策、解决的问题和最终结果，不要只说“参与开发”。

### 1. 可以怎么组织

建议按这个结构回答：

- 项目目标是什么。
- 团队规模和分工。
- 我负责哪些模块。
- 我做了哪些关键设计。
- 遇到什么问题，我怎么解决。
- 最后有什么效果。

### 2. 角色可以包括哪些

如果是 AI 客服或 Agent 项目，可以说：

- 负责 RAG 知识库建设。
- 负责文档解析和向量入库。
- 负责 Agent 流程编排。
- 负责工具调用封装。
- 负责模型评测集和效果评估。
- 负责接口对接和上线监控。
- 参与技术选型和架构设计。

### 3. 面试注意点

不要把团队成果全说成自己一个人做的，也不要说得太虚。最好用“我主要负责”和“我参与设计”区分。

### 4. 结合跨境支付项目怎么说（面试口径）

> 在这个跨境支付智能客服项目里，我主要负责 AI 能力和业务系统之间的工程落地。具体包括 RAG 知识库建设、支付异常问题的检索优化、订单和通道查询工具封装，以及 Agent 流程里的状态设计和保护节点。
>
> 比如付款失败解释这个场景，我负责把用户问题映射到订单查询、通道回执查询、风控原因查询和知识库检索几个步骤，并把结果整理成模型可用的上下文。为了避免模型乱答，我还做了引用约束和敏感信息过滤。
>
> 项目里我不只是写接口，也参与了技术选型和评估指标设计，比如用真实工单评测准确率、人工改写率和转人工率。这样能证明优化不是凭感觉，而是有数据闭环。

**钩子**：可以引到“个人负责 RAG/Agent/工具封装/评测闭环，而不是泛泛参与”。
