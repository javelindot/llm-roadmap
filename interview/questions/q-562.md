# 详细介绍一下Moe?优缺点各自是啥?

## 答案

**一句话**：MoE 是混合专家模型，通过 router 为每个 token 选择少量专家参与计算，实现大参数容量和较低激活成本的平衡。

### 原理

MoE 通常包含多个 expert 和一个 router。每个 token 经过 router 选择 top-k 个 expert，只有被选中的 expert 参与计算。

### 优点

- 总参数量大，模型容量强。
- 每次只激活部分参数，计算成本可控。
- 适合多领域、多任务知识承载。

### 缺点

- 路由训练复杂。
- expert 负载不均衡。
- 多卡通信成本高。
- 部署和调优更难。

### 结合跨境支付项目怎么说（面试口径）

> 在跨境支付里，MoE 可以理解成不同专家擅长不同领域，比如客服、风控、合规、通道知识。但生产选型时，我会看延迟、成本和稳定性，不会只因为 MoE 参数大就选它。
>
> 对高频客服场景，Dense 小模型加 RAG 可能更稳；复杂疑难问题再考虑更强模型或 MoE。

**钩子**：可以引到“router、top-k expert、负载均衡、复杂业务领域”。
