# Prompt是如何生成的，优化目标是什么，任务是什么？

## 答案

生产 Prompt 通常不是由模型自由生成，而是由版本化模板和可信运行时上下文组装：

```text
系统职责与安全边界
 + 当前任务说明
 + 输出 Schema
 + 可用工具定义
 + 会话状态
 + 检索证据/工具结果
 + 用户输入
```

### 1. Prompt 的来源

- 产品和业务规则定义目标与禁止事项；
- 领域专家提供标准流程、术语和边界案例；
- 工具服务提供机器可读 Schema；
- RAG 动态注入当前版本证据；
- 会话状态机注入已确认槽位；
- 用户输入作为不可信数据放在独立区块。

身份、租户、权限和商户 ID 由后端绑定，不允许模型生成或覆盖。

### 2. 任务拆分

不同节点使用不同 Prompt：

- 意图与实体抽取；
- Query Rewrite；
- 知识检索；
- 工具选择；
- 证据整合和回答；
- 输出验证或拒答判断。

理解节点只输出结构化意图，生成节点不能自行执行工具，验证节点只检查 claim。拆分后每个 Prompt 更短、更可评测。

### 3. 优化目标

Prompt 优化是多目标问题：

- 任务正确率；
- faithfulness 和引用正确率；
- 工具与参数准确率；
- JSON/Schema 通过率；
- 拒答、追问和转人工准确率；
- 提示注入和越权防护；
- P95/P99、token 和单次成本；
- 多语言和多轮稳定性。

不能只优化“回答看起来更自然”。

### 4. 优化流程

1. 建立最小、明确的 baseline；
2. 在冻结黄金集上运行；
3. 将错误分成知识、路由、格式、工具、安全和表达；
4. 一次只改变一个变量；
5. 做消融和回归；
6. 影子流量观察真实分布；
7. 小流量灰度并保留回滚。

Prompt、模型、工具 Schema、知识版本和解码参数一起版本化，否则无法复现实验。

### 5. 常用优化手段

- 明确输入输出和禁止事项；
- 使用枚举、JSON Schema 或约束解码；
- 提供少量边界清晰的正反例；
- 将长规则改为决策表；
- 对相似工具提供选择条件；
- 先抽取证据，再基于证据回答；
- 删除重复、冲突和无效背景；
- 证据不足时要求追问或拒答；
- 用小模型处理简单分类，复杂任务再升级。

### 6. 安全边界

外部文档、网页、邮件和工具结果都可能含提示注入，只能作为数据，不能覆盖系统指令。执行层重新验证工具白名单、参数、对象归属和权限。

跨境支付 Prompt 可以指导模型解释付款状态，但订单状态、金额、汇率和风控结果必须来自权威工具；退款、付款和解冻仍由确定性工作流执行。
