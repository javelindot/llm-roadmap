# 🔥🔥 1.简历拷打

## 答案

**一句话**：简历拷打不要背项目，要能把每段经历讲成“背景、目标、职责、难点、方案、结果、复盘”，并且所有数字和技术细节能对上。

### 1. 面试官会怎么拷

常见追问包括：

- 这个项目为什么做？
- 你具体负责哪一块？
- 技术选型为什么这么选？
- 遇到最大的难点是什么？
- 指标提升多少，怎么评估？
- 如果重做会怎么优化？
- 你说的这些是你做的，还是团队做的？

### 2. 回答方法

建议用 STAR 结构，但不要太模板化：

- S：业务背景是什么。
- T：项目目标是什么。
- A：你做了哪些关键动作。
- R：结果和指标是什么。
- 复盘：你学到了什么，后续怎么迭代。

### 3. 结合跨境支付项目怎么说（面试口径）

> 我的项目背景是跨境支付客服和风控问题处理成本比较高。用户问一笔付款为什么失败，客服要查订单、通道、风控和知识库多个系统，所以我们做了一个 RAG + 工具调用 + Agent 编排的智能客服/辅助决策系统。
>
> 我主要负责 RAG 检索优化、工具调用封装和 Agent 状态设计。比如针对付款未到账问题，我把订单查询、通道回执、风控原因和知识库 SOP 串成流程，并通过元数据过滤和 rerank 提高回答准确率。
>
> 项目难点是答案必须准确、合规、可追溯，不能让模型编。我的解决方式是工具事实优先、RAG 引用约束、敏感信息脱敏和 badcase 评测闭环。

**钩子**：可以引到“你具体负责的模块、真实指标、badcase 复盘、如果重做怎么优化”。
