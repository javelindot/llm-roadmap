# transformer八股，讲讲模型训练方面的经历

## 答案

回答可分为 Transformer 原理和个人训练经历两部分。

### 1. Transformer 核心

- 输入由 token embedding 与位置编码组成；
- Self-Attention：

\[
\operatorname{softmax}(QK^\top/\sqrt{d_k})V
\]

- 多头机制学习不同子空间；
- FFN 对每个位置做共享的非线性变换；
- 残差与 LayerNorm 保证深层优化；
- Encoder 双向理解，Decoder 使用 causal mask 自回归生成；
- 长度 \(n\) 下稠密 Attention 为 \(O(n^2d)\)。

应能继续解释 Q/K/V 是否可共享、为什么除 \(\sqrt{d_k}\)、Pre-Norm 与 Post-Norm、MHA/GQA/MQA、RoPE、KV Cache 和 FlashAttention。

### 2. 训练经历

> 我训练过[真实模型]解决[任务]。数据经过[清洗、脱敏、切分]，使用[模型和损失]，在[卡型/数量]上以[精度和并行方式]训练。训练中监控 loss、梯度范数、显存和[业务指标]。遇到[真实问题]后，我通过单卡复现和 bad batch 定位，最终用[措施]解决。模型经过消融、回归、影子和灰度评估后上线。

不能只背 Transformer 再说“调用 Hugging Face 完成训练”。需要讲清数据、超参数、显存估算、checkpoint 选择和失败排查。

### 跨境支付场景

训练支付文本模型时按国家、语言、币种和时间切片；交易与身份数据先脱敏；高风险意图同时看召回和误判；模型输出必须经过知识、工具、权限与审计约束。

### 面试总结

Transformer 八股验证原理，训练经历验证落地。把公式、复杂度和现代优化讲清，再用一条真实训练链路证明自己处理过数据、数值稳定、评估和上线问题。
