# 如何计算卷积层的输出大小？

## 答案

**一句话**：经典公式 **O = ⌊(I + 2P - K) / S⌋ + 1**——给定输入大小 I、kernel K、padding P、stride S，输出大小一行算清。

### 1. 标准公式
$$O = \left\lfloor \frac{I + 2P - K}{S} \right\rfloor + 1$$
- I：输入空间尺寸（H 或 W）
- K：kernel size
- P：padding（两边各 P）
- S：stride

### 2. 三个 padding 模式
- **"valid"**（无 padding）：P = 0
- **"same"**：自动 padding 让 O = ⌈I/S⌉
- **"full"**：P = K - 1，输出最大

### 3. Dilation（空洞卷积）扩展公式
$$O = \left\lfloor \frac{I + 2P - D(K-1) - 1}{S} \right\rfloor + 1$$
- D：dilation rate（默认 1）

### 4. 通道维度
- 输入：(B, C_in, H_in, W_in)
- 输出：(B, **C_out**, H_out, W_out)
- C_out = kernel 数量
- 每个 kernel 形状：(C_in, K, K)

### 5. 参数量公式
$$\text{params} = K \times K \times C_{in} \times C_{out} + C_{out}$$
（最后的 + C_out 是 bias）

### 6. FLOPs 估算
$$FLOPs = 2 \times K^2 \times C_{in} \times C_{out} \times H_{out} \times W_{out}$$

### 7. 常见配置例子
| 输入 | K | P | S | 输出 |
|------|---|---|---|------|
| 32×32 | 3 | 1 | 1 | 32×32（same） |
| 32×32 | 3 | 0 | 1 | 30×30 |
| 224×224 | 7 | 3 | 2 | 112×112 |
| 224×224 | 3 | 1 | 2 | 112×112 |
| 14×14 | 1 | 0 | 1 | 14×14（1x1 conv） |

### 8. 1×1 卷积特殊用途
- 改变通道数（升降维）
- 跨通道融合
- ResNet bottleneck、Inception、MobileNet 必备

### 9. 一句话总结
**O = ⌊(I + 2P - K)/S⌋ + 1**——背下这个公式 + 知道 dilation 扩展，CNN 维度计算就拿到了。

## 结合项目谈（口语版 · 约 20 秒）

> 这个公式我们跨境支付里写**KYB 证件识别**的 ViT 投影层时碰到过——准确说不是卷积，但 ViT patch embedding 本质就是 stride=K=patch_size 的 conv2d：
>
> 输入 224×224，patch_size=16，K=16, S=16, P=0 → 输出 14×14 = 196 个 patch。
>
> 这就是为什么 ViT-B/16 一张 224 图变成 196 个 token。如果改成 patch_size=14，输出 16×16 = 256 token，序列变长。
>
> 还有一个细节坑：我们做过把 OCR 截图按 224×224 喂 ViT，但发票尺寸经常 1024×512 这种非方形。直接缩放会失真，我们后来做了"按比例缩放 + zero padding 到 224×224"，padding 部分用 attention mask 屏蔽掉，效果稳很多。
>
> 一句话：**这个公式是 CV 基础题**，工业 LLM 里直接用得不多，但**多模态项目里 patch embedding 维度怎么算、padding 怎么处理**经常会撞到。
