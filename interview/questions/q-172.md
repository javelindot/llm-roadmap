# AI RAG项目是你参与了还是全程你自己做的？

## 答案

**一句话**：回答这个问题要明确自己的参与边界，既不要夸大全程独立，也不要把自己说成只做边角料。

### 核心回答
可以按模块说明：
- 需求阶段：参与业务场景梳理和指标定义。
- 数据阶段：参与文档整理、清洗、切片策略。
- 检索阶段：参与 embedding、向量库、混合检索、rerank。
- 生成阶段：参与 prompt、引用、拒答策略。
- 评估阶段：参与测试集、bad case、RAGAS/人工评估。
- 工程阶段：参与接口、日志、监控、权限、灰度。

重点讲自己负责的决策和结果，而不是说"全程都是我一个人做"。

### 1. 这类问题怎么回答

面试官问这个，通常不是只想听"是不是你做的"，而是想判断：

- 你有没有真实参与。
- 你负责的模块边界是什么。
- 你遇到过哪些问题。
- 你有没有独立思考和优化。
- 你能不能讲出结果和指标。

回答时最好不要说得太虚，也不要把团队成果都揽到自己身上。

### 2. 推荐回答结构

可以按"项目背景 → 我的职责 → 关键难点 → 我的改进 → 效果"来讲。

这样既真实，又能体现能力。

### 3. 面试总结

核心是讲清楚自己的贡献和决策点，尤其是你怎么解决 RAG 里的解析、检索、评估和上线问题。

### 4. 结合跨境支付项目怎么说（面试口径）

> 我不是一个人把整个 RAG 平台从 0 到 1 全包了，但核心链路我参与比较深。我们做的是跨境支付知识助手，主要服务客服、运营和风控同学。
>
> 我负责过文档解析和切片策略、混合检索方案、prompt 和引用格式、bad case 分析这几块。比如通道规则表格一开始召回很差，国家限制、币种和结算周期经常被切散。我后来把表格按业务单元保留，并加国家、币种、通道元数据过滤，效果明显稳定了。
>
> 所以如果问我是不是全程自己做，我会说不是单兵作战，但我对核心 RAG 链路和关键优化点是实际负责过的，也能讲清楚每一步为什么这么设计。

**钩子**：可以继续追问切片策略、混合检索、评估指标和 bad case 如何优化。
