# 2.你都读过哪些关于多模态的技术报告

## 答案

这是阅读真实性题，不应只报模型名字。建议从自己确实精读过的报告中选 3 至 5 篇，分别讲清架构、数据、训练目标、评测和一个可迁移的结论。下面是截至 2026 年 6 月可选的阅读脉络，不代表个人已经读过。

### 1. 基础路线

- **CLIP**：图文对比学习，理解大规模弱配对数据如何得到可迁移视觉表示；
- **Flamingo**：视觉编码器、Perceiver Resampler 与冻结语言模型的跨模态连接；
- **BLIP-2**：用 Q-Former 在冻结视觉编码器和冻结 LLM 之间进行轻量对齐；
- **LLaVA**：视觉指令微调如何把图像表示接入语言模型。

这些工作适合回答多模态系统如何从“对齐表示”发展到“视觉指令跟随”。

### 2. 统一图像、视频和音频

- **LLaVA-OneVision**：单图、多图和视频之间的任务迁移；
- **Qwen2.5-VL**：动态分辨率、视觉定位、文档与视频理解；
- **Qwen2.5-Omni**：统一文本、图像、音频和视频，并进行流式语音生成；
- **Qwen3-VL**：长交错多模态上下文、空间时间建模和更强视觉推理；
- **Qwen3-Omni**：Thinker-Talker MoE、统一感知与生成、低时延流式语音。

读报告时应比较：

```text
视觉/音频编码器
-> 投影器或跨模态适配器
-> LLM 主干
-> 输入 token 化与位置编码
-> 预训练、指令微调和偏好优化
-> 单图/多图/视频/音频评测
```

### 3. 面试示例

> 我重点读过 BLIP-2、LLaVA-OneVision 和 Qwen3-VL。BLIP-2 让我理解冻结单模态模型时，Q-Former 如何减少跨模态对齐成本；LLaVA-OneVision 重点展示图像训练向多图和视频任务迁移；Qwen3-VL 则进一步处理长交错上下文、视频时间对齐和多层视觉特征。阅读时我不仅看 benchmark，还比较视觉 token 数、数据配方、空间 grounding、视频采样和推理成本。

只保留真实读过的报告，并准备回答“最关键的一张架构图”“一个消融实验”和“一个局限”。

### 4. 跨境支付场景

多模态可用于票据、护照、账单和合同理解，联合版面、图像、OCR 文本与语言推理。但高风险字段应由 OCR/文档模型、规则和人工交叉验证；视觉语言模型生成的解释不能直接作为 KYC 或资金决策事实。

### 5. 面试总结

高质量回答不是罗列十篇论文，而是选择少量真实精读报告，讲清技术演进、架构差异、训练数据、评测缺陷以及如何落到具体业务。
