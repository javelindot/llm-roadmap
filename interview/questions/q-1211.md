# lora rank 设的多大 alpha 有什么作用

## 答案

LoRA 假设权重更新可以用低秩矩阵表示：

```text
W' = W + ΔW
ΔW = (α / r) BA
```

其中 `A ∈ R^(r×d_in)`、`B ∈ R^(d_out×r)`，基础权重 W 冻结，只训练 A、B。

### 1. Rank 的作用

`r` 决定更新子空间的容量：

- rank 小：参数少、显存低、训练快，但可能欠拟合；
- rank 大：表达能力强，但参数、优化器状态和通信增加；
- rank 超过任务需要后，收益通常递减，还可能增加过拟合。

可训练参数约与：

```text
r × (d_in + d_out)
```

成正比。

### 2. Alpha 的作用

标准 LoRA 使用 `α/r` 缩放更新：

- `α` 控制 Adapter 更新相对基础权重的尺度；
- 相同 A/B 下，alpha 越大，ΔW 影响越强；
- alpha 不是学习率，但会与学习率、初始化和 rank 共同影响梯度及最终更新；
- 不同实现的 scaling 可能不同，必须检查框架配置。

常见起点是 `α` 与 `r` 相等，或设为 `2r`，但不是通用最优。

### 3. 如何选择 Rank

可从 `r=8/16/32/64` 做受控 sweep，保持：

- 相同数据和训练 token；
- 相同 target modules；
- 相同学习率搜索范围；
- 相同评测集和随机种子；
- 同时记录质量、显存、吞吐和 checkpoint 大小。

格式或风格适配往往较低 rank 即可；复杂领域、多任务或工具行为可能需要更高 rank。不同层还可以使用 rank pattern，而不是所有模块统一 rank。

### 4. Target Modules

常见选择：

- Attention 的 `q_proj、k_proj、v_proj、o_proj`；
- FFN 的 `gate_proj、up_proj、down_proj`；
- 必要时 Embedding 或 LM Head。

只训练 Q/V 参数最省，但任务变化大时可能不足。增加 FFN Adapter 会提升容量，也增加参数。

### 5. Rank 与 Alpha 的耦合

标准 LoRA 中 rank 增大但 alpha 不变，会使 `α/r` 变小。比较不同 rank 时必须说明 scaling，否则结果可能来自更新幅度而非 rank 容量。

Rank-Stabilized LoRA 使用：

```text
α / sqrt(r)
```

减少高 rank 下梯度和更新尺度随 rank 衰减的问题。使用 PEFT 等框架时要确认是否启用 rsLoRA。

### 6. 训练与评测

- 监控训练/验证 loss 和梯度范数；
- 检查 Adapter 是否真正挂载到目标层；
- 保存 base model revision、Tokenizer 和 LoRA config；
- 与 Prompt/RAG baseline 比较；
- 评估通用能力遗忘、格式、安全和多语言分桶；
- 合并权重前后做数值和生成回归。

跨境支付客服可从低 rank 学习术语、格式和工具调用；国家规则、费率和实时状态仍应放在 RAG/工具中，不通过提高 rank 写入模型记忆。
