# 电商平台推荐系统差异比较

## 答案

比较电商平台推荐系统，不能只比较模型名称。不同平台的供给、用户意图、流量入口、履约方式和商业目标不同，会决定召回、排序和约束的差异。

### 1. 通用主链路

大多数平台都采用：

> 场景识别与硬过滤 → 多路召回 → 粗排 → 多任务精排 → 列表重排 → A/B 与反馈闭环。

共同目标包括点击、加购、下单、支付、履约、复购和长期价值；共同约束包括库存、价格、物流、商家质量、广告和合规。

### 2. 主要差异维度

| 平台形态 | 用户意图 | 供给特点 | 推荐侧重点 |
|---|---|---|---|
| 搜索型货架电商 | 意图明确 | SKU 标准化较强 | Query-商品相关性、价格、库存、转化 |
| 首页信息流电商 | 意图较弱 | 商品规模大 | 兴趣发现、多样性、长期价值 |
| 内容电商 | 内容消费与购物交织 | 视频、达人、商品关系复杂 | 内容兴趣、作者信任、商品转化联合建模 |
| 低价/活动驱动平台 | 价格和活动敏感 | 供给与补贴变化快 | 性价比、活动约束、供给效率 |
| 跨境电商 | 国家与履约约束强 | 多语言、多币种、长链路 | 本地化、税费、物流时效、支付可用性 |
| B2B 电商 | 决策周期长 | 询报价、阶梯价和合同 | 企业画像、采购周期、线索质量 |

这些是业务形态上的抽象，不代表某一家公司的内部实现固定不变。

### 3. 数据与标签差异

- 搜索场景更依赖 Query、类目、属性和精确匹配；
- 推荐场景更依赖长期行为、session 和探索；
- 内容电商需要同时建模内容互动和商品交易，处理归因链路；
- B2B 的正样本稀疏且延迟长，询盘、报价、审批和复购都可能是目标；
- 跨境场景需要国家、语言、币种、税费、物流和支付成功等额外标签。

同一个“点击”在不同场景含义不同。内容点击可能是娱乐消费，搜索点击更接近明确采购，因此样本权重和漏斗不能直接复用。

### 4. 召回和精排差异

货架搜索通常增加 BM25、类目属性和 Query-Item 双塔；首页推荐增加 ItemCF、序列、热门和探索；内容电商还要召回作者、视频、直播间和挂载商品；跨境电商则必须前置国家可售、库存、物流、税费和支付方式过滤。

精排目标可能是：

```text
pCTR、pATC、pOrder、pPaySuccess、
pCancel、pRefund、pChargeback、LongTermValue
```

不同平台对这些目标的权重不同。只优化 GMV 可能偏向高价商品，只优化 CTR 可能出现标题党，只优化支付成功率又可能牺牲覆盖率。

### 5. 冷启动与探索

- 新用户：地域热门、上下文、入口和首轮交互；
- 新商品：标题、属性、图片、多模态内容和商家先验；
- 新商家：准入质量、类目供给和受控探索；
- 新市场：跨语言迁移、本地规则和小流量校准。

探索必须设置质量、库存、履约和风险底线，不能为了长尾覆盖放开不合规商品。

### 6. 跨境支付的作用边界

SUNRATE、连连国际、PingPong 这类跨境支付服务商通常不拥有电商平台的商品推荐主链路，但可以在授权和合规边界内提供：

- 国家、币种和支付方式覆盖；
- 聚合后的支付成功率与失败类型；
- 手续费、汇兑和结算时效；
- 拒付、退款和风险信号；
- 商户可用的收款、付款和 FX 产品。

这些信号可帮助优化“下单后能否顺利付款”，但不能跨商户共享用户级交易数据，也不能由支付公司替平台决定商品兴趣。更贴近支付公司的推荐任务，是向商户推荐适用的收款账户、付款方式、换汇和收单方案。

### 7. 如何公平比较

平台效果不能直接比较公开 CTR 或 GMV。应在同一平台内通过：

- 召回：Recall@K、覆盖率、独占贡献；
- 排序：GAUC、NDCG、LogLoss、校准；
- 交易：下单、支付成功、净 GMV；
- 生态：长尾、新品、商家公平和负反馈；
- 风险：退款、拒付、投诉和欺诈损失；
- 系统：P95/P99、成本和降级率。

推荐系统的差异首先来自业务约束和数据生成机制，模型架构是这些差异之后的工程选择。
