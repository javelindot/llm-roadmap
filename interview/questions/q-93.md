# 卷积层和全连接层的区别是什么？

## 答案

**一句话**：全连接层 = **每个输出与所有输入相连**；卷积层 = **局部连接 + 权重共享**——这两个特性让 CNN 参数少、平移不变、适合图像。

### 1. 全连接层（FC / Linear / Dense）
- 公式：y = W·x + b
- W 形状：(out_dim, in_dim)
- 每个输出节点连接**所有**输入
- 参数量 = in_dim × out_dim

### 2. 卷积层（Conv）
- 公式：y[i] = Σ W[k] · x[i+k]
- 每个输出节点只看**局部窗口**输入（kernel 大小）
- 所有位置**共享同一组权重**
- 参数量 = kernel_h × kernel_w × in_ch × out_ch

### 3. 三大核心区别

| 维度 | 全连接 | 卷积 |
|------|--------|------|
| **连接方式** | 全局 | 局部 |
| **参数共享** | ❌ | ✅ |
| **参数量** | 大（in × out） | 小（kernel × ch） |
| **平移不变性** | ❌ | ✅ |
| **空间结构感知** | ❌ | ✅ |
| **输入尺寸** | 固定 | 可变 |

### 4. 为什么图像用卷积不用 FC
- 图像 224×224×3 ≈ 15w 维，FC 第一层就要几千万参数
- 图像局部相关性强（邻近像素相关）
- 物体出现在哪里都要识别 → 平移不变
- 卷积天然满足上面三条

### 5. 为什么文本/Transformer 用 FC 不用卷积
- LLM 的 FFN 就是 FC（gate / up / down）
- Attention 已经处理"位置和上下文"，FC 只需做特征变换
- 卷积的"局部窗口"假设不适合长程依赖

### 6. 卷积可以看作受限的 FC
- 把卷积展开成稀疏 + 共享权重的 FC 矩阵
- FC 的"过分自由"反而是过拟合的祸根

### 7. 实战经验
- CV 任务：CNN backbone + FC head
- NLP：Transformer（FC + Attention）
- 多模态：Vision Encoder（CNN/ViT） + LLM（FC）+ 投影层
- LLM 时代 FC 反而是 Transformer 的 80% 参数（FFN）

### 8. 一句话总结
**卷积 = 局部 + 共享，FC = 全局 + 独立**——选哪个看数据是否有局部相关性和平移不变性。

## 结合项目谈（口语版 · 约 25 秒）

> 这道题工业 LLM 落地里直接体会到——**Transformer 里 80% 参数其实就是 FC**（FFN 的 up / gate / down），卷积只在多模态接 Vision Encoder 时碰得到。
>
> 我们跨境支付里**KYB 证件识别**用过两个对比：
>
> **早期方案**：CNN 提特征 + FC 分类。营业执照各字段位置固定，CNN 的平移不变性反而是负担——我们其实不希望"公司名"和"注册号"位置混淆。
>
> **后来方案**：ViT 当 backbone + FC 投影层接 LLM。ViT 把图像切 patch 后直接走 attention（FC），**没有卷积**。这反而更适合证件识别——位置信息通过 positional encoding 显式给。
>
> 一个有趣观察：**ViT 在大数据下普遍打败 CNN**，因为 FC 的表达能力上限更高，数据够多时不需要"局部/共享"这种归纳偏置。CNN 在小数据下还有优势。
>
> 我对这道题的总结：**面试要会讲两者区别和适用场景**，但 LLM 时代你会更多碰到 FC，**FC 的参数管理（LoRA 旁挂、量化、低秩近似）是更值得花时间的话题**。
