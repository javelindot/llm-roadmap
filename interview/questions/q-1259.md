# 如何分析训练结果并改进训练数据的构造方式？

## 答案

分析训练结果不能只看一条总 loss 曲线。应把训练稳定性、泛化、数据分桶和真实任务错误对应起来，再决定修改数据、目标函数还是模型。

### 1. 先确认实验可信

- 固定代码、模型、Tokenizer、随机种子和数据版本；
- 验证 train/valid/test 无重复和实体泄漏；
- 检查 label、mask、padding 和 loss reduction；
- 确认参数确实更新，梯度没有 NaN/Inf；
- 保持 baseline 与新实验的 token budget、硬件和评测一致。

实验不可复现时，先修训练链路，不能根据偶然波动改数据。

### 2. 阅读训练曲线

**Train 与 Valid 同时下降**

模型仍在学习，可继续训练或扩大数据。

**Train 下降、Valid 回升**

过拟合或数据分布不一致。考虑 early stopping、去重、增加多样性、正则和减少 epoch。

**两者都不降**

可能是学习率、数据标签、模型容量、输入截断、梯度或任务定义问题。

**Loss 正常但业务指标差**

训练目标与业务目标错位，例如 token CE 下降但工具成功率不升，或对比 loss 下降但 Recall@K 没改善。

### 3. 做错误分桶

每个 bad case 标注：

- 任务、语言、长度和难度；
- 国家、币种、产品和通道；
- 高频、长尾、新实体；
- 事实错误、漏答、格式、拒答或越权；
- 是否缺知识、召回错误、模型推理错误；
- 训练集中是否有相似样本。

总体平均分可能掩盖小语种、长文本或高风险样本退化。

### 4. 分析数据覆盖

- 标签分布和样本长度；
- 来源、时间、用户与业务实体分布；
- 重复率与模板占比；
- 正负样本难度；
- 噪声、冲突和标注分歧；
- train/test 距离；
- 高损失样本与高置信错样本。

高损失不一定是有价值的 hard case，也可能是错标或损坏数据，需要人工抽样判断。

### 5. 按问题改造数据

| 现象 | 数据改进 |
|---|---|
| 长尾召回差 | 增加长尾正样本和 hard negative |
| 数字/日期错误 | 构造最小变化对和程序校验 |
| 过度拒答 | 增加安全可回答边界样本 |
| 格式错误 | 加入完整 Schema 示例和失败样本 |
| 输出冗长 | 长度匹配偏好对和精简示范 |
| 多语言退化 | 语言配比、翻译质量和原生标注 |
| 记住模板 | 去重、改写和来源多样化 |
| 工具选错 | 增加相似工具、边界参数和失败恢复轨迹 |

优先修复最影响业务且最可验证的一类问题，做单变量实验，而不是一次重做全部数据。

### 6. 数据版本与闭环

每条样本保留来源、许可证、标注者、版本、质量分和进入训练的原因。线上 bad case 需脱敏、去重、人工复核，并与原测试集隔离。新数据加入后必须同时检查目标分桶提升和通用能力回归。

跨境支付中，动态费率、订单状态和通道结果不适合做成记忆型训练样本；训练数据用于学习流程和边界，实时事实继续由工具提供。
