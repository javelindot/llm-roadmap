# self attention 计算 优缺点

## 答案

Self-Attention 让序列中每个位置根据内容动态聚合其他位置的信息。输入 `X` 的形状为 `[B, N, d_model]`，核心计算是：

```text
Q = X * W_Q
K = X * W_K
V = X * W_V

Scores = Q * K^T / sqrt(d_k) + Mask
A = softmax(Scores)
Output = A * V
```

多头注意力把 `d_model` 划分成 `h` 个 head，每个 head 独立计算，再拼接并做输出投影：

```text
MultiHead(X) = Concat(head_1, ..., head_h) * W_O
```

### 1. 为什么除以 `sqrt(d_k)`

假设 Q、K 各维均值为 0、方差约为 1，点积由 `d_k` 项相加，方差会随 `d_k` 增长。维度较大时 logits 绝对值变大，softmax 容易饱和，梯度变小。

除以 `sqrt(d_k)` 后，logit 方差回到相对稳定的量级，使不同 head dimension 下的训练更稳定。

### 2. Mask 的作用

- **Padding mask**：屏蔽补齐 token；
- **Causal mask**：Decoder 中屏蔽未来位置；
- **Attention mask**：限制局部窗口、文档边界或指定连接。

Mask 通常在 softmax 前把不可见位置加上极小值。需要处理整行全部被 mask 的情况，否则可能出现 NaN。

### 3. 优点

1. **长距离依赖路径短**：任意两个 token 一层即可直接交互。
2. **训练并行度高**：相较 RNN，不需要沿时间步顺序计算。
3. **内容自适应**：注意力权重根据当前输入动态变化。
4. **多头分工**：不同 head 可以关注实体、语法、位置和关系。
5. **架构统一**：文本、图像、语音和多模态都能使用类似机制。

### 4. 缺点

1. **时间和显存复杂度高**：完整注意力矩阵为 `[N, N]`，计算近似 `O(N²d)`，注意力显存近似 `O(N²)`。
2. **长上下文成本高**：序列翻倍，注意力矩阵元素约变为四倍。
3. **缺少天然顺序信息**：必须加入 RoPE、绝对或相对位置编码。
4. **注意力权重不等于解释**：高权重不能直接证明因果贡献。
5. **自回归解码受内存带宽限制**：需要 KV Cache，长上下文和高并发会占用大量显存。
6. **可能关注噪声**：上下文过长、重复或冲突时，相关证据会被稀释。

### 5. 常见优化

- FlashAttention：通过 IO-aware tiling 减少 HBM 读写，不改变精确注意力结果；
- MQA/GQA：减少 KV head 和 KV Cache；
- sliding-window、block-sparse 或 linear attention：降低长序列成本，但可能牺牲全局交互；
- PagedAttention：改善服务端 KV Cache 分配和碎片；
- prefix caching：复用相同前缀的 KV；
- KV Cache 量化和上下文压缩。

### 6. 跨境支付中的工程取舍

长客服历史、合规规则和交易日志不应无差别塞入上下文。先用检索和工具选择与当前国家、币种、产品及订单相关的证据，再让 Self-Attention 建模这些信息，通常比单纯扩大窗口更准确、更便宜。多租户服务还必须隔离 prefix/KV cache，避免不同商户上下文串用。
