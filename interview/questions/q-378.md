# 怎么提高RAG的召回率？

## 答案

**一句话**：提高 RAG 召回率，就是让相关知识更容易被检索到，核心方法包括优化 query、优化 chunk、混合召回、扩大候选集、元数据过滤和评测反馈。

### 1. Query 侧提升

- Query rewrite，把口语化问题改成标准业务表达。
- Multi-query，从多个角度生成检索问题。
- 同义词扩展，比如付款、出款、payout。
- 关键词增强，比如错误码、币种、国家、通道名。
- 用户意图分类后选择不同召回策略。

### 2. 文档侧提升

- 按语义切分，不把标题和正文切散。
- 标题注入，避免 chunk 失去上下文。
- 加元数据，比如国家、通道、版本。
- 清理重复和过期文档。
- 对表格、错误码、FAQ 用不同切分策略。

### 3. 检索策略提升

- 向量召回 + BM25 混合召回。
- 适当增大候选 topK。
- 使用 rerank 提升排序。
- 对高价值知识加权。
- 对不同业务场景设置不同索引。

### 4. 结合跨境支付项目怎么说（面试口径）

> 在跨境支付 RAG 里，提高召回率我会先从 query 入手。用户说“钱没到”，知识库里可能写的是“payout pending due to bank processing”，所以要做 query rewrite，把口语问题扩展成付款未到账、payout pending、银行处理中、到账时效等表达。
>
> 文档侧我会做标题注入和元数据。比如一个 chunk 内容只有“处理时间为 1-3 个工作日”，如果没有标题，模型不知道这是哪种通道、哪个国家、哪类交易。把“巴西 / PIX / 付款到账时效”注入进去，召回会更准。
>
> 检索侧我会用混合召回。支付业务里错误码、通道名、币种必须精确匹配，向量召回不够稳定。最后用真实工单评估 recall@k，看优化是否真的提高召回率。

**钩子**：可以引到“query rewrite、标题注入、混合召回、recall@k 评估”。
