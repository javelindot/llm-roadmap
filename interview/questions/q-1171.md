# 项目中如何选择模型及原因

## 答案

项目中的模型选型是多目标工程决策，核心不是选择参数最大或榜单最高的模型，而是在任务质量、延迟、成本、安全和维护能力之间找到满足上线门槛的方案。

> 任务拆分 → 硬约束过滤 → 业务黄金集 → 候选实测 → 灰度上线 → 分层路由与持续评估。

### 1. 按任务拆模型

一个跨境支付智能系统通常包含不同任务：

- 意图识别、分类和 Query 改写；
- Embedding 与向量召回；
- Cross-Encoder Rerank；
- RAG 回答和摘要；
- 工具选择与参数抽取；
- KYC/KYB、回单和截图的多模态解析；
- 实时风险或支付成功率预测。

这些任务不应强行共用一个生成模型。实时风险使用专用分类或序列模型；检索使用 Embedding 和 Reranker；生成模型负责自然语言理解、证据整合和解释。

### 2. 先设置硬门槛

候选模型必须满足：

- 数据是否允许发送到外部服务；
- 中文、英文及目标国家语言能力；
- 上下文长度和长文档稳定性；
- JSON Schema、工具调用和流式输出能力；
- 目标硬件下的 P95/P99、QPS 和显存；
- 商用许可、版本稳定性和供应商退出方案；
- 私有化、审计、日志脱敏和数据保留要求。

不满足安全、许可或 SLA 的模型，无论公开 benchmark 多高都直接淘汰。

### 3. 用真实业务集比较

从脱敏线上问题构建黄金集，按国家、币种、语言、产品、难度和风险分桶。至少覆盖：

- KYC/KYB、收款、付款、换汇、退款和拒付；
- 多轮指代、缺少订单号和多意图问题；
- 长文档、多证据和相互冲突的规则；
- 无答案、过期知识和需要转人工；
- 工具选择、参数填写和权限校验；
- 提示注入、跨租户访问和敏感信息。

评测包括正确性、faithfulness、引用准确率、工具成功率、结构通过率、安全违规率和拒答准确率。非确定性模型要重复运行，统计均值、方差和最坏情况。

### 4. 实测性能和总成本

在真实输入长度、并发和目标硬件上压测：

- TTFT、TPOT、P50/P95/P99；
- QPS、批处理效率、超时率；
- 输入输出 token 和上下文缓存命中；
- GPU 显存、利用率和单位成功任务成本；
- 高峰容量、故障切换和降级能力。

成本不能只看每百万 token 单价，还要看一次业务问题是否被解决。较便宜模型如果频繁重试或转人工，实际成本可能更高。

### 5. 先建立基线，再决定是否微调

优先使用 Prompt、RAG、工具和结构化约束建立基线。以下情况再考虑 SFT 或 LoRA：

- 领域术语、输出格式和工具调用存在稳定错误模式；
- 有足量、高质量、合规的训练数据；
- 知识不是高频变化事实；
- 微调收益能在冻结测试集上稳定复现；
- 训练和版本维护成本低于收益。

国家规则、汇率、通道状态和商户实时数据放在 RAG 或工具中，不通过微调写入模型参数；权限和资金状态机必须由代码保证。

### 6. 采用分层路由

生产中通常不是一个模型处理全部请求：

- 高频分类和改写走小模型；
- 普通 RAG 问答走中等模型；
- 复杂多证据问题升级到强模型；
- 高风险请求增加 verifier 和人工审批；
- 超时或服务不可用时回退到搜索结果、规则模板或人工。

路由器也需要单独评测，防止为了省成本把复杂任务错误路由给弱模型。

### 7. 最终决策和上线

建立模型卡，记录：

> `质量、安全、语言、结构化输出、工具调用、延迟、吞吐、成本、部署、许可、维护`

先使用硬门槛过滤，再比较 Pareto 前沿。候选模型通过离线评测后，依次进行历史回放、shadow、小流量灰度和 A/B；模型升级时保留版本、prompt、数据和回滚路径。

最终选择应明确“适用范围”和“不适用范围”。例如某模型适合低风险知识问答，不代表它可以自动执行退款、付款或账户解冻。
