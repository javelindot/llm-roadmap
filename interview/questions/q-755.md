# relu的缺点是啥？如何改进？

## 答案

ReLU 的主要缺点来自负半轴完全为 0、正半轴无界和函数不平滑。

### Dying ReLU

当神经元输入长期落在负区间时，输出和梯度都为 0，参数可能再也无法更新。过大学习率、负偏置和不合适初始化会加剧。

可使用 Leaky ReLU：

$$
f(x)=\max(\alpha x,x)
$$

或 PReLU 学习 $\alpha$，保证负区仍有梯度；也可用 ELU、GELU、SiLU。配合 He 初始化、较小学习率和归一化同样有帮助。

### 非零中心与无界输出

ReLU 输出非负，可能使后层梯度方向偏移；正半轴无界，在异常激活或大权重下可能产生数值不稳定。BatchNorm/LayerNorm、合理初始化、weight decay 和 gradient clipping 可控制尺度。

### 不平滑与表达

ReLU 在 0 处不可导，框架通常定义一个次梯度，实际影响有限；但硬折线可能不如 GELU/SiLU 的平滑门控适合 Transformer。ReLU 还会永久丢弃负值信息。

### 取舍

Leaky/PReLU 修复死亡问题但引入超参数；ELU 计算更贵；GELU/SiLU 平滑但不产生严格稀疏。CNN 端侧常因 kernel 简单继续使用 ReLU/ReLU6，大模型 FFN 更常使用 GELU 或 SwiGLU。应以收敛、精度和目标硬件延迟共同选择。
