# 训练和推理的时候的BN均值和方差都是怎么得到的

## 答案

### 训练阶段

对输入张量的每个通道，Batch Normalization 使用当前 mini-batch 的样本统计量进行归一化：

$$
\mu_B=\frac{1}{m}\sum_{i=1}^{m}x_i,\qquad
\sigma_B^2=\frac{1}{m}\sum_{i=1}^{m}(x_i-\mu_B)^2
$$

然后计算：

$$
\hat{x}_i=\frac{x_i-\mu_B}{\sqrt{\sigma_B^2+\epsilon}},\qquad
y_i=\gamma\hat{x}_i+\beta
$$

其中 $\gamma$、$\beta$ 是可学习参数。卷积网络通常在 `N、H、W` 维度上统计，每个通道得到一组均值和方差。

训练时还会维护用于推理的 running mean 和 running variance。例如采用指数滑动平均：

$$
\mu_{run}\leftarrow(1-momentum)\mu_{run}+momentum\cdot\mu_B
$$

方差同理。不同框架对 `momentum` 的定义方向可能相反，使用时应看具体 API。某些实现归一化时使用有偏方差，但更新 running variance 时会做无偏修正，这也是框架间数值略有差异的原因。

### 推理阶段

推理时不能依赖当前 batch，否则同一个样本会因同行样本不同而得到不同结果，也无法稳定处理 batch size 为 1 的请求。因此推理使用训练期间累积的 running mean 和 running variance：

$$
y=\gamma\frac{x-\mu_{run}}{\sqrt{\sigma_{run}^2+\epsilon}}+\beta
$$

### 工程注意点

- batch 很小时，批统计量噪声大，可使用 SyncBatchNorm、GroupNorm 或 LayerNorm。
- 分布式训练中的普通 BN 只统计单卡 batch；SyncBN 会跨卡聚合统计量。
- 微调数据少或分布不稳定时，常冻结 BN 的 running statistics，必要时也冻结 $\gamma$、$\beta$。
- 训练和推理模式必须正确切换，例如 PyTorch 中分别调用 `model.train()` 和 `model.eval()`。
