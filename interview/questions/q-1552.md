# 请详细解释一下 Transformer 模型中的自注意力机制是如何工作的?它为什么比 RNN 更适合处理长序列?

## 答案

Self-Attention 让序列中的每个 token 根据当前内容动态聚合其他 token 的信息。它通过 Q/K 计算相关性，再用相关性权重加权 V。

### 1. 计算过程

输入：

```text
X in R^(B x S x H)
```

线性投影：

```text
Q = XW_Q
K = XW_K
V = XW_V
```

拆成多个 head 后：

```text
Q, K, V: [B, heads, S, d_head]
```

缩放点积 Attention：

```text
scores = QK^T / sqrt(d_head)
weights = softmax(scores + mask)
output = weights V
```

多个 head 拼接后经输出投影，再经过残差、归一化和 FFN。不同 head 可学习语法、指代、位置或语义等关系。

### 2. Mask

- Encoder Self-Attention：通常双向，只 mask padding；
- Decoder Self-Attention：使用 causal mask，位置 `t` 看不到未来；
- 业务还可加入 block、local 或权限相关 mask。

### 3. 为什么比 RNN 更适合长序列

**路径更短**

RNN 中位置 1 到位置 `S` 的信息需经过 `O(S)` 次状态传递；全局 Attention 可一次直接交互，理论路径长度为 `O(1)`，更容易学习长距离依赖。

**训练可并行**

RNN 的时间步递归依赖前一步，难以并行；Transformer 训练时可同时计算全部位置，更适合 GPU/TPU 和大规模数据。

**容量更灵活**

每个 token 根据内容选择不同上下文，不必把全部历史压缩到固定 hidden state。

### 4. 代价与边界

标准 Attention 的 score 矩阵是 `S x S`：

```text
时间/显存复杂度约 O(S^2)
```

长序列会受到显存和 IO 限制；自回归 Decode 仍逐 token 串行，还要维护 KV Cache。FlashAttention 改善 IO，不改变标准 Attention 的二次计算量；滑窗、稀疏、线性 Attention 和状态空间模型则改变可见模式或复杂度。

标称长上下文也不保证能正确使用中间信息，应评测 lost-in-the-middle 和多跳检索。

### 5. 跨境支付场景

模型理解“这笔 USD payout 因收款账户无效而失败”时，可让“失败”直接关注币种、交易类型和错误原因。但实时状态和金额不能只依赖 Attention 记忆，仍需工具校验。

### 6. 面试总结

Self-Attention 的核心是 `softmax(QK^T/sqrt(d))V`。它比 RNN 更适合大规模长序列训练，因为依赖路径短、全序列可并行、无需固定状态瓶颈；代价是二次 Attention 与 KV Cache。
