# 问模型训练、RAG、embedding、transformer等基础八股

## 答案

这是一组综合基础题，面试时应按“训练 -> Transformer -> Embedding -> RAG -> 评测与部署”形成完整链路，而不是散点背术语。

### 1. 模型训练

基本流程：

```text
数据采集与清洗
-> tokenizer/特征处理
-> 前向传播
-> loss
-> 反向传播
-> optimizer 更新
-> 验证与 checkpoint
-> 离线/线上评测
```

常见问题：

- 过拟合：正则、Dropout、数据增强、早停；
- 梯度消失/爆炸：残差、Norm、初始化、gradient clipping；
- OOM：减小 micro-batch、checkpoint、ZeRO/FSDP、LoRA；
- 数据泄漏：按时间、用户或实体切分；
- 类别不均衡：重采样、类别权重、Focal Loss。

### 2. Transformer

核心组件：

- Token Embedding；
- Position Encoding/RoPE；
- Multi-Head Self-Attention；
- FFN/SwiGLU；
- Residual；
- LayerNorm/RMSNorm。

注意力：

```text
Attention(Q,K,V) =
softmax(QK^T / sqrt(d_k)) V
```

Encoder 通常双向理解，Decoder 使用 causal mask 自回归生成。推理分 Prefill 和 Decode，并使用 KV Cache。

### 3. Embedding

Embedding 将文本、图片或实体映射到稠密向量，使语义相近的对象在向量空间靠近。

常见训练：

- 对比学习；
- 双塔；
- in-batch negatives；
- hard negative；
- supervised similarity。

常见相似度：

```text
cosine
dot product
L2 distance
```

评估看 Recall@K、MRR、NDCG、语义簇、跨语言和 hard case。向量维度越大不一定越好，还要考虑索引内存和检索延迟。

### 4. RAG

标准流程：

```text
文档解析
-> 分块
-> metadata
-> embedding/index
-> query rewrite
-> hybrid retrieval
-> rerank
-> context assembly
-> generation
-> citation/verification
```

常见优化：

- BM25 + dense hybrid；
- metadata filter；
- query expansion；
- parent-child chunk；
- Reranker；
- 去重和多样性；
- 无答案检测；
- 引用验证；
- 索引增量更新。

### 5. 评测

RAG 不能只看最终回答：

- 召回：Recall@K、HitRate@K；
- 排序：MRR、NDCG；
- 生成：事实一致性、完整性、引用支持率；
- 系统：P95、成本、失败率；
- 安全：越权检索、隐私泄露、提示注入。

### 6. 跨境支付场景

知识库按国家、币种、产品、规则版本和权限过滤；模型解释订单问题前先查实时交易工具。政策和账务状态不能只依赖向量召回，必须有结构化过滤、版本控制和权威系统校验。

### 7. 面试总结

建议用一条主线回答：Transformer/Embedding 提供表示能力，RAG 提供外部知识，训练让模型适配任务，评测与部署保证真实可用。每个模块都要能说出原理、指标、失败模式和业务边界。
