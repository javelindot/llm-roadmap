# Qwen3 模型架构

## 答案

**一句话**：Qwen3 是阿里 2025 年发布的开源旗舰系列，**同时提供 Dense（0.6B-32B）和 MoE（30B-A3B / 235B-A22B）两条线**，统一支持**think / non-think 双模式**——架构是现代 LLM 标配（Pre-LN + RMSNorm + RoPE + GQA），创新点在**双模式推理**和 **MoE 高效化**。

### 1. 系列规格（2025-06 发布版）
| 模型 | 类型 | 总参 | 激活参 | 长度 |
|------|------|------|--------|------|
| Qwen3-0.6B / 1.7B / 4B / 8B / 14B / 32B | Dense | 同 | 同 | 32k-128k |
| Qwen3-30B-A3B | MoE | 30B | 3B | 128k |
| Qwen3-235B-A22B | MoE | 235B | 22B | 128k |

### 2. 核心架构（标准现代 LLM）
- **位置编码**：RoPE（base 大、原生长上下文）
- **归一化**：Pre-LN + RMSNorm
- **注意力**：GQA（4-8 groups）
- **激活函数**：SwiGLU
- **Vocab**：BPE 词表 ~152k

### 3. MoE 设计要点（235B-A22B）
- 128 个 expert，每 token 路由到 8 个
- **细粒度专家**：比传统 8-expert MoE 路由更精
- **shared expert**：部分 expert 始终激活（DeepSeek-MoE 风格）
- **load balancing loss**：避免某些 expert 过冷
- **激活参 22B**：推理只算 22B，但能力接近 235B Dense

### 4. 🌟 双模式（thinking / non-thinking）
Qwen3 最大创新：**一个模型支持两种推理模式**
- **think mode**：开启长 CoT，输出 `<think>...</think><answer>...</answer>`
- **non-think mode**：直接回答，速度快

切换方式：
- 用户 prompt 带 `/think` 或 `/no_think`
- 框架参数 `enable_thinking=False`

应用：
- 简单任务 → non-think 省 token
- 复杂任务 → think 精确推理

### 5. 训练数据规模
- 预训练 36T tokens（4x Qwen2.5）
- 119 种语言
- SFT 数百万条
- DPO + RLHF 多轮对齐

### 6. 性能定位（社区评测）
- Qwen3-235B-A22B：开源最强之一，部分指标超 GPT-4o
- Qwen3-32B Dense：单 H100 可推，性价比之王
- Qwen3-4B：边缘部署友好

### 7. 与 Qwen2.5 对比
- 词表扩到 152k（多语言）
- MoE 重新设计（细粒度 + shared）
- 双模式（新增）
- 上下文 32k → 128k 起步
- 多语言能力大幅提升

### 8. 一句话总结
**Qwen3 = 现代 LLM 架构 + MoE 优化 + 双模式推理**——架构没革命，但工程优化到极致，是国内开源旗舰。

## 结合项目谈（口语版 · 约 30 秒）

> 我们跨境支付**新一代 base 升级到 Qwen3**，三个点对我们特别有用。
>
> **第一是双模式**。这是 Qwen3 我最喜欢的设计——同一个模型，简单工单（"查我的订单到账了吗"）走 non-think 模式，0.3 秒回；复杂工单（"我跨境付款被卡了，涉及三个商户、两个币种、四笔关联交易"）走 think 模式，让模型走完整推理链路。**一个模型同时覆盖两种需求**，推理成本和质量都对得起业务。
>
> **第二是 MoE**。我们调研过 Qwen3-30B-A3B——30B 总参但激活 3B，推理速度像 3B 模型，能力接近 14B Dense。客服回复这种高 QPS 场景挺合适，单卡 H100 能跑 50+ QPS。
>
> **第三是 128k 长上下文**。风控调查 Agent 经常拉一堆历史交易塞 prompt，Qwen2.5 32k 上限不够，Qwen3 直接 128k 起步，原生支持，外推都不用。
>
> 我们现在的部署组合：客服 Qwen3-30B-A3B MoE（高 QPS + 双模式）+ 风控 Qwen3-14B Dense（长上下文 + think 模式深推）+ 边缘 Qwen3-4B（简单分类）。
>
> 一句话：**Qwen3 的工程价值高于学术价值**——没有架构革命，但每个细节都对工业场景特别友好。
