# 🔥 human feedback是怎么被agent消化吸收的？有没有用rl进行策略更新？

## 答案

Human feedback 被 Agent 吸收有三种时间尺度：当前会话立即修正、跨会话写入记忆，以及离线更新模型/策略。不能把用户一句反馈直接当作可信长期规则。

### 当前会话

用户纠正后，Agent 更新任务 state、约束或计划，重新执行受影响步骤。结构化记录“原行动、反馈、修正和证据”，并让 verifier 确认结果。无需更新模型参数。

### 长期记忆

将稳定偏好或经验证的经验抽取为 memory，带用户/租户作用域、来源、时间、置信度和可撤销版本。个人偏好只影响该用户，事实纠错需要外部证据。低质量或恶意反馈不能直接进入共享记忆。

### 离线训练

反馈可构造成：

- SFT：问题与人工修正后的目标回答/轨迹；
- preference pair：修正结果为 chosen，原失败结果为 rejected；
- 工具/路由标签：正确工具、参数和步骤；
- reward signal：任务是否完成、人工评分或可验证结果。

数据去隐私、去重、审核后，可做 DPO、Reward Model + PPO/GRPO，或训练 router/critic。是否用 RL 取决于奖励是否可验证、是否需要在线探索。多数 Agent 产品先用 Prompt、workflow、memory 和 SFT/DPO，成本更低且稳定。

上线采用离线回归、shadow 和小流量 A/B，监控任务成功率、安全、成本与遗忘。反馈闭环必须防止 reward hacking、少数用户投毒和新策略破坏通用能力。
