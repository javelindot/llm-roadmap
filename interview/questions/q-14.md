# CLIP架构？

## 答案

**一句话**：CLIP 是一个图文双塔模型，用图像编码器和文本编码器分别把图片和文本映射到同一个向量空间，再通过对比学习让匹配的图文更近、不匹配的图文更远。

### 1. 架构组成
- **Image Encoder**：通常是 ResNet 或 ViT，把图片编码成图像向量。
- **Text Encoder**：通常是 Transformer，把文本编码成文本向量。
- **Projection Layer**：把图像向量和文本向量投影到同一语义空间。
- **Contrastive Loss**：一个 batch 里有 N 对图文，正确匹配是正样本，其余 N-1 个是负样本。
- **Similarity Matrix**：计算 N 张图和 N 段文本两两相似度，得到 N x N 的矩阵。

训练目标是：图片能找到对应文本，文本也能找到对应图片，所以通常同时做 image-to-text 和 text-to-image 两个方向的对比学习。

### 2. 为什么有效

CLIP 不需要每张图都有人标具体类别，而是用大规模图文对学习开放语义空间。因此它可以做：
- 零样本分类：用文本 prompt 当类别描述。
- 图文检索：输入图片找文本，或输入文本找图片。
- 多模态相似度：判断图片和描述是否一致。
- 多模态 RAG：把图片和文本都转成向量做召回。

### 3. 面试总结

CLIP 的核心不是分类头，而是**图文对齐空间**。它把"图片像什么"和"文本说什么"放到同一个 embedding 空间里比较。

## 结合项目谈（口语版 · 面试现场约 50 秒）

> 结合跨境支付业务，我会把 CLIP 理解成一个图文一致性和多模态检索工具。比如商户入驻时会上传营业执照、店铺截图、商品图片、物流单据，同时也会填写行业、商品描述、经营国家。如果只看文本，可能会漏掉图片里的风险；如果只看 OCR，也可能看不出图片语义。
>
> CLIP 的价值在于把图片和文本映射到同一个向量空间。比如商户描述自己卖办公用品，但上传的商品图更像虚拟币、博彩或高风险商品，我们就可以用 CLIP 做图文相似度和异常检测，给风控一个辅助信号。再比如做材料审核，输入"公司营业执照"可以召回相似证照图片，辅助判断资料类型。
>
> 所以我不会把 CLIP 只当成图像分类模型，而是把它放在 KYB、商品合规、资料一致性审核这些多模态风控场景里，用来补充传统 OCR 和规则引擎的不足。

**自然带出去的钩子**：
- "CLIP 适合做图文一致性，不一定直接做最终风控决策。"
- "在支付业务里我会把它作为风控特征之一，和 OCR、规则、交易行为一起用。"
- "它能帮助发现商户描述和实际经营图片不一致的问题。"
