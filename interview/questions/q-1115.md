# 模型微调的训练数据构建的关键点有哪些？

## 答案

微调数据构建的核心不是数量，而是目标能力、覆盖范围、标注一致性、数据安全和评测隔离。完整流程是：

> 定义能力 → 数据采集 → 清洗去重 → 标注与质检 → 配比采样 → 格式化 → 数据切分 → 训练前审计 → 线上回流。

### 1. 从目标能力定义数据 Schema

先明确模型要学什么：

- 意图识别和槽位抽取；
- 支付领域问答与合规话术；
- Query 改写和结构化输出；
- 工具选择与参数生成；
- 风控调查报告或商户材料摘要；
- 越权拒答和转人工。

每种能力定义输入、输出、允许证据、风险等级和评分标准。不能把所有任务简单拼成同一种问答对。

例如工具调用样本至少包含：

> `messages、available_tools、expected_tool、arguments、tool_result、final_response`

### 2. 数据来源和合规处理

可使用：

- 脱敏后的真实客服会话和工单；
- 业务专家编写的标准案例；
- 产品文档、SOP 和错误码生成的合成问题；
- 线上 bad case 与人工改写；
- 工具调用和流程 trace；
- 对抗与安全测试样本。

跨境支付数据需要：

- 删除或替换姓名、证件、银行卡、账户和交易标识；
- 保持同一会话内实体替换的一致性；
- 按租户、地区和用途授权；
- 限制原始数据保留周期；
- 记录数据来源、处理版本和使用范围；
- 防止训练样本恢复真实商户或用户。

### 3. 清洗、去重和泄漏检查

- 规则去除空回答、乱码、模板残留和工具异常；
- 精确哈希与 MinHash/embedding 近似去重；
- 删除错误规则、过期话术和未经确认的人工回复；
- 统一系统 Prompt、角色、特殊 token 和工具 schema；
- 检查训练集与验证/测试集的 query、答案、文档和实体重叠；
- 过滤 benchmark 污染和模型生成内容中的错误引用。

多轮会话不能随意截断，否则前文订单号丢失后，后续回答会变成错误样本。

### 4. 标注规范和一致性

标注指南应明确：

- 事实依据和可接受答案范围；
- 必须追问的信息；
- 可披露与不可披露的风控内容；
- 什么情况调用工具、拒答或转人工；
- 严重错误等级；
- 格式、语气和引用要求。

高风险样本至少双人标注并由业务专家仲裁。计算标注一致性，持续分析分歧来自指南不清、业务规则冲突还是标注错误。

### 5. 正样本、困难样本和拒答样本

只训练标准正例会导致模型在线上遇到边界问题时强行回答。数据应包含：

- 信息完整和信息缺失；
- 高频和长尾意图；
- 相似工具的 hard negative；
- 错误币种、错误订单归属和过期规则；
- 无答案、证据冲突和低置信场景；
- 提示注入、越权和敏感信息请求；
- 需要人工审批的资金动作。

拒答样本要区分“无法回答”“无权限”“需要补充信息”和“必须人工处理”，避免模型学成统一拒绝。

### 6. 数据配比与采样

配比由线上任务分布和风险共同决定：

- 高频任务保证基本覆盖；
- 长尾和高风险任务适当过采样；
- 多语言、国家、币种和产品分桶；
- 旧样本根据规则版本衰减或下架；
- 保留一定通用指令数据，降低灾难性遗忘；
- 合成数据设置上限，避免模型学习单一生成器风格。

不能只复制少数样本实现过采样，优先扩充真实变体和困难负样本。

### 7. SFT 与偏好数据

SFT 样本重点检查答案正确、格式一致、证据充分和过程可执行。

DPO/偏好数据要求同一 prompt 下 chosen 与 rejected 的差异确实对应目标偏好，例如：

- chosen 引用正确，rejected 无依据承诺；
- chosen 正确转人工，rejected 越权执行；
- chosen 参数完整，rejected 猜测订单或币种。

避免只让 chosen 更长、更礼貌，否则模型会学到长度和格式偏置。偏好对过于容易也无法提供有效梯度。

### 8. 数据切分和版本管理

按时间、商户或业务事件分组切分，保证同一会话、同一模板变体和同一事件不跨集合。测试集在模型与 Prompt 选择结束前保持冻结。

每个数据版本记录：

- 数据源和时间范围；
- 清洗、脱敏和去重规则；
- 各任务及分桶数量；
- tokenizer/chat template；
- 标注指南版本；
- 已知风险和质量报告。

### 9. 训练前后验收

训练前抽样人工审计，并用小规模 overfit 测试检查标签、mask 和模板。训练后除总体分数外，还要评估通用能力、长尾、安全、格式、校准和不同国家/语言分桶。

LIMA 等研究说明少量高质量示范可以显著改善对齐表现，但生产微调仍需要覆盖真实分布、困难边界和安全约束。数据质量高不等于只保留“漂亮答案”，而是让每种关键决策都有正确、可验证的监督。
