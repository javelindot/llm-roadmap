# 什么是 instruction tuning？在多模态场景下如何做？

## 答案

Instruction Tuning 是用“指令/输入 -> 期望输出”的监督数据继续训练预训练模型，使其学会理解自然语言任务、遵循格式和进行多任务泛化。它本质上通常是 SFT，不等于偏好优化或 RL。

### 1. 文本指令微调

训练样本可写为：

```text
system: 角色与边界
user: 指令 + 输入
assistant: 高质量目标回答
```

使用 teacher forcing 做 token-level cross-entropy，通常只对 assistant 输出计算 loss。数据混合问答、抽取、分类、工具和拒答任务，让模型从单一 next-token 预训练转为可交互助手。

### 2. 多模态架构

典型视觉语言模型：

```text
image/video
-> vision encoder
-> projector/resampler
-> visual tokens
-> LLM
-> text/tool/grounding output
```

也可能使用 cross-attention、统一 tokenizer 或原生多模态主干。音频还需 audio encoder/codec，视频包含时序采样与位置建模。

### 3. 多模态指令数据

任务应覆盖：

- 图像描述和 VQA；
- OCR、文档与图表理解；
- grounding、坐标和区域引用；
- 多图对比；
- 视频时序问答；
- 视觉工具调用；
- 无法看清、证据不足和安全拒答。

样本必须包含媒体引用与对话模板，保证 `<image>` 等占位符、视觉 token 和目标答案对齐。

### 4. 训练阶段

常见路线：

1. **模态对齐**：冻结或部分冻结 LLM/视觉编码器，训练 projector；
2. **多模态 SFT**：训练 projector、LLM 的 LoRA 或部分/全部参数；
3. **偏好/可验证优化**：改善帮助性、grounding、工具和安全；
4. **高分辨率/视频扩展**：继续训练动态分辨率与时序能力。

是否解冻视觉编码器取决于领域差异和资源。文档、医学、遥感等与通用图像差异大时，完全冻结可能限制效果。

### 5. 数据质量难点

- 教师模型对图像幻觉；
- OCR 文本与答案不一致；
- 坐标系、分辨率和裁剪错误；
- 多图顺序错位；
- 视频帧遗漏关键事件；
- 合成数据模板单一；
- 训练集和 benchmark 图片泄漏。

需用 OCR/检测器/规则/人工验证，保留原图与标注来源。

### 6. 评测

- VQA、OCR、DocVQA、ChartQA、grounding；
- 多图/视频与长上下文；
- 幻觉、引用区域和无答案；
- 文本通用能力回归；
- 图像 token、TTFT、显存和成本；
- 目标业务集人工评测。

### 7. 跨境支付场景

可微调票据、护照、账单和支付截图理解。号码、金额、印章和身份字段要由 OCR/规则/人工交叉验证，VLM 的自然语言答案不能直接成为 KYC 或资金事实。

### 8. 面试总结

多模态 instruction tuning 是“先对齐视觉/音频表示，再用多任务指令数据训练模型遵循跨模态任务”。关键在媒体与文本对齐、grounding、数据验证以及文本能力回归。
