# LoRA为什么有效

## 答案

LoRA 有效的核心假设是：预训练模型已经拥有大部分通用能力，下游适配所需的权重变化通常集中在一个低维子空间，不需要重新学习整个高维参数矩阵。

### 1. 数学形式

原权重：

```text
W ∈ R^(d_out x d_in)
```

LoRA 只学习：

```text
Delta W = B A

A ∈ R^(r x d_in)
B ∈ R^(d_out x r)
r << min(d_in, d_out)
```

前向：

```text
y = W x + (alpha/r) B A x
```

新增参数从：

```text
d_out * d_in
```

降为：

```text
r * (d_in + d_out)
```

### 2. 为什么低秩假设合理

- 大模型高度过参数化；
- 下游任务通常只改变部分行为和表示方向；
- Transformer 不同层存在冗余；
- 微调梯度和权重增量常呈低有效秩；
- 冻结基座可保留已有通用能力，减少小数据过拟合。

LoRA 不是证明所有任务增量都严格低秩，而是用受限容量换取较好的效果/成本比。

### 3. 工程优势

- 可训练参数和优化器状态少；
- 更低显存；
- 迭代快；
- 一个基座可挂多个 Adapter；
- 可动态切换或合并，无额外推理层；
- 便于按客户、语言和任务隔离版本。

### 4. 什么时候可能失效

- 领域与基座差异极大；
- 需要学习新语言或深层知识；
- 数据规模很大、任务复杂；
- rank 太小；
- target modules 选错；
- 需要同时重塑视觉、语言和跨模态模块；
- 多个 Adapter 合并发生方向冲突。

此时可提高 rank/覆盖层、使用 DoRA/PiSSA 等变体，或考虑继续预训练和全参微调。

### 5. 为什么不只是“参数少”

参数少本身不保证有效。LoRA 的价值是：

- 低秩结构保留矩阵变换的方向组合能力；
- 零/小初始化使训练起点接近原模型；
- 残差式增量不破坏基座主路径；
- 可对不同线性层选择性注入。

### 6. 跨境支付

LoRA 适合学习支付客服格式、报告结构、工具协议和多语言表达。实时费率、汇率、交易状态和合规名单不应写进 Adapter，而应由 RAG/工具提供。

**面试总结**：LoRA 有效不是因为“大模型随便改一点就行”，而是预训练能力已足够，下游变化通常具有较低内在维度。
