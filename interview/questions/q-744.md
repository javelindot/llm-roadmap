# 为什么不直接用相似度召回？而是过一遍大模型？

## 答案

相似度召回与“大模型过一遍”解决的目标不同。双塔相似度适合全库低成本召回，大模型或 cross-encoder 适合在小候选集上做细粒度相关性判断。

### 相似度召回的限制

双塔分别把 query 和文档压缩成单个向量，只在最后做点积。它速度快、文档向量可离线索引，但存在信息瓶颈：

- 难处理否定、数字、时间和多个约束；
- query 与文档 token 无细粒度交互；
- 训练负样本与线上分布可能不一致；
- 高相似不等于真正支持答案。

### 大模型/重排器的价值

Cross-encoder 将 query 与候选文档联合编码，可以让每个 token 交互；LLM 还能判断证据完整性、时效、权限或多跳关系。它通常用于 top-50/100 重排、证据过滤或生成，而不是扫描百万文档。

### 合理架构

先用 BM25 与 dense ANN 多路召回保证覆盖，再由 cross-encoder 或小型 LLM 精排；只有复杂、低置信 query 才使用更大模型。高频结果可缓存，大模型判断可蒸馏到轻量 reranker。

### 如何证明必要

在标注集上比较 Recall@K、nDCG、Precision@K、最终答案正确率和延迟。若大模型只带来微小增益却显著增加 P99 和成本，就不应使用。简单精确查询甚至 BM25 可能优于大模型，系统应按 query 路由，而不是一刀切。
