# Deepspeed的zero1、zero2、zero3中混合精度的使用方式及其它显存优化方法

## 答案

混合精度和 ZeRO 是两个正交维度：混合精度降低单个张量的字节数并利用 Tensor Core，ZeRO 切分模型状态。ZeRO-1、2、3 都可以配合 FP16 或 BF16 使用，区别是“哪些状态被切分”，不是每个 stage 使用一种不同精度。

### 1. 混合精度训练的基本结构

前向和反向通常使用 FP16/BF16：

```text
低精度参数或计算副本
低精度 activation
低精度 gradient 通信或存储
FP32 master parameter
FP32 optimizer states
```

Adam 通常需要 FP32 一阶、二阶动量，并用 FP32 master weight 做稳定更新。FP16 动态范围较小，常使用 dynamic loss scaling：

```text
scaled_loss = loss * loss_scale
backward(scaled_loss)
grad = grad / loss_scale
```

发现 inf/nan 时跳过更新并降低 scale。BF16 与 FP32 的指数范围接近，通常不需要 loss scaling，但尾数精度更低，仍应监控数值稳定性。

### 2. ZeRO-1

切分：

- optimizer states；
- 与优化器更新相关的 FP32 master parameters。

不切分：

- 低精度模型参数副本；
- 梯度；
- activation。

适合优化器状态占显存大、但参数和梯度仍能放下的情况。Adam 的状态通常是主要节省来源。

### 3. ZeRO-2

在 ZeRO-1 基础上继续切分：

- gradients。

每个 rank 只保留自己负责的梯度分片，并更新对应参数分片，更新后同步参数。它进一步减少梯度显存，但低精度模型参数仍在每卡完整复制。

### 4. ZeRO-3

继续切分：

- parameters；
- gradients；
- optimizer states。

某层计算前按需 AllGather 参数，计算后可释放；反向再收集需要的参数，梯度通过 ReduceScatter 回到负责的 rank。ZeRO-3 最省模型状态显存，但通信、调度和小张量开销更高。

可概括为：

```text
Stage 1：切 optimizer states
Stage 2：再切 gradients
Stage 3：再切 parameters
```

ZeRO 不自动解决 activation 显存。长序列训练中，activation 可能超过模型状态，需要单独优化。

### 5. 其他显存优化方法

**Activation**

- gradient/activation checkpointing；
- FlashAttention 或 memory-efficient attention；
- sequence/context parallel；
- 缩短序列、动态 padding、sample packing；
- selective recomputation。

**参数与优化器**

- LoRA/QLoRA，只训练少量 Adapter；
- 8-bit optimizer；
- CPU/NVMe optimizer 或 parameter offload；
- 更低精度 FP8/INT8，但要有硬件和 Kernel 支持；
- 使用更省状态的优化器。

**并行与训练策略**

- tensor parallel、pipeline parallel；
- sequence/context parallel；
- micro batch + gradient accumulation；
- gradient accumulation boundary 上再做更新；
- 合理 bucket，重叠通信和计算；
- 及时释放无用张量，避免 Python 引用导致显存不回收。

### 6. 选型与排查

先估算每卡峰值：

```text
model states
+ activations
+ temporary buffers
+ communication buckets
+ allocator fragmentation
```

如果参数能放下但 Adam 状态过大，优先 ZeRO-1；梯度也成为瓶颈时用 ZeRO-2；模型参数本身放不下时用 ZeRO-3。若长上下文导致 OOM，单纯从 ZeRO-2 升到 ZeRO-3 可能治标不治本，应先看 activation。

验收同时看峰值显存、tokens/s、MFU、通信占比、loss 曲线和最终业务质量。跨境支付模型还需回归金额、币种、日期、工具参数与合规拒答，不能只确认训练不 OOM。
