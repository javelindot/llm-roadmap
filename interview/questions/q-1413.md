# 后训练用的哪个框架？你用过什么框架？如何使用deepspeed进行分布式训练？脚本是你自己写的吗？

## 答案

这是项目真实性核验题，应按真实经历回答。可以先列框架，再讲自己实际负责的训练脚本、配置、启动方式和排障过程，不能把团队公共框架全部说成自己从零开发。

### 1. 常见后训练框架

- PyTorch；
- Hugging Face Transformers/Trainer；
- PEFT；
- TRL；
- DeepSpeed；
- PyTorch FSDP；
- Megatron-Core/Megatron-LM；
- LLaMA-Factory、Axolotl 等训练封装；
- vLLM 等 rollout/inference engine。

SFT 可用 Transformers + PEFT + DeepSpeed；PPO/GRPO 等 RL 训练还需要 rollout、reward、reference policy 和分布式调度。

### 2. DeepSpeed 基本使用

训练代码中初始化：

```python
model_engine, optimizer, train_loader, scheduler = deepspeed.initialize(
    model=model,
    model_parameters=trainable_parameters,
    training_data=train_dataset,
    config="ds_config.json",
)

for batch in train_loader:
    outputs = model_engine(**batch)
    loss = outputs.loss
    model_engine.backward(loss)
    model_engine.step()
```

配置示例：

```json
{
  "bf16": {"enabled": true},
  "gradient_accumulation_steps": 8,
  "gradient_clipping": 1.0,
  "zero_optimization": {
    "stage": 2,
    "overlap_comm": true,
    "contiguous_gradients": true,
    "reduce_scatter": true
  }
}
```

启动：

```bash
deepspeed --num_gpus 8 train.py \
  --deepspeed ds_config.json
```

如果用 Hugging Face Trainer，可以传入 DeepSpeed 配置，由 Trainer 完成初始化。

### 3. 选 ZeRO Stage

- ZeRO-1：分片 optimizer state；
- ZeRO-2：再分片 gradient；
- ZeRO-3：再分片 parameter；
- 长序列 OOM 还要处理 activation，ZeRO 不会自动解决；
- 跨节点网络弱时，ZeRO-3 的参数通信可能成为瓶颈。

### 4. 自己写了什么

建议按真实情况表述：

> 底层通信和 ZeRO 分片不是我从零实现的，我基于 PyTorch/Transformers/DeepSpeed 编写和维护训练入口、Dataset/Collator、loss mask、LoRA 配置、DeepSpeed JSON、评测与 checkpoint 逻辑，并负责启动、日志和 OOM/NaN 排查。

如果只改过配置，就明确说“基于现有脚本完成适配和实验”，不要说整个框架都是自己写的。

### 5. 能体现实践的细节

- global batch 的计算；
- gradient accumulation；
- loss mask；
- bf16/fp16 稳定性；
- gradient checkpointing；
- ZeRO checkpoint 保存与恢复；
- 多机 hostfile 和网络配置；
- NCCL timeout/通信故障；
- 每 rank 显存和 tokens/s；
- resume 后 scheduler/optimizer 状态一致性。

### 6. 面试总结

回答重点不是报框架名，而是说明数据如何进入模型、DeepSpeed 如何初始化和启动、为什么选择某个 ZeRO stage、自己负责哪些脚本，以及遇到 OOM、NaN、通信和 checkpoint 问题如何解决。
