# PEFT-LoRA及变种

## 答案

**一句话**：LoRA 是在冻结原模型参数的基础上，为部分权重加低秩矩阵增量，只训练少量参数来完成微调。

### 核心回答
LoRA 把权重更新写成：

```text
W' = W + BA
```

其中 `W` 冻结，`A/B` 是低秩矩阵，rank 很小，所以训练参数少、显存低、易切换。

常见变种：
- **QLoRA**：基础模型 4bit 量化，只训练 LoRA，显存更省。
- **AdaLoRA**：动态分配不同层的 rank。
- **DoRA**：把权重方向和幅度分开建模，提升表达。
- **LoRA+**：对 A/B 使用不同学习率。
- **rsLoRA**：改缩放方式，让大 rank 更稳定。

调参重点：rank、alpha、target modules、dropout、学习率和数据质量。

### 1. LoRA 为什么省资源

全量微调会更新原模型的大量参数，显存和存储成本都高。LoRA 冻结原权重，只在部分线性层上加低秩增量矩阵，因此训练参数量很小。

常见会挂 LoRA 的位置：

- attention 里的 `q_proj`、`k_proj`、`v_proj`、`o_proj`
- MLP 里的 `up_proj`、`down_proj`、`gate_proj`

rank 越大表达能力越强，但参数量和过拟合风险也更高。

### 2. LoRA 变种怎么理解

- **QLoRA**：用 4bit 量化加载基座模型，大幅省显存，适合单卡/小资源微调。
- **AdaLoRA**：不同层动态分配 rank，把容量给更重要的层。
- **DoRA**：把权重更新拆成方向和幅度，提升低秩适配能力。
- **rsLoRA**：改善大 rank 时的缩放稳定性。

### 3. 面试总结

LoRA 的价值不是只省显存，而是让业务可以维护多个轻量 adapter，方便灰度、回滚和多场景切换。

### 4. 结合跨境支付项目怎么说（面试口径）

> 在跨境支付里，我会把 LoRA 当成垂直领域模型微调的首选。比如我们想让模型更懂风控规则、商户审核话术、拒付处理流程，没有必要全量微调大模型。全量微调成本高，而且一旦训坏回滚也麻烦。
>
> 用 LoRA 的好处是可以按业务场景维护 adapter：风控解释一个、客服问答一个、对账助手一个。底座模型保持稳定，业务层 adapter 可以小步迭代。某个场景效果不好，也可以单独回滚，不影响其他业务。
>
> 如果资源比较紧，我会考虑 QLoRA；如果数据量不大，我会控制 rank 和学习率，避免模型学到某些审核员的口吻，而不是学到真正的业务规则。

**钩子**：业务微调优先 LoRA/QLoRA，全量微调成本高且回滚不方便。
