# Transformer 各子层功能及去掉 FFN 的影响

## 答案

Transformer Block 主要由 Attention、FFN、Residual 和 Normalization 组成；Embedding、位置编码和输出层位于 Block 外部。Attention 与 FFN 的功能互补：

> Attention 在 token 之间混合信息，FFN 在每个 token 内进行特征变换。

### 1. 各子层作用

**Token Embedding**

将离散 token ID 映射为连续向量。

**Position Encoding/RoPE**

为 Attention 注入顺序和相对位置信息，否则 Self-Attention 对排列本身不敏感。

**Multi-Head Attention**

通过 Q/K 匹配并聚合 V，使每个位置读取其他位置的信息。多头在不同子空间学习实体、位置、指代和长距离关系。

**FFN/MLP**

对每个位置独立执行：

```text
FFN(x) = W_2 activation(W_1 x)
```

现代模型常用 SwiGLU：

```text
FFN(x) = W_down[
  SiLU(W_gate x) ⊙ (W_up x)
]
```

它扩大中间维度、增加非线性，并完成通道维特征组合。

**Residual Connection**

保留原表示并建立短梯度路径，使深层网络更容易优化。

**LayerNorm/RMSNorm**

稳定激活和梯度尺度。Pre-Norm 通常比原始 Post-Norm 更容易训练深层模型。

**Output Projection/LM Head**

将隐藏状态映射到词表 logits，常与输入 Embedding 权重共享。

### 2. 去掉 FFN 会发生什么

- 参数量和计算量明显下降；
- token 之间仍可通过 Attention 交换信息；
- 每个 token 内的非线性特征变换显著减弱；
- 模型表示容量、知识存储和任务性能通常下降；
- Attention 需要同时承担路由和特征变换，效率变差；
- 深层堆叠仍含 softmax 等非线性，但不能等价替代高容量 FFN。

许多 Transformer 中 FFN 参数占比很高，MoE 也主要扩展 FFN Experts，说明它是模型容量的重要载体。

### 3. 如何验证影响

在相同数据、参数预算和训练 FLOPs 下比较：

- 原始 Attention + FFN；
- 删除 FFN；
- 缩小 FFN；
- 用 GLU/MoE 或其他 mixer 替代；
- 通过增加层数补回相近参数。

评估 loss、下游任务、长上下文、吞吐和显存。直接删除 FFN 后参数量变小，若不控制预算，不能判断下降来自结构还是容量减少。

### 4. 跨境支付场景

Attention 可以关联订单、币种、错误码和通道规则，FFN 则把聚合后的信息变换为更抽象的业务表示。是否适合支付任务仍取决于训练数据和外部证据；模型结构不能替代实时工具、权限和规则校验。
