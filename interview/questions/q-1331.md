# 5.Transformer八股。讲讲自注意力机制。了解什么位置编码方式？

## 答案

Self-Attention 让序列中每个 token 根据当前内容动态读取其他 token。它解决 RNN 难并行和长距离依赖路径长的问题，但标准实现的注意力矩阵对序列长度是 `O(n^2)`。

### 1. Q、K、V

输入：

```text
X: [B, N, d_model]
```

线性投影：

```text
Q = X W_Q
K = X W_K
V = X W_V
```

多头后：

```text
Q,K,V: [B, h, N, d_head]
```

### 2. Scaled Dot-Product Attention

```text
S = Q K^T / sqrt(d_head) + Mask
A = softmax(S)
O = A V
```

除以 `sqrt(d_head)` 是因为点积方差会随维度增大，缩放可避免 softmax 饱和。Decoder 使用 causal mask；padding 和 packed sequence 还需要有效长度/边界约束。

多头结果拼接后：

```text
Y = Concat(head_1...head_h) W_O
```

不同 head 可学习局部、指代、语法和长距离关系。

### 3. 位置编码为什么需要

纯 Attention 对 token 顺序没有天然感知，因此需要注入位置：

**正弦绝对位置**

- 原始 Transformer；
- 无训练参数；
- 使用不同频率的 sin/cos。

**可学习绝对位置**

- BERT、早期 GPT 常见；
- 训练长度内灵活；
- 外推通常较弱。

**相对位置 Bias**

- T5、Transformer-XL 等；
- 根据 token 相对距离加 attention bias；
- 对距离模式更直接。

**RoPE**

- 旋转 Q/K，使内积包含相对位置信息；
- Llama、Qwen 等现代 LLM 常见；
- 可结合 NTK/YaRN 等缩放扩展长度。

**ALiBi**

- 对不同 head 添加随距离递增的线性 bias；
- 结构简单、长度外推较自然；
- 表达方式与 RoPE 不同。

### 4. 当前方向

- GQA/MQA/MLA 降低 KV Cache；
- FlashAttention 优化精确 Attention 的 IO；
- Sliding Window、Block Sparse 和混合注意力降低长序列成本；
- 线性注意力和状态空间模型探索亚二次复杂度。

### 5. 跨境支付

长合规文档可使用 RoPE 长上下文模型，但仍应先按权限、国家、通道和规则版本检索。位置编码解决“能处理长序列”，不保证模型会准确找到中间证据。
