# State是怎么初始化的？

## 答案

答：State用TypedDict定义字段结构，设置init函数，传入类对象，或业务规定的默认值，
给各属性赋值，完成State初始化。
初始化完成后，调用ainvoke(State)执行流程。
在运行过程中，每个节点基于当前State计算，返回新字典覆盖更新，不会直接修改原
State。

### 补充说明

LangGraph 里的 State 可以理解成一次图执行的工作内存。初始化时要把必要字段放进去，比如用户输入、会话 ID、当前意图、工具结果、检索结果、错误次数、是否转人工等。

好的 State 设计要满足三个要求：字段清晰、默认值明确、节点之间只读写自己负责的字段。这样后续排查时能知道每个节点对状态做了什么更新。

### 结合跨境支付项目怎么说（面试口径）

> 在跨境支付客服 Agent 里，我会把 State 初始化成一个结构化对象。比如包含 `user_query`、`merchant_id`、`order_id`、`intent`、`payment_status`、`risk_summary`、`channel_receipt`、`retrieved_docs`、`need_human`、`trace_id` 这些字段。
>
> 用户刚进来时，很多字段是空的。意图识别节点填 intent，订单查询节点填 payment_status，风控节点填 risk_summary，RAG 节点填 retrieved_docs，最后答案节点根据这些状态生成回复。
>
> 这样做的好处是流程可回放。如果一笔付款失败解释错了，可以看 State 里到底是订单状态查错、通道回执缺失，还是生成节点理解错。

**钩子**：可以引到“State 是工作内存、节点字段边界、trace_id 和流程回放”。
