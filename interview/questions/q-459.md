# 🔥 介绍一下RLHF流程，包括哪几步

## 答案

经典 RLHF 可以概括为 **SFT → 偏好数据与 Reward Model → PPO 策略优化**。预训练通常是前置阶段，不严格算 RLHF 本身。

### 1. SFT：监督微调

- 收集高质量指令—回答数据。
- 在预训练模型上做监督学习。
- 让模型先学会遵循指令、对话格式和基本任务行为。

SFT 模型通常同时作为后续 Policy 的初始模型和 Reference Model 的来源。

### 2. 收集人类偏好

对同一个 Prompt，让模型生成多个候选回答，由标注者：

- 两两比较 chosen / rejected；
- 或对多个答案排序；
- 按正确性、有用性、安全性等 Rubric 判断。

标注一致性和数据覆盖决定后续对齐上限。

### 3. 训练 Reward Model

Reward Model 输入 Prompt 和回答，输出一个标量分数。常见 Pairwise Loss：

```text
L_RM = -log sigmoid(r(x, y_chosen) - r(x, y_rejected))
```

目标是让 chosen 的分数高于 rejected。

### 4. PPO 优化 Policy

当前 Policy 生成回答，Reward Model 打分，再用 PPO 更新 Policy。目标通常包括：

```text
最终奖励 = RM 分数 - β × KL(Policy || Reference)
```

KL 项限制模型不要偏离 SFT 模型过远。训练中常同时维护：

- Policy / Actor
- Reference Model
- Reward Model
- Value / Critic

### 5. 评测与迭代

需要评估：

- 人类偏好胜率
- 事实正确和任务完成率
- 安全性与拒答行为
- 通用能力是否退化
- Reward Hacking 和长度偏置

上线 Bad Case 会继续回流，形成新的偏好数据。

### 6. 为什么现在也常用 DPO

DPO 直接用 chosen/rejected 优化 Policy，不显式训练 Reward Model 和 Critic，工程更简单。但 RLHF/PPO 仍适合需要在线采样、多步决策或复杂奖励的场景。

**面试总结**：RLHF 的核心不是 PPO，而是把人类偏好转成可优化信号，并用参考策略约束模型在提升偏好的同时保留原能力。
