# 什么是批量标准化（Batch Normalization, BN）？作用是什么？

## 答案

**一句话**：BN = **沿 batch 维度计算每个特征的均值方差并标准化**，外加可学习的缩放和偏移——目的是稳定训练、加速收敛、提升泛化。**在 LLM/NLP 里几乎被 LayerNorm 取代**。

### 1. BN 的公式
对一个 batch B = {x_1, ..., x_m} 的某个特征维度：
$$\mu_B = \frac{1}{m}\sum x_i, \quad \sigma_B^2 = \frac{1}{m}\sum(x_i - \mu_B)^2$$
$$\hat{x}_i = \frac{x_i - \mu_B}{\sqrt{\sigma_B^2 + \epsilon}}$$
$$y_i = \gamma \hat{x}_i + \beta$$
- γ、β 是可学习参数

### 2. 作用
- **加速收敛**：归一化让 loss surface 更平滑
- **缓解 internal covariate shift**：每层输入分布稳定
- **正则化效果**：batch 统计带噪声
- **允许更大学习率**
- **降低对初始化的敏感度**

### 3. 训练 vs 推理的差异
- **训练**：用当前 batch 的均值/方差
- **推理**：用训练时累积的**滑动平均**统计量
- 这是 BN 在 small batch / 推理时的痛点根源

### 4. BN 的痛点
- **依赖 batch 大小**：batch=1 / 2 时统计不准
- **序列任务不友好**：variable length、padding 干扰
- **训练/推理不一致**：滑动统计偏差
- **分布式训练**：跨节点同步 batch stats 复杂（SyncBN）

### 5. BN vs LayerNorm vs InstanceNorm vs GroupNorm
| 类型 | 沿哪个维度 | 适用 |
|------|-----------|------|
| **BN** | batch | CV / CNN |
| **LayerNorm** | feature | **NLP / Transformer** |
| **InstanceNorm** | 单样本通道 | Style Transfer |
| **GroupNorm** | 通道分组 | 小 batch CV |

### 6. 为什么 LLM 用 LayerNorm 不用 BN
- LLM 序列长度可变，BN 难处理
- 大模型 batch 通常小（甚至 1），BN 不准
- LayerNorm 每个 token 独立归一化，没 batch 依赖
- LayerNorm 训练/推理一致

### 7. RMSNorm 是 LN 的进一步简化
- 去掉 mean 中心化
- 只做 RMS scale
- 快 10-20%、效果几乎一样
- 现代 LLM 默认

### 8. 一句话总结
**BN 是 CV 时代的瑰宝，NLP/Transformer 时代被 LayerNorm/RMSNorm 取代**——理由是 batch 依赖性和序列任务不兼容。

## 结合项目谈（口语版 · 约 25 秒）

> 跨境支付里我们做过一些**多模态 KYB 识别**用到 BN——CLIP-ViT 的早期版本里 ViT 之前的 CNN backbone 有 BN，这些场景默认 BN 用得起来。
>
> 但 LLM 这条线我们**全部 LayerNorm / RMSNorm**，从来不用 BN。原因很直接——
>
> 第一是 **batch size**。我们 SFT 14B 模型时 micro-batch 经常是 1，BN 完全没法用，统计量根本不准；改 batch=1 时 BN 退化成 InstanceNorm，没意义。
>
> 第二是 **序列长度变化**。客服对话长度从 50 token 到 4k token 都有，packing 之后更复杂，BN 没法处理这种动态。
>
> 第三是 **训练/推理一致性**。BN 推理时用滑动均值，业务侧偶尔会出现"训练好的模型推理时和验证集不一样"——这种 debug 噩梦我们经历过一次，那次还是 CV 项目。
>
> 切到 LayerNorm（或现在的 RMSNorm）之后这些问题全消失，**每个 token 独立归一化、batch 大小 / 序列长度 / 训练推理都一致**，工程友好性碾压。
>
> 一句话：**BN 学一下理解原理就好**，工业 NLP/LLM 全是 LayerNorm/RMSNorm 的天下。
