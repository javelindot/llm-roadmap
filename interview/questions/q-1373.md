# encoder中的self-attention和decoder中的self-attention有什么区别

## 答案

Encoder self-attention 和 Decoder self-attention 都是在同一序列内部做注意力，但核心区别是 mask 和使用方式不同。

### 1. Encoder 中的 self-attention

Encoder 通常用于理解输入序列，例如 BERT、Transformer Encoder、检索模型。它的 self-attention 是双向的：

```text
第 i 个 token 可以看见序列中所有非 padding token
```

所以它没有 causal mask，只有 padding mask。这样每个位置都能同时利用左边和右边上下文。

例如输入：

```text
这笔 USD 出款为什么 pending
```

“pending”可以关注前面的“USD 出款”，“出款”也可以关注后面的“pending”。

### 2. Decoder 中的 self-attention

Decoder 用于自回归生成，例如 GPT、LLaMA、Qwen。它的 self-attention 带 causal mask：

```text
第 i 个 token 只能看见 1..i 的 token
不能看见未来 token
```

否则训练时会偷看答案，推理时也不符合逐 token 生成过程。

### 3. 训练和推理差异

训练时：

- Encoder 可以整句并行理解；
- Decoder 也可以并行计算所有位置，但通过 causal mask 遮住未来 token。

推理时：

- Encoder 通常一次性编码输入；
- Decoder 逐 token 生成，并使用 KV cache 复用历史 K/V。

### 4. 和 cross-attention 的区别

在原始 Encoder-Decoder Transformer 中，Decoder 里还有一层 cross-attention：

```text
Decoder hidden states 作为 Q
Encoder outputs 作为 K/V
```

它不是 decoder self-attention。self-attention 看的是 decoder 已生成的 token，cross-attention 看的是 encoder 输入内容。

### 5. 面试总结

一句话：Encoder self-attention 是双向理解，通常只遮 padding；Decoder self-attention 是自回归生成，必须用 causal mask 遮未来 token，并在推理时配合 KV cache。BERT 属于前者，GPT/Qwen/LLaMA 属于后者。
