# triton学过吗，传统方法部署模型的流程是怎么样的

## 答案

“Triton”有两个常见含义，应先确认面试官问的是 OpenAI Triton 编程语言，还是 NVIDIA Triton Inference Server。

### 1. OpenAI Triton Language

它是用 Python 风格 DSL 编写自定义 GPU kernel 的工具。开发者显式设计 block、内存访问、并行和 autotune，编译器生成 GPU 代码。适合：

- 融合 softmax、norm、activation，减少 HBM 往返和 kernel launch；
- 自定义量化、MoE、Attention 和特殊 shape 算子；
- 用 block tiling、共享/片上存储与访存重排提高性能。

实现流程是 PyTorch 基线 → profiler 找瓶颈 → Triton kernel → 与参考实现做精度/梯度测试 → 多 shape autotune → benchmark。官方[矩阵乘教程](https://triton-lang.org/main/getting-started/tutorials/03-matrix-multiplication.html)展示了 block GEMM、指针计算和 autotune。

### 2. NVIDIA Triton Inference Server

它是模型服务系统，支持 TensorRT、ONNX Runtime、PyTorch/Python backend 等，通过 model repository 和 `config.pbtxt` 配置版本、输入输出、实例与调度。能力包括动态 batching、多模型并发、sequence batching、ensemble、HTTP/gRPC、指标和健康检查。

### 3. 传统模型部署流程

1. 固化预处理、模型与后处理；
2. 导出 TorchScript/ONNX/SavedModel 等；
3. 验证数值一致性和动态 shape；
4. 用 TensorRT/ONNX Runtime 优化、量化和融合；
5. 封装 HTTP/gRPC 服务；
6. 配置 batching、并发、超时和资源；
7. 容器化并部署到 Kubernetes；
8. 压测 P50/P99、吞吐、显存和精度；
9. 灰度、监控、版本回滚和数据漂移告警。

### 面试总结

OpenAI Triton 用于写高性能 GPU kernel，NVIDIA Triton 用于模型服务。传统部署是导出、优化、服务化、压测、灰度与监控的完整流程，不能只停在 ONNX 转换。
