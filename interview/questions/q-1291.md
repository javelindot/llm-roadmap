# 这个AI平台商用了吗？基座模型用的什么？介绍一下整个平台的架构？

## 答案

平台已经用于内部生产场景，包括商户客服辅助、支付异常解释、知识问答和文档处理；“商用”不表示模型直接控制付款、换汇或账户处置。资金与合规动作仍由业务系统、规则、权限和人工审批完成。

### 1. 基座模型选型

平台不绑定单一模型，采用 Model Gateway 统一接入：

- Qwen/Llama 系列本地模型承担数据敏感、稳定高频任务；
- 小模型负责分类、Query Rewrite、字段抽取；
- 较强模型负责复杂多文档推理和报告生成；
- Embedding、Reranker、OCR/VLM 是独立模型服务；
- 必要时接入外部模型，但只能处理经过审批和脱敏的数据。

模型版本按业务黄金集选择，而不是固定认为某个模型最好。评估包括：

- 知识问答和引用；
- 工具调用、JSON schema；
- 中英文支付术语；
- 金额、币种、日期；
- 安全拒答；
- P95/P99、吞吐和成本。

### 2. 平台分层架构

```text
业务接入层
-> AI Gateway
-> Agent / RAG / Workflow
-> Model Gateway
-> Model Serving

          |
          +-> Tool Gateway
          +-> Knowledge Platform
          +-> Evaluation & Observability
          +-> Security & Governance
```

**业务接入层**

- 客服工作台、运营后台、商户门户和内部 API；
- 不让各业务直接调用底层模型。

**AI Gateway**

- 认证、租户、限流、配额；
- Prompt 和模型路由；
- trace_id、审计、缓存；
- 超时、重试、降级和灰度。

**Agent / Workflow**

- LangGraph 或状态机；
- 意图识别、任务分解、工具编排；
- 对确定性流程使用显式节点；
- 高风险节点进入 HITL。

**RAG 与知识平台**

- 文档解析、切片、去重和版本；
- Elasticsearch/BM25 + 向量检索；
- metadata filter、rerank、引用；
- 国家、币种、产品、生效时间和权限过滤。

**Tool Gateway**

- 对接订单、账务、通道、风控、工单；
- schema 校验、最小权限、幂等；
- 读写工具分离；
- 写操作不由自由文本直接触发。

**Model Serving**

- vLLM 等推理引擎；
- continuous batching、KV Cache、量化；
- 多模型路由、弹性副本；
- GPU 实时与离线任务隔离。

### 3. 数据与训练层

- 知识文档进入 RAG，不把经常变化的规则硬记进模型；
- 历史工单经脱敏后用于 SFT、评测和错误分析；
- 实时订单、余额、汇率只通过授权工具查询；
- Hugging Face + PEFT + DeepSpeed 负责领域微调；
- 模型、数据、Prompt、索引和评测集统一版本化。

### 4. 生产治理

- PII、账户和交易字段脱敏；
- 租户与行列权限；
- Prompt injection 和工具参数校验；
- 全链路 audit log；
- 离线回归、shadow、灰度、A/B；
- 模型和知识版本一键回滚；
- 低置信、越权、工具异常时转人工。

### 5. 商用验收

上线门槛不仅是回答准确率：

- 任务完成率和人工改写率；
- 工具参数及执行成功率；
- 引用和事实一致性；
- 敏感信息、越权和错误资金建议；
- P95/P99、错误率和容量；
- 单个成功任务成本；
- 业务连续性、审计和回滚。

平台的核心价值是把模型能力标准化、安全地提供给多个业务，而不是为每个场景重复搭建一个不可治理的聊天 Demo。
