# 🔥 介绍一下function calling和MCP

## 答案

**一句话**：Function Calling 是模型按 schema 调用具体工具；MCP 更像统一的工具/资源接入协议，让模型应用以标准方式连接外部系统。

### 1. Function Calling

开发者定义工具名称、描述和参数 schema。模型判断需要工具时，输出工具名和参数，业务系统校验后执行工具，再把结果返回给模型生成答案。

重点是：模型不直接执行业务，只提出调用意图，真正执行的是后端。

### 2. MCP

MCP 可以理解成模型上下文协议，把外部工具、资源、文件、数据库等能力标准化暴露给模型应用。它解决的是不同工具接入方式不统一的问题。

### 3. 区别

- Function Calling：一次模型到工具的调用机制。
- MCP：更通用的工具和上下文接入协议。
- Function Calling 偏模型 API 能力。
- MCP 偏生态和标准化连接。

### 4. 结合跨境支付项目怎么说（面试口径）

> 在跨境支付客服项目里，Function Calling 可以用来查订单、查通道回执、查风控大类、查知识库。比如用户问“这笔付款为什么失败”，模型生成查询订单工具的参数，后端校验商户权限后执行。
>
> MCP 则可以把这些内部能力标准化接入，比如订单系统、风控系统、知识库、工单系统都作为受控工具或资源暴露给 Agent。
>
> 但不管是 Function Calling 还是 MCP，资金类动作都不能直接放开。查询可以开放，退款、解冻、重新付款必须经过权限、审批和审计。

**钩子**：可以引到“模型只生成调用意图、后端执行校验、MCP 标准化工具接入、资金操作保护”。
