# 把自己实习模型的结构和训练流程在ide里用伪代码实现

## 答案

这类题主要考察候选人是否真的做过项目：能不能把数据输入、模型结构、loss、训练循环、评测和保存流程写出来。回答时必须按自己的真实项目改参数，不能背一段与项目无关的代码。

下面以“跨境支付工单意图分类模型”为例。模型使用预训练 Encoder，加分类头；如果实际项目是生成式大模型或多模态模型，可以替换 backbone 和 loss，但训练主流程类似。

### 1. 数据和模型结构

```python
class PaymentTicketDataset:
    def __init__(self, samples, tokenizer, max_length=512):
        self.samples = samples
        self.tokenizer = tokenizer
        self.max_length = max_length

    def __getitem__(self, index):
        sample = self.samples[index]
        encoded = self.tokenizer(
            sample["text"],
            max_length=self.max_length,
            truncation=True,
            padding="max_length"
        )
        return {
            "input_ids": tensor(encoded["input_ids"]),
            "attention_mask": tensor(encoded["attention_mask"]),
            "label": tensor(sample["label"])
        }


class PaymentIntentModel(Module):
    def __init__(self, pretrained_model, hidden_size, num_labels):
        super().__init__()
        self.encoder = load_pretrained_encoder(pretrained_model)
        self.dropout = Dropout(0.1)
        self.classifier = Linear(hidden_size, num_labels)

    def forward(self, input_ids, attention_mask, labels=None):
        outputs = self.encoder(
            input_ids=input_ids,
            attention_mask=attention_mask
        )
        pooled = outputs.last_hidden_state[:, 0]  # [CLS]
        logits = self.classifier(self.dropout(pooled))

        loss = None
        if labels is not None:
            loss = cross_entropy(logits, labels)

        return {"loss": loss, "logits": logits}
```

标签可以是：

### 2. 训练流程

```python
model = PaymentIntentModel(
    pretrained_model="bert-like-model",
    hidden_size=768,
    num_labels=NUM_LABELS
).to(device)

optimizer = AdamW(
    model.parameters(),
    lr=2e-5,
    weight_decay=0.01
)

scheduler = LinearWarmupScheduler(
    optimizer,
    warmup_ratio=0.1,
    total_steps=len(train_loader) * epochs
)

best_f1 = 0.0

for epoch in range(epochs):
    model.train()
    optimizer.zero_grad()

    for step, batch in enumerate(train_loader):
        batch = move_to_device(batch, device)

        with autocast():
            output = model(
                input_ids=batch["input_ids"],
                attention_mask=batch["attention_mask"],
                labels=batch["label"]
            )
            loss = output["loss"] / grad_accum_steps

        scaler.scale(loss).backward()

        if (step + 1) % grad_accum_steps == 0:
            scaler.unscale_(optimizer)
            clip_grad_norm_(model.parameters(), 1.0)
            scaler.step(optimizer)
            scaler.update()
            scheduler.step()
            optimizer.zero_grad()

    metrics = evaluate(model, valid_loader)

    if metrics["macro_f1"] > best_f1:
        best_f1 = metrics["macro_f1"]
        save_checkpoint(model, optimizer, scheduler)
```

### 3. 评测流程

```python
def evaluate(model, loader):
    model.eval()
    predictions = []
    labels = []

    with no_grad():
        for batch in loader:
            output = model(
                input_ids=batch["input_ids"].to(device),
                attention_mask=batch["attention_mask"].to(device)
            )
            predictions.extend(argmax(output["logits"], dim=-1))
            labels.extend(batch["label"])

    return {
        "accuracy": accuracy(predictions, labels),
        "macro_f1": macro_f1(predictions, labels),
        "confusion_matrix": confusion_matrix(predictions, labels)
    }
```

支付业务不能只看 accuracy，因为 `chargeback`、`kyc_review` 等少数类可能更重要。应同时看 macro-F1、关键类别 recall、误分成本和分国家/语言表现。

### 5. 面试总结

手写伪代码时要覆盖五件事：Dataset、模型 forward、loss、optimizer/scheduler、train/eval loop。跨境支付项目还要补充数据脱敏、类别不均衡、关键风险类 recall 和线上灰度指标。最重要的是代码和自己真实做过的模型结构一致。
