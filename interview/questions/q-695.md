# 如果DPO训练的过程中发现loss不降，如何定位原因？从数据构造和训练参数两个角度分析。

## 答案

DPO loss 不下降时，先确认实现和指标是否正确。参考策略与初始策略相同时，常见 DPO loss 初始值约为 $\log 2$；loss 不是越接近 0 越好，还要看 chosen/rejected reward margin 和验证集偏好准确率。

### 数据构造排查

- 检查 `prompt/chosen/rejected` 是否同题配对，是否有字段颠倒、空答案、截断或 chat template 错位。
- chosen 与 rejected 若几乎相同，偏好信号太弱；若差异只在长度或格式，模型会学习捷径。
- 去重并按 prompt 划分数据集，防止泄漏。
- 检查标注一致率、领域和长度分布，人工抽查高 loss 样本。
- 确保 completion 部分参与 loss，而 prompt token 按实现正确 mask。
- 统计初始模型对 pair 的 log-probability margin；若多数 chosen 本就远高于 rejected，梯度可能很小。

### 训练参数排查

- 学习率过小几乎不更新，过大则 margin 振荡或梯度异常；先在小数据上验证能否过拟合。
- 检查 optimizer 是否拿到可训练参数，LoRA target module、`requires_grad` 和梯度范数是否正常。
- $\beta$ 控制相对参考模型的偏好尺度。过小信号弱，过大可能饱和，应结合 margin 与 KL 调节。
- 检查有效 batch、gradient accumulation、warmup、clip、bf16/fp16 和 scheduler。
- policy 与 reference 必须使用同一 tokenizer、模板和序列处理，reference 应冻结且数值稳定。

### 定位流程

固定 32 到 100 个高质量 pair，关闭复杂分布式与数据混合，观察单 batch loss、chosen/rejected log-prob、reward accuracy 和梯度。若小样本都不能下降，优先查实现；若能过拟合但全量不降，重点查数据噪声、配比和超参数。最终用 held-out 人工偏好验证，防止只优化训练 loss。
