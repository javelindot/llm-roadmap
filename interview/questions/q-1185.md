# Dropout在训练和推理时的实现方式及区别

## 答案

Dropout 在训练时随机屏蔽部分神经元，减少特征共适应和过拟合；推理时通常关闭随机屏蔽，使用完整网络。

### 1. 训练阶段

设丢弃概率为 `p`，保留概率为 `1-p`。现代框架通常采用 Inverted Dropout：

```text
m_i ~ Bernoulli(1 - p)
y_i = x_i × m_i / (1 - p)
```

被丢弃位置输出 0，保留位置除以 `1-p`。因此：

```text
E[y_i] = x_i
```

训练输出的期望与推理输出一致，无需在部署时再缩放。

例如 `p=0.2` 时，训练阶段约 20% 的元素被置零，其余元素乘以 `1/0.8=1.25`。

### 2. 推理阶段

标准推理中：

```text
y = x
```

不再采样 mask，也不做额外缩放。推理结果因此更稳定，且使用训练期间形成的多个随机子网络的近似集成效果。

早期也可以训练时不缩放、推理时乘 `1-p`，但 PyTorch、TensorFlow 等主流实现采用 Inverted Dropout。

### 3. 框架中的关键区别

PyTorch 中：

```python
model.train()  # Dropout 随机置零
model.eval()   # Dropout 变为恒等映射
```

两个常见误区：

- `torch.no_grad()` 只关闭梯度记录，不会自动关闭 Dropout；
- `model.eval()` 会切换 Dropout 和 BatchNorm 行为，但不会自动关闭梯度。

标准推理通常同时使用 `model.eval()` 和 `torch.inference_mode()`。

### 4. Transformer 中的 Dropout

常见位置包括：

- Embedding 或位置编码之后；
- Attention 权重；
- Attention 输出投影；
- FFN 中间层或输出；
- 残差分支。

Attention Dropout 随机屏蔽的是注意力连接，不是直接删除 token。大规模预训练模型的数据量和正则来源较多，有些架构会使用很小的 Dropout，甚至设为 0；下游小数据微调时再根据过拟合情况调整。

### 5. MC Dropout

如果需要近似估计不确定性，可以在推理时有意保持 Dropout 开启，多次前向：

```text
ŷ = mean(f_1(x), ..., f_T(x))
uncertainty = variance(f_1(x), ..., f_T(x))
```

这叫 Monte Carlo Dropout。它不是普通线上推理默认行为，会增加多次计算成本，也不能替代严格的概率校准。

### 6. 工程检查

- 验证训练与评估模式是否正确切换；
- 固定随机种子只能帮助复现，不代表每个 GPU kernel 都完全确定；
- 检查分布式训练中各 rank 的随机状态；
- 对分类概率评估 ECE/Brier，而不是认为 Dropout 自动带来校准；
- 导出 ONNX/TensorRT 前确认 Dropout 节点已按推理语义消除。

跨境支付的意图分类或风险辅助模型中，Dropout 可降低小样本过拟合，但线上概率仍需按国家、币种和产品分桶校准；资金决策不能把 MC Dropout 方差直接当成唯一风险阈值。
