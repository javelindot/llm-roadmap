# 接触过哪些推理加速的方法。(vllm的page-attention, kv cache, prefix cachemla，flash-attention直接安排一套)

## 答案

推理加速应按模型、算子、KV Cache、调度、并行和业务输入分层，并区分 TTFT、逐 token 延迟和吞吐。

### 1. KV Cache

自回归 decode 缓存历史 K/V，避免每步重算全部历史。代价是显存随层数、KV head、序列和并发线性增长。

- GQA/MQA 减少 KV head；
- MLA 缓存低维 KV latent；
- KV INT8/FP8 量化降低字节；
- Sliding Window 限制缓存长度。

### 2. PagedAttention

vLLM 将 KV 切成固定 block，用 block table 管理非连续显存，减少预留和碎片，并支持前缀共享。它不改变 Attention 数学和有效 KV 数据量。

### 3. Prefix Cache

对相同 system prompt、工具 schema 或共享文档前缀复用 prefill KV，减少 TTFT。缓存键必须包含模型、adapter、token 和权限版本。

### 4. FlashAttention

通过分块和 online softmax 减少 HBM 往返，不显式存完整 Attention 矩阵。它是精确 Attention kernel，降低显存和时间，但稠密算术仍为 \(O(n^2)\)。

### 5. 其他

- continuous batching、chunked prefill；
- speculative decoding；
- FP8/INT8/INT4 权重量化；
- 算子融合、CUDA Graph、编译；
- Tensor/Expert/Context Parallel；
- prefill/decode 分离；
- 蒸馏和小模型路由；
- RAG、摘要和短输出减少 token。

### 如何组合

例如 Qwen/Llama GQA 模型 + FlashAttention + vLLM Paged KV + prefix cache + continuous batching；高接受率场景再加 speculative decoding。MLA 需要模型架构原生支持，不能给普通模型靠配置直接开启。

### 面试总结

KV Cache 避免重算，GQA/MLA 减少缓存，FlashAttention 优化单次 Attention IO，PagedAttention 优化在线缓存分配，prefix cache 复用公共 prefill。它们解决不同层次问题，可组合但不可互相替代。
