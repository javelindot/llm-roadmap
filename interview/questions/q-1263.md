# 四塔多目标方案相较经典双塔在 CTR/跨类目召回上带来哪些可量化收益？

## 答案

“四塔”不是统一标准结构，必须先说明四个塔各自编码什么。一种常见多目标设计是为点击与转化分别建立 User/Item 表示：

```text
User-CTR tower    ↔ Item-CTR tower
User-CVR tower    ↔ Item-CVR tower
```

底层 embedding 或部分专家可以共享，任务顶部保持独立；也可以把长期兴趣、短期兴趣、Item 和 Category 作为四塔。结构含义不同，收益不能直接横比。

### 1. 相较经典双塔可能提升什么

经典双塔用一套 User/Item 向量优化单一相似度，容易被高频点击目标主导。四塔多目标可以：

- 分离点击兴趣与深层转化意图；
- 共享相关信号，同时减少目标冲突；
- 为跨类目兴趣保留独立表示；
- 对稀疏目标利用高频目标迁移；
- 按目标召回多路候选再融合。

但参数、索引和在线召回链路会增加，目标相关性差时还可能产生负迁移。

### 2. 可量化收益不能预设

不能脱离数据写“CTR 提升 5%”。正确做法是与同样本、同特征、同训练预算的双塔比较：

**离线召回**

- Recall@K、HitRate@K、MRR/NDCG；
- 跨类目 Recall@K；
- 长尾和新类目覆盖；
- 候选重合率与独占贡献；
- 向量索引大小和召回延迟。

**在线业务**

- CTR、长点击率；
- CVR、GMV 或任务完成率；
- 跨类目点击/转化率；
- 新类目渗透、覆盖率和多样性；
- 用户留存、负反馈；
- P95 延迟和计算成本。

报告收益时给出相对提升、置信区间、实验流量和周期，而不是只给离线 AUC。

### 3. 归因实验

至少做以下消融：

1. 双塔单目标；
2. 双塔多目标共享底座；
3. 四塔但不共享；
4. 四塔 + shared experts/gating；
5. 去掉跨类目序列特征；
6. 单路召回与多路融合。

这样才能判断收益来自塔数、多目标监督、额外特征还是候选融合，而不是把所有变化归功于“四塔”。

### 4. 负迁移与样本偏差

- 点击多、转化少，loss 尺度差异大；
- CVR 只在点击后可观测，存在样本选择偏差；
- 热门类目会主导梯度；
- 多塔表示可能互相失去可比性；
- 跨类目候选增加后，排序层不一定能正确利用。

可使用 loss weighting、uncertainty weighting、MMoE/PLE、分任务采样和分塔蒸馏，并单独校准各路分数。

### 5. 跨境支付中的可迁移思路

类似多目标结构可用于“商户与产品/通道匹配”，同时建模成功率、成本、时效和风险，但这不是电商 CTR 项目经历。支付路由最终还必须满足币种、国家、牌照、限额和合规硬约束，模型分数不能覆盖规则门禁。
