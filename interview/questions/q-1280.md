# clip训练方式？temperature的作用

## 答案

CLIP 使用大规模图文配对数据做对比学习：图像编码器和文本编码器分别生成向量，将同一条样本中的图文作为正样本，batch 内其他组合视为负样本，在共享空间中拉近正对、推远负对。

### 1. 训练流程

一个 batch 有 `N` 对图文：

```text
image_features = image_encoder(images)
text_features  = text_encoder(texts)

I = L2Normalize(image_projection(image_features))
T = L2Normalize(text_projection(text_features))
```

归一化后点积等价于 cosine similarity：

```text
similarity = I @ T^T
```

再用可学习温度构造 logits：

```text
logits_per_image = similarity / tau
logits_per_text  = logits_per_image^T
```

原始 CLIP 实现常写成：

```text
logit_scale = exp(s)
logits = logit_scale * similarity
```

其中 `s` 是可学习参数，初始值对应 `tau ≈ 0.07`。工程实现通常会限制 `logit_scale`，避免训练中指数增长造成数值不稳定。

正样本位于相似度矩阵对角线：

```text
labels = [0, 1, 2, ..., N - 1]
```

损失同时计算 image-to-text 和 text-to-image 两个方向的交叉熵。

### 2. Temperature 的作用

Softmax 概率为：

```text
p(j | i) = exp(sim(i,j) / tau)
           / sum_k exp(sim(i,k) / tau)
```

**较小的 `tau`**

- logits 差距被放大；
- 分布更尖锐；
- 梯度更集中在 hard negatives；
- 区分性可能更强；
- 过小会导致梯度不稳定、过度关注噪声负样本。

**较大的 `tau`**

- 分布更平滑；
- 更多负样本参与梯度；
- 训练较稳定；
- 过大时正负样本差异被压平，判别能力变弱。

温度本质上控制对比学习任务的难度和梯度分配。让模型学习温度，比固定一个值更能适应数据和 batch，但仍需监控其范围。

### 3. Batch 和分布式训练

CLIP 将 batch 内其他样本作为负样本，因此有效 batch 越大，负样本越丰富。多卡训练通常 AllGather 各卡的图像和文本 embedding，再构造全局相似度矩阵；标签索引必须加入 rank 偏移，否则正样本位置会错。

还要处理：

- 重复图片或重复文案造成 false negative；
- 图文描述过短、错配和噪声；
- 数据语言、国家和品类分布；
- 大 batch 下学习率和温度稳定性；
- embedding gather 的通信和显存开销。

### 4. 训练后如何使用

CLIP 可以零样本分类：

1. 把类别写成 Prompt；
2. 编码所有类别文本；
3. 编码待测图像；
4. 计算相似度并做 softmax。

也可以用于图搜文、文搜图、多模态召回或作为下游模型特征。跨境支付中可用于凭证、账单、KYC 材料和错误截图的辅助检索，但不能仅凭相似度作合规或资金决策。

### 5. 评估

- image-to-text / text-to-image Recall@K；
- 零样本分类准确率；
- 分语言、分文档类型和长尾覆盖；
- embedding uniformity、重复与 false negative；
- 检索 P95/P99、向量存储和 ANN 成本；
- 敏感材料的租户隔离与误召回。
