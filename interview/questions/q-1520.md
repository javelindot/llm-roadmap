# 实现一个包含环检测功能的链表反转的代码，如果有环，返回nullptr，如果无环返回反转后的head node

## 答案

可以先用快慢指针检测环；确认无环后再原地反转。两遍扫描仍是 `O(n)` 时间、`O(1)` 额外空间，而且不会在有环链表上破坏结构。

### 1. C++ 实现

```cpp
struct ListNode {
    int val;
    ListNode* next;
    explicit ListNode(int x) : val(x), next(nullptr) {}
};

ListNode* reverseIfAcyclic(ListNode* head) {
    // Floyd 环检测
    ListNode* slow = head;
    ListNode* fast = head;

    while (fast != nullptr && fast->next != nullptr) {
        slow = slow->next;
        fast = fast->next->next;
        if (slow == fast) {
            return nullptr;
        }
    }

    // 无环，原地反转
    ListNode* prev = nullptr;
    ListNode* curr = head;

    while (curr != nullptr) {
        ListNode* next = curr->next;
        curr->next = prev;
        prev = curr;
        curr = next;
    }

    return prev;
}
```

### 2. 正确性

- 若有环，快指针每次走两步、慢指针走一步，最终会在环内相遇；
- 若快指针到达 `nullptr`，说明链表无环；
- 反转时维护不变量：`prev` 指向已反转部分的头，`curr` 指向未处理部分；
- 每次先保存 `next`，再修改 `curr->next`，避免丢失后续链表。

### 3. 复杂度

```text
时间：O(n)
空间：O(1)
```

虽然检测与反转各遍历一次，但 `O(2n)` 仍是 `O(n)`。

### 4. 边界条件

- 空链表返回 `nullptr`；
- 单节点无环返回自身；
- 单节点自环返回 `nullptr`；
- 头节点在环内或尾部形成环都能检测。

题目接口用 `nullptr` 同时表示“空链表反转结果”和“检测到环”，调用方无法区分两种情况。实际工程更适合返回 `std::pair<bool, ListNode*>` 或状态码。

### 5. 面试总结

先检测、后反转是最清楚且安全的实现。关键是 Floyd 判环条件、保存 next 指针，以及指出接口对空链表与有环状态存在歧义。
