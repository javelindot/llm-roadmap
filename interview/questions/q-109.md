# 人工智能、机器学习、深度学习的关系是什么？

## 答案

**一句话**：三者是**层层包含**关系——**AI ⊃ ML ⊃ DL**，AI 是终极目标、ML 是实现 AI 的主流路径、DL 是 ML 里基于神经网络的子分支。

### 1. 三层定义

**人工智能（AI）**
- 目标：让机器具备智能行为（推理、感知、决策）
- 历史：1956 达特茅斯会议起源
- 范围最大：包括符号 AI、专家系统、ML、DL、Agent、LLM

**机器学习（ML）**
- 方式：让机器从数据中学习规律
- 算法：决策树、SVM、Naive Bayes、KNN、Random Forest、XGBoost、神经网络
- 思想：用数据 + 算法替代手写规则

**深度学习（DL）**
- 方法：多层神经网络（深网）
- 核心：自动特征学习
- 典型：CNN、RNN、Transformer
- 是 ML 的一个子集

### 2. 关系图
```
                  AI
                  ↓
   ┌──────────────┼──────────────┐
   ↓              ↓              ↓
 符号 AI       机器学习         其他
                  ↓
   ┌──────────────┼──────────────┐
   ↓              ↓              ↓
 传统 ML       深度学习         强化学习
                  ↓
              CNN / RNN / Transformer / LLM
```

### 3. 时代演进
| 时代 | 代表 | 特点 |
|------|------|------|
| 1950s-80s | 符号 AI、专家系统 | 规则驱动 |
| 1990s-2010s | 传统 ML（SVM、GBDT）| 特征工程主导 |
| 2012-2018 | DL（AlexNet → ResNet）| 自动特征 |
| 2017+ | Transformer | 序列建模革命 |
| 2020+ | LLM | 通用智能涌现 |
| 2024+ | LLM Agent / 多模态 | 自治系统 |

### 4. 三者的边界（容易混淆）
- "规则引擎"是 AI 但不是 ML
- "线性回归"是 ML 但不是 DL
- "ChatGPT"是 AI 也是 ML 也是 DL 也是 LLM
- "Agent"基于 LLM，是 AI 的一种工程形态

### 5. 实战选择
- 任务**有明确规则** → 规则引擎（非 ML）
- 任务**有结构化特征 + 中等数据** → 传统 ML（XGBoost、LightGBM）
- 任务**有大规模非结构化数据** → DL（CV/NLP）
- 任务**需要通用理解/对话/推理** → LLM

### 6. 一句话总结
**AI 是目标，ML 是路径，DL 是当前最有效的子路径，LLM 是 DL 在 NLP 上的极致体现**——理解这条链路有助于在业务里选对工具。

## 结合项目谈（口语版 · 约 25 秒）

> 这道题工业落地我会用一个**反向案例**讲——很多人觉得"上 AI 就是上 LLM"，其实**最有效的方案常常是混合**。
>
> 我们跨境支付有三层方案，对应三个层级：
>
> **规则引擎层（AI 的最早形式）**——风控里的"金额>10万 + 跨境 + 新商户"这种硬规则，**绝对不能用 LLM 替代**。规则简单、合规可解释、推理快。一年拦截 90% 的明显风险靠规则。
>
> **传统 ML 层**——商户违约预测、客户流失预测这种结构化任务，**XGBoost 仍然打 LLM**。10w 行客户特征、几十维数值，XGB 跑 1 分钟 AUC 0.92；上 LLM 反而慢、贵、效果不一定好。
>
> **DL / LLM 层**——客服对话、风控调查推理、KYB 证件识别这种非结构化、需要语义理解的任务，必须 DL/LLM。
>
> 我跟新人介绍这三层时会说："**AI 不是越深越好，是越合适越好**"——能用规则解决的不上 ML，能用传统 ML 的不上 DL，能用小 DL 的不上 LLM。
>
> 这道题面试要会讲三者关系，但**面试官真正想听的是"你知道不同层级各自的适用场景"**——这才是工程师视角，不是学生背概念。

**🪝 顺嘴带出去的钩子**：
- "我们的混合架构（规则 + XGB + LLM）选型逻辑可以聊。"
- "什么时候上 LLM、什么时候不上是个 ROI 决策可以展开。"
- "传统 ML 在金融场景为什么还活着可以讲讲。"
