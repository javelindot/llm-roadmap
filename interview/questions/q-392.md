# 结构化调度，主Agent只输出结构化指令，子Agent只接收固定格式。边界

## 答案

由代码锁死，不可能越权；

### 补充说明

结构化调度的核心是让主 Agent 不直接自由发挥，而是输出固定格式的指令，比如：

```json
{
  "target_agent": "risk_agent",
  "task_type": "analyze_risk_reason",
  "input": {
    "transaction_id": "...",
    "risk_code": "..."
  }
}
```

子 Agent 只接收 schema 允许的字段，只执行绑定工具。调度器在代码层校验目标 Agent、任务类型、字段、权限和工具白名单，校验不通过就拒绝执行。

### 结合跨境支付项目怎么说（面试口径）

> 在跨境支付多 Agent 里，我会让主 Agent 只输出结构化调度指令，不能直接拼自然语言让子 Agent 猜。比如要分析风控原因，只能发给风控 Agent 一个固定 schema，里面包含交易 ID、风险大类和必要上下文，不能把完整用户聊天和所有订单信息都传过去。
>
> 子 Agent 也只接收固定格式，工具由代码绑定。风控 Agent 没有退款工具，客服 Agent 没有解除风控工具，所以即使模型输出越权意图，调度器也会拒绝。
>
> 这种边界不是靠 prompt 约束，而是代码和 schema 锁死。支付场景涉及资金和敏感数据，必须这样做。

**钩子**：可以引到“结构化 JSON 指令、schema 校验、工具白名单、边界靠代码锁死”。
