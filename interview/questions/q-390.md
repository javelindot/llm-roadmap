# 介绍一下State。

## 答案

答：（1）State是LangGraph的传递状态。包含的内容有：
追踪器Tracker、Flow列表、领域（包括槽位和固定响应）、
输入输出数据、中间结果缓存、
最大动作次数限制、当前动作次数、节点执行信息等等。
（2）State仅在一次图流程中有效，是临时变量，不需要序列化。
（3）State是字典对象，其中一个元素是Tracker

### 补充说明

State 可以理解成 LangGraph 单次执行里的共享数据对象，节点通过读取 State 做判断，再返回部分字段更新 State。它通常包括输入、业务上下文、工具结果、检索结果、错误信息、当前步骤和控制变量。

State 和 Tracker 的区别可以这样讲：State 是本次图执行的临时工作内存，Tracker 更偏跨轮会话追踪和持久化信息。State 里可以引用 Tracker，但两者生命周期不同。

### 结合跨境支付项目怎么说（面试口径）

> 在跨境支付客服 Agent 里，State 会保存一次问题处理过程中的信息，比如用户问题、订单号、交易状态、通道回执、风控大类、检索到的 SOP、是否需要转人工、当前节点执行次数。
>
> Tracker 则保存更长期的会话信息，比如当前用户正在咨询哪笔订单、已确认哪些槽位、历史摘要。每次图流程开始时，可以从 Tracker 初始化 State，流程结束后再把必要结果回写 Tracker。
>
> 这样设计能让单次流程清晰，也能支持多轮对话连续性。

**钩子**：可以引到“State 临时工作内存、Tracker 跨轮追踪、流程结束回写”。
