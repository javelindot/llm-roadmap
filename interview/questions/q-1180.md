# 多目标模型的优缺点

## 答案

多目标模型在一个模型中联合预测多个相关目标，例如推荐中的点击、收藏、下单、支付成功、退款和长期价值。它通过共享表示提高数据利用率，但也会引入任务冲突和目标权衡问题。

### 1. 优点

- **共享统计强度**：高频任务为低频任务提供表示，缓解收藏、转化等标签稀疏；
- **减少重复计算**：共享 Embedding 和底层网络，通常比多个完全独立模型节省训练与在线特征开销；
- **学习完整漏斗**：可显式建模曝光、点击、转化等任务关系；
- **提升泛化**：相关辅助任务形成归纳偏置，降低主任务过拟合；
- **输出口径一致**：同一特征版本和样本时间点产生多种概率，便于统一排序；
- **扩展业务目标**：在点击之外引入质量、风险和长期价值，避免单目标系统走向极端。

### 2. 缺点

- **负迁移**：点击与长期价值、支付成功与覆盖率等目标可能方向相反；
- **梯度冲突**：共享参数上不同任务梯度夹角为负，某个任务提升会损害另一个任务；
- **标签尺度差异**：高频点击 loss 可能淹没低频转化或拒付任务；
- **样本空间不同**：转化只在点击后可观测，退款又有更长的成熟窗口；
- **调参复杂**：专家数、共享层、任务权重和最终融合权重都需实验；
- **故障半径更大**：新增任务或标签污染可能影响所有任务；
- **可解释性下降**：很难确认收益来自共享表示、样本增加还是目标融合；
- **线上耦合**：一个任务需要快速迭代时，可能被统一模型的发布周期限制。

### 3. 常见结构

- Shared-Bottom：简单高效，任务差异大时负迁移明显；
- MMoE：每个任务使用独立 Gate 组合共享 Experts；
- PLE：增加 shared 和 task-specific Experts，逐层分离共享与专属知识；
- AITM：利用注意力显式传递有序转化漏斗的信息；
- AdaTT：多层任务到任务融合，显式学习任务关系；
- STEM：进一步把共享与任务专属建模扩展到 Embedding 层。

结构不能解决全部问题，还需配合 loss weighting、采样、梯度冲突处理、去偏和校准。

### 4. 如何控制负迁移

- 先分析任务标签相关性、条件相关性和梯度 cosine similarity；
- 对强相关任务共享更多，对冲突任务使用专属 Expert 或独立模型；
- 使用 uncertainty weighting、GradNorm 等动态 loss 权重；
- 使用 PCGrad 等方法处理冲突梯度；
- 对不同样本空间使用 ESMM/ESCM² 或反事实校正；
- 监控每个任务的 Gate、专家利用率和分桶指标；
- 新增任务必须做逐任务消融，确认没有损害原任务。

### 5. 线上排序

多任务输出需要校准后再融合：

```text
Score =
w1 × f(pClick)
+ w2 × f(pOrder)
+ w3 × f(pPaySuccess)
- w4 × f(pRefund)
- w5 × f(pChargeback)
```

模型 loss 权重和线上业务权重不是同一概念：前者控制训练梯度，后者表达产品价值。线上权重应通过约束实验和长期指标决定。

### 6. 跨境支付场景

支付产品推荐或通道路由可以联合预测：

> `pOpen、pUse、pSuccess、pTimeout、pReturn、pChargeback`

共享国家、币种、商户和通道表示有价值，但成功率、成本、覆盖率和风险天然冲突。最终排序必须在牌照、KYB、制裁、额度和通道状态等硬约束之后进行，资金动作仍由确定性系统执行。

评测不能只看平均 AUC，还要按国家、币种、通道、新老商户和高风险交易分桶，并联合观察成功率、覆盖率、净收益、风险损失和概率校准。
