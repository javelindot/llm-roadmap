# MMOE如何解决负迁移问题

## 答案

负迁移指多任务联合训练后，某个任务反而比单任务模型更差，常见原因是任务相关性弱、梯度方向冲突、标签尺度不同或大任务主导共享参数。

MMoE 通过“共享多个 Expert + 每个任务独立 Gate”缓解负迁移：

```text
h_t = sum_i gate_t[i] * expert_i(x)
```

不同任务可以选择不同专家组合，不再像 Shared-Bottom 那样强制所有任务使用同一份共享表示。

### 1. MMoE 如何缓解

**多个 Expert 提供可分化容量**

部分 Expert 可以学习任务共性，部分 Expert 可以逐渐偏向特定任务模式。

**任务独立 Gate**

每个任务根据输入动态分配 Expert 权重。例如支付成功率更关注通道和网络状态，拒付风险更关注商户、用户和历史行为。

**任务专属 Tower**

共享表示之后，各任务使用独立输出网络和 loss，保留任务特性。

### 2. 为什么不能说完全解决

经典 MMoE 的 Expert 仍然是所有任务共享的，多个任务的梯度仍会同时更新它们，因此仍可能出现：

- Gate 都选择相同 Expert；
- Expert 输出高度相似；
- 大样本任务压制小任务；
- 任务梯度方向冲突；
- 某任务 loss 尺度过大；
- 稀疏任务没有足够训练信号。

### 3. 如何诊断

- 与每个任务的单任务模型比较；
- 查看各任务 Gate 分布和熵；
- 检查 Expert 使用率与输出相似度；
- 计算任务梯度 cosine similarity；
- 做任务组合消融；
- 观察分人群、长尾和关键场景指标；
- 检查每个任务 loss 与梯度范数。

### 4. 进一步优化

- 调整任务 loss 权重；
- 使用 GradNorm、uncertainty weighting；
- 使用 PCGrad/CAGrad 缓解梯度冲突；
- 给任务增加专属 Expert；
- 使用 PLE，把 shared expert 和 task-specific expert 显式分开；
- 调整任务采样比例；
- 对 Gate 加负载均衡或多样性正则；
- 对冲突严重的任务拆模。

### 5. 跨境支付场景

同时预测通道成功率、成本、时效和风险时，“成功率最大化”和“风险最小化”可能冲突。MMoE 能让任务使用不同专家，但制裁、牌照、限额等不能只作为一个可学习任务，而应作为硬过滤规则。

### 6. 面试总结

MMoE 通过任务独立 Gate 为每个任务选择不同 Expert 组合，减少硬共享导致的表示冲突。但它只是缓解负迁移，不是彻底消除；仍要结合单任务基线、Gate/Expert 诊断、损失平衡和 PLE 或梯度冲突方法。
