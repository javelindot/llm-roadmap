# coding：一道链表mid

## 答案

题干只写了“一道链表 mid”，没有给输入输出，无法给出唯一代码。面试题库中应尽量补全题目。对于未知的 mid 链表题，可以先用“操作位置”和“是否需要回退”判断方法。

### 1. 识别题型

- 要找中点、倒数第 N 个、判断环：快慢指针；
- 要改变节点顺序：反转或分段反转；
- 要连接两个链表：dummy node + 双指针；
- 要按顺序排序：归并排序；
- 要复制带随机指针链表：哈希表或原地穿插；
- 要做 LRU：哈希表 + 双向链表。

### 2. 分段反转通用模板

很多 mid 题，例如“反转第 left 到 right 个节点”“K 个一组反转”，都依赖局部反转。下面以反转区间为例：

```python
def reverse_between(head, left, right):
    dummy = ListNode(0, head)
    before = dummy

    for _ in range(left - 1):
        before = before.next

    cur = before.next

    for _ in range(right - left):
        moved = cur.next
        cur.next = moved.next
        moved.next = before.next
        before.next = moved

    return dummy.next
```

### 3. 为什么使用 dummy node

如果 `left == 1`，头节点本身会变化。dummy node 把“修改头节点”和“修改中间节点”统一起来，减少边界分支。

### 4. 复杂度

```text
时间复杂度 O(n)
空间复杂度 O(1)
```

### 5. 容易出错的点

- 断链后丢失 `next`；
- 区间边界少走或多走一步；
- 忘记返回新头节点；
- 空链表和单节点；
- 反转后没有接回剩余链表；
- 用节点值交换代替节点重排，而题目要求移动节点。

### 6. 面试总结

原题缺失时只能给通用框架，不能声称这就是原题答案。mid 链表题通常是“定位 + 断链 + 反转/合并 + 重连”，写代码时用 dummy node 统一头节点边界，并在每次改 `next` 前保存后继节点。
