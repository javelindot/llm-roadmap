# 如何对AIGC的文本进行评估？

## 答案

AIGC 文本评估应先按场景定义 rubric，再组合确定性指标、事实验证、LLM Judge、人工评审和线上指标。一个 BLEU 或总分无法覆盖质量。

### 1. 通用维度

- 指令遵循、相关性和完整性；
- 事实正确、引用忠实和时效性；
- 连贯、可读、风格和简洁性；
- 多样性、重复率和模板化；
- 安全、偏见、隐私与版权风险；
- 格式、结构化 schema 和工具参数合法性。

### 2. 自动指标

- 有参考文本：ROUGE、BLEU、BERTScore，但表达不同不代表质量差；
- 无参考：困惑度、重复 n-gram、长度与可读性只作诊断；
- 事实任务：实体/数值比对、检索证据、NLI/faithfulness；
- 代码/数学：单元测试和最终答案等可验证指标优先；
- 多样性：Distinct-n、Self-BLEU 等，但高多样性不等于高质量。

### 3. LLM-as-Judge

使用明确 rubric、逐维评分或 pairwise 比较，随机交换答案顺序，避免位置与长度偏差。用多 Judge 或人工集校准一致性，训练 Judge 与最终 Judge 尽量独立。

### 4. 人工与线上

人工盲评处理主观与高风险样本，记录一致性。线上看任务完成、追问、编辑、投诉、留存、延迟和成本，并防止只优化点击。

跨境支付文本必须检查金额、币种、订单状态和规则引用；语言再流畅，只要事实或权限错误就应判不合格。

### 面试总结

AIGC 评估要以业务 rubric 为中心，确定性任务优先用可执行验证，开放文本用 Judge 与人工交叉校准，再用线上任务指标闭环。流畅度不能掩盖事实和安全错误。
