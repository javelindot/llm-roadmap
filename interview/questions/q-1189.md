# 什么是投机采样？

## 答案

投机采样或 Speculative Decoding 使用一个更快的 Draft Model 连续提出多个 token，再让 Target Model 一次并行验证，从而减少大模型串行解码次数。经典算法在使用修正拒绝采样时，可以保持与 Target Model 相同的输出分布。

### 1. 基本流程

设 Draft Model 分布为 `q`，Target Model 分布为 `p`：

1. Draft 自回归提出 `γ` 个 token；
2. Target 对整段候选做一次前向，并行得到每个位置的 `p`；
3. 依次接受候选 token，接受概率为：

```text
min(1, p(x) / q(x))
```

4. 第一个被拒绝的位置，从修正分布重新采样；
5. 如果全部接受，还可以从 Target 的下一个位置再采一个 token；
6. 重复直到结束。

Greedy 解码可以简化为“Draft token 与 Target argmax 相同则接受”，但随机采样若要严格保持目标分布，需要完整的接受与修正步骤。

### 2. 为什么能加速

普通自回归生成 `γ` 个 token 需要 Target 串行运行 `γ` 次。投机采样中，Target 可以利用 GPU 并行能力一次验证短序列；如果大多数 Draft token 被接受，一次 Target 调用就推进多个 token。

速度收益大致取决于：

- Draft 与 Target 的接受率；
- 每轮提出的 token 数；
- Draft 推理成本；
- Target 并行验证效率；
- KV Cache、显存带宽和 batch；
- 输出长度与请求调度。

接受率低时，Draft 的计算会变成额外开销。

### 3. Draft Model 如何选择

理想 Draft 应：

- 显著快于 Target；
- tokenizer 和词表兼容，或有可靠映射；
- 输出分布与 Target 足够接近；
- 在目标语言和业务域中保持高接受率；
- 不占用过多额外显存。

可以使用同系列小模型、Target 的早退层、蒸馏模型，或多 token prediction heads。Draft 越强不一定越好，因为自身成本也会上升。

### 4. 优点与限制

优点：

- 经典算法不需要修改 Target 权重；
- 可以保持 Target 的采样分布；
- 更适合低 batch、解码受内存带宽限制的场景；
- 可与量化、KV Cache 等优化组合。

限制：

- 不保证所有负载都加速；
- 高并发 continuous batching 下收益可能下降；
- Draft 增加模型加载、显存和调度复杂度；
- Prompt 很长、输出很短时，Prefill 占主导，收益有限；
- 业务域差异会降低接受率；
- 动态 speculation length 需要额外策略。

### 5. 评测方式

不能只报告 tokens/s，应同时比较：

- 单请求 P50/P95/P99；
- inter-token latency；
- Target 调用次数；
- Draft token 接受率和每轮接受长度；
- 吞吐、GPU 利用率和显存；
- 相同解码参数下的输出分布或精确一致性；
- 不同 batch、输入长度和输出长度分桶。

### 6. 跨境支付场景

客服 RAG 的低并发长回答可能受益，但订单查询、意图识别和结构化工具调用通常输出很短，投机采样收益有限。是否采用应由真实流量压测决定，并确保 Draft 和 Target 的版本、tokenizer、审计日志及降级路径可控。
