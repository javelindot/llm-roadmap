# GPT 的结构和预训练目标是什么？

## 答案

**一句话**：GPT = **Decoder-only Transformer + Next-Token Prediction**——用因果掩码堆叠的 transformer decoder 学"给定前文猜下一个 token"，是现代 LLM 的祖师爷架构。

### 1. 结构（GPT-1 起的范式）
- N 层 Transformer **Decoder block**
- 每个 block：Masked Multi-Head Attn + FFN + 残差 + LN
- 输入：tokens → embedding + position
- 输出：每个位置的下一个 token 概率分布
- 关键：**只有 Decoder，没有 Encoder**（区别于原始 Transformer）

### 2. Decoder block 内部
```
x → LN → MHA(causal mask) + residual → LN → FFN + residual
```
- Pre-LN（GPT-2 后默认）
- Causal mask：每个位置只能 attend ≤ 当前位置

### 3. 预训练目标：Next-Token Prediction（Causal LM）
$$L = -\sum_t \log P_\theta(x_t | x_{<t})$$
- 标准 cross-entropy
- 整个序列一次性算（teacher forcing）
- causal mask 保证不偷看未来

### 4. GPT 系列演化
| 版本 | 年份 | 参数 | 重要变化 |
|------|------|------|---------|
| GPT-1 | 2018 | 117M | 引入 decoder-only + 预训练-微调范式 |
| GPT-2 | 2019 | 1.5B | Pre-LN、zero-shot 涌现 |
| GPT-3 | 2020 | 175B | In-context learning |
| GPT-3.5 | 2022 | - | RLHF + ChatGPT |
| GPT-4 | 2023 | ~1.8T (MoE) | 多模态 |
| GPT-4o | 2024 | - | 原生多模态 + 实时 |

### 5. 与 BERT 的区别
| 维度 | GPT（Decoder-only） | BERT（Encoder-only） |
|------|-------------------|---------------------|
| Attention | Causal | 双向 |
| 训练目标 | Next-token | Masked LM |
| 适合 | 生成 | 理解 |
| 代表 | 全部现代 LLM | 老一代 |

### 6. 现代 GPT 风格 LLM 的共性
- Decoder-only
- Pre-LN + RMSNorm
- RoPE 位置编码
- GQA/MLA 注意力
- SwiGLU 激活
- BPE / SentencePiece tokenizer

### 7. 为什么 Decoder-only 赢了
- 单一目标（next-token）简洁
- 训练和推理一致
- 自回归生成天然
- Scaling Law 可观
- BERT 风格在大模型上效果反而不如 decoder-only

### 8. 一句话总结
**GPT = Decoder-only Transformer + Next-Token Prediction**——这两个简单设计被 Scaling Law 砸出整个 LLM 时代。

## 结合项目谈（口语版 · 约 20 秒）

> 我们跨境支付的 LLM 主线全部是 **Decoder-only**（Qwen/Llama/Mistral 系列），就是 GPT 范式的开源继承者。
>
> 实际工程里 GPT 范式给我们的影响有几个具体点：
>
> **1. SFT 数据组织**：next-token 训练目标决定 SFT 数据必须是连续文本。我们客服 SFT 把"User: 问题\nAssistant: 答案"拼成单条，loss 只在 Assistant 段算（详见 q-53 的 loss masking 经验）。
>
> **2. KV Cache 是 decoder-only 的命脉**：因果掩码意味着推理时前文 K/V 可以缓存，每次只算新 token。这是为什么 KV Cache 压缩（GQA、MLA）成了工程焦点。
>
> **3. Pretrain → SFT → DPO 三段式**：GPT-3 → InstructGPT → ChatGPT 这条路径成了所有商业 LLM 的标准，我们的 base 选型也按这条路径——base 模型预训练好、我们做 SFT + DPO。
>
> **4. 涌现 vs 微调**：GPT-3 的 in-context learning 让我们意识到很多任务不需要训练，prompt 就能解决。我们风控规则的快速试错就靠这个——业务想加个新规则，先 prompt 试，效果可以再做数据精调。
>
> 一句话：**GPT 的"decoder-only + next token"是简洁到极致的设计**，整个 LLM 行业的工程范式都是这个简单内核的工程化延伸。
