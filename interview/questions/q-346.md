### 补充说明

理解节点的提示词要约束模型只做“理解和结构化输出”，不要在这个节点直接回答用户或执行工具。通常会要求输出固定 JSON，包括意图、槽位、命令、置信度、是否需要追问、是否需要转人工。

约束手段包括：

- 明确角色：只做意图理解。
- 给出可选命令枚举。
- 给出槽位 schema。
- 给出决策规则表。
- 要求无法判断时输出 unknown。
- 禁止编造不存在的字段和命令。
- 输出后用代码做 JSON Schema 校验。

### 结合跨境支付项目怎么说（面试口径）

> 在跨境支付客服 Agent 里，理解节点我会约束它只输出结构化指令。比如用户说“这笔美金怎么还没到”，理解节点输出意图是 `payment_delay_query`，槽位里可能有币种 USD，但订单号缺失，于是下一步命令是追问订单号，而不是直接回答。
>
> 可用命令必须是白名单，比如查询订单、查询通道回执、查询风控大类、检索知识库、转人工。模型不能自己发明“强制放款”这种命令。
>
> 输出后还要通过 JSON Schema 校验，校验不过就降级或重新解析。这样可以把模型的不确定性限制在理解层，不让它直接影响资金动作。

**钩子**：可以引到“理解节点只输出 JSON、命令白名单、槽位 schema、模型不能直接执行资金动作”。
