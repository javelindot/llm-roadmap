# 4.讲讲微调（全量和高效）。LoRA的原理。了解多少别的变种？

## 答案

微调可分为全参数微调和参数高效微调（PEFT）。全参改变所有权重，能力上限高但成本和遗忘风险大；PEFT 只训练少量新增或选定参数，更适合企业领域适配和多租户模型管理。

### 1. 全参数微调

所有参数都参与反向传播：

- 优点：容量最大，适合大规模高质量数据和深层能力变化；
- 缺点：参数、梯度、优化器状态显存高，训练慢；
- 容易灾难性遗忘；
- 每个业务版本都要保存完整模型。

常结合 BF16、FlashAttention、gradient checkpoint、FSDP/ZeRO 和张量并行。

### 2. LoRA 原理

冻结原矩阵 `W`，训练低秩增量：

```text
W' = W + (alpha / r) * B * A

A: [r, d_in]
B: [d_out, r]
```

新增参数：

```text
r * (d_in + d_out)
```

远小于原矩阵 `d_in * d_out`。LoRA 常插入 Q/K/V/O 和 MLP 线性层，可动态加载，也可合并后部署。

### 3. 如何选择

**选全参**

- 数据规模大且质量高；
- 需要改变底层领域语言或复杂能力；
- 有足够算力；
- 允许保存独立完整模型。

**选 LoRA/PEFT**

- 几千到几十万条领域指令；
- 主要学习格式、风格、工具协议和有限领域行为；
- 需要多个业务 Adapter；
- 显存和迭代周期有限。

若业务事实频繁变化，应放 RAG/工具，不要靠全参微调记忆。

### 4. 常见变体

- QLoRA：4-bit 量化冻结基座，训练 LoRA；
- AdaLoRA：按重要性动态分配 rank；
- DoRA：分解权重方向与幅值；
- rsLoRA：调整 rank scaling，改善大 rank 稳定性；
- LoRA+：对 A/B 使用不同学习率；
- PiSSA/OLoRA：改初始化，让低秩子空间更贴近原权重；
- Prefix/Prompt Tuning：训练虚拟 token；
- Adapter：插入瓶颈网络；
- IA3：训练激活缩放向量。

### 5. 评估

不能只看训练 loss，需要比较：

- 任务成功率、格式和工具准确率；
- 通用能力回归；
- 不同 rank 和 target modules 的消融；
- 合并前后 logits/效果；
- 量化后的质量、显存和吞吐。

### 6. 跨境支付

LoRA 可学习客服语气、调查报告结构和工具调用格式。订单状态、费率、汇率、制裁名单和规则版本必须实时查询。高风险合规结论仍由规则和人工审批。
