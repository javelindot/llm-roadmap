# 项目：未来希望专注大模型哪个方向？（对齐 / 推理加速 / 长上下文？）为什么？

## 答案

应选择一个与真实经历、岗位和长期问题相连的方向，不要同时说三个方向都想深入。

### 参考：可靠 Agent 与推理系统

> 我未来更希望专注可靠 Agent 和推理系统，它连接了对齐、推理加速与长上下文，而目标是让模型在真实业务里低成本、可验证地完成任务。
>
> 我过去在[真实 RAG/工具/训练项目]中发现，瓶颈往往不是模型不会生成，而是证据、工具、状态、权限和延迟不能形成闭环。因此我想继续研究：如何用可验证奖励提升多步规划，如何用 workflow/验证器控制高风险动作，以及如何通过缓存、模型路由和推理服务满足 SLO。
>
> 近期我会重点积累[具体技术]，例如 Agent 评测、GRPO/DPO、vLLM/SGLang 或长上下文压缩；长期希望做能落到真实业务指标的系统。

也可选择对齐、推理加速或长上下文，但都应回答：

- 为什么重要；
- 自己已有何证据；
- 核心未解问题是什么；
- 未来一年准备做什么；
- 与应聘岗位如何匹配。

跨境支付特别适合“可靠性”主线，因为模型必须连接实时事实、权限、幂等与审计。

### 面试总结

聚焦一个稳定问题，而不是追逐名词。用过去项目证明兴趣，用具体技术规划说明下一步，并解释该方向如何产生可验证业务价值。
