# Agent的并发控制实现方法

## 答案

Agent 并发控制要区分“同一任务内并行子步骤”和“多用户任务并发”，并针对模型、工具、数据库和副作用动作分别限流。不能直接对所有工具 `gather`。

### 1. 并发模型

- I/O 密集工具：`asyncio`/异步客户端；
- CPU 密集解析：进程池或独立 worker；
- GPU 推理：集中模型服务与 continuous batching；
- 长任务：消息队列 + workflow/actor；
- 有副作用动作：按业务实体串行或乐观并发控制。

只有互不依赖且只读的步骤才适合并行。

### 2. 受控异步实现

```python
import asyncio
from collections.abc import Awaitable, Callable


class ToolExecutor:
    def __init__(self, limits: dict[str, int]):
        self._semaphores = {
            name: asyncio.Semaphore(limit)
            for name, limit in limits.items()
        }

    async def run(
        self,
        tool_name: str,
        call: Callable[[], Awaitable[object]],
        timeout: float,
    ):
        semaphore = self._semaphores[tool_name]
        async with semaphore:
            return await asyncio.wait_for(call(), timeout=timeout)


async def run_independent_steps(executor, steps):
    async def run_one(step):
        try:
            result = await executor.run(
                tool_name=step["tool_name"],
                call=step["call"],
                timeout=step["timeout"],
            )
            return {"id": step["id"], "ok": True, "result": result}
        except asyncio.TimeoutError:
            return {"id": step["id"], "ok": False, "error": "timeout"}
        except Exception as exc:
            return {
                "id": step["id"],
                "ok": False,
                "error": type(exc).__name__,
            }

    async with asyncio.TaskGroup() as group:
        tasks = [group.create_task(run_one(step)) for step in steps]

    return [task.result() for task in tasks]
```

这段代码为每类工具设置独立 semaphore 和超时；生产中还要加入全局请求预算、rate limiter、trace、重试分类和取消传播。

### 3. 状态与一致性

- 每个任务使用独立 `task_id/session_id`；
- 状态外置并带版本号；
- 并行步骤返回结构化结果，由聚合节点按 ID 合并；
- 写操作使用幂等键；
- 同一订单/账户可用乐观锁或实体队列串行；
- 不持有数据库锁等待 LLM 或远程工具；
- 失败后从 checkpoint 恢复。

### 4. 背压与公平

- 有界队列，满时拒绝或降级；
- 按租户和优先级做加权公平；
- 限制单任务 fan-out；
- 根据队列年龄、P95 和下游配额扩缩容；
- 对 429/503 指数退避并加 jitter；
- 超过 deadline 的任务取消，不继续浪费 token；
- 使用 circuit breaker 隔离故障工具。

### 5. 跨境支付场景

订单、通道和规则查询可并行；退款、冻结和账务写入必须按订单串行、幂等并审批。即使请求超时重试，也不能产生重复扣款或退款。

### 6. 面试总结

Agent 并发的核心是依赖分析、分资源池限流、背压、状态隔离、取消与幂等。并行只读步骤，串行有副作用步骤，并用集中模型服务承接 GPU 并发。
