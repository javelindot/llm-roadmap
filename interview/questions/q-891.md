# 模型选型问题，为什么选qwen3-8b，如何估算模型开销

## 答案

只有在真实对比实验支持时才能说“选择 Qwen3-8B”。合理理由是它在能力、中文/多语言、推理/非推理模式、Agent 能力和部署成本之间达到项目所需平衡，而不是因为 8B 流行。

### 1. 官方配置

[Qwen3-8B 模型卡](https://huggingface.co/Qwen/Qwen3-8B)给出的当前配置：

- 8.2B 参数，非 embedding 约 6.95B；
- 36 层，hidden size 4096；
- 32 个 Q head、8 个 KV head，head dim 128；
- 原生 32,768 context，可通过 YaRN 到 131,072；
- 支持 thinking/non-thinking 与工具调用。

### 2. 选型实验

与 4B、14B 和其他同级模型在相同 prompt、量化、硬件上比较：

- 业务正确率、工具成功率和拒答；
- 中文、长尾和长上下文；
- TTFT、ITL、P99、吞吐和单位成本；
- 峰值显存、并发和稳定性；
- LoRA/量化后的质量回退。

### 3. 权重与 KV 开销

权重下界：

- BF16：约 \(8.2B\times2\approx16.4\) GB；
- INT8：约 8.2 GB；
- INT4：理论约 4.1 GB，实际还需 scale、metadata 和运行时空间。

BF16 KV Cache 每 token：

\[
2\times36\times8\times128\times2
=147456\text{ Bytes}
\]

约 144 KiB/token。单请求 32K KV 约 4.5 GiB，实际还需权重、workspace、碎片和并发预算。

### 4. 计算与输出

Dense 模型每 token 粗略前向 FLOPs 约 \(2P\)，8.2B 即约 16.4 GFLOPs，另有长上下文 Attention 和系统开销。thinking mode 可能产生更长输出，成本必须按真实平均输出 token 估算。

### 面试总结

选择 Qwen3-8B 应由业务质量和 SLO 实验决定。其 8.2B、36 层、32/8 GQA 使能力与部署成本较均衡，但长上下文 KV 和 thinking 输出仍可能成为主要成本。
