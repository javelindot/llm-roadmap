# 跨域问题的解决方法有哪些

## 答案

“跨域”有两种常见含义，面试时应先确认是 Web 跨域，还是机器学习中的 Domain Shift。

### 1. Web 跨域

浏览器同源策略要求协议、域名、端口一致。常见解决方法：

- 服务端配置 CORS，精确设置允许的 origin、method、header；
- 同域反向代理，例如通过 Nginx/API Gateway 转发；
- 服务端到服务端请求，因为同源策略主要约束浏览器；
- 特殊场景使用 `postMessage` 做安全的跨窗口通信。

带 Cookie 时不能把 `Access-Control-Allow-Origin` 配为 `*`，还要设置 credentials、CSRF 防护和白名单。JSONP 只支持 GET 且安全性差，已不是首选。

### 2. 机器学习跨域

源域与目标域分布不同，可采用：

- 在目标域继续预训练或微调；
- 特征对齐，如 MMD、CORAL、对抗式域适配；
- 伪标签、自训练和一致性正则；
- 多域训练、领域专家或 Adapter/LoRA；
- 数据重加权、数据增强和不变特征学习；
- 无标签目标域下做 Unsupervised Domain Adaptation；
- 无法拿到源数据时做 Source-Free Adaptation；
- 上线后进行漂移监控和分域校准。

跨境支付天然存在国家、语言、币种和通道分布差异。训练时应按域切片评估，不应把中国银行卡数据上的高 AUC 直接外推到巴西 PIX。规则差异由本地知识和风控系统约束，模型只迁移可共享的表征。

### 面试总结

先澄清“跨域”的含义。Web 跨域优先使用严格 CORS 或同域网关；机器学习跨域则通过目标域数据、表征对齐、重加权和漂移监控解决，核心是验证目标域而不是只优化源域指标。
