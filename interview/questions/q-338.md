# LLM Ops了解吗？公司代码挪到现场部署会有各种问题，就算用Docker，之前也要做一

## 答案

件事，防止逆向工程，顺带做一个加速处理。还有一个新出来的Agent Ops。
答：

### 正式回答

LLM Ops 可以理解成围绕大模型应用的研发、部署、评估、监控和运营体系。它不仅包括模型部署，还包括数据集管理、Prompt 版本管理、RAG 知识库版本、评测集、线上监控、badcase 回流、成本统计和安全审计。

现场部署时，即使用 Docker，也要处理很多问题：

- 环境变量和密钥配置。
- GPU 驱动、CUDA、模型依赖版本。
- 内网网络和证书。
- 模型权重和知识库数据同步。
- 日志、监控、告警。
- 版本回滚和灰度。
- 代码混淆或镜像加固，防止被随意逆向。

Agent Ops 则更关注 Agent 每一步的 trace、工具调用、失败恢复、成本、权限和效果评估。

### 结合跨境支付项目怎么说（面试口径）

> 在跨境支付项目里，LLM Ops 非常重要。比如智能客服上线后，不只是模型能跑，还要知道每次回答用了哪个模型、检索了哪些文档、调用了哪些工具、有没有命中敏感信息、成本是多少、用户是否满意。
>
> 如果把代码搬到客户现场部署，即使用 Docker，也要提前处理 GPU 驱动、模型权重、向量库、配置密钥、内网访问和日志采集。支付场景还要特别关注数据不出域、调用审计和权限隔离。
>
> Agent Ops 方面，我会记录每个 Agent 的决策链路。比如一笔付款失败解释，系统查了订单、通道、风控和知识库，每一步都要能追踪。这样出了错才能定位是检索错、工具错还是模型生成错。

**钩子**：可以引到“Prompt/模型/知识库版本管理、Agent trace、现场私有化部署、审计和回滚”。
