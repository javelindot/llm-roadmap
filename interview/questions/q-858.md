# 如何解决对话系统中时延问题

## 答案

先把对话系统拆成端点检测、ASR、检索/工具、LLM prefill、decode、TTS 和网络排队，并分别测 P50/P95/P99。语音系统还要关注首音节延迟，而不只是总完成时间。

### 1. 流式与并行

- 流式 ASR，用户未完全说完时做增量意图判断；
- endpointing/VAD 减少无效等待，但避免过早截断；
- ASR 部分结果稳定后提前检索；
- 无依赖的知识检索和只读工具并行；
- LLM token 流式传给 TTS，TTS 分句或分块合成；
- 支持用户打断，取消下游无效任务。

### 2. 减少模型工作量

- 小模型路由简单 FAQ，复杂任务升级大模型；
- 压缩 system prompt、历史和工具 schema；
- RAG 只返回相关证据，长期对话做摘要与结构化记忆；
- 限制生成长度，先给结论再补充；
- 缓存公共前缀、热点答案、ASR/TTS 结果。

### 3. 服务优化

- continuous batching、Paged KV、chunked prefill；
- prefix caching、量化、GQA/MLA 和 speculative decoding；
- 按 deadline 调度，长请求与短请求隔离；
- 连接池、异步 I/O、超时、熔断和就近部署；
- 流量预测与预热，避免冷启动。

### 4. 体验与正确性

可先回应“正在查询”，但不能用低延迟占位内容冒充事实。支付客服中的余额、到账和退款状态必须等待权威系统；超时应转人工或明确稍后返回。

### 面试总结

对话时延优化依靠 ASR→检索→LLM→TTS 的流式流水线、可并行工具和更少上下文，再结合推理服务调度。应分别优化首字/首音节延迟、逐 token 延迟和端到端 P99。
