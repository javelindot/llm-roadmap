# llama结构

## 答案

LLaMA 不是一个固定配置，而是一个模型家族。Llama 1–3 的主干是 dense decoder-only Transformer；Llama 4 又引入原生多模态和 Mixture-of-Experts。因此回答结构前应说明具体代际。

### 1. Llama 1–3 的公共骨架

```text
Token IDs
-> Token Embedding
-> N x Decoder Block
-> Final RMSNorm
-> LM Head
-> Vocabulary Logits
```

每层通常为：

```text
X
-> RMSNorm
-> Causal Self-Attention
-> Residual Add
-> RMSNorm
-> SwiGLU FFN
-> Residual Add
```

它是 decoder-only 自回归模型，通过 causal mask 预测下一个 token：

```text
L = -sum_t log p(x_t | x_<t)
```

### 2. RMSNorm 和 Pre-Norm

LLaMA 使用 Pre-Norm，将归一化放在 Attention/FFN 前：

```text
X1 = X + Attention(RMSNorm(X))
X2 = X1 + FFN(RMSNorm(X1))
```

RMSNorm 只按均方根缩放，不做均值中心化。Pre-Norm 让深层残差路径更稳定。

### 3. RoPE

Rotary Position Embedding 作用于 Q/K：

```text
Q_pos = rotate(Q, position)
K_pos = rotate(K, position)
```

旋转后的内积包含相对位置信息。它不是把一个 position embedding 直接加到 token embedding 上。

长上下文扩展还涉及训练长度、RoPE scaling、插值和注意力实现，不能只修改配置中的最大长度。

### 4. Attention

早期尺寸可使用标准 MHA；较新的 Llama 2 大模型和 Llama 3 系列广泛使用 GQA：

```text
num_kv_heads < num_query_heads
```

多个 Query head 共享较少的 K/V head，降低 KV Cache 和 decode 内存带宽，同时尽量保留 MHA 质量。

推理分为：

- Prefill：并行处理 Prompt，生成每层 KV Cache；
- Decode：每步只计算新 token 的 Q/K/V，复用历史 K/V。

### 5. SwiGLU FFN

```text
FFN(x)
= W_down(
    SiLU(W_gate x)
    * (W_up x)
  )
```

它使用门控分支和逐元素乘法，比普通 ReLU/GELU FFN 具有更强表达能力。LLaMA 还通常减少线性层 bias。

### 6. Tokenizer 和输出

- 文本经 SentencePiece/BPE 类 tokenizer 转为 token；
- hidden states 经 LM Head 投影到词表；
- 输出是 logits，不是概率；
- temperature、top-p/top-k 等只在采样阶段使用；
- 不同代际的词表、上下文和特殊 token 不同，不能混用 tokenizer。

### 7. Llama 3 的重要变化

Llama 3 仍保持 dense decoder-only 主干，但扩大训练数据和词表，使用 GQA，并提供更长上下文的后续版本。具体层数、hidden、head 数取决于 8B、70B、405B 等尺寸。

模型能力提升不能只归因于结构，也来自数据质量、规模、训练配方和后训练。

### 8. Llama 4 的区别

Meta 2025 年公布的 Llama 4 Scout/Maverick 不再只是经典 dense 文本 Decoder：

- 原生多模态，文本与视觉共同训练；
- 使用 MoE，仅激活部分专家；
- 采用 early-fusion 路线处理多模态 token；
- 总参数量与每 token 激活参数量不同；
- 部署需要考虑 expert routing、通信和多模态输入成本。

因此不能把“LLaMA 永远是 dense decoder-only”作为家族级结论。经典架构题通常仍在问 Llama 1–3，需要根据上下文确认。
