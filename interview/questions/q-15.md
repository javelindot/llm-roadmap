# 投影层（Projection Layer）的作用是什么？

## 答案

**一句话**：投影层是多模态模型里的**翻译官**——把视觉/语音 encoder 输出的特征，"翻译"到 LLM 能听懂的 embedding 空间里。

### 1. 解决的核心问题
- **维度对齐**：CLIP-ViT 输出 1024 维，LLaMA embedding 4096 维，直接拼对不上。
- **语义对齐**：Vision encoder 训出来的特征空间和文本 embedding 空间不在一个语义系上，需要映射。
- **信息压缩**：一张图 ViT 切成 576 个 patch，全塞进 LLM 上下文会爆——常用投影层顺手做下采样。

### 2. 主流形式
| 形式 | 代表模型 | 特点 |
|------|---------|------|
| **线性投影**（一层 Linear） | LLaVA v1 | 最简单，效果一般 |
| **MLP**（两层 + GELU） | LLaVA v1.5 / v1.6 | 简单有效，是当前主流 |
| **Q-Former** | BLIP-2 | 可学习 query 抽取压缩特征 |
| **Perceiver Resampler** | Flamingo | 跨注意力压缩 |
| **Cross-Attention** | Flamingo / Idefics | LLM 每层都插模态 token |

### 3. 训练策略（这是面试常考点）
- **阶段一（pretrain）**：冻 Vision Encoder + 冻 LLM，**只训投影层**——用 image-text 对齐（image caption）。
- **阶段二（instruction tuning）**：解冻 LLM，训投影层 + LLM，少量 vision 指令数据。
- 好处：**只训投影层时**仅 1% 左右参数可训，效率极高。

### 4. 关键设计 trade-off
- **简单（Linear/MLP） vs 表达力（Q-Former）**：MLP 训得快但保留全部 patch 信息；Q-Former 能压缩但训练复杂。
- **patch 数 vs 上下文窗口**：高分辨率图需要更多 patch，投影层要负责降维。
- **是否共享 LLM tokenizer**：有的把图像 token 接成 special token 走 LLM tokenizer。

### 5. 一句话总结
**投影层是多模态最便宜的"对接"——参数少、训练快、改一改就能换 vision encoder**，但表达力上限就是它的上限。

## 结合项目谈（口语版 · 约 40 秒）

> 投影层这块我们跨境支付里用过一次，场景是商户入网时的 **KYB 证件识别 + 票据 OCR**。商户上传的营业执照、税务登记证、税单这些，过去走传统 OCR + 规则提取错误率挺高，尤其是手写体和印章遮挡。
>
> 我们后来选了一个多模态方案：CLIP-ViT 做 vision encoder + Qwen 做 LLM，中间接一个两层 MLP 当投影层。**只训投影层**这一招特别香——5w 张带标注的证件样本，单卡 A100 跑两天就能收敛，比全量微调省太多了。
>
> 上线之后字段提取准确率比规则方案高了 12 个点左右，关键字段（公司名、统一社会信用代码、有效期）准确率到了 99%+。手写体那块也好不少，因为 LLM 有语义先验，能联系上下文猜，规则系统只看像素。
>
> 投影层我们用的就是 MLP，没上 Q-Former——量级不到、复杂度不值。
>
> 顺便发现一个隐藏好处：投影层独立，意味着我们可以**冷启动期复用通用多模态模型**，等业务数据攒够了再 finetune 投影层做领域适配，迁移成本很低。

**🪝 顺嘴带出去的钩子**：
- "证件识别里中文 OCR 我们怎么调投影层 patch 数能聊聊。"
- "为什么不上 Q-Former 而用 MLP，这个选型可以展开。"
- "多模态 + LLM 的成本和准确率怎么平衡，我们有几条经验。"
