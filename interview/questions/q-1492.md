# 项目中使用了LSTM？为什么使用它，不用Transformer？LSTM的不足和优势？

## 答案

这是项目选型题，应先说明当时的数据规模、序列长度、时延预算和实际对比结果，不能泛泛地说 LSTM 一定优于 Transformer。

### 1. 为什么可能选择 LSTM

- 序列较短，长距离依赖不是主要矛盾；
- 数据规模有限，LSTM 的归纳偏置更强、基线更稳定；
- 需要流式逐步更新隐藏状态；
- 在线部署在 CPU 或低算力设备，模型规模和时延受限；
- 项目已有成熟 LSTM 链路，新增 Transformer 的收益不足以覆盖复杂度；
- 对照实验中，Transformer 的业务收益没有显著提升。

面试时应给出真实的序列长度、参数量、P95 时延和离线/线上指标；没有做过对照实验就明确说这是工程判断，不应编造数据。

### 2. LSTM 的优势

- 门控缓解普通 RNN 的梯度消失；
- 按时间自然维护状态，适合流式输入；
- 单步增量推理只需保存隐藏状态；
- 对短序列和中小数据集通常是可靠基线；
- 不需要构造全序列两两 Attention。

### 3. LSTM 的不足

- 时间步必须串行，训练并行度低；
- 长序列信息压缩在固定状态中，仍可能遗忘；
- 很难像 Self-Attention 一样直接建立任意位置联系；
- 多层双向 LSTM 的训练与部署也可能较慢；
- 大规模预训练与多模态扩展生态弱于 Transformer。

### 4. Transformer 的取舍

Transformer 训练时可并行处理 token，长距离依赖更直接，也更容易利用预训练模型。但标准 Attention 的时间和显存随序列长度近似二次增长；在线逐 token 推理还要维护 KV Cache。

当前工程也可选择线性 Attention、状态空间模型或 LSTM/Transformer 混合结构，因此选型应基于工作负载，而不是只比较模型年代。

### 5. 跨境支付场景

若根据最近几十笔交易预测下一次支付偏好或短期风险，LSTM 可能以较低成本满足需求；若要联合数百至数千个事件、设备关系、文本备注和跨周期依赖，Transformer 更有优势。

无论哪种模型，时间切分、延迟标签、概念漂移和可解释护栏都比单纯换架构更重要。

### 6. 面试总结

LSTM 的核心优势是短序列、流式状态和较轻部署；不足是串行训练和长依赖瓶颈。回答“为什么没用 Transformer”时必须回到数据、序列、资源、时延和实测收益。
