# 什么是数据并行？如何提升通信效率？通信如何协同与计算？

## 答案

**一句话**：数据并行是每张卡放一份完整模型，切分训练数据，各自算梯度，再通过通信把梯度同步成一致模型。

### 核心回答
- **基本流程**：每张 GPU 拿不同 mini-batch，前向和反向后得到本地梯度，通过 AllReduce 求平均，再更新相同参数。
- **优点**：实现简单，扩展吞吐。
- **瓶颈**：模型越大，梯度同步通信越重。

提升通信效率的方法：
- 使用高速互联：NVLink、InfiniBand。
- 梯度 bucket 化，边算边传。
- 通信与反向计算重叠。
- 梯度压缩或混合精度通信。
- ZeRO/FSDP 切分参数、梯度、优化器状态。
- 合理增大 batch size，减少同步频率。

通信协同计算的关键是 overlap：后面层还在反传时，前面层梯度已经开始 AllReduce。

### 1. 数据并行为什么有效

数据并行的前提是每张卡都有一份完整模型，但处理不同数据。这样单步训练可以同时消化更大的 batch，提升吞吐。每张卡算出的梯度代表自己那部分样本，AllReduce 后得到全局平均梯度，保证所有卡更新后仍然是同一份模型。

### 2. 通信效率怎么优化

通信优化通常有三个方向：

- **减少通信量**：混合精度、梯度压缩、ZeRO/FSDP 切分状态。
- **提升通信带宽**：NVLink、InfiniBand、合理拓扑。
- **隐藏通信时间**：把梯度分 bucket，某些层梯度一算完就开始同步，和后续反向计算重叠。

如果 overlap 做得不好，GPU 会出现"算一会儿、等一会儿"的状态，利用率很低。

### 3. 面试总结

数据并行看起来简单，但大模型训练的瓶颈经常在通信。回答时要讲出 AllReduce、bucket、overlap 和 ZeRO/FSDP，面试官会觉得你不是只知道概念。

### 4. 结合跨境支付项目怎么说（面试口径）

> 如果我们训练支付领域模型，比如风控解释、合规问答、商户审核助手，数据并行就是每张卡拿不同的业务样本训练，最后把梯度同步。它的好处是吞吐能上去，适合数据量比较大的 SFT 或偏好训练。
>
> 但实际我不会只看卡数，而会看 GPU 利用率。如果训练时很多时间卡在 AllReduce，说明通信拖慢了计算。这时可以调 batch size、梯度累积、bucket size，或者看通信和反向计算有没有重叠。
>
> 对业务来说，这类垂直模型经常要随着规则变化重新微调，所以训练效率和成本很重要。不是说卡越多越好，而是要让算力真正用在训练上。

**钩子**：数据并行简单但通信容易成为瓶颈，工程上要看 overlap 和显存切分。
