# LORA的秩一般是如何选取的

## 答案

LoRA rank \(r\) 决定每个权重增量最多包含多少个独立方向：

\[
\operatorname{rank}(\Delta W)
=\operatorname{rank}(BA)\le r
\]

不存在通用最优 rank，应按任务、模块、数据和预算做消融。

### 推荐流程

1. 明确 target modules，从 `q/v` 或 `q/k/v/o` 建立基线；
2. 扫描如 4、8、16、32，复杂任务可继续到 64/128；
3. 保持数据、训练 token、学习率搜索预算一致；
4. 比较业务指标、通用能力回归、过拟合、峰值显存和 adapter 大小；
5. 增大 rank 不再提升时停止；
6. 若仍欠拟合，先判断瓶颈是数据、模块覆盖还是 rank，而非只继续增大。

### 影响因素

- 格式/风格适配通常较小 rank 即可；
- 领域偏移、复杂工具或 FFN 适配可能需要更大 rank；
- 同一个 rank 在 Q/V 与大 FFN 矩阵上的容量意义不同；
- 数据少时大 rank 更易过拟合；
- 标准 scale 为 \(\alpha/r\)，大 rank 可尝试 rsLoRA 的 \(\alpha/\sqrt r\)；
- AdaLoRA 可按层重要性动态分配 rank。

### 面试总结

rank 是容量超参数，不是固定经验数。用小 rank 起步，逐步扫描并同时看质量、回归和资源；若提升饱和，应调整数据或目标模块，而不是盲目加 rank。
