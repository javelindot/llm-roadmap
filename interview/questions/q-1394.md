# 如果直接把这些特征输入到一个 LR 或其它简单模型做融合，和你现在的模型相比效果差距大吗？

## 答案

这题要用消融实验回答，而不是主观说复杂模型一定更好。LR 或浅层模型应作为强基线：保持相同数据切分、特征可用范围和评测口径，再比较离线效果、线上收益、延迟、稳定性和可解释性。

### 1. 公平对比方式

至少做三组：

```text
A: 原始基础特征 -> LR
B: 全部人工交叉特征 -> LR/FM/GBDT
C: 相同原始特征 -> 当前复杂模型
```

同时控制：

- 训练/验证/测试切分；
- 负采样方式；
- 标签窗口；
- 特征时间点，防止穿越；
- 超参数搜索预算；
- 阈值和评测脚本。

### 2. 看哪些指标

不能只看一个 AUC：

- AUC、LogLoss、PR-AUC；
- NDCG、Recall@K；
- 概率校准；
- 关键人群和长尾切片；
- P95 延迟、CPU/GPU 成本；
- 特征生产复杂度；
- 线上 CTR/CVR、成功率或投诉率。

跨境支付风险融合中，还要看高金额、高风险国家、新商户和稀有欺诈模式的 Recall，以及人工审核量。

### 3. 复杂模型为什么可能更好

- 学习非线性关系；
- 自动建模高阶特征交叉；
- 学习序列行为；
- 使用多模态或文本表示；
- 多任务共享信息；
- 对长尾组合有更强表达能力。

例如：

```text
国家 x 币种 x 支付方式 x 商户行业 x 时间段
```

LR 需要显式构造交叉，复杂模型可以自动学习。

### 4. LR 为什么可能已经足够

- 数据量小；
- 主要关系接近线性；
- 特征工程已经非常成熟；
- 标签噪声大，模型容量不是瓶颈；
- 线上延迟和解释性要求高；
- 复杂模型收益没有覆盖维护成本。

因此如果复杂模型只提升极小幅度，却增加大量延迟和故障面，生产上可能仍选择 LR。

### 5. 项目回答模板

> 我们把 LR 作为 baseline，并保证相同数据和特征时点。复杂模型在某些离线指标和长尾切片上有提升，主要来自非线性交叉/序列建模；但我不会只报一个整体数字，还会比较线上指标、延迟和成本。具体提升幅度应使用项目真实实验数据，不能现场编造。如果 LR 与复杂模型差距很小，我会优先排查标签、特征和评测口径，而不是继续堆模型。

### 6. 面试总结

正确回答是“通过受控消融确定差距”。LR 是验证复杂模型增益是否真实的关键基线；最终选型应看增量业务价值是否覆盖推理成本、维护复杂度和风险，而不是默认模型越复杂越好。
