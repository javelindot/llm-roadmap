# LORA一般加到哪?rank设置有没有技巧

## 答案

LoRA 常加在 Transformer 的线性层上，优先是 Attention 投影，也可以覆盖 FFN。具体 target modules 和 rank 应通过任务、模型架构和资源预算做消融，不能机械复制配置。

### 1. 常见 Target Modules

Attention：

```text
q_proj
k_proj
v_proj
o_proj
```

FFN：

```text
gate_proj
up_proj
down_proj
```

最省参数的配置常只加 `q_proj/v_proj`；任务变化较大时可覆盖全部 Attention，甚至 Attention+FFN。

Embedding、LM Head、Norm 通常不作为第一选择，但词表扩展、特定输出头或特殊任务可能需要单独训练，并通过 `modules_to_save` 保存。

### 2. Rank 的含义

```text
Delta W = (alpha/r) * B A
```

`r` 决定更新子空间容量：

- rank 小：省显存、训练快，可能欠拟合；
- rank 大：容量更高，参数和优化器状态增加；
- 超过任务需要后收益递减；
- 数据少时高 rank 也可能过拟合。

可训练参数近似：

```text
r * (d_in + d_out)
```

### 3. Rank 选择方法

常见实验起点可用：

```text
r = 8, 16, 32, 64
```

但这只是 sweep 范围，不是通用最优。保持以下条件一致：

- 相同 base；
- 数据和 token；
- target modules；
- 训练步数；
- 学习率搜索；
- eval set；
- 随机种子。

记录质量、显存、吞吐和 checkpoint 大小。

### 4. Alpha 与 Scaling

标准 LoRA：

```text
scaling = alpha / r
```

不同 rank 对比时要控制 scaling。常从 `alpha=r` 或 `2r` 起步。rsLoRA 使用 `alpha/sqrt(r)` 改善高 rank 下的缩放稳定性。

### 5. 按任务判断

- 风格、格式：低 rank 常够；
- 工具调用、领域问答：中等 rank；
- 多任务、复杂推理或大分布迁移：更高 rank、更多模块或全参；
- 不同层需求不同：可使用 rank pattern/AdaLoRA；
- 先比较 Prompt/RAG baseline，避免把动态知识硬塞进 Adapter。

### 6. 训练排查

- 确认 Adapter 真正挂到目标层；
- 打印 trainable parameters；
- 监控 gradient norm；
- 检查合并前后输出；
- 保存 base revision/tokenizer；
- 做通用、安全和多语言回归。

### 7. 跨境支付场景

支付客服可从 Attention LoRA 的低/中 rank 起步，若工具调用和多语言欠拟合再覆盖 FFN。实时订单、费率和规则仍由 RAG/工具提供，不能靠无限增大 rank 解决。

### 8. 面试总结

LoRA 通常先加 Q/K/V/O，再按需求扩到 FFN。Rank 是容量与成本旋钮，应通过受控 sweep 和业务评测选择，并与 alpha、target modules、学习率一起调。
