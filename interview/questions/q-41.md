# 什么是张量并行？优缺点？

## 答案

**一句话**：张量并行（TP）就是把**单个矩阵乘法**沿行或列切到多张卡上算、再 all-reduce 合并——**省显存、加速猛，但通信密集，强依赖 NVLink 这种高带宽互联**。

### 1. 基本原理（Megatron-LM 思路）
对 Linear 层 Y = X · W：
- **列切（Column Parallel）**：W 按列切 → 每张卡产出 Y 的一部分列 → 拼接得完整 Y
- **行切（Row Parallel）**：W 按行切，X 也按列切 → 每张卡算部分和 → all-reduce 累加

Transformer 里 attention 和 MLP 都是"先列切后行切"，**两次切之间不通信**，只在最后 all-reduce 一次（前向 1 次 + 反向 1 次）。

### 2. Attention 切法
- Q/K/V 三个投影矩阵按 head 切（自然切）
- 输出投影 O 按行切
- 一个 attention block 前向 1 次 all-reduce

### 3. MLP 切法
- W1 列切（fc1）
- W2 行切（fc2）
- 一个 MLP block 前向 1 次 all-reduce

### 4. 优点
- **单卡显存显著降低**：每卡只放 1/TP 参数
- **计算并行度高**：算子级并行，理论加速接近线性
- **延迟低**：相比 PP 的串行流水线，TP 是同步并行
- **无气泡**：不像 PP 有 startup/cooldown

### 5. 缺点
- **通信极重**：每个 transformer block 至少 2 次 all-reduce（前 1 反 1）
- **强依赖高速互联**：必须 NVLink 或 IB，跨节点慢得没法用
- **TP size 通常 ≤ 8**：超过单节点 NVLink 域就崩
- **batch 不可切（dim 维度）**：和 DP 不一样

### 6. TP 适用边界
- 单节点 8 卡（NVLink 域内）TP=2/4/8
- 跨节点 TP 几乎不可用
- 通常和 PP + DP 组合：**节点内 TP，节点间 PP，全局 DP**

### 7. Sequence Parallel（SP）补充
TP 中有些算子（LayerNorm、Dropout）激活没切——SP 把序列维度也切了，进一步省显存，吃 reduce-scatter 替代 all-reduce。

### 8. 一句话总结
**TP = NVLink 域内的算力榨干机**，跨节点不要用；和 PP + DP 组合是大模型训练的标配。

## 结合项目谈（口语版 · 约 40 秒）

> 我们跨境支付训练 14B 客服模型时，**TP 是我们默认开的**——单节点 8×H100 配 NVSwitch，TP=2 起，能把单卡显存压下来不少。
>
> 一个细节：我们做过 **TP=2 vs TP=4 vs TP=8** 的对比。14B 这种规模 TP=2 性价比最高——再切多了，all-reduce 通信开销吃掉了算力收益，14B 没那么大、单卡塞 7B 也不爆，TP=2 是甜点。
>
> 70B 调研时用的是 **TP=8 + PP=2**，**8 卡 TP 必须在 NVLink 域内**，跨节点带宽撑不住 all-reduce，这是 TP 的硬约束。
>
> 推理侧 vLLM 也是 TP 起家——同样跨节点 TP 几乎不可用，所以我们多副本部署而不是跨节点 TP。
>
> 一个踩坑：**Sequence Parallel 一开始没开**，激活显存比预期大很多，开了之后能多塞 50% batch size。建议长上下文场景 SP 直接默认开。
>
> 我对 TP 的总结：**算子级并行的最大杠杆，但杠杆只在 NVLink 域内有效**。
