# 如何确定你项目的评估指标迭代是否符合当前需求？

## 答案

判断评估指标是否符合当前需求，不能只看指标是否上涨，而要检查它是否能反映当前业务目标、是否覆盖关键失败模式、是否与线上结果相关。一个实用方法是建立“业务目标 -> 用户行为 -> 模型能力 -> 离线指标 -> 线上指标”的指标树。

### 1. 先明确当前阶段的目标

同一个项目在不同阶段的目标可能不同：

- 冷启动阶段：先保证基本可用和安全；
- 效果优化阶段：提升准确率、召回率或任务成功率；
- 规模化阶段：关注延迟、成本、稳定性和容量；
- 商业化阶段：关注转化、留存、收入和长期价值。

跨境支付客服 Agent 的目标不能只写“回答准确”，应拆成：

```text
正确识别问题
-> 查询正确系统
-> 给出与订单状态一致的解释
-> 完成材料引导或问题解决
-> 不产生资金、合规和隐私风险
```

### 2. 指标要覆盖效果、效率和风险

例如：

| 维度 | 指标 |
|---|---|
| 效果 | 意图准确率、工具调用准确率、事实一致率、任务完成率 |
| 效率 | P95 延迟、Token 成本、人工处理时长、转人工率 |
| 风险 | 幻觉率、越权操作率、敏感信息泄露率、错误承诺率 |
| 业务 | 工单重开率、商户满意度、一次解决率、运营成本 |

如果只优化一次解决率，模型可能通过过度承诺减少当次追问，却增加后续投诉，所以必须同时设置风险护栏。

### 3. 验证指标与真实需求的相关性

可以做三层验证：

1. **人工案例评审**：高分样本是否真的好，低分样本是否确实有问题；
2. **历史回放**：离线指标能否识别线上已知事故和高频 bad case；
3. **线上实验**：离线提升是否带来 A/B 实验中的业务改善。

还要计算不同版本的离线指标变化与线上 KPI 变化是否同向。如果长期不相关，说明指标定义、数据分布或评测集需要调整。

### 4. 做分层和切片

总平均值容易掩盖问题。跨境支付至少按以下维度切片：

- 国家和地区；
- 币种和支付方式；
- 新老商户；
- 支付、退款、出款、拒付、KYC 等问题类型；
- 高低金额；
- 正常路径与异常路径；
- 不同语言；
- 高风险和普通场景。

例如整体准确率上涨，但小语种 KYC 问题召回率下降，就不能直接上线。

### 5. 指标也要版本化

每次指标迭代要记录：

- 指标定义和计算公式；
- 数据时间范围；
- 标签来源和标注规范；
- 去重、过滤和切分方式；
- 阈值；
- 与上一版差异；
- 哪些业务风险被新增覆盖。

否则同名“准确率”在不同实验中可能根本不可比。

### 6. 上线决策

我会设置：

```text
主指标：必须显著提升
护栏指标：不得明显恶化
分层指标：关键切片必须达标
系统指标：延迟、成本、错误率在预算内
```

线上先灰度，再逐步扩量，并保留回滚条件。

### 7. 面试总结

评估指标是否符合需求，最终要看它能否预测真实业务价值并暴露风险。我会用指标树连接业务目标和模型能力，通过人工评审、历史回放、分层评测和 A/B 实验验证，并同时设置效果、效率、风险和成本护栏。
