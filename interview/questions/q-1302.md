# 三个wise训练区别

## 答案

“三个 wise”在排序场景中通常指 Pointwise、Pairwise 和 Listwise。三者的核心区别是监督单位：单个候选、候选对、完整候选列表。

### 1. Pointwise

把每个 `(query, item)` 当成独立分类或回归样本：

```text
score_i = f(query, item_i)
L = CE(label_i, sigmoid(score_i))
```

也可以预测 CTR、CVR 或多级相关性分数。

**优点**

- 数据和实现简单；
- 能利用单样本标签；
- 分数较容易做概率校准；
- 适合多任务预测。

**缺点**

- 不直接优化候选之间的顺序；
- 忽略同一 Query 内部竞争；
- 类别不均衡严重；
- 高 AUC/LogLoss 不一定意味着 Top-K 最优。

### 2. Pairwise

对同一 Query 构造正负候选对：

```text
score_pos = f(q, d_pos)
score_neg = f(q, d_neg)

L_ranknet
= -log sigmoid(score_pos - score_neg)
```

或使用 margin loss：

```text
L = max(0, margin - score_pos + score_neg)
```

**优点**

- 直接学习谁应排前；
- 相对偏好通常更易标注；
- 适合 hard negative；
- RankNet、LambdaRank、Reward Model 常用。

**缺点**

- 候选多时 pair 数量可达 `O(n^2)`；
- pair 采样会改变训练分布；
- 只优化局部顺序，不保证全列表最优；
- 分数不一定校准；
- 可能出现非传递关系。

### 3. Listwise

一次输入同一 Query 下的整个候选列表，直接优化列表分布或排序指标。

ListNet 的一种形式：

```text
P_y(i) = softmax(label_i)
P_s(i) = softmax(score_i)
L = -sum_i P_y(i) log P_s(i)
```

ListMLE 则最大化正确排列的概率：

```text
P(permutation)
= product_k
  exp(score_{pi_k})
  / sum_{j>=k} exp(score_{pi_j})
```

**优点**

- 与 NDCG、MAP 等列表目标更一致；
- 能处理多级相关性；
- 显式考虑同一列表内部关系。

**缺点**

- 每个 Query 的候选数不等，batch 和 padding 复杂；
- 计算和显存更高；
- 日志中往往只有部分曝光与标签；
- 训练列表必须接近线上候选分布；
- 对 label noise 和位置偏差敏感。

### 4. LambdaRank/LambdaMART 放在哪里

它们常基于 Pairwise 梯度，但按交换两个候选对 NDCG 的变化 `Delta NDCG` 加权：

```text
lambda_ij
≈ pairwise_gradient_ij * |Delta NDCG_ij|
```

因此训练形式像 Pairwise，优化意图更接近 Listwise 指标。

### 5. 如何选择

- 有独立概率标签、需要校准：Pointwise；
- 有点击/偏好对、重视相对顺序：Pairwise；
- 候选组稳定、有多级标签、重视 Top-K：Listwise；
- 工业系统常先 Pointwise 多任务预测，再 Pairwise/Listwise 重排；
- 召回双塔常用对比学习，本质也包含一个正样本与多个负样本的相对训练。

### 6. 跨境支付示例

对可用通道排序：

- Pointwise：分别预测每个通道成功率、成本和时效；
- Pairwise：学习同一交易下 A 通道是否优于 B；
- Listwise：对所有合规通道整体排序。

国家、币种、牌照、限额和风险属于硬过滤，不应交给排序 loss 学习。离线比较 NDCG、AUC、校准和分桶指标，线上再看成功率、成本、时效与风险 guardrail。
