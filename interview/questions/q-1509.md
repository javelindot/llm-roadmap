# 除了大模型，你还有做其他 NLP 的项目吗？

## 答案

这道题考察基础 NLP 是否扎实，以及是否所有问题都只会套大模型。应选择一个真实项目，讲清任务、数据、基线、模型、指标和上线约束。

### 1. 可选择的项目类型

- 文本分类：工单意图、情感、风险类别；
- 序列标注：NER、地址和票据字段抽取；
- 语义匹配：FAQ、搜索、相似工单；
- 文本聚类：问题发现和知识库整理；
- OCR 后处理：纠错、版面字段校验；
- 传统检索与排序：BM25、双塔、Cross-Encoder；
- 主题模型、关键词抽取或规则系统。

只能讲真正做过的项目，并准备解释为什么没有直接使用 LLM。

### 2. 示例回答

> 除了大模型，我做过支付工单意图分类和实体抽取。早期基线是 TF-IDF + Logistic Regression，优点是快、可解释；之后使用预训练 Encoder 联合预测意图和订单号、币种、国家等实体。难点是类别长尾、多语言混写和标签噪声。我们通过层级标签、hard negative、弱监督清洗和按时间切分改善。离线除 macro-F1 外还看关键类别 recall，线上看转人工率、误路由率和 P95 时延。

### 3. 为什么传统 NLP 仍有价值

- 固定标签和高吞吐任务成本低；
- 小模型延迟稳定，便于端侧或 CPU 部署；
- 结构化输出更容易约束；
- 可建立强 baseline，验证 LLM 是否真的带来增益；
- 高风险链路可把 LLM 放在解释或兜底位置。

### 4. 跨境支付边界

意图分类和实体抽取可以辅助路由工单，但订单状态、余额、制裁和合规结论必须查询权威系统。模型抽取的订单号也要格式、权限和存在性校验。

### 5. 面试总结

回答重点是展示完整 NLP 方法论：传统基线、预训练小模型、数据治理、离线指标和线上业务指标，而不是罗列算法名。
