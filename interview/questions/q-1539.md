# 如何优化 Agent 的规划效率？

## 答案

优化 Agent 规划效率的目标不是让模型“想得更短”，而是在保持任务成功率和安全性的前提下，减少不必要的规划 token、模型调用、工具调用和重规划。

### 1. 先做任务路由

- FAQ 和单工具查询不调用复杂 planner；
- 固定流程走状态机或模板；
- 多约束、多工具任务才启用动态规划；
- 高风险任务直接进入受控 workflow 和审批。

最有效的优化通常是让简单任务不规划。

### 2. 分层规划

先生成短的高层计划，只展开当前一步：

```text
goal
-> 3~5 个里程碑
-> 只细化 next action
-> 根据结果局部重规划
```

避免一次生成几十步后环境变化导致整份计划作废。稳定子流程可封装为 skill/macro tool。

### 3. 缩小工具空间

- 按意图检索相关工具，而不是把全部工具放进 Prompt；
- 工具 schema 简洁、互斥边界清楚；
- 合并高频连续调用为批量工具；
- 对相互独立的只读工具并行执行；
- 工具结果返回结构化摘要和原始引用。

工具太多会增加 token、误选和规划搜索空间。

### 4. 模型和上下文优化

- 小模型做路由和参数抽取，强模型处理复杂规划；
- 缓存稳定计划模板、工具选择和公共前缀；
- 只保留当前目标、约束、关键证据和未完成步骤；
- 使用 structured output 限制计划格式；
- 设置最大步骤、token、时间和费用预算；
- 低置信度或重复失败时及时转人工。

### 5. 执行中控制

- 每步执行后只验证受影响的前提；
- 失败按错误类型定向修复，不全量反思；
- 检测重复状态和循环；
- 达成目标立即停止；
- 使用 verifier 判断结果，不让 planner 自己宣布成功；
- 长任务 checkpoint，避免故障后从头规划。

### 6. 评测

同时看：

- 成功率与严重错误；
- 平均/尾部步骤数；
- 规划 token 和模型调用数；
- 无效、重复和失败工具调用；
- 首次计划命中率、重规划率；
- P95、成本和人工接管。

只降低步骤数可能把工作转移到更昂贵的大模型，必须看端到端成本。

### 7. 跨境支付场景

“查询订单状态”直接路由单工具；“调查跨境付款失败”才展开订单、通道、账务和规则检查。退款、冻结等动作进入固定审批流程，禁止 planner 自行搜索可执行路径。

### 8. 面试总结

规划效率优化的优先级是：任务路由、分层按需展开、工具检索与 skill 化、并行只读调用、状态压缩、局部重规划和预算/停止条件。
