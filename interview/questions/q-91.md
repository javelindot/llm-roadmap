# 数据增强的目的是什么？常用方法有哪些？

## 答案

**一句话**：数据增强 = **在不增加新样本的前提下扩充数据多样性**，目的是**提升泛化、降低过拟合、平衡类别**——CV 有几何/颜色增强，NLP 有同义词/回译/EDA，LLM 时代主要是**模型生成增强**和**指令重写**。

### 1. 三个目的
- **提升泛化**：模型看到更多分布变体，OOD 鲁棒性更强
- **降低过拟合**：避免对训练集的死记
- **解决类别不均衡**：稀有类别多增强几倍

### 2. CV 数据增强
- 几何：翻转、旋转、缩放、裁剪、Affine
- 颜色：亮度、对比度、饱和度、HSV
- 噪声：高斯噪声、椒盐
- 高级：MixUp、CutMix、AutoAugment、RandAugment

### 3. NLP 传统数据增强
| 方法 | 思路 | 风险 |
|------|------|------|
| **同义词替换** | 用 WordNet 替换 | 改变语义 |
| **EDA**（Easy Data Aug）| 替换/插入/交换/删除 | 简单粗暴 |
| **回译**（Back Translation） | 中→英→中 | 质量看翻译模型 |
| **TF-IDF 替换** | 替换低 TF-IDF 词 | 保留关键信息 |
| **Mixup-NLP** | embedding 级混合 | 语义可能塌 |

### 4. LLM 时代的数据增强
- **模型生成增强**：让 LLM 改写/扩写同一条样本，生成 N 种表达
- **指令重写**：同一答案换 N 种问法
- **难度递增（self-instruct）**：让 LLM 自己出题
- **思维链生成**：用 GPT-4 给样本加 CoT
- **Reject sampling 增强**：让模型多采样、选高分回炒

### 5. 对齐场景的数据增强
- 偏好对增强：同一 prompt 不同模型采样配对
- length-balanced sampling：避免长度偏置
- 难度分桶：按 reward 差值分桶采样

### 6. 关键原则
- 增强必须**保留任务标签**（同义替换不能改类别）
- 多样性 > 数量
- 增强后要做**质量过滤**（GPT-4 judge 抽检）
- 不能让分布远离测试集

### 7. 一句话总结
**CV 几何/颜色增强、NLP 同义/回译增强、LLM 时代靠模型生成增强**——核心思想都是"造更多分布内的样本，让模型见多识广"。

## 结合项目谈（口语版 · 约 30 秒）

> 我们跨境支付里数据增强是**业务模型常态**，因为业务数据天然不均衡——可疑交易只占 0.1%、特殊客诉占 1%，不增强模型根本学不到这些类别。
>
> 三个具体场景：
>
> **场景一：风控类别不均衡**。可疑交易标注样本只有 5k 条，正常交易 50w 条，类别比 1:100。我们做了三件事：负样本下采样到 5w、可疑样本用 GPT-4 改写扩到 2w、加上 SMOTE-NLP 风格的 embedding 级混合。最终 F1 提升 18 个点。
>
> **场景二：客服 prompt 多样性**。客户问"我的退款到账了吗"有 100 种表达，原始数据只覆盖 20 种。我们让 GPT-4 对每条样本生成 5 种改写，**保持答案不变**，prompt 多样性大幅提升。这就是经典的 instruction augmentation。
>
> **场景三：长尾偏好对**。DPO 数据集里复杂客诉的样本少，我们用 **reject sampling** ——让 SFT 模型对长尾 prompt 多采样 8 次，找到 chosen 和 rejected 配成新 pair，长尾场景准确率提升 12 个点。
>
> 一句话：**LLM 时代数据增强主力是"让模型自己生成"**，但**必须用更强的模型做质量过滤**，否则会污染分布。

**🪝 顺嘴带出去的钩子**：
- "GPT-4 改写生成数据的质量过滤流程我们有方法。"
- "可疑交易类别不均衡的增强策略可以聊。"
- "reject sampling 数据增强在 DPO 上的效果有数据。"
