# 强化学习奖励模型怎么训练的

## 答案

Reward Model（RM）给定 prompt 和完整回答输出标量分数，使人类偏好的回答得分更高。

### 1. 数据

对同一 prompt 从一个或多个策略模型采样多个回答，由标注者排序或成对比较，形成：

\[
(x,y_w,y_l)
\]

其中 \(y_w\) 是 chosen，\(y_l\) 是 rejected。数据按 prompt 切分，覆盖正常、安全、长短、语言和困难样本，并控制长度与模型来源偏差。

### 2. 模型

在预训练/SFT 模型顶部增加 scalar head，取 EOS 或最后非 padding token 的 hidden state：

\[
r_\phi(x,y)=w^\top h_{\rm EOS}
\]

也可用多头 RM 分别预测正确性、安全和风格。

### 3. Loss

Bradley-Terry 模型：

\[
P(y_w\succ y_l|x)
=\sigma(r_w-r_l)
\]

\[
\mathcal L_{\rm RM}
=-\log\sigma(r_w-r_l)
\]

多候选可构造多个 pair 或使用 listwise loss。绝对 reward 通常没有跨 prompt 的统一意义。

### 4. 评估

- held-out pairwise accuracy；
- 按领域、长度、语言和安全切片；
- reward 与长度/格式/模型来源相关性；
- OOD、对抗和高分错误样本；
- 与人工盲评的一致性；
- policy 优化后是否出现 reward 上升但真实胜率下降。

### 5. 迭代

随着 policy 变化，持续收集新生成的困难偏好，重训或 ensemble RM，并用 KL、独立 Judge 和规则防 reward hacking。

跨境支付中，事实与权限用可验证规则打分，RM 主要评价表达和帮助性，不能决定资金操作。

### 面试总结

RM 用同 prompt 的 chosen/rejected 训练标量排序头，常用 pairwise logistic loss。重点难点是偏好数据质量、长度捷径、分布漂移和 reward hacking，而不是只看训练准确率。
