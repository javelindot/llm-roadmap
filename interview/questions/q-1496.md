# SGD随机梯度分类器原理

## 答案

SGD 随机梯度分类器通常指“用随机梯度下降训练的线性分类器”，例如 scikit-learn 的 `SGDClassifier`。SGD 是优化方法，不是独立的分类模型结构。

### 1. 模型与目标

线性分类器的打分为：

```text
z = w^T x + b
```

根据损失函数可得到不同模型：

- `hinge loss`：近似线性 SVM；
- `log loss`：逻辑回归，可输出概率；
- modified Huber 等鲁棒损失。

训练目标通常为：

```text
min  average_loss(y, w^T x + b) + lambda * regularization(w)
```

正则可选 L1、L2 或 Elastic Net。

### 2. 随机更新

每次随机取一个样本或小批次：

```text
w <- w - learning_rate * gradient(loss + regularization)
```

遍历多轮并打乱样本。与全量梯度下降相比，单次更新成本低，可处理超大规模稀疏数据，也支持 `partial_fit` 增量学习，但梯度噪声更大、对学习率较敏感。

### 3. 使用要点

- 数值特征先标准化，否则不同尺度会导致优化困难；
- 稀疏文本可直接使用稀疏矩阵；
- 设置学习率调度、早停和迭代轮数；
- 类别不平衡使用 class weight、采样和合适指标；
- 概率输出要检查校准；
- 时间数据按时间切分，避免未来信息泄漏。

### 4. 跨境支付场景

可用它对高维稀疏的商户、国家、币种、MCC 和设备特征做快速风险基线，并通过增量训练适应新数据。优点是训练和推理快、系数可解释；缺点是只能学习线性边界，复杂交互需显式构造或换非线性模型。

### 5. 面试总结

SGDClassifier 是“线性模型 + 指定损失 + 正则化 + 随机梯度更新”。回答时要区分优化器与模型，并说明 hinge/log loss、学习率、标准化和增量学习。
