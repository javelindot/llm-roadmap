# 多路召回策略中各策略召回文档的好坏如何评估

## 答案

评估多路召回不能只看每一路“召回了多少”，而要看相关性、独占贡献、与其他路的重合、覆盖范围以及延迟成本。目标是判断该通道是否为融合结果提供了不可替代的增量候选。

### 1. 建立统一评测集

评测集应包含：

- query/用户上下文；
- 一个或多个相关文档；
- 多级相关性标签；
- 时间、国家、语言、用户类型等切片；
- 无答案 query；
- 时间切分后的新文档和长尾文档。

RAG 场景还要标注正确文档版本和权限范围。

### 2. 单路质量

每一路分别计算：

- Recall@K；
- HitRate@K；
- Precision@K；
- MRR/NDCG@K；
- 空结果率；
- 文档和主题覆盖率；
- 长尾、新文档和多语言覆盖；
- P95 延迟、索引新鲜度和单位成本。

### 3. 独占和边际贡献

最重要的是：

```text
独占命中率 =
只有该通道命中的正例 / 全部正例
```

以及：

```text
边际增益 =
融合全部通道的 Recall
- 移除该通道后的 Recall
```

还可做逐路加入实验，观察增量曲线。某一路单独 Recall 不低，但和主通道高度重合，边际贡献可能很小。

### 4. 通道重合与互补性

- 候选 Jaccard overlap；
- 正例重合率；
- 多路命中次数；
- source-rank 分布；
- 不同场景下的互补性；
- 被后续 Reranker 选入 top-K 的比例。

多路命中可作为相关性信号，但也要防止所有通道都只召回热门文档。

### 5. 消融与下游评测

只评召回不够，还要比较：

- 移除通道后 NDCG、回答正确率的变化；
- 该通道候选被粗排/精排保留的比例；
- 最终引用正确率；
- 线上点击、任务完成或问题解决率；
- 通道超时时系统是否可降级。

### 6. 跨境支付 RAG 示例

可以有：

- BM25：精确匹配错误码；
- 向量召回：匹配语义相近问题；
- 结构化召回：按国家、币种、产品、版本过滤；
- 图谱召回：关联支付状态、通道和处理流程；
- 热点召回：当前大面积通道故障公告。

结构化通道可能总量不大，却能准确命中国家和版本约束，独占价值很高。权限和时效错误属于硬错误，不能被总 Recall 掩盖。

### 7. 面试总结

多路召回应按“单路 Recall + 独占命中 + 边际增益 + 重合互补 + 下游收益 + 延迟成本”评估。真正有价值的通道不是返回最多，而是在关键场景提供其他通道找不到的正确候选。
