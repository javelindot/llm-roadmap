# 我只说了过去word2vec窗口太小什么，还有很多其实：具体见图

## 答案

如果问题是在比较 Word2Vec 与 BERT/上下文模型，“窗口太小”只是一个表面差异。Word2Vec 的主要限制还有以下几点。

### 1. 静态词向量

一个词只有一个固定向量，无法区分多义词。例如“bank”在银行与河岸语境下表示相同。BERT 会根据整句上下文为每次出现生成不同表示。

### 2. 局部共现目标

Skip-gram/CBOW 主要根据固定窗口预测中心词或上下文词，擅长学习局部共现，但不显式建模任意两位置的动态关系。扩大窗口会增加噪声，并不能等价于 Self-Attention。

### 3. 组合性不足

Word2Vec 给出词级表示，却不直接提供完整句子编码器；词序、否定、指代和句法组合需要额外模型。BERT 通过多层 Transformer 联合建模上下文。

### 4. OOV 与词形问题

传统词表对未登录词和低频词不友好。现代模型使用 WordPiece/BPE 等子词切分，虽然仍可能切分不理想，但可减少完全 OOV。

### 5. 预训练与任务迁移

Word2Vec 通常作为固定或轻量微调的词嵌入；BERT 可端到端微调，也可通过 Adapter/LoRA 适配分类、抽取和检索任务。

### 6. 仍然有价值的地方

Word2Vec 训练和推理便宜、可解释、适合轻量相似词和冷启动 baseline。现代方法更强不代表所有场景都需要大模型。

跨境支付中，“chargeback”“settlement”“pending”等词依赖上下文，BERT 类模型更适合理解；但海量商户或商品 ID 的简单近邻表示仍可能使用 Word2Vec 思想。

### 面试总结

Word2Vec 的局限不仅是窗口小，更关键是静态、局部、缺少组合语义和 OOV 处理。BERT 用子词、双向上下文和端到端迁移改进这些问题，但代价是更高的计算与服务成本。
