# LoRA微调时的优化目标是什么？用的什么LOSS？

## 答案

LoRA 改变的是“哪些参数参与优化”，通常不改变任务本身的优化目标。做 SFT 时仍使用自回归语言模型的 token-level Cross Entropy，只是冻结原权重 `W`，训练低秩增量 `BA`。

### 1. LoRA 参数化

```text
W' = W + Delta W
Delta W = (alpha / r) * B * A
```

其中：

```text
A: [r, d_in]
B: [d_out, r]
r << min(d_in, d_out)
```

训练时 `W` 冻结，只更新 `A/B`。

### 2. SFT 的 Loss

对目标 token：

```text
L_CE
= -1/N * sum_t m_t log p_theta(y_t | x, y_<t)
```

`m_t` 是 loss mask：

- system/user/padding：通常不计 loss；
- assistant response：计 loss；
- 工具调用任务可选择监督 tool-call token；
- 多轮对话要明确哪些 assistant turn 参与训练。

PyTorch 中通常是：

```python
loss = cross_entropy(
    logits.view(-1, vocab_size),
    labels.view(-1),
    ignore_index=-100,
)
```

### 3. 不同任务的 Loss

LoRA 也可用于其他目标：

- 分类：Cross Entropy/BCE；
- 回归：MSE/Huber；
- Embedding：contrastive、triplet、InfoNCE；
- Reranker：pointwise/pairwise/listwise；
- DPO：偏好对比目标；
- PPO/GRPO：策略目标 + KL；
- 多模态：语言 loss + 视觉/对齐辅助 loss。

所以“LoRA 用什么 loss”应回答“取决于下游任务，LoRA 本身没有专属 loss”。

### 4. 是否加正则

可以对 LoRA 参数使用 dropout、weight decay、早停等，但不是必须额外添加“低秩损失”。rank 本身已形成容量约束。某些变体会改变缩放、分解或优化器，但主任务 loss 仍由任务定义。

### 5. 常见错误

- 未屏蔽 Prompt，模型学会复述用户输入；
- padding 参与 loss；
- logits 和 labels 未右移或重复右移；
- 量化基座未正确 prepare；
- tool result 被当成模型应生成内容；
- 只看训练 loss，不看业务准确率。

**面试总结**：SFT 时优化目标仍是最大化正确回答 token 的条件似然，LoRA 只是把更新限制在低秩子空间。
