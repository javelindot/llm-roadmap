# Lora的优势和缺点是什么？

## 答案

LoRA 的核心优势是用少量低秩参数适配大模型，降低训练显存和版本存储；核心缺点是容量受限、超参数和插层敏感，而且它只减少反向训练成本，不会让基础模型的前向计算消失。

### 1. 优势

**参数高效**

```text
Delta W = B A
```

可训练参数从 `d_out*d_in` 降为 `r*(d_in+d_out)`。

**显存和训练成本低**

- 冻结基座，无需保存全部梯度和优化器状态；
- 可结合 QLoRA 在单卡训练较大模型；
- 迭代速度快。

**多任务管理方便**

- 一个基座挂多个 Adapter；
- checkpoint 小；
- 可动态加载、组合或按租户切换；
- 合并后通常不增加额外推理算子。

**保留基座能力**

增量式更新比小数据全参微调更不容易破坏通用能力。

### 2. 缺点

**容量有限**

复杂领域迁移、新语言和深层能力改变可能受 rank 限制。

**超参数敏感**

- rank；
- alpha/scaling；
- target modules；
- dropout；
- learning rate；
- 初始化。

**不一定减少推理成本**

基座仍需完整前向；未合并、多 Adapter 或动态路由还会增加管理和少量计算开销。

**合并与组合风险**

- base revision 不匹配；
- 多 Adapter 冲突；
- 量化权重合并复杂；
- 不同模板和 tokenizer 造成效果偏差。

**容易把动态知识写死**

用 LoRA 训练费率、名单和规则后，更新与删除困难。

### 3. 什么时候适合

- 风格、格式和工具协议；
- 中小规模高质量领域数据；
- 多客户/多任务 Adapter；
- 算力有限、快速迭代。

### 4. 什么时候考虑其他方案

- 动态事实：RAG/Tool；
- 只有离线偏好：DPO；
- 大规模领域无标注数据：继续预训练；
- LoRA 容量经消融证明不足：部分解冻或全参；
- 端侧推理：还需蒸馏和量化。

### 5. 跨境支付

LoRA 可学习客服语气、调查报告和工具格式；交易状态、汇率、费率、制裁名单及审批规则由实时系统提供。

**面试总结**：LoRA 解决“如何低成本改变模型行为”，不解决知识时效、事实核验和推理服务成本的全部问题。
