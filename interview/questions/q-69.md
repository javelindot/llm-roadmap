# 为什么 Adam + L2 正则化效果不好？AdamW 是如何解决的？

## 答案

**一句话**：Adam 里的 L2 正则会被**二阶矩 v 缩放掉**，导致大梯度参数被过度衰减、小梯度参数几乎不衰减；**AdamW 把 weight decay 从梯度里拆出来**，直接作用于参数本身，恢复"正则应有的样子"。

### 1. L2 和 Weight Decay 的本质
- **L2**：往 loss 加 `λ × ||θ||²`，反传时梯度多一个 `λθ`
- **Weight Decay**：每步更新时把参数乘 `(1 - λ × lr)`
- 在 **SGD 下两者数学等价**

### 2. 为什么在 Adam 下不等价
Adam 更新规则：
$$\theta \leftarrow \theta - lr \cdot \frac{\hat{m}}{\sqrt{\hat{v}} + \epsilon}$$

如果加 L2，等价于梯度变成 `g + λθ`，然后整个被 `√v` 缩放：
- **梯度大的参数**：v 大，缩放严重，L2 项贡献被压扁
- **梯度小的参数**：v 小，缩放轻，L2 项相对放大

→ 结果是"该正则的没被正则、不该被正则的反而正则化得猛"，**严重违背 L2 的初衷**。

### 3. AdamW 怎么解
直接把 weight decay 从梯度里**拆出来**：
$$\theta \leftarrow \theta - lr \cdot \frac{\hat{m}}{\sqrt{\hat{v}} + \epsilon} - lr \cdot \lambda \cdot \theta$$

- 梯度项还是 Adam 自适应缩放
- weight decay 项**直接作用于 θ**，不被 v 缩放
- 恢复了原本"每个参数等比例衰减"的效果

### 4. 实验结果（Loshchilov 2019 原论文）
- AdamW 在 ImageNet / WMT 上比 Adam + L2 提升 1-3%
- 收敛更快、泛化更好
- 训练大模型几乎都用 AdamW

### 5. 工程现实
- PyTorch 里 `optim.AdamW(...)` 直接用
- HuggingFace 默认 AdamW
- weight_decay 经验值：**0.01 - 0.1**（LLM 训练）
- LayerNorm / Embedding 通常**不加 weight decay**（NO_DECAY_PARAMS）

### 6. 一句话总结
**Adam + L2 = "假正则"，AdamW = "真正则"**——LLM 训练几乎全用 AdamW，weight_decay=0.01-0.1，norm/embedding 排除在外。

## 结合项目谈（口语版 · 约 30 秒）

> 我们跨境支付训练**默认 AdamW**，这没什么好选的——HuggingFace、DeepSpeed、Megatron 全是 AdamW 默认。
>
> 但有个细节我们踩过：一开始我们直接 `optim.AdamW(model.parameters(), weight_decay=0.01)`，**把 LayerNorm 和 embedding 也 decay 进去了**，跑了一段发现 loss 比预期高一些。后来排查发现这两类参数本来就该 0 decay——LayerNorm 的 γ、β 是缩放因子不该衰减；embedding 是查找表也不该衰减。
>
> 改成 **NO_DECAY_PARAMS 分组**之后效果改善：
> ```python
> no_decay = ["bias", "LayerNorm", "layernorm", "embed", "norm"]
> params = [
>   {"params": [p for n,p in model.named_parameters() if not any(nd in n for nd in no_decay)], "weight_decay": 0.01},
>   {"params": [p for n,p in model.named_parameters() if any(nd in n for nd in no_decay)], "weight_decay": 0.0},
> ]
> ```
>
> 这块 HuggingFace 的 Trainer 默认就是这么做的，自己写训练脚本要注意。
>
> 我对这道题的总结：**AdamW 是答案，但工程里 weight_decay 怎么分组比"用 AdamW"更值得讲**——这是真有人调过参才会注意的。
