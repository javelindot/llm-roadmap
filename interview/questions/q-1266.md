# 除了向量召回和关键词召回，还有哪些召回方式，了不了解全文召回，如何做的？

## 答案

召回不是只有 Dense Vector 和简单关键词匹配。生产系统通常按“精确、语义、实体、行为、结构和时效”建立多路召回，再统一融合与重排。

### 1. 全文召回是什么

全文检索先对文档做分析并建立倒排索引：

```text
term → [(doc_id, term_frequency, positions), ...]
```

在线 Query 经过相同或兼容的 analyzer，查找包含 term 的文档并使用 BM25 等算法打分。

离线流程：

1. 文档字段设计；
2. 分词、大小写和 Unicode 归一化；
3. 停用词、词干和同义词处理；
4. 建立 term dictionary 与 posting list；
5. 保存词频、文档频率、字段长度和位置；
6. 设置版本、权限和生效时间字段。

在线流程：

1. Query 分词与术语归一化；
2. Boolean/phrase/prefix/fuzzy 查询；
3. ACL 和元数据过滤；
4. BM25 字段加权；
5. 返回候选与高亮；
6. 与其他召回融合、rerank。

全文召回不等于只做字符串包含。它支持短语、字段、位置、同义词、模糊匹配、过滤和相关性排序。

### 2. 其他召回方式

**FAQ/规则召回**

标准问、同义问、错误码和确定性映射，延迟低、可解释。

**实体召回**

识别商品、商户、国家、币种、机构或订单实体，按实体 ID、别名和关系查询。

**结构化检索**

SQL、列式存储、时序库和业务 API，适合实时状态、范围、聚合和精确条件。

**图召回**

通过知识图谱或行为图做邻居、多跳和路径召回。

**协同过滤**

ItemCF、UserCF、矩阵分解，根据用户-物品行为相似召回。

**双塔/多兴趣召回**

将用户/Query 与候选编码为向量，ANN 检索；多兴趣模型为一个用户生成多个向量。

**Learned Sparse Retrieval**

如 SPLADE，学习高维稀疏词项权重，兼顾神经语义与倒排索引。

**时效/热门/规则召回**

新内容、趋势、运营白名单或安全兜底，通常作为补充通道。

### 3. 跨境支付中的组合

- 错误码、SWIFT/SEPA/PIX、API 字段：全文/BM25；
- “款没到”等口语：Dense；
- 商户、账户、通道：实体或结构化工具；
- 国家、币种和产品关系：图或元数据；
- 当前订单、余额和汇率：业务 API；
- 高频标准问题：FAQ。

权限、租户和生效时间应在召回前过滤。不同通道返回的候选随后去重、融合和精排，不能把所有结果直接塞给模型。
