# 做过纯 CV 的任务吗？比如图像识别、语义分割、图像理解等

## 答案

> 纯 CV 我做过几个，但说实话**多模态（CV + LLM）才是我的主战场**，纯 CV 任务我做过下面几个：
>
> **1. KYB 证件版面分割**（语义分割，做过）
> - **任务**：商户上传的营业执照、税单，识别哪块是公司名、哪块是注册号、哪块是印章
> - **方案**：Attention U-Net，输入 512×512，输出像素级 mask
> - **数据**：5w 张标注证件，按字段类别分 8 类
> - **训练**：单卡 A100 跑 24 小时，mIoU 92%
> - **trick**：模拟印章遮挡、低质量扫描的数据增强
>
> **2. 证件 OCR 检测**（图像识别 + 检测，做过）
> - **任务**：在证件图像里定位文本框位置
> - **方案**：PaddleOCR 现成模型 + 业务数据微调
> - **结果**：检测准确率 95%，配合 LLM 识别字段
>
> **3. 客户头像质量检测**（图像分类，做过）
> - **任务**：KYC 上传的身份证 selfie 是否合规（清晰、正脸、无遮挡）
> - **方案**：MobileNet 轻量分类器，5 个标签
> - **训练**：迁移学习 + finetune
> - **结果**：实时部署，单图推理 <50ms
>
> **4. 商户营业场所图像识别**（场景分类，调研未生产）
> - **任务**：商户上传经营场所照片，识别"店铺 / 办公室 / 居家 / 仓库"
> - **方案**：ViT-B + linear probe
> - **数据少**：仅 1w 张，效果一般，最后切到 Qwen-VL 多模态做
>
> **5. 多模态 VLM 任务**（主战场）
> - 证件字段提取（ViT + 投影 + Qwen-14B）
> - 客户头像 vs 证件照人脸比对（CLIP face embedding）
> - 流程图理解、UI 截图理解（Qwen-VL）
>
> **没做过 / 不擅长的**：
> - 自动驾驶 / 实时目标检测（YOLO 家族）
> - 医学影像
> - 3D 视觉（点云、SLAM）
> - 生成类（StyleGAN、SD 训练）
>
> **我的 CV 技术栈**：
> - **检测 / 分割**：PaddleOCR、MMDetection、Detectron2 一般够用，不自训
> - **分类**：迁移学习 + ResNet / ViT / EfficientNet
> - **多模态**：CLIP、ViT、LLaVA、Qwen-VL（更熟）
> - **生成**：SD 系列、ControlNet（业务侧短期试过）
>
> **几个观察**：
>
> 1. **纯 CV 在工业里大部分被 VLM 替代**——以前要训分类器的任务，现在 Qwen-VL prompt 一下就行
>
> 2. **小数据场景仍需要 CV**——5k 以下样本，VLM 不如专门微调的小 CNN
>
> 3. **实时性 CV 仍不可替代**——目标检测、人脸识别 < 10ms 的场景 VLM 跑不动
>
> 4. **金融场景结构化任务（证件 / 票据）适合 VLM**——可解释、可追溯
>
> 一句话：**我做过纯 CV 但不是专精**——LLM 时代 CV 任务很多被 VLM 蚕食，我更擅长把"视觉理解 + 语言推理"组合起来用。
