# Deepspeed的三阶段zero是什么

## 答案

DeepSpeed ZeRO 通过在数据并行进程间切分训练状态，消除每张 GPU 上重复保存完整副本的浪费。

### 1. 三阶段

| 阶段 | 分片对象 | 节省显存 | 主要代价 |
|---|---|---|---|
| ZeRO-1 | 优化器状态 | 尤其节省 Adam 的一阶、二阶矩 | 参数和梯度仍复制 |
| ZeRO-2 | 优化器状态、梯度 | 再节省梯度显存 | 更多梯度通信 |
| ZeRO-3 | 优化器状态、梯度、模型参数 | 三类状态都按数据并行度分片 | 前后向按需聚合参数，通信和工程复杂度最高 |

记忆方式：

```text
Stage 1: optimizer states
Stage 2: + gradients
Stage 3: + parameters
```

### 2. 需要注意

- ZeRO 主要处理模型状态，不自动消除激活显存；
- 激活可用 gradient checkpointing、序列并行等优化；
- ZeRO-Offload 可把部分状态放到 CPU，ZeRO-Infinity 可进一步利用 NVMe；
- Stage 3 checkpoint 保存和加载要正确处理分片；
- Stage 越高不一定越快，需结合模型大小和网络带宽选择；
- LoRA 与全参数训练的显存瓶颈不同，不能照搬配置。

### 3. 面试总结

ZeRO-1 分优化器状态，ZeRO-2 再分梯度，ZeRO-3 再分模型参数。它用更多通信和复杂度换取更低单卡显存，选择阶段时要看模型是否能放下、网络拓扑和目标吞吐。
