# React项目中使用过哪些Hooks及其原理

## 答案

React Hooks 让函数组件使用状态、Effect、Context、Ref 和并发特性。回答项目题时应说明用在什么场景，而不只是列名称。

### 常用 Hooks

- `useState`：保存影响渲染的局部状态。set 函数把更新加入队列，React 在下一次 render 使用新 state；连续依赖旧值时使用函数式更新。
- `useReducer`：状态转移复杂或多个字段相关时，以 reducer 统一管理 action。
- `useEffect`：在 commit 后与网络、订阅、定时器等外部系统同步；依赖变化前先执行上次 cleanup。它不是普通数据派生工具。
- `useLayoutEffect`：DOM 变更后、浏览器绘制前同步执行，适合测量布局，但会阻塞绘制。
- `useRef`：保存跨 render 的可变值或 DOM 引用，修改 `current` 不触发渲染。
- `useContext`：读取最近 Provider 的值；Provider value 变化会使订阅组件重新渲染。
- `useMemo`/`useCallback`：缓存计算结果或函数引用，是性能优化而非语义保证，应在实际重算或子组件 memo 场景使用。
- `useTransition`、`useDeferredValue`：将非紧急更新延后，保持输入交互响应。

### 原理与规则

React 按组件每次 render 中 Hook 的调用顺序，将状态槽与调用对应。因此 Hook 必须在组件顶层调用，不能放在条件、循环或普通函数中。状态更新会触发重新渲染，闭包捕获的是创建该 render 时的 props/state。

自定义 Hook 只是组合其他 Hooks 复用有状态逻辑，不共享状态实例。项目中还要关注依赖数组、Effect cleanup、竞态取消和不必要重渲染，不能通过滥用 memo 掩盖状态设计问题。
