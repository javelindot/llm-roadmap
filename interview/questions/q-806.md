# 🔥🔥 位置编码

## 答案

Attention 对 token 顺序本身不敏感，位置编码用于告诉模型“谁在前、谁在后、相距多远”。

### 1. 绝对位置编码

- 原始 Transformer 使用固定正弦/余弦：

\[
PE_{(pos,2i)}=\sin(pos/10000^{2i/d})
\]

\[
PE_{(pos,2i+1)}=\cos(pos/10000^{2i/d})
\]

- BERT 使用可学习绝对位置 embedding。

它们通常与 token embedding 相加，实现简单，但可学习表超过训练长度没有参数，外推困难。

### 2. 相对位置

在 Attention score 或 value 中加入相对距离 bias。T5 relative bias、Transformer-XL、DeBERTa 等属于此类。它更直接表示 token 间距离，但实现和缓存方式不同。

### 3. RoPE

对 Q/K 做与位置相关的二维旋转，使点积依赖相对位置。LLaMA、Qwen 等广泛采用，可结合位置插值、YaRN 等扩长，但不能天然无限外推。

### 4. ALiBi

按头给 Attention score 加随距离线性递减的 bias，不增加位置 embedding，外推和实现较简洁，但表达形式比 RoPE 更受限。

### 5. 选型与评估

比较长度外推、短上下文损失、KV Cache 兼容性、计算开销和目标硬件。长文本还受训练长度、Attention 复杂度和 lost-in-the-middle 影响，位置编码只是其中一环。

### 面试总结

位置编码主要分绝对、相对、RoPE 和 ALiBi。现代 LLM 常用 RoPE，因为它把相对距离融入 Q/K 点积且便于扩长；但真正的长上下文能力还需要长序列训练和系统支持。
