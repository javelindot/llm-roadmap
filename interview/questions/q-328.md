# 有没有Agent的记忆系统？

## 答案

答：短期滑动窗口记忆、结构化工作记忆、向量长期记忆、用户持久化记忆。

### 补充说明

Agent 记忆可以分层理解：

- **短期记忆**：当前会话最近几轮对话，用于保持连续性。
- **工作记忆**：当前任务的结构化状态，比如槽位、工具结果、执行步骤。
- **长期记忆**：跨会话保存的用户偏好、历史问题、常用实体。
- **知识记忆**：通过 RAG 检索企业知识库，不直接放在对话上下文里。

工程上不要把所有记忆都塞进 prompt。更合理的是短期上下文放模型，结构化状态放 State/Redis/DB，长期记忆用向量库或关系库按需召回。

### 结合跨境支付项目怎么说（面试口径）

> 在跨境支付客服 Agent 里，我会把记忆分成几层。短期记忆用于理解用户连续追问，比如用户先问“这笔付款为什么失败”，后面问“那怎么处理”，系统要知道还是同一笔订单。工作记忆保存订单号、交易状态、通道回执、风控大类和已查询过的工具结果。
>
> 长期记忆我会非常谨慎。比如商户常用币种、常见问题类型可以保存，但银行卡号、证件号、完整风控细节不能直接作为长期记忆存进模型侧。涉及敏感信息要脱敏、设置过期时间，并受权限控制。
>
> 所以我理解 Agent 记忆不是越多越好，而是分层、可控、可审计。支付场景尤其要避免因为记忆污染导致越权回答或拿旧状态回答新问题。

**钩子**：可以引到“短期记忆、工作记忆、长期记忆、敏感信息不能无脑持久化”。
