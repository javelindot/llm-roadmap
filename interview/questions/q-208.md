# 项目中使用过哪些大语言模型（LLM）？不同大语言模型的核心区别是什么？各自适配什么样的业务场景？

## 答案

**一句话**：不同 LLM 的差异主要体现在推理能力、中文能力、工具调用、长上下文、多模态、成本、私有化部署和生态支持上，选型不能只看榜单，要看业务场景。

### 1. 项目中可能使用过哪些模型

常见会接触几类：

- **商用闭源模型**：如 GPT、Claude、Gemini、通义、文心、豆包等，优点是效果稳定、工具调用和推理能力强，缺点是成本、数据合规和供应商依赖需要评估。
- **开源模型**：如 Qwen、Llama、DeepSeek、Yi、GLM 等，优点是可私有化、可微调、数据可控，缺点是部署、推理优化和评估成本更高。
- **Embedding 模型**：用于 RAG 向量检索，重点看语义召回、中文效果、维度、吞吐和成本。
- **Rerank 模型**：用于对召回结果重排，提升最终上下文质量。
- **多模态模型**：用于图片、票据、PDF 截图、证件材料理解。

### 2. 核心区别

选模型时我主要看这些维度：

- **推理能力**：复杂问题拆解、数学、代码、长链路分析。
- **工具调用能力**：能不能稳定生成结构化参数、按流程调用工具。
- **中文和领域术语能力**：金融、支付、合规术语是否理解准确。
- **长上下文能力**：能不能处理长合同、长工单、交易日志。
- **多模态能力**：能不能理解图片、PDF、证件、截图。
- **私有化部署能力**：是否支持本地部署、量化、vLLM 加速。
- **成本和延迟**：高频客服场景对成本和响应速度非常敏感。

### 3. 场景适配

- 客服问答：优先稳定、便宜、中文好、支持 RAG 的模型。
- 风控解释：需要推理能力强、输出可控、不能胡编。
- 合规/KYC：更看重准确性、可审计和私有化。
- 文档抽取：需要长上下文和结构化输出能力。
- Agent 工具调用：需要 function call 稳定、格式遵循好。
- 离线评估/标注：可以用更强但成本更高的模型。

### 4. 面试总结

我不会简单说某个模型最好，而是会按任务分层：强模型做复杂推理和评估，中等模型做高频问答，小模型做分类、抽取和简单判断，Embedding/Rerank 模型负责检索质量。这样整体成本和效果更平衡。

### 5. 结合跨境支付项目怎么说（面试口径）

> 在跨境支付项目里，我会按业务场景选模型。比如客服问答量大，我会优先用成本可控、中文能力好的模型，加 RAG 来保证答案来自知识库；如果是风控拦截原因解释，我会用推理能力更强、输出更稳定的模型，因为它要把交易金额、国家、设备、商户画像和历史行为串起来分析；如果是 KYC/KYB 材料识别，就需要多模态或文档理解模型。
>
> 另外我会把模型分层使用。线上主链路不直接依赖超大模型，避免延迟和稳定性问题；复杂疑难工单、风控复核、规则分析可以调用更强模型；高频、低风险任务可以用小模型或本地模型。这样既能控制成本，也能保证关键场景的效果。
>
> 我比较关注的是模型评估闭环。每次模型切换前，会用真实工单、支付失败原因、通道回执、合规问答做测试集，评估准确率、幻觉率、引用命中率和人工满意度，而不是只看通用 benchmark。
