# 7.分层召回的结构是什么？

## 答案

“分层召回”通常有两种含义：一是工业推荐中的多阶段候选漏斗，二是先在类目或聚类层定位区域，再在区域内部检索的分层索引。面试时应先确认语境，再回答结构。

### 1. 多阶段候选漏斗

```text
全量候选
-> L0 资格与硬规则过滤
-> L1 多路宽召回
-> L2 合并、去重和轻量粗排
-> L3 高质量候选
-> 精排与重排
```

- **L0**：过滤权限、库存、地域、时效和安全限制；
- **L1**：并行执行协同过滤、向量、热门、内容、图和实时召回；
- **L2**：去重、通道配额、轻模型打分和候选截断；
- **L3**：保留数百到数千个候选送入较重的精排模型；
- **重排**：处理多样性、业务规则、探索和频控。

越靠前越强调覆盖率和吞吐，越靠后越强调精度。每层都要记录候选数、耗时、覆盖率和正样本保留率。

### 2. 分层索引

另一种结构是：

```text
用户/Query
-> 类目、主题或向量簇路由
-> 簇内 ANN/BM25 检索
-> 跨簇合并
```

它通过先缩小搜索空间降低延迟，但路由错误会造成不可恢复的召回损失，因此常保留多簇路由、全局热门和兜底召回。

### 3. 跨境支付场景

支付通道路由可设计为：

1. L0 根据牌照、制裁、币种、金额上限和商户合同过滤；
2. L1 由历史成功通道、相似交易、低成本、低时延和探索通道多路召回；
3. L2 用轻量模型估计成功率并去重、限流；
4. L3 精排综合成功率、成本、时延和风险；
5. 重排满足通道容量、集中度和熔断规则。

任何模型召回都不能重新加入 L0 已判定不合法的通道。

### 4. 面试总结

分层召回本质是用逐层缩小候选集换取性能，并通过多路召回保持覆盖。要同时回答每层输入输出、候选预算、模型复杂度、监控指标和召回损失的兜底机制。
