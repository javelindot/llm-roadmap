# 如何降低 Transformer 的计算复杂度？常见的稀疏注意力变体有哪些？

## 答案

标准 Self-Attention 对长度 $n$ 的序列需要计算 $n\times n$ 分数矩阵，时间复杂度约为 $O(n^2d)$，注意力矩阵内存为 $O(n^2)$。优化可从精确计算的内存实现、限制连接范围和低秩近似三方面进行。

### 精确但更省内存

FlashAttention 通过分块和在线 softmax 减少 HBM 读写，不显式存储完整注意力矩阵。它不改变数学结果，渐近计算复杂度仍是二次，但实际速度和显存显著改善。

MQA/GQA 主要减少 K/V 头和 KV Cache，改善自回归推理带宽，并不把 prompt self-attention 的序列复杂度降为线性。

### 稀疏注意力

- Sliding Window/Local Attention：每个 token 只看邻近窗口，复杂度约 $O(nw)$，适合局部相关任务。
- Longformer：局部窗口加少量 global token，兼顾局部与全局信息。
- BigBird：局部、随机和全局连接组合，并具有一定理论表达能力。
- Block Sparse：按块定义固定或可学习稀疏模式，便于硬件加速。
- Dilated/Strided Attention：使用膨胀或跨步连接扩大感受野。
- Routing/Reformer：通过 LSH 或聚类让相似 token 互相注意，降低候选对数量。

### 近似与压缩

Linformer 在序列维度低秩投影 K/V；Performer 用核特征近似 softmax attention；Nyströmformer 用 landmark 近似注意力矩阵。还有 memory token、分层摘要和 recurrent memory，将远程历史压缩后复用。

选择时要考虑任务需要的全局依赖、实际 kernel 支持和近似误差。理论复杂度下降不代表端到端更快，小序列或不规则稀疏可能被调度开销抵消。评估应同时看质量、吞吐、峰值显存和不同长度下的延迟。
