# 分布式训练及显存优化技术有哪些？

## 答案

**一句话**：分布式训练分**并行策略**（DP/TP/PP/SP/EP）和**显存优化**（ZeRO/Offload/Gradient Checkpointing/Mixed Precision/Flash-Attn）两类，工业实践是 **3D/4D 并行 + ZeRO + 混合精度 + LoRA** 组合拳。

### 1. 并行策略（按切什么切）
| 策略 | 切什么 | 通信代价 | 适用 |
|------|--------|---------|------|
| **DP（Data Parallel）** | 切 batch | All-reduce 梯度 | 入门、单层模型小 |
| **TP（Tensor Parallel）** | 切单层算子 | All-reduce 激活 | 同节点高速互联 |
| **PP（Pipeline Parallel）** | 切层 | P2P 激活 | 跨节点、大模型 |
| **SP（Sequence Parallel）** | 切序列长度 | 配合 TP | 长上下文 |
| **EP（Expert Parallel）** | 切 MoE expert | All-to-all | MoE 模型 |
| **CP（Context Parallel）** | Ring Attention 切 seq | 高 | 超长上下文 |

### 2. 3D / 4D 并行的典型拓扑
- 单节点内：**TP**（NVLink 高带宽）
- 跨节点：**PP**（带宽要求低）
- 全局复制：**DP**（数据切分）
- 大模型常用 **TP=8 × PP=4 × DP=N** 拼起来

### 3. 显存优化技术
- **ZeRO**（DeepSpeed）：
  - ZeRO-1：切 optimizer state（FP32 Adam state）
  - ZeRO-2：+ 切 gradients
  - ZeRO-3：+ 切 parameters（全切）
- **Offload**：把 optimizer state / 参数 offload 到 CPU/NVMe
- **Gradient Checkpointing**：用计算换显存，反传时重算激活
- **混合精度（AMP）**：FP16/BF16 计算 + FP32 master
- **Flash-Attention**：tiling + 不存中间 attention 矩阵
- **LoRA / QLoRA**：只训低秩 + 4bit 量化 base

### 4. 显存账本（粗算公式）
训练显存 ≈ **参数 × 2** (FP16) + **梯度 × 2** + **Adam state × 8** + 激活 + buffer

7B 模型全量训练 ≈ 7B × 16 = **112GB**（不含激活）
LoRA 微调 ≈ 7B × 2 + LoRA × 16 ≈ **15GB**（QLoRA 进一步压到 8GB）

### 5. 工业选型经验
- **<7B 全参**：单机 8 卡 H100/A100 + ZeRO-2 + AMP
- **7B-70B 全参**：多机 + TP + PP + ZeRO-2 / FSDP
- **>70B**：3D 并行 + ZeRO-3 + Megatron-LM / DeepSpeed
- **预算紧** / 仅做对齐：**LoRA / QLoRA**

### 6. 一句话总结
**显存优化看 ZeRO，速度优化看 3D 并行，省钱靠 LoRA + 量化**——三个旋钮一起拧，几乎所有规模都能跑起来。

## 结合项目谈（口语版 · 约 40 秒）

> 我们跨境支付训练这块主要跑过 7B-14B 量级，分享下我们的拓扑选择。
>
> **客服 SFT（Qwen-14B 全参）**：单机 8 卡 H100 + ZeRO-2 + BF16 + Gradient Checkpointing + Flash-Attention 2，batch size 32，单步 1.5 秒能跑稳。一开始上 ZeRO-3 反而慢，因为通信比省的显存代价大，14B + H100 完全用不着 ZeRO-3。
>
> **风控模型（Qwen-7B + LoRA）**：QLoRA + 4bit NF4 + LoRA rank=64，单卡 A100 就能跑，显存占 18G，训 5w 条样本 4 小时。运营那边想改规则随时重训，成本可控。
>
> **领域多模态（13B + ViT）**：用 DeepSpeed ZeRO-2 + Tensor Parallel（NVLink 域内）跑，分布式策略选得不对的话激活太大会爆。
>
> 踩过的坑：第一次用 FSDP 没开 `use_orig_params=True`，checkpoint 加载出来权重对不上；ZeRO-3 跨节点慢得离谱，最后切回 ZeRO-2；BF16 在 H100 默认开，A100 还得 FP16 + loss scaling。
>
> 我的经验是：**先估算显存账，再选并行策略，最后用 ZeRO 兜底**，不要一上来就堆复杂度。
