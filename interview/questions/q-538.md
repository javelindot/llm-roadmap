# 讲一下prompt书写的经验,如何判断prompt是否达标,如何改进prompt

## 答案

**一句话**：好的 prompt 要目标清楚、上下文足够、约束明确、输出格式稳定，并且要通过评测集验证，而不是凭感觉。

### 经验

- 明确角色和任务。
- 给必要业务背景。
- 限制回答依据和边界。
- 指定输出格式。
- 给正反例。
- 要求不确定时说明不知道。
- 用 badcase 迭代。

### 结合跨境支付项目怎么说（面试口径）

> 在跨境支付客服项目里，我会要求模型必须基于订单工具结果和 RAG 检索内容回答，不能编造通道状态，也不能承诺一定到账。输出格式会包含原因、当前状态、下一步建议和引用依据。
>
> 判断 prompt 是否达标，不看单条 demo，而看真实工单评测：准确率、幻觉率、人工改写率、转人工率是否改善。

**钩子**：可以引到“输出格式约束、依据约束、badcase 迭代、真实工单评测”。
