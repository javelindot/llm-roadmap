# LoRA中的scale参数的作用及设置方法是什么？

## 答案

LoRA 将权重更新写为：

\[
\Delta W=sBA
\]

标准缩放为：

\[
s=\frac{\alpha}{r}
\]

`r` 决定低秩容量，`alpha` 决定更新的整体强度。采用缩放后，不同 rank 的更新幅度更容易保持在可控范围。

### 设置方法

1. 先固定 target modules，从较小 rank 如 8/16 开始；
2. 选择 `alpha` 使 scale 处于基线范围，例如 `alpha=r` 或 `2r` 只是实验起点，不是定律；
3. 联合扫描 rank、alpha、learning rate 与 dropout；
4. 比较目标任务、通用能力回归、梯度范数和 \(\|\Delta W\|/\|W\|\)；
5. rank 增大后若标准 scale 过弱，可尝试 rsLoRA 的 \(\alpha/\sqrt r\)；
6. 不同 Attention/FFN 模块可使用不同 rank/alpha，但会增加调参复杂度。

### 常见误区

- scale 大不等于学习快，可能造成遗忘和输出崩坏；
- LoRA 合并后 scale 已写入权重，部署时不要重复乘；
- QLoRA 的量化误差会影响最优 scale；
- 推理时动态调 adapter scale 只能在验证过的范围内使用。

### 面试总结

LoRA scale 控制低秩分支相对基座的强度，标准形式是 \(\alpha/r\)，rsLoRA 是 \(\alpha/\sqrt r\)。没有通用最优值，应与 rank、学习率、模块和数据通过消融共同确定。
