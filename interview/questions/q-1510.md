# 看起来大模型是你的长板啊，那你能讲讲你学习大模型的路线吗？

## 答案

学习路线不应只是课程和论文清单，而要体现“原理、实现、项目、评测和生产”的递进，并用可验证产出证明掌握程度。

### 1. 基础阶段

- 线性代数、概率、优化和信息论；
- Python、PyTorch、自动微分与混合精度；
- NLP 的 tokenization、embedding、语言模型和 seq2seq；
- 手写 Attention、Transformer Block 和训练循环。

这一阶段至少能解释参数量、显存、梯度、复杂度和数值稳定性。

### 2. 预训练模型与微调

- 阅读 Transformer、BERT、GPT、Llama 等核心工作；
- 使用 Transformers 完成 SFT；
- 比较全参数、LoRA、QLoRA；
- 理解数据模板、packing、学习率、checkpoint 和评测；
- 做小规模可复现实验，而不是直接训练超大模型。

### 3. 应用系统

- RAG：解析、切分、Embedding、混合检索、Rerank 和评测；
- Agent：工具 schema、状态机、权限、记忆和失败恢复；
- 多模态：OCR、视觉编码器、投影器和文档理解；
- 建立 baseline、错误分类和端到端测试集。

### 4. 对齐与强化学习

- 先掌握 SFT、reward model、DPO；
- 再理解 PPO、GAE、GRPO、KL 和 importance sampling；
- 用可验证任务做小规模 RL；
- 重点观察 reward hacking、entropy、KL 和能力回归。

### 5. 训练与推理工程

- DDP、FSDP/ZeRO、张量/流水线/序列并行；
- FlashAttention、KV Cache、PagedAttention；
- vLLM/SGLang、continuous batching、量化和投机解码；
- TTFT、TPOT、goodput、成本和故障恢复。

### 6. 当前学习方法

对新技术采用：

```text
原始论文/官方文档
-> 最小复现
-> 与强基线对比
-> 真实负载压测
-> 写出局限与适用边界
```

不要仅追模型榜单。未来重点可放在高质量数据、可验证 RL、长上下文与记忆、推理系统、多模态 Agent 和安全评测。

### 7. 跨境支付实践

可用支付客服/RAG 做应用入口，再逐步扩展到文档理解、调查 Agent 和模型服务。整个过程中权限、审计、数据驻留、事实来源和人工审批始终作为系统约束。

### 8. 面试总结

一条可信路线是：基础原理 -> 小模型实现 -> SFT/PEFT -> RAG/Agent -> 对齐训练 -> 分布式与推理优化 -> 评测安全。每一阶段都应有代码、实验或上线结果，而不只是“看过”。
