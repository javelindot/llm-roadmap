# 了解哪些agent开发框架，例如langchain和LlamaIndex，他们核心应用场景有何不同？

## 答案

LangChain 与 LlamaIndex 都能构建 RAG 和 Agent，但历史侧重点不同，当前能力也有重叠。选型应看数据链路、工作流状态、持久化和现有生态，而不是只看组件数量。

### LangChain/LangGraph

LangChain 提供模型、Prompt、Tool、Retriever 和 structured output 等通用集成。复杂 Agent 更常使用 LangGraph，以有状态图表达节点、分支、循环、checkpoint、streaming 和 human-in-the-loop。

它适合工具调用多、流程复杂、需要持久化恢复和显式状态控制的应用。优点是集成广，缺点是抽象层多、版本变化快，需要限制依赖边界。

### LlamaIndex

LlamaIndex 起点是“让私有数据可被 LLM 使用”，在 connector、文档解析、node/chunk、index、retriever、query engine 和 RAG 评测方面较完整，也提供 Agent 与 Workflow。

它适合数据源多、索引和检索策略复杂、需要快速搭建知识问答与数据 Agent 的场景。

### 如何选择

- 重点是复杂状态机与多工具编排：优先评估 LangGraph。
- 重点是文档接入、索引和 RAG：优先评估 LlamaIndex。
- 流程固定、可靠性要求高：工作流引擎加自研薄层可能更简单。

生产评估包括 checkpoint、幂等、权限、trace、测试、升级成本和延迟。框架只解决编排，不替代工具安全、数据治理和任务评测。具体 API 应以当前官方文档为准。
