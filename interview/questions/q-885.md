# vllm是怎么加速的

## 答案

vLLM 通过 KV Cache 内存管理、请求调度和高效 kernel 共同加速 LLM 服务，主要收益常是并发吞吐和显存利用率，而非所有场景单请求都同倍加速。

### 1. Paged KV Cache

将每个请求的 KV Cache 切成固定 block，用 block table 映射到非连续显存。避免为最大长度预留连续空间，减少碎片，并支持前缀块共享和动态增长。

### 2. Continuous Batching

每个调度 step 都可加入新请求、移除完成请求，不必等待固定 batch 中最长序列结束，提高 GPU 利用率。

### 3. Prefill 与 Decode

- chunked prefill 将长 prompt 拆块，减少长 prefill 阻塞 decode；
- prefix caching 复用相同 system prompt、工具 schema 和文档前缀；
- speculative decoding 用 draft/n-gram/EAGLE/MTP 一次验证多个 token；
- disaggregated prefill/decode 可分别扩容算力密集和带宽密集阶段。

### 4. Kernel 与并行

- FlashAttention、FlashInfer、FlashMLA；
- 融合算子、CUDA Graph、编译优化；
- Tensor/Pipeline/Data/Expert/Context Parallel；
- FP8、INT8、INT4 和 KV Cache 量化；
- Multi-LoRA、结构化输出和多模态调度。

### 5. 评估

用真实 prompt/output 长度和并发测 TTFT、ITL/TPOT、吞吐、P99、显存和单位成本。continuous batching 提升吞吐时可能增加低并发延迟，需要调 scheduler。

当前能力参考 [vLLM 官方文档](https://docs.vllm.ai/en/stable/)。

### 面试总结

vLLM 的核心是分页 KV Cache 与 continuous batching，再叠加 chunked prefill、prefix cache、speculative decoding、量化和并行。它主要解决动态并发服务的内存与调度效率。
