# 10. 4.18 讲讲attention的各个变体

## 答案

Attention 变体可以按“信息来源、可见范围、头共享、复杂度和系统实现”分类。

### 1. 信息来源

- **Self-Attention**：Q/K/V 来自同一序列；
- **Cross-Attention**：Q 来自当前序列，K/V 来自编码器、图像或外部记忆；
- **Co-Attention**：两个模态或序列双向交互。

### 2. 可见范围

- **Bidirectional**：BERT 可看左右上下文；
- **Causal**：Decoder 只能看当前及之前 token；
- **Sliding Window/Local**：只看附近窗口，复杂度接近 \(O(nw)\)；
- **Sparse/Block Sparse/Global-Local**：Longformer、BigBird 等选择少量连接；
- **Dilated/Chunk Attention**：按间隔或块组织长序列。

### 3. Head 与 KV 组织

- **MHA**：每个 Query Head 有独立 K/V；
- **MQA**：所有 Q Head 共享一组 K/V，KV 最省；
- **GQA**：若干 Q Head 共享一组 K/V，质量和缓存折中；
- **MLA**：缓存低维 KV latent，并解耦 RoPE 部分，进一步压缩缓存；
- **Multi-Query Latent/Grouped 变体**继续优化并行与带宽。

### 4. 算法近似

- **Linear Attention**：通过核技巧改变乘法顺序，避免显式 \(n^2\) 矩阵；
- **Low-rank Attention**：Linformer 等压缩序列维；
- **LSH/Cluster Attention**：Reformer 等只关注相近桶；
- **Retrieval/Memory Attention**：从外部记忆选相关内容。

近似线性不代表所有长度都更快，kernel 常数、因果实现和质量要实测。

### 5. 系统实现

- **FlashAttention**：精确 Attention 的 IO-aware tiled kernel，不改变数学结果和 \(O(n^2)\) 算术复杂度；
- **PagedAttention**：推理服务中的 KV block 管理，不是新的 Attention 数学公式；
- **Prefix/Chunked Prefill**：缓存或调度优化。

### 面试总结

回答变体时要分层：Self/Cross 讲来源，Causal/Local 讲可见性，MHA/GQA/MLA 讲 KV 组织，Linear/Sparse 讲复杂度，Flash/Paged 讲 kernel 与缓存系统，避免把它们混为一类。
