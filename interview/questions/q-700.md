# decoder部分训练和推理有什么区别？

## 答案

Decoder 训练与推理使用同一个因果语言模型，但输入来源、并行方式、缓存、随机性和目标不同。

### 训练阶段

训练采用 teacher forcing。输入是完整真实序列的前缀，标签是右移一位的真实 token：

$$
\mathcal{L}=-\sum_t\log P_\theta(x_t\mid x_{<t})
$$

因果 mask 保证位置 $t$ 看不到未来，但所有位置的 logits 可以在一次前向中并行计算。训练通常启用 dropout，保存激活用于反向传播，并更新参数。padding token 和不参与监督的 prompt token需按任务正确 mask。

### 推理阶段

模型没有真实的下一 token，必须将自己上一步选出的 token 追加到输入，自回归循环生成。因此步骤之间串行，会产生 exposure bias 和误差累积。

推理先用 prefill 并行处理 prompt，保存每层 KV Cache；随后 decode 每次只处理一个新 token，读取历史 K/V，避免重复计算。模型切到 eval 模式，不做反向传播和 dropout，并通过 greedy、beam、top-p 等策略选择 token。

### 性能差异

训练侧主要是大矩阵并行计算，关注 tokens/s、激活和优化器显存；decode 常是 batch 小、访存密集，受 KV Cache 和显存带宽限制。推理还需处理动态 batching、停止条件、量化和请求调度。

验证 loss 时仍可使用 teacher forcing；真正评估生成质量则应运行自回归解码。二者不能混为同一种评测。
