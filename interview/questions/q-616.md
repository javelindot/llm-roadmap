# 5．为什么有了 SFT 之后还需要 RLHF

## 答案

SFT 的目标是模仿示范答案，RLHF 的目标是进一步优化人类偏好或明确的行为准则，两者解决的问题不同。

### SFT 的局限

SFT 使用 token-level 交叉熵：

$$
\mathcal{L}_{SFT}=-\sum_t\log p_\theta(y_t\mid x,y_{<t})
$$

它擅长教会模型回答格式、任务知识和基本指令遵循，但有几个限制：

- 高质量完整示范成本高，而比较两个答案哪个更好通常更容易标注。
- 交叉熵把参考答案当作唯一目标，不能直接表达“两个答案都合理，但其中一个更有帮助”。
- 训练按局部 token 似然优化，而用户关心的是完整回答的正确性、安全性、风格和任务完成度。
- SFT 数据覆盖有限，模型在自身生成分布上仍可能出现啰嗦、迎合、拒答不当或安全问题。

### RLHF 补充了什么

典型 RLHF 先收集同一 prompt 下的回答偏好，训练 Reward Model，再用 PPO 等算法提高高奖励回答的概率，同时通过 KL 惩罚限制模型偏离参考策略。它能利用排序数据优化序列级偏好，并针对帮助性、安全性、事实性等目标调整行为。

现代方案也可用 DPO、IPO 等直接偏好优化，跳过显式在线强化学习，但本质仍是在 SFT 基础上利用偏好对齐信号。

### 是否一定需要

不一定。若任务有可验证奖励、SFT 数据足够且输出空间窄，SFT 可能已经满足要求；偏好优化也可能引入 reward hacking、过度拒答或能力退化。因此通常先做高质量 SFT 建立能力和稳定策略，再根据线上评测决定是否使用 RLHF/DPO，并持续用独立安全与能力基准验证。
