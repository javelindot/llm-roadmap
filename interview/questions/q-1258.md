# 损失函数：CE和对比损失

## 答案

交叉熵 CE 学习“样本属于哪个标签或下一个 token 是什么”，对比损失学习“哪些样本在表示空间中应接近或远离”。两者可以单独使用，也可以在分类、Embedding 和 RAG 中联合训练。

### 1. 交叉熵

多分类中，真实标签为 `y`：

```text
L_CE = -log p(y | x)
```

等价写法：

```text
L_CE = -Σ_c y_c * log p_c
```

作用是提高正确类别 logit、压低其他类别。语言模型的 next-token loss、意图分类和 reranker 二分类都可使用 CE。

优点：

- 目标明确，实现稳定；
- 直接优化类别概率；
- 易于处理多分类和 token 级任务。

局限：

- 类别数大时 full softmax 成本高；
- 只给类别标签时，不显式约束类内紧凑与类间几何结构；
- 类别不平衡和标签噪声会影响校准。

### 2. 对比损失

InfoNCE 的常见形式：

```text
L_i =
-log [
  exp(sim(q_i, d_i+) / tau)
  /
  Σ_j exp(sim(q_i, d_j) / tau)
]
```

其中 `d_i+` 是正样本，batch 内其他文档可作为负样本，`tau` 是温度。

目标是让 Query 与正样本相似度高于负样本。常用于：

- Sentence Embedding；
- Dense Retrieval；
- 图文对齐；
- 表征预训练；
- 去重和聚类。

### 3. 二者的联系

InfoNCE 本质上也可以看成“在一个正样本和多个负样本中做交叉熵分类”。区别主要在监督单位：

- 普通 CE 的类别通常是固定标签；
- 对比学习的“类别”由当前样本、正对和负样本动态构成；
- 对比学习更强调表示空间和相对相似度。

### 4. 关键工程点

**负样本**

- 随机负样本太简单，梯度价值低；
- hard negative 能提升区分力；
- 伪负样本会把真正相关文本推远；
- in-batch negative 要避免同主题或重复答案互相误伤。

**温度 tau**

- 较小温度使分布更尖锐，更强调困难样本；
- 过小会导致梯度集中和训练不稳定；
- 与 batch size、相似度范围和归一化共同调参。

**向量归一化**

L2 normalize 后点积等于 cosine，有利于控制相似度尺度；是否归一化要与训练目标和向量索引一致。

### 5. 联合训练

例如支付 Query-Document 检索：

```text
L_total =
    L_contrastive
    + lambda * L_domain_classification
```

对比损失学习相关文本接近，分类 CE 辅助学习产品、国家或文档类型。权重需要消融，避免辅助任务压过检索目标。

生成模型 SFT 主要使用 token CE；Embedding 模型主要使用对比目标；Cross-Encoder reranker 可使用 CE、pairwise 或 listwise ranking loss。选择依据是线上任务和评测，而不是哪种损失“更先进”。
