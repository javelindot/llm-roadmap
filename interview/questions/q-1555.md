# 什么是词元化?请比较一下 BPE 和 WordPiece 这两种主流的子词切分算法。

## 答案

词元化（Tokenization）把原始文本转换为模型可处理的 token ID 序列。完整流程通常包括 Unicode 规范化、预切分、子词切分、特殊 token 插入和词表 ID 映射。

### 1. 为什么使用子词

按词切分会遇到未登录词和巨大词表；按字符/字节切分序列又太长。子词在两者间折中：

- 高频词保留为一个 token；
- 低频词拆成可组合片段；
- 控制词表大小；
- 对新词、拼写变化和多语言更稳健。

### 2. BPE

BPE 从较小单位开始，反复合并训练语料中频率最高的相邻 token 对：

```text
l o w
lo w
low
```

训练后保存 merge rules，编码时按规则合并。

特点：

- 目标直观，按频率贪心合并；
- GPT 系列常用 byte-level BPE；
- 字节回退可以避免真正的 OOV；
- 词表与序列长度折中良好；
- 可能把语义相似词切成不同片段。

### 3. WordPiece

WordPiece 与 BPE 类似，也逐步构建子词词表，但合并选择更偏向提高语言模型似然，而不只是最高原始 pair frequency。常见近似评分为：

```text
score(a, b)
= freq(ab) / (freq(a) * freq(b))
```

BERT WordPiece 常用 `##` 表示词中后续片段，例如：

```text
playing -> play + ##ing
```

编码通常选择能覆盖单词的最长子词；无法覆盖时可能回退 `[UNK]`，取决于实现。

### 4. 对比

| 维度 | BPE | WordPiece |
|---|---|---|
| 合并依据 | 高频 pair | 似然/归一化关联度 |
| 常见模型 | GPT、RoBERTa | BERT |
| OOV | Byte BPE 基本可避免 | 传统实现可能 `[UNK]` |
| 标记方式 | 依实现 | 常见 `##suffix` |
| 训练 | 简单直接 | 目标更接近语言建模 |

实际 tokenizer 还包括 SentencePiece Unigram、byte fallback、正则预切分等，不能只由“BPE/WordPiece”名称推断全部行为。

### 5. 对模型的影响

- 词表越大，Embedding/LM Head 参数更多，但序列可能更短；
- 切分过碎会增加上下文和推理成本；
- 数字、代码、中文、低资源语言需单独评测；
- tokenizer 一旦更换，旧权重和 token ID 通常不能直接兼容；
- 业务术语若被切得过碎，可评估扩词表，但需重新训练对应 embedding。

### 6. 跨境支付场景

应检查 `USD、IBAN、SWIFT、payout、拒付、错误码、金额和多语言姓名` 的切分效率。tokenizer 只改变表示，不负责校验账号或金额；高风险字段仍应结构化解析。

### 7. 面试总结

BPE 以高频相邻对合并为主，WordPiece 更偏向选择提升词表语言模型效果的子词。两者都在词级与字符级间折中，但 OOV、前缀标记和具体编码行为取决于实现。
