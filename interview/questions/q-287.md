# Agent 怎么做的？

## 答案

> 这是个开放问题，我从**我们风控调查 Agent 的实战**出发完整讲一遍。
>
> **整体技术栈**：
> - 编排：LangGraph（状态图）
> - LLM：Qwen-14B（决策、规划）
> - 工具：内部 MCP Server 暴露 30+ 工具
> - 状态持久化：Redis
> - 监控：OpenTelemetry + Grafana
>
> **核心架构**：
>
> ```
>      ┌──────────────┐
>      │ 工单输入     │
>      └──────┬───────┘
>             ▼
>      ┌──────────────┐
>      │ 规划 (Plan)  │ ← LLM 输出步骤列表
>      └──────┬───────┘
>             ▼
>      ┌──────────────┐
>      │ 执行 (Exec)  │ ← 逐步调用工具
>      └──────┬───────┘
>             ▼
>      ┌──────────────┐
>      │ 反思 (Reflect)│ ← LLM 判断是否够了
>      └──────┬───────┘
>      不够 ↺  够↓
>             ▼
>      ┌──────────────┐
>      │ HITL 复核    │
>      └──────┬───────┘
>             ▼
>      ┌──────────────┐
>      │ 输出报告     │
>      └──────────────┘
> ```
>
> **关键模块详解**：
>
> **1. 规划层**
> ```python
> async def plan(state: State) -> dict:
>     prompt = build_plan_prompt(state.ticket, available_tools)
>     plan = await llm.invoke(prompt, response_format=Plan)
>     # 校验 plan 合法性（步数、必含步骤）
>     validate_plan(plan)
>     return {"plan": plan}
> ```
>
> **2. 执行层**
> ```python
> async def execute_step(state: State) -> dict:
>     step = state.plan.next_step()
>     args = render_args(step.args_template, state.context)
>     try:
>         result = await tools[step.tool].invoke(args, timeout=10)
>     except (TimeoutError, ToolError) as e:
>         result = ErrorResult(retry=True if isinstance(e, TimeoutError) else False)
>     state.context[step.id] = result
>     return {"context": state.context}
> ```
>
> **3. 反思层**
> ```python
> async def reflect(state: State) -> str:
>     prompt = build_reflect_prompt(state.context, state.plan)
>     decision = await llm.invoke(prompt)  # "enough" | "more_data" | "human"
>     return decision
> ```
>
> **4. HITL 节点**
> ```python
> async def hitl_wait(state: State):
>     await job_queue.publish_for_human_review(state)
>     # 阻塞等待，可中断
>     human_decision = await job_queue.wait_for_decision(state.job_id)
>     return {"human_decision": human_decision}
> ```
>
> **状态定义**：
> ```python
> class InvestigationState(TypedDict):
>     ticket: dict
>     plan: Optional[Plan]
>     context: dict
>     iteration: int
>     conclusion: Optional[str]
>     human_decision: Optional[str]
> ```
>
> **几个工程要点**：
>
> **1. max_iterations 兜底**
> ```python
> if state["iteration"] > 8: route_to_human()
> ```
>
> **2. 工具调用幂等**
> 工具层强制带 idempotency_key，重试不重复执行
>
> **3. 状态持久化**
> 每个节点出口存 Redis，崩了从 checkpoint 恢复
>
> **4. trace 全链路**
> 每个节点 emit span 到 OpenTelemetry
>
> **5. 评测体系**
> - 离线：黄金集 500 条人工标
> - 线上：监管复核通过率、漏检率
> - 回流：每月失败 case 复盘
>
> **6. 灰度上线**
> 5% → 20% → 50% → 100%，每阶段跑一周观察指标
>
> **7. 监控告警**
> - p95 延迟超阈值
> - 失败率 > 5%
> - HITL 队列积压
>
> **效果数据**：
> - 单工单调查时间从 30 分钟 → 3 分钟
> - 证据召回率 +40%
> - 误报率 -15%
> - 运营从"信息搬运工"变成"决策审核员"
>
> **踩过的坑（值得讲）**：
> - 一开始用 LangChain `AgentExecutor`，后来切 LangGraph（详见 q-13）
> - LLM 计划质量不稳，加 JSON Schema 强约束
> - Agent 偶尔走神（去查无关账户），加 step adherence 监控
> - 反思层太宽松会过早结论，prompt 加 "证据 checklist"
>
> 一句话：**Agent 做的就是"LLM 决策 + 工程框架兜底 + 评测飞轮迭代"**——技术不难，工程上线难，业务效果靠飞轮。

**🪝 顺嘴带出去的钩子**：
- "Plan 失败 replan 的具体策略可以聊。"
- "反思层 prompt 怎么避免过早结论可以展开。"
- "Agent 上线后的飞轮怎么转可以讲讲。"
