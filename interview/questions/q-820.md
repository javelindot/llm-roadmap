# 搜索中 RAG 的向量检索会受到长尾商品影响，你会如何缓解？

## 答案

长尾商品交互少、名称新颖、属性稀疏，纯协同 embedding 容易质量差；同时热门商品在训练负样本、索引邻域和线上反馈中占优势。需要从表示、召回、训练和评估共同处理。

### 1. 强化内容表示

- 融合标题、类目、品牌、属性、图片和规格；
- 用商品知识图谱或类目层级补充关系；
- 冷启动商品由内容 encoder 生成 embedding，而非等待交互；
- 对缺失属性做质量检测，不让 LLM 编造商品事实。

### 2. 多路混合召回

- BM25/稀疏检索保留型号、SKU 和精确属性；
- 稠密向量召回语义同义表达；
- 类目、品牌、价格和库存做 metadata 过滤；
- 规则或探索通道给长尾新品保留候选配额；
- 使用 RRF 或学习融合，防止单一向量通道垄断。

### 3. 训练与索引

- 对热门曝光做 inverse propensity 或频次去偏；
- 对长尾 query/item 过采样，但校准真实线上分布；
- 挖掘同类目的 hard negative，避免只学热度；
- 采用多任务学习、对比学习和 teacher distillation；
- 检查 ANN 参数、量化误差是否让稀疏长尾邻居更易丢失。

### 4. 评估

按 head/mid/tail、冷启动、新品和类目切片报告 Recall@K、NDCG、覆盖率、多样性及转化。总体指标可能被热门商品掩盖。

### 面试总结

缓解长尾要用多模态内容表示解决冷启动，用 BM25+向量+规则多路召回抵抗热度偏差，再通过曝光去偏、hard negative 和分桶评估验证长尾收益。
