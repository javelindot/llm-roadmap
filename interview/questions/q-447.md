# 🔥 讲一下GRPO

## 答案

**一句话**：GRPO（Group Relative Policy Optimization）是一种不训练 Critic 的策略优化方法。它对同一个 Prompt 采样一组回答，用组内奖励的相对高低构造 Advantage，再用类似 PPO 的裁剪目标更新策略。

### 1. 核心流程

对每个 Prompt `q`：

1. 用旧策略采样 `G` 个回答 `o_1 ... o_G`。
2. 使用规则、验证器或 Reward Model 得到奖励 `r_1 ... r_G`。
3. 在组内标准化奖励：

```text
A_i = (r_i - mean(r_1...r_G)) / (std(r_1...r_G) + eps)
```

4. 用 `A_i` 更新当前策略，并加入 KL 约束避免偏离参考模型。

简化目标可表示为：

```text
min(
  ratio_i · A_i,
  clip(ratio_i, 1-ε, 1+ε) · A_i
) - β · KL(πθ || πref)
```

其中 `ratio_i = πθ(o_i|q) / πold(o_i|q)`。

### 2. 与 PPO 的主要区别

- PPO 通常训练 Value/Critic 来估计 Advantage。
- GRPO 用同一 Prompt 下多个回答的组内相对奖励作为 Baseline。
- 省掉 Critic 后，显存占用和训练复杂度更低。

### 3. 优点

- 特别适合数学、代码等可用规则验证答案的任务。
- 同题多采样后，能学习“哪些推理路径相对更好”。
- 不需要单独维护价值模型。

### 4. 局限

- 每个 Prompt 要生成多个回答，采样成本高。
- 组内奖励都相同时，Advantage 信号很弱。
- 效果高度依赖 Reward 的可靠性，仍可能 Reward Hacking。
- 属于在线策略优化，旧样本不能像普通监督数据那样无限复用。
- 对开放式任务，奖励标准比数学正确性更难设计。

**面试总结**：GRPO 的关键不是“没有价值估计”，而是用同题多个候选的相对奖励替代单独的 Critic。
