# Transformer 的原理，从用户输入到模型输出，中间经历了什么？

## 答案

> 我从**端到端流程**讲，再点出每步的内部机制。
>
> **完整流程**（以 GPT 风格 decoder-only LLM 为例）：
>
> ```
> 用户输入文本
>   ↓ Tokenizer：BPE / SentencePiece 切成 token IDs
> [101, 2345, 6789, ...]
>   ↓ Embedding 查表
> (seq_len, hidden_dim) 矩阵
>   ↓ 加位置编码（Sinusoidal / RoPE）
> (seq_len, hidden_dim) 含位置信息
>   ↓ 进入 N 层 Transformer Decoder Block
>   每一层做：
>     ↓ RMSNorm（Pre-LN 风格）
>     ↓ Multi-Head Self-Attention（含 causal mask）
>     ↓ Residual 残差
>     ↓ RMSNorm
>     ↓ FFN（SwiGLU 激活）
>     ↓ Residual
>   ↓ 最后 Norm
>   ↓ LM Head（Linear，hidden_dim → vocab_size）
> (seq_len, vocab_size) logits
>   ↓ Softmax → 下一 token 概率分布
>   ↓ 采样（greedy / top-p / top-k）
> 下一个 token ID
>   ↓ Detokenize → 文字
> 用户看到的输出
> ```
>
> **关键步骤细节**：
>
> **1. Tokenization**
> 文本切成 subword 单元。中文常按字符 + BPE 合并。Qwen 词表 152k，覆盖中英多语言。
>
> **2. Embedding 查表**
> 每个 token ID 对应一个 hidden_dim 维向量（Qwen-14B 是 5120 维）。
>
> **3. 位置编码**
> 现代 LLM 用 RoPE——对 Q/K 做"位置相关的旋转"，让 attention 内积自然包含相对位置信息。
>
> **4. Self-Attention（核心）**
> ```
> Q = X @ W_q, K = X @ W_k, V = X @ W_v
> scores = Q @ K^T / √d_k  + causal_mask
> attn = softmax(scores)
> output = attn @ V
> ```
> Multi-head：把 hidden 分到多个 head 并行算，最后拼起来。GQA：多个 Q 共享一组 KV，省 KV cache。
>
> **5. FFN（占大头参数）**
> ```
> SwiGLU: x → (Swish(x @ W_gate) * (x @ W_up)) @ W_down
> ```
> 一般 hidden_dim 扩到 4x 再压回来。这是 LLM 80% 参数所在。
>
> **6. Residual + Norm**
> 每个子层都接残差 + Pre-LN，让深层模型稳定训练，梯度好传。
>
> **7. N 层堆叠**
> Qwen-14B 是 48 层 decoder block。
>
> **8. LM Head**
> 最后 hidden 投影到词表大小，softmax 后是下一 token 概率分布。
>
> **9. 采样**
> - greedy：取概率最大
> - top-p / top-k：截断后采样
> - temperature 调节平滑度
>
> **10. 自回归循环**
> 生成的 token 拼回输入，继续算下一 token——直到 EOS 或达 max_length。
>
> **推理时的工程优化**：
>
> - **KV Cache**：缓存历史 K/V，每次只算新 token
> - **Prefill vs Decode**：处理输入用并行 prefill（算力密集），生成用串行 decode（带宽密集）
> - **Flash-Attention**：IO 优化，省 5x HBM 读写
> - **vLLM Paged Attention**：KV cache 按 page 管理，提升并发
> - **量化（INT8 / AWQ / GPTQ）**：参数低精度，推理快、显存省
>
> **训练时的关键差异**：
> - 训练时整个序列**一次性算完**（teacher forcing），靠 causal mask 防偷看未来
> - loss 是每个位置的 cross-entropy
> - 反传更新所有参数
>
> 一句话：**Transformer = 切 token → embed + 位置 → 多层 attention + FFN → softmax → 采样**——简单的内核被规模和工程优化堆出整个 LLM 时代。
