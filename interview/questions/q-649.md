# 介绍多路召回策略。

## 答案

多路召回是让不同召回通道从不同视角产生候选，再进行去重、配额和融合，以兼顾相关性、覆盖率、实时性和探索能力。

### 常见召回通道

- 协同过滤：item-based/user-based CF、矩阵分解，利用共现和行为相似性。
- 向量召回：双塔模型编码用户/query 与 item，通过 ANN 检索语义或兴趣相似候选。
- 内容与关键词：BM25、标签、类目、知识图谱，适合精确约束和冷启动。
- 热门与趋势：全局、地域、时间窗口或人群热门，保证基本覆盖。
- 规则与运营：关注、订阅、白名单、活动内容和业务兜底。
- 实时兴趣：根据最近点击、搜索、会话序列召回短期兴趣。
- 探索通道：新 item、长尾和多样化候选，缓解反馈回路。

### 融合策略

最简单是每路固定配额后合并去重，但流量和用户差异大时利用率不高。更常见做法是为候选保留来源、通道分数和 rank，经过分数归一化后加权，或训练轻量融合模型。Reciprocal Rank Fusion 等方法在各路分数不可比时也很实用。

配额可根据用户活跃度、场景、通道历史增益和实时负载动态调整。相同 item 被多路命中本身也是强特征，可传给粗排与精排。

### 系统设计与评估

各通道并行执行并设置独立超时、熔断和最低返回数，慢通道不能阻塞整体。缓存稳定通道，实时通道单独更新；统一做权限、库存和已消费过滤。

离线分别看每路 Recall@K、覆盖率、重合率、独占贡献和延迟，再做增量消融。线上关注 CTR/CVR、时长、多样性、新内容曝光和系统成本。多路召回的价值不是通道越多越好，而是每一路提供可量化的增量候选。
