# 问有没有其他方法: 最小堆，再排序O(nlogk)

## 答案

这道题的原始题干缺失，只能从“最小堆，再排序 O(nlogk)”推断大概率是在问 Top K 或第 K 大/小。面试时要先向面试官确认：是求第 k 大、无序 top k，还是要求 top k 有序输出。三种问题复杂度不一样。

### 1. 最小堆方案

如果是求最大的 k 个数：

1. 维护大小为 k 的最小堆；
2. 遍历数组；
3. 堆没满就入堆；
4. 堆满后，如果当前数大于堆顶，则替换堆顶；
5. 最后堆中就是 top k。

复杂度：

```text
遍历 n 个元素，每次堆操作 O(logk)
总复杂度 O(nlogk)
空间复杂度 O(k)
```

如果还要求 top k 从大到小有序输出，需要再排序堆里的 k 个元素：

```text
O(nlogk + klogk)
```

通常面试里会简写为 O(nlogk)，因为 k <= n 且主过程是堆维护，但严格写法要加上最后排序。

### 2. 快速选择

如果只要求无序 top k，可以用 Quickselect：

```text
平均 O(n)
最坏 O(n^2)
空间 O(1) 或 O(logn)
```

找到第 k 大的分界点后，左/右侧就是 top k。若要求有序，再对 k 个元素排序：

```text
平均 O(n + klogk)
```

这是面试里常问的“有没有比堆更快”的方法。

### 3. 全量排序

直接排序：

```text
O(nlogn)
```

代码最简单，适合 n 不大、一次性离线处理、或者面试官不强调最优复杂度的情况。

### 4. 计数或桶

如果元素范围有限，比如分数、年龄、状态码，可以用计数排序/桶：

```text
O(n + R)
```

其中 R 是值域大小。跨境支付里如果是“按有限风险等级取 top k”，这种方法可能比堆更直接。

### 5. 流式数据怎么选

如果数据是流式到达，不能一次性放入内存，最小堆是最稳的：

- 只保留 k 个元素；
- 每来一个新元素 O(logk) 更新；
- 适合实时排行榜、异常交易 top k、失败率最高通道 top k。

### 6. 面试总结

题干缺失时，我会先确认输出要求。求流式 top k 用大小为 k 的堆，复杂度 O(nlogk)，有序输出再加 O(klogk)；离线无序 top k 可以用 Quickselect，平均 O(n)；值域有限时可以用桶或计数排序。
