# 工作性质,Python用的多还是C++用的多,有无转正,是部署到车机芯片上测实车表现还是部署到测试服务里去验证性能指标?

## 答案

工作内容更偏服务端 LLM 应用与算法工程，不是车机端算法部署。技术栈可以分为三层说明。

### 1. Python 为主

Python 主要用于：

- 数据清洗、脱敏、标注和评测；
- PyTorch/Transformers 微调；
- Embedding、Reranker 和 RAG 实验；
- Agent 编排、工具调用和离线回放；
- FastAPI/异步任务服务；
- 指标分析、A/B 实验和自动化脚本。

在跨境支付场景中，还会用 Python 对国家、币种、通道、错误码和交易状态做数据处理及模型评测。

### 2. C++ 的使用边界

C++ 通常用于性能敏感或底层部署环节：

- 自定义 CUDA/推理算子；
- TensorRT、ONNX Runtime 插件；
- 高性能检索和特征服务；
- 端侧 SDK 或已有 C++ 服务接入；
- 延迟、内存和并发优化。

如果工作没有涉及自定义算子和端侧 Runtime，就应明确 C++ 不是主要开发语言，而不是为了匹配岗位夸大使用比例。

### 3. 部署环境

跨境支付 LLM 系统通常部署在测试或生产服务集群：

```text
模型服务
 + 向量/搜索服务
 + Agent 编排
 + 支付业务工具
 + 监控和审计
```

验证指标包括答案正确率、召回、工具成功率、P95/P99、QPS、GPU 显存、成本和安全违规率。它与车机芯片实车验证是不同工程场景，不应包装成端侧部署经历。

如果岗位真正涉及车机，还需要额外回答：

- 芯片算力、内存和功耗；
- 模型量化、编译和算子支持；
- 实时性、冷启动和热管理；
- 仿真、台架、封闭道路与实车测试；
- 功能安全和故障降级。

### 4. 转正问题

有无转正属于个人履历事实，应直接、简洁说明结果和原因。没有转正不等于能力不足，可以如实说明实习周期、名额、毕业安排或个人选择；不能编造转正结果。
