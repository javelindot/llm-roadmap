# 卷积核的大小是否一定越大越好？

## 答案

**一句话**：**不是**——大卷积核感受野大但参数和算力暴涨、容易过拟合；**主流做法是用多层小卷积（3×3 堆叠）等效大感受野**，这就是 VGG 的核心思想。

### 1. 大 kernel 的好处
- 单层感受野大
- 一次能"看到"更大局部
- 早期 AlexNet 用 11×11 first layer

### 2. 大 kernel 的代价
- **参数爆炸**：K² 增长
  - 7×7 = 49 个参数 / channel
  - 3×3 = 9 个参数 / channel
- **FLOPs 暴涨**
- **过拟合风险**：参数多但有效信号未必相应增加
- **空间细节丢失**：大 kernel 平滑掉小尺度特征

### 3. 小 kernel 堆叠的优势（VGG 思路）
- **两层 3×3 ≈ 一层 5×5 感受野**，但参数只有 2×9 = 18 vs 25
- **三层 3×3 ≈ 一层 7×7 感受野**，参数 27 vs 49
- 每层都有非线性激活，**表达力更强**
- 更深 → 更多组合特征

### 4. 现代设计中的 kernel 选择
| 网络 | 主流 kernel | 备注 |
|------|------------|------|
| AlexNet | 11×11, 5×5, 3×3 | 早期 |
| VGG | **3×3** | 堆叠出深度 |
| ResNet | 3×3 + 1×1 | bottleneck |
| Inception | 1×1, 3×3, 5×5 混合 | 多尺度 |
| ConvNeXt | **7×7 depthwise** | 大 kernel 复兴 |
| MobileNet | 3×3 depthwise | 轻量 |

### 5. 近年大 kernel 复兴
- **ConvNeXt**（2022）：用 7×7 depthwise + 通道扩展
- **RepLKNet**：用 31×31 等超大 kernel
- 大 kernel + **depthwise** 结合，参数仍可控
- 受 ViT 启发，恢复"大感受野"思想

### 6. 选 kernel 的经验
- **3×3 是默认安全选择**：参数小、效果好
- **1×1 用于通道调整 + 跨通道融合**
- **5×5 / 7×7 用于早期下采样或大感受野需求**
- **多尺度 Inception** 在多模态/检测里有用
- **Depthwise 大 kernel** 是轻量化新趋势

### 7. 一句话总结
**3×3 是默认、1×1 调通道、大 kernel 配 depthwise 复兴**——卷积核选择是 trade-off，从来不是越大越好。

## 结合项目谈（口语版 · 约 20 秒）

> 这道题工业 LLM 落地里**用得很少**——我们跨境支付的 LLM 主线全是 Transformer，没卷积。
>
> 唯一相关是**多模态 KYB 识别**，我们 vision encoder 用 ViT 不用 CNN，patch_size=16 相当于一次性 16×16 卷积（一种"大 kernel + 不重叠"的极端形式）。这个设计有个隐含选择——**牺牲精细局部特征换全局 attention**。后来发现对印章遮挡这种细节场景效果一般，我们换成 ViT-B/14 patch=14 + 高分辨率 448 输入，patch 数翻倍但能捕捉小细节。
>
> 这其实印证了"kernel 选择是 trade-off"的核心思想——**ViT 大 patch 类似大 kernel，捕捉全局好但丢局部**；缩小 patch 类似多层小 kernel 堆叠，更精细但慢。
>
> 我对这道题的总结：**面试要会讲 VGG 的"3×3 堆叠 = 大感受野"经典思想**，但工业 LLM 里更多是 Transformer 取代了 CNN 这件事——理解 trade-off 比记网络名重要。

- "多模态项目里 vision encoder 怎么选我们做过对比。"
