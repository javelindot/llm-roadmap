# 怎么用大模型结合知识图谱做问答的呢？

## 答案

**一句话**：大模型结合知识图谱做问答，核心是让大模型负责理解问题和组织答案，让知识图谱负责提供结构化实体关系和可解释依据。

### 1. 为什么结合知识图谱

RAG 更擅长从文本里找相关段落，但有些问题需要结构化关系推理，比如：

- 某个商户关联了哪些账户。
- 某个收款人是否和高风险主体有关。
- 某个国家支持哪些支付通道。
- 某类错误码对应哪些处理动作。

知识图谱可以表达实体、关系和路径，让答案更可解释。

### 2. 实现流程

常见流程是：

1. 从文档、数据库、日志中抽取实体和关系。
2. 构建图谱，比如 Neo4j。
3. 用户提问后识别实体和意图。
4. 大模型生成 Cypher 或调用图查询工具。
5. 图数据库返回实体关系路径。
6. 大模型基于图结果和文本知识生成答案。

### 3. 技术关键点

- 实体抽取要准确。
- 同名实体要做消歧和归一。
- 图谱关系要有业务含义。
- 图查询要受权限控制。
- 大模型生成 Cypher 要做安全校验。
- 图谱结果最好和 RAG 文本证据结合。

### 4. 结合跨境支付项目怎么说（面试口径）

> 在跨境支付业务里，知识图谱可以用来表示商户、法人、账户、收款人、设备、IP、国家、通道、风险标签之间的关系。比如风控人员问“这个收款人是否和历史高风险商户有关联”，单纯 RAG 很难回答，但图谱可以查出它是否共用设备、银行卡、IP 或法人信息。
>
> 大模型在这里负责把自然语言问题转换成查询意图，比如识别商户名、收款人、风险类型，然后调用图查询工具。图谱返回关系路径后，模型再解释成自然语言，比如“该收款人与两个高风险账户共享设备指纹，因此建议人工审核”。
>
> 我会把图谱和 RAG 结合。图谱回答结构化关系，RAG 提供制度依据和处理 SOP，这样答案既有关系证据，也有业务解释。
