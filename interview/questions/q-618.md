# 加那个reward头需要什么库？

## 答案

增加 reward head 本身不需要特殊库，本质是在 Transformer 最后隐藏状态上增加一个输出维度为 1 的线性层：

```python
self.score = torch.nn.Linear(hidden_size, 1, bias=False)
```

对每条 `prompt + response`，取 EOS 或最后一个非 padding token 的隐藏状态，得到标量 reward。

### 常见实现选择

- **纯 PyTorch**：`torch.nn.Linear` 加自定义 Bradley-Terry pairwise loss，控制最清晰；
- **Transformers**：使用 `AutoModelForSequenceClassification.from_pretrained(..., num_labels=1)`；
- **TRL**：使用 [`RewardTrainer`](https://huggingface.co/docs/trl/main/reward_trainer) 处理 chosen/rejected 数据、padding 和训练循环；
- **PEFT**：显存有限时给 backbone 加 LoRA，同时保证 scalar head 可训练；
- **DeepSpeed/FSDP**：模型较大时做分布式分片，它们解决的是训练规模，不是 reward head 定义。

若问的是 PPO 的 critic/value head，而非独立 Reward Model，可使用 TRL 的 `AutoModelForCausalLMWithValueHead` 或自定义 value head。两者虽然都是标量头，但用途不同：Reward Model 评估整段回答，critic 估计当前策略下的期望回报并计算 advantage。

### 关键实现细节

- tokenizer 与 chat template 必须和 backbone 匹配；
- chosen/rejected 应来自同一 prompt；
- padding 时不能误取 pad token 的隐藏状态；
- 训练 pairwise score difference，而不是要求绝对 reward 有统一刻度；
- 检查长度、格式和模型来源偏差，防止 reward head 学捷径。

### 面试总结

reward head 本质是 `Linear(hidden_size, 1)`，PyTorch 就能实现。工程上常用 Transformers 的 Sequence Classification 模型与 TRL RewardTrainer；要区分独立 Reward Model 的 score head 和 PPO critic 的 value head。
