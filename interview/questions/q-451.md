# 🔥 lora训练，怎么做的，参数，rag的流程细节。

## 答案

这道题包含两个部分，回答时应明确区分：**LoRA 用于改变模型行为或领域能力，RAG 用于在推理时注入可更新的外部知识**。

### 一、LoRA 训练怎么做

#### 1. 数据准备

- 明确任务：问答、分类、抽取、风格或工具调用。
- 清洗去重，统一 Prompt 模板。
- 划分训练、验证和真实业务测试集。
- 检查答案质量、长度分布和隐私数据。

#### 2. 关键参数

| 参数 | 常见起点 | 作用 |
|------|----------|------|
| `r` | 8 / 16 / 32 / 64 | 低秩容量 |
| `lora_alpha` | 常设为 `r` 或 `2r` | 增量缩放 |
| `lora_dropout` | 0–0.1 | 缓解过拟合 |
| `target_modules` | q/k/v/o，必要时加 FFN | 注入位置 |
| learning rate | 约 `1e-4` 到 `2e-4` 起调 | 更新速度 |
| epochs | 1–3 起调 | 避免小数据过拟合 |
| max sequence length | 按任务分布设置 | 影响显存和长文本能力 |

还要设置有效 Batch Size、梯度累积、Warmup、Weight Decay、BF16/FP16，以及是否使用 4bit QLoRA。

#### 3. 训练与评估

```text
加载基座模型
→ 冻结原参数
→ 注入 LoRA
→ 监督微调
→ 离线评测
→ 合并或以 Adapter 方式部署
```

不能只看 Loss，要比较任务指标、通用能力回归、幻觉率和线上样本。

### 二、RAG 的完整流程

#### 1. 离线入库

```text
文档采集 → 清洗解析 → 分块 → 元数据
→ Embedding → 向量/关键词索引
```

分块需要考虑标题层级、语义边界、重叠长度、表格和版本权限。

#### 2. 在线检索

```text
用户问题
→ 意图识别 / Query Rewrite
→ 元数据过滤
→ 向量 + BM25 混合召回
→ Rerank
→ 上下文去重与排序
```

#### 3. 生成与治理

```text
构造 Prompt
→ LLM 基于证据生成
→ 引用来源
→ 事实 / 格式校验
→ 反馈与 Bad Case 回流
```

评测应拆成检索 Recall@K、Rerank 指标、答案正确性、忠实度、引用准确率、延迟和成本。

**面试总结**：需要稳定行为、固定格式或领域表达时考虑 LoRA；知识频繁更新、要求引用和权限控制时优先 RAG；实际系统常组合使用。
