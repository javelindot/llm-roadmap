# PDF 解析存在哪些问题？

## 答案

**一句话**：PDF 是个"印刷格式"不是"数据格式"——**视觉上看到什么 ≠ 文件里存了什么**，所以解析时会撞上**版式错乱、表格丢结构、扫描页 OCR、字体编码、公式/图表丢失**等一堆坑。

### 1. 五大经典问题

**1) 版式还原乱**
- PDF 是按"视觉位置"存的，不是"阅读顺序"
- 双栏、报刊式版面提取出来段落混乱
- 直接 `pdftotext` 可能把左右栏交错混合

**2) 表格丢结构**
- 表格在 PDF 里是"线条 + 文本框"
- 提取后只剩文本碎片，行列关系丢失
- 跨页表格、合并单元格、嵌套表更难

**3) 扫描页是图像**
- 整页就是一张 JPEG/PNG
- 必须先 OCR
- 印章、手写、低质量扫描进一步雪上加霜

**4) 字体编码问题**
- 自定义字体子集化后字符映射错乱
- 中文 PDF 可能出现"乱码"或"⼿写"这种相似字
- ToUnicode CMap 缺失时无解

**5) 公式 / 图表 / 图片**
- 数学公式以图片或 LaTeX 形式存
- 图表数据丢失
- 流程图、组织架构图变成"文本块乱序"

### 2. 进阶问题

**6) 阅读顺序**
- 多栏、脚注、表格旁注混杂
- 简单顺序提取 ≠ 人类阅读顺序

**7) 元信息丢失**
- 标题、章节层级在 PDF 里没有显式标记
- 需要靠字号、字体推断

**8) 跨页问题**
- 表格、段落、章节跨页要拼接
- 页眉页脚污染内容

### 3. 主流解析工具对比
| 工具 | 优势 | 劣势 |
|------|------|------|
| **PyMuPDF (fitz)** | 快、提取细致 | 表格弱 |
| **pdfplumber** | 表格友好 | 慢 |
| **PDF.js** | 浏览器渲染 | Node 重 |
| **MinerU**（OpenDataLab） | **专为 LLM 设计、识别公式表格** | 较新 |
| **Unstructured** | 端到端结构化 | 黑盒 |
| **Marker** | Markdown 输出、效果好 | GPU 推荐 |
| **PaddleOCR / RapidOCR** | OCR 强 | 仅图像 |
| **GPT-4V / Qwen2.5-VL** | 多模态终极方案 | 贵 |

### 4. 工业流水线（推荐）
```
PDF
  → 检测是否扫描页（pixel ratio 判定）
  → 文本页 PyMuPDF/MinerU 提取
  → 扫描页 PaddleOCR / OCR 模型
  → 表格 pdfplumber / Camelot / Marker
  → 公式 LaTeX-OCR / VLM
  → 重建阅读顺序 + 段落归并
  → 切块 + embedding 入库
```

### 5. 多模态 PDF 解析新趋势
- 用 **VLM 直接看页面图像**生成 Markdown
- GPT-4V、Qwen2.5-VL、InternVL3 都能做
- 优点：保留版面、公式、图表
- 缺点：成本高、慢

### 6. 一句话总结
**PDF 解析 = 工程苦活**——文本 PDF 用 MinerU / Marker、扫描页 OCR、复杂版面上 VLM，没有一招通杀，必须按文档类型组合方案。

## 结合项目谈（口语版 · 约 30 秒）

> PDF 解析在我们跨境支付里**实打实痛**——RAG 知识库 70% 来源是 PDF，包括各国汇款政策、合规手册、KYB 证件、银行回单、对账单。
>
> 几个具体痛点 + 解：
>
> **痛点一：表格丢结构**。汇款手续费表是按"国家×币种×渠道"的二维表，pdfplumber 提取后变成一行一行的文本，丢了行列对应。我们后来用 **Marker** 直接转 Markdown，表格结构基本保留，准确率从 50% 涨到 85%。
>
> **痛点二：扫描页 KYB 证件**。商户上传的营业执照、税单经常是手机拍照存 PDF，画质差、有印章遮挡。我们用 **Qwen2.5-VL** 直接看图提结构化字段，比传统 OCR + 规则提取准确率高 12 个点，但单页成本 0.02 元，量大时心痛。
>
> **痛点三：阅读顺序**。监管文件经常多栏 + 脚注 + 表格混排，简单提取后顺序全乱。我们最后切到 **MinerU**（OpenDataLab 的专为 LLM 的解析），它自带版面分析能保留阅读顺序，质量比 pdfplumber 好很多。
>
> 现在我们的 PDF 流水线是 **MinerU 主线 + VLM 兜底**：MinerU 处理 90% 普通 PDF，VLM 处理复杂版面和扫描页。
>
> 一句话：**PDF 是 RAG 的最大数据敌人**，没有银弹只有组合拳。

- "MinerU vs Marker 在金融文档上的对比我们做过。"
- "VLM 直接看 PDF 的成本 / 质量 trade-off 可以聊。"
- "扫描页 KYB 识别我们怎么压成本可以展开。"
