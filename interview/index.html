先选主战场

三类 LLM 岗位地图

岗位名称经常混用，判断重点应看 JD 中的实际交付物。先选一个主方向深入，再补相邻能力。

ROLE 01工程落地

应用 / Agent 工程师

把模型接入真实业务，交付 RAG、Agent、工作流、智能客服或 Copilot 类产品。

核心能力Prompt、RAG、Agent、评测、后端工程、业务抽象
面试重点项目深挖、系统设计、bad case、效果与成本权衡
优先入口RAG / Agent 系统设计 →
ROLE 02算法研究

算法 / 训练工程师

围绕数据、预训练、SFT、偏好优化和模型评测，提升模型能力或垂域效果。

核心能力Transformer、训练范式、LoRA、RLHF / DPO、实验设计
面试重点原理推导、论文理解、训练调参、数据与评测闭环
优先入口LLM 原理与微调 →
ROLE 03性能基建

基建 / 推理工程师

负责模型服务、推理加速、资源调度和稳定性，让模型跑得更快、更稳、更省。

核心能力vLLM、KV Cache、量化、并行计算、GPU / K8s
面试重点吞吐与延迟、显存估算、压测、故障定位、容量规划
优先入口推理优化 / 部署题库 →

按背景开始

不要从第 1 页顺序背到最后

不同背景的短板不同。下面三条路径能把内容压缩成一条可执行路线。

CAMPUS · 应届 / 校招

先建立面试基本盘

重点证明基础、学习能力和项目潜力，算法与自我介绍不能留明显短板。

确定岗位方向与 30 天节奏 重写课程 / 实习 / 毕设经历 补 LLM 原理与算法题型 按分类完成第一轮真题
EXPERIENCED · 社招

把经验转成可验证价值

重点准备项目决策、业务结果、复杂故障和规模化经验，避免只罗列技术名词。

量化最近 3 年核心成果 准备系统设计与扩容方案 围绕目标 JD 定向刷题 计算 TC 并准备谈薪筹码
SWITCH · 转型 AI

保留原领域优势再补 AI

不要把过去经验清零。用原行业知识构造一个可演示、可评测、可追问的 LLM 项目。

选择最接近原能力的岗位 重组可迁移能力与项目故事 补齐 RAG / Agent 工程链路 用真题暴露知识盲区
最小行动：今天只做三件事：选定目标岗位、写出一个 1 分钟自我介绍、从对应分类随机刷 10 题。完成后再扩展计划。
