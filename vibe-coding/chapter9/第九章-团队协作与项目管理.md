```

│   ├─ 效率最大化                                            │
```

## 9.3 Cursor Rules 团队化

### 团队级规则结构

```
vibe-coding/
├── .cursor/

│   │   ├── api.md               # API 设计规范

│   │   └── security.md          # 安全规范
│   └── config.json              # 团队配置
```

### 全局规则

```markdown
# .cursor/rules/_global.md

## 团队代码规范

### 通用原则
- 代码优先运行，无警告无错误
- 遵循 KISS 原则（Keep It Simple, Stupid）
- SOLID 原则应用于面向对象设计
- DRY 原则（Don't Repeat Yourself）

### 命名规范
- 变量/函数: camelCase
- 类/类型/组件: PascalCase
- 常量: UPPER_SNAKE_CASE
- 文件: kebab-case

### 注释规范
- 函数/方法添加 JSDoc
- 复杂逻辑添加行内注释
- 业务决策添加 TODO 说明

### 错误处理
- 同步: try-catch
- 异步: .catch() 或 try-await
- 统一错误格式: { success, error, data }
```

### TypeScript 规则

```markdown
# .cursor/rules/typescript.md

## TypeScript 最佳实践

### 类型定义
- 禁止使用 any，使用 unknown 替代
- 优先使用接口而非类型别名
- 使用泛型而非函数重载
- 导出类型而非实现

### 类型推断
- 变量初始化时推断类型
- 函数返回值明确标注
- 复杂类型添加注释

### 类型检查
- strict 模式开启
- noImplicitAny: true
- strictNullChecks: true
```

### React 规则

```markdown
# .cursor/rules/react.md

## React 开发规范

### 组件规范
- 使用函数组件 + Hooks
- Props 使用 TypeScript interface
- 组件文件导出默认导出
- 组件放在 components/ 目录

### Hooks 规范
- 自定义 Hook 以 use 开头
- Hook 放在 hooks/ 目录
- 依赖数组必须完整

### 状态管理
- 本地状态: useState
- 共享状态: Context 或 Zustand
- 服务端状态: TanStack Query
```

### ADR (Architecture Decision Records)

```markdown
# docs/adr/003-use-claude-for-complex-refactoring.md

## ADR-003: 使用 Claude 进行复杂重构

### 状态
已通过 - 2024-03-15

### 背景
项目规模增长，代码重构需求频繁，传统方式效率低下。

### 决策
使用 Claude Code 作为复杂重构的主要工具。

### 理由
1. 200K 上下文可理解全项目
2. 推理能力强，适合复杂逻辑
3. 可控性强，每步可确认

### 后果
- 重构效率提升 50%
- 需要人工最终审查
- 团队需要培训 Claude 使用

### 使用规范
1. 重构前创建备份分支
2. 每次重构不超过 5 个文件
3. 重构后必须通过所有测试
4. Code Review 必须人工完成
```

## 9.7 CI/CD 集成

### AI 相关的 CI 流程

```yaml
# .github/workflows/ci.yml
name: CI

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

jobs:
  lint-and-test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Setup Node
        uses: actions/setup-node@v4
        with:
          node-version: '20'

      - name: Install dependencies
        run: npm ci

      - name: Lint
        run: npm run lint

      - name: Type check
        run: npm run type-check

      - name: Test
        run: npm run test

  ai-code-review:
    runs-on: ubuntu-latest
    needs: lint-and-test
    if: github.event_name == 'pull_request'
    steps:
      - uses: actions/checkout@v4

      - name: Claude Code Review
        uses: anthropic/claude-code-action@v1
        with:
          api-key: ${{ secrets.ANTHROPIC_API_KEY }}
          review-scope: |
            - Logic correctness
            - Security issues
            - Performance concerns
```

### GitHub Actions 市场集成

| Action | 用途 |
|--------|------|
| **anthropic/claude-code-action** | Claude Code 自动化 |
| **github/super-linter** | 代码质量检查 |
| ** actions/stale** | 自动关闭 stale issues |
| ** github/codeql-action** | 代码安全扫描 |
