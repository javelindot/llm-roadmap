# 第八章：个人开发最佳实践

## 8.1 Vibe Coding 工作流设计

### 标准工作流

```
┌─────────────────────────────────────────────────────────────┐
│                   AI 辅助开发工作流                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   ┌─────────┐                                              │
│   │ 需求    │  ← 明确要做什么                              │
│   │ 澄清    │                                              │
│   └────┬────┘                                              │
│        ↓                                                   │
│   ┌─────────┐                                              │
│   │ AI     │  ← 使用 AI 生成代码                          │
│   │ 生成   │                                              │
│   └────┬────┘                                              │
│        ↓                                                   │
│   ┌─────────┐                                              │
│   │ 人工    │  ← 审查 AI 生成的代码                        │
│   │ 审查   │                                              │
│   └────┬────┘                                              │
│        ↓                                                   │
│   ┌─────────┐                                              │
│   │ 迭代    │  ← 根据审查反馈优化                          │
│   │ 优化   │                                              │
│   └────┬────┘                                              │
│        ↓                                                   │
│   ┌─────────┐                                              │
│   │ 测试    │  ← 验证代码正确性                            │
│   │ 验证   │                                              │
│   └────┬────┘                                              │
│        ↓                                                   │
│   ┌─────────┐                                              │
│   │ 提交    │  ← 提交到版本控制                            │
│   │ 部署   │                                              │
│   └─────────┘                                              │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 各阶段要点

| 阶段 | 要点 | 常见问题 |
|------|------|----------|
| **需求澄清** | 明确输入、输出、边界 | 需求模糊 |
| **AI 生成** | 提供上下文、明确要求 | 上下文不足 |
| **人工审查** | 检查逻辑、安全、风格 | 过度信任 AI |
| **迭代优化** | 明确反馈、逐步调整 | 反馈不清 |
| **测试验证** | 覆盖边界、异常情况 | 测试不足 |
| **提交部署** | 小步提交、记录变更 | 提交粒度粗 |

## 8.2 Cursor 工作流

### 日常开发模式

```
┌─────────────────────────────────────────────────────────────┐
│                   Cursor 日常使用模式                         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   场景 1: 快速添加功能                                       │
│   ─────────────────────────                                │
│   ⌘K → "添加用户头像上传功能"                                │
│   → 预览代码 → Accept                                       │
│                                                             │
│   场景 2: 理解代码                                          │
│   ─────────────────                                        │
│   ⌘L → "解释这个函数的逻辑"                                  │
│   → 查看解释 → 有疑问继续追问                                 │
│                                                             │
│   场景 3: 修复 Bug                                          │
│   ────────────────                                         │
│   ⌘L → "这个函数返回 undefined 是为什么"                     │
│   → 查看分析 → 应用修复建议                                   │
│                                                             │
│   场景 4: 重构代码                                          │
│   ─────────────────                                        │
│   ⌘K → "把这个类重构为函数式组件"                             │
│   → 预览 → 逐文件确认                                       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 快捷键速查

| 任务 | 快捷键 | 效果 |
|------|--------|------|
| **生成代码** | ⌘K | 打开 Composer |
| **对话问答** | ⌘L | 打开 Chat |
| **接受建议** | ⌘Enter | 接受 AI 建议 |
| **全部接受** | ⌘⇧Enter | 接受所有建议 |
| **撤销** | ⌘Z | 撤销操作 |
| **多行补全** | Tab | 接受补全 |

### Cursor Rules 配置

```markdown
# .cursorrules - 我的个人配置

## 代码风格
- 2 空格缩进
- 单引号
- 分号结尾
- async/await 优先

## React 组件
- 函数组件
- TypeScript
- Tailwind CSS
- 组件文件单个导出

## 错误处理
- 始终 try-catch
- 用户友好的错误消息
- 错误日志记录

## 命名规范
- 组件: PascalCase
- 函数: camelCase
- 常量: UPPER_SNAKE_CASE
- 文件: kebab-case
```

### 常见 AI 生成代码问题

| 问题类型 | 表现 | 解决方法 |
|----------|------|----------|

## 8.6 版本控制策略

### Git 提交规范

```
┌─────────────────────────────────────────────────────────────┐
│                   Git 提交规范                               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   格式:                                                     │
│   <type>(<scope>): <subject>                               │
│                                                             │
│   Type:                                                     │
│   - feat: 新功能                                           │
│   - fix: Bug 修复                                          │
│   - refactor: 重构                                          │
│   - docs: 文档更新                                         │
│   - style: 代码格式                                         │
│   - test: 测试相关                                          │
│   - chore: 构建/工具                                       │
│                                                             │
│   示例:                                                     │
│   feat(auth): add user registration                        │
│   fix(api): handle null response                           │
│   refactor(components): simplify Button                     │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### AI 辅助 Git 操作

# Claude Code 或 Cursor ⌘L

## 8.7 测试策略

### AI 生成测试

```
┌─────────────────────────────────────────────────────────────┐
│                   测试生成策略                               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   1. 单元测试                                               │
│   ──────────                                               │
│   ⌘K → "为这个函数生成单元测试"                              │
│   → Jest/Vitest 测试用例                                   │
│   → 覆盖正常/边界/异常                                      │
│                                                             │
│   2. 集成测试                                               │
│   ──────────                                               │
│   ⌘L → "为这个 API 端点生成集成测试"                         │
│   → Supertest/Playwright 测试                              │
│   → 覆盖成功/失败/权限                                     │
│                                                             │
│   3. E2E 测试                                              │
│   ──────────                                               │
│   ⌘L → "为这个用户流程生成 E2E 测试"                         │
│   → Playwright/Cypress 场景                                │
│   → 覆盖关键路径                                           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 测试覆盖要点

| 测试类型 | 覆盖要点 | AI 生成难度 |
|----------|----------|-------------|
| **正常路径** | 典型输入、预期输出 | ⭐ 简单 |
| **边界条件** | 空值、零、最大值 | ⭐⭐ 中等 |
| **异常处理** | 错误输入、超时、崩溃 | ⭐⭐⭐ 较难 |
| **安全测试** | 注入、越权、伪造 | ⭐⭐⭐⭐ 难 |
| **性能测试** | 负载、并发 | ⭐⭐⭐⭐⭐ 最难 |
