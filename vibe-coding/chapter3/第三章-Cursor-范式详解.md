# 第三章：Cursor 范式详解

## 3.1 Cursor 定位与特点

### 为什么 Cursor 是"最像 IDE 的 AI 工具"

Cursor 是由前苹果工程师打造的 AI 代码编辑器，基于 VS Code 深度定制。与传统 IDE 插件不同，Cursor 从底层将 AI 能力与编辑器深度融合。

```
┌─────────────────────────────────────────────────────────────┐
│                        Cursor 核心定位                       │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   传统 IDE:                                                │
│   ┌─────────────┐                                          │
│   │   Editor    │  ← 代码编辑是核心                          │
│   └─────────────┘                                          │
│         ↓                                                   │
│   ┌─────────────┐                                          │
│   │ AI Plugin   │  ← AI 是附加组件                          │
│   └─────────────┘                                          │
│                                                             │
│   Cursor:                                                   │
│   ┌─────────────┐                                          │
│   │     AI      │  ← AI 是核心                              │
│   └─────────────┘                                          │
│         ↓                                                   │
│   ┌─────────────┐                                          │
│   │   Editor    │  ← 编辑器是 AI 的界面                     │
│   └─────────────┘                                          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 核心优势

| 优势 | 说明 |
|------|------|
| **Composer 模式** | 一次性生成和修改多个文件 |
| **Agent 模式** | 自主执行多步骤任务 |
| **规则配置** | Cursor Rules 自定义项目规则 |
| **数据库感知** | Schema 理解、智能 SQL 生成 |
| **跨文件理解** | 项目级代码分析和修改 |
| **精美界面** | 现代化的用户界面和交互 |

## 3.3 Composer 模式（⌘K）

### 核心功能

Composer 模式是 Cursor 最强大的功能，允许你一次性生成或修改多个文件。

#### 使用方式

1. **⌘K** 打开 Composer 输入框
2. 描述你想要实现的功能
3. AI 生成多个相关文件
4. 逐个预览和确认

#### 实战示例

```
场景: 创建用户认证系统

用户输入:
"帮我创建一个用户认证系统，包括:
- 用户注册和登录
- JWT token 验证
- 密码加密存储
- 中间件保护路由"

AI 输出:
┌─────────────────────────────────────────┐
│  📄 models/User.ts          [新建]       │
│  📄 routes/auth.ts           [新建]       │
│  📄 middleware/auth.ts       [新建]       │
│  📄 utils/jwt.ts            [新建]       │
│  📄 utils/password.ts       [新建]       │
│  📄 tests/auth.test.ts      [新建]       │
└─────────────────────────────────────────┘

点击任一文件可预览，Accept All 接受全部
```

### Composer 进阶技巧

| 技巧 | 说明 |
|------|------|
| **@文件夹引用** | @components 引用整个文件夹 |
| **@文件引用** | 引用具体文件提供上下文 |
| **@文档引用** | 引用项目文档 |
| **@Git** | 引用 Git 历史 |
| **@代码** | 引用特定代码片段 |
| **多轮迭代** | 分步骤引导 AI 生成 |

## 3.4 Chat 模式（⌘L）

### 核心功能

Chat 模式提供对话式的 AI 交互，适合解释代码、答疑解惑。

#### 使用方式

1. **⌘L** 打开 Chat 面板
2. 提问或描述问题
3. AI 在侧边栏回答
4. 可以直接应用建议到代码

#### 界面布局

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   ┌─────────────────────┬─────────────────────────────┐   │

│   │    代码编辑器        │      Chat 面板               │   │
│   │                     │                             │   │

│   │  │               │  │                             │   │

│   │  │   (选中代码)   │  │                             │   │
│   │  │               │  │  [Apply] [Copy] [Regenerate] │   │
```

## 3.7 Cursor Rules 配置

### 什么是 Cursor Rules

Cursor Rules 允许你为项目定义自定义规则，AI 会遵循这些规则生成代码。

### 创建方式

1. 创建 `.cursorrules` 文件（项目根目录）
2. 创建 `.cursor/rules/` 目录下的 `.md` 文件

### 规则示例

```markdown
# .cursorrules

## 代码风格
- 使用 TypeScript strict 模式
- 使用 2 空格缩进
- 使用单引号
- 使用 async/await 而非 Promise

## 组件规范
- 组件使用 PascalCase
- Props 使用 camelCase
- 每个组件单独一个文件
- 组件文件放在 components 目录

## 命名规范
- 变量使用 camelCase
- 常量使用 UPPER_SNAKE_CASE
- 类型使用 PascalCase
- 文件名使用 kebab-case

## 安全规范
- 永远不要硬编码密钥
- 所有 API 调用使用环境变量
- 输入必须验证和清理
```

### 进阶配置

```markdown
# .cursor/rules/typescript.md

## TypeScript 最佳实践

### 类型定义
- 避免使用 `any`，优先使用 `unknown`
- 使用接口而非类型别名（除非需要联合类型）
- 使用泛型而非重载

### 导入
- 使用绝对导入（配置了 paths）
- 按以下顺序分组：
  1. React 和框架
  2. 第三方库
  3. 内部模块
  4. 类型定义
```

## 3.9 数据库与 API 集成

### 数据库 Schema 感知

Cursor 可以连接数据库，理解 Schema 并生成 SQL：

```typescript
// 连接到 PostgreSQL
// Cursor 自动读取 schema:

// Schema:
// users: id, email, name, created_at
// posts: id, user_id, title, content, published_at
// comments: id, post_id, user_id, body, created_at

// 用户输入:
"帮我写一个获取用户及其最近5篇博客及其评论的查询"

// AI 生成:
const query = `
  SELECT
    u.name as user_name,
    p.title as post_title,
    p.created_at as post_date,
    c.body as comment_body,
    c.created_at as comment_date
  FROM users u
  LEFT JOIN posts p ON p.user_id = u.id
  LEFT JOIN comments c ON c.post_id = p.id
  WHERE u.id = $1
  ORDER BY p.created_at DESC, c.created_at DESC
  LIMIT 25;
`;
```

### API 集成

| 功能 | 说明 |

| **请求生成** | 根据定义生成请求代码 |
| **类型推断** | 从响应推断类型 |

## 3.11 小结

本章深入介绍了 Cursor 范式：

### 关键要点

1. **三大核心模式**：Composer(⌘K)、Chat(⌘L)、Agent(⌘I)
2. **Composer 最强大**：一次性处理多文件生成
3. **Cursor Rules**：项目级自定义规则
4. **智能模型切换**：根据任务选择最优模型
5. **数据库感知**：Schema 理解与 SQL 生成

### 实战技巧

1. **日常开发**：⌘K 生成 + ⌘L 解释
2. **新项目**：⌘I Agent 初始化
3. **复杂重构**：⌘K 分步骤执行
4. **学习探索**：⌘L 对话问答
