## 2.3 交互模式对比

### 三种交互模式

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   Mode 1: Tab Completion (补全模式)                         │
│   ─────────────────────────────────────                     │
│   你输入代码 ↓                                              │
│   AI 预测并显示建议                                         │
│   按 Tab 接受建议                                           │
│                                                             │
│   示例: GitHub Copilot                                      │
│   ┌─────────────────────────────────────┐                   │
│   │ function calculateTotal(items) {  │                   │
│   │   // AI 补全:                      │                   │
│   │   return items.reduce((sum, item) │                   │
│   │     => sum + item.price, 0);       │                   │
│   │ }                    [Tab 接受]    │                   │
│   └─────────────────────────────────────┘                   │
│                                                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   Mode 2: Inline/Chat (对话模式)                            │
│   ─────────────────────────────────────                     │
│   你描述需求 ↓                                              │
│   AI 在当前文件/选区生成代码                                 │
│   确认后插入                                               │
│                                                             │
│   示例: Copilot Chat, Claude for VS Code                   │
│   ┌─────────────────────────────────────┐                   │
│   │ // AI: 为这个函数添加输入验证        │                   │
│   │ function validateEmail(email) {     │                   │
│   │   const regex = /^[^\s@]+@[^\s@]+  │                   │
│   │     \.[^\s@]+$/;                    │                   │
│   │   return regex.test(email);         │                   │
│   │ }                                   │                   │
│   └─────────────────────────────────────┘                   │
│                                                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   Mode 3: Agent (代理模式)                                  │
│   ─────────────────────────────────                         │
│   你描述目标 ↓                                              │
│   AI 自主规划步骤 ↓                                         │
│   执行文件操作、命令、代码生成                                │
│   返回结果并解释                                           │
│                                                             │
│   示例: Cursor Composer, Claude Code                        │
│   ┌─────────────────────────────────────┐                   │
│   │ > 帮我创建一个用户认证系统            │                   │
│   │                                     │                   │
│   │ Thinking:                            │                   │
│   │ 1. 创建用户模型 User.js              │                   │
│   │ 2. 创建认证路由 auth.js              │                   │
│   │ 3. 添加 JWT 验证中间件               │                   │
│   │ 4. 编写测试用例                      │                   │
│   │                                     │                   │
│   │ ✅ 已完成所有文件创建                 │                   │
│   └─────────────────────────────────────┘                   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 交互模式选择指南

| 场景 | 推荐模式 | 原因 |
|------|----------|------|
| 快速补全简单代码 | Tab 补全 | 最快、最直接 |
| 添加功能、修改代码 | Inline/Chat | 上下文清晰 |
| 新项目、从零构建 | Agent | 自主规划执行 |
| 复杂重构、跨文件修改 | Agent + 人工监督 | 自动化 + 审核 |

## 2.9 小结

本章全面对比了主流 AI 编程工具：

### 关键要点

1. **工具生态丰富**：从 IDE 插件到 CLI 工具，从个人到企业，选择多样
2. **交互模式演进**：从 Tab 补全到 Agent 模式，能力逐步增强
3. **上下文是关键**：上下文窗口大小直接影响 AI 理解能力
4. **价格差异大**：从免费到 $100+/月，需根据需求选择
5. **生态决定体验**：IDE 集成、外部工具支持影响日常使用

1. 为什么 Claude 的上下文能力是目前最强的？
2. Tab 补全和 Agent 模式各适合什么场景？
