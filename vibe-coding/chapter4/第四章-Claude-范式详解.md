```
└─────────────────────────────────────────────────────────────┘
```

### Constitutional AI

Claude 采用 Constitutional AI 技术，确保 AI 行为符合人类价值观：

```python
# Constitutional AI 简化示意

class Claude:
    def generate_response(self, prompt):
        # 1. 生成初始响应
        response = self.model.generate(prompt)

        # 2. 根据原则自我审查
        principles = [
            "helpful, not harmful",
            "honest, not deceptive",
            "safe, not dangerous"
        ]

        for principle in principles:
            if not self.check_alignment(response, principle):
                response = self.revise(response, principle)

        return response
```

## 4.6 Claude Skills

### 什么是 Skills

Skills 是 Claude 的任务级能力封装，将复杂任务模板化。

### 执行流程

```
┌─────────────────────────────────────────────────────────────┐
│                    Skills 执行流程                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   1. Discovery (发现)                                       │
│      └─> 扫描 skills 目录，提取可用技能                      │
│                                                             │
│   2. Activation (激活)                                      │
│      └─> 检测到技能触发词，加载对应 SKILL.md                  │
│                                                             │
│   3. Execution (执行)                                       │
│      └─> 依据技能指导完成用户请求                            │
│                                                             │
│   4. Feedback (反馈)                                        │
│      └─> 返回执行结果和后续建议                              │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### Skills 文件结构

```markdown
# skills/commit-writer/SKILL.md

## 名称
commit-writer

## 触发词
- "写 commit"
- "帮我提交"
- "commit changes"

## 描述
根据 Git diff 自动生成规范的 commit message

## 执行步骤
1. 读取 git diff --staged
2. 分析变更类型和内容
3. 生成符合 Conventional Commits 规范的 message
4. 显示给用户确认

## 输出格式
<type>(<scope>): <description>

[optional body]

[optional footer]
```
