### 风险类型

| 风险类型 | 描述 | 影响 | 缓解措施 |
|----------|------|------|----------|
| **数据泄露** | 代码被用于训练或泄露 | 高 | 本地部署、VPN |
| **Prompt 注入** | 恶意指令注入 | 中 | 输入过滤、规则 |
| **API 滥用** | 过度调用或未授权使用 | 中 | 配额、监控 |
| **密钥泄露** | API 密钥暴露 | 高 | 密钥管理服务 |
| **合规违规** | 违反数据法规 | 高 | 法律审查 |

### 代理服务配置

```yaml
# api-proxy/config.yaml
server:
  host: 0.0.0.0
  port: 8080
  ssl:
    enabled: true
    cert: /path/to/cert.pem
    key: /path/to/key.pem

upstream:
  anthropic:
    base_url: https://api.anthropic.com
    timeout: 120s

auth:
  enabled: true
  api_key_header: X-API-Key
  valid_keys:
    - team-key-1
    - team-key-2

rate_limit:
  enabled: true
  requests_per_minute: 100
  burst: 20

audit:
  enabled: true
  log_path: /var/log/api-proxy/audit.log
  include_request_body: false
  include_response_body: false
```

### Nginx 反向代理

```nginx
# /etc/nginx/conf.d/anthropic-proxy.conf

server {
    listen 443 ssl;
    server_name api-proxy.company.com;

    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;

    # 认证
    auth_basic "Anthropic API Proxy";
    auth_basic_user_file /etc/nginx/.htpasswd;

    # 限流
    limit_req_zone $binary_remote_addr zone=api:10m rate=10r/s;
    limit_req zone=api burst=20;

    location /v1/ {
        proxy_pass https://api.anthropic.com/v1/;
        proxy_set_header Host api.anthropic.com;
        proxy_set_header Authorization $http_authorization;
        proxy_set_header X-API-Key $http_x_api_key;

        proxy_read_timeout 120s;
        proxy_connect_timeout 10s;

        # 审计日志
        access_log /var/log/nginx/anthropic-access.log;
    }
}
```

### MCP 服务器开发示例

```python
# internal_code_search/server.py
from mcp.server import Server
from mcp.types import Tool, Resource
from starlette.applications import Starlette
import httpx

server = Server("internal-code-search")

@server.list_tools()
async def list_tools() -> list[Tool]:
    return [
        Tool(
            name="search_code",
            description="Search internal codebase",
            inputSchema={
                "type": "object",
                "properties": {
                    "query": {"type": "string"},
                    "repo": {"type": "string"},
                    "limit": {"type": "number", "default": 10}
                }
            }
        )
    ]

@server.call_tool()
async def call_tool(name: str, arguments: dict) -> str:
    if name == "search_code":
        # 调用内部代码搜索 API
        async with httpx.AsyncClient() as client:
            response = await client.post(
                "https://internal-search.company.com/api/search",
                json={
                    "query": arguments["query"],
                    "repo": arguments.get("repo"),
                    "limit": arguments.get("limit", 10)
                },
                headers={"Authorization": f"Bearer {get_internal_token()}"}
            )
            return response.json()
```

## 10.9 小结

### 核心要点

1. **安全第一**：代码隐私、数据保护是首要考虑
2. **本地部署**：完全控制，适合高安全要求场景
3. **API 代理**：认证、限流、审计是企业必备
4. **合规**：SOC2、GDPR 等合规要求不可忽视
5. **成本**：预算控制和优化策略必须到位
